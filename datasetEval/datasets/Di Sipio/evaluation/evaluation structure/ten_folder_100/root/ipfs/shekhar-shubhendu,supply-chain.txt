b"# Supply Chain Management\n\t\n \t\n ------------------------\t\t\n\nSupply chain management on blockchain using Angular 4 + Truffle + IPFS + Ethereum\n\n\n## How to use\nThere are 2 small parts to successfully running this project.\n\n### Prerequisite\n\n1. Node.js\n2. IPFS\n3. Truffle Framework (`npm install -g truffle` or on linux `sudo npm install -g truffle`)\n4. Testrpc (`npm install -g ethereumjs-testrpc` or on linux `sudo npm install -g ethereumjs-testrpc`)\n\n### Part 1\n\n1. `git clone https://github.com/shekhar-shubhendu/supply-chain.git`\n2. `cd supply-chain`\n3. `npm install`\n4. `npm install -g @angular/cli@latest`\n(linux users might have to do: `sudo npm install -g @angular/cli@latest`)\n\n### Part 2\n\n4. Open a new terminal and start IPFS daemon with `ipfs daemon`\n6. Start testrpc in new terminal with `testrpc -l 47000000000000`.\n7. From inside the project directory run `truffle compile` to compile your contracts\n8. And `truffle migrate` to deploy those contracts to the network\n9. Now finally, start the project with `npm start`. Navigate to `http://localhost:4200/`. The app will automatically reload if you change any of the source files.\n10. Make sure there are no errors in browser console\n\n### Login\n\n1. This project uses your keystore file to login to the application.\n2. For now, the accounts that can login are hardcoded in `auth.service.ts` with there role. Future builds will feature a screen for proper role and account management.\n3. You can find the keystore for default accounts and there password in 'keys' folder.\n\n### URLs\n\n1. `/login`\n2. `/retailer`\n3. `/distributor`\n4. `/manufacturer`\n5. `/supplier`\n\n### Todo:\n\n1.
