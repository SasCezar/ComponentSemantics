b"# [ipfs-wiki-system] build your wiki system base on ipfs and markdown\n\n[![jaywcjlove/sb](https://jaywcjlove.github.io/sb/lang/chinese.svg)](README-zh.md)\n\n\xef\xa3\xbf The goal of the project is to build a wiki system on ipfs. The wiki data is **decentration** and **not modifiable**.\n\nIf anyone interest this project, they can direct clone this repo to yourself.\n\nWelcome to commit any question or issue to me.\n\nFeel free to **star** and **fork**.\n\n## Explanation\n\n[English](README.md) | [\xe4\xb8\xad\xe6\x96\x87](README-zh.md)\n\n## How to use\n\n### Create Workplace\n\n```\nmkdir workplace\ncd workplace\ngit clone git@github.com:daijiale/ipfs-wiki-system.git\n```\n\n### Build Wiki System\n\nExec these command:\n\n```\ncd workplace/ipfs-wiki-system/wiki-release\nll\n```\n\nThe output:\n\n- **wiki-release**\n\n  - **index.html** //markdown\xe6\xa8\xa1\xe6\x9d\xbf\xe6\xb8\xb2\xe6\x9f\x93\n  - **navigation.md** //\xe5\xaf\xbc\xe8\x88\xaamarkdown\n  - **index.md** //\xe9\xa6\x96\xe9\xa1\xb5markdown\n\nEveryone can refer this repo to make yourself wiki content by `markdown` format.\n\n### Mount Location Node\n\n![](http://daijiale-cn.oss-cn-hangzhou.aliyuncs.com/djl-blog-pic/ipfs/publish-to-block.png)\n\nRemember the hash of the files root directory: QmV5ZVQxXURKPDcVDW8WjpLCiQYvNzg173XcB6rYFevoXm\n\n### Public To Master Net\n\nJust execute `ipfs daemon`, the wiki file would to send to the ipfs net.\n\nThe example is [this](https://ipfs.io/ipfs/QmV5ZVQxXURKPDcVDW8WjpLCiQYvNzg173XcB6rYFevoXm/#!index.md)\n\n### Public To IPNS\n\nSince one hash corresponding one not modifable content, if update a website, the website hash would be changed, and other old links can't find this new website.\n\nFor resolve this problem, `ipfs` support a basic component names `IPNS`.\n\nipfs allow user to create an additional message with a file hash and a private key, and use a pubkeyhash to indicate your lastest website version.\n\nThe operation command is blow:\n\n![](http://daijiale-cn.oss-cn-hangzhou.aliyuncs.com/djl-blog-pic/ipfs/ipns.png)\n\nFinish it now, we have a fixed link to bundle a website: `QmPS5NRXPCeAUtofKbW7c58Qm4PpM8mPEVJvaooE13LF78`\n\n### Bundling Validation\n\n```\nipfs name resolve QmPS5NRXPCeAUtofKbW7c58Qm4PpM8mPEVJvaooE13LF78\n\n/ipfs/QmV5ZVQxXURKPDcVDW8WjpLCiQYvNzg173XcB6rYFevoXm\n```\n\n**Use IPNS To Access Fixed Node Hash:**\n\n<https://ipfs.io/ipns/QmPS5NRXPCeAUtofKbW7c58Qm4PpM8mPEVJvaooE13LF78>\n\nWhen verification successful, we will see this result:\n\n![](http://daijiale-cn.oss-cn-hangzhou.aliyuncs.com/djl-blog-pic/ipfs/ipfs-wiki-demo.png)\n\n### Decentration Validation\n\nNow we assume the node that recived our data is first node. Then we create another node in location to simulate the second node.\n\nOpen web console:\n\n![](http://daijiale-cn.oss-cn-hangzhou.aliyuncs.com/djl-blog-pic/ipfs/ipfs-web-consolve.png)\n\nIn the second node, we also can use `IPFS HASH` to find the `ipfs-wiki-system` content of first node.\n\nAnd we can see our data has spreaded to 275 nodes in the ipfs console. In other words, there are 274 nodes have recorded our data in there own 'book'. So we don't warry about any disaster. If one node is well, our data would not lost.\n\n![](http://daijiale-cn.oss-cn-hangzhou.aliyuncs.com/djl-blog-pic/ipfs/ipfs-peer-id.png)\n\n### Project Demo\n\n![](http://daijiale-cn.oss-cn-hangzhou.aliyuncs.com/djl-blog-pic/ipfs/ipfs-wiki-demo.png)\n\n[-> demo](https://ipfs.io/ipfs/QmV5ZVQxXURKPDcVDW8WjpLCiQYvNzg173XcB6rYFevoXm/#!index.md)\n\n## Reference\n\n- [How To Build A BlockChain Service On IPFS - YuanChunLi](http://liyuechun.org/2017/09/18/ipfs-blockchain/)\n- [Youtube Lab](https://www.youtube.com/watch?v=iUVLuXjPAfg)\n- [protocol lab](https://protocol.ai/)\n- [Filecoin Miner Early Plan](https://filecoin.io/)\n- [IPFS Development Doc](https://ipfs.io/docs/)\n- [IPFS----Replace HTTP Protocol?](http://www.jianshu.com/p/ddccae89a49a)\n- [IPFS--Killer To HTTP](http://mp.weixin.qq.com/s/0nHPzty51knpYOZZaBIA8g)\n"
