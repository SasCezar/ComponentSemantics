b'# [IPFS](http://ipfs.io) API for kotlin to access a IPFS node via http\n\n[![Release](https://jitpack.io/v/ligi/ipfs-api-kotlin.svg)](https://jitpack.io/#ligi/ipfs-api-kotlin)\n\n# Get it\n\nYou can get the artifact via jitpack - here you see how to use with gradle - for other build systems have a look [here](https://jitpack.io/#ligi/ipfs-api-kotlin)\n\n``` groovy\nrepositories {\n  maven { url "https://jitpack.io" }\n}\n```\n\n``` groovy\ndependencies {\n  compile \'com.github.ligi:ipfs-api-kotlin:$current_version\'\n}\n```\n\n# Use it with kotlin\n\n### add a string\n```kotlin\nval multihash = LocalIPFS().add.string("test-string").Hash\n```\n\n### get a string\n```kotlin\nval content = LocalIPFS().get.cat(multihash)\n```\n\n### get commit from version\n```kotlin\nval commit = LocalIPFS().info.version().Commit\n```\n\nNote: when you are not running a local IPFS daemon you can also use infura just replace LocalIPFS() with InfuraIPFS() or write your own configuration.\n\n# Apps that use this library\n\n* [IPFSDroid](http://github.com/ligi/IPFSDroid)\n* [Kontinuum](http://github.com/ligi/kontinuum)\n* [RHazOS Management console](https://github.com/RHazDev/IPFS-Android)\n\nPlease send a message or even PR if you write/find an app that also uses this library.\n\n# Dependencies \n\nWe depend on [okhttp](http://square.github.io/okhttp) and [moshi](https://github.com/square/moshi) which play well together with [okio](https://github.com/square/okio)\n\n# License \n\nMIT\n'
