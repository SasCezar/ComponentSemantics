[\xe2\x9c\x94\xef\xb8\x8f] Easy to use\n* [\xe2\x9c\x94\xef\xb8\x8f] Login\n* [\xe2\x9c\x94\xef\xb8\x8f] 2FA (bad location)\n* [\xe2\x9c\x94\xef\xb8\x8f] 2FA (sms pin enabled)\n* [\xe2\x9c\x94\xef\xb8\x8f] Multi-Session\n* [\xe2\x9c\x94\xef\xb8\x8f] Multi-Platform (Windows 10, Mac OSX, Linux and Raspberry PI)\n* [\xe2\x9c\x94\xef\xb8\x8f] Error management feature (bad pin, bad password)\n* [\xe2\x9c\x94\xef\xb8\x8f] Screenshot and Verbose logger\n* [\xe2\x9c\x94\xef\xb8\x8f] Like Mode Classic: bot selects a random hashtag from a config list and likes 1 random photo, and can repeat this all time.\n* [\xe2\x9c\x94\xef\xb8\x8f] Like Mode Realistic: bot selects a random hashtag from a config list and likes fast 10-12 photos, it sleeps 15-20min and repeats this all time. Sleeps at night.\n* [\xe2\x9c\x94\xef\xb8\x8f] Like Mode Competitor Users: it selects an account, selects random followers, likes 10-12 photo and sleeps 15-20min. Sleeps at night.\n* [\xe2\x9c\x94\xef\xb8\x8f] Like Mode Super like: it selects random hashtag from a config list and likes 3 random photos of the same user.\n* [\xe2\x9c\x94\xef\xb8\x8f] Comment Mode Classic: it selects random comments and random hashtags and writes comments under photos.\n* [\xe2\x9c\x94\xef\xb8\x8f] Follow/Defollow Classic: follow 30 users, defollow first and rotate (in loop). This method is not detected from socialblade. 300 follow-defollow/day.\n\n## \xf0\x9f\x91\xa8\xe2\x80\x8d\xf0\x9f\x92\xbb Desktop setup (GUI)\n1. Run `npm install @social-manager-tools/igbot@latest --save` (Available: @latest, @beta and @nightly)\n2. Get `config.js` file from `/node_modules/@social-manager-tools/igbot/configs/` folder.\n3. Remove `.tpl` suffix from `config.js.tpl` file in `configs` folder and fill it properly.\n4. On your code require library, config and run bot, example:\n```javascript\nconst config = require ("./config");\nconst Bot = require("@social-manager-tools/igbot");\nlet bot = new Bot(config);\n\n(async () => {\n\tawait bot.start();\n\n\tlet api = await bot.api();\n\n\tlet response = await api.login.flow();\n\n\tif (response.status) {\n\t\tresponse = await api.twofa.flow();\n\t}\n\n\tif (response.status) {\n\t\tresponse = await api.mode.flow();\n\t}\n\n\tawait bot.stop();\n})();\n```\n5. If it works add a star \xf0\x9f\x8c\x9f at this project \xe2\x9d\xa4\xef\xb8\x8f\n6. If you want to help me: **donate on [paypal](http://paypal.ptkdev.io)/[ko-fi](http://coffee.ptkdev.io)** or become a **[backer on patreon](http://patreon.ptkdev.io)**.\n\n## \xf0\x9f\x90\xb3 Docker setup\nIf you prefer to run this using Docker, an official container is available from the [Docker Hub](https://hub.docker.com/u/socialmanagertools).\n\nIn order to run it, copy the `config.js.tpl` file (template of available in `configs` folder), configure it as you prefer, then use it through volume mapping,\nlike in this example:\n\n```sh\ndocker run --restart=always --name=socialmanagertools-igbot -d -v /path/to/configs/config.js:/app/configs/config.js socialmanagertools/igbot:amd64\n```\n\n**AVAILABLE TAGS:** `amd64` (64bit), `i386` (32bit),`armv7` (Raspberry PI 2 and Raspberry PI 3), `armv8` (Raspberry PI ARMv8). All point to the master github repository (stable version) but is available `-nightly` suffix, example: `armv7-nightly`.\n\n**WARNING:** with docker is mandatory edit `config.js` and set `chrome_headless` on `enabled` (or to true if you use `v0.9.X` version) and set `chrome_executable_path` to `/usr/bin/chromium-browser`. Without this manual fix docker don\'t work.\n\n## \xf0\x9f\x93\x9a Documentation\n1. Run `npm run docs-init`\n2.
