It\'s tiny in size with an [incredible performance](https://github.com/jarun/nnn/wiki/Performance).\n\n`nnn` is also a disk usage analyzer, a fuzzy app launcher, a batch renamer and a file picker. The [plugin repository](https://github.com/jarun/nnn/tree/master/plugins#nnn-plugins) has tons of plugins and documentation to extend the capabilities further. You can _plug_ new functionality _and play_ with a custom keybind instantly. There\'s an independent [(neo)vim plugin](https://github.com/mcchrish/nnn.vim).\n\nIt runs smoothly on the Raspberry Pi, Termux [on Android](https://www.youtube.com/watch?v=AbaauM7gUJw), Linux, macOS, BSD, Cygwin and Linux subsystem for Windows. `nnn` works seamlessly with DEs and GUI utilities. It\'s nearly zero-config (with sensible defaults) and can be setup in less than 5 minutes.\n\nAdd to that an awesome [Wiki](https://github.com/jarun/nnn/wiki)!\n\n## Features\n\n- Modes\n  - Light (default), detail\n  - Disk usage analyzer (block/apparent)\n  - File picker, (neo)vim plugin\n- Navigation\n  - *Navigate-as-you-type* with dir auto-select\n  - Contexts (_aka_ tabs/workspaces) with custom colors\n  - Sessions, bookmarks; pin and visit a dir\n  - Remote mounts (needs sshfs, rclone)\n  - Familiar shortcuts (arrows, <kbd>~</kbd>, <kbd>-</kbd>, <kbd>@</kbd>), quick reference\n  - CD on quit (*easy* shell integration)\n- Sorting\n  - Ordered pure numeric names by default (visit _/proc_)\n  - Case-insensitive version (_aka_ natural) sort\n  - By file name, modification/access time, size, extension\n- Search\n  - Instant filtering with *search-as-you-type*\n  - Regex and substring match\n  - Subtree search to open or edit files (using plugin)\n- Mimes\n  - Open with desktop opener or specify a custom app\n  - Create, list, extract, mount (FUSE based) archives\n  - Option to open all text files in EDITOR\n- Information\n  - Detailed file information\n  - Media information (using plugin)\n- Convenience\n  - Run plugins and commands with custom keybinds\n  - FreeDesktop compliant trash (needs trash-cli)\n  - Cross-dir file/all/range selection\n  - Batch rename selection or dir entries\n  - Copy (as), move (as), delete, archive, link selection\n  - Create (with parents), rename, duplicate (anywhere) files and dirs\n  - Spawn a shell, run apps, run commands, execute file\n  - Hovered file set as `$nnn` at prompt and spawned shell\n  - Lock terminal (needs a locker)\n- Minimal deps, minimal config\n- Widely available\n- Unicode support\n- Follows Linux kernel coding style\n- Highly optimized, static analysis integrated code\n\n## Quickstart\n\n1. Install the [utilities you may need](https://github.com/jarun/nnn#utility-dependencies) based on your regular workflows.\n2. Configure [cd on quit](https://github.com/jarun/nnn/wiki/Basic-use-cases#configure-cd-on-quit).\n3. Optionally open all text files in `$EDITOR` (fallback vi): `export NNN_USE_EDITOR=1`.\n4. For additional functionality [install plugins](https://github.com/jarun/nnn/tree/master/plugins#installing-plugins) and the GUI app launcher [`nlaunch`](https://github.com/jarun/nnn/tree/master/misc/nlaunch).\n\nNotes:\n\n1. Don\'t memorize keys. Arrows, <kbd>/</kbd> and <kbd>q</kbd> suffice. Press <kbd>?</kbd> for help on keyboard shortcuts anytime.\n2. To set `nnn` as the default file manager, follow these [instructions](https://github.com/jarun/nnn/wiki/Advanced-use-cases#configure-as-default-fm).\n\n## Installation\n\n#### Library dependencies\n\nA curses library with wide char support (e.g. ncursesw), libreadline (`make O_NORL=1` to drop) and standard libc.\n\n#### Utility dependencies\n\n| Dependency | Installation | Operation |\n| --- | --- | --- |\n| xdg-open (Linux), open(1) (macOS), cygstart (Cygwin) | base | desktop opener |\n| file, coreutils (cp, mv, rm), xargs | base | file type, copy, move and remove |\n| tar, (un)zip [atool/bsdtar for more formats] | base | create, list, extract tar, gzip, bzip2, zip |\n| archivemount, fusermount(3) | optional | mount, unmount archives |\n| sshfs, [rclone](https://rclone.org/), fusermount(3) | optional | mount, unmount remotes |\n| trash-cli | optional | trash files (default action: rm) |\n| vlock (Linux), bashlock (macOS), lock(1) (BSD) | optional | terminal locker (fallback: [cmatrix](https://github.com/abishekvashok/cmatrix)) |\n| advcpmv (Linux) ([integration](https://github.com/jarun/nnn/wiki/Advanced-use-cases#show-cp-mv-progress)) | optional | copy, move progress |\n| `$VISUAL` (else `$EDITOR`), `$PAGER`, `$SHELL` | optional | fallback vi, less, sh |\n\n#### From a package manager\n\nInstall `nnn` from your package manager. On Ubuntu 18.04:\n\n    $ sudo apt-get install pkg-config libncursesw5-dev libreadline-dev\n    $ make\n    $ sudo make strip install\n\n`PREFIX` is supported, in case you want to install to a different location.\n\nVisit the [developer guides](https://github.com/jarun/nnn/wiki/Developer-guides) for compilation notes on the Pi, Cygwin and other compilation modes.\n\n#### Shell completion\n\nCompletion scripts for Bash, Fish and Zsh are [available](misc/auto-completion). Refer to your shell\'s manual for installation instructions.\n\n## Usage\n\n#### Configuration\n\nThere is no config file. Associated files are stored under `${XDG_CONFIG_HOME:-$HOME/.config}/nnn/`.\n\n`nnn` supports the following optional (set if you need) environment variables.\n\n| Example `export` | Description |\n| --- | --- |\n| `NNN_OPENER=mimeopen` | custom file opener |\n| `NNN_BMS=\'d:~/Documents;D:~/Docs archive/\'` | key-bookmark pairs [max 10] |\n| `NNN_PLUG=\'m:nmount;t:imgthumb;x:_chmod +x $nnn\'` | key-plugin (or cmd) pairs (<kbd>:key</kbd> to run) [max 15] |\n| `NNN_USE_EDITOR=1` | open text files in `$VISUAL` (else `$EDITOR`, fallback vi) |\n| `NNN_CONTEXT_COLORS=\'1234\'` | specify per context color [default: \'4444\' (all blue)] |\n| `NNN_SSHFS_OPTS=\'sshfs -o reconnect,idmap=user\'` | specify SSHFS options |\n| `NNN_RCLONE_OPTS=\'rclone mount --read-only\'` | specify rclone options |\n| `NNN_IDLE_TIMEOUT=300` | idle seconds to lock terminal [default: disabled] |\n| `NNN_COPIER=copier` | clipboard copier script [default: none] |\n| `NNN_TRASH=1` | trash files to the desktop Trash [default: delete] |\n\n#### Cmdline options\n\n```\nusage: nnn [OPTIONS] [PATH]\n\nThe missing terminal file manager for X.\n\npositional args:\n  PATH   start dir [default: .]\n\noptional args:\n -a      use access time\n -b key  open bookmark key\n -c      cli-only opener\n -d      detail mode\n -e name load session by name\n -E      EDITOR for undetached edits\n -f      run filter as cmd on prompt key\n -H      show hidden files\n -i      nav-as-you-type mode\n -K      detect key collision\n -n      version sort\n -o      open files on Enter\n -p file selection file [stdout if \'-\']\n -r      use advcpmv patched cp, mv\n -s      string filters [default: regex]\n -S      du mode\n -t      disable dir auto-select\n -v      show version\n -h      show help\n```\n\n#### Keyboard and mouse\n\nThe list below is from the **dev branch**. Press <kbd>?</kbd> in `nnn` to see the keybinds in your installed version.\n\n```\n NAVIGATION\n         Up k  Up          PgUp ^U  Scroll up\n       Down j  Down        PgDn ^D  Scroll down\n       Left h  Parent      ~ ` @ -  HOME, /, start, last\n  Ret Right l  Open              . Help & settings, file details and archive listing are shown in the `$PAGER`.\n2. To change shortcuts modify key bindings in `nnn.h` and compile.\n\n| Mouse click | Function |\n|---| --- |\n| Left single on context number | Visit context |\n| Left single on top row after context numbers | Visit parent |\n| Left single/double on last 2 rows | Toggle nav-as-you-type |\n| Left single | Select context or entry |\n| Left double | Select context or open entry |\n\n#### Lead key\n\nThe Lead/Leader key provides a powerful multi-functional navigation mechanism. It is case-sensitive and understands contexts, bookmarks and location shortcuts.\n\n| Follower key | Function |\n|:---:| --- |\n| <kbd>1-4</kbd> | Go to/create selected context |\n| key | Go to bookmarked location |\n| <kbd>\'</kbd> | Go to first file in directory |\n| <kbd>~</kbd> <kbd>`</kbd> <kbd>@</kbd> <kbd>-</kbd> | Visit HOME, `/`, start, last visited dir |\n| <kbd>.</kbd> | Toggle show hidden files |\n\nWhen the filter is on, <kbd>/</kbd> works as an additional Lead key.\n\n#### Help\n\n    $ nnn -h\n    $ man nnn\nTo lookup keyboard shortcuts at runtime, press <kbd>?</kbd>.\n\n## Elsewhere\n\n- [Wikipedia](https://en.wikipedia.org/wiki/Nnn_(file_manager))\n- [ArchWiki](https://wiki.archlinux.org/index.php/Nnn)\n- [FOSSMint](https://www.fossmint.com/nnn-linux-terminal-file-browser/)\n- [gHacks Tech News](https://www.ghacks.net/2019/11/01/nnn-is-an-excellent-command-line-based-file-manager-for-linux-macos-and-bsds/)\n- Hacker News [[1](https://news.ycombinator.com/item?id=18520898)] [[2](https://news.ycombinator.com/item?id=19850656)]\n- [It\'s FOSS](https://itsfoss.com/nnn-file-browser-linux/)\n- LinuxLinks [[1](https://www.linuxlinks.com/nnn-fast-and-flexible-file-manager/)] [[2](https://www.linuxlinks.com/bestconsolefilemanagers/)]\n- [Suckless Rocks](https://suckless.org/rocks/)\n- [Ubuntu Full Circle Magazine - Issue 135](https://fullcirclemagazine.org/issue-135/)\n\n## Developers\n\n- [Arun Prakash Jana](https://github.com/jarun) (Copyright \xc2\xa9 2016-2019)\n- [0xACE](https://github.com/0xACE)\n- [Anna Arad](https://github.com/annagrram)\n- [KlzXS](https://github.com/KlzXS)\n- [Sijmen J.
