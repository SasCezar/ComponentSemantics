Run M-x markdown-toc-refresh-toc -->\n**Table of Contents**\n\n- [Centaur Emacs](#centaur-emacs)\n    - [Features](#features)\n    - [Prerequisite](#prerequisite)\n        - [OS](#os)\n        - [GNU Emacs](#gnu-emacs)\n        - [Dotfiles](#dotfiles)\n    - [Quick Start](#quick-start)\n        - [Install](#install)\n        - [Update](#update)\n        - [Docker](#docker)\n    - [Customization](#customization)\n        - [Customize-group](#customize-group)\n        - [Manual](#manual)\n    - [Screenshots](#screenshots)\n    - [FAQ](#faq)\n\n<!-- markdown-toc end -->\n\nThis is an Emacs distribution that aims to enhance the default\nEmacs experience. It alters a lot of the default settings,\nbundles a plethora of additional packages and adds its own core\nlibrary to the mix. The final product offers an easy to use Emacs\nconfiguration for Emacs newcomers and lots of additional power for\nEmacs power users.\n\nIt\'s able to run on Windows, GNU Linux and macOS. It is compatible **ONLY with\nGNU Emacs 25.1 and above**. In general you\'re advised to always run with the\nlatest stable release - currently **26.3**.\n\n## Features\n\n- Out of box.\n- Clean and Fast.\n- Quick fuzzy search.\n- Better Org/Markdown support.\n- Support multiple programming languages\n    - C/C++/Object-C/C#/Java\n    - Python/Ruby/Perl/PHP/Shell/Powershell/Bat\n    - Javascript/Typescript/JSON/YAML\n    - HTML/CSS/XML\n    - Golang/Swift/Rust/Dart/Elixir\n    - ...\n- Auto completion.\n- Fly syntax check.\n- Fly spell check.\n- Git integration.\n- Project/Workspace integration.\n- Pomodor integration.\n- Support docker.\n- Better Chinese support:\n    - Chinese calendar\n    - Youdao dictionary\n    - Pinyin search\n\n## Prerequisite\n\n### OS\n\n- GNU Linux\n- macOS\n- Windows (Cygwin/MSYS)\n\n### GNU Emacs\n\nPlease refer to [Installing Emacs](http://wikemacs.org/index.php/Installing_Emacs).\n\n### Dotfiles\n\nRecommend to use [Centaur Dotfiles](https://github.com/seagle0128/dotfiles).\n\n## Quick Start\n\n### Install\n\n``` shell\nmv ~/.emacs.d ~/.emacs.d.bak\ngit clone --depth 1 https://github.com/seagle0128/.emacs.d.git ~/.emacs.d\n```\n\nor download the [zip\npackage](https://github.com/seagle0128/.emacs.d/archive/master.zip) directly and\nextract to `~/.emacs.d`.\n\nThen start Emacs. Wait for a while to install packages at the first startup. Enjoy! :smile:\n\n### Update\n\n``` emacs-lisp\n# Update Centaur Emacs (then restart), including configurations and packages\nM-x centaur-update\nM-x centaur-update-and-restart\n\n# Update Emacs configurations only\nM-x centaur-update-config\n\n# Update ~/.dotfiles if it exists\nM-x centaur-update-dotfiles\n\n# Update packages only (then restart)\nM-x centaur-update-packages\nM-x centaur-update-packages-and-restart\n\n# Update all including configurations, packages and dotfiles\nM-x centaur-update-all\n```\n\n### Docker\n\n``` shell\ncd ~/.emacs.d/Dockerfile\ndocker build -t centaur/emacs .\ndocker run -it centaur/emacs bash\n```\n\n## Customization\n\n### Customize Group\n\n`M-x customize-group` and select `centaur`. Set and save the configurations, then restart Emacs.\n\n### Manual\n\nCopy `custom-template.el` to `custom.el` and change the configurations, then restart Emacs.\n\nFor Example:\n\n``` emacs-lisp\n(setq centaur-logo nil)                        ; Logo file or nil (official logo)\n(setq centaur-full-name "user name")           ; User full name\n(setq centaur-mail-address "user@email.com")   ; Email address\n(setq centaur-proxy "127.0.0.1:1080")          ; Network proxy\n(setq centaur-package-archives \'emacs-china)   ; Package repo: melpa, melpa-mirror, emacs-china, netease or tuna\n(setq centaur-theme \'dark)                     ; Color theme: default, classic, colorful, dark, light, day or night\n(setq centaur-dashboard nil)                   ; Use dashboard at startup or not: t or nil\n(setq centaur-lsp \'eglot)                      ; Set LSP client: lsp-mode, eglot or nil\n(setq centaur-chinese-calendar nil)            ; Use Chinese calendar or not: t or nil\n(setq centaur-benchmark t)                     ; Enable initialization benchmark or not: t or nil\n```\n\nThe default package archives is `melpa`. You can change it in `custom.el`, or\nswitch manually via `M-x switch-package-archives` anytime.\n\nFor the personal configurations, you could put to `~/.emacs.d/custom-post.el`.\n\n## Screenshots\n\n![Dashboard](https://user-images.githubusercontent.com/140797/56488858-4e5c4f80-6512-11e9-9637-b9395c46400f.png\n"Dashboard (default theme)")\n\n![color_icons](https://user-images.githubusercontent.com/140797/56474931-8bddc000-64b3-11e9-93d8-0204c3752790.png\n"Main (color icons)")\n\n![markdown_magit](https://user-images.githubusercontent.com/140797/56489193-7f894f80-6513-11e9-9c95-ea1ef41efb1f.png\n"Markdown and Magit")\n\n![org_classic](https://user-images.githubusercontent.com/140797/56489410-2837af00-6514-11e9-9e04-a1663ac479e8.png\n"Org (classic theme)")\n\n![main_light](https://user-images.githubusercontent.com/140797/56489516-85cbfb80-6514-11e9-9dd4-8602bf8a4ec7.png\n"Main (light theme)")\n\n## FAQ\n\n1. How to display icons correctly in `Centaur Emacs`?\n\n    `all-the-icons` are necessary. Run `M-x all-the-icons-install-fonts` to\n    install the resource fonts. On Windows, the fonts should be installed manually.\n    `all-the-icons` only support GUI. If you don\'t like color icons,\n    `(setq all-the-icons-color-icons nil)` to disable. Please refer to\n    [all-the-icons.el](https://github.com/domtronn/all-the-icons.el) for details.\n\n    If some icons are not displayed correctly although `all-the-icons` fonts are\n    installed correctly, please install [symbola](http://users.teilar.gr/~g1951d/).\n    Refer to [#121](https://github.com/seagle0128/.emacs.d/issues/121) for more\n    details.\n\n1. The packages cannot be installed, what should I do?\n\n   Generally it\'s due to connection issue. Please refer to\n   [#98](https://github.com/seagle0128/.emacs.d/issues/98).\n   - `M-x package-refresh-contents` and try again.\n   - `(setq gnutls-algorithm-priority "NORMAL:-VERS-TLS1.3")`.\n   - Use other mirror of elpa.\n   - Change another network to retry.\n\n1. How to search Chinese via pinyin?\n\n    In Emacs, `C-s !`. If you just want to search `!`, use `C-s \\!`.\n\n1. How to use the Centaur Dashboard?\n\n    If `centaur-dashboard` is non-nil the Centaur Dashboard will be opened at\n    startup automatically. After startup, you could use `F2` to reopen it\n    anytime. In the dashboard, please press `?` or `h` to get the help. You can\n    easily jump to the sections or buttons with keybindings, e.g. Homepage(`H`),\n    Session(`R`, `L`), Settings(`S`), Update(`U`), Recent Files (`r`),\n    Bookmarks(`m`) and Projects(`p`).\n\n1. Does Centaur Emacs support Language Server Protocol (LSP)?\n\n    The LSP feature is supported and enabled by default in Centaur Emacs.\n    `lsp-mode` is the default client, and `eglot` is another choice. Use `(setq\n    centaur-lsp nil)` to disable LSP if you don\'t like it.\n\n    To use LSP you should install the language servers. Refer to [lsp-mode:\n    supported-languages](https://github.com/emacs-lsp/lsp-mode#supported-languages)\n    or [eglot: Connecting to a\n    server](https://github.com/joaotavora/eglot#connecting-to-a-server) for\n    the details.\n\n1. How to enable `plantuml` in `org-mode`?\n\n    Put `(setq org-plantuml-jar-path "<path of plantumx.x.x.jar>")` in `custom.el`.\n\n1. Why the Emacs environment variables and `exec-path` are different between GUI\n   and terminal?\n\n    Please refer to #33. You should instead set environment variables in startup\n    files like .profile, .bash_profile or .zshenv, then `Centaur Emacs` is able\n    to recognize and import the environment variables.\n\n## Donate\n\nIf you think the it\'s helpful for you, please consider paying a cup of coffee\nfor me.
