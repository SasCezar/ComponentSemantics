 Re-set `org-todo\' & `org-headline-done\' faces to make them respect\n     underlying faces (i.e. don\'t override the :height or :background of\n     underlying faces).\n  -  Make statistic cookies respect underlying faces.\n  -  Fontify item bullets (make them stand out)\n  -  Fontify item checkboxes (and when they\'re marked done), like TODOs that\n     are marked done.\n  -  Fontify dividers/separators (5+ dashes)\n  -  Fontify #hashtags and @at-tags, for personal convenience; see\n     `doom-org-special-tags` to disable this.\n- **Resources that may interest you:**\n  - To get dimmed sidebars and brighter source buffers (like in the\n    [screenshots]), see [solaire-mode].\n  - To get line number highlighting for `nlinum`, set\n    `nlinum-highlight-current-line` to non-nil (or see [hlinum] for `linum`).\n  - In the meantime, check\n    out [my mode-line configuration][mode-line] in my [emacs.d].\n\n## Install\n\n### Doom Emacs\n\nThe built-in `:ui doom` module installs `doom-themes` for you, and loads\n`doom-one` by default. You can change the theme by changing the `doom-theme`\nvariable from `~/.doom.d/config.el`:\n\n``` emacs-lisp\n(setq doom-theme \'doom-city-lights)\n```\n\nOr by loading it manually:\n\n``` emacs-lisp\n(load-theme \'doom-city-lights t)\n```\n\n### Manually\n\n`doom-themes` is available on MELPA.\n\n- [Straight](https://github.com/raxod502/straight.el) users can install it with\n  `M-x straight-use-package RET doom-themes`\n- And package.el users: `M-x package-install RET doom-themes`\n\nHere is a example configuration for `doom-theme`:\n\n```emacs-lisp\n(use-package doom-themes\n  :config\n  ;; Global settings (defaults)\n  (setq doom-themes-enable-bold t    ; if nil, bold is universally disabled\n        doom-themes-enable-italic t) ; if nil, italics is universally disabled\n  (load-theme \'doom-one t)\n\n  ;; Enable flashing mode-line on errors\n  (doom-themes-visual-bell-config)\n  \n  ;; Enable custom neotree theme (all-the-icons must be installed!)\n  (doom-themes-neotree-config)\n  ;; or for treemacs users\n  (setq doom-themes-treemacs-theme "doom-colors") ; use the colorful treemacs theme\n  (doom-themes-treemacs-config)\n  \n  ;; Corrects (and improves) org-mode\'s native fontification.\n  (doom-themes-org-config))\n```\n\n[The wiki contains details for customizing the neotree theme][wiki].\n\n## Common Issues\n\n+ If you use `nlinum` or `linum` in org-mode, the larger headline sizes in some\n  themes [could bleed into the line numbers](https://github.com/hlissner/emacs-doom-themes/issues/86).\n\n  Fix this by setting :height explicitly for your line number plugins, after\n  you\'ve loaded the theme. e.g.\n\n  ```emacs-lisp\n  (let ((height (face-attribute \'default :height)))\n    ;; for all linum/nlinum users\n    (set-face-attribute \'linum nil :height height)\n    ;; only for `linum-relative\' users:\n    (set-face-attribute \'linum-relative-current-face nil :height height)\n    ;; only for `nlinum-relative\' users:\n    (set-face-attribute \'nlinum-relative-current-face nil :height height))\n  ```\n\n## Contribute\n\nI welcome contributions of any kind, be they pull requests, bug reports or elisp\npointers.
