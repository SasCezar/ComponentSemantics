b'# Emacs PHP Mode\n\n[![Emacs: 26.3](https://img.shields.io/badge/Emacs-26.3-blue.svg)](https://www.gnu.org/software/emacs/)\n[![lang: PHP 7](https://img.shields.io/badge/lang-PHP%207-brightgreen.svg)](http://php.net/manual/migration70.php)\n[![lang: PHP 5](https://img.shields.io/badge/lang-PHP%205-green.svg)](http://php.net/downloads.php)\n[![Build Status](https://github.com/emacs-php/php-mode/workflows/CI/badge.svg)](https://github.com/emacs-php/php-mode/actions)\n[![melpa badge][melpa-badge]][melpa-link]\n[![melpa stable badge][melpa-stable-badge]][melpa-stable-link]\n[![GPL v3](https://img.shields.io/badge/license-GPL_v3-green.svg)](http://www.gnu.org/licenses/gpl-3.0.txt)\n\nThis is a major mode development project to support PHP coding in GNU Emacs.\nThis fork builds on the work of:\n\n1. Turadg Aleahmad (Original Author)\n2. Aaron S. Hawley\n3. Lennart Borgman\n4. Eric James Michael Ritz\n5. USAMI Kenta (@zonuexe)\n\nPlease submit any bug reports or feature requests by creating issues on [the GitHub page for PHP Mode][php-mode].  Alternatively you may also request features via [the FeatHub page][feathub] for the entire [PHP suite for GNU Emacs][php-suite].\n\nInstallation\n------------\n\n**PHP Mode works on Emacs 24.3 or later.**  PHP Mode may work with older versions of Emacs but this is not guaranteed.  Bug reports for problems related to using PHP Mode with older versions of Emacs will most like *not* be addressed.\n\nThe current support policy can be found on the [Supported Version] page.\n\n### **(RECOMMENDED)** Install from MELPA\n\n[![melpa badge][melpa-badge]][melpa-link] [![melpa stable badge][melpa-stable-badge]][melpa-stable-link]\n\nWith GNU Emacs 24 or later then you can use its [package][] feature (or [Cask][]) to install PHP Mode from [MELPA][] or [MELPA Stable][].\n\n### Install using OS package system\n\nPHP Mode is available from [package systems provided by several OSs](https://repology.org/project/emacs:php-mode/versions).  This is the easiest way to add PHP to Emacs and can be installed with just `sudo apt-get install elpa-php-mode`.\n\nThe package name on Debian, Ubuntu, and some derived operating systems is [`elpa-php-mode`](https://packages.debian.org/buster/elpa-php-mode).  The packages provided for these operating systems are stable, but since they are behind the latest version of PHP Mode, consider the possibility of not incorporating the latest PHP feature support and performance improvements.\n\nAlso, the `php-elisp` package provided by [Debian 9 (stretch)](https://packages.debian.org/stretch/php-elisp) and [Ubuntu 18.10](https://packages.ubuntu.com/cosmic/php-elisp) and earlier is [extremely old](https://github.com/emacs-php/php-mode/issues/430), so **PLEASE DO NOT INSTALL IT**.\n\n### Manual installation\n\nPlease `git clone` this project or download and unarchive tar or zip file from [php-mode releases].\n\nYou can choose one of the following **A**, **B**, **C** installation methods.\n\n#### **A**: `(load php-mode-autoloads.el)` *(RECOMMENDED)*\n\nThis is an initialization method that achieves the same performance and ease of use as using a package manager.\n\nBy moving the downloaded file to the extracted path of the current directory and executing the `make` command, byte compilation and `php-mode-autoloads.el` is generated.  Just load the file from `init.el` and you are ready to use.\n\n```el\n;; Put follow code into init.el\n(when (file-directory-p "~/path/to/php-mode")\n  (load "~/path/to/php-mode/php-mode-autoloads.el"))\n\n;; Any code below is *unnecessary*\n;; (require \'php-mode)\n;; (add-to-list \'load-path (expand-file-name "~/path/to/php-mode"))\n;; (add-to-list \'auto-mode-alist \'("\\\\.php\\\\\'" . php-mode))\n```\n#### **B**: `(autoload \'php-mode)`\n\nThis is for advanced users who want to reduce the slight increase in reading when Emacs starts.\n\nAlso in this case, it is recommended to byte compile with `make`.\n\n```el\n;; Put follow code into init.el\n(autoload \'php-mode (expand-file-name "~/path/to/php-mode/php-mode") "\\\nMajor mode for editing PHP code.\n\n\\\\{php-mode-map}\n\n\\(fn)" t nil)\n\n(add-to-list \'auto-mode-alist \'("\\\\.\\\\(?:php\\\\|phtml\\\\)\\\\\'" . php-mode))\n\n;; Any code below is *unnecessary*\n;; (add-to-list \'load-path (expand-file-name "~/path/to/php-mode"))\n;; (require \'php-mode)\n```\n\n#### **C**: `(require \'php-mode)` *(NOT RECOMMENDED)*\n\nLoad `php-mode` synchronously from a specific path.  It will load 10 times the size of the code compared to method **A**, and how much the startup time will depend on the performance of your machine\'s CPU and file system.\n\n```el\n(require \'php-mode "~/path/to/php-mode/php-mode")\n(add-to-list \'load-path (expand-file-name "~/path/to/php-mode"))\n```\n\nReporting Bugs\n--------------\n\nWhen reporting a bug please run the function `M-x php-mode-debug` and include its output in your bug report.  This helps up reproduce any problem you may have.\n\n\nExperimental and In-Progress Features\n-------------------------------------\n\n### CC Mode, CEDET, EDE, and Semantic\n\nIn 2013 Daniel Haxney began rewriting parts of PHP Mode in terms of Emacs\' built-in CC Mode.  This laid the foundation for incorporating some of the inherit IDE-ish features of Emacs, such as CEDET, EDE, and Semantic.  Support for these tools continues to improve thanks to the work of Andrea Turso, Steven R\xc3\xa9mot, Joris Steyn, and others.  If you wish to test, contribute to, or simply experiment with such features then [this thread is a good place to start](https://github.com/emacs-php/php-mode/issues/256).\n\n### PHP 7 Support\n\nPHP 7 has been released.  PHP Mode supports the following features and changes from PHP 7:\n\n1. Type-hints for return values in functions and methods receive syntax highlighting in the same way as type-hints for function and method parameters.\n\n2. PHP Mode treats `yield from` as keyword in the same way it already does for a sole `yield`.\n\n3. It recognizes `strict_types` as a special declaration in the same way as `ticks`.\n\n\nFeatures\n--------\n\n### New Keywords\n\nNow PHP Mode supports syntax highlighting for new keywords which PHP 5.4 introduced, e.g. those related to traits, such as `insteadof`.  Also supported are the older keywords `clone` and `default`.\n\n### Constants\n\nSyntax highlighting includes every magic constant and predefined constant listed on the official PHP site.  However, some constants from specific extensions are not currently included.\n\n### Traits, Interfaces, and Namespaces\n\nTraits, interfaces, and namespaces now appear under Imenu listings. Fontification behaves properly for namespaces as well, so that code like `namespace Foo\\Bar\\Baz` no longer looks like a warning.  This is also true for namespace aliases, e.g. `use <namespace> as <alias>`; currently the aliased name is not listed in Imenu, but future versions will address this.\n\n### Treatment of Underscores\n\nPHP Mode treats underscores as \xe2\x80\x98symbol constituents\xe2\x80\x99 (in Emacs terminology) so that you can use keys like `M-f` and `M-b` to move through the individual parts of a variable name like `$foo_bar_baz`.\n\n### Chained Method Calls\n\nPHP Mode can align method calls over multiple lines anchored around the `->` operator, e.g.:\n\n```php\n$object->foo()\n       ->bar()\n       ->baz();\n```\n\nThis behaviour is off by default, but you can customize the variable `php-lineup-cascaded-calls` to enable this.\n\n**Note:** Alignment will only work if you use one of the php-mode coding styles or inherit one of the styles.\n\n### Nested Array Formatting\n\nNested function calls and `array()` structures now look better by default (or at least in my opinion).  Here is an example of the style:\n\n```php\n$results = Post::model()->find(\n    array(\n        \'select\' => \'title\',\n        \'condition\' => \'postID=:postID\',\n        \'params\' => array(\':postID\' => 10),\n    )\n);\n```\n\n### Anonymous Functions\n\nAnonymous functions such as\n\n```php\n$greet = function($name) { ... };\n```\n\nwill now appear on Imenu; in this case the name will be `$greet`.\n\n### Flymake Support\n\nBy customizing the variable `php-executable` you can enable Flymake mode in order to see warnings and errors in real-time as you write code.\n\n### Search Local Documentation\n\nThe key command `C-c C-f` will search the PHP website for documentation on the word under the cursor.  However, if you have a [local copy of the PHP documentation](http://us2.php.net/download-docs.php) then PHP Mode will try searching that documentation first.  All you need to do is customize the variable `php-manual-path` and give it the path to your copy of the documentation.  If PHP Mode cannot find something locally then it will still fallback on searching the PHP website.\n\n### Executing Regions of PHP\n\nThe command `php-send-region`, which is bound to `C-c C-r` by default, will execute the selected region of PHP code.  In conjunction with the Emacs command `C-x h` you can use this to execute an entire file.  Any output will appear in a buffer called `*PHP*`.\n\n### PHPDoc Tag / Annotation Highlighting\n\nPHPDoc is a documentation format similar to [JavaDoc](https://en.wikipedia.org/wiki/Javadoc).\n\nThere are `@param`, `@return`, `@var`... etc in the notation called **tag**, look at [list of tags defined by phpDocumentor2](https://phpdoc.org/docs/latest/references/phpdoc/tags/index.html).  (These tags are compatible with static type checkers like PhpStorm and [Phan](https://github.com/etsy/phan).)\n\nIn addition, it also partially supports notation called **annotation**.  Annotation has a slightly different grammar from tag, and the example is `@Annotation(attr1="vvv", attr2="zzz")`.\n\n[Symfony](http://symfony.com/) project and [Go! AOP](https://github.com/goaop/framework) and some projects/frameworks use annotation grammer based on [Doctrine Annotations](http://docs.doctrine-project.org/projects/doctrine-common/en/latest/reference/annotations.html).\n\n```php\n/**\n * Summary of Product class\n *\n * @copyright 2112 John Doe\n * @license https://spdx.org/licenses/Apache-2.0.html Apache License 2.0\n * @ORM\\Entity\n * @ORM\\Table(name="product")\n */\nclass Product\n{\n    /**\n     * @ORM\\Id\n     * @ORM\\Column(type="integer")\n     * @ORM\\GeneratedValue(strategy="AUTO")\n     */\n    protected $id;\n\n    /**\n     * @ORM\\Column(type="string", length=100)\n     */\n    protected $name;\n\n    /**\n     * @ORM\\Column(type="decimal", scale=2)\n     */\n    protected $price;\n\n    /**\n     * @ORM\\Column(type="text")\n     */\n    protected $description;\n}\n```\n\nThe annotations are the lines that begin with the `@` character, and PHP Mode will give these special highlighting to help them stand out.\n\nPHP Mode has not fully supported [PSR-5: PHPDoc (Draft)](https://github.com/phpDocumentor/fig-standards/blob/master/proposed/phpdoc.md) yet.  We want to support them, but the current implementation still limited.  See issue [#478](https://github.com/emacs-php/php-mode/issues/478) for details.\n\n### Coding Styles\n\nBy default PHP Mode tries to provide a reasonable style for indentation and formatting, which you can use via the function `php-enable-default-coding-style`.  However, it provides other options suited for particular projects which you may find useful.  Other coding styles are available through these functions:\n\n1. `php-enable-pear-coding-style`\n2. `php-enable-drupal-coding-style`\n3. `php-enable-wordpress-coding-style`\n4. `php-enable-symfony2-coding-style`\n5. `php-enable-psr2-coding-style`\n\nThey will help format your code for PEAR/PSR-2 projects, or work on Drupal, WordPress, and Symfony2 software, respectively.  You may enable any of them by default by running `M-x customize-group <RET> php` and looking for the \xe2\x80\x98PHP Mode Coding Style\xe2\x80\x99 option.  You may also enable any of these via a hook, e.g.\n\n```lisp\n(add-hook \'php-mode-hook \'php-enable-default-coding-style)\n```\n\n#### Symfony2 Style\n\nWith this style method call chains can be formatted with indented continuation and a hanging semi-colon:\n\n```php\n    $user1\n        ->setCreateDate(new \\DateTime(\'2007-05-07 01:34:45\'))\n        ->setLastDate(new \\DateTime(\'2012-08-18 19:03:02\'))\n        ->setUsername(\'jay\')\n    ;\n```\n\nThis style is used widely throughout Symfony2 source code even if it is not explicitly mentioned in their conventions documents.\n\n### Extra Constants\n\nIf you commonly use a framework or library that defines a set of constants then you may wish to customize the value of `php-extra-constants`.  It is a list of strings that PHP Mode will treat as additional constants, i.e. providing them the same level syntax highlighting that PHP Mode uses for built-in constants.\n\n### Web Mode Constants and Keywords\n\nIf you use [Web Mode][] then PHP Mode will attempt to use any additional PHP constants and keywords that Web Mode allows you to define.\n\n### Avoid HTML Template Compatibility\n\nMany developers use PHP Mode to edit pure PHP scripts (e.g. files with only PHP and no HTML). A basic compatibility layer with HTML has historically been part of PHP Mode but it does not work perfectly and can cause some bad side effects such as slowness and incorrect font locking.  Configuring the `php-template-compatibility` property with a `nil` will cancel any attempt of HTML compatibility.  [Web Mode](http://web-mode.org/) is a great alternative to PHP Mode if you need to work with PHP scripts that do contain HTML and other markup.\n\n### Subword Mode\n\nGNU Emacs comes with [Subword Mode][], a minor mode that allows you to navigate the parts of a [camelCase][] as if they were separate words.  For example, PHP Mode treats the variable `$fooBarBaz` as a whole name by default.  But if you enable Subword Mode then Emacs will treat the variable name as three separate words, and therefore word-related commands (e.g. `M-f`, `M-b`, `M-d`) will only affect the camelCase part of the name under the cursor.\n\nIf you want to always use Subword Mode for PHP files then you can add this to your Emacs configuration:\n\n```lisp\n(add-hook \'php-mode-hook (lambda () (subword-mode 1)))\n```\n\nThe key-binding `C-c C-w` will also toggle Subword Mode on and off.\n\n### Amaka Support\n\nViewing and editing build scripts for [Amaka](http://trashofmasters.github.io/amaka/) will automatically enable PHP Mode.\n\n### Insert current class/namespace\n\n```el\n(with-eval-after-load \'php-mode\n  (define-key php-mode-map (kbd "C-c C--") \'php-current-class)\n  (define-key php-mode-map (kbd "C-c C-=") \'php-current-namespace))\n```\n\nOther Packages for PHP programming\n----------------------------------\n\n- Completions\n  -
