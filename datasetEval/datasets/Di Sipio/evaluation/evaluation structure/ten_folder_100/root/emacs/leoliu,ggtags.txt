b'=========================================================\n Emacs frontend to GNU Global source code tagging system\n=========================================================\n\nThis package is part of `GNU ELPA <http://elpa.gnu.org>`_ (``M-x\nlist-packages``) and is also available on `MELPA\n<https://melpa.org/#/ggtags>`_.\n\nThe goal is to make working with GNU Global in Emacs as effortlessly\nand intuitively as possible and to integrate tightly with standard\nemacs packages. ``ggtags.el`` is tested in Emacs 24 and 25. Patches,\nfeature requests and bug reports are welcome. Thanks.\n\nFeatures\n~~~~~~~~\n\n#. Build on ``compile.el`` for asynchronicity and its large\n   feature-set.\n#. Automatically update Global\'s tag files when needed with tuning for\n   large source trees.\n#. Intuitive navigation among multiple matches with mode-line display\n   of current match, total matches and exit status.\n#. Read tag with completion.\n#. Show definition at point.\n#. Jump to #include files.\n#. Support search history and saving a search to register/bookmark.\n#. Query replace.\n#. Manage Global\'s environment variables on a per-project basis.\n#. Highlight (definition) tag at point.\n#. Abbreviated display of file names.\n#. Support all Global search backends: ``grep``, ``idutils`` etc.\n#. Support `exuberant ctags <http://ctags.sourceforge.net/>`_ and\n   ``pygments`` backend.\n#. Support all Global\'s output formats: ``grep``, ``ctags-x``,\n   ``cscope`` etc.\n#. Support projects on remote hosts (e.g. via ``tramp``).\n#. Support eldoc.\n#. Search ``GTAGSLIBPATH`` for references and symbols.\n\nScreenshot\n~~~~~~~~~~\n\n.. figure:: http://i.imgur.com/wx8ZPGe.png\n   :width: 500px\n   :target: http://i.imgur.com/wx8ZPGe.png\n   :alt: ggtags.png\n\nWhy GNU Global\n~~~~~~~~~~~~~~\n\nThe opengrok project composed a feature comparison `table\n<https://github.com/OpenGrok/OpenGrok/wiki/Comparison-with-Similar-Tools>`_\nbetween a few tools.\n\nInstall Global and plugins\n~~~~~~~~~~~~~~~~~~~~~~~~~~\n\n1. Compile and install Global with ``exuberant-ctags``\n   ::\n\n     ./configure --prefix=<PREFIX> --with-exuberant-ctags=/usr/local/bin/ctags\n     make && make install\n\n   The executable ``ctags`` is unfortunately named because ``emacs``\n   also includes a command of the same name. So make sure it is from\n   http://ctags.sourceforge.net. See ``plugin-factory/PLUGIN_HOWTO``\n   (``plugin-factory/README`` for Global < 6.5) in GNU Global\n   source for further information.\n\n2. Install ``pygments`` plugin\n\n   The ``pygments`` plugin has been included in ``global`` since\n   version ``6.3.2``. ``pip install pygments`` is the only step\n   required. Note the plugin is not activated by the default\n   ``gtags.conf`` or ``.globalrc``. See\n   ``global/plugin-factory/PLUGIN_HOWTO.pygments`` for details.\n\n   The following instructions are for older ``global``.\n   ::\n\n     pip install pygments\n     git clone https://github.com/yoshizow/global-pygments-plugin.git\n     cd global-pygments-plugin/\n     sh reconf.sh\n     ./configure --prefix=<PREFIX> --with-exuberant-ctags=/usr/local/bin/ctags\n     make && make install\n     cp sample.globalrc $HOME/.globalrc\n\n   Make sure the value of ``<PREFIX>`` agree with step 1.\n\nConfig\n~~~~~~\n\nGlobal with ``exuberant-ctags`` and ``pygments`` plugins can support\ndozens of programming languages. For example, to enable\n``ggtags-mode`` for C/C++/Java modes::\n\n    (add-hook \'c-mode-common-hook\n              (lambda ()\n                (when (derived-mode-p \'c-mode \'c++-mode \'java-mode)\n                  (ggtags-mode 1))))\n\nAlso see https://github.com/leoliu/ggtags/wiki for more examples.\n\nUsage\n~~~~~\n\nOpen any file in a project and type ``M-x ggtags-mode``. Use ``M-.``\n(``ggtags-find-tag-dwim``) to find the tag at point. If the project\nhas not been indexed (i.e. no ``GTAGS`` file exists), ``ggtags`` will\nask for the project root directory and index it recursively.\nAlternatively one can invoke ``ggtags-create-tags`` to index a\ndirectory. The mode line will display the directory name next to the\nbuffer name. If point is at a valid definition tag, it is underlined.\n\n``ggtags`` is similar to the standard ``etags`` package. For example\nthese keys ``M-.``, ``M-,`` and ``C-M-.`` should work as expected in\n``ggtags-mode``.\n\nThe following search commands are available:\n\nggtags-find-tag-dwim\n\n   Find a tag by context.\n\n   If point is at a definition tag, find references, and vice versa.\n   If point is at a line that matches ``ggtags-include-pattern``, find\n   the include file instead.\n\n   To force finding a definition tag, call it with a prefix (``C-u``).\n\nggtags-find-tag-mouse\n\n   Like ``ggtags-find-tag-dwim`` but suitable for binding to mouse\n   events.\n\nggtags-find-definition\n\n   Find definition tags. With ``C-u`` ask for the tag name with\n   completion.\n\nggtags-find-reference\n\n   Find reference tags. With ``C-u`` ask for the tag name with completion.\n\nggtags-find-other-symbol\n\n   Find tags that have no definitions. With ``C-u`` ask for the tag\n   name with completion.\n\nggtags-find-tag-regexp\n\n   Find definition tags matching a regexp. By default it lists all\n   matching tags in the project. With ``C-u`` restrict the lists to a\n   directory of choice.\n\nggtags-idutils-query\n\n   Use idutils to find matches.\n\nggtags-grep\n\n   Grep for lines matching a regexp. This is usually the slowest.\n\nggtags-find-file\n\n   Find a file from all the files indexed by ``gtags``.\n\nggtags-query-replace\n\n   Do a query replace in all files found in a search.\n\nHandling multiple matches\n+++++++++++++++++++++++++\n\nWhen a search finds multiple matches, a buffer named\n``*ggtags-global*`` is popped up and ``ggtags-navigation-mode`` is\nturned on to facilitate locating the right match.\n``ggtags-navigation-mode`` makes a few commands in the\n``*ggtags-global*`` buffer globally accessible:\n\n``M-n``\n\n   Move to the next match.\n\n``M-p``\n\n   Move to the previous match.\n\n``M-}``\n\n   Move to next file.\n\n``M-{``\n\n   Move to previous file.\n\n``M-=``\n\n   Move to the file where navigation session starts.\n\n``M-<``\n\n   Move to the first match.\n\n``M->``\n\n   Move to the last match.\n\n``C-M-s`` or ``M-s s``\n\n   Use ``isearch`` to find the match.\n\n``RET``\n\n   Found the right match so exit navigation mode. Resumable by\n   ``M-x tags-loop-continue``.\n\n``M-,`` (``M-*`` if Emacs < 25)\n\n   Abort and go back to the location where the search was started.\n\nMiscellaneous commands\n++++++++++++++++++++++\n\nCommands are available from the ``Ggtags`` menu in ``ggtags-mode``.\n\nggtags-prev-mark\n\n   Move to the previously (older) visited location. Unlike ``M-,``\n   (``M-*`` if Emacs < 25) this doesn\'t delete the location from the\n   tag ring.\n\nggtags-next-mark\n\n   Move to the next (newer) visited location.\n\nggtags-view-tag-history\n\n   Pop to a buffer listing all visited locations from newest to\n   oldest. The buffer is a next error buffer and works with standard\n   commands ``next-error`` and ``previous-error``. In addition ``TAB``\n   and ``S-TAB`` move to next/prev entry, and ``RET`` visits the\n   location. ``M-n`` and ``M-p`` move to and display the next/previous\n   entry.\n\nggtags-view-search-history\n\n   View or re-run past searches as kept in\n   ``ggtags-global-search-history``.\n\nggtags-kill-file-buffers\n\n   Kill all file-visiting buffers of current project.\n\nggtags-toggle-project-read-only\n\n   Toggle opening files in ``read-only`` mode. Handy if the main\n   purpose of source navigation is to read code.\n\nggtags-visit-project-root\n\n   Open the project root directory in ``dired``.\n\nggtags-delete-tags\n\n   Delete the GTAGS, GRTAGS, GPATH and ID files of current project.\n\nggtags-explain-tags\n\n  Explain how each file is indexed in current project.\n\nggtags-browse-file-as-hypertext\n\n   Use ``htags`` to generate HTML of the source tree. This allows\n   browsing the project in a browser with cross-references.\n\nIntegration with other packages\n+++++++++++++++++++++++++++++++\n\n* eldoc\n\n  ``Eldoc`` support is set up by default on emacs 24.4+. For older\n  versions set, for example, in the desired major mode:\n\n  ::\n\n     (setq-local eldoc-documentation-function #\'ggtags-eldoc-function)\n\n* imenu\n\n  Emacs major modes usually have excellent support for ``imenu`` so\n  this is not enabled by default. To use:\n  ::\n\n    (setq-local imenu-create-index-function #\'ggtags-build-imenu-index)\n\n* hippie-exp\n  ::\n\n     (setq-local hippie-expand-try-functions-list\n                 (cons \'ggtags-try-complete-tag hippie-expand-try-functions-list))\n\n* company\n\n  ``company`` can use ``ggtags`` as completion source via\n  ``company-capf`` which is enabled by default.\n\n* helm\n\n  If ``helm-mode`` is enabled ``ggtags`` will use it for completion if\n  ``ggtags-completing-read-function`` is nil.\n\nNEWS\n~~~~\n\n(devel)\n+++++++\n\n#.
