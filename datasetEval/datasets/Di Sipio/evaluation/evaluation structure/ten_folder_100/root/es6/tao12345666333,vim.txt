Vundle will auto manage the `.vim` directory\xef\xbc\x8call plugins which has been configured will be downloaded to `~/.vim/bundle/` by default, please keep the `.vim` directory clean before use it. When Vundle installing plugins `git clone` operation will be triggered\xef\xbc\x8cthe search operation need `curl` .\n\n#### Configuration(one part)\n\n```vim\n" let Vundle manage Vundle\nPlugin \'gmarik/vundle\'\n\n" ============================================================================\n" Active plugins\n" You can disable or add new ones here:\n\n" Plugins from github repos:\n\n" Better file browser\nPlugin \'scrooloose/nerdtree\'\n" Code commenter\nPlugin \'scrooloose/nerdcommenter\'\n" Class/module browser\nPlugin \'majutsushi/tagbar\'\n" Code and files fuzzy finder\nPlugin \'kien/ctrlp.vim\'\n" Extension to ctrlp, for fuzzy command finder\nPlugin \'fisadev/vim-ctrlp-cmdpalette\'\n" Zen coding\nPlugin \'mattn/emmet-vim\'\n" Git integration\nPlugin \'motemen/git-vim\'\n" Tab list panel\nPlugin \'kien/tabman.vim\'\n\n```\n\n#### Supported operations\n\n|   command             |    description        |\n|-----------------------|:---------------------:|\n|  :PluginList          |   list all Plugins    |\n|  :PluginInstall(!)    | install/update Plugin |\n|  :PluginSearch(!) foo |search Plugin about foo|\n|  :PluginClean(!)      |  clean unused Plugins |\n|  :PluginUpdate        |      update Plugins   |\n\n\n### Project file browser\xef\xbc\x88NERDTree\xef\xbc\x89\n\nIn this configuration\xef\xbc\x8cI use [**NERDTree**](https://github.com/scrooloose/nerdtree) as file browser. The NERDTree allows you to explore your filesystem and to open files and directory. It also allows you to hide files or set bookmarks etc. In NERDTree window input `?` can get the operation guide. This configuration filters out `.pyc`, `.git`, `.hg`, `.svn` etc.\n\n#### Configuration\n\n```vim\n" auto open or close NERDTree\nautocmd vimenter * if !argc() | NERDTree | endif\nautocmd bufenter * if (winnr("$") == 1 && exists("b:NERDTreeType") && b:NERDTreeType == "primary") | q | endif\n\n" NERDTree -----------------------------\n\n" toggle nerdtree display\nmap <F3> :NERDTreeToggle<CR>\n" open nerdtree with the current file selected\nnmap ,t :NERDTreeFind<CR>\n" don;t show these file types\nlet NERDTreeIgnore = [\'\\.pyc$\', \'\\.pyo$\']\n```\n\n#### Supported operation\n\n|  shortcut key         |    description             |\n|-----------------------|:--------------------------:|\n|      F3               | open/close NERDTree        |\n|      ,t               |open NERDTree and select current file|\n\n\n### Syntax checking\n\nIn this configuration\xef\xbc\x8cI use [**Syntastic**](https://github.com/scrooloose/syntastic) plugin for syntax checking. Support `C/C++/Go/Python/Haskell/Ruby/JavaScript` etc. For JavaScript, I use `eslint` as checker, so it can check ES6 and JSX etc. You can see [JSLint, JSHint\xe5\x92\x8cESLint\xe7\x9a\x84\xe5\xaf\xb9\xe6\xaf\x94\xe5\x8f\x8aVim\xe9\x85\x8d\xe7\xbd\xae](http://moelove.info/2015/11/28/JSLint-JSHint-ESLint%E5%AF%B9%E6%AF%94%E5%92%8CVim%E9%85%8D%E7%BD%AE/) for more details, when you want to change checker tools, just modify a little setting.\n\n#### Configuration\n\n```vim\n" Syntastic ------------------------------\n\n" show list of errors and warnings on the current file\nnmap <leader>e :Errors<CR>\n" turn to next or previous errors, after open errors list\nnmap <leader>n :lnext<CR>\nnmap <leader>p :lprevious<CR>\n" check also when just opened the file\nlet g:syntastic_check_on_open = 1\n" syntastic checker for javascript.\n" eslint is the only tool support JSX.\n" If you don\'t need write JSX, you can use jshint.\n" And eslint is slow, but not a hindrance\n" let g:syntastic_javascript_checkers = [\'jshint\']\nlet g:syntastic_javascript_checkers = [\'eslint\']\n" don\'t put icons on the sign column (it hides the vcs status icons of signify)\nlet g:syntastic_enable_signs = 0\n" custom icons (enable them if you use a patched font, and enable the previous \n" setting)\nlet g:syntastic_error_symbol = \'\xe2\x9c\x97\'\nlet g:syntastic_warning_symbol = \'\xe2\x9a\xa0\'\nlet g:syntastic_style_error_symbol = \'\xe2\x9c\x97\'\nlet g:syntastic_style_warning_symbol = \'\xe2\x9a\xa0\'\n```\n\n#### Features\n\nWhen you save files, it will check syntax automatically, and display syntax errors.\n\n#### Supported operations\n\n|   shortcut key        |    description             |\n|-----------------------|:--------------------------:|\n|       `\\e`            |     list syntax errors     |\n|       `\\n`            |     turn to next error     |\n|       `\\p`            |     turn to previous error |\n'
