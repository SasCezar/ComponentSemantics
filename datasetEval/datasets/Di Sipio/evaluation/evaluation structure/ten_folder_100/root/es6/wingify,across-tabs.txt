Safely enables [cross-origin](https://developer.mozilla.org/en-US/docs/Web/HTTP/Access_control_CORS) communication among different browser tabs. Uses `PostMessage` API for communication.\n2. Easy to hook custom callback at various levels. Eg: executing a custom method in Child\'s tab on receiving a message from Parent tab.\n3. Option to provide `data-tab-opener="name"` attribute on the target link/button(which opens up a new tab), so that it remains to disable until `Child` tab initiates a handshake and is received by the `Parent` tab\n4. Fully fledged API to get information regarding the tabs(Parent and Child tabs) and other communication-related methods.\n5. Exports in a UMD format i.e. library works everywhere.\n6. Only `~4 KB` gzipped.\n\n### Installation\n\n1. Via [npm](https://www.npmjs.com/)\n\n```\n$ npm install across-tabs\n```\n\n2. Via [bower](https://bower.io/)\n\n```\n$ bower install across-tabs\n```\n\n3. Via [cdnjs](https://cdnjs.com), using `<script>` tag\n\n```\n<script type="text/javascript" src="https://cdnjs.cloudflare.com/ajax/libs/across-tabs/1.0.0/across-tabs.min.js"></script>\n```\n\n**Note**: Use the desired version of the library in place of `1.0.0`. Checkout [cdnjs across-tabs](https://cdnjs.com/libraries/across-tabs).\n\n### Flow Diagram\n\n- Opener/Parent tab(`P`) opens up a new Child tab(`C`).\n- `C` initiates a handshake with the `P` tab by sending a `postMessage`.\n- `P` acknowledges the request and sends `C` it\'s identity i.e. `UUID` along with `P` information.\n- This sets up a communication channel between Parent and Child tab.\n- Now, `P` and `C` can share custom messages with each other.\n- Whenever `C` gets closed/refreshed, `P` is notified.\n- Whenever `P` is closed/refreshed, all children of `P` tab gets notified.\n\n<img src="https://raw.githubusercontent.com/wingify/across-tabs/master/parent-tab-communication.jpg" />\n\n**Explanation of diagram**\n\n- Parent(`P`) opens Child tab(`C1`) at `t=1`.\n- `c1a` - When `C1` initiates a handshake with the Parent.\n- `P1` - When `P` receives `C1` message.\n- `P2` - `P` acknowledges the request and sends the `C1` its identity.\n- `c1b` - When `C1` receives an acknowledgement message along with identity from `P`.\n\n---\n\nTotal Tabs Associated: **1** | Opened Tabs: **1** | Closed Tabs: **0**\n\n---\n\n- Parent(`P`) opens CHild tab(`C2`) at `t=10`.\n- `c2a` - When `C2` initiates a handshake with the Parent.\n- `P3` - When `P` receives `C2` message.\n- `P4` - `P` acknowledges the request and sends the `C2` its identity.\n- `c2b` - When `C2` receives acknowledgemnet message along with identity from `P`.\n\n---\n\nTotal Tabs Associated: **2** | Opened Tabs: **2** | Closed Tabs: **0**\n\n---\n\n- `c1c` - Tab `C1` closes.\n- `P5` - `P` is notified about the `C1`. Parent updates the list.\n\n---\n\nTotal Tabs Associated: **2** | Opened Tabs: **1** | Closed Tabs: **1**\n\n---\n\n- Parent(`P`) opens CHild tab(`C3`) at `t=25`.\n- `c3a` - When `C3` initiates a handshake with the Parent.\n- `P6` - When `P` receives `C3` message.\n- `c2c` - Tab `C2` sends a `custom` message.\n- `P7` - When `P` receives a message from tab `C2`. It processes it.\n- `P8` - `P` acknowledges the request and sends the `C3` its identity.\n- `c3b` - When `C3` receives acknowledgemnet message along with identity from `P`.\n\n---\n\nTotal Tabs Associated: **3** | Opened Tabs: **2** | Closed Tabs: **1**\n\n---\n\n- When Parent Tab `P` is closed, all the opened tabs are notified about it.\n\n### Usage\n\n##### Create an instance / reference before using.\n\n**Opener(Parent) tab**\n\n```\nvar config = {\n  onHandshakeCallback: function () { ... },\n  onPollingCallback: function () { ... },\n  onChildCommunication: function () { ... }\n}\nvar parent = new AcrossTabs.Parent(config);\n```\n\n`Constructor` accepts a configurable `Object` with the following keys.\n\n- `heartBeatInterval`: The time interval for polling tabs statuses\n- `removeClosedTabs`: Pass it as `true` to have only opened tabs in the list\n- `shouldInitImmediately`: Pass it as `false` to create an instance but initialize it later\n- `onHandshakeCallback`: Callback to be called when a successful connection has been established\n- `onChildCommunication`: Callback to be called when child sends message\n- `onPollingCallback`: Callback to be called every time a tab is polled for its status\n- `origin`: whitelist `origin` for securing `postMessage` communication. It will discard the malicious messages trying to trick the behavior. Eg. http://example.com\n- `parse`: parser used when parsing messages, defaults to `JSON.parse`\n- `stringify`: stringifier used when converting data into messages, defaults to `JSON.stringify`\n\n| Config Keys               | default        | accepts                            |\n| ------------------------- | -------------- | ---------------------------------- |\n| **heartBeatInterval**     | 500 msec       | A number representing milliseconds |\n| **removeClosedTabs**      | false          | Boolean                            |\n| **shouldInitImmediately** | true           | Boolean                            |\n| **onHandshakeCallback**   | Undefined      | Function as callback               |\n| **onChildCommunication**  | Undefined      | Function as callback               |\n| **onPollingCallback**     | Undefined      | Function as callback               |\n| **origin**                | \'\\*\'           | String(url)                        |\n| **parse**                 | JSON.parse     | Function                           |\n| **stringify**             | JSON.stringify | Function                           |\n\n**New(Child tab)**\n\n```\nvar config = {\n  onReady: onReady,\n  onInitialize: onInitialize,\n  isSiteInsideFrame: false, // dont set if not required\n  handshakeExpiryLimit: 4000, // msec\n  onParentDisconnect: onParentDisconnect,\n  onParentCommunication: onParentCommunication\n};\nvar child =  new AcrossTabs.Child(config);\n```\n\n`Constructor` accepts a configurable `Object` with the following keys.\n\n- `handshakeExpiryLimit`: Time to wait for getting an acknowledgement from Parent window for child\'s request\n- `onReady`: Callback to be invoked once child instance is ready\n- `onInitialize`: Callback when a child instance is actually initiated\n- `onParentDisconnect`: Callback to be invoked when Parent gets disconnected\n- `onParentCommunication`: Callback to be invoked whenever Parent communicates with the child tab\n- `isSiteInsideFrame`: If the library is loaded inside an iframe in the child tab, this needs to be set `true` for maintaining proper window/frame(s) references\n- `origin`: whitelist `origin` for securing `postMessage` communication. It will discard the malicious messages trying to trick the behavior. Eg. http://example.com\n- `parse`: parser used when parsing messages, defaults to `JSON.parse`\n- `stringify`: stringifier used when converting data into messages, defaults to `JSON.stringify`\n\n| Config Keys               | default        | accepts                                |\n| ------------------------- | -------------- | -------------------------------------- |\n| **handshakeExpiryLimit**  | 5000 msec      | A number representing milliseconds     |\n| **isSiteInsideFrame**     | null           | If child tab has actual site in a fram |\n| **onReady**               | Undefined      | Function as callback                   |\n| **onInitialize**          | Undefined      | Function as callback                   |\n| **onParentDisconnect**    | Undefined      | Function as callback                   |\n| **onParentCommunication** | Undefined      | Function as callback                   |\n| **origin**                | \'\\*\'           | String(url)                            |\n| **parse**                 | JSON.parse     | Function                               |\n| **stringify**             | JSON.stringify | Function                               |\n\n**Example** is included in the `example` folder. `Vanilla JS` and `Vue js` versions are there to test out.\n_Note:_ Run `npm install` if you wish to run `vuejs` example since the example needs the `vue-js` library to work.\n\n### API\n\nRefer [above section](#create-an-instance--reference-before-using) on how to create an instance before hitting API methods.\n\n##### Opener Tab(Parent) Methods\n\n- **`openNewTab`**\n\n  Saves `data` in specific `key` in sessionStorage. If the key is not provided, the library will warn.\n  Following types of JavaScript objects are supported:\n\n  | Parameter | Description                                   |\n  | --------- | --------------------------------------------- |\n  | config    | For opening a new tab i.e. URL and windowName |\n\n  ```\n    parent.openNewTab({url: \'http://example.com\', windowName: \'AcrossTab\'});\n  ```\n\n- **`enableElements`**\n\n  Links / buttons can be given a data attribute: `data-tab-opener="name"`. On clicking, the library finds elements with that attribute and adds `disabled="disabled"` attribute to that element. The `disabled` attribute is removed once the Child tab communicates back to the opener i.e. Parent. Though, this method is called internally but can also be invoked(may be within a timer) to be on a safer side.\n\n  ```\n    parent.enableElements();\n  ```\n\n- **`getAllTabs`**\n\n  Returns the list of all the tabs.\n  If `removeClosedTabs: true` is provided while instantiating, it will return only the opened tabs as closed tabs would be removed.\n\n  ```\n    parent.getAllTabs()\n  ```\n\n- **`getOpenedTabs`**\n\n  Returns the list of all `opened` tabs.\n\n  ```\n    parent.getOpenedTabs();\n  ```\n\n- **`getClosedTabs`**\n\n  Returns the list of all `closed` tabs.\n  If `removeClosedTabs: true` is provided while instantiating, it will return empty list as closed tabs would be removed.\n\n  ```\n    parent.getClosedTabs();\n  ```\n\n- **`closeAllTabs`**\n\n  Closes all the opened tabs.\n\n  ```\n    parent.closeAllTabs()\n  ```\n\n- **`closeTab`**\n\n  Closes a particular tab whose id is provided.\n\n  | Parameter | Description                |\n  | --------- | -------------------------- |\n  | id        | id of the tab to be closed |\n\n  ```\n    parent.closeTab(\'57cd47da-d98e-4a2d-814c-9b07cb51059c\');\n  ```\n\n- **`broadCastAll`**\n\n  Sends a same `message` to all the opened tabs.\n\n  | Parameter | Description   |\n  | --------- | ------------- |\n  | msg       | msg to be ent |\n\n  ```\n    parent.broadCastAll(\'Hello my dear Child! A greeting from Parent.\');\n  ```\n\n- **`broadCastTo`**\n\n  Sends a `message` to a particular opened tab.\n\n  | Parameter | Description                  |\n  | --------- | ---------------------------- |\n  | id        | id of the tab to send an msg |\n  | msg       | msg to be sent               |\n\n  ```\n    parent.broadCastTo(\'57cd47da-d98e-4a2d-814c-9b07cb51059c\', \'Hey! Can you run the script: worker.js? Thanks!\');\n  ```\n\n##### New Tab(Child) Methods\n\n- **`getTabInfo`**\n\n  Return id, name, and parentName of the child tab.\n\n  ```\n    child.getTabInfo();\n  ```\n\n- **`sendMessageToParent`**\n\n  Sends a `message` to the `Parent` tab.\n\n  | Parameter | Description    |\n  | --------- | -------------- |\n  | msg       | msg to be sent |\n\n  ```\n    child.sendMessageToParent(\'Hey Parent! I\\\'m done with my work.\');\n  ```\n\n### Browser Support\n\nTested in Chrome, Firefox and IE(versions >= 9).\n\n### Sites using across-tabs\n\n- app.vwo.com\n- _[Add your site]_ - See [Contributing](#contributing) section\n\n### Development Stack\n\n- Webpack based `src` compilation & bundling and `dist` generation.\n- ES6 as a source of writing code.\n- Exports in a [UMD](https://github.com/umdjs/umd) format so the library works everywhere.\n- Linting with [ESLint](http://eslint.org/).\n- ES6 test setup with [Karma](https://karma-runner.github.io/1.0/index.html), [Jasmine](https://jasmine.github.io/2.0/introduction.html) and [isparta](https://github.com/deepsweet/isparta-loader).\n- Test coverage with [Istanbul]() and [Coveralls](https://coveralls.io/).\n\n### Process\n\n```\nES6 source files\n       |\n       |\n    webpack\n       |\n       +--- babel, eslint\n       |\n       o--- tests and covergae\n       |\n  ready to use\n     library\n  in UMD format\n```\n\n### Scripts\n\n- `npm run build` - produces production version(minified) of the library under the `dist` folder\n- `npm run dev` - produces development version(un minified) of the library and runs a watcher to detect file changes.\n- `npm run test` - well ... it runs the tests :)\n\n### Contributing\n\n1.
