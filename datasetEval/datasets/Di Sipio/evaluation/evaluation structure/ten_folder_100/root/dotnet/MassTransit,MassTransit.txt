| 2.0 |\n\n## Getting started with MassTransit\n\nIn order to get started with MassTransit, you can have a look at the\ndocumentation, which is located at [http://masstransit-project.com/MassTransit](http://masstransit-project.com/MassTransit).\n\n### Simplest possible thing:\n\n`install-package MassTransit.RabbitMq`\n\nand then:\n\n```csharp\n// Message Definition\nclass MyMessage\n{\n\tpublic string Value { get; set; }\n}\n\n// Code Snippet for Console Application\nasync Task Main()\n{\n\tvar bus = Bus.Factory.CreateUsingRabbitMq(sbc =>\n\t{\n\t\tvar host = sbc.Host(new Uri("rabbitmq://localhost:/"), h =>\n\t\t{\n\t\t\th.Username("guest");\n\t\t\th.Password("guest");\n\t\t});\n\n\t\tsbc.ReceiveEndpoint(host, "my_queue", endpoint =>\n\t\t{\n\t\t\tendpoint.Handler<MyMessage>(async context =>\n\t\t\t{\n\t\t\t\tawait Console.Out.WriteLineAsync($"Received: {context.Message.Value}");\n\t\t\t});\n\t\t});\n\t});\n\n\tawait bus.StartAsync();\n\n\tawait bus.Publish(new MyMessage { Value = "Hello, World." });\n\n\tConsole.ReadLine();\n\n\tawait bus.StopAsync();\n}\n```\n\nYou will also need to set up RabbitMQ; \n\n 1. **Install Erlang** using the [installer](http://www.erlang.org/download.html). (Next -> Next ...)\n 2. **Install RabbitMQ** using the [installer](http://www.rabbitmq.com/download.html). (Next -> Next ...) You now have a RabbitMQ broker (look in `services.msc` for it) that you can [log into](http://localhost:15672/#/) using `guest`, `guest`. You can see message rates, routings and active consumers using this interface. \n \n**Please note** \n\nYou need to add the management interface before you can login. First, from an elevated command prompt, change directory to the sbin folder within the RabbitMQ Server installation directory e.g. `%PROGRAMFILES%\\RabbitMQ Server\\rabbitmq_server_3.5.3\\sbin\\`\n\nNext, run the following command to enable the rabbitmq management plugin:\n\n`rabbitmq-plugins.bat enable rabbitmq_management`\n\n### Downloads\n\nDownload from NuGet \'MassTransit\' [Search NuGet for MassTransit](http://nuget.org/packages?q=masstransit)\n\nDownload the continuously integrated Nuget packages from [AppVeyor](https://ci.appveyor.com/project/phatboyg/masstransit/build/artifacts).\n\n### Supported transports\n\nWe support RabbitMQ and Azure Service Bus message brokers.\n\n## Mailing list\n\n[MassTransit Discuss](http://groups.google.com/group/masstransit-discuss)\n\n## Gitter Chat \n\nWhile attendance is pretty light, there is a Gitter chat room available:\n\n[![Join the chat at https://gitter.im/MassTransit/MassTransit](https://badges.gitter.im/Join%20Chat.svg)](https://gitter.im/MassTransit/MassTransit?utm_source=badge&utm_medium=badge&utm_campaign=pr-badge&utm_content=badge)\n\n## GitHub Issues\n\n**Pay attention**\n\nPlease do not open an issue on github, unless you have spotted an actual bug in MassTransit. \nIf you are unsure, ask on the mailing list, and if we confirm it\'s a bug, we\'ll ask you to create the issue. \nIssues are not the place for questions, and they\'ll likely be closed.\n\nThis policy is in place to avoid bugs being drowned out in a pile of sensible suggestions for future \nenhancements and calls for help from people who forget to check back if they get it and so on.\n\n## Building from Source\n\n 1.
