b'Lando\n=====\n\n![screenshot](https://raw.githubusercontent.com/lando/lando/master/docs/.vuepress/public/images/lando-screenshot.png)\n\nLando is for developers who want to:\n\n* Quickly specify and painlessly spin up the [services](http://docs.devwithlando.io/config/services.html) and [tooling](http://docs.devwithlando.io/config/tooling.html) needed to develop their projects.\n* Ship these local development dependencies in a per-project, lives-in-git [config file](http://docs.devwithlando.io/config/lando.html)\n* Automate complex [build steps](http://docs.devwithlando.io/config/services.html#build-steps), testing setups, deployments or other [repeated-more-than-once workflows](http://docs.devwithlando.io/config/events.html)\n* Avoid the built-in-masochism of directly using `docker` or `docker-compose`\n\nIt\'s a free, open source, cross-platform, **local** development environment and DevOps tool built on [Docker](http://docker.com) container technology and developed by [Tandem](http://thinktandem.io). Designed to work with most major languages, frameworks and services, Lando provides an easy way for developers of all skill levels to specify simple or complex requirements for their projects, and then quickly get to work on them.\n\n**Think of it as your local development dependency management and automation tool.** Note that while you _can_ run Lando in production it is _highly_ discouraged, not recommended and 100% not supported!\n\nWhat is it good for?\n--------------------\n\n*   Mimicking your production environment locally\n*   Standardizing your teams dev environments and tooling on OSX, Windows and Linux\n*   Integrating with hosting providers like [Pantheon](https://pantheon.io)\n*   Customizing or extending tooling, deployment options and basically any other functionality\n*   Running CI tests locally, running local tests in CI\n*   Using a single local development environment for *all* your projects\n*   Freeing yourself from the tyranny of inferior local development products\n\nWait, doesn\'t Docker Compose do this?\n-------------------------------------\n\nYes! Well, sort of. You can think of Lando as both an abstraction layer and superset of Docker Compose as well as a Docker Compose utility.\n\n**As an abstraction layer** Lando vastly reduces the complexity of spinning up containers by exposing only the most relevant config for a given "service" and setting "sane defaults". Lando also provides "recipes" which are common combinations of services and their tooling that satisfy a given development use case eg Drupal, Python, Laravel, Dotnet, etc.\n\n**As a superset** Lando provides ways for developers to run complex commands, build steps and automation on their services without the hassle of custom Dockerfiles or long "docker exec" commands. Think `lando yarn add express`. Think clear my applications cache after I import a database. Think install this `php-extension` before my appserver starts and then `composer install` after it does.\n\n**As a utility** Lando handles some of the more arduous setup required for a *good* Docker Compose setup eg proxying, nice urls, cross-application networking (think Vue.js frontend talking to separate Laravel backend), host-container file permission handling, file sharing, per-container SSL certificate handling, ssh-key handling, etc.\n\n**And!** If you don\'t like the default choices we\'ve made all of the above is highly configurable down to the Docker Compose level itself!\n\nBegin by
