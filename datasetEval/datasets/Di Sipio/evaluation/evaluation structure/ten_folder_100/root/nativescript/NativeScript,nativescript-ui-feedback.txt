b'# Welcome to Progress NativeScript UI\n[Progress NativeScript UI](https://www.nativescript.org/ui-for-nativescript) is a set of free* components that enable implementing rich-ui applications for iOS and Android by using NativeScript. Progress NativeScript UI is built on top of natively implemented components targeting iOS and Android.\n\n> NOTE: *Although the components are free, they are not open-source and their code is proprietry. [Read the components\' license for details](https://github.com/NativeScript/nativescript-ui-feedback/blob/master/LICENSE.md).\n\n# Contributing to NativeScript UI\n\n## Issue Reports\n> Please take a minute to read our [NativeScript Code of Conduct](https://github.com/NativeScript/codeofconduct) before proceeding with posting issues or discussing. The purpose of this guide is to make communication and cooperation within our forums a pleasure for you and the other members.\n\nIn order to provide a fast response and issue resolution to all submitted issues we **require** that all issue reports are done with our predefined "issue template" that is automatically populated in the issue submission form. **Please fill** that template and **DO NOT DELETE IT**, that information is necessary and helpful to us in order to provide you with a timely response. If the "issue template" information is not provided **your issue may be closed immediately** without a response.\n\n## Interested in Contributing to the Codebase?\nYou have a feature request that is not part of our immediate plans? Or there is an urgent bug that you want to fix right away? Fear not, get in touch with us at nativescriptplugins@progress.com and tell us about your ideas. Contributions to the NativeScript UI are possible after signing a mutual NDA.\n\n# Overview\n## Distribution\n**Progress NativeScript UI** is distributed via npm. You may download the package that contains the component that you want to use from the following locations:\n\n- **Calendar**: https://www.npmjs.com/package/nativescript-ui-calendar\n- **Chart**: https://www.npmjs.com/package/nativescript-ui-chart\n- **ListView**: https://www.npmjs.com/package/nativescript-ui-listview\n- **DataForm**: https://www.npmjs.com/package/nativescript-ui-dataform\n- **SideDrawer**: https://www.npmjs.com/package/nativescript-ui-sidedrawer\n- **Gauge**: https://www.npmjs.com/package/nativescript-ui-gauge\n- **AutoComplete**: https://www.npmjs.com/package/nativescript-ui-autocomplete\n\n## Getting started\nFor more information on how to use Progress NativeScript UI, please visit the documentation pages here:\n- [https://docs.nativescript.org/ui/professional-ui-components/overview](https://docs.nativescript.org/ui/professional-ui-components/overview)\n- [https://docs.nativescript.org/angular/ui/professional-ui-components/overview](https://docs.nativescript.org/angular/ui/professional-ui-components/overview)\n- [https://docs.nativescript.org/vuejs/ns-ui/getting-started](https://docs.nativescript.org/vuejs/ns-ui/getting-started)\n\nYou can also take a look at the sample applications which are available on GitHub and contain simple scenarios covering most features of the components in Progress NativeScript UI:\n- [https://github.com/NativeScript/nativescript-ui-samples](https://github.com/NativeScript/nativescript-ui-samples)\n- [https://github.com/NativeScript/nativescript-ui-samples-angular](https://github.com/NativeScript/nativescript-ui-samples-angular)\n- [https://github.com/NativeScript/nativescript-ui-samples-vue](https://github.com/NativeScript/nativescript-ui-samples-vue)\n\n## Feedback\nProgress NativeScript UI is constantly improved and extended so we\'re looking for your feedback.
