of [Angular Reactive Forms](https://angular.io/guide/reactive-forms)\n- \xf0\x9f\x98\xb1 Drag and Drop Editor (coming \xf0\x9f\x94\x9c)\n\n**Supported UI libs**\n\n| UI get started with Ngx Formly. Also check out our [demos](https://formly.dev) for further examples.\n\n#### 1. Install @angular/forms and @ngx-formly/core packages:\n```bash\n  npm install @angular/forms @ngx-formly/core --save\n```\n\n#### 2. Choose and install your UI (pre-defined types/templates) package:\n\n| UI                                                | Npm package name            | NgModule                    |\n| ------------------------------------------------- | --------------------------- | --------------------------- |\n| [Bootstrap](https://getbootstrap.com)             | `@ngx-formly/bootstrap`     | `FormlyBootstrapModule`     |\n| [Material2](https://github.com/angular/material2) | `@ngx-formly/material`      | `FormlyMaterialModule`      |\n| [Ionic](https://ionicframework.com)               | `@ngx-formly/ionic`         | `FormlyIonicModule`         |\n| [PrimeNG](http://primefaces.org/primeng/#/)       | `@ngx-formly/primeng`       | `FormlyPrimeNGModule`       |\n| [Kendo](http://www.telerik.com/kendo-angular-ui)  | `@ngx-formly/kendo`         | `FormlyKendoModule`         |\n| [NativeScript](https://www.nativescript.org)      | `@ngx-formly/nativescript`  | `FormlyNativescriptModule`  |\n\n```bash\n  npm install @ngx-formly/<package-name> --save\n```\n\n#### 3. Import the `FormlyModule` and UI (pre-defined types/templates):\n\n```typescript\nimport {NgModule} from \'@angular/core\';\nimport {ReactiveFormsModule} from \'@angular/forms\';\nimport {FormlyModule} from \'@ngx-formly/core\';\n\n/**\n * - Bootstrap:     import {FormlyBootstrapModule} from \'@ngx-formly/bootstrap\';\n * - Material2:     import {FormlyMaterialModule} from \'@ngx-formly/material\';\n * - Ionic:         import {FormlyIonicModule} from \'@ngx-formly/ionic\'\n * - PrimeNG:       import {FormlyPrimeNGModule} from \'@ngx-formly/primeng\';\n * - Kendo:         import {FormlyKendoModule} from \'@ngx-formly/kendo\';\n * - NativeScript:  import {FormlyNativescriptModule} from \'@ngx-formly/nativescript\';\n */\nimport {FormlyBootstrapModule} from \'@ngx-formly/bootstrap\';\n\n@NgModule({\n  imports: [\n    ...,\n    ReactiveFormsModule,\n    FormlyModule.forRoot(),\n\n    /**\n     * - Bootstrap:    FormlyBootstrapModule\n     * - Material2:    FormlyMaterialModule\n     * - Ionic:        FormlyIonicModule\n     * - PrimeNG:      FormlyPrimeNGModule\n     * - Kendo:        FormlyKendoModule\n     * - NativeScript: FormlyNativescriptModule\n     */\n    FormlyBootstrapModule,\n  ],\n})\nexport class AppModule {}\n```\n\n#### 4. Define the form config in your component:\n\n```typescript\nimport {Component} from \'@angular/core\';\nimport {FormGroup} from \'@angular/forms\';\nimport {FormlyFieldConfig} from \'@ngx-formly/core\';\n\n@Component({\n  selector: \'app\',\n  template: `\n    <form [formGroup]="form" (ngSubmit)="submit(model)">\n      <formly-form [form]="form" [fields]="fields" [model]="model"></formly-form>\n      <button type="submit" class="btn btn-default">Submit</button>\n    </form>\n  `,\n})\nexport class AppComponent {\n  form = new FormGroup({});\n  model = { email: \'email@gmail.com\' };\n  fields: FormlyFieldConfig[] = [{\n    key: \'email\',\n    type: \'input\',\n    templateOptions: {\n      label: \'Email address\',\n      placeholder: \'Enter email\',\n      required: true,\n    }\n  }];\n\n  submit(model) {\n    console.log(model);\n  }\n}\n```\n\nFrom there, it\'s just JavaScript. Allowing for DRY, maintainable, reusable forms.\n\n## Using with Angular-CLI\n\n`@ngx-formly/schematics` provides CLI commands for setting up a project and _eventually_ generating other features (e.g. wrapper components).\nBuilt on top of Schematics, this tool integrates with the Angular CLI.\n\n### Installation\n\nInstall `@ngx-formly/schematics` from npm:\n\n`npm install @ngx-formly/schematics --save-dev`\n\n### Default Schematics Collection\n\nTo use `@ngx-formly/schematics` as the default collection in your Angular CLI project,\nadd it to your `angular.json`:\n\n```bash\nng config cli.defaultCollection @ngx-formly/schematics\n```\n\nThe [collection schema](src/schematics/src/collection.json) defines the available schematics to run.\n\nThe `@ngx-formly/schematics` extend the default `@schematics/angular` collection and so all existing Angular CLI commands are available.\nIf you want to set defaults for schematics such as generating components with scss file, you must change the schematics package name from `@schematics/angular` to `@ngx-formly/schematics` in `angular.json`:\n\n```json\n"schematics": {\n  "@ngx-formly/schematics:component": {\n    "styleext": "scss"\n  }\n}\n```\n\n### Quick Start with Schematics\n\n#### Install @angular/forms and @ngx-formly/core packages and choose a UI theme to install:\n```bash\n  ng add @ngx-formly/schematics --ui-theme=material\n```\n\n## Roadmap\n\nSee the [issues labeled enhancement](https://github.com/ngx-formly/ngx-formly/labels/enhancement)\n\n## Credits\n\n* [Abdellatif Ait boudad](https://github.com/aitboudad)\n* [Mohammed Zama Khan](https://twitter.com/mohamedzamakhan)\n* [Kent C.
