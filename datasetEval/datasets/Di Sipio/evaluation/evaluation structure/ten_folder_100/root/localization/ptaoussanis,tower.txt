Unfortunately, they can be... cumbersome. We can do much better in Clojure.\n\nTower\'s an attempt to present a **simple, idiomatic internationalization and localization** story for Clojure. It wraps standard Java functionality where possible - with warm, fuzzy, functional love.\n\n## Library status: EOL but stable\n\n**Last updated:** Jan 2016\n\nTower\'s latest beta (`3.1.0-beta4`) is **stable and quite useable in production**, but is likely the last major release for the library (modulo unexpected bug fixes). **Future development work** is going to be focused on:\n\n  * [Tempura] - Clojure/Script **translations** API\n  * To be determined - Clojure/Script **localization** API\n\n##### Why the EOL?\n\nTower\'s quite useable as it is but was initially designed before things like Reactjs (indeed ClojureScript) were available. The API does the job, but it\'s not what I\'d write if I wrote it from scratch today w/o the historical baggage.\n\nRather than try to shoehorn a significant set of potentially-breaking changes into Tower (pressuring stable users to migrate), it seemed logical to instead focus future work on alternative lib/s that could be **optionally migrated to in stages**.\n\nAs I find time to publish these libs, I\'ll make a best effort to also document the procedure necessary for **migrating from Tower** for those users that want to do that.\n\nI apologise for the stress/trouble that this might cause, but hope that the present solution will turn out to be relatively painless in most cases. Long term, I think that the benefits will be worth it.\n\n\\- [Peter Taoussanis]\n\n## Features\n * Small **all-Clojure** library\n * Simple wrappers for standard Java **localization features**\n * **Simple, map-based** translation dictionary format. No XML or resource files!\n * Rails-like, pure-Clojure **translation function** with ClojureScript support\n * Automatic dev-mode **dictionary reloading** for rapid REPL development\n * Seamless **markdown support** for translators\n * **Ring middleware**\n * TODO: export/import to allow use with **industry-standard tools for translators**\n\n## Getting started\n\nAdd the necessary dependency to your project:\n\n```clojure\n[com.taoensso/tower "3.0.2"]\n```\n\nAnd setup your namespace imports:\n\n```clojure\n(ns my-clj-ns ; Clojure namespace\n  (:require [taoensso.tower :as tower :refer (with-tscope)]))\n\n;; Requires v3.1+\n(ns my-cljs-ns ; ClojureScript namespace\n  (:require [taoensso.tower :as tower :refer-macros (with-tscope)]))\n```\n\n### Translation\n\nThe `make-t` fn handles translations. You give it a config map which includes your dictionary, and get back a `(fn [locale-or-locales k-or-ks & fmt-args])`:\n\n```clojure\n(def my-tconfig\n  {:dictionary ; Map or named resource containing map\n   {:en   {:example {:foo         ":en :example/foo text"\n                     :foo_comment "Hello translator, please do x"\n                     :bar {:baz ":en :example.bar/baz text"}\n                     :greeting "Hello %s, how are you?"\n                     :inline-markdown "<tag>**strong**</tag>"\n                     :block-markdown* "<tag>**strong**</tag>"\n                     :with-exclaim!   "<tag>**strong**</tag>"\n                     :with-arguments  "Num %d = %s"\n                     :greeting-alias :example/greeting\n                     :baz-alias      :example.bar/baz}\n           :missing  "|Missing translation: [%1$s %2$s %3$s]|"}\n    :en-US {:example {:foo ":en-US :example/foo text"}}\n    :de    {:example {:foo ":de :example/foo text"}}\n    :ja "test_ja.clj" ; Import locale\'s map from external resource\n    }\n   :dev-mode? true ; Set to true for auto dictionary reloading\n   :fallback-locale :de})\n\n(def t (tower/make-t my-tconfig)) ; Create translation fn\n\n(t :en-US :example/foo) => ":en-US :example/foo text"\n(t :en    :example/foo) => ":en :example/foo text"\n(t :en    :example/greeting "Steve") => "Hello Steve, how are you?"\n\n;;; Translation strings are escaped and parsed as inline or block Markdown:\n(t :en :example/inline-markdown) => "&lt;tag&gt;<strong>strong</strong>&lt;/tag&gt;"\n(t :en :example/block-markdown)  => "<p>&lt;tag&gt;<strong>strong</strong>&lt;/tag&gt;</p>" ; Notice no "*" suffix here, only in dictionary map\n(t :en :example/with-exclaim)    => "<tag>**strong**</tag>" ; Notice no "!" suffix here, only in dictionary map\n(t :en :example/with-arguments 42 "forty two") =>   "Num 42 = forty two"\n```\n\nIt\'s simple to get started, but there\'s a number of advanced features for if/when you need them:\n\n#### Loading dictionaries from disk/resources\n\nJust use a string for the `:dictionary` and/or any locale value(s) in your config map. Be sure to check that the appropriate files are available on your classpath or one of Leiningen\'s resource paths (e.g. `resources/`).\n\n#### Reloading dictionaries on modification**\n\nEnable the `:dev-mode?` option and you\'re good to go!\n\n#### Scoping translations**\n\nUse `with-tscope` if you\'re calling `t` repeatedly within a specific translation-namespace context:\n```clojure\n(with-tscope :example\n  [(t :en :foo)\n   (t :en :bar/baz)]) => [":en :example/foo text" ":en :example.bar/baz text"]\n```\n\n#### Missing translations**\n\nThese are handled gracefully. `(t :en-US :example/foo)` will search for a translation as follows:\n\n 1. `:example/foo` in the `:en-US` locale.\n 2. `:example/foo` in the `:en` locale.\n 3. `:example/foo` in the dictionary\'s fallback locale.\n 4. `:missing` in any of the above locales.\n\nYou can also specify fallback keys that\'ll be tried before other locales. `(t :en-US [:example/foo :example/bar]))` searches:\n\n 1. `:example/foo` in the `:en-US` locale.\n 2. `:example/bar` in the `:en-US` locale.\n 3. `:example/foo` in the `:en` locale.\n 4. `:example/bar` in the `:en` locale.\n 5. `:example/foo` in the fallback locale.\n 6. `:example/bar` in the fallback locale.\n 7. `:missing` in any of the above locales.\n\nAnd even fallback locales. `(t [:fr-FR :en-US] :example/foo)` searches:\n\n 1. `:example/foo` in the `:fr-FR` locale.\n 2. `:example/foo` in the `:fr` locale.\n 3. `:example/foo` in the `:en-US` locale.\n 4. `:example/foo` in the `:en` locale.\n 5. `:example/foo` in the fallback locale.\n 6. `:missing` in any of the above locales.\n\nIn all cases, translation requests are logged upon fallback to fallback locale or :missing key.\n\n#### ClojureScript translations (early support, v3.1+)\n\n```clojure\n(def ^:private tconfig\n  {:fallback-locale :en\n   ;; Inlined (macro) dict => this ns needs rebuild for dict changes to reflect.\n   ;; (dictionary .clj file can be placed in project\'s `/resources` dir):\n   :compiled-dictionary (tower-macros/dict-compile* "my-dict.clj")})\n\n(def t (tower/make-t tconfig)) ; Create translation fn\n\n(t :en-US :example/foo) => ":en-US :example/foo text"\n```\n\nThere\'s two notable differences from JVM translations:\n\n 1. The dictionary is provided in a _pre-compiled_ form so that it can be inlined directly into your Cljs.\n 2. Since we lack a locale-aware Cljs `format` fn, your translations _cannot_ use JVM locale formatting patterns.\n\nThe API is otherwise exactly the same, including support for all decorators.\n\n##### Use with React (Reagent/Om/etc.)\n\nReact presents a bit of a challenge to translations since it **automatically escapes all text content** as a security measure.\n\nThis has two important implications for use with Tower\'s translations:\n\n 1. Content intended to allow _translator-controlled inline styles_ needs to provided to React with the `dangerouslySetInnerHTML` property.\n 2. All other content should get a `:<key>!`-style translation to prevent double escaping (Tower already escapes translations not marked with an exlamation point).\n\n### Localization\n\nCheck out `fmt`, `parse`, `lsort`, `fmt-str`, `fmt-msg`:\n\n```clojure\n(tower/fmt   :en-ZA 200       :currency) => "R 200.00"\n(tower/fmt   :en-US 200       :currency) => "$200.00"\n(tower/parse :en-US "$200.00" :currency) => 200\n\n(tower/fmt :de-DE 2000.1 :number)               => "2.000,1"\n(tower/fmt :de-DE (java.util.Date.))            => "12.06.2012"\n(tower/fmt :de-DE (java.util.Date.) :date-long) => "12. Juni 2012"\n(tower/fmt :de-DE (java.util.Date.) :dt-long)   => "12 giugno 2012 16.48.01 ICT"\n\n(tower/lsort :pl ["Warsaw" "Krak\xc3\xb3w" "\xc5\x81\xc3\xb3d\xc5\xba" "Wroc\xc5\x82aw" "Pozna\xc5\x84"])\n=> ("Krak\xc3\xb3w" "\xc5\x81\xc3\xb3d\xc5\xba" "Pozna\xc5\x84" "Warsaw" "Wroc\xc5\x82aw")\n\n(mapv #(tower/fmt-msg :de "{0,choice,0#no cats|1#one cat|1<{0,number} cats}" %)\n        (range 5))\n=> ["no cats" "one cat" "2 cats" "3 cats" "4 cats"]\n```\n\nIt\'s that simple.
