Run `git clone https://github.com/jeremykenedy/laravel-auth.git laravel-auth`\n2. Create a MySQL database for the project\n    * ```mysql -u root -p```, if using Vagrant: ```mysql -u homestead -psecret```\n    * ```create database laravelAuth;```\n    * ```\\q```\n3. From the projects root run `cp .env.example .env`\n4. Configure your `.env` file\n5. Run `composer update` from the projects root folder\n6. From the projects root folder run:\n```\nphp artisan vendor:publish --tag=laravelroles &&\nphp artisan vendor:publish --tag=laravel2step\n```\n7. From the projects root folder run `sudo chmod -R 755 ../laravel-auth`\n8. From the projects root folder run `php artisan key:generate`\n9. From the projects root folder run `php artisan migrate`\n10. From the projects root folder run `composer dump-autoload`\n11. From the projects root folder run `php artisan db:seed`\n12. Compile the front end assets with [npm steps](#using-npm) or [yarn steps](#using-yarn).\n\n#### Build the Front End Assets with Mix\n##### Using Yarn:\n1. From the projects root folder run `yarn install`\n2. From the projects root folder run `yarn run dev` or `yarn run production`\n  * You can watch assets with `yarn run watch`\n\n#### Optionally Build Cache\n1. From the projects root folder run `php artisan config:cache`\n\n###### And thats it with the caveat of setting up and configuring your development environment. I recommend [Laravel Homestead](https://laravel.com/docs/6.x/homestead)\n\n### Seeds\n##### Seeded Roles\n  * Unverified - Level 0\n  * User  - Level 1\n  * Administrator - Level 5\n\n##### Seeded Permissions\n  * view.users\n  * create.users\n  * edit.users\n  * delete.users\n\n##### Seeded Users\n\n|Email|Password|Access|\n|:------------|:------------|:------------|\n|user@user.com|password|User Access|\n|admin@admin.com|password|Admin Access|\n\n##### Themes Seed List\n  * [ThemesTableSeeder](https://github.com/jeremykenedy/laravel-auth/blob/master/database/seeds/ThemesTableSeeder.php)\n  * NOTE: A lot of themes render incorrectly on Bootstrap 4 since their core was built to override Bootstrap 4. From the projects root folder run ```composer update```\n  4. Add the service provider to ```/config/services.php```\n     * Example:\n\n     ```\n        \'twitch\' => [\n            \'client_id\'   => env(\'TWITCH_KEY\'),\n            \'client_secret\' => env(\'TWITCH_SECRET\'),\n            \'redirect\'    => env(\'TWITCH_REDIRECT_URI\'),\n        ],\n     ```\n\n  5. Add the API credentials to ``` /.env  ```\n     * Example:\n\n      ```\n         TWITCH_KEY=YOURKEYHERE\n         TWITCH_SECRET=YOURSECRETHERE\n         TWITCH_REDIRECT_URI=http://YOURWEBSITEURL.COM/social/handle/twitch\n      ```\n\n  6. Add the social media login link:\n      * Example:\n      In file ```/resources/views/auth/login.blade.php``` add ONE of the following:\n         * Conventional HTML:\n        ```\n        <a href="{{ route(\'social.redirect\', [\'provider\' => \'twitch\']) }}" class="btn btn-lg btn-primary btn-block twitch">Twitch</a>\n        ```\n         * Use Laravel HTML Facade with [Laravel Collective](https://laravelcollective.com/):\n\n        ```\n        {!!
