{badge_netlify}\n\nThe Gradle plugin `de.fayard.buildSrcVersions` goal is to make it as painless as possible to upgrade your project to the latest and greatest version of everything.\n\nIt does that by extracting all your dependencies, searching for available dependencies updates\nand generating Kotlin code, Groovy code or Gradle properties.\n\nThe plugin was called before `de.fayard.buildSrcVersions`, which still exists and is currently identical.\n\nhttps://github.com/jmfayard/buildSrcVersions/issues/104[The goal in the future is to focus on :refreshVersions]\n\n== Installation\n\nEdit your root `build.gradle(.kts)` file\n\nWe recommend to use the `plugins` block:\n\n[source,kotlin,subs=attributes]\n----\nbuildscript {\n    //...\n}\nplugins {\n  id("de.fayard.buildSrcVersions") version "{plugin_version}"\n}\n// Don\'t put any code before the buildscripts {} and plugins {} block\n----\n\nUpdating Gradle is usually a good idea. You get less bugs, more features and more build speed, and it\'s as simple as this:\n\n`$ ./gradlew wrapper --gradle-version {gradle_version}`\n\n\n== The task `:buildSrcVersions`\n\nThe task `:buildSrcVersions` is well suited for complex multi-module Gradle projects: Android, Kotlin multi-platform, ...\n\nIt generates this kind of code inside the Gradle `buildSrc` module.\n\nhttps://github.com/jmfayard/buildSrcVersions/issues/88[Read the friendly documentation]\n\n[[img-carbon-Libs-Version]]\nimage::carbon-Libs-Versions.png[]\n\n\n== Requirements\n\n* Kotlin, JVM and Android projects.\n* Gradle 4.3+, 5+, 6+\n* The IDE integration works\n** for both `build.gradle` (Groovy) and `build.gradle.kts` (Kotlin) files.\n** in IntelliJ IDEA 2018.03 and better\n** in Android Studio 3.5 and better.\n\n== Articles\n\nNot only code, but also good documentation can help you upgrade an old project to the latest and greatest.\n\nI have a series of articles called `Built with Gradle`:\n\n- https://dev.to/jmfayard/my-project-is-open-to-kotlin-contributors-m4b[#hacktoberfest : project is open to contributors!]\n- https://dev.to/jmfayard/helping-out-with-gradle-builds-for-hacktoberfest-1c1c[Helping out with Gradle builds for hacktoberfest]\n- https://dev.to/jmfayard/the-one-gradle-trick-that-supersedes-all-the-others-5bpg[Use the Build Scan ! The plugin should do nothing unless it\'s explicitly called.\n- **Be Lazy.** Use code generation to avoid writing tedious code manually. Find available updates automatically instead of googling them. The plugin should do the right thing by default, no configuration should be mandatory. Leverage IDE tooling support (auto-completion, ...).\n- **Be Humble.** Render to the programmer the things that are the programmer\'s. This include using the generated code and updating the dependencies.\n\n\n== FAQ\n\nQuestions? {issues}[Look at the existing issues], then ask your own.\n\n\n[[FAQ]]\nimage::{image_faq}[title="FAQ",alt="IDE Integration", width="600", link="{issues}"]\n\n\n\n== Changelog\n\nSee {master}/CHANGELOG.md[CHANGELOG.md]\n\nIf you migrate from an older version, two configuration options have been removed: `rejectedVersionKeywords` and `useFdqnFor`.\n\nSee https://github.com/jmfayard/buildSrcVersions/issues/53:[#53 Plugin configuration]\n\n\n== Contributing\n\n- This project is licensed under the MIT License.
