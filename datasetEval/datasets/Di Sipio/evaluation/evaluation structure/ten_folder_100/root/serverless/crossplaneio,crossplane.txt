With Crossplane you\ncan manage your applications and infrastructure across clouds, regions, and\nclusters. Crossplane is composable and Kubernetes native. Deploy it to an\nexisting Kubernetes cluster to manage cloud infrastructure like databases,\nbuckets, and caches using `kubectl`. Deploy it standalone to manage Kubernetes\nclusters, the workloads running on them, and the cloud infrastructure those\nworkloads depend upon.\n\n## Architecture\n\n![Architecture diagram](docs/media/crossplane-overview.png)\n\nCrossplane builds on the Kubernetes control plane. It is composed of [Stacks] -\neasy to install packages of Kubernetes [custom resources and controllers] that\nextend Crossplane with new functionality. A stack can extend Crossplane with the\nability to manage the infrastructure of a new cloud provider, or to deploy and\nmanage a new cloud-native application. This enables Crossplane users to:\n\n1. Install the **infrastructure stacks** for their desired clouds to enable\n   on-demand, **cloud-portable provisioning of infrastructure** services like\n   databases, caches, and Kubernetes clusters.\n1. Define **portable application workloads**, including their cloud\n   infrastructure dependencies, and schedule them to the most appropriate\n   infrastructure across any cloud.\n1. Package their workloads as an **application stack** that others may easily\n   deploy.\n\nThe full vision and architecture of the Crossplane project is described in our\n[architecture document].\n\n## Get Started\n\nTake a look at the [getting started] guide to learn how to install Crossplane,\ninstall a stack, and provision cloud infrastructure using `kubectl`. Refer to\nthe Crossplane [documentation] for comprehensive guides to using Crossplane,\nincluding deploying an application packaged as a stack.\n\n## Infrastructure Stacks\n\nThe following infrastructure stacks are currently available for use with\nCrossplane.
