b'<img alt="Exoframe" src="./logo/png/exo_blue.png" width="300">\n\n> Simple Docker deployment tool\n\n[![Build Status](https://travis-ci.org/exoframejs/exoframe.svg?branch=master)](https://travis-ci.org/exoframejs/exoframe)\n[![Coverage Status](https://coveralls.io/repos/github/exoframejs/exoframe/badge.svg?branch=master)](https://coveralls.io/github/exoframejs/exoframe?branch=master)\n[![npm](https://img.shields.io/npm/v/exoframe.svg)](https://www.npmjs.com/package/exoframe)\n[![license](https://img.shields.io/github/license/mashape/apistatus.svg?maxAge=2592000)](https://opensource.org/licenses/MIT)\n\nExoframe is a self-hosted tool that allows simple one-command deployments using Docker.\n\n## Features\n\n- One-command project deployment\n- SSH key based auth\n- Rolling updates\n- Deploy tokens (e.g. to deploy from CI)\n- Deploy secrets (e.g. to hide sensitive env vars)\n- Automated HTTPS setup via letsencrypt \\*\n- Automated gzip compression \\*\n- Rate-limit support \\*\n- Basic HTTP Auth support \\*\n- Simple access to the logs of deployments\n- Docker-compose support\n- Simple function deployments\n- Multiple deployment endpoints and multi-user support\n- Simple update procedure for client, server and Traefik\n- Optional automatic subdomain assignment (i.e. every deployment gets its own subdomain)\n- Swarm mode deployments\n- Complex recipes support (i.e. deploy complex systems in one command)\n\n\\* Feature provided by [Traefik](https://traefik.io/)\n\n## Demo\n\n[![asciicast](https://asciinema.org/a/129255.png)](https://asciinema.org/a/129255)\n\n## Installation and Usage\n\nTo run Exoframe you will need two parts - Exoframe CLI and [Exoframe server](https://github.com/exoframejs/exoframe-server).  \nFor server install instructions see [server installation docs section](docs/ServerInstallation.md).\n\nTo install Exoframe CLI you can either download one of the pre-packaged binaries from [releases page](https://github.com/exoframejs/exoframe/releases) or install it using npm (needs at least Node 8.x):\n\n```\nnpm install exoframe -g\n```\n\nMake sure you have [Exoframe server](https://github.com/exoframejs/exoframe-server) deployed, set it as your endpoint using:\n\n```\nexoframe endpoint http://you.server.url\n```\n\nThen login using:\n\n```\nexoframe login\n```\n\nThen you will be able to deploy your projects by simply running:\n\n```\nexoframe\n```\n\nYou can find a list of all commands and options in the [docs](./docs/README.md).\n\n## Docs\n\n- [Basics](docs/Basics.md)\n- [Server Installation](docs/ServerInstallation.md)\n- [Server Configuration](docs/ServerConfiguration.md)\n- [Advanced topics](docs/Advanced.md)\n- [Function deployments](docs/Functions.md)\n- [FAQ](docs/FAQ.md)\n- [Contribution Guidelines](docs/Contributing.md)\n- [Templates guide](docs/TemplatesGuide.md)\n- [Recipes guide](docs/RecipesGuide.md)\n- [Using nightly versions](docs/Nightly.md)\n- [Using development and debug versions](docs/Development.md)\n- [Tutorials, articles, video and related links](docs/Links.md)\n- [Change Log](CHANGELOG.md)\n\n## Special thanks\n\nThanks to:\n\n- [Ivan Semenov](https://www.behance.net/ivan_semenov) for making [an awesome logo](./logo/README.md)\n\n## License\n\nLicensed under MIT.\n'
