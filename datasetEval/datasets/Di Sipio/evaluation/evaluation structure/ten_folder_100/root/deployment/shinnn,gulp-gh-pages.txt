b'# gulp-gh-pages\n\n[![npm version](https://img.shields.io/npm/v/gulp-gh-pages.svg)](https://www.npmjs.com/package/gulp-gh-pages)\n[![Build Status](https://travis-ci.org/shinnn/gulp-gh-pages.svg?branch=master)](https://travis-ci.org/shinnn/gulp-gh-pages)\n[![Build status](https://ci.appveyor.com/api/projects/status/iqmi2ijhabfg0cwb/branch/master?svg=true)](https://ci.appveyor.com/project/ShinnosukeWatanabe/gulp-gh-pages/branch/master)\n[![Coverage Status](https://img.shields.io/coveralls/shinnn/gulp-gh-pages.svg)](https://coveralls.io/github/shinnn/gulp-gh-pages)\n\n[gulp](http://gulpjs.com/) plugin to publish contents to [Github pages](https://pages.github.com/)\n\n## Installation\n\n[Use](https://docs.npmjs.com/cli/install) [npm](https://docs.npmjs.com/getting-started/what-is-npm).\n\n```\nnpm install --save-dev gulp@next gulp-gh-pages\n```\n\n## Usage\n\nDefine a `deploy` task in your `gulpfile.js` (as below) which can be used to push to `gh-pages` going forward.\n\n```javascript\nconst {src, task}= require(\'gulp\');\nconst ghPages = require(\'gulp-gh-pages\');\n\ntask(\'deploy\', () => src(\'./dist/**/*\').pipe(ghPages()));\n```\n\nNow, you should be able to call your task by doing:\n\n```\ngulp deploy\n```\n\n## API\n\n```javascript\nconst ghPages = require(\'gulp-gh-pages\');\n```\n\n### ghPages([*options*])\n\n*options*: `Object`  \nReturn: [`stream.Transform`](https://nodejs.org/api/stream.html#stream_class_stream_transform)\n\n#### options.remoteUrl\n\nType: `string`  \nDefault: URL for the remote of the current dir (assumes a git repository)\n\nBy default `gulp-gh-pages` assumes the current working directory is a git repository and uses the URL of the remote designated by `origin`. If your `gulpfile.js` is not in a git repository, or if you want to push to a different remote url, you can specify it. Ensure you have write access to the repository.\n\n#### options.branch\n\nType: `string`  \nDefault: `"gh-pages"`\n\nThe branch where deploy will by done. Change to "master" for `username.github.io` projects.\n\n#### options.cacheDir\n\nType: `string`  \nDefault: `.publish`\n\nSet the directory path to keep a cache of the repository. If it doesn\'t exist, gulp-gh-pages automatically create it.\n\n#### options.push\n\nType: `boolean`  \nDefault: `true`\n\nAllow you to make a build on the defined branch without pushing it to master. Useful for dry  run.\n\n#### options.force\n\nType: `boolean`  \nDefault: `false`\n\nForce adding files to the `gh-pages` branch, even if they are ignored by `.gitignore` or `.gitignore_global`.\n\n#### options.message\n\nType: `string`  \nDefault: `"Update [timestamp]"`\n\nEdit commit message.\n\n## License\n\n[MIT License](./LICENSE) \xc2\xa9 2014 [Micheal Benedict](https://github.com/rowoot), 2015 - 2018 [Shinnosuke Watanabe](https://github.com/shinnn)\n'
