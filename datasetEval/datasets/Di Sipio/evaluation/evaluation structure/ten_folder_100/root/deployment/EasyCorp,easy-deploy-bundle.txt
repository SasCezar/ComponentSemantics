b'EasyDeployBundle\n================\n\n**EasyDeployBundle is the easiest way to deploy your Symfony applications.**\n\n### Features\n\n  * Zero dependencies. No Python. No Ruby. No Capistrano. No Ansible. Nothing.\n  * Zero configuration files. No YAML. No XML. No JSON. Just pure PHP awesomeness.\n  * Multi-server and multi-stage deployment (e.g. "production", "staging", "qa").\n  * Zero downtime deployments.\n  * Supports Symfony 2.7+, Symfony 3.x and Symfony 4.x applications.\n  * Compatible with GitHub, BitBucket, GitLab and your own Git servers.\n\n### Requirements\n\n  * Your local machine: PHP 7.1 or higher and a SSH client.\n  * Your remote servers: they allow SSH connections from the local machine.\n  * Your application: it can use any version of Symfony (2.7+, 3.x, 4.x).\n\n### Documentation\n\n* [Installation](doc/installation.md)\n* [Getting Started](doc/getting-started.md)\n* [Configuration](doc/configuration.md)\n* [Default Deployer](doc/default-deployer.md)\n* [Custom Deployer](doc/custom-deployer.md)\n\n#### Tutorials\n\n* [Creating a Local SSH Configuration File](doc/tutorials/local-ssh-config.md)\n* [Troubleshooting Connection Issues to Remote SSH Servers](doc/tutorials/remote-ssh-config.md)\n* [Cloning the Application Code on Remote Servers](doc/tutorials/remote-code-cloning.md)\n\n> **NOTE**\n> EasyDeploy does not "provision" servers (like installing a web server and the\n> right PHP version for your application); use Ansible if you need that.\n> EasyDeploy does not deploy containerized applications: use Kubernetes if you\n> need that.\n'
