Create a firebase project with the name `pipelinedashboard-dev` via their firebase console https://console.firebase.google.com (*note: this must have a credit card assigned for external http access, but no charge required*)\n2. Turn on **OAuth** authentication on the project (we use GitHub here, but you can use another or allow multiple)\n   1. If using GitHub OAuth, create an OAuth App on GitHub\n   2. Enter the 2 OAuth private keys from GitHub into the Firebase Authentication\n3. Click **Databases** and create an empty `firestore` database (indexes, security, collections and rules will all be automatically created later on as part of the deployment)\n4. Deploying\n  i. Update `{{ FIREBASE_FUNCTIONS_URL }}` in file `functions/src/environments/environment.ts` with your function subdomain, for example `us-central1-pipelinedashboard-test`\n  ii. Update `{{ GITHUB_WEBHOOK_SECRET }}` in file `functions/src/environments/environment.ts` with your private secret key (random string), this is used to protect your webhook function, for example `pipelinedashboard-test-123`\n\n#### Angular\n\n*Note: Make sure you have done the firebase steps above*\n\n1. Now we need to add your firebase keys to your local project\n   1. Duplicate the file `web/environments/environment.ts` to `web/environments/environment.local.ts`\n   2. Go to your firebase project config page and select `config` to see your project keys in `json` format\n   3. Then copy/paste them to the file `web/environments/environment.local.ts` (this file is ignored by git and is not saved to the project)\n2. `cd web`\n3. `npm install`\n4. `npm start`\n\nThen visit `http://localhost:4200/`\n\n#### DEPLOY\n\n1. Deploy all the code (db, functions and UI - you don't have to use the UI, you can connect from your local code, instructions below)\n   1. `npm --prefix web run build:local`\n   2. Login in with firebase cli `firebase login` and then deploy `firebase deploy --project pipelinedashboard-dev`\n\n**NOTE: After `web/` changes you can still use the UI code locally, but any `functions` / `rules` / `indexes` changes will require another deployment**\n\n#### Before contributing\n\nDo not forget to run `npm run list` in `web` or `functions` directory depending on where you made your changes before commiting, these will run the lint checks for you\n\n### RUNNING AUTOMATED TESTS\n\nCurrently in prototype mode.\n\n### Commit message standards\n\nWe use the Angular standards...\n\n> We have very precise rules over how our git commit messages can be formatted.
