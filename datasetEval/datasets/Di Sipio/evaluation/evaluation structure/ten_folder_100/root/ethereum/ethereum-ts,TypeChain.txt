you will never call not existing method again\n- IDE support - works with any IDE supporting Typescript\n- extendible - work with many different APIs: `ethers.js`, `truffle`, `Web3.js 1.0` or you can create your own target\n- frictionless - works with simple, JSON ABI files as well as with Truffle style ABIs\n\n## Installation\n\n```bash\nnpm install --save-dev typechain\n```\n\nYou will also need to install a desired target for example `typechain-target-ethers`. You should not commit them. Read more in FAQ section.\n\nExample:\n\n```\ntypechain --target ethers --outDir app/contracts \'./node_modules/neufund-contracts/build/contracts/*.json\'\n```\n\n## Demo \xf0\x9f\x8f\x8e\xef\xb8\x8f\n\n![Demo](https://media.giphy.com/media/3oFzmqgHxrPZFhBst2/giphy.gif)\n\n[Example usage](https://github.com/Neufund/commit.neufund.org/pull/331/files)\n\n## Getting started \xf0\x9f\x93\x9a\n\n### Motivation\n\nInteracting with blockchain in Javascript is a pain. Web3 interface is sluggish and when using it\nwith Typescript it gets even worse. Often, you can\'t be sure what given method call will\nactually do without looking at ABI file. TypeChain is here to solve these problems (as long as you\nuse Typescript).\n\n### How does it work?\n\nTypeChain is code generator - provide ABI file and you will get Typescript class with flexible\ninterface for interacting with blockchain. Depending on the target parameter it can generate typings for\ntruffle, web3 1.0.0 or ethers.\n\n### Step by step guide\n\nInstall typechain with `yarn add --dev typechain` and install desired target.\n\nRun `typechain --target=your_target` (you might need to make sure that it\'s available in your path if you installed it\nonly locally), it will automatically find all `.abi` files in your project and generate Typescript\nclasses based on them. You can specify your glob pattern: `typechain --target=your_target "**/*.abi.json"`.\n`node_modules` are always ignored. We recommend git ignoring these generated files and making typechain part of your build process.\n\nThat\'s it! Now, just import contract bindings as any other file `import { MyAwesomeContract } from \'./contracts/MyAwesomeContract\'` and start interacting with it. We use named exports because of\n[this](https://blog.neufund.org/why-we-have-banned-default-exports-and-you-should-do-the-same-d51fdc2cf2ad).\n\n## Targets \xf0\x9f\x8e\xaf\n\n### Ethers.js\n\nUse `ethers` target to generate wrappers for [ethers.js](https://github.com/ethers-io/ethers.js/) lib.\n\n### Truffle\n\nTruffle target is great when you use truffle contracts already. Check out [truffle-typechain-example](https://github.com/ethereum-ts/truffle-typechain-example) for more details. It require installing [typings](https://www.npmjs.com/package/truffle-typings) for truffle library itself.\n\nNow you can simply use your contracts as you did before and get full type safety, yay!\n\n### Web3-1.0.0\n\nGenerates typings for contracts compatible with latest Web3.js version. Typings for library itself are now part of `Web3 1.0.0` library so nothing additional is needed. For now it needs explicit cast as shown [here](https://github.com/krzkaczor/TypeChain/pull/88/files#diff-540a9b8840419be93ddb8d4b53325637R8), this will be fixed after improving official typings.\n\n### Your own target\n\nThis might be useful when you\'re creating a library for users of your smartcontract and you don\'t want to lock yourself into any API provided by Web3 access providing library. You can generate basically any code (even for different languages than TypeScript!) that based on smartcontract\'s ABI.\n\n## Migration guide\n\nFor users of 0.x.x versions: the only breaking change is extraction of targets for separate packages so now you need to install `typechain-target-${name}` for each target.\n\n## FAQ \xf0\x9f\xa4\x94\n\n### Q: Should I commit generated files to my repository?\n\nA: _NO_ \xe2\x80\x94 we believe that no generated files should go to git repository. You should git ignore them\nand make `typechain` run automatically for example in post install hook in package.json:\n\n```\n"postinstall":"typechain"\n```\n\nWhen you update ABI, just regenerate files with TypeChain and Typescript compiler will find any\nbreaking changes for you.\n\n### Q: How do I customize generated code?\n\nA: You can create your own target and generate basically any code.\n\n### Q: Generated files won\'t match current codestyle of my project :(\n\nA: We will automatically format generated classes with `prettier` to match your coding preferences (just make sure to use `.prettierrc` file).\n\nFurthermore, we will silent tslint for generated files with `/* tslint:disable */` comments.\n\n### Usage as API\n\nYou may want to use `ts-generator` api to kick off whole process by api:\n\n```typescript\nimport { tsGenerator } from "ts-generator";\nimport { TypeChain } from "typechain/dist/TypeChain";\n\nasync function main() {\n  const cwd = process.cwd();\n\n  await tsGenerator(\n    { cwd },\n    new TypeChain({\n      cwd,\n      rawConfig: {\n        files: "your-glob-here",\n        outDir: "optional out dir path",\n        target: "your-target",\n      },\n    }),\n  );\n}\n\nmain().catch(console.error);\n```\n\n### Running tests\n\n```\nyarn           # install all dependencies\nyarn test      # runs tests + linting\nyarn test:fix  # autofix any errors + run tests\n```\n\n### Debugging \xf0\x9f\x90\x9e\n\n```sh\nDEBUG=typechain typechain\n```\n\n# Licence\n\nKrzysztof Kaczor (krzkaczor) MIT | [Github](https://github.com/krzkaczor) | [Twitter](https://twitter.com/krzkaczor)\n'
