b'Bh \xc2\xb7 Bootstrap Helpers\n======================\n\nA set of Ruby helpers that streamlines the use of\n[Bootstrap 3 components](http://getbootstrap.com/components) in HTML views.\n\nThe **full documentation** is available at [rubydoc.info](http://www.rubydoc.info/github/Fullscreen/bh/master/Bh/Helpers).\n\n[![Build Status](http://img.shields.io/travis/Fullscreen/bh/master.svg)](https://travis-ci.org/Fullscreen/bh)\n[![Coverage Status](http://img.shields.io/coveralls/Fullscreen/bh/master.svg)](https://coveralls.io/r/Fullscreen/bh)\n[![Dependency Status](http://img.shields.io/gemnasium/Fullscreen/bh.svg)](https://gemnasium.com/Fullscreen/bh)\n[![Code Climate](http://img.shields.io/codeclimate/github/Fullscreen/bh.svg)](https://codeclimate.com/github/Fullscreen/bh)\n[![Online docs](http://img.shields.io/badge/docs-\xe2\x9c\x93-green.svg)](http://www.rubydoc.info/github/Fullscreen/bh/master/Bh/Helpers)\n[![Gem Version](http://img.shields.io/gem/v/bh.svg)](http://rubygems.org/gems/bh)\n\nBootstrap 3 is a great framework, but requires many lines of HTML code\neven for simple components.\nFor instance, you need to write the following HTML to show a _dismissible alert_:\n\n```html\n<div class="alert alert-info alert-dismissible" role="alert">\n  <button type="button" class="close" data-dismiss="alert">\n    <span aria-hidden="true">&times;</span>\n    <span class="sr-only">Close</span>\n  </button>\n  You accepted the Terms of service.\n</div>\n```\n\nWriting this for every _dismissible alert_ is cumbersome,\nrepetitive, and prone to errors.\n\nWith Bh, you can achieve the same result with just one line of code:\n\n```rhtml\n<%= alert_box \'You accepted the Terms of service.\', dismissible: true %>\n```\n\n`alert_box` is only one of the helpers provided by Bh.\nThe other ones are: `bootstrap_css`, `bootstrap_js`, `bootstrap_theme_css`,\n`button_to`, `button`, `dropdown`, `font_awesome_css`, `form_for`,\n`icon`, `modal`, `navbar`, `nav`, `panel_row`, `panel` and `progress_bar`.\n\nHow to install\n==============\n\nBh is compatible with **Rails 3.2**, **Rails 4**, **Padrino** and **Middleman**.\n\nTo include the Bh gem in your project:\n\n1. Add `gem \'bh\', \'~> 1.3\'` to the `Gemfile` file of your Rails, Padrino or Middleman project.\n2. *Only if you are using Middleman*: open `config.rb` and add `activate :bh`.\n3. *Only if you are using Padrino*: open `app.rb` and add `register Bh`.\n\nThat\xe2\x80\x99s all!\nFrom now on, you will be able to use any Bh helper in your views.\n\nA comprehensive guide to Bh helpers\n===================================\n\nAll the helpers available in Bh are detailed on the [Bh homepage](http://fullscreen.github.io/bh):\n\n[![Bh homepage](https://cloud.githubusercontent.com/assets/7408595/4195282/5e823a9c-37bc-11e4-865d-bbc04aab38ec.png)](http://fullscreen.github.io/bh)\n\nPlease proceed to [http://fullscreen.github.io/bh](http://fullscreen.github.io/bh) for more details and examples on how to use Bh.\n\nHow to try it in your browser\n=============================\n\nThe `examples` folder includes three sample apps that you can rapidly run in\nyour browser and edit as you please, in order to visually appreciate the result\nof Bh.\n\nIf you\xe2\x80\x99d like to run the _test Rails app_:\n\n1. Download the source code\n1. `cd` into the `examples/rails` folder\n1. Run `bundle install`\n1. Run `bundle exec rails server`\n1. Point your browser to [http://0.0.0.0:3000](http://0.0.0.0:3000)\n\nIf you prefer to see the helpers inside a _test Middleman app_:\n\n1. Download the source code\n1. `cd` into the `examples/middleman` folder\n1. Run `bundle install`\n1. Run `bundle exec middleman`\n1. Point your browser to [http://0.0.0.0:4567](http://0.0.0.0:4567)\n\nIf you prefer to see the helpers inside a _test Padrino app_:\n\n1. Download the source code\n1. `cd` into the `examples/padrino` folder\n1. Run `bundle install`\n1. Run `bundle padrino start`\n1. Point your browser to [http://0.0.0.0:3000](http://0.0.0.0:3000)\n\n\nHow to update\n=============\n\nBh strictly follows [Semantic Versioning](http://semver.org).\nBy indicating the version number in you are a manager of this project, remember to upgrade the [Bh gem](http://rubygems.org/gems/bh)\nwhenever a new feature is added or a bug gets fixed.\n\nMake sure all the tests are passing on [Travis
