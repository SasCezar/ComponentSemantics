b'# Material\n\n[![Backers on Open Collective](https://opencollective.com/material/backers/badge.svg)](#backers)\n[![Sponsors on Open Collective](https://opencollective.com/material/sponsors/badge.svg)](#sponsors) \n\nDaemonite\'s Material UI is a cross-platform and fully responsive front-end interface based on Google Material Design developed using Bootstrap 4.\n\nThe basic idea behind this project is to combine the front-end technology of the popular Bootstrap framework with the visual language of Google Material Design.\n\n> A visual language for our users that synthesizes the classic principles of good design with the innovation and possibility of technology and science. This is material design.\n> http://www.google.com/design/spec/material-design/introduction.html\n\nFeel free to check out our [documentation site](http://daemonite.github.io/material/) or let us know what you think at the [Daemon Labs](http://labs.daemon.com.au) forum.\n\nThe primary goal of this project is to give all Bootstrap components and elements a Google Material Design look, so it allows web developers to continue using the exact same Bootstrap HTML markup they are familiar with, but presents them a final outcome that is in line with the principles and specifics of Google Material Design.\n\nA secondary goal of this project is to add support for some unique Google Material Design components such as floating buttons, pickers, and steppers, to name a few, which cannot be achieved by transforming existing Bootstrap components.\n\nBecause these components will require additional markup (some may require additional JavaScript), they will be documented separately in Material\'s documentation.\n\n## :pray:\n\nThis project exists thanks to all the people who contribute.\n\n<a href="https://github.com/Daemonite/material/graphs/contributors"><img src="https://opencollective.com/material/contributors.svg?width=890&button=false" /></a>\n\n### Backers\n\nThank you to all our backers! The docs may also be run locally.\n\n### Running documentation locally\n\n1. Run through the [tooling setup](http://daemonite.github.io/material/docs/4.1/getting-started/tooling/#getting-started) to install Jekyll and other Ruby dependencies with `bundle install`.\n2. Run `npm install` to install Node.js dependencies.\n3. Run `npm run dist` and `npm run docs` to rebuild distributed CSS and JavaScript files, as well as docs assets.\n4. From the root `/daemonite-material` directory, run `npm run docs-serve` in the command line.\n5. Open `http://localhost:9999/material` in your browser, and voil\xc3\xa0.\n\n## Quick start\n\nSeveral quick start options are available:\n\n- Clone the repo: `git clone https://github.com/Daemonite/material.git`\n- Download the [latest release](https://github.com/Daemonite/material/releases)\n- Install with [bower](https://bower.io/): `bower install daemonite-material`\n- Install with [npm](https://www.npmjs.com/): `npm install daemonite-material`\n'
