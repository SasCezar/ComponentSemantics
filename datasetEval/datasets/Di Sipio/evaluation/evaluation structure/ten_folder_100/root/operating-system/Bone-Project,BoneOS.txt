Issues](https://github.com/Bone-Project/BoneOS/issues)\r\n\r\n\r\nWelcome! This is the official and main repository of BoneOS. This project is currently in high active development ,\r\nand is predominately written in C ofcourse some parts such as  hardware communication, scripting, building, etc ...will use\r\nother languages.\r\nIf you find any issue using this OS please request an issue [**here**](https://github.com/Bone-Project/BoneOS/issues)!\r\nIf you have any contributions or pull requests, which will be highly appreciated, please issue a pull request\r\n[**here**](https://github.com/Bone-Project/BoneOS/pulls)!\r\n\r\n# WARNINGS\r\n---\r\n\r\n - Installing VirtualBox on Ubuntu may lead to problems.\r\n\r\n\r\n# Discussions\r\n---\r\n\r\nAny Questions, Comments, Ideas, Feature Requests, etc. may be brought up [here](https://github.com/Bone-Project/BoneOS/issues).\r\n\r\n# Installation\r\n---\r\n\r\nThere are two ways to install BoneOS: one for the normal user to get it into his Virtual Machine and the other for the developer who want to contribute (building from source).\r\n\r\n#### Normal Installation (For user)\r\n  To install a spsecific version , go to the top of readme where you will see the version number listed as well as installing the Image File via SourceForge or Github Releases. Choose whichever you would like and when you finish installing the ISO(Image File) plug it into your Virtual Machine. [**here**](https://www.youtube.com/watch?v=rBjlaEAzUZo&feature=youtu.be) is a video showing how to do this using VirtualBox as the Virtual Machine Emulator. You can download Virtual Box [**here**](https://www.virtualbox.org/). [Here](https://www.youtube.com/watch?v=yDiwl6AxNrc&feature=youtu.be) is a video showing how to get it in VMWare. VMWare is paid , there if a free trial though. You can get more Information at our Offical Download Page [here](https://boneos.org/download.html).\r\n\r\n#### Nightly Builds (For Developer Or Normal User)\r\n\r\nYou can  get the nightly builds for the latest updated iso for the master branch [here](https://bintray.com/boneos/BoneOS/download_file?file_path=BoneOS-master.iso) .\r\n\r\n#### Building From Source (For developer)\r\n\r\n First make sure you have a Ubuntu Terminal (If you are on Windows, use the new Linux Subsystem Feature). Then download the dependencies using `bash setup.sh`. Then compile BoneOS using `make`. Now that you have qemu after you executed `setup.sh` , you can go ahead and run on qemu using `make qemu_compile` for running the binary or `make qemu_iso` to run the actual Image File(.ISO).\r\n\r\n#### Cloud9 Collaboration (For Developer)\r\n\r\n You can collaborate with the Main BoneOS Developer in Cloud9. Cloud9 Is an Online IDE , with an Ubuntu Terminal. We use Cloud9IDE Terminal + NoVNC to make this possible.\r\n the workspace is located [here](https://ide.c9.io/amanuel2/osdev) . You can learn more [here](https://community.c9.io/t/cant-install-grub/11653/5?u=phpcoder231).\r\n\r\n If you want to setup NoVNC Server once you are in an Cloud9IDE You can use the command\r\n\r\n ```sh\r\n make no_vnc\r\n ```\r\n\r\n#### Extra Options (For Developer)\r\n\r\n There are extra options when building(`make`ing) , this OS. You can set the compiler instead of the default\r\n gcc(`$(CC)`) like so :\r\n\r\n ```sh\r\n  $ CC=clang make #If you want to run via clang\r\n  $ CC=gcc-5 make #If you want to run via GCC Version 5\r\n  $ ./configure --cc=gcc-5 && make # Optionally use the configure script to store variable permanently\r\n ```\r\n\r\n You can see more in [Docs/Configuration](https://docs.boneos.org/en/latest/Configuration/Configuration/)\r\n\r\n#### More Info(For Developer)\r\nFor more information, go [here](https://docs.boneos.org) for documentation.\r\n\r\n# Directory Structure\r\n----\r\nOur directory structure currently is as follows:\r\n\r\n- ***docs*** : Documentation for BoneOS\r\n- ***apps*** : Applications\r\n   - ***term*** : Terminal\r\n- ***.github*** : Github related files.\r\n   - ***ISSUE_TEMPLATE.md*** : Github Issues Template\r\n- ***arch*** : Architecture Dependant Codes\r\n   - ***@(ARCH)*** : Architecture name.\r\n     - ***boot***: GRUB Bootloader information\r\n       - ***multiboot***: Grub Multiboot information\r\n     - ***drv***  : Device Drivers for @(ARCH) are located here.\r\n     - ***io*** : I/O Software Code\r\n     - ***link***: Where the linker is located.\r\n     - ***init***: Main kernel function getting called by GRUB Bootloader\r\n  - ***shared*** : Shared piece of code between Proccecor Families\r\n     - ***x86*** : Shared piece of code for x86 family\r\n- ***bin*** : Non Adminstrator Terminal Commands\r\n- ***sbin*** : Adminstrator Terminal Commands\r\n- ***config*** : Configuration Files\r\n- ***hdd*** : Default files on first time use\r\n- ***platform*** : Platform Dependant Code\r\n   - ***pc*** : PC Platform\r\n- ***libc***: C Library constructed and used by BoneOS, non-archecture and non-platform dependant code.\r\n- ***utils***: Utilities for devs.\r\n   - ***cross_compiler***: Script for installing the cross compiler\r\n   - ***no_vnc***: Setup No_vnc server in Cloud9IDE\r\n   - ***watch.sh*** : Nodemon like script\r\n- ***include***: All Include Files(.h).\r\n- ***var*** : Common Variables used across the system.\r\n\r\n# Documentation\r\n ---\r\nYou can take a look at the Documentation online hosted by gitbook [**here**](https://docs.boneos.org). Documentation are for presistant information , not code details.\r\n\r\n# Architectures\r\n---\r\nArchitecture          |   Status    | Plan to support |\r\n------------- | ------------- | -----------|\r\nx86(32 BIT) |    Ready   |  Already Supported        |\r\nx64| On Progress  |    Currently updating to support this architecture       |\r\nPowerPC| Unavailable  |   Yes, in later releases       |\r\nArm| Unavailable   |    Yes, in later releases       |\r\nMIPS | Unavailable|     Yes, in later releases     |\r\n\r\nif there are architectures you want BoneOS to support , please request an issue to talk about it.\r\n\r\n\r\n# FAQ\r\n---\r\n\r\n1) **What is BoneOS Goal?**\r\n Our goal is to make an Operating System for \'Everyone\'.
