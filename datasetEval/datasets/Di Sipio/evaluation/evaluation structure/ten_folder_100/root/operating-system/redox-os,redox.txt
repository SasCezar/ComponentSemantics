b'<p align="center">\n<img alt="Redox" width="346" src="https://gitlab.redox-os.org/redox-os/assets/raw/master/logos/redox/logo.png">\n</p>\n\n**Redox** is an operating system written in Rust, a language with focus on safety and high performance. Redox, following the microkernel design, aims to be secure, usable, and free. Redox is inspired by previous kernels and operating systems, such as SeL4, MINIX, Plan 9, and BSD.\n\nRedox _is not_ just a kernel, it\'s a **full-featured Operating System**, providing packages (memory allocator, file system, display manager, core utilities, etc.) that together make up a functional and convenient operating system. You can loosely think of it as the GNU or BSD ecosystem, but in a memory safe language and with modern technology. See [this list](#ecosystem) for overview of the ecosystem.\n\nThe website can be found at https://www.redox-os.org.\n\nPlease make sure you use the **latest nightly** of `rustc` before building (for more troubleshooting, see ["Help! Redox won\'t compile!"](#compile-help)).\n\n[![Travis Build Status](https://travis-ci.org/redox-os/redox.svg?branch=master)](https://travis-ci.org/redox-os/redox)\n[![Downloads](https://img.shields.io/github/downloads/redox-os/redox/total.svg)](https://gitlab.redox-os.org/redox-os/redox/tags)\n[![MIT licensed](https://img.shields.io/badge/license-MIT-blue.svg)](./LICENSE)\n![Rust Version](https://img.shields.io/badge/rust-nightly%202017--10--03-lightgrey.svg)\n\n## Contents\n\n* [What it looks like](#screenshots)\n* [Ecosystem](#ecosystem)\n* [Help! Redox won\'t compile! </a>\n\nSometimes things go wrong when compiling. Try the following before opening an issue:\n\n1. Run `rustup update`\n1. Run `make clean pull`.\n1. Make sure you have **the latest version of Rust nightly!** ([rustup.rs](https://www.rustup.rs) is recommended for managing Rust versions. If you already have it, run `rustup`).\n1. Update **GNU Make**, **NASM** and **QEMU/VirtualBox**.\n1. Pull the upstream master branch (`git remote add upstream git@gitlab.redox-os.org:redox-os/redox.git; git pull upstream master`).\n1. Update submodules (`git submodule update --recursive --init`).\n\nand then rebuild!\n\n## <a name="contributing"> Contributing to Redox </a>\n\nIf you\'re interested in this project, and you\'d like to help us out, [here](CONTRIBUTING.md) is a list of ways you can do just that.\n\n## <a name="cloning-building-running"> Cloning, Building and Running </a>\n\nRedox is big, even compressed. Downloading the full history may take a lot of bandwidth, and can even be costly on some data plans. Clone at your own risk!\n\n### <a name="quick-setup" /> Quick Setup </a>\n\n```bash\n$ cd path/to/your/projects/folder/\n\n# Run bootstrap setup\n$ curl -sf https://gitlab.redox-os.org/redox-os/redox/raw/master/bootstrap.sh -o bootstrap.sh && bash -e bootstrap.sh\n\n# Change to project directory\n$ cd redox\n\n# Build Redox\n$ make all\n\n# Launch using QEMU\n$ make qemu\n# Launch using QEMU without using KVM (Kernel-based Virtual Machine). Try if QEMU gives an error.\n$ make qemu kvm=no\n```\n\n#### QEMU with KVM\n\nTo use QEMU with KVM (Kernel-based Virtual Machine), which is faster than without KVM, you need a CPU with Intel\xc2\xae Virtualization Technology (Intel\xc2\xae VT) or AMD Virtualization\xe2\x84\xa2 (AMD-V\xe2\x84\xa2) support. Most systems have this disabled by default, so you may need to reboot, go into the BIOS, and enable it.\n\n### <a name="manual-setup"> Manual Setup </a>\n\nTo manually clone, build and run Redox using a Unix-based host, run the following commands (with exceptions, be sure to read the comments):\n```bash\n$ cd path/to/your/projects/folder/\n\n# HTTPS\n$ git clone https://gitlab.redox-os.org/redox-os/redox.git --origin upstream --recursive\n# SSH\n$ git clone git@gitlab.redox-os.org:redox-os/redox.git --origin upstream --recursive\n\n$ cd redox/\n\n# Install/update dependencies\n$ ./bootstrap.sh -d\n\n# Install rustup.rs\n$ curl https://sh.rustup.rs -sSf | sh\n$ source $HOME/.cargo/env\n\n# Install the sysroot manager Xargo\n$ cargo install xargo\n\n# For successive builds start here. If this is your first build, just continue\n\n# Update git submodules\n$ git submodule update --recursive --init\n\n# Build Redox\n$ make all\n\n# Launch using QEMU\n$ make qemu\n\n# Launch using QEMU without using KVM (Kernel-based Virtual Machine). Try if QEMU gives an error.\n$ make qemu kvm=no\n\n# Launch using QEMU without using KVM (Kernel-based Virtual Machine) nor Graphics\nmake qemu kvm=no vga=no\n```\n\n### <a name="setup-using-docker"> Setup using Docker </a>\nWe also provide docker image. After cloning this repository, please follow README under the `docker` directory.\n\n### Updating the codebase using the Makefile\nTo update the codebase run:\n\n```\nmake pull; make fetch\n```\n\n`make pull` pulls and updates the submodules, and `make fetch` updates the sources for cookbook recipes.\n'
