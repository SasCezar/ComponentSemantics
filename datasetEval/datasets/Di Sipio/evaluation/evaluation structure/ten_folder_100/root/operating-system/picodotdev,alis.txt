b"# alis\n\nArch Linux Install Script (alis) installs unattended, automated and customized Arch Linux system.\n\nThis a simple bash script for an easy and fast way of installing Arch Linux, follow the [Arch Way](https://wiki.archlinux.org/index.php/Arch_Linux) of doing things and learn what this script does. This will allow you to know what is happening. \n\nPlease, don't ask for support for this script in Arch Linux forums, first read\nthe [Arch Linux wiki](https://wiki.archlinux.org), the [Installation Guide](https://wiki.archlinux.org/index.php/Installation_guide) and the [General\nRecomendations](https://wiki.archlinux.org/index.php/General_recommendations), later\ncompare those commands with the commands of this script.\n\nFor new features, improvements and bugs fill an issue in GitHub or make a pull request. You can test it in a VirtualBox virtual machine (recommended) before run it in real hardware. If you test it in real hardware please send me an email to pico.dev@gmail.com with the machine description and tell me if something goes wrong or all works fine. [Pull request](https://github.com/picodotdev/alis/pulls) and [new feature request](https://github.com/picodotdev/alis/issues) are welcome!\n\n**Warning! This script deletes all partitions of the persistent storage**\n\nCurrently these scripts are for me but maybe they are useful for you too.\n\n### Features\n\n* GPT, UEFI, BIOS\n* Support for SATA, NVMe and MMC\n* LVM and no LVM\n* _root_ partition encrypted and no encrypted\n* LVM on LUKS when LVM and encrypted\n* File system formats ext4, btrfs (no swap), xfs\n* Optional file swap\n* WPA WIFI network installation\n* Periodic TRIM for SSD storage\n* VirtualBox guest utils\n* Intel processors microcode\n* Users creation and add to sudoers\n* Common and custom packages installation\n* AUR utility installation (aurman, yay)\n* Retry packages download on connection/mirror error\n* Desktop environments (GDM, KDE, XFCE, Mate, Cinnamon, LXDE), display managers (GDM, SDDM, Lightdm, lxdm) and no desktop environment\n* Additional kernel installation (linux-lts, linux-hardened, linux-zen)\n* Kernel compression\n* Graphic controllers (intel, nvidia, amd) with optionally early KMS start\n* GRUB, rEFInd, systemd-boot bootloaders\n* Script for download installation and recovery scripts and configuration files\n* Installation log with all commands executed and output in a file and/or asciinema video\n* Wait after installation for an abortable reboot\n\n### Installation\n\nInternet connection is required, with wireless WIFI connection see [Wireless_network_configuration](https://wiki.archlinux.org/index.php/Wireless_network_configuration#Wi-Fi_Protected_Access) to bring up WIFI connection before starting installation with alis.\n\n```\n# # Start the system with lastest Arch Linux installation media\n# loadkeys [keymap]\n# curl https://raw.githubusercontent.com/picodotdev/alis/master/download.sh | bash, or with URL shortener curl -sL https://bit.ly/2F3CATp | bash\n# # Edit alis.conf and change variables values with your preferences\n# vim alis.conf\n# ./alis.sh\n```\n\n### Installation with asciinema v\xc3\xaddeo\n\nAs another form of log.\n\n```\n# # Start the system with lastest Arch Linux installation media\n# loadkeys [keymap]\n# curl https://raw.githubusercontent.com/picodotdev/alis/master/download.sh | bash, or with URL shortener curl -sL https://bit.ly/2F3CATp | bash\n# ./alis-asciinema.sh\n# # Edit alis.conf and change variables values with your preferences\n# vim alis.conf\n# ./alis.sh\n# exit\n# ./alis-reboot.sh\n```\n\n### Recovery\n\n```\n# # Start the system with lastest Arch Linux installation media\n# loadkeys [keymap]\n# curl https://raw.githubusercontent.com/picodotdev/alis/master/download.sh | bash, or with URL shortener curl -sL https://bit.ly/2F3CATp | bash\n# # Edit alis-recovery.conf and change variables values with your last installation with alis\n# vim alis-recovery.conf\n# # Optional asciinema video\n# ./alis-asciinema-recovery.sh\n# ./alis-recovery.sh\n```\n\n###
