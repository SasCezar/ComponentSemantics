b'# Heroku Django Starter Template\n\nAn utterly fantastic project starter template for Django 2.0.\n\n## Features\n\n- Production-ready configuration for Static Files, Database Settings, Gunicorn, etc.\n- Enhancements to Django\'s static file serving functionality via WhiteNoise.\n- Latest Python 3.6 runtime environment.\n\n## How to Use\n\nTo use this project, follow these steps:\n\n1. Create your working environment.\n2. Install Django (`$ pipenv install django`)\n3. Create a new project using this template\n\n## Creating Your Project\n\nUsing this template to create a new Django app is easy::\n\n    $ django-admin.py startproject --template=https://github.com/heroku/heroku-django-template/archive/master.zip --name=Procfile helloworld\n\n(If this doesn\'t work on windows, replace `django-admin.py` with `django-admin`)\n\nYou can replace ``helloworld`` with your desired project name.\n\n## Deployment to Heroku\n\n    $ git init\n    $ git add -A\n    $ git commit -m "Initial commit"\n\n    $ heroku create\n    $ git push heroku master\n\n    $ heroku run python manage.py migrate\n\nSee also, a [ready-made application](https://github.com/heroku/python-getting-started), ready to deploy.\n\n\n## License: MIT\n\n## Further Reading\n\n- [Gunicorn](https://warehouse.python.org/project/gunicorn/)\n- [WhiteNoise](https://warehouse.python.org/project/whitenoise/)\n- [dj-database-url](https://warehouse.python.org/project/dj-database-url/)\n'
