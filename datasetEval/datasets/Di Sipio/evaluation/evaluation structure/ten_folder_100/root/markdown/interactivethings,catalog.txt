Stable code is in [master](https://github.com/interactivethings/catalog/tree/master).\n\n# Catalog\n\nCatalog lets you create beautiful living and fully interactive style guides using Markdown and React components.\n\nPlease read the [Catalog documentation](https://docs.catalog.style/) (built with Catalog!) for detailed installation and usage instructions.\n\n## Installation\n\n### yarn\n\n```\nyarn add catalog react react-dom\n```\n\n### npm\n\n```\nnpm install catalog react react-dom --save\n```\n\n## Development\n\n> Please make sure that you have Node >= 6 and [yarn](https://yarnpkg.com/) installed.\n\n### Build process\n\nStart the build process in watch mode:\n\n```\nmake\n```\n\n### Docs\n\nTo edit Catalog documentation, run:\n\n```\nmake docs\n```\n\n> Note: this uses the local Catalog build from `dist/`. Run `make` before/alongside `make docs`.\n\n### Linking\n\nWhen developing Catalog you want to link it locally:\n\n```\nyarn link\n```\n\nYou can then link to this version in your project (or one of the examples):\n\n```\nyarn link catalog\n```\n\n### Tests\n\nTo run [Jest](https://facebook.github.io/jest/) tests in watch mode:\n\n```\nmake test-watch\n```\n\n## Creating a Release\n\nBump Catalog's version:\n\n```\nmake version\n```\n\nThen push including tags:\n\n```\ngit push && git push --tags\n```\n\nThe CI server will automatically run tests, build and publish the new version to npm\n\n## CI commands\n\n> These usually run automatically on the CI server\n\n### Create a build\n\n```\nmake build\n```\n\n### Create a documentation build\n\n```\nmake build-docs\n```\n\n### Publish to npm and generate GitHub release notes\n\n```\nmake publish\n```\n\n## Credits\n\nCatalog is developed by [many people](https://github.com/interactivethings/catalog/blob/master/AUTHORS) at [Interactive Things](https://www.interactivethings.com/), a User Experience and Data Visualization Studio based in Z\xc3\xbcrich.\n"
