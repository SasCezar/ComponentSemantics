b'# MMMarkdown\nMMMarkdown is an Objective-C framework for converting [Markdown][] to HTML. It is compatible with OS X 10.7+, iOS 8.0+, tvOS, and watchOS.\n\nUnlike other Markdown libraries, MMMarkdown implements an actual parser. It is not a port of the original Perl implementation and does not use regular expressions to transform the input into HTML. MMMarkdown tries to be efficient and minimize memory usage.\n\n[Markdown]: http://daringfireball.net/projects/markdown/\n\n## API\nUsing MMMarkdown is simple. The main API is a single class method:\n\n    #import <MMMarkdown/MMMarkdown.h>\n    \n    NSError  *error;\n    NSString *markdown   = @"# Example\\nWhat a library!";\n    NSString *htmlString = [MMMarkdown HTMLStringWithMarkdown:markdown error:&error];\n    // Returns @"<h1>Example</h1>\\n<p>What a library!</p>"\n\nThe markdown string that is passed in must be non-nil.\n\nMMMarkdown also supports a number of Markdown extensions:\n\n    #import <MMMarkdown/MMMarkdown.h>\n    \n    NSString *markdown   = @"~~Mistaken~~";\n    NSString *htmlString = [MMMarkdown HTMLStringWithMarkdown:markdown extensions:MMMarkdownExtensionsGitHubFlavored error:NULL];\n    // Returns @"<p><del>Mistaken</del></p>"\n\n## Setup\nAdding MMMarkdown to your project is easy.\n\nIf you\xe2\x80\x99d like to use [Carthage](https://github.com/Carthage/Carthage), add the following line to your `Cartfile`:\n\n```\ngithub "mdiep/MMMarkdown"\n```\n\nOtherwise, you can:\n\n0. Add MMMarkdown as a git submodule. (`git submodule add https://github.com/mdiep/MMMarkdown <path>`)\n\n0. Add `MMMarkdown.xcodeproj` to your project or workspace\n\n0. Add `MMMarkdown.framework` to the \xe2\x80\x9dLink Binary with Libraries" section of your project\'s \xe2\x80\x9cBuild Phases\xe2\x80\x9d.\n\n0. Add `MMMarkdown.framework` to a \xe2\x80\x9dCopy Files\xe2\x80\x9d build phase that copies it to the `Frameworks` destination.\n\n## License\nMMMarkdown is available under the [MIT License][].\n\n[MIT License]: http://opensource.org/licenses/mit-license.php\n\n'
