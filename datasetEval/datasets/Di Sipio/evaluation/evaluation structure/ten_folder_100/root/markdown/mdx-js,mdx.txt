\xf0\x9f\x9a\x80\n\n[![Build Status][build-badge]][build]\n[![Join the community on Spectrum][spectrum-badge]][spectrum]\n\n[MDX][website] is an authorable format that lets you seamlessly use JSX in your markdown documents.\nYou can import components, like interactive charts or notifications, and export\nmetadata.\nThis makes writing long-form content with components a blast.\n\n## Example\n\nSee MDX in action:\n\n```jsx\nimport { Chart } from \'../components/chart\'\n\n# Here\xe2\x80\x99s a chart\n\nThe chart is rendered inside our MDX document.\n\n<Chart />\n```\n\n## Intro\n\nMDX is markdown for the component era.\nIt lets you write JSX embedded inside markdown.\nThat\xe2\x80\x99s a great combination because it allows you to use markdown\xe2\x80\x99s often terse\nsyntax (such as `# heading`) for the little things and JSX for more advanced\ncomponents.\n\n:heart: **Powerful**: MDX blends markdown and JSX syntax to fit perfectly in\nReact/JSX-based projects.\n\n:computer: **Everything is a component**: Use existing components inside your\nMDX and import other MDX files as plain components.\n\n:wrench: **Customizable**: Decide which component is rendered for each markdown\nelement (`{ h1: MyHeading }`).\n\n:books: **Markdown-based**: The simplicity and elegance of markdown remains,\nyou interleave JSX only when you want to.\n\n:fire: **Blazingly blazing fast**: MDX has no runtime, all compilation occurs\nduring the build stage.\n\n> [Watch some of these features in action][intro]\n\n## Getting started\n\n```shell\nnpm init mdx\n```\n\nVisit [`mdxjs.com`][website] for more info, and check out [Contribute][] below\nto find out how to help out.\n\n## Why?\n\nBefore MDX, some of the benefits of writing markdown were lost when integrating\nwith JSX.\nImplementations were often template string-based which required lots of escaping\nand cumbersome syntax.\n\n> \xe2\x80\x9cMDX \\[\xe2\x80\xa6] is extremely useful for using design system components to render\n> markdown and weaving interactive components in with existing markdown.\xe2\x80\x9d\n>\n> \xe2\x80\x94 [@chrisbiscardi][tweet]\n\nMDX seeks to make writing with markdown *and* JSX simpler while being more\nexpressive.\nThe possibilities are endless when you combine components (that can even be\ndynamic or load data) with the simplicity of markdown for long-form content.\nA nice example of this is [mdx-deck][], a great way to create slides with MDX.\n\n*   Fast\n*   No runtime compilation\n*   [Pluggable][remark-plugins]\n*   Element to React component mapping\n*   React component `import`/`export`\n*   Customizable layouts\n*   [Webpack loader](https://mdxjs.com/getting-started/webpack)\n*   [Parcel plugin](https://mdxjs.com/getting-started/parcel)\n*   [Next.js plugin](https://mdxjs.com/getting-started/next)\n*   [Gatsby plugin](https://mdxjs.com/getting-started/gatsby)\n\n##
