Output to pagers, write to files, automate with cron, etc.\n\n`haxor-news` comes with a handy **optional auto-completer with interactive help**:\n\n![Imgur](http://i.imgur.com/seKUiur.png)\n\n## completions and an auto-completion menu with interactive help:\n\n    $ haxor-news\n\nIf available, the auto-completer also automatically displays comments through a pager.\n\nWithin the auto-completer, the same syntax applies:\n\n    haxor> hn <command> [params] [options]\n\n![Imgur](http://i.imgur.com/L2rzgb3.png)\n\n![Imgur](http://i.imgur.com/FL2pyC0.png)\n\n## Customizable Highlighting\n\nYou can control the ansi colors used for highlighting by updating your `~/.haxornewsconfig` file.\n\nColor options include:\n\n```\n\'black\', \'red\', \'green\', \'yellow\',\n\'blue\', \'magenta\', \'cyan\', \'white\'\n```\n\nFor no color, set the value(s) to `None`.\n\n![Imgur](http://i.imgur.com/lzoRxfW.png)\n\n## Commands\n\n![Imgur](http://i.imgur.com/oqvUbj8.png)\n\n## View Posts\n\nView the Top, Best, Show, Ask, Jobs, New, and Onion posts.\n\nUsage:\n\n    $ hn [command] [limit]  # post limit default: 10\n\nExamples:\n\n    $ hn top\n    $ hn show 20\n\n![Imgur](http://i.imgur.com/tjGPszv.png)\n\n## View a Post\'s Linked Web Content\n\nAfter viewing a list of posts, you can view a post\'s linked web content by referencing the post `#`.\n\nThe HTML contents of the post\'s link are **formatted for easy-viewing within your terminal**.  If available, the formatted output is sent to a pager.\n\nSee the [View in a Browser](#view-in-a-browser) section to view the contents in a browser instead.\n\nUsage:\n\n    $ hn view [#]\n\nExample:\n\n    $ hn view 1\n    $ hn view 8\n\n![Imgur](http://i.imgur.com/FoTCPAp.png)\n\n## View and Filter a Post\'s Comments\n\n### View All Comments\n\nAfter viewing a list of posts, you can view a post\'s comments by referencing the post `#`.\n\nExamples:\n\n    $ hn view 8 -c\n    $ hn view 8 --comments > comments.txt\n\n#### Paged Comments\n\nIf running with the auto-completer, comments are automatically paginated.  To get the same pagination without the auto-completer, append `| less -r`:\n\n    $ hn view 8 -c | less -r\n\n![Imgur](http://i.imgur.com/t32ITvN.png)\n\n### Filter on Unseen Comments\n\nFilter comments to expand only those you have not yet seen.  Unseen comments are denoted with a `[!]` and are fully expanded.\n\nSeen comments will be truncated with [...] and will be shown to help provide context to unseen comments.\n\nExamples:\n\n    $ hn view 8 -cu\n    $ hn view 8 --comments_unseen | less -r\n\n![Imgur](http://i.imgur.com/tCVpSIs.png)\n\n### Filter on Recent Comments\n\nFilter comments to expand only those **posted within the past 60 minutes**.\n\nOlder comments will be truncated with [...] and will be shown to help provide context to recent comments.\n\nExamples:\n\n    $ hn view 8 -cr | less -r\n    $ hn view 8 --comments_recent\n\n![Imgur](http://i.imgur.com/diOjxIr.png)\n\n### Filter with Regex\n\nFilter comments based on a given regular expression query.\n\nExamples:\n\n    $ hn view 2 -cq "(?i)programmer" | less -r\n    $ hn view 2 --comments_regex_query "(?i)programmer" > programmer.txt\n\n*Case insensitive regex: `(?i)`*\n\n![Imgur](http://i.imgur.com/SlKtIpS.png)\n\n### Hide Non-Matching Comments\n\nWhen filtering comments for unseen, recent, or with regex, non-matching comments are collapsed to provide context.  To instead hide non-matching comments, pass the `-ch\\--comments_hide` flag.  Hidden comments will be displayed as `.`.\n\nExample:\n\n    $ hn view 8 -cu -ch | less -r\n\n![Imgur](http://i.imgur.com/qPopK7X.png)\n\n## Filter the Monthly Hiring Post\n\nHacker News hosts a monthly hiring post where employers post the latest job openings.\n\nUsage:\n\n    $ hn hiring [regex filter]\n\nExamples:\n\n    $ hn hiring ""\n    $ hn hiring "(?i)JavaScript|Node"\n    $ hn hiring "(?i)(Node|JavaScript).*(remote)" > remote_jobs.txt\n\n*Case insensitive regex: `(?i)`*\n\n![Imgur](http://i.imgur.com/Lwz8iwG.png)\n\nTo search a different monthly hiring post other than the latest, use the hiring post id.\n\nUsage:\n\n    $ hn hiring [regex filter] [post id]\n\n## Filter the Freelancers Post\n\nHacker News hosts a monthly freelancers post where employers and freelancers post availabilities.\n\nUsage:\n\n    $ hn freelance [regex filter]\n\nExamples:\n\n    $ hn freelance ""\n    $ hn freelance "(?i)JavaScript|Node"\n    $ hn freelance "(?i)(Node|JavaScript).*(remote)" > remote_jobs.txt\n\n*Case insensitive regex: `(?i)`*\n\n![Imgur](http://i.imgur.com/TnBDFGk.png)\n\nTo search a different monthly hiring post other than the latest, use the hiring post id.\n\nUsage:\n\n    $ hn freelance [regex filter] [post id]\n\n## Combine With Pipes and Redirects\n\nOutput to pagers, write to files, automate with cron, etc.\n\nExamples:\n\n    $ hn view 1 -c | less\n    $ hn freelance "(?i)(Node|JavaScript).*(remote)" > remote_jobs.txt\n\n![Imgur](http://i.imgur.com/x2aj7SM.png)\n\n## View User Info\n\nUsage:\n\n    $ hn user [user id]\n\n![Imgur](http://i.imgur.com/oTALQQI.png)\n\n## View Onions\n\nUsage:\n\n    $ hn onion [limit]  # post limit default: all\n\n![Imgur](http://i.imgur.com/MubWRNG.png)\n\n## View in a Browser\n\nView the linked web content or comments in your default browser instead of your terminal.\n\nUsage:\n\n    $ hn <command> [params] [options] -b\n    $ hn <command> [params] [options] --browser\n\n## Windows Support\n\n`haxor-news` has been tested on Windows 10.\n\n### Pager Support\n\nPager support on Windows is more limited as discussed in the following [ticket](https://github.com/donnemartin/haxor-news/issues/16).  Users can direct output to a pager with the `| more` command:\n\n    $ hn view 1 -c | more\n\n### Config File\n\nOn Windows, the `.haxornewsconfig` file can be found in `%userprofile%`.  For example:\n\n    C:\\Users\\dmartin\\.haxornewsconfig\n\n### `cmder` and `conemu`\n\nAlthough you can use the standard Windows command prompt, you\'ll probably have a better experience with either [cmder](https://github.com/cmderdev/cmder) or [conemu](https://github.com/Maximus5/ConEmu).\n\n## Installation\n\n### Pip Installation\n\n[![PyPI version](https://badge.fury.io/py/haxor-news.svg)](http://badge.fury.io/py/haxor-news) [![PyPI](https://img.shields.io/pypi/pyversions/haxor-news.svg)](https://pypi.python.org/pypi/haxor-news/)\n\n`haxor-news` is hosted on [PyPI](https://pypi.python.org/pypi/haxor-news).  The following command will install `haxor-news`:\n\n    $ pip install haxor-news\n\nYou can also install the latest `haxor-news` from GitHub source which can contain changes not yet pushed to PyPI:\n\n    $ pip install git+https://github.com/donnemartin/haxor-news.git\n\nIf you are not installing in a virtualenv, run with `sudo`:\n\n    $ sudo pip install haxor-news\n\nOnce installed, run the optional `haxor-news` auto-completer with interactive help:\n\n    $ haxor-news\n\nRun commands:\n\n    $ hn <command> [params] [options]\n\n### Virtual Environment Installation\n\nIt is recommended that you install Python packages in a [virtualenv](http://docs.python-guide.org/en/latest/dev/virtualenvs/) to avoid potential issues with dependencies or permissions.\n\nTo view `haxor-news` `virtualenv` installation instructions, click [here](https://github.com/donnemartin/haxor-news/blob/master/INSTALLATION.md).\n\n### Mac OS X 10.11 El Capitan Users\n\nThere is a known issue with Apple and its included python package dependencies (more info at https://github.com/pypa/pip/issues/3165).
