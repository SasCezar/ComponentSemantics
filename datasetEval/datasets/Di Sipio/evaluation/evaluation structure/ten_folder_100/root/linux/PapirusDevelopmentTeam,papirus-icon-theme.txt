install Papirus from our official [PPA](https://launchpad.net/~papirus/+archive/ubuntu/papirus):\n\n```\nsudo add-apt-repository ppa:papirus/papirus\nsudo apt-get update\nsudo apt-get install papirus-icon-theme\n```\n\nor download .deb packages from [here](https://launchpad.net/~papirus/+archive/ubuntu/papirus/+packages?field.name_filter=papirus-icon-theme).\n\n**NOTE:** Now the daily builds of the papirus-icon-themes package are placed in [`ppa:papirus/papirus-dev`](https://launchpad.net/~papirus/+archive/ubuntu/papirus-dev).\n\n### Debian and derivatives\n\nDebian users also can install Papirus from our [PPA](https://launchpad.net/~papirus/+archive/ubuntu/papirus), but the commands will differ:\n\n```\nsudo sh -c "echo \'deb http://ppa.launchpad.net/papirus/papirus/ubuntu bionic main\' > /etc/apt/sources.list.d/papirus-ppa.list"\n\nsudo apt-get install dirmngr\nsudo apt-key adv --recv-keys --keyserver keyserver.ubuntu.com E58A9D36647CAE7F\nsudo apt-get update\nsudo apt-get install papirus-icon-theme\n```\n\n### Papirus Installer\n\nUse the scripts to install the latest version directly from this repo (independently of your distro):\n\n**NOTE:** Use the same script to update icon themes.\n\n#### ROOT directory (recommended)\n\n```\nwget -qO- https://git.io/papirus-icon-theme-install | sh\n```\n\n#### HOME directory for GTK\n\n```\nwget -qO- https://git.io/papirus-icon-theme-install | DESTDIR="$HOME/.icons" sh\n```\n\n#### HOME directory for KDE\n\n```\nwget -qO- https://git.io/papirus-icon-theme-install | DESTDIR="$HOME/.local/share/icons" sh\n```\n\n#### \\*BSD systems\n\n```\nwget -qO- https://git.io/papirus-icon-theme-install | env DESTDIR="/usr/local/share/icons" sh\n```\n\n#### Uninstall\n\n```\nwget -qO- https://git.io/papirus-icon-theme-uninstall | sh\n```\n\n### Third-party packages\n\nPackages listed in this section are third-party packages. absolute path instead of the icon name in a .desktop file or in the source code which makes them unthemable.\n\n### Hardcoded application icons\n\nTo deal with hardcoded application icons we recommend using [hardcode-fixer](https://github.com/Foggalong/hardcode-fixer). Papirus supports most of the applications in the [list](https://github.com/Foggalong/hardcode-fixer/blob/master/tofix.csv). If [hardcode-fixer](https://github.com/Foggalong/hardcode-fixer) doesn\'t support your favorite app yet, please open an issue [here](https://github.com/Foggalong/hardcode-fixer/issues) or edit your .desktop file manually.\n\n### Hardcoded tray icons\n\nTo fix hardcoded tray icons Papirus supports [Hardcode-Tray](https://github.com/bil-elmoussaoui/Hardcode-Tray) script. A list of supported applications is available [here](https://github.com/bil-elmoussaoui/Hardcode-Tray/tree/master/data/database).\n\n**NOTE:** To get Papirus to work right with Hardcode-Tray, use the hardcode-tray option `--conversion-tool RSVGConvert`:\n\n```\nsudo -E hardcode-tray --conversion-tool RSVGConvert --size 22 --theme Papirus\n```\n\n**Size recommendations:**\n\n- Unity 22px\n- KDE 22px\n- GNOME 22px ([see](https://github.com/PapirusDevelopmentTeam/papirus-icon-theme#manual-fixes) for more info)\n- XFCE 22px ([see](https://github.com/PapirusDevelopmentTeam/papirus-icon-theme#manual-fixes) for more info)\n- Pantheon 24px\n- Cinnamon 16px\n\n\n![hardcode-tray](https://i.imgur.com/6hFm6aj.png)\n\n**NOTE**: Some Electon-based applications have blurred tray icon on KDE (see [bug report](https://bugs.kde.org/show_bug.cgi?id=366062)). To solve this issue pass the following environment variable to the app: `XDG_CURRENT_DESKTOP=Unity wire-desktop`\n\n## KDE colorscheme\n\nSupport for monochrome icons for KDE colorscheme is now available:\n- Papirus - for dark plasma theme & light color scheme\n- Papirus Dark - for dark plasma theme & color scheme\n- Papirus Light - for light plasma theme & color scheme\n\n![kde-color-scheme](https://i.imgur.com/oM1qhQH.png)\n\n**NOTE:** Non-KDE apps don\'t support KDE colorscheme on the system tray, but you can replace color manually.\n\n## Folder\'s color\n\nPapirus has [Folder Color](http://foldercolor.tuxfamily.org/) v0.0.80+ support that allows you to change a color of a folder.\n\nAvailable colors:\n\n![Folder Color Preview](https://i.imgur.com/jP3mRci.png)\n\nFor KDE, colors of individual folders can be changed using [dolphin-folder-color](https://github.com/audoban/dolphin-folder-color).\n\nAlso, you can use our [papirus-folders](https://github.com/PapirusDevelopmentTeam/papirus-folders) script to apply the color of folders system-wide.\n\n## Extras\n\n- [Papirus theme for LibreOffice](https://github.com/PapirusDevelopmentTeam/papirus-libreoffice-theme)\n- [Papirus themes for FileZilla](https://github.com/PapirusDevelopmentTeam/papirus-filezilla-themes)\n- [Papirus theme for SMPlayer](https://github.com/PapirusDevelopmentTeam/papirus-smplayer-theme)\n- [Papirus themes for Claws Mail](https://github.com/PapirusDevelopmentTeam/papirus-claws-mail-theme)\n- [Papirus themes for Thunderbird](https://github.com/PapirusDevelopmentTeam/thunderbird-theme-papirus)\n- [Papirus theme for aMule](https://github.com/PapirusDevelopmentTeam/papirus-amule-theme)\n\n## Recommendations\n\n- Recommended use Papirus icons alongside one of the following GTK themes:\n  - The patched version of [sni-qt](https://github.com/bilelmoussaoui/sni-qt) for [Hardcode-Tray](https://github.com/bilelmoussaoui/Hardcode-Tray) doesn\'t work without it.\n- [Dash to Dock](https://extensions.gnome.org/extension/307/dash-to-dock/) **\xc2\xb9**\n- [No Symbolic Icons](https://extensions.gnome.org/extension/1304/no-symbolic-icons/)\n- [Status Area Horizontal Spacing](https://extensions.gnome.org/extension/355/status-area-horizontal-spacing/)\n\n**\xc2\xb9** On Ubuntu 18.04+ it is pre-installed.\n</details>\n\n<details>\n<summary>For Xfce users</summary>\n\nHere are a few recommendations for Xfce users.\n\n#### Thunar File Manager\n\nGo to `Edit` \xe2\x86\x92 `Preferences...`. Click on `Side Pane` tab. Under `Side Pane`, look for `Icon Size` and set to `Very Small`.\n\n![thunar-prefecences](https://i.imgur.com/Iu1TIEa.png)\n\n#### Notification Area\n\nGo to `Settings Manager` \xe2\x86\x92 `Panel` \xe2\x86\x92 `Items` tab. Select `Notification Area` item and click on `Edit currently selected item` button. Under `Appearance` set the following options:\n\n- Set `Maximum icon size (px)` to `24`\n- Uncheck `Show frame`\n\n![xfce4-notification-area](https://i.imgur.com/MopCZBZ.png)\n</details>\n\n<details>\n<summary>For elementary/Pantheon users</summary>\n\nWith light wallpaper, we recommend disable `use-transparency` option on wingpanel:\n\n```\ngsettings set org.pantheon.desktop.wingpanel use-transparency false\n```\n\n</details>\n\n## Icon request\n\n- Application name\n- Icon name (see desktop-file option **Icon** on `/usr/share/applications`)\n- Original icon image\n- Small description and/or a link to the official webpage\n\n**NOTE**: We do NOT support Windows/Wine/Crossover or other NOT native Linux-apps.
