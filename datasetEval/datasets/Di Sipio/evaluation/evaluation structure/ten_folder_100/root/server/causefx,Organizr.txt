src="https://user-images.githubusercontent.com/16184466/53667702-fcdcc600-3c2e-11e9-8828-860e531e8096.png">\n\n##### Usage\n```\ndocker create \\\n  --name=organizr \\\n  -v <path to data>:/config \\\n  -e PGID=<gid> -e PUID=<uid>  \\\n  -p 80:80 \\\n  organizrtools/organizr-v2\n```\n##### Parameters\nThe parameters are split into two halves, separated by a colon, the left hand side representing the host and the right the container side. For example with a port -p external:internal - what this shows is the port mapping from internal to external of the container. So `-p 8080:80` would expose port 80 from inside the container to be accessible from the host\'s IP on port 8080 and `http://192.168.x.x:8080` would show you what\'s running INSIDE the container on port 80.\n\n* `-p 80` - The port(s)\n* `-v /config` - Mapping the config files for Organizr\n* `-e PGID` Used for GroupID - see below for explanation\n* `-e PUID` Used for UserID - see below for explanation\n\n##### Info\n* Shell access whilst the container is running: `docker exec -it organizr /bin/bash`\n* To monitor the logs of the container in realtime: `docker logs -f organizr`\n* Container version number: `docker inspect -f \'{{ index .Config.Labels "build_version" }}\' organizr`\n* Image version number: `docker inspect -f \'{{ index .Config.Labels "build_version" }}\' organizrtools/docker-organizr-v2`\n\n<img src="https://user-images.githubusercontent.com/16184466/53614287-a9b73480-3b96-11e9-9c8e-e32b4ae20c0d.png">\n\n<p align="center"><a href="https://www.browserstack.com"><img src="https://avatars2.githubusercontent.com/u/1119453?s=200&v=4g"></a></p>\n<p align="center"><a href="https://www.browserstack.com">BrowserStack</a> for allowing us to use their platform for testing</p>\n'
