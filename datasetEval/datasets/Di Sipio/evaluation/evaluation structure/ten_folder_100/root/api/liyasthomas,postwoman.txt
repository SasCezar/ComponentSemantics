Simple design is the best design.\n\n - Faster, lighter, cleaner, minimal & responsive\n\n\xe2\x9a\xa1\xef\xb8\x8f **Fast**: Send requests and get/copy responses in real-time! Fast software is the best software.\n\n**Methods:**\n - `GET` - Retrieve information about the REST API resource\n - `HEAD` - Retrieve response headers identical to those of a GET request, but without the response body.\n - `POST` - Create a REST API resource\n - `PUT` - Update a REST API resource\n - `DELETE` - Delete a REST API resource or related component\n - `OPTIONS` - Describe the communication options for the target resource\n - `PATCH` - Apply partial modifications to a REST API resource\n\n_History entries are synced with local session storage_\n\n\xf0\x9f\x8c\x88 **Make it yours**: Customizable combinations for background, foreground and accent colors: because customization === freedom. [Customize now \xe2\x9c\xa8](https://postwoman.io/settings).\n\n**Customizations:**\n - Choose theme: Kinda Dark (default), Clearly White, Just Black and System theme\n - Choose accent color: Green (default), Yellow, Pink, Red, Purple, Orange, Cyan and Blue\n - Toggle multi-colored frames\n\n_Customized themes are also synced with local session storage_\n\n\xf0\x9f\x94\xa5 **PWA**: Install as a [PWA](https://developers.google.com/web/progressive-web-apps) (100/100 Lighthouse score) on your device.\n\n**Features:**\n - Instant loading with [Service Workers](https://developers.google.com/web/fundamentals/primers/service-workers)\n - Offline support\n - Low RAM/memory and CPU usage\n - [Add to Home Screen](https://developers.google.com/web/fundamentals/app-install-banners) (button in footer)\n - [Desktop PWA](https://developers.google.com/web/progressive-web-apps/desktop) support (button in footer)\n - ([full features](https://developers.google.com/web/progressive-web-apps))\n\n\xf0\x9f\x9a\x80 **Request**: Retrieve data from a URL without having to do a full page refresh.\n\n - Choose `method`\n - Enter `URL`\n - Enter `Path`\n\n**Features:**\n - Copy/share public "Share URL"\n - Generate request code for JavaScript XHR, Fetch and cURL\n - Copy generated request code to clipboard\n - Import cURL\n - Label requests\n\n\xf0\x9f\x94\x8c **WebSocket**: Establish full-duplex communication channels over a single TCP connection.\n\n - Send and receive data\n\n\xf0\x9f\x93\xa1 **Server Sent Events**: Receive a stream of updates from a server over a HTTP connection without resorting to polling.\n\n  - Receive data\n\n\xf0\x9f\x8c\x8d **GraphQL**: GraphQL is a query language for APIs and a runtime for fulfilling those queries with your existing data.\n\n - Set endpoint and get schemas\n - Multi-column docs\n\n\xf0\x9f\x94\x90 **Authentication**: Allows to identify the end user.\n\n**Types:**\n - None\n - Basic authentication using username and password\n - Token based authentication\n\n\xf0\x9f\x93\xa2 **Headers**: Describes the format the body of your request is being sent as.\n\n - Add or remove Header list\n\n\xf0\x9f\x93\xab **Parameters**: Use request parameters to set varying parts in simulated requests.\n\n\xf0\x9f\x93\x83 **Request Body**: Used to send and receive data via the REST API.\n\n**Options:**\n - Set Content Type\n - Add or remove Parameter list\n - Toggle between key-value and RAW input Parameter list\n\n\xf0\x9f\x91\x8b **Responses**: Contains the status line, headers and the message/response body.\n\n - Copy response to clipboard\n - Download response to a local file\n - View preview for HTML responses\n\n_HTML responses have "Preview HTML" feature_\n\n\xe2\x8f\xb0 **History**: Request entries are synced with local session storage to reuse with a single click.\n\n**Fields:**\n - Label\n - Timestamp\n - Method\n - Status code\n - URL\n - Path\n\n_History entries can be sorted by any fields_\n\n_Histories can be deleted one-by-one or all together_\n\n\xf0\x9f\x93\x81 **Collections**: Keep your API requests organized with collections and folders. Reuse them with a single click.\n\n**Options:**\n - Create infinite collections, folders and requests\n - Edit, delete, move, export, import and replace\n\n_Export, import and replace collections with JSON files_\n\n_Collections are synced with local session storage_\n\n\xf0\x9f\x8c\x90 **Proxy**: Enable Proxy Mode from Settings to access blocked APIs.\n\n**Features:**\n - Hide your IP address\n - Fixes [CORS](https://developer.mozilla.org/en-US/docs/Web/HTTP/CORS) (Cross Origin Resource Sharing) issues\n - Access APIs served in non-HTTPS (`http://`)\n - Use custom Proxy URL\n\n_Official Postwoman Proxy is hosted by ApolloTV - [Privacy policy](https://apollotv.xyz/legal)_\n\n\xf0\x9f\x93\x9c **Pre-Request Scripts**: Snippets of code associated with a request that are executed before the request is sent.\n\n**Use-cases:**\n - Including the timestamp in the request headers\n - Sending a random alphanumeric string in the URL parameters\n\n_Requests with Pre-Request Scripts are indicated in History entries_\n\n\xe2\x8c\xa8\xef\xb8\x8f **Keyboard Shortcuts**: Optimized for efficiency.\n\n**Shortcuts:**\n - Send Request <kbd>Ctrl</kbd> + <kbd>G</kbd>\n - Save to Collections <kbd>Ctrl</kbd> + <kbd>S</kbd>\n - Copy Request Link <kbd>Ctrl</kbd> + <kbd>K</kbd>\n - Reset Request <kbd>Ctrl</kbd> + <kbd>L</kbd>\n\n**To find out more, please check out [Postwoman Wiki](https://github.com/liyasthomas/postwoman/wiki).**\n\n## Demo \xf0\x9f\x9a\x80 [![Website](https://img.shields.io/website?url=https%3A%2F%2Fpostwoman.io&logo=Postwoman)](https://postwoman.io)\n\n[https://postwoman.io](https://postwoman.io)\n\n## Usage \xf0\x9f\x92\xa1\n\n1. Specify your request method\n2. Type in your API URL\n3. Add API path\n4. Send request\n5. Get response!\n\nYou\'re done!\n\n## Built with \xf0\x9f\x94\xa7\n\n* [Chromium](https://github.com/chromium/chromium) - Thanks for being so fast!\n* HTML - For the web framework\n* CSS - For styling components\n* JavaScript - For magic!\n* [Vue](https://vuejs.org/) - To add to the JavaScript magic!\n* [Nuxt](https://nuxtjs.org/) - To add to the Vue magic!\n\n## Developing \xf0\x9f\x91\xb7\n\n1. [Clone this repo](https://help.github.com/en/articles/cloning-a-repository) with git.\n2. Install dependencies by running `npm install` within the directory that you cloned (probably `postwoman`).\n3. Start the development server with `npm run dev`.\n4. Open development site by going to [http://localhost:3000](http://localhost:3000) in your browser.\n\n#### Or, with docker-compose:\n\n1. [Clone this repo](https://help.github.com/en/articles/cloning-a-repository) with git.\n2. Run `docker-compose up`\n3. Open development site by going to [http://localhost:3000](http://localhost:3000) in your browser.\n\n## Docker \xf0\x9f\x90\xb3 [![Docker Cloud Build Status](https://img.shields.io/docker/cloud/build/liyasthomas/postwoman?logo=Docker)](https://hub.docker.com/r/liyasthomas/postwoman)\n\n```bash\n#pull\ndocker pull liyasthomas/postwoman\n\n#run\ndocker run -p 3000:3000 liyasthomas/postwoman:latest\n\n#build\ndocker build -t postwoman:latest\n```\n\n## Releasing \xf0\x9f\x8f\xb7\xef\xb8\x8f\n\n1.
