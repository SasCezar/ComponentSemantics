[![pyVersion37](https://img.shields.io/badge/python-3.7-blue.svg)](https://www.python.org/download/releases/3.7/)\n\n- Download this source code into a working directory.\n\n- Install the requirements using pip:\n\n```sh\nmake install\n```\n\nThis will install all the required packages and libraries for using PokeAPI\n\n- Set up the local developer environment using the following command:\n\n```sh\nmake setup\n```\n\n- Run the server using the following command:\n\n```sh\nmake serve\n```\n\nVisit localhost:8000 to see the running website!\n\nIf you ever need to wipe the database use this command:\n\n```sh\nmake wipe_db\n```\n\n## Database setup\n\nStart Django shell\n\n```sh\npython manage.py shell --settings=config.local\n```\n\nRun the build script with\n\n```py\nfrom data.v2.build import build_all\nbuild_all()\n```\n\nEach time the build script is run, it will iterate over each table in the database, wipe it, and rewrite each row using the data found in data/v2/csv.\n\nIn informal tests on a Windows PC with a SSD and a 2.50 GHz processor, building against a PostgresQL database took approximately 6 minutes, and building against a SQLite database took about 7.5 minutes or longer, with some varying results.\n\nThe option to build individual portions of the database was removed in order to increase performance of the build script.\n\n## Docker Compose\n\nThere is also a multi-container setup, managed by [Docker Compose](https://docs.docker.com/compose/). This setup allow you to deploy a production-like environment, with separate containers for each services.\n\nStart the process using\n\n```sh\ndocker-compose up --build\n```\n\nYou can specify the `-d` switch to start in detached mode.\nThis will bind port 80 and 443. Unfortunately, unlike the `docker` command, there is no command line arguments to specify ports. If you want to change them, edit the `docker-compose.yml` file.\n\nAfter that, start the migration process\n\n```sh\ndocker-compose exec app python manage.py migrate --settings=config.docker-compose\n```\n\nAnd then, import the data using the shell\n\n```sh\ndocker-compose exec app python manage.py shell --settings=config.docker-compose\n```\n\nThen use the `build_all()` method in the shell to populate the database.\n\n```py\nfrom data.v2.build import build_all\nbuild_all()\n```\n\nBrowse [localhost/api/v2/](http://localhost/api/v2/) or [localhost/api/v2/pokemon/bulbasaur/](http://localhost/api/v2/pokemon/bulbasaur/)\n\nFor the moment, this setup doesn\'t allow you to use the `scale` command.\n\n## Docker (obsolete)\n\nThe application can be built and run as a Docker container for easy deployments\n\nFrom the root directory of the cloned repo\n\n```sh\ndocker build -t pokeapi .\n```\n\nRun the container on host port 8000\n\n```sh\ndocker run -d -p 8000:8000 pokeapi\n```\n\n## Contributing\n\nThis project exists thanks to all the people who contribute.
