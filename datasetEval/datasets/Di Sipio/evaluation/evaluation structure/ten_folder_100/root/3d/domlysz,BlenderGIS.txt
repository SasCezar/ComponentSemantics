b'Blender GIS\n==========\nBlender minimal version : 2.8\n\n\n[Full documentation](https://github.com/domlysz/BlenderGIS/wiki/Home)\n--------------------\n\n[FAQ](https://github.com/domlysz/BlenderGIS/wiki/FAQ) - [Quick start guide](https://github.com/domlysz/BlenderGIS/wiki/Quick-start)\n\n\n[Basemaps](https://github.com/domlysz/BlenderGIS/wiki/Basemaps)\n--------------------\nDisplay web map service like OpenStreetMap directly in Blender\n\n![](https://raw.githubusercontent.com/wiki/domlysz/blenderGIS/Blender27x/images/basemaps_demo.gif)\n\n[SRTM download](https://github.com/domlysz/BlenderGIS/wiki/SRTM)\n--------------------\nGet SRTM topographic data and apply it as height texture\n\n![](https://raw.githubusercontent.com/wiki/domlysz/blenderGIS/Blender27x/images/srtm_demo.gif)\n\n\n[ESRI Shapefile import / export](https://github.com/domlysz/BlenderGIS/wiki/Shapefile-import)\n--------------------\n\nA [Shapefile](http://en.wikipedia.org/wiki/Shapefile) is a popular geospatial vector data format for geographic information system software.\n\nThis tool can import into Blender most of shapefile feature type. It can also uses attributes data to define Z elevation values or Z extrusion values.\n\nExporter script can export a mesh to pointZ, polylineZ or polygonZ shapefile. Note that currently this tool does not re-export attribute data include in the dbase file linked to the shapefile. So if you want to import a shapefile for edit it into Blender and then re-export it, you will lose attribute data.\n\n\n[Georeferenced raster importer](https://github.com/domlysz/BlenderGIS/wiki/Import-georef-raster)\n--------------------\n\nImport geotiff or common image format georeferenced with a [world file](http://en.wikipedia.org/wiki/World_file).\n\nYou can import the raster as a plane mesh, as backgound image for orthographic view, as UV texture mapping on a mesh or [as DEM](https://github.com/domlysz/BlenderGIS/wiki/Import-DEM-grid) for warp a mesh with the displace modifier.\n\nESRI ASCII GRID format is also supported through a dedicated import tool.\n\n[OpenStreetMap import](https://github.com/domlysz/BlenderGIS/wiki/OSM-import)\n--------------------\n\n![](https://raw.githubusercontent.com/wiki/domlysz/blenderGIS/Blender27x/images/osm_demo.gif)\n\n\n[Georeferenced render output](https://github.com/domlysz/BlenderGIS/wiki/Make-a-georef-render)\n--------------------\n\nThis is a tool to create a new camera correctly setup for produce a map render. Georeferencing data (worldfile) are writing in text file accessible from the Blender text editor.\n\n\n[Delaunay triangulation & Voronoi diagram](https://github.com/domlysz/BlenderGIS/wiki/Make-terrain-mesh-with-Delaunay-triangulation)\n--------------------\n\nThis script computes [Delaunay triangulation](http://en.wikipedia.org/wiki/Delaunay_triangulation) in 2.5D. This triangulation is suitable for create a 3D terrain mesh from [points cloud](http://en.wikipedia.org/wiki/Point_cloud) or [contour lines](http://en.wikipedia.org/wiki/Contour_line)\n\nThe script can also compute [Voronoi tessellation](http://en.wikipedia.org/wiki/Voronoi) in 2D which is the dual of delaunay triangulation. Voronoi diagram is suitable to make neighborhood analysis map.\n\n\n[Terrain analysis](https://github.com/domlysz/BlenderGIS/wiki/Terrain-analysis)\n--------------------\n\nThis part of Blender GIS is designed to assist in the analysis of the topography : height, slope and azimuth (aspect).\n\nThere are 2 tools, one to build materials nodes setup for Cycles engine, and a second to configure the color ramp as usual in common GIS software (reclassify values and apply color ramp presets).\n\n[Georeferencing management](https://github.com/domlysz/BlenderGIS/wiki/Gereferencing-management)\n--------------------\nHandle various projection systems with reprojection capabilities and compatibility with some others addons\n'
