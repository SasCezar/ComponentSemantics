href="https://github.com/UniversalViewer/universalviewer/graphs/contributors"><img src="https://opencollective.com/universalviewer/contributors.svg?width=890&button=false" /></a>\n\nThe Universal Viewer is a community-developed open source project on a mission to help you share your content with the world http://universalviewer.io\n\n## Features:\n\n- **Zoomable**\n  <p><a href="https://openseadragon.github.io/">OpenSeadragon</a> image zooming using the <a href="http://iiif.io/api/image/2.0/">IIIF image API</a></p>\n- **Embeddable**\n  <p>YouTube-style embedding with deep links to specific pages/zoom regions.</p>\n- **Themable**\n  <p>Fork the <a href="https://github.com/UniversalViewer/uv-en-GB-theme">UV theme</a> on github to create your own.</p>\n- **Configurable**\n  <p>The UV has its own visual configuration editor allowing all settings to be customised.</p>\n- **Extensible**\n  <p>Supports IIIF Presentation API v3 (in development), allowing <a href="http://universalviewer.io/examples/?manifest=http://files.universalviewer.io/manifests/nelis/ecorche.json">3D</a>, <a href="http://universalviewer.io/examples/?manifest=http://wellcomelibrary.org/iiif/b17307922/manifest">audio</a>, <a href="http://universalviewer.io/examples/?manifest=http://wellcomelibrary.org/iiif/b16659090/manifest">video</a>, and <a href="http://universalviewer.io/examples/?manifest=http://wellcomelibrary.org/iiif/b17502792/manifest">pdf</a> viewing experiences.</p>\n\n[More about the Universal Viewer](https://github.com/UniversalViewer/universalviewer/wiki/About)\n\n## Stable Builds\n\nSee the [GitHub releases page](https://github.com/UniversalViewer/UniversalViewer/releases).\n\n## Getting Started\n\nFor a "hello world" getting started example see [uv-hello-world](https://github.com/UniversalViewer/uv-hello-world).\n\nFor an example of how to use the Universal Viewer in a website starter project that uses [biiif](https://github.com/edsilv/biiif) to generate your IIIF data, see [uv-app-starter](https://github.com/UniversalViewer/uv-app-starter).\n\n## Building the Universal Viewer\n\nAll command-line operations for building the UV are scripted using [Grunt](http://gruntjs.com/) which is based on [Node.js](http://nodejs.org/). To get set up:\n\n1. Install [Node.js](http://nodejs.org), if you haven\'t already\n\n1. Install the grunt command line interface:\n\n    `npm install -g grunt-cli`\n\n<!--\n1. Install [Sorcery](https://github.com/Rich-Harris/sorcery):\n\n    npm install -g sorcery\n-->\n1. Clone the `universalviewer` repository and `examples` sub module:\n\n    `git clone https://github.com/UniversalViewer/universalviewer.git --recursive`\n\n1. On the command line, go in to the `universalviewer` folder\n\n1. Run\n\n    `npm install`\n\n> If switching between branches, ensure that you check out the correct branch in the `examples` sub-repository too. By default this should be `gh-pages`.\n\n## Debug Builds\n\nTo build the debug version of the viewer, just run (on the command line, in the `universalviewer` folder):\n\n    grunt build\n\nThis will compile the [.less](http://lesscss.org) files into .css and [.ts](http://typescriptlang.org) files into .js.\n\n## Distribution Builds\n\nTo build the distribution version of the UV, just run (on the command line, in the `universalviewer` folder):\n\n    grunt build --dist\n\n## Examples\n\nTo view the examples run:\n\n    grunt examples\n\n## Backers\n\nSupport us with a monthly donation and help us continue our activities.
