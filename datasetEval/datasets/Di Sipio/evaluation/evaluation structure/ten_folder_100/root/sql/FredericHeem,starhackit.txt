b'<img alt="Starhack" src="./client/src/app_public/landing/img/starhackit-logo-text.png" height="250">\n\nStarHackIt\n==========\nStarHackIt is a fullstack starter kit composed of:\n\n* React web frontend\n* React native mobile app \n* Node backend\n* Deployment by Ansible with docker images\n\nFullstack web application starter kit written in modern Javascript with react and node.js with the following features:\n\n* Authentication: username/password, facebook, google authentication etc ...\n* Authorization: scheme using user, group and permission  \n* Scalable by using a micro services based architecture, a.k.a message queues\n* Relational database: postgres, mysql, sqlite, mssql etc, ...\n* Logging\n* 100% results in [Lighthouse test](https://developers.google.com/web/tools/lighthouse/)\n  \n## Frontend Technologies\n\n* [React](https://facebook.github.io/react/): libraries to build user interfaces.\n* [Mobx](https://mobx.js.org/): Simple, scalable state management\n* [Webpack](http://webpack.github.io/): module bundler for the browser\n* [Emotion](https://emotion.sh): Style React Components with Style.\n* [i18next](http://i18next.com/): internationalization\n\nFor more information about the frontend, see its [README](client/README.md)\n\n## Mobile App Technologies\n\nFor more information about the mobile app, see its [README](mobile/README.md)\n\n## Backend Technologies\n\n* [Koa](http://koajs.com/): next generation web framework for Node.js.\n* [Sequelize](http://docs.sequelizejs.com/en/latest/): Object Relationship Management (ORM) supporting majors relational SQL database.\n* [PostgreSQL](http://www.postgresql.org/): the world\'s most advanced open source relational database.\n* [Passport](http://passportjs.org/): authentication framework with more than 140 authentication strategies: username/password, facebook , google, github etc ...\n* [Winston](https://github.com/winstonjs/winston): a multi-transport async logging library.\n* [Nodemailer](https://github.com/andris9/Nodemailer): send email with various provider.\n\nFor more information about the backend, see its [README](server/README.md)\n\n## Dev Technologies\n\n* [Docker](https://www.docker.com/): Debug your app, not your environment\n* [Mocha](http://mochajs.org/):
