On drag end, calls an action for you to handle list mutation.\n* Disable sorting within a list while still allowing dragging in and out of the list (sort order is determined by host app instead of the user).\n\n\n\n### Demo\n\nhttps://kaliber5.github.io/ember-drag-sort/\n\n\n\n## Versions, branches and jQuery\n\n* Version 1.x, `gen-0` branch is based on jQuery.\n* Version 2.x, `gen-1` branch (current development head) has got rid of jQuery. It yields `item` and `index`.\n\n```handlebars\n{{#drag-sort-list\n  items         = items1\n  dragEndAction = (action \'dragEnd\')\n  as |item|\n}}\n  {{item.name}}\n{{/drag-sort-list}}\n```\n\n### The drag end action\n\nIt is called on the source list component when the drag\'n\'drop operation is complete. It\'s called with a single argument -- an object with the following properties:\n\n| Property      | Type   | Description                                                            |\n|:--------------|:-------|:-----------------------------------------------------------------------|\n| `group`       | String | Group provided to the `drag-sort-list` component.                      |\n| `draggedItem` | <any>  | The list item being dragged.                                           |\n| `sourceList`  | Array  | The list where the sorting was initiated.                              |\n| `sourceIndex` | Number | The initial index of the item in the source list.                      |\n| `targetList`  | Array  | The list where sorting was finished. Will be the same as `sourceList`. |\n| `targetIndex` | Number | The resulting index of the dragged item in the target list.            |\n\nWhen sorting within one list, `targetIndex` assumes that the dragged item is not in the list.\n\n> For example, when your list is `[\'a\', \'b\', \'c\']` and you put `b` after `c`, `sourceIndex` will be `1` and `targetIndex` will be `2`. The initial index of `c` was `2`, so you could suppose that target index after `c` is `3`.\n>\n> But `targetIndex` is calculated as if the dragged item `b` is not in the list: `[\'a\', \'c\']`. Thus, next index after `c` will be `2`.\n>\n> This is because the array has three items with indexes `0`, `1` and `2`, so putting an item to position `3` would make no sense.\n\nHere\'s the reference implementation of the `dragEndAction` action:\n\n```js\n  actions: {\n    dragEndAction ({sourceList, sourceIndex, targetList, targetIndex}) {\n      if (sourceList === targetList && sourceIndex === targetIndex) return\n\n      const item = sourceList.objectAt(sourceIndex)\n\n      sourceList.removeAt(sourceIndex)\n      targetList.insertAt(targetIndex, item)\n    }\n  }\n```\n\nThe `dragEndAction` action *must* be a closure action.\n\nCorrect:\n\n    dragEndAction = (action \'dragEnd\')\n\nIncorrect:\n\n    dragEndAction = \'dragEnd\'\n\n\n\n### The determine foreign position action\n\nYou may want to let the user drag items in and out of a list, without letting him rearrange items within a list. In that case the order of items is determined by the app.\n\nHere\'s a use case. Your CMS allows the admin to put widgets in page areas. The admin panel has a number of lists representing page header, sidebar, footer, etc. The admin is allowed to rearrange widgets to his liking. And there\'s a list of unused widgets, the admin can drag items to and from that list, but unused items should always be sorted alphabetically.\n\nTo achieve that, pass a closure action `determineForeignPositionAction` into the list of unused items. This will prevent the user from sorting items in that list.\n\nWhen the user drags a foreign items into such a list, the action will be called to determine the position of the item. Essentially, by running that action the `ember-drag-sort` addon asks the host app to suggest desired position of the dragged item.\n\nThe action is only called for foreign items. When the user drags an item out of the unsortable list but then drags the item back, it will appear on its original position.\n\nThe `determineForeignPositionAction` is called with with a single argument -- an object with the following properties:\n\n| Property      | Type  | Description                                   |\n|:--------------|:------|:----------------------------------------------|\n| `draggedItem` |       | The item being dragged.                       |\n| `items`       | Array | The list where the item should be positioned. |\n\nThis action must return an **integer** -- the desired position of the item.\n\nThe simplest implementation is to always put the item into the end of the list:\n\n```js\ndetermineForeignPosition ({/* draggedItem,  */items}) {\n  return items.length\n}\n```\n\nTo sort items alphabetically, you  can use lodash:\n\n```js\ndetermineForeignPosition ({draggedItem, items}) {\n  return _.sortedIndex(items.toArray(), draggedItem)\n}\n```\n\nOr do it by hand:\n\n```js\ndetermineForeignPosition ({draggedItem, items}) {\n    return Ember.A(items.slice()) // make sure not to mutate the list; `Ember.A()` is typically redundant\n      .addObject(draggedItem)\n      .sortBy(\'name\')\n      .indexOf(draggedItem)\n}\n```\n\n**`determineForeignPositionAction` must not actually sort the list**. It\'s only purpose is to suggest desired item position, which is necessary to display the placeholder.\n\n**`determineForeignPositionAction` must passed as a closure action**.\n\nCorrect:\n\n    determineForeignPositionAction = (action \'determineForeignPosition\')\n\nIncorrect:\n\n    determineForeignPositionAction = \'determineForeignPosition\'\n\n\n\n\n### drag-sort-list arguments reference\n\n| Argument                         | Type                                         | Default value | Description                                                                                                                                                                                     |\n|:---------------------------------|:---------------------------------------------|:--------------|:------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|\n| `items`                          | Ember Array                                  | <required>    | An array of items to display and offer sorting.                                                                                                                                                 |\n| `dragEndAction`                  | Closure action                               | <required>    | This callback will be called on source list when sorting is complete. See above for details.                                                                                                    |\n| `determineForeignPositionAction` | Closure action or `undefined`                | `undefined`   | When provided, used to determine the position of the placeholder when dragging a foreign item into the list. When not provided, the user is able to determine the order. See above for details. |\n| `group`                          | <any>                                        | `undefined`   | Used to restrict dragging between multiple lists to only some of those lists. Typically a string.                                                                                               |\n| `draggingEnabled`                | Boolean                                      | `true`        | Disables sorting. Useful when `dragEndAction` is an async operation.                                                                                                                            |\n| `childClass`                     | String                                       | `""`          | HTML class applied to list item components.                                                                                                                                                     |\n| `childTagName`                   | String                                       | `"div"`       | `tagName` applied to list item components.                                                                                                                                                      |\n| `handle`                         | String, typically `"[draggable]"`, or `null` | `null`        | Selector of the drag handle element. When provided, items can only be dragged by handle. :warning: The handle element *must* have `draggable="true"` attribute.                                 |\n| `isHorizontal`                   | Boolean                                      | `false`       | Displays the list horizontally. :warning: Horizontal lists don\'t work well when nested.   |\n| `isRtl`                          | Boolean                                      | `false`       | RTL - Right to left. It also assumes the following classes dynamically:\n\n| HTML class         | Applied when...                                                                                                                                                                   |\n|:-------------------|:----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|\n| `-isEmpty`         | The given list is empty.                                                                                                                                                          |\n| `-draggingEnabled` | Dragging is enabled via the `draggingEnabled` attribute.                                                                                                                          |\n| `-isDragging`      | Dragging is in progress and the given list is either a source list or belongs to the same group as the source list.                                                               |\n| `-isDraggingOver`  | Dragging is in progress and the placeholder is within the given list. This class is removed from a list when an item is dragged into a different list.                            |\n| `-isExpanded`      | Dragging is in progress and the given list is either empty or contains only the dragged item. Used to give some height to the list, so that the item can be dragged back into it. |\n\nThe individual item component has HTML class `dragSortItem`. It also assumes the following classes dynamically:\n\n\n| HTML class          | Applied when...                                                               |\n|:--------------------|:------------------------------------------------------------------------------|\n| `-isDragged`        | The given item is the one being dragged. Used to hide the item from the list. |\n| `-isDraggingOver`   | Dragged item is positioned either above/before or below/after the given item.              |\n| `-placeholderBefore` | Dragged item is positioned above/before the given item.                              |\n| `-placeholderAfter` | Dragged item is positioned below/after the given item.                              |\n\n\n\n### CSS concerns\n\nWhen dragging, the dragged item is hidden via the `-isDragged` HTML class that applies `display: none`.\n\nThe placeholder (drop target) is shown via the `-placeholderBefore` or `-placeholderAfter` HTML classes. These classes apply padding to the given list item, and the placeholder is an absolutely positioned `:before` pseudo-element. A similar pseudo-element is applied to an `-isExpanded` list (see above).\n\nFor sorting to work correctly, you must not apply padding to the list HTML element. If you need some padding on the list, apply it to its parent element.\n\nYou must not apply any padding or margin to list item elements either. If you need padding between list items, apply it to HTML elements that you pass into list items.\n\nAvoid collapsing margins between list items and between list item and list. Collapsing margins may cause a [jumping glitch](https://github.com/kaliber5/ember-drag-sort/issues/7).\n\n\n\n### Events\n\nThere\'s an Ember service called `dragSort`. You can listen to the following events on it, using the [Evented API](https://www.emberjs.com/api/classes/Ember.Evented.html).\n\nEach event is called with as single argument, which is an object with properties. For the description of properties, see `dragEndAction` documentation above.\n\n| Event name | Description                                                                        | Argument properties                                                                                   |\n|:-----------|:-----------------------------------------------------------------------------------|:------------------------------------------------------------------------------------------------------|\n| `start`    | Sorting has started.                                                               | `group`, `draggedItem`, `sourceList`, `sourceIndex`                                                   |\n| `sort`     | Dragged item has been moved within a list. The list is referenced as `targetList`. | `group`, `draggedItem`, `sourceList`, `sourceIndex`, `targetList`, `oldTargetIndex`, `newTargetIndex` |\n| `move`     | Item has been dragged into a different list.                                       | `group`, `draggedItem`, `sourceList`, `sourceIndex`, `oldTargetList`, `newTargetList`, `targetIndex`  |\n| `end`      | Sorting has ended.                                                                 | `group`, `draggedItem`, `sourceList`, `sourceIndex`, `targetList`, `targetIndex`  |\n\n\n\n## Test helpers\n\n### trigger\n\n`trigger` is a low-level test helper that can be imported like this:\n\n```js\nimport trigger from \'ember-drag-sort/utils/trigger\'\n```\n\nIt accepts three arguments:\n\n| Argument    | Type                                     | Description                                                                                    |\n|:------------|:-----------------------------------------|:-----------------------------------------------------------------------------------------------|\n| `element`   | String, DOM element or jQuery collection | Selector or element to trigger an operation on.                                                |\n| `eventName` | String                                   | For list: `dragenter`; for list item: `dragstart`, `dragover` or `dragend`.                    |\n| `above`     | Boolean                                  | Only for `dragover`. Whether to put placeholder above (`true`) or below (`false`) target item. |\n\nThe order of operations is the following:\n\n1. `dragstart` on the element to drag.\n2. `dragenter` on target list that the dragged element should be moved into (optional).\n3. `dragover` on target element, the one that the dragged element should be dropped next to. Provide third argument to indicate above or below.\n4. `dragover` on the dragged element.\n\nAfter performing the operations, you must [wait for async behavior](https://guides.emberjs.com/v2.13.0/testing/testing-components/#toc_waiting-on-asynchronous-behavior).\n\nSee this addon\'s integration test for example.\n\n\n\n### sort\n\n`sort` is a page object component\n\nIf you want to provide custom descriptors for the `dragSortList` page object component, use the spread operator:\n\n```js\nimport {create, hasClass, visitable} from can describe its content. For example, you can describe the following template:\n\n```handlebars\n{{#drag-sort-list\n  items         = items\n  dragEndAction = (action \'dragEndAction\')\n  as |item|\n}}\n  <div class = \'drag-sort-item-content\'>\n    {{item.name}}\n  </div>\n{{/drag-sort-list}}\n```\n\n...by importing the page object component factory from `{dragSortList}` and passing your item description into it like this:\n\n```js\nimport {create, visitable} from \'ember-cli-page-object\'\nimport {dragSortList} from \'<your-app-name>/tests/pages/components/drag-sort-list\'\n\nexport default create({\n  visit:        visitable(\'/\'),\n  sortableList: dragSortList({\n    title: text()\n  })\n})\n```\n\nIn a test, list items are available as `sortableList.items()`. Item content is available as `sortableList.items(index).content`.\n\nFor example, to assert the title of the first item in a list, using the page object from the last example, you can do this:\n\n```js\nassert.equal(sortableList.items(0).content.title, "Foo")\n```\n\n\n\n#### Providing the drag handle selector\n\nIf you\'re using drag handles, you must pass a drag handle selector to the factory as the second argument:\n\n```js\nimport {create, visitable} from \'ember-cli-page-object\'\nimport {dragSortList} from \'<your-app-name>/tests/pages/components/drag-sort-list\'\n\nexport default create({\n  visit:        visitable(\'/\'),\n  sortableList: dragSortList({}, \'[draggable]\')\n})\n```\n\nIf you don\'t want to provide custom list item content, pass an empty object.\n\nThis assumes that every list item has a handle with the same selector.\n\nThe selector must be discoverable within list item content. In other words, the handle is accessed via `listItem.querySelector(handleSelector)`.\n\nIf you\'re describing a nested list, use the `>` combinator in your drag handle selector, so that drag handles of child items aren\'t selected.\n\n\n\n#### Sorting the dragSortList page object component\n\nInside your acceptance test, you can use the `sort` method on the `dragSortList` page object component.\n\nTo **rearrange items within a single list**, call `dragSortList.sort()` with three arguments:\n\n| Argument      | Type    | Required | Description                                                                                                                     |\n|:--------------|:--------|:---------|:--------------------------------------------------------------------------------------------------------------------------------|\n| `sourceIndex` | Integer | yes      | Zero-based index of the item to pick up.                                                                                        |\n| `targetIndex` | Integer | yes      | Zero-based index of the item to drop picked item on top of, calculated while the picked item is still on its original position. |\n| `above`       | Boolean | yes      | Whether to drop picked item above (`true`) or below (`false`) target item.                                                      |\n\nAfter executing `sort`, perform a wait using `await` or `andThen()`.\n\nExample:\n\n```js\ntest(\'sorting a list\', async function (assert) {\n  await page.visit()\n\n  const list = page.sortableList\n\n  await list.sort(0, 1, false)\n\n  const expectedTitles = [\'Bar\', \'Foo\', \'Baz\', \'Quux\']\n\n  assert.equal(list.items().count, 4)\n\n  expectedTitles.forEach((expectedTitle, k) => {\n    m = `List #0 item #${k} content title`\n    expect(list.items(k).content.title, m).equal(expectedTitle)\n  })\n}))\n```\n\n\n\nTo **move an item from one list to another**, call `dragSortList.move()` with four arguments:\n\n| Argument      | Type                  | Required                        | Description                                                                                                                                                                                                                                |\n|:--------------|:----------------------|:--------------------------------|:-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|\n| `sourceIndex` | Integer               | yes                             | Zero-based index of the item to pick up.                                                                                                                                                                                                   |\n| `targetList`  | Page object component | yes                             | The page object of the other sortable list component.                                                                                                                                                                                      |\n| `targetIndex` | Integer               | no                              | Zero-based index of the item to drop picked item on top of, calculated while the picked item is still on its original position. When omitted, adds item to the end of the target list. **Must** be omitted when moving into an empty list. |\n| `above`       | Boolean               | yes if `targetList` is provided | Whether to drop picked item above (`true`) or below (`false`) target item.                                                                                                                                                                 |\n\nAfter executing `sort`, perform a wait using `await` or `andThen()`.\n\nExample:\n\n```js\nconst list1 = page.sortableList1\nconst list2 = page.sortableList2\n\nawait list1.sort(0, list2, 1, false)\n```\n\nThis will pick the first item from `list1` and drop it below the second item of `list2`.\n\nSee this addon\'s acceptance test for example.\n\n\n\n## Development\n\n### Do not use npm, use yarn\n\nThis project uses [Yarn](https://yarnpkg.com/) to lock dependencies.
