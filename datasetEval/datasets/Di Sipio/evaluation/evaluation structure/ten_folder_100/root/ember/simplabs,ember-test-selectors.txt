b'ember-test-selectors\n==============================================================================\n\n[![Latest NPM release][npm-badge]][npm-badge-url]\n[![TravisCI Build Status][travis-badge]][travis-badge-url]\n\n[npm-badge]: https://img.shields.io/npm/v/ember-test-selectors.svg\n[npm-badge-url]: https://www.npmjs.com/package/ember-test-selectors\n[travis-badge]: https://img.shields.io/travis/simplabs/ember-test-selectors/master.svg?label=TravisCI\n[travis-badge-url]: https://travis-ci.org/simplabs/ember-test-selectors\n\nEnabling better element selectors in [Ember.js](http://emberjs.com) tests\n\nFeatures\n------------------------------------------------------------------------------\n\n- Removes attributes starting with `data-test-` from HTML tags and\n  component/helper invocations in your templates for production builds\n\n- Removes properties starting with `data-test-` from your JS objects like\n  component classes for production builds\n\n- Automatically binds properties starting with `data-test-` on all components\n  for development/testing builds\n\nMore information on why that is useful are available on our\n[blog](http://simplabs.com/blog/2016/03/04/ember-test-selectors.html)!\n\n[![ember-test-selectors](https://cloud.githubusercontent.com/assets/2922250/25236119/0cc8e13a-25b5-11e7-8a5b-f29589384833.png)\n](https://embermap.com/video/ember-test-selectors)\n\n\nCompatibility\n------------------------------------------------------------------------------\n\n- Ember 2.16 or above\n- Ember CLI 2.14 or above\n- Node.js 8 or above\n\n\nInstallation\n------------------------------------------------------------------------------\n\n```bash\nember install ember-test-selectors\n```\n\n\nUsage\n------------------------------------------------------------------------------\n\nIn your templates you are now able to use `data-test-*` attributes, which are\nautomatically removed from `production` builds:\n\n```hbs\n<article>\n  <h1 data-test-post-title data-test-resource-id={{post.id}}>{{post.title}}</h1>\n  <p>{{post.body}}</p>\n  <button data-test-like-button>Like</button>\n</article>\n```\n\nOnce you\'ve done that you can use attribute selectors to look up and interact\nwith those elements:\n\n```js\nassert.dom(\'[data-test-post-title]\').hasText(\'Ember is great!\');\n\nawait click(\'[data-test-like-button]\');\n```\n\n### Usage with Components\n\nYou can use the same syntax also for component invocations:\n\n```hbs\n<Spinner @color="blue" data-test-spinner>\n```\n\nInside the `Spinner` component template the `data-test-spinner` attribute will\nbe applied to the element that has `...attributes` on it, or on the component\nwrapper `div` element if you don\'t use `tagName = \'\'`.\n\n\n### Usage with Curly Components\n\nIf you still use the old curly invocation syntax for components you can pass\n`data-test-*` arguments to the components and they will automatically be bound\non the wrapper element too:\n\n```handlebars\n{{spinner color="blue" data-test-spinner=true}}\n```\n\nPlease note that the automatic argument binding only works for components based\non `@ember/component`, but not `@glimmer/component`.\n\n\n### Usage in Ember addons\n\nIf you want to use ember-test-selectors in an addon make sure that it appears\nin the `dependencies` section of the `package.json` file, not in the\n`devDependencies`. This ensures that the selectors are also stripped correctly\neven if the app that uses the addon does not use ember-test-selectors itself.\n\n\nConfiguration\n------------------------------------------------------------------------------\n\nYou can override when the `data-test-*` attributes should be stripped from the\nbuild by modifying your `ember-cli-build.js` file:\n\n```js\nvar app = new EmberApp({\n  \'ember-test-selectors\': {\n    strip: false\n  }\n});\n```\n\n`strip` accepts a `Boolean` value and defaults to `!app.tests`, which means\nthat the attributes will be stripped for production builds, unless the build\nwas triggered by `ember test`. That means that if you use\n`ember test --environment=production` the test selectors will still work, but\nfor `ember build -prod` they will be stripped out.\n\nLicense\n------------------------------------------------------------------------------\n\nember-test-selectors is developed by and &copy;\n[simplabs GmbH](http://simplabs.com) and contributors.
