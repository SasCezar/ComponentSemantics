b"# I2P\n\nThis is the source code for the reference Java implementation of I2P.\n\nLatest release: https://geti2p.net/download\n\n## Installing\n\nSee INSTALL.txt or https://geti2p.net/download for installation instructions.\n\n## Documentation\n\nhttps://geti2p.net/how\n\nFAQ: https://geti2p.net/faq\n\nAPI: http://docs.i2p-projekt.de/javadoc/\nor run 'ant javadoc' then start at build/javadoc/index.html\n\n## How to contribute / Hack on I2P\n\nPlease check out [HACKING.md](docs/HACKING.md) and other documents in the docs directory.\n\n## Building packages from source\n\nTo get development branch from source control: https://geti2p.net/newdevelopers\n\n### Prerequisites\n\n- Java SDK (preferably Oracle/Sun or OpenJDK) 1.7.0 or higher\n  - Non-linux operating systems and JVMs: See https://trac.i2p2.de/wiki/java\n  - Certain subsystems for embedded (core, router, mstreaming, streaming, i2ptunnel)\n    require only Java 1.6\n- Apache Ant 1.7.0 or higher\n- The xgettext, msgfmt, and msgmerge tools installed from the GNU gettext package\n  http://www.gnu.org/software/gettext/\n- Build environment must use a UTF-8 locale.\n\n### Ant build process\n\nOn x86 systems do:\n\n    ant pkg\n\nOn non-x86, use one of the following instead:\n\n    ant installer-linux\n    ant installer-freebsd\n    ant installer-osx\n\nRun 'ant' with no arguments to see other build options.\n\n### Gradle build process\n\nFull builds of installers or updates are not yet possible, but the code can be\ncompiled with:\n\n    ./gradlew assemble\n\nThis will download dependencies over the clearnet by default, including Gradle\nitself.
