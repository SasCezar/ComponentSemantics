b'# Microsoft Graph Toolkit\n\n<img align="left" height="150" src="https://github.com/microsoftgraph/microsoft-graph-toolkit/raw/master/assets/graff.png" title="Graff the Giraffe">\n\n[![NPM](https://img.shields.io/npm/v/@microsoft/mgt.svg)](https://www.npmjs.com/package/@microsoft/mgt) [![Licence](https://img.shields.io/github/license/microsoftgraph/microsoft-graph-toolkit.svg)](https://github.com/microsoftgraph/microsoft-graph-toolkit) [![code style: prettier](https://img.shields.io/badge/code_style-prettier-ff69b4.svg)](https://github.com/microsoftgraph/msgraph-sdk-javascript) [![stackoverflow](https://img.shields.io/stackexchange/stackoverflow/t/microsoft-graph-toolkit.svg)](https://stackoverflow.com/questions/tagged/microsoft-graph-toolkit)\n[![Build Status](https://dev.azure.com/microsoft-graph-toolkit/microsoft-graph-toolkit/_apis/build/status/microsoftgraph.microsoft-graph-toolkit?branchName=master)](https://dev.azure.com/microsoft-graph-toolkit/microsoft-graph-toolkit/_build/latest?definitionId=1&branchName=master) [![Published on webcomponents.org](https://img.shields.io/badge/webcomponents.org-published-blue.svg)](https://www.webcomponents.org/element/@microsoft/mgt)\n\nThe Microsoft Graph Toolkit is a collection of web components powered by the Microsoft Graph.\n\nComponents are functional and work automatically with the Microsoft Graph\n\nComponents work with any web framework and on all modern browsers. IE 11 is also supported\n\n[Here is a quick jsfiddle](https://jsfiddle.net/metulev/9phqxLd5/)\n\n## Components & Documentation\n\nThe toolkit currently includes the following components:\n\n* [mgt-login](https://docs.microsoft.com/graph/toolkit/components/login)\n* [mgt-person](https://docs.microsoft.com/graph/toolkit/components/person)\n* [mgt-person-card](https://docs.microsoft.com/graph/toolkit/components/person-card)\n* [mgt-people](https://docs.microsoft.com/graph/toolkit/components/people)\n* [mgt-people-picker](https://docs.microsoft.com/graph/toolkit/components/people-picker)\n* [mgt-agenda](https://docs.microsoft.com/graph/toolkit/components/agenda)\n* [mgt-tasks](https://docs.microsoft.com/graph/toolkit/components/tasks)\n\nAnd the following providers:\n\n* [Msal Provider](https://docs.microsoft.com/graph/toolkit/providers/msal)\n* [SharePoint Provider](https://docs.microsoft.com/graph/toolkit/providers/sharepoint)\n* [Teams Provider](https://docs.microsoft.com/graph/toolkit/providers/teams)\n* [Simple Provider](https://docs.microsoft.com/graph/toolkit/providers/custom)\n\n[View the full documentation](https://docs.microsoft.com/graph/toolkit/overview)\n\n## Getting Started\n\n[Watch the Getting Started Video](https://www.youtube.com/watch?v=oZCGb2MMxa0)\n\nYou can use the components by referencing the loader directly (via unpkg), or installing the npm package\n\n### Use via mgt-loader:\n\n```html\n<script src="https://unpkg.com/@microsoft/mgt/dist/bundle/mgt-loader.js"></script>\n```\n\nYou can then start using the components in your html page. Here is a full working example with the Msal provider:\n\n```html\n<script src="https://unpkg.com/@microsoft/mgt/dist/bundle/mgt-loader.js"></script>\n<mgt-msal-provider client-id="[CLIENT-ID]"></mgt-msal-provider>\n<mgt-login></mgt-login>\n\n<!-- <script>\n    // alternatively, you can set the provider in code and provide more options\n    mgt.Providers.globalProvider = new mgt.MsalProvider({clientId: \'[CLIENT-ID]\'});\n</script> -->\n```\n\n> NOTE: MSAL requires the page to be hosted in a web server for the authentication redirects. If you are just getting started and want to play around, the quickest way is to use something like [live server](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer) in vscode.\n\n### Use via NPM:\n\nThe benefits of using MGT through NPM is that you have full control of the bundling process and you can bundle only the code you need for your site. First, add the npm package:\n\n```bash\nnpm install @microsoft/mgt\n```\n\nNow you can reference all components at the page you are using:\n\n```html\n<script src="node_modules/@microsoft/mgt/dist/es6/components.js"></script>\n```\n\nOr, just reference the component you need and avoid loading everything else:\n\n```html\n<script src="node_modules/@microsoft/mgt/dist/es6/components/mgt-login/mgt-login.js"></script>\n```\n\nSimilarly, to add a provider, you can add it as a component:\n\n```html\n<script src="node_modules/@microsoft/mgt/dist/es6/components/providers/mgt-msal-provider.js"></script>\n\n<mgt-msal-provider client-id="[CLIENT-ID]"></mgt-msal-provider>\n```\n\nor, add it in your code:\n\n```html\n<script type="module">\n  import { Providers, MsalProvider } from \'@microsoft/mgt\';\n\n  Providers.globalProvider = new MsalProvider({ clientId: \'[CLIENT-ID]\' });\n</script>\n```\n\n## Providers\n\nThe components work best when used with a [provider](https://docs.microsoft.com/graph/toolkit/providers). The provider exposes authentication and Microsoft Graph apis used by the components to call into the Microsoft Graph.\n\nThe toolkit contains providers for [MSAL](https://docs.microsoft.com/graph/toolkit/providers/msal), [SharePoint](https://docs.microsoft.com/graph/toolkit/providers/sharepoint), and [Teams](https://docs.microsoft.com/graph/toolkit/providers/teams). You can also create your own providers by extending the [IProvider](https://docs.microsoft.com/graph/toolkit/providers/custom) abstract class.\n\n## Contribute\n\nWe enthusiastically welcome contributions and feedback.
