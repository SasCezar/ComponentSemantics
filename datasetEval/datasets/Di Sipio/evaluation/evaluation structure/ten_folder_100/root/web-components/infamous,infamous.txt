b'infamous\n[![All Contributors](https://img.shields.io/badge/all_contributors-2-orange.svg?style=flat-square)](#contributors)\n========\n\nCreate 3D CSS+WebGL apps declaratively with HTML.\n\n[Site](https://infamous.io) \xe2\x80\x94 [Forums](https://forums.infamous.io) \xe2\x80\x94 [Examples](https://infamous.io/docs/examples.html) \xe2\x80\x94 [Docs](https://infamous.io/docs/index.html)\n\n<h2> About </h2>\n\nInfamous gives you general-purpose [Custom HTML\nElements](https://developers.google.com/web/fundamentals/architecture/building-components/customelements)\nthat the browser understands, for defining 2D or 3D scenes rendered with CSS3D,\nWebGL, or both mixed together.\n\nIf you prefer imperative JavaScript, you can also use the imperative JavaScript\nAPI.\n\nInfamous\' "mixed mode" let\'s us combine traditional HTML elements with new ones\nthat render to WebGL (powered by Three.js), which means we can render\ntraditional elements and 3D objects like spheres and cubes (and soon any 3D\nmodel) together in the same 3D space (with lighting and shadow not just on the\nWebGL objects, but on the DOM elements too!).\n\nUnlike traditional CSS `box-shadow` which is flat and boring, with Infamous we\ncan give UIs __*real*__ shadows! [See for\nyourself!](https://codepen.io/trusktr/pen/rpegZR)\n\nInfamous is built on the [Web Component\nstandards](https://www.w3.org/standards/techs/components#w3c_all), making it\npossible to write 3D scenes declaratively using custom HTML elements,\nregardless of which view layer you prefer. This makes it possible for you write\n3D scenes using popular HTML frameworks like (but not limited to)\n[React](https://facebook.github.io/react), [Vue.js](https://vuejs.org),\n[Meteor](http://meteor.com), [Angular](https://angular.io),\n[Ember.js](https://www.emberjs.com), or even the great\n[jQuery](http://jquery.com).\n\nSupported browsers are Google Chrome, Mozilla Firefox, Opera, and Microsoft\nEdge. You\'re luck may vary with anything lower than Edge, though in theory this\nshould at least work in IE11 but it might require some additional transpile\nsteps and WebGL feature detection guards. CSS3D rendering may even work in IE10\nwith scenes that only have one level of nesting.
