b'# Base Components Recipes\n\nView examples for base components in small bites. Each recipe demonstrates a working example of the base components to build pages and apps quickly. Use and customize the base components and the recipes in your apps.\n\nBase component recipes open up the source code for the base components shown in https://developer.salesforce.com/docs/component-library. We transpiled the base components into the `c` namespace so that you can use the components in your projects. Explore the inner workings of the components and use the source code to build new components with your own requirements. The possibilities are endless with the source in your hands!\n\nBase components implement Lightning Design System and are developed using Lightning Web Components. A `c` namespace component can contain components in the `lightning` namespace.\n\n## Getting Started\n\nWe recommend using a scratch org to work with base component recipes on the Salesforce platform.\n\n1. Set up your environment by following the steps in the [Lightning Web Components Dev Guide](https://developer.salesforce.com/docs/component-library/documentation/lwc/lwc.install_setup_develop), which includes:\n\n    - Install the Salesforce CLI.\n    - Install Visual Studio Code and the Salesforce Extension Pack, or your own favorite code editor.\n    - Set up a Dev Hub org so you can work with Lightning web components in scratch orgs.\n\n2. Authenticate with your Dev Hub org and provide it with an alias, as shown by `mybaseorg` in the following command.\n\n```bash\nsfdx force:auth:web:login -d -a mybaseorg\n```\n\n3. Clone the base-component-recipes repository.\n\n```bash\ngit clone git@github.com:salesforce/base-components-recipes.git\ncd base-components-recipes\n```\n\n4. Create a scratch org and provide it with an alias, as shown by `base-recipes` in the following command.\n\n```bash\nsfdx force:org:create -s -f config/project-scratch-def.json -a base-recipes\n```\n\nIf you get an error "You do not have access to the [ScratchOrgInfo] object", make sure you have enabled your org as a Dev Hub org. See [Enable Dev Hub in Your Org](https://developer.salesforce.com/docs/atlas.en-us.sfdx_setup.meta/sfdx_setup/sfdx_setup_enable_devhub.htm). Alternatively, sign up for a Dev Hub org at https://developer.salesforce.com/promotions/orgs/dx-signup.\n\n5. Push the base-components-recipes code to your scratch org.\n\n```bash\nsfdx force:source:push\n```\n\n6. Open the scratch org.\n\n```bash\nsfdx force:org:open\n```\n\n7. In App Launcher, select the **Base Components** app. This app shows the base component recipes at [base-components-recipes/example/lwc](https://github.com/salesforce/base-components-recipes/tree/master/examples/lwc).\n\n## Use Base Components on Salesforce Platform\n\nCreate a helloWorld Lightning web component that uses a base component, `c-button`. We\'ll use Visual Studio Code in this example.\n\n1. In Visual Studio code, open the Command Palette by pressing **Ctrl+Shift+P** on Windows or **Cmd+Shift+P** on macOS.\n2. Type SFDX.\n3. Select SFDX: Create Lightning Web Component.\n4. Press Enter to accept the default `force-app/main/default/lwc` directory.\n5. Enter helloWorld for the name of the new component.\n6. Press Enter. A `helloWorld` bundle is created in `force-app/main/default/lwc`.\n7. In the HTML file, `helloWorld.html`, copy and paste the following code. Save the file.\n\n```html\n<!--helloWorld.html-->\n\n<template>\n    <c-button label="{greeting}" title="greeting"> </c-button>\n</template>\n```\n\n8.  In the JavaScript file, `helloWorld.js`, copy and paste the following code. Save the file.\n\n```javascript\n//helloWorld.js\n\nimport { LightningElement, api } from \'lwc\';\nexport default class HelloWorld extends LightningElement {\n    @api greeting = \'Hello World\';\n}\n```\n\n9. In the XML file, `helloWorld.js-meta.xml`, copy and paste the following code. Save the file.\n\n```xml\n<?xml version="1.0" encoding="UTF-8"?>\n<LightningComponentBundle xmlns="http://soap.sforce.com/2006/04/metadata">\n    <apiVersion>47.0</apiVersion>\n    <isExposed>true</isExposed>\n    <targets>\n        <target>lightning__AppPage</target>\n        <target>lightning__RecordPage</target>\n        <target>lightning__HomePage</target>\n  </targets>\n</LightningComponentBundle>\n```\n\n10. Push your changes.\n\n```bash\nsfdx force:source:push\n```\n\nYour helloWorld component is now ready for action. You can add this component to your apps and pages via the Lightning App Builder.\nFor more information, see the [Quick Start: Lightning Web Components](https://trailhead.salesforce.com/content/learn/projects/quick-start-lightning-web-components/create-a-hello-world-lightning-web-component) Trailhead project.\n\n## [Experimental] Use Base Components Outside of Salesforce Platform\n\n Base Components in the `c` namespace are intended for use on the Salesforce platform. However, we realize the desire to use them in non-Salesforce scenarios. Currently, many components will work, but others will not, due to dependencies on the Salesforce platform. Our goal is to provide Base Components for non-Salesforce use cases, and we\xe2\x80\x99ve created the [LWC Storybook PoC](https://github.com/reiniergs/lwc-storybooks-poc) to show what this might look like in the future. Please click the Watch button on the repo for updates as we move forward.\n\n## Documentation\n\nBase components in the `c` namespace map to components in the `lightning` namespace. Create custom labels that you can use in your Lightning web components.\n\n#### Update Labels\n\nTo update the Custom Labels metadata, go to the `force-app/main/default/labels directory`. The labels are available in the `CustomLabels.labels-meta.xml` file. For an example of a CustomLabels definition, see the [Metadata API Dev Guide](https://developer.salesforce.com/docs/atlas.en-us.api_meta.meta/api_meta/meta_customlabels.htm).\n\n#### Customize Labels\n\nTo customize your labels, from Setup, enter Custom Labels in the Quick Find box, then select **Custom Labels**. Our labels are shipped in English and can be translated to other languages. For more information, see the [Translate Labels](#translate-labels) section below and [Custom Labels](https://help.salesforce.com/articleView?id=cl_about.htm&type=5) in Salesforce Help.\n\n#### Use Labels\n\nImport the labels from the `@salesforce/label` scoped module. Reference your label name using the `c` namespace.\n\n```javascript\n// myComponent.js\n\nimport cardTitleLabel from \'@salesforce/label/c.LightningCard_cardTitle\';\nimport loadingLabel from \'@salesforce/label/c.LightningControl_loading\';\nimport { LightningElement } from \'lwc\';\n\nexport default class MyComponent extends LightningElement {\n    label = {\n        cardTitleLabel,\n        loadingLabel\n    };\n}\n```\n\nUse your labels in a Lightning web component.\n\n```html\n<!-- myComponent.html -->\n\n<template>\n    <c-card title="{label.cardTitleLabel}">\n        {label.loadingLabel}\n    </c-card>\n</template>\n```\n\n#### Translate Labels\n\nEnable the Translation Workbench to support translations in your org. See [Enable and Disable the Translation Workbench](https://help.salesforce.com/articleView?id=customize_wbench.htm) in Salesforce Help. Next, modify `sfdx-project.json` to include the path for the translation files, which are available in the `optional` folder.\n\n```json\n{\n    "packageDirectories": [\n        {\n            "path": "force-app",\n            "default": true\n        },\n        {\n            "path": "examples",\n            "default": false\n        },\n        {\n            "path": "optional"\n        }\n    ],\n    "namespace": "",\n    "sfdcLoginUrl": "https://login.salesforce.com",\n    "sourceApiVersion": "46.0"\n}\n```\n\nAfter updating `sfdx-project.json`, push the files to your scratch org.\n\n```bash\nsfdx force:source:push\n```\n\n## Contributing\n\nWe are not accepting contributions at this time.
