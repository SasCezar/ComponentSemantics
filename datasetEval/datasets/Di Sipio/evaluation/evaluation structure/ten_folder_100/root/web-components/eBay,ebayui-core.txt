Getting Started\n\nThe eBayUI core components are available as the `@ebay/ebayui-core` package on [NPM](https://www.npmjs.com/package/@ebay/ebayui-core).\n\nUse npm or yarn to add the package dependency to your project:\n\n```sh\nyarn add @ebay/ebayui-core\n```\n\n### Custom Tags\n\nOnce the package dependency is added, the eBay [customs tags](https://markojs.com/docs/custom-tags/) are now available for use in your Marko templates. For example, to use an `ebay-menu` component:\n\n#### _template.marko_\n\n```marko\n<ebay-menu text="Sort" type="radio">\n    <ebay-menu-item>Price</ebay-menu-item>\n    <ebay-menu-item>Time</ebay-menu-item>\n    <ebay-menu-item>Distance</ebay-menu-item>\n</ebay-menu>\n```\n\nOn Marko v3, remember to include the component\'s resources in your `browser.json` file.\n\n#### _browser.json_\n\n```js\n{\n    "dependencies": [\n        "@ebay/ebayui-core/ebay-menu"\n    ]\n}\n```\n\n### Attributes\n\nAttributes provide initial state for a component. We can see that the menu has `text` and `type` attributes:\n\n#### _template.marko_\n\n```marko\n<ebay-menu text="Sort" type="radio">\n    <ebay-menu-item>Price</ebay-menu-item>\n    <ebay-menu-item>Time</ebay-menu-item>\n    <ebay-menu-item>Distance</ebay-menu-item>\n</ebay-menu>\n```\n\nPassing new attributes to an ebayui component will always reset it\'s internal state. If you want to persist this state yourself, events are exposed which allow you to synchronize the state into your own components, for example:\n\n```marko\nclass {\n    onCreate() {\n        this.state = {\n            dialogIsOpen: false\n        }\n    }\n\n    handleDialogClose() {\n        this.state.dialogIsOpen = false;\n    }\n\n    handleDialogOpen() {\n        this.state.dialogIsOpen = true;\n    }\n}\n\n<ebay-dialog\n    open=state.dialogIsOpen\n    on-dialog-open(\'handleDialogOpen\')\n    on-dialog-close(\'handleDialogClose\')>\n    ...\n</ebay-dialog>\n```\n\n#### Pass-Through Attributes\n\nHTML attributes can be used on any component, and they will be passed through to the most prominent tag of the component. The most prominent tag is usually the root or form control, but individual components will note if it varies for specific cases.\n\nExample of static usage:\n\n```marko\n<ebay-button id="my-button"/>\n```\n\nFor using pass-through attributes dynamically, they should be sent through the `html-attributes` attribute:\n\n```marko\n$ const myAttributes = { id: \'my-button\' };\n<ebay-button html-attributes=myAttributes/>\n```\n\nStatic and dynamic pass-through attributes can be used simultaneously (html-attributes takes precedence in conflicts):\n\n```marko\n$ const myAttributes = { id: \'my-button\' };\n<ebay-button html-attributes=myAttributes type="submit"/>\n```\n\n### Events\n\nEvents can also be handled using [Marko syntax](https://markojs.com/docs/events/) (or the legacy syntax for [Marko v3](http://v3.markojs.com/docs/marko-widgets/taglib-api/#w-on)):\n\n#### _template.marko_\n\n```marko\n<ebay-menu text="Sort" type="radio" on-change("onMenuChange")>\n    <ebay-menu-item>Price</ebay-menu-item>\n    <ebay-menu-item>Time</ebay-menu-item>\n    <ebay-menu-item>Distance</ebay-menu-item>\n</ebay-menu>\n```\n\n## Releases &amp; Milestones\n\nFor upcoming roadmap and release history, please refer to our [releases](https://github.com/eBay/ebayui-core/releases) and [milestones](https://github.com/eBay/ebayui-core/milestones) pages.\n\n### Versioning\n\nThe ebayui-core package follows strict [Semantic Versioning](http://semver.org).\n\nGiven a version number MAJOR.MINOR.PATCH:\n\n* MAJOR version is incremented when we make incompatible API changes\n* MINOR version is incremented when we add functionality in a backwards-compatible manner\n* PATCH version is incremented when we make backwards-compatible bug fixes.\n\n### Issues\n\nPlease use our [issues page](https://github.com/eBay/ebayui-core/issues) to ask questions, report issues or submit feature requests.\n\nTo help track your issue, our admins will assign it with one or more coloured labels:\n\n* Black: Issue Type (e.g. bug, question, test case)\n* White: Resolution (e.g. wont fix, invalid, duplicate)\n* Gray: Status (e.g. backlog, in progress, help wanted)\n* Red: Blocker (e.g. dependency, discussion, design)\n* Green: Module (e.g. button, radio, dialog)\n* Blue: Aspect (e.g. build, documentation, website)\n* Yellow: Semver Guidance (e.g. breaking change, backwards compatible)\n* Purple: Sprint (e.g. sprint 1, sprint 2, etc)\n\n## Contributing\n\nLooking to contribute to eBay UI?
