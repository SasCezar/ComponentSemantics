b'# &lt;details-menu&gt; element\n\nA menu that\'s opened with a &lt;details> button.\n\n## Installation\n\n```\n$ npm install @github/details-menu-element\n```\n\n## Usage\n\n```js\nimport \'@github/details-menu-element\'\n```\n\n```html\n<details>\n  <summary>Robots</summary>\n  <details-menu role="menu">\n    <button type="button" role="menuitem">Hubot</button>\n    <button type="button" role="menuitem">Bender</button>\n    <button type="button" role="menuitem">BB-8</button>\n  </details-menu>\n</details>\n```\n\nUse `data-menu-button` and `data-menu-button-text` to update the button\'s text on menu item activation.\n\n```html\n<details>\n  <summary>Preferred robot: <span data-menu-button>None</span></summary>\n  <details-menu role="menu">\n    <button type="button" role="menuitem" data-menu-button-text>Hubot</button>\n    <button type="button" role="menuitem" data-menu-button-text>Bender</button>\n    <button type="button" role="menuitem" data-menu-button-text>BB-8</button>\n  </details-menu>\n</details>\n```\n\nUse `data-menu-button` and `data-menu-button-contents` to update the button\'s HTML content on menu item activation.\n\n```html\n<details>\n  <summary>Preferred robot: <span data-menu-button>None</span></summary>\n  <details-menu role="menu">\n    <button type="button" role="menuitem" data-menu-button-contents><img src="hubot.png"> Hubot</button>\n    <button type="button" role="menuitem" data-menu-button-contents><img src="bender.png"> Bender</button>\n    <button type="button" role="menuitem" data-menu-button-contents><img src="bb8.png"> BB-8</button>\n  </details-menu>\n</details>\n```\n\nUse `label[tabindex="0"][role=menuitemradio/menuitemcheckbox]` when dealing with radio and checkbox inputs menu items. Check states of the input element and the label will be synchronized.\n\n```html\n<details>\n  <summary>Preferred robot</summary>\n  <details-menu role="menu">\n    <label tabindex="0" role="menuitemradio">\n      <input type="radio" name="robot" value="Hubot"> Hubot\n    </label>\n    <label tabindex="0" role="menuitemradio">\n      <input type="radio" name="robot" value="Bender"> Bender\n    </label>\n    <label tabindex="0" role="menuitemradio">\n      <input type="radio" name="robot" value="BB-8"> BB-8\n    </label>\n  </details-menu>\n</details>\n```\n\n### Events\n\n- `details-menu-select` (cancelable) - fired on `<details-menu>` with `event.detail.relatedTarget` being the item to be selected.\n- `details-menu-selected` - fired on `<details-menu>` with `event.detail.relatedTarget` being the item selected, after label is updated and menu is closed.\n\n### Deferred loading\n\nMenu content can be loaded from a server by embedding an\n[`<include-fragment>`][fragment] element.\n\n[fragment]: https://github.com/github/include-fragment-element/\n\n```html\n<details>\n  <summary>Robots</summary>\n  <details-menu src="/robots" preload>\n    <include-fragment>Loading\xe2\x80\xa6</include-fragment>\n  </details-menu>\n</details>\n```\n\nThe `src` attribute value is copied to the `<include-fragment>` the first\ntime the `<details>` button is toggled open, which starts the server fetch.\n\nIf the `preload` attribute is present, the server fetch will begin on mouse\nhover over the `<details>` button, so the content may be loaded by the time\nthe menu is opened.\n\n## Browser support\n\nBrowsers without native [custom element support][support] require a [polyfill][].\n\n- Chrome\n- Firefox\n- Safari\n- Microsoft Edge\n\n[support]: https://caniuse.com/#feat=custom-elementsv1\n[polyfill]: https://github.com/webcomponents/custom-elements\n\n## Development\n\n```\nnpm install\nnpm test\n```\n\n## License\n\nDistributed under the MIT license.
