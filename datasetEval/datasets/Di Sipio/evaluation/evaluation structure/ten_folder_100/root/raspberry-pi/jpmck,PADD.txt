b'# PADD\n\nPADD (formerly Chronometer2) is a more expansive version of the original chronometer.sh that is included with [Pi-Hole](https://pi-hole.net). PADD provides in-depth information about your Pi-hole.\n\n![PADD](https://jpmck.com/img/blog/padd.png)\n\n## Setup PADD\n*More in-depth information about setting up PADD can be found in this repo\xe2\x80\x99s [wiki](https://github.com/jpmck/PADD/wiki/Setup).*\n\n- Get a copy of PADD by running:\n```bash\ncd ~\nwget -N https://raw.githubusercontent.com/jpmck/PADD/master/padd.sh\n```\n- Make PADD executable by running\n```bash\nsudo chmod +x padd.sh\n```\n- Set PADD to auto run by adding the following to the end of `~/.bashrc`:\n```bash\n# Run PADD\n# If we\xe2\x80\x99re on the PiTFT screen (ssh is xterm)\nif [ "$TERM" == "linux" ] ; then\n  while :\n  do\n    ./padd.sh\n    sleep 1\n  done\nfi\n```\nOne line version\n```bash\ncd ~ ; echo "if [ \\"\\$TERM\\" == \\"linux\\" ] ; then\\n  while :\\n  do\\n    ./padd.sh\\n    sleep 1\\n  done\\nfi" | tee ~/.bashrc -a\n```\n- Reboot your Pi-Hole by running `sudo reboot`. PADD should now run when your Pi-Hole has completed booting.\n\n## Updating PADD\n- Just run\n```bash\ncd ~\nwget -N https://raw.githubusercontent.com/jpmck/PADD/master/padd.sh\n```\n\n**Note: if you are already running Chronometer2 v1.3.1 or below, you\xe2\x80\x99ll need to follow [these instructions](https://github.com/jpmck/PADD/wiki/Updating-from-Chronometer2)!**\n\n## FAQ\n*Answers to frequently asked questions can be found in this repo\xe2\x80\x99s [wiki](https://github.com/jpmck/PADD/wiki/FAQ).*\n'
