b"# glslViewer [![Build Status](https://travis-ci.org/patriciogonzalezvivo/glslViewer.svg?branch=master)](https://travis-ci.org/patriciogonzalezvivo/glslViewer)\n\n![](images/00.gif)\n\n[![Donate](https://www.paypalobjects.com/en_US/i/btn/btn_donate_SM.gif)](https://www.paypal.com/cgi-bin/webscr?cmd=_s-xclick&hosted_button_id=4BQMKQJDQ9XH6)\n\nGlslViewer is a flexible console-base OpenGL Sandbox to display 2D/3D GLSL shaders without the need of an UI. You can definitely make your own UI or wrapper using the Python Module (include) or any other tool that communicates back/forth with glslViewer thought the standard POSIX console In/Out or OSC.\n\nGlslViewer support both 2D shaders and/or 3D shaders when a geometry (PLY, OBJ or GLTF) is provided.\n\n![](images/scene.png)\n\nGlslViewer gives support to:\n\n* fragment and vertex shaders on GLSL 120 (compatible with WebGL 1.0 and OpenGL ES 2.0)\n* resolve `#include` dependencies\n* adding/deleting `#defines`, plus a rich set of automatically generated set of defines.\n* adding/deleting uniforms (float, int, vec2, vec3 and vec4)\n* import of Textures\n* import of Cubemaps and spherical harmonics\n* import PLY, OBJ or GLTF files (and their dependencies)\n* default vert/frag shaders with PBR lighting model\n* hot reload of files on changes\n* One default light and one default camera\n* Interactive commands thought POSIX console IN/OUT or OSC\n* headless rendering and image export\n* PNG sequence export\n\n![](images/01.gif)\n\n## Wiki themes\n\n* What is GlslViewer [![Watch the There is not extra languages or meta data on top.
