b'QtAwesome - Font Awesome support for Qt applications\n====================================================\n\nDescription\n-----------\n\nQtAwesome is a simple library that can be used to add [Font Awesome](http://fortawesome.github.io/Font-Awesome/) icons to your [Qt application](http://qt-project.org/).\n\nNOTE: Though the name is QtAwesome and currently it\'s very Font Awesome based, you can use every other icon/glyph font you want.\n\nThe class can also be used to manage your own dynamic code-drawn icons, by adding named icon-painters.\n\n\nUpdated to FontAwesome 4.7.0\n----------------------------\n\nThis library has been updated to Font Awesome version **4.7.0**.\n\n* In the 4.5.0 version the _linux name has been changed to fa_linux. (Makes the naming of conflicting/invalid names more consistent, like fa_try and fa_500px)\n*  You can find the previous FontAwesome 4 c++11 library in the [c++11 branch](https://github.com/gamecreature/QtAwesome/tree/c++11).\n*  You can find the previous FontAwesome 3 library in the [fontawesome-3 branch](https://github.com/gamecreature/QtAwesome/tree/fontawesome-3).\n\n\n**Note about previous c++11**\n\nI removed the C++11 requirement. And moved the c++11 code to a c++11 branch.\nIt\'s not that I don\'t like c++11, but the typed enum made the code less flexible then it is now.\nJust integers it is. Simpler is better.\n\n\n\n\nInstallation\n------------\n\nThe easiest way to include QtAweome in your project is to copy the QtAwesome directory to your\nproject tree and add the following `include()` to your Qt project file:\n\n    include(QtAwesome/QtAwesome.pri)\n\nNow you are good to go!\n\n\nUsage\n-----\n\nYou probably want to create a single QtAwesome object for your whole application:\n\n````\n    QtAwesome* awesome = new QtAwesome( qApp )\n    awesome->initFontAwesome();     // This line is important as it loads the font and initializes the named icon map\n\n````\n\n* Add an accessor to this object (i.e. a global function, member of your application object, or whatever you like).\n* Use an icon name from the [Font Awesome Cheatsheet](http://fortawesome.github.io/Font-Awesome/cheatsheet/).\n\n\nExample\n--------\n\n```c++\n// You should create a single object of QtAwesome.\nQtAwesome* awesome = new QtAwesome( qApp );\nawesome->initFontAwesome();\n\n// Next create your icon with the help of the icon-enumeration (no dashes):\nQPushButton* beerButton new QPushButton( awesome->icon( fa::beer ), "Cheers!" );\n\n// You can also use \'string\' names to access the icons. (The string version omits the \'fa-\' or \'icon-\' prefix and has no dashes )\nQPushButton* coffeeButton new QPushButton( awesome->icon( "coffee" ), "Black please!" );\n\n// When you create an icon you can supply some options for your icons:\n// The available options can be found at the "Default options"-section\n\nQVariantMap options;\noptions.insert( "color" , QColor(255,0,0) );\nQPushButton* musicButton = new QPushButton( awesome->icon( fa::music, options ), "Music" );\n\n// You can also change the default options.\n// for example if you always would like to have green icons you could call)\nawesome->setDefaultOption( "color-disabled", QColor(0,255,0) );\n\n// You can also directly render a label with this font\nQLabel* label = new QLabel( QChar( fa::group ) );\nlabel->setFont( awesome->font(16) );\n\n```\n\nExample custom painter\n----------------------\n\nThis example registers a custom painter for supporting a duplicate icon (it draws 2 "plus marks"):\n\n```c++\nclass DuplicateIconPainter : public QtAwesomeIconPainter\n{\npublic:\n    virtual void paint( QtAwesome* awesome, QPainter* painter, const QRect& rectIn, QIcon::Mode mode, QIcon::State state, const QVariantMap& options  )\n    {\n        int drawSize = qRound(rectIn.height()*0.5);\n        int offset = rectIn.height() / 4;\n        QChar chr = QChar( static_cast<int>(fa::plus) );\n\n        painter->setFont( awesome->font( drawSize ) );\n\n        painter->setPen( QColor(100,100,100) );\n        painter->drawText( QRect( QPoint(offset*2, offset*2), QSize(drawSize, drawSize) ), chr , QTextOption( Qt::AlignCenter|Qt::AlignVCenter ) );\n\n        painter->setPen( QColor(50,50,50) );\n        painter->drawText( QRect( QPoint(rectIn.width()-drawSize-offset, rectIn.height()-drawSize-offset), QSize(drawSize, drawSize) ), chr , QTextOption( Qt::AlignCenter|Qt::AlignVCenter ) );\n\n    }\n};\n\nawesome->give("duplicate", new DuplicateIconPainter() );\n```\n\n\nDefault options:\n----------------\n\n  The following options are default in the QtAwesome class.\n\n```c++\nsetDefaultOption( "color", QColor(50,50,50) );\nsetDefaultOption( "color-disabled", QColor(70,70,70,60));\nsetDefaultOption( "color-active", QColor(10,10,10));\nsetDefaultOption( "color-selected", QColor(10,10,10));\n\nsetDefaultOption( "text", QString() );      // internal option\nsetDefaultOption( "text-disabled", QString() );\nsetDefaultOption( "text-active", QString() );\nsetDefaultOption( "text-selected", QString() );\n\nsetDefaultOption( "scale-factor", 0.9 );\n```\n\n  When creating an icon, it first populates the options-map with the default options from the QtAwesome object.\n  After that the options are expanded/overwritten by the options supplied to the icon.\n\n  It is possible to use another glyph per icon-state. For example to make an icon-unlock symbol switch to locked when selected,\n  you could supply the following option:\n\n```c++\n  options.insert("text-selected", QString( fa::lock ) );\n```\n\nColor and text options have the following structure:\n`keyname-iconmode-iconstate`\n\nWhere iconmode normal is empty\nAnd iconstate On is off.\n\nSo the list of items used is:\n\n- color\n- color-disabled\n- color-active\n- color-selected\n- color-off\n- color-disabled-off\n- color-active-off\n- color-selected-off\n- text\n- text-disabled\n- text-active\n- text-selected\n- text-off\n- text-disabled-off\n- text-active-off\n- text-selected-off\n\n\nLicense\n-------\n\nMIT License.
