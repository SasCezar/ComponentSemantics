b'# fontfaceonload\n\n[![Build Status](https://img.shields.io/travis/zachleat/fontfaceonload/master.svg)](https://travis-ci.org/zachleat/fontfaceonload)\n[![Dependency Status](https://david-dm.org/zachleat/fontfaceonload.svg)](https://david-dm.org/zachleat/fontfaceonload)\n[![devDependency Status](https://david-dm.org/zachleat/fontfaceonload/dev-status.svg)](https://david-dm.org/zachleat/fontfaceonload#info=devDependencies)\n\n## [Demo](http://zachleat.github.io/fontfaceonload/demo/demo.html)\n\n## Usage\n\nUse with any existing `@font-face` declaration.\n\n```css\n@font-face {\n    font-family: My Custom Font Family;\n    /* src and other properties as normal */\n}\n```\n\nInclude the library. Call the JavaScript.\n\n```js\nFontFaceOnload("My Custom Font Family", {\n    success: function() {},\n    error: function() {},\n    timeout: 5000 // in ms. Optional, default is 10 seconds\n});\n```\n\n### Use with Content Fonts\n\nTo allow the fallback font to be visible while the `@font-face` is loading, simply use `FontFaceOnload` like so:\n\n```js\nFontFaceOnload("My Custom Font Family", {\n    success: function() {\n        document.documentElement.className += " my-custom-font-family";\n    }\n});\n```\n\nand then use the class to scope your usage of the custom font:\n\n```css\nbody {\n    font-family: sans-serif;\n}\n.my-custom-font-family body {\n    font-family: My Custom Font Family, sans-serif;\n}\n```\n\nAn alternative approach that will also eliminate the FOIT as well as not require you to change your CSS is to use the [`loadCSS` library](https://github.com/filamentgroup/loadCSS#usage-example-with-content-fonts) to load the `@font-face` with a Data URI source block dynamically. `loadCSS` is smaller than `fontfaceonload`. The limitations to this approach include requiring you to manage which format to load (WOFF, WOFF2, TTF) and it will not work as well with icon fonts (since you need to a CSS class to style other sibling elements, like the descriptive text).\n\n### Use with Icon Fonts\n\nTo hide the fallback font so that weird fallback characters are not visible while the icon font is loading, use FontFaceOnload with the `glyphs` option like so:\n\n```js\nFontFaceOnload("My Custom Font Icon", {\n    success: function() {\n        document.documentElement.className += " my-custom-font-icon";\n    },\n    glyphs: "\\uE600\\uE601\\uE602\\uE605" // Optional, default is "". Useful for icon fonts: a few code points from your custom font icon.\n});\n```\n\nand then use the class to scope your usage of the custom font:\n\n```css\n.icon {\n    display: none;\n}\n.my-custom-font-family .icon {\n    display: inline-block;\n    font-family: My Custom Font Icon, sans-serif;\n}\n```\n\n## How it Works\n\nThis uses the [CSS Font Loading Module](http://dev.w3.org/csswg/css-font-loading/) when available (currently in Chrome 35+ and Opera 22+). When that isn\xe2\x80\x99t available, it uses a very similar approach to the one used in the [TypeKit Web Font Loader](https://github.com/typekit/webfontloader) (which is currently 7.1KB GZIP).\n\nBasically, it creates an element with a font stack including the web font and a default serif/sans-serif typeface.  It then uses a test string and measures the dimensions of the element at a certain interval. When the dimensions are different than the default fallback fonts, the font is considered to have loaded successfully.\n\nIf you\xe2\x80\x99d like a full polyfill for the CSS Font Loading Module, follow along with [Bram Stein\xe2\x80\x99s Font Loader](https://github.com/bramstein/fontloader). I believe the specification has changed since he launched this polyfill, but he\xe2\x80\x99s working on an updated version.\n\n## Building the project\n\nRun these commands:\n\n * `npm install`\n * `bower install`\n * `grunt init`\n * `grunt` as normal.\n\n## Configuring Grunt\n\nRather than one giant `Gruntfile.js`, this project is using a modular Grunt setup. Each individual grunt configuration option key has its own file located in `grunt/config-lib/` (readonly upstream configs, do not modify these directly) or `grunt/config/` (project specific configs). You may use the same key in both directories, the objects are smartly combined using [Lo-Dash merge](http://lodash.com/docs#merge).\n\nFor concatenation in the previous Gruntfile setup, you\xe2\x80\x99d add another key to the giant object passed into `grunt.initConfig` like this: `grunt.initConfig({ concat: { /* YOUR CONFIG */ } });`. In the new configuration, you\xe2\x80\x99ll create a `grunt/config/concat.js` with `module.exports = { /* YOUR CONFIG */ };`.\n\n## License\n\n[MIT License](http://en.wikipedia.org/wiki/MIT_License)\n\n## Alternatives\n\n* [Font Face Observer](https://github.com/bramstein/fontfaceobserver) by [@bram_stein](https://twitter.com/bram_stein)\n* [Zenfonts](https://github.com/gaborlenard/zenfonts) by [@GaborLenard](https://twitter.com/GaborLenard)\n'
