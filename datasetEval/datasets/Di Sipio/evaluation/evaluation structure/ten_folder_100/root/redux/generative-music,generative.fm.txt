b'# generative.fm\n\nA platform for playing [generative music](https://medium.com/@metalex9/introduction-to-generative-music-91e00e4dba11) in the browser.\n\nhttps://generative.fm\n\n## Overview\n\n### Audio\n\nThis site uses [Tone.js](https://tonejs.github.io/) to play audio in the browser through the [Web Audio API](https://developer.mozilla.org/en-US/docs/Web/API/Web_Audio_API).\n\n#### Samples\n\nMost of the samples are provided by the [Versilian Studios Chamber Orchestra 2 Community Edition](https://vis.versilstudios.com/vsco-community.html) and the [Versilian Community Sample Library](https://vis.versilstudios.com/vcsl.html). Other samples provided by the [Sonatina Symphonic Orchestra](https://github.com/peastman/sso). The [singing bowl samples](https://kasper.bandcamp.com/album/singing-bowls) were provided by [Kasper](https://kasper.bandcamp.com/). [Lo-Fi Hip-Hop Drum Kit samples are from ItsLucid](https://soundpacks.com/free-sound-packs/lo-fi-hip-hop-drum-kit/). All other samples were specially recorded for the site. Sample files can be found in the [generative-music/samples.generative.fm] repository.\n\n### Music\n\nSource code for the music generators can be found in the [generative-music/pieces-alex-bainter](https://github.com/generative-music/pieces-alex-bainter) repository.\n\n## Release Schedule\n\nSee [milestones](https://github.com/generative-music/generative.fm/milestones) for scheduled releases and to see which features and bug fixes will be included in each.\n\n## Issues and Feedback\n\nProblems or general feedback can be reported [here on Github](https://github.com/generative-music/site/issues) or by sending an email to [alex@alexbainter.com](mailto:alex@alexbainter.com?subject="Generative.fm"). Thanks!\n\n### Status\n\nYou can view which new features and bug fixes are currently planned, in progress, or waiting to be deployed to the site by visiting the [Generative.fm project board](https://github.com/orgs/generative-music/projects/2).\n\n## Installation\n\nYou will need [Node.js](https://nodejs.org/en/) installed on your machine.\n\n1. Clone to your machine.\n2. Navigate to the project\'s directory.\n3. Run `npm i`.\n4. Run `npm start`.\n5. Navigate to [localhost:9999](http://localhost:9999).\n\n### Local Sample Files\n\nRequests for sample files from the generative.fm sample CDN are blocked, see [#38](https://github.com/generative-music/generative.fm/issues/38). In order to play music locally, you will need to follow these instructions:\n\n1. Install the build prerequisites specified in [BUILD_README.md](https://github.com/generative-music/samples.generative.fm/blob/master/BUILD_README.md).\n2. Clone the [generative-music/samples.generative.fm] repository to a directory adjacent to the one this project was cloned to.\n3. Run `npm i`.\n4. Run `npm run build:samples`.\n\nThis will create the sample files on your machine to be served during local development.\n\n## Contributing\n\nSee [CONTRIBUTING.md](docs/CONTRIBUTING.md).\n\n[generative-music/samples.generative.fm]: https://github.com/generative-music/samples.generative.fm\n'
