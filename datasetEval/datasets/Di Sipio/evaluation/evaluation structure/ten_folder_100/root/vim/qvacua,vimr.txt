The goal is to build an editor that uses Neovim inside with many of the convenience GUI features similar to those present in modern editors. We mainly use Swift, but also use Objective-C when its C-nature helps.\n\nThere are other Neovim GUIs for macOS, e.g. [NyaoVim](https://github.com/rhysd/NyaoVim), [neovim-dot-app](https://github.com/rogual/neovim-dot-app), [Oni](https://www.onivim.io), etc., so why?\n\n- play around (obviously) with [Neovim](https://github.com/qvacua/neovim),\n- play around with Swift (and especially with [RxSwift](https://github.com/ReactiveX/RxSwift)) and\n- (most importantly) have fun!\n\n## (Reusable) Components\n\n* [RxMessagePort](https://github.com/qvacua/vimr/blob/develop/RxPack/RxMessagePort.swift): RxSwift wrapper for local and remote `CFMessagePort`.\n* [RxMsgpackRpc](https://github.com/qvacua/vimr/blob/develop/RxPack/RxMsgpackRpc.swift): implementation of MsgpackRpc using RxSwift.\n* [RxNeovimApi](https://github.com/qvacua/vimr/blob/develop/RxPack/RxNeovimApi.swift): RxSwift wrapper of Neovim API.\n* [NvimView](https://github.com/qvacua/vimr/tree/develop/NvimView): Cocoa Framework which bundles everything, e.g. Neovim\'s `runtime`-files, needed to embed Neovim in a Cocoa App. See the [wiki](https://github.com/qvacua/vimr/wiki/NvimView-Framework).\n\n---\n\nIf you want to support VimR financially, you can use [Bountysource](https://www.bountysource.com/teams/vimr). Big thanks to [all](https://www.bountysource.com/teams/vimr/backers) who did support: We list our spendings in the [wiki](https://github.com/qvacua/vimr/wiki/How-we-use-the-donations).\n\n## Download\n\nPre-built binaries can be found under [Releases](https://github.com/qvacua/vimr/releases).\n\n## Some Features\n\n* Basic input including Emojis and Hangul (+Hanja): We don\'t know whether other input systems work...\n* Markdown preview\n* Generic HTML preview (retains the scroll position when reloading)\n* Fuzzy file finder a la Xcode\'s "Open Quickly..."\n* Trackpad support: Pinching for zooming and two-finger scrolling.\n* Ligatures: Turned off by default. Turn it on in the Preferences.\n* Command line tool.\n* (Simple) File browser\n* Flexible workspace model a la JetBrain\'s IDEs\n\nWe will gradually create feature [issues](https://github.com/qvacua/vimr/issues) with more details. For the current status see the [project board](https://waffle.io/qvacua/vimr).\n\n## How to Build\n\nFirst after cloning the VimR source tree you need to initialize git submodules\n\n```bash\ngit submodule init\ngit submodule update\n```\n\nYou have to use Xcode 10.2. First install `homebrew`, then in the project root:\n\n```bash\nxcode-select --install # install the Xcode command line tools, if you haven\'t already\nbrew bundle\n\n./bin/build_vimr.sh # VimR.app will be placed in build/Build/Products/Release/\n```\n\nIf the build fails for some reason, do the following and build again:\n\n```\ncd NvimView/neovim\ngit reset --hard @\ngit clean -fd\nmake distclean\ncd ..\n./bin/build_vimr.sh\n```\n\n## Project Setup\n\n### Artifacts Hierarchy\n\n```\nVimR.app\n+-- NvimView.framework\n \xc2\xa0\xc2\xa0 +-- NVimView\n \xc2\xa0\xc2\xa0 +-- runtime files for Neovim\n \xc2\xa0\xc2\xa0 +-- NvimServer\n \xc2\xa0\xc2\xa0  \xc2\xa0\xc2\xa0 +-- libnvim\n \xc2\xa0\xc2\xa0  \xc2\xa0\xc2\xa0 +-- other libs for Neovim\n```\n\n### Libraries for Neovim\n\n* The library `libiconv` is linked by the linker flag `-liconv`. The version bundled with macOS will be linked.\n* The library `libintl` is pre-built in `third-party/libintl/lib` and linked by "Other Linker Flags" of the NvimServer target.\n* Other libraries used by Neovim, which are automatically built by building `libnvim`, are linked by "Other Linker Flags" of the NvimServer target.\n\n## License\n\n[MIT](https://github.com/qvacua/vimr/blob/master/LICENSE)\n\n---\n\nIf you are here for VimR-MacVim, use the [macvim/master](https://github.com/qvacua/vimr/tree/macvim/master) branch and the version [0.8.0 (32)](https://github.com/qvacua/vimr/releases/tag/v0.8.0-32).\n'
