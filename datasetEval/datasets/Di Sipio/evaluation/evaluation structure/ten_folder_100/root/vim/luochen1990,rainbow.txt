b'Rainbow Parentheses Improved \n===\n>\thelp you read complex code by showing diff level of parentheses in diff color !!\n\nDescription [(\xe8\xbf\x99\xe9\x87\x8c\xe6\x9c\x89\xe4\xb8\xad\xe6\x96\x87\xe7\x89\x88)](https://github.com/luochen1990/rainbow/blob/master/README_zh.md)\n---------------------------------------------------------------------------------------------------\n\nAs everyone knows, the most complex codes were composed of a mass of different kinds of parentheses (typically: lisp).\nThis plugin will help you read these codes by showing different levels of parentheses in different colors.\nYou can also find this plugin in **[www.vim.org](http://www.vim.org/scripts/script.php?script_id=4176)**.\n\n#### lisp\n![lisp](https://raw.githubusercontent.com/luochen1990/rainbow/demo/lisp.png)\n#### html\n![html](https://raw.githubusercontent.com/luochen1990/rainbow/demo/html.png)\n#### [more](https://github.com/luochen1990/rainbow/blob/demo/more.md)\n\n### What is improved ?\n\n- no limit of parentheses levels.\n- separately edit guifgs and ctermfgs (the colors used for highlighting).\n- now you can design your own parentheses  such as \'begin\' and \'end\'.\n- you can also configure anything separately for different types of files.\n- now you can even decide to let some operators (like + - * / , ==) highlighted with the parentheses together.\n- dot separated combined filetype support (`:h ft`).\n- json style configuration used, more understandable and readable, easier for advanced configuration.\n- the code is shorter and easier to read now.\n- smoother and faster.\n- the Chinese document is added.\n\n### Referenced:\n- http://www.vim.org/scripts/script.php?script_id=1561 (Martin Krischik)\n- http://www.vim.org/scripts/script.php?script_id=3772 (kien)\n\nInstall\n-------\n\n#### install via Plug:\n\n```vim\nPlug \'luochen1990/rainbow\'\nlet g:rainbow_active = 1 "set to 0 if you want to enable it later via :RainbowToggle\n```\n\n#### install manually:\n\n- first, execute the following commands (for windows users, use `~/vimfiles` instead of `~/.vim`)\n\n\t```sh\n\tgit clone https://github.com/luochen1990/rainbow.git\n\tcd rainbow\n\tcp plugin/* ~/.vim/plugin\n\tcp autoload/* ~/.vim/autoload\n\t```\n\n- second, add the follow sentences to your `.vimrc` or `_vimrc` :\n\n\t```vim\n\tlet g:rainbow_active = 1 "set to 0 if you want to enable it later via :RainbowToggle\n\t```\n\n- third, restart your vim and enjoy coding.\n\nConfigure\n---------\n\nThere is an example for advanced configuration, add it to your vimrc and edit it as you wish (just keep the format).\n\nNote: you can remove these lines safely since they are all included by the [source code](https://github.com/luochen1990/rainbow/blob/master/autoload/rainbow_main.vim)).\n\n```vim\nlet g:rainbow_conf = {\n\\\t\'guifgs\': [\'royalblue3\', \'darkorange3\', \'seagreen3\', \'firebrick\'],\n\\\t\'ctermfgs\': [\'lightblue\', \'lightyellow\', \'lightcyan\', \'lightmagenta\'],\n\\\t\'guis\': [\'\'],\n\\\t\'cterms\': [\'\'],\n\\\t\'operators\': \'_,_\',\n\\\t\'parentheses\': [\'start=/(/ end=/)/ fold\', \'start=/\\[/ end=/\\]/ fold\', \'start=/{/ end=/}/ fold\'],\n\\\t\'separately\': {\n\\\t\t\'*\': {},\n\\\t\t\'markdown\': {\n\\\t\t\t\'parentheses_options\': \'containedin=markdownCode contained\', "enable rainbow for code blocks only\n\\\t\t},\n\\\t\t\'lisp\': {\n\\\t\t\t\'guifgs\': [\'royalblue3\', \'darkorange3\', \'seagreen3\', \'firebrick\', \'darkorchid3\'], "lisp needs more colors for parentheses :)\n\\\t\t},\n\\\t\t\'haskell\': {\n\\\t\t\t\'parentheses\': [\'start=/(/ end=/)/ fold\', \'start=/\\[/ end=/\\]/ fold\', \'start=/\\v\\{\\ze[^-]/ end=/}/ fold\'], "the haskell lang pragmas should be excluded\n\\\t\t},\n\\\t\t\'vim\': {\n\\\t\t\t\'parentheses_options\': \'containedin=vimFuncBody\', "enable rainbow inside vim function body\n\\\t\t},\n\\\t\t\'perl\': {\n\\\t\t\t\'syn_name_prefix\': \'perlBlockFoldRainbow\', "solve the [perl indent-depending-on-syntax problem](https://github.com/luochen1990/rainbow/issues/20)\n\\\t\t},\n\\\t\t\'stylus\': {\n\\\t\t\t\'parentheses\': [\'start=/{/ end=/}/ fold contains=@colorableGroup\'], "[vim css color](https://github.com/ap/vim-css-color) compatibility\n\\\t\t},\n\\\t\t\'css\': 0, "disable this plugin for css files\n\\\t}\n\\}\n```\n\n- \'guifgs\': a list of `guifg` (`:h highlight-guifg`), i.e. colors for gui interface, will be used in order\n- \'guis\': a list of `gui` (`:h highlight-gui`), will be used in order\n- \'ctermfgs\': a list of `ctermfg` (`:h highlight-ctermfg`)\n- \'cterms\': a list of `cterm` (`:h highlight-cterm`)\n- \'operators\': describe the operators you want to highlight (note: be careful about special characters which needs escaping, you can find more examples [here](https://github.com/luochen1990/rainbow/issues/3), and you can also read the [vim help about syn-pattern](http://vimdoc.sourceforge.net/htmldoc/syntax.html#:syn-pattern)). note that this option will be overwritten by the `step` part of `parentheses`.\n- \'parentheses\': a list of parentheses definitions, a parentheses definition contains parts like `start=/(/`, `step=/,/`, `stop=/)/`, `fold`, `contained`, `containedin=someSynNames`, `contains=@Spell`, see `:h syntax` for more details. notice that the `step` part is defined by this plugin so it is not described by the official vim doc.\n- \'parentheses_options\': parentheses options shared between different parentheses, things like `containedin=xxxFuncBody`, `contains=@Spell` (or \'contains=@NoSpell\') often appears here. this option is often used to solve [3rd-party-plugin-compatibility]() problems.\n- \'separately\': configure for specific filetypes (decided by &ft), key `*` for filetypes without separate configuration, value `0` means disable rainbow only for this type of files, value `"default"` means keep the default shim for this filetype (notice: the default shim config will change between plugin version).\n- \'syn_name_prefix\': add a prefix to name of the syntax definition, this option is often used to solve [3rd-party-plugin-compatibility]() problems.\n- \'after\': execute some vim commands after the rainbow syntax rules is defined. it is often used like `[\'syn clear xxx\']` to solve [3rd-party-plugin-compatibility]() problems.\n- keep a field empty to use the default setting.\n\nTo get more advanced config examples, try to search throught this [tag](https://github.com/luochen1990/rainbow/issues?utf8=%E2%9C%93&q=label%3A%22config+reference%22+).\n\nUser Command\n------------\n\n- **:RainbowToggle**\t\t--you can use it to toggle this plugin.\n\n3rd Party Plugin Compatibility\n------------------------------\n\nYou should notice that this plugin is very special, Vim plugins is expected to provide syntax definitions vertically, i.e. one filetype, one syntax definition set:\n\n```\n----------------------------------------------------\n|      cpp       |      java       |    python     |\n|                |                 |               |\n| syn cppKeyword | syn javaKeyword | syn pyKeyword |\n| syn cppFunc    | syn javaFunc    | syn pyLambda  |\n| syn cppParen   | syn javaParen   | syn pyParen   |\n| ...            | ...             | ...           |\n----------------------------------------------------\n```\n\nBut this plugin provide syntax definitions horizontally, i.e. parentheses syntax for all filetypes:\n\n```\n----------------------------------------------------\n|      cpp       |      java       |    python     |\n|                |                 |               |\n| syn cppKeyword | syn javaKeyword | syn pyKeyword |\n| syn cppFunc    | syn javaFunc    | syn pyLambda  |\n| ...            | ...             | ...           |\n----------------------------------------------------\n|                     rainbow                      |\n|                                                  |\n| syn cppRainbow   syn javaRainbow   syn pyRainbow |\n----------------------------------------------------\n```\n\nYou can notice that, to provide rainbow parentheses, this plugin have to define it\'s own syntax rules, and these rules will overwrite the parentheses syntax provided by the filetype plugin.\n\nIt works well at most of the time, but in some special cases, when the parentheses syntax rule is depended somewhere else (e.g. indent, spell checking, nested syntax rules), the things depend on the original syntax rules will be broken.\n\nThis plugin has provide some mechanisms to solve the compatibility problems, and have provided default configurations to solve compatibility problems with the default vim syntax files.\n\nBut if this plugin is conflicted with some other plugins, you will probably have to solve them by yourself. First, you can search on [this issue tag](https://github.com/luochen1990/rainbow/issues?utf8=%E2%9C%93&q=label%3A%22confliction+with+other+third-party+syntax+plugins%22+) to find whether there is somebody else had the same problem and solved it. Second, you can read the following content about troubleshooting.\n\nTroubleshooting\n---------------\n\n- rainbow doesn\'t work at all: find out the current parentheses syntax name, and use the `after` mechanism to clear these syntax.\n- rainbow doesn\'t work inside some structure: find out the syntax region name, and use the `parentheses_options` mechanism like `\'containedin=xxx\'`.\n- spell checking doesn\'t work inside parentheses: use the `parentheses_options` mechanism like `\'contains=@Spell\'`.\n- spell checking works inside parentheses but you don\'t want it: use the `parentheses_options` mechanism like `\'contains=@NoSpell\'`.\n- indent not works correctly: check the indent script you are using and search `synID` inside it to find which syntax name prefix it depends on, and use the `syn_name_prefix` mechanism to solve it.\n\nThe following keymappings will help you to check the syntax name and definitions under the cursor, add them to your vimrc and restart vim:\n\n```\nnnoremap <f1> :echo synIDattr(synID(line(\'.\'), col(\'.\'), 0), \'name\')<cr>\nnnoremap <f2> :echo ("hi<" . synIDattr(synID(line("."),col("."),1),"name") . \'> trans<\'\n\\ . synIDattr(synID(line("."),col("."),0),"name") . "> lo<"\n\\ . synIDattr(synIDtrans(synID(line("."),col("."),1)),"name") . ">")<cr>\nnnoremap <f3> :echo map(synstack(line(\'.\'), col(\'.\')), \'synIDattr(v:val, "name")\')<cr>\nnnoremap <f4> :exec \'syn list \'.synIDattr(synID(line(\'.\'), col(\'.\'), 0), \'name\')<cr>\n```\n\nMove your cursor to a parentheses and press the keys to use them.\n\n------------------------------------------------------------------\n**Rate this script if you like it, and I\'ll appreciate it and improve this plugin for you because of your support!\n\nJust go to [this page](http://www.vim.org/scripts/script.php?script_id=4176) and choose `Life Changing` and click `rate`**\n'
