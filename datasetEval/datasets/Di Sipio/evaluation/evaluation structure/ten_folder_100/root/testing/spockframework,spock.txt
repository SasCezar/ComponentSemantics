Add https://jitpack.io as a respository\n2. Use `org.spockframework.spock` as `groupId` and the normal `artifact-id`\n\n```groovy\nrepositories {\n    // ...\n    maven { url 'https://jitpack.io' }\n}\n\ndependencies {\n        compile 'org.spockframework.spock:spock-core:spock-1.3'\n        compile 'org.spockframework.spock:spock-spring:spock-1.3'\n}\n```\n3. For intermediate releases you can also use the commit-hash as version, e.g. compile `com.github.spockframework.spock:spock-core:d91bf785a1`\n\nModules\n-------\n* spock-core -- Core framework. This is the only mandatory module.\n* spock-specs -- Specifications for spock-core, implemented using Spock. Not required for using Spock.\n* spock-spring -- Integration with the [Spring TestContext Framework](http://docs.spring.io/spring/docs/4.1.5.RELEASE/spring-framework-reference/html/testing.html#testcontext-framework).\n* spock-tapestry -- Integration with the [Tapestry 5](http://tapestry.apache.org/tapestry5/) IoC container.\n* spock-guice -- Integration with [Guice](http://code.google.com/p/google-guice/) 2/3.\n* spock-unitils -- Integration with [Unitils](http://www.unitils.org/).\n* spock-report -- Interactive, business-friendly HTML reports.\n\nBuilding\n--------\n\n### Supported versions\nSpock is supported for Java version 7, and 8 (with Groovy 2.5 Java 9+ are supported as well).\n\nSpock is supported for Groovy version 2.4+\n\nThe tests are testing spock with a specific versions (variants) of groovy. Default is groovy version 2.4\n\nThe groovy 2.4 and 2.5 variant should pass on all supported JDK versions:\n\n```\n./gradlew clean build\n```\n\n(Windows: `gradlew clean build`).\nAll build dependencies, including\nthe [build tool](http://www.gradle.org) itself, will be downloaded\nautomatically (unless already present).\n\nContributing\n------------\nContributions are welcome!
