b'# AiForms.Effects for Xamarin.Forms\n\nAiForms.Effects is the effects library that provides you with more flexible functions than default by targetting only Android and iOS in a Xamarin.Forms project.\n\n[Japanese](./README-ja.md)\n\n![Build status](https://kamusoft.visualstudio.com/NugetCI/_apis/build/status/AiForms.Effects)\n\n## Features\n* [Floating](#floating)\n    * arrange some floating views (e.g. FAB) at any place over a page.\n* [Feedback](#feedback)\n    * add touch feedback effect (color and sound) without command.\n* [AddTouch](#addtouch)\n    * add touch event ( begin, move, end, cancel ).\n* [SizeToFit](#sizetofit)\n    * make font size adjusted to fit the Label size.\n* [Border](#border)\n    * add border to a view.\n* [ToFlatButton](#toflatbutton)\n\t* alter Button to flat (for Android)\n* [AddText](#addtext)\n\t* add one line Text to a view.\n* [AddCommand](#addcommand)\n    * add Command function to a view.\n* [AddNumberPicker](#addnumberpicker)\n    * add NumberPicker function to a view.\n* [AddTimePicker](#addtimepicker)\n\t* add TimePicker function to a view.\n* [AddDatePicker](#adddatepicker)\n\t* add DatePicker function to a view.\n* [AlterLineHeight](#alterlineheight)\n    * alter LineHeight of Label and Editor.\n* [AlterColor](#altercolor)\n\t* alter Color of an element which it cannot change color.\n* [Placeholder](#placeholder)\n\t* show placeholder on Editor.\n\n## **Trigger Property (1.4.0~)**\n\nThough toggling an effect had used On property in less than ver.1.4.0, As of ver.1.4.0, an effect can be enabled by setting one or more main properties of the effect.\nThese properties are named "Trigger Properties".\n\n **Trigger properties** correspond to main properties such as Command and LongCommand in case of AddCommand Effect.\n\nIn this document,  trigger properties are written "trigger".\n\n\n### Old (~1.3.1)\n\n```xml\n<Label Text="Text" ef:AddCommand.On="true" ef:AddCommand.Command="{Binding GoCommand}" />\n```\nAlways needs On property.\n\n### New (1.4.0~)\n\n```xml\n<Label Text="Text" ef:AddCommand.Command="{Binding GoCommand}" />\n```\n\nNeed not On property when specified Trigger Property.\n\n### To keep traditional\n\nIf an effect had been used to dynamically toggle to with the On property specified, it may not correctly work on the trigger properties mode.\nTo keep traditional mode, you can disable trigger properties by specifying the static config property at any place in .NETStandard project as the following code.\n\n```csharp\nAiForms.Effects.EffectConfig.EnableTriggerProperty = false;\n```\n\n## Minimum Device and Version etc\n\niOS:iPhone5s,iPod touch6,iOS9.3  \nAndroid:version 5.1.1 (only FormsAppcompatActivity) / API22\n\n## Nuget Installation\n\n```bash\nInstall-Package AiForms.Effects\n```\n\nYou need to install this nuget package to .NETStandard project and each platform project.\n\n### for iOS project\n\nYou need to write the following code in AppDelegate.cs:\n\n```csharp\npublic override bool FinishedLaunching(UIApplication app, NSDictionary options) {\n    \n    global::Xamarin.Forms.Forms.Init();\n    AiForms.Effects.iOS.Effects.Init();  //need to write here\n\n    ...\n    return base.FinishedLaunching(app, options);\n}\n```\n\n### for Android global::Xamarin.Forms.Forms.Init(this, bundle);\n    AiForms.Effects.Droid.Effects.Init(); //need to write here\n    ...\n}\n```\n\n## Floating\n\nThis is the effect that arranges some floating views (e.g. FAB) at any place on a page.\nThe arranged elements are displayed more front than a ContentPage and are not affected a ContentPage scrolling.\n\n### How to use\n\nThis sample is that an element is arranged at above 25dp from the vertical end and left 25dp from the horizontal end.\n\n```xml\n<ContentPage xmlns:ef="clr-namespace:AiForms.Effects;assembly=AiForms.Effects">\n    \n    <ef:Floating.Content>\n        <ef:FloatingLayout>\n            <!-- This element is arranged at above 25dp from the vertical end and left 25dp from the horizontal end. -->\n            <ef:FloatingView \n                VerticalLayoutAlignment="End" \n                HorizontalLayoutAlignment="End"\n                OffsetX="-25" OffsetY="-25" >\n                 <!-- Code behind handling / ViewModel binding OK -->\n                 <Button Clicked="BlueTap" BackgroundColor="{Binding ButtonColor}" \n                         BorderRadius="28" WidthRequest="56" HeightRequest="56" \n                         Text="+" FontSize="24"\n                         TextColor="White" Padding="0" />\n            </ef:FloatingView>\n        </ef:FloatingLayout>\n    </ef:Floating.Content>\n\n    <StackLayout>\n        <Label Text="MainContents" />\n    </StackLayout>\n</ContentPage>\n```\n\n<img src="images/floating.jpg" width="600" /> \n\n### Property\n\n* Content (trigger)\n    * The root element to arrange floating views.\n    * This property is of type FloatingLayout.\n\n> Note that this effect doesn\'t work without trigger property because it hasn\'t an On property. \n\n### FloatingLayout\n\nThis is the Layout that can freely arrange several FloatingView\'s over a page.\n\n### FloatingView\n\nIt is a view that FloatingLayout arranges.\nThis view is used to specify HorizontalLayoutAlignment, VerticalLayoutAlignment, OffsetX, OffsetX and determine itself position.\nThis view can be set any VisualElements.\n\n#### Properties\n\n* HorizontalLayoutAlignment (defalut: Center)\n    * The horizontal position enumeration value (Start / Center / End / Fill)\n* VerticalLayoutAlignment (default: Center)\n    * The vertical position enumeration value (Start / Center / End / Fill)\n* OffsetX\n    * The adjustment relative value from the horizontal layout position. (without Fill)\n* OffsetY\n    * The adjustment relative value from the vertical layout position. (without Fill)\n* Hidden\n    * The bool value whether a view is hidden or shown.\n    * On Android, If IsVisible is false when a page is rendered, there is the issue that views are never displayed. This method is used to avoid the issue. If there is some problem using IsVisible, use this instead of.\n    * In internal, this property uses Opacity and InputTransparent properties.\n\n\n## Feedback\n\nThis is the effect that adds touch feedback effects (color and sound) to a view.\nThis effect can be made use of with others effect (for example, AddNumberPicker and AddDatePicker) simultaneously.\nHowever, AddCommand can\'t be used along with this effect because AddCommand contains this functions.\n\n### Properties\n\n* EffectColor (trigger)\n    * Touch feedback color. (default: transparent)\n* EnableSound (trigger)\n    * Touch feedback system sound. (default: false)\n\n## AddTouch\n\nThis is the effect that adds touch events (begin, move, end, cancel) to a view.\nEach touch events provides location property and can be taken X and Y position.\n\n### Properties\n\n* On\n    * Effect On / Off\n\n> Since this effect hasn\'t any trriger property, control by On property.\n\n### TouchRecognizer events\n\n* TouchBegin\n* TouchMove\n* TouchEnd\n* TouchCancel\n\n### Demo\n\nhttps://youtu.be/9zrVQcr_Oqo\n\n### How to use\n\nThis effect usage is a little different from the other effects.\nFirst of all, set On attached property to a target control and set the value to true in XAML.\n\n```xml\n<?xml version="1.0" encoding="UTF-8"?>\n<ContentPage \n    ...\n    xmlns:ef="clr-namespace:AiForms.Effects;assembly=AiForms.Effects">\n    <StackLayout HeightRequest="300" ef:AddTouch.On="true" x:Name="container" />\n</ContentPage>\n```\n\nIn turn, use AddTouch.GetRecognizer method, get a TouchRecognizer in code.\nThis recognizer can be used to handle each touch events.\n\n```cs\nvar recognizer = AddTouch.GetRecognizer(container);\n\nrecognizer.TouchBegin += (sender, e) => {\n    Debug.WriteLine("TouchBegin");\n};\n\nrecognizer.TouchMove += (sender, e) =>  {\n    Debug.WriteLine("TouchMove");\n    Debug.WriteLine($"X: {e.Location.X} Y:{e.Location.Y}"); \n};\n\nrecognizer.TouchEnd += (sender, e) => {\n    Debug.WriteLine("TouchEnd");\n};\n\nrecognizer.TouchCancel += (sender, e) => {\n    Debug.WriteLine("TouchCancel");\n};\n```\n\n## SizeToFit\n\nThis is the effect that make font size adjusted to fit the Label size.\nThis can be used only Label. \n\n### Properties\n\n* On\n    * Effect On/Off (true is On)\n* CanExpand\n    * Whether font size is expanded when making it fit. (Default true)\n    * If false, font size won\'t be expanded and only shrinked.\n\n> Since this effect hasn\'t any trriger property, control by On property.\n\n### Demo\n\nhttps://youtu.be/yMjcFOp38XE\n\n### How to write with Xaml\n\n```xml\n<ContentPage \n\txmlns="http://xamarin.com/schemas/2014/forms" \n\txmlns:x="http://schemas.microsoft.com/winfx/2009/xaml" \n\txmlns:ef="clr-namespace:AiForms.Effects;assembly=AiForms.Effects"\n\tx:Class="AiEffects.TestApp.Views.BorderPage">\n\t<Label Text="LongText..." ef:SizeToFit.On="true" ef.SizeToFit.CanExpand="false"\n\t\t\tHeightRequest="50" Width="200"  />\n</ContentPage>\n```\n\n## Border\n\nThis is the effect that add border to a view.  \nEntry, Picker, DatePicker and TimePicker on iOS have a border by default.  \nWhen specifying their width 0, it is possible that hide border.\n\n<img src="images/border_ios.gif" /> <img src="images/border_droid.gif" />\n\n### Properties\n\n* On\n\t* Effect On/Off (true is On)\n* Width (trigger)\n\t* Border width (default 0)\n* Color\n\t* Border color (default transparent)\n* Radius (trigger)\n\t* Border radius (default 0)\n\n### How to write with Xaml\n\n```xml\n<ContentPage \n\txmlns="http://xamarin.com/schemas/2014/forms" \n\txmlns:x="http://schemas.microsoft.com/winfx/2009/xaml" \n\txmlns:ef="clr-namespace:AiForms.Effects;assembly=AiForms.Effects"\n\tx:Class="AiEffects.TestApp.Views.BorderPage">\n\t<StackLayout Margin="4" \n        ef:Border.Width="2" ef:Border.Radius="6" ef:Border.Color="Red">\n\t\t<Label Text="hoge" />\n        <Label Text="fuga" />\n\t</StackLayout>\n</ContentPage>\n```\n\n### Limitations\n\n* On Android Entry, Picker, DatePicker, TimePicker\'s input underline is hidden if this effect attached.\n* On Android, Button is not displayed correctly. Use [ToFlatButton](#toflatbutton) for button.\n* On Android WebView, Frame, ScrollView are not supported.\n* On Android ListView and TableView overflow background from border.\n* Using AddCommand simultaneously is not supported.\n\n\n## ToFlatButton\n\nThis is the effect that alter Button to flat(only Android).  \nIf this effect is used, you will be able to design like iOS\'s Button.  \nAnd also this effect will enable BorderRadius, BorderWidth and BorderColor of default button properties to use by Android.\n\n<img src="images/toflat1.png" height="400" /> <img src="images/toflat2.png" height="400" />\n\n### Supported View\n\n* Button (Android)\n\n### Properties\n\n* On\n    * Effect On/Off (true is On)\n* RippleColor (trigger)\n\t* Ripple effect color.(default none)\n\n\n### How to write with Xaml\n\n```xml\n<Button Text="ButtonText" \n\tef:ToFlatButton.On="true" \n\tef:ToFlatButton.RippleColor="Red"\n\tBorderWidth="4" BorderColor="Green" BorderRadius="10" \n/>\n```\n\n## AddText\n\nThis is the effect that add one line text into a view.  \nIf you use this effect, for example you will be able to show a information that validations or character count etc.  \nYou will be able to change text position(top-left,top-right,bottom-left,bottom-right), text color,font size and margin by specifying property.\n\n<img src="images/addtext_ios.gif" /> <img src="images/addtext_droid.gif" />\n\n### Supported View\n\n* Label\n* Entry\n* Editor\n* StackLayout\n* AbsoluteLayout\n\nand more.\n\n### Properties\n\n* On\n\t* Effect On/Off (true is On)\n* Text (trigger)\n\t* added text\n* TextColor\n\t* Default Red\n* BackgroundColor\n\t* BackgroundColor of inner text view.\n\t* Default Transparent\n* FontSize\n\t* Default 8\n* Margin\n\t* Distance from a side of target view to inner text view.\n\t* Default 0,0,0,0\n* Padding\n\t* Padding of inner text view. \n\t* Default 0,0,0,0\n* HorizontalAlign\n\t* horizontal text position(Start or End). Default End.\n* VerticalAlign\n\t* vertical text position(Start or End). Default Start.\n\n### How to write with Xaml\n\n```xml\n<ContentPage \n\txmlns="http://xamarin.com/schemas/2014/forms" \n\txmlns:x="http://schemas.microsoft.com/winfx/2009/xaml" Text="{Binding Title}"\n\t\t\tef:AddText.On="true" ef:AddText.TextColor="Red" \n\t\t\tef:AddText.FontSize="10" ef:AddText.Margin="4,8,4,8" \n\t\t\tef:AddText.Padding="2,4,2,4" ef:AddText.BackgroundColor="#A0F0F0E0"\n\t\t\tef:AddText.HorizontalAlign="End"\n\t\t\tef:AddText.VerticalAlign="Start" \n\t\t\tef:AddText.Text="{Binding TitleMessage}" />\n\t</StackLayout>\n</ContentPage>\n```\n\n### Limitations\n\nWhen device rotates, text position will not be right in case android.\n\n## AddCommand\n\nThis Effect add Command function to a view.
