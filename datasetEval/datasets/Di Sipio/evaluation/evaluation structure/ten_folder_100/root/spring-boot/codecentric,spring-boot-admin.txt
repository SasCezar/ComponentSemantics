community project provides an admin interface for [Spring Boot <sup>\xc2\xae</sup>](http://projects.spring.io/spring-boot/ "Official Spring-Boot website") applications.\n\nIt provides the following features for registered applications.\n\n* Show health status\n* Show details, like\n  * JVM & memory metrics\n  * micrometer.io metrics\n  * Datasource metrics\n  * Cache metrics\n* Show build-info number\n* Follow and download logfile\n* View jvm system- & environment-properties\n* View Spring Boot Configuration Properties\n* Support for Spring Cloud\'s postable /env- &/refresh-endpoint\n* Easy loglevel management\n* Interact with JMX-beans\n* View thread dump\n* View http-traces\n* View auditevents\n* View http-endpoints\n* View scheduled tasks\n* View and delete active sessions (using spring-session)\n* View Flyway / Liquibase database migrations\n* Download heapdump\n* Notification on status change (via e-mail, Slack, Hipchat, ...)\n* Event journal of status changes (non persistent)\n\n## Getting Started\n\n[A quick guide](http://codecentric.github.io/spring-boot-admin/2.1.6/#getting-started) to get started can be found in our docs.\n\nThere are introductory talks Snapshot builds\nYou can access snapshot builds from the sonatype snapshot repository by adding the following to your `repositories`:\n```xml\n<repository>\n\t<id>sonatype-nexus-snapshots</id>\n\t<name>Sonatype Nexus Snapshots</name>\n\t<url>https://oss.sonatype.org/content/repositories/snapshots/</url>\n\t<snapshots>\n\t\t<enabled>true</enabled>\n\t</snapshots>\n\t<releases>\n\t\t<enabled>false</enabled>\n\t</releases>\n</repository>\n```\n\n##
