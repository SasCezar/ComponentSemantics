to discuss hosting under the\n[circe organization][circe-org] on GitHub.\n\n* [circe-benchmarks][circe-benchmarks]: Benchmarks for comparing the performance of circe and other JSON libraries for the JVM.\n* [circe-config][circe-config]: A library for translating between HOCON, Java properties, and JSON documents.\n* [circe-derivation][circe-derivation]: Experimental generic derivation with improved compile times.\n* [circe-fs2][circe-fs2]: A library that provides streaming JSON parsing and decoding built on [fs2][fs2] and [Jawn][jawn].\n* [circe-iteratee][circe-iteratee]: A library that provides streaming JSON parsing and decoding built on [iteratee.io][iteratee] and [Jawn][jawn].\n* [circe-jackson][circe-jackson]: A library that provides [Jackson][jackson]-supported parsing and printing for circe.\n* [circe-spray][circe-spray]: A library that provides JSON marshallers and unmarshallers for [Spray][spray] using circe.\n* [circe-yaml][circe-yaml]: A library that uses [SnakeYAML][snakeyaml] to support parsing YAML 1.1\n  into circe\'s `Json`.\n\n### Related projects\n\nThe following open source projects are either built on circe or provide circe support:\n\n* [Actor Messenger][actor-im]: A platform for instant messaging.\n* [akka-http-json][akka-http-json]: A library that supports using circe for JSON marshalling and\n  unmarshalling in [Akka HTTP][akka-http].\n* [akka-stream-json][akka-stream-json]: A library that provides JSON support for stream based applications using Jawn as a parser with a convenience example for circe.\n* [Argus][argus]: Generates models and circe encoders and decoders from JSON schemas.\n* [Blackdoor JOSE][jose-json-circe]: circe JSON support for blackdoor JOSE and JWT.\n* [circe-kafka][circe-kafka]: Implicit conversion of Encoder and Decoder into Kafka Serializer/Deserializer/Serde\n* [cornichon][cornichon]: A DSL for JSON API testing.\n* [Cosmos][cosmos]: An API for [DCOS][dcos] services that uses circe.\n* [crjdt][crjdt]: A conflict-free replicated JSON datatype in Scala.\n* [diffson][diffson]: A Scala diff / patch library for JSON.\n* [elastic4s][elastic4s]: A Scala client for [Elasticsearch][elasticsearch] with circe support.\n* [Enumeratum][enumeratum]: Enumerations for Scala with circe integration.\n* [Featherbed][featherbed]: A REST client library with circe support.\n* [Finch][finch]: A library for building web services with circe support.\n* [fintrospect][fintrospect]: HTTP contracts for [Finagle][finagle] with circe support.\n* [fluflu][fluflu]: A [Fluentd][fluentd] logger.\n* [Github4s][github4s]: A GitHub API wrapper written in Scala.\n* [content-api-models][guardian-content-api-models]: The Guardian\'s Content API Thrift models.\n* [http4s][http4s]: A purely functional HTTP library for client and server applications.\n* [IdeaLingua][izumi-r2]: Staged Interface Definition and Data Modeling Language & RPC system currently targeting Scala, Go, C# and TypeScript. Scala codegen generates models and JSON codecs using circe.\n* [Iglu Schema Repository][iglu]: A [JSON Schema][json-schema] repository with circe support.\n* [jsactor][jsactor]: An actor library for Scala.js with circe support.\n* [jwt-circe][jwt-circe]: A [JSON Web Token][jwt] implementation with circe support.\n* [kadai-log][kadai-log]: A logging library with circe support.\n* [msgpack4z-circe][msgpack4z-circe]: A [MessagePack][msgpack] implementation with circe support.\n* [ohNoMyCirce][ohNoMyCirce]: Friendly compile error messages for [shapeless][shapeless]\'s Generic, [circe][circe-org]\'s Encoder & Decoder and [slick][slick]\'s case class mapping.\n* [play-circe][play-circe]: circe support for [Play!][play].\n* [pulsar4s][pulsar4s]: A Scala client for [Apache-Pulsar][pulsar] with circe support.\n* [Rapture][rapture]: Support for using circe\'s parsing and AST in Rapture JSON.\n* [roc][roc]: A PostgreSQL client built on Finagle.\n* [sangria-circe][sangria-circe]: circe marshalling for [Sangria][sangria], a [GraphQL][graphql]\n  implementation.\n* [scalist][scalist]: A [Todoist][todoist] API client.\n* [scala-jsonapi][scala-jsonapi]:  Scala support library for integrating the JSON API spec with Spray, Play! or Circe\n* [scala-json-rpc]: [JSON-RPC][json-rpc] 2.0 library for Scala and Scala.js \n* [scalatest-json-circe]: Scalatest matchers for Json with appropriate equality and descriptive error messages.\n* [Scio][scio]: A Scala API for Apache Beam and Google Cloud Dataflow, uses circe for JSON IO\n* [seals][seals]: Tools for schema evolution and language-integrated schemata (derives circe encoders and decoders).\n* [shaclex][shaclex]: RDF validation using SHACL or ShEx. \n* [Slick-pg][slick-pg]: [Slick][slick] extensions for PostgreSQL.\n* [sttp][sttp]: Scala HTTP client.\n* [telepooz][telepooz]: A Scala wrapper for the [Telegram Bot API][telegram-bot-api] built on circe.\n* [Zenith][zenith]: Functional HTTP library built on circe.\n\n### Examples\n\nThe following projects provide examples, templates, or benchmarks that include circe:\n\n* https://github.com/alanphillips78/akka-http-microservice-blueprint\n* https://github.com/bneil/fcs_boilerplate\n* https://github.com/gvolpe/simple-http4s-api\n* https://github.com/vitorsvieira/akka-http-circe-json-template\n* https://github.com/stephennancekivell/some-jmh-json-benchmarks-circe-jackson\n* https://github.com/pauljamescleary/scala-pet-store\n\n## Contributors and participation\n\ncirce is a fork of [Argonaut][argonaut], and if you find it at all useful, you should thank\n[Mark Hibberd][markhibberd], [Tony Morris][tonymorris], [Kenji Yoshida][xuwei-k], and the rest of\nthe [Argonaut contributors][argonaut-contributors].\n\ncirce is currently maintained by [Travis Brown][travisbrown], [Alexandre Archambault][archambault],\nand [Vladimir Kostyukov][vkostyukov].
