the next set of JSON Schema IETF Internet Draft (I-D) documents.\nFor the latest released I-Ds, please see the [Specification page](http://json-schema.org/documentation.html) on the website.\n\n## Call for contributions and feedback\n\nReviews, comments and suggestions are most welcome!\nPlease read our [guidelines for contributing](CONTRIBUTING.md).\n\n## Status\nFor the current status of assigned based on [Sensible Github Labels](https://github.com/Relequestual/sensible-github-labels).\n\n## Contents\n\n* Makefile - scripts to build the Internet-Draft txt/html\n* _Internet-Draft sources_\n    * jsonschema-core.xml - source for JSON Schema\'s "core" I-D\n    * jsonschema-validation.xml - source for the validation vocabulary I-D\n    * jsonschema-hyperschema.xml - source for the hyper-schema vocabulary I-D\n    * relative-json-pointer.xml - source for the Relative JSON Pointer I-D\n* _meta-schemas and recommended output formats_\n    * schema.json - JSON Schema "core" and Validation meta-schema\n    * hyper-schema.json - JSON Hyper-Schema meta-schema\n    * links.json - JSON Hyper-Schema\'s Link Description Object meta-schema\n    * hyper-schema-output.json - The recommended output format for JSON Hyper-Schema links\n\nType "make" at a shell to build the .txt and .html spec files.\n\nDescriptions of the xml2rfc, I-D documents, and RFC processes:\n\n* https://xml2rfc.tools.ietf.org/authoring/draft-mrose-writing-rfcs.html\n* https://www.ietf.org/tao.html\n* https://www.ietf.org/ietf-ftp/1id-guidelines.html\n* https://www.rfc-editor.org/rfc/rfc7322.txt\n\n## Test suites\n\nConformance tests for JSON Schema and its vocabularies may be found\n[in their own
