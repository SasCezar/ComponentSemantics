b'## &lt;ngx-avatar&gt;\n[![npm version](https://badge.fury.io/js/ngx-avatar.svg)](https://badge.fury.io/js/ngx-avatar) [![npm](https://img.shields.io/npm/dt/ngx-avatar.svg)](https://www.npmjs.com/package/ngx-avatar) [![Build Status](https://travis-ci.org/HaithemMosbahi/ngx-avatar.svg?branch=master)](https://travis-ci.org/HaithemMosbahi/ngx-avatar)\n[![Angular Style Guide](https://mgechev.github.io/angular2-style-guide/images/badge.svg)](https://angular.io/styleguide)\n![size](https://img.shields.io/bundlephobia/minzip/ngx-avatar.svg)\n\nA universal avatar component for Angular 2+ applications that fetches / generates avatar based on the information you have about the user. The component has a fallback system that if for example an invalid Facebook ID is used it will try google ID and so on.\n\nYou can use this component whether you have a single source or a multiple avatar sources. In this case the fallback system will fetch the first valid avatar.\n\nMoreover, the component can shows name initials or simple value as avatar.\n\n![Angular Avatar component preview](https://cdn.rawgit.com/HaithemMosbahi/ngx-avatar/0bac9072/demo.png)\n\n\n Supported avatar sources:\n\n * Facebook\n * Google\n * Twitter\n * Vkontakte (VK)\n * Skype\n * Gravatar\n * Github\n * Custom image\n * name initials\n * value\n\n The fallback system uses the same order as the above source list, Facebook has the highest priority, if it fails, google source will be used, and so on.\n\n If you enjoy watching videos, check out this [tutorial](https://medium.com/letsboot/lets-play-with-ngx-avatar-ec585dc39161) on medium which explains how to use ngx-avatar in your angular application.\n \n Check out this [link](https://stackblitz.com/edit/ngx-avatar-demo) to play with ngx-avatar :grinning:\n\n## Installation\n\nInstall avatar component using [NPM](https://www.npmjs.com/)::\n\n```bash\n$ npm install ngx-avatar --save\n```\nOr [download as ZIP](https://github.com/HaithemMosbahi/ngx-avatar/archive/master.zip).\n\n\n## Usage\n\n1. Import AvatarModule :\n\nOnce you have installed ngx-avatar, you can import it in your `AppModule`:\n\n```typescript\nimport { BrowserModule } from \'@angular/platform-browser\';\nimport { NgModule } from \'@angular/core\';\nimport { HttpClientModule } from \'@angular/common/http\';\n\nimport { AppComponent } from \'./app.component\';\n\n// Import your AvatarModule\nimport { AvatarModule } from \'ngx-avatar\';\n\n@NgModule({\n  declarations: [\n    AppComponent\n  ],\n  imports: [\n    BrowserModule,\n    HttpClientModule,\n    // Specify AvatarModule as an import\n    AvatarModule\n  ],\n  providers: [],\n  bootstrap: [AppComponent]\n})\nexport class AppModule { }\n```\n\nStarting from version 3.4.0:\n- `HttpClientModule` is mandatory in order to fetch the avatar from external sources (Gravatar, Google, ...).\n\n2. Start using it:\n\nOnce the AvatarModule is imported, you can start using the component in your Angular application:\n\n```html\n<ngx-avatar></ngx-avatar>\n```\n## Examples\n\n```html\n<ngx-avatar For example, the avatar module comes with a set of default colors used to randomly fill the backgroud color of the avatar. Thus, it\'s possible to chnage the default list of colors and to pass your own list.\n\nAll you need to do is to configure the AvatarModule by calling **forRoot** method. The forRoot method takes an AvatarConfig Object that contains the overridden options. \n\nAvatarConfig interface has two properties: \n  * **avatarColors:** allows the user to override the default avatar colors by providing a new set of colors\n  * **sourcePriorityOrder:** allows the user to change the avatar source priority order. If you want the avatar component to look for user initials first, twitter before facebook or any order you want, this is can be done using the sourcePriorityOrder property\n\nThe following code shows an example on how to import the AvatarModule with your own source priority order. \nWith the given order, the avatar component will look first for the custom avatar image and then for user initials and after that it will look the rest of sources.\n\n```typescript\nimport { BrowserModule } from \'@angular/platform-browser\';\nimport { NgModule } from \'@angular/core\';\nimport { HttpClientModule } from "@angular/common/http";\n\nimport { AppComponent } from \'./app.component\';\nimport { UserService } from "./user.service";\nimport { AvatarModule, AvatarSource } from \'ngx-avatar\';\n\nconst avatarSourcesOrder = [AvatarSource.CUSTOM, AvatarSource.INITIALS];\n\n@NgModule({\n  declarations: [\n    AppComponent  ],\n  imports: [\n    BrowserModule,\n    HttpClientModule,\n    // import AvatarModule in your app with your own configuration\n    AvatarModule.forRoot({\n      sourcePriorityOrder: avatarSourcesOrder\n    })\n  ],\n  providers: [UserService],\n  bootstrap: [AppComponent]\n})\nexport class AppModule { }\n\n```\n\nHere\'s an example on how to import the AvatarModule with your own set of colors.\n\n* Starting from version 3.1, overriding the avatar configuration can be done as follows:\n\n```typescript\nimport { BrowserModule } from \'@angular/platform-browser\';\nimport { NgModule } from \'@angular/core\';\n\nimport { AppComponent } from \'./app.component\';\nimport { UserService } from "./user.service";\nimport { AvatarModule } from "ngx-avatar";\nimport { HttpClientModule } from "@angular/common/http";\n\nconst avatarColors = ["#FFB6C1", "#2c3e50", "#95a5a6", "#f39c12", "#1abc9c"];\n\n@NgModule({\n  declarations: [\n    AppComponent  ],\n  imports: [\n    BrowserModule,\n    HttpClientModule,\n    // import AvatarModule in your app with your own configuration\n    AvatarModule.forRoot({\n      colors: avatarColors\n    })\n  ],\n  providers: [UserService],\n  bootstrap: [AppComponent]\n})\nexport class AppModule { }\n\n```\n\n* Users who use a prior version of ngx-avatar ( < 3.1 ) can override the configuration as follows:\n\n```typescript\nimport { BrowserModule } from \'@angular/platform-browser\';\nimport { NgModule } from \'@angular/core\';\n\nimport { AppComponent } from \'./app.component\';\nimport { UserService } from "./user.service";\nimport { AvatarModule,AvatarConfig } from "ngx-avatar";\nimport { HttpClientModule } from "@angular/common/http";\n\nconst avatarConfig = new AvatarConfig([\'red\',\'blue\',\'pink\']);\n\n@NgModule({\n  declarations: [\n    AppComponent  ],\n  imports: [\n    BrowserModule,\n    HttpClientModule,\n    // import AvatarModule in your app with your own configuration\n    AvatarModule.forRoot(avatarConfig)\n  ],\n  providers: [UserService],\n  bootstrap: [AppComponent]\n})\nexport class AppModule { }\n\n```\n\n **Avatar Styling**\n\n In addition to the style attribute, ngx-avatar style can be customized using css classes. Thus, the generated code offers two css classes that can be overridden :\n * **avatar-container** : class that represents the avatar container - the host element. Styles in this class will be applied on the avatar wether is an image or text.\n * **avatar-content** : css class that represents the avatar element which is embedded inside the avatar-container.\n \n To overcome Angular\'s view encapsulation, you may need to use the /deep/ operator to target it. Here\'s an example that shows how to override ngx-avatar style : \n \n ```html\n   <ngx-avatar class="my-avatar" value="HM"> </ngx-avatar>\n ```\n Your css file might look like this \n \n ```css\n .my-avatar /deep/ .avatar-content {\n     background-color : red !important;\n }\n ```\n\n## Release Notes & History\n* 3.6.0: Angular 8 support\n* 3.5.0: export Avatar component for Angular elements and ng upgrade\n* 3.4.0: http module is removed from the library dependencies.
