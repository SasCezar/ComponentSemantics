Install Composer using detailed installation instructions [here](https://getcomposer.org/doc/00-intro.md#installation-linux-unix-osx)\n```\n$ wget https://getcomposer.org/composer.phar\n$ chmod +x composer.phar\n$ mv composer.phar /usr/local/bin/composer\n```\n2. Install Node.js using detailed installation instructions [here](https://nodejs.org/en/download/package-manager/)\n```\n$ yum install npm\n```\n3. Clone repository\n```\n$ git clone https://github.com/Labs64/laravel-boilerplate.git\n```\n4. Change into the working directory\n```\n$ cd laravel-boilerplate\n```\n5. Copy `.env.example` to `.env` and modify according to your environment\n```\n$ cp .env.example .env\n```\n6. Install composer dependencies\n```\n$ composer install --prefer-dist\n```\n7. An application key can be generated with the command\n```\n$ php artisan key:generate\n```\n8. Execute following commands to install other dependencies\n```\n$ npm install\n$ npm run dev\n```\n9. Run these commands to create the tables within the defined database and populate seed data\n```\n$ php artisan migrate --seed\n```\n\n## Troubleshooting\n\n- If you get an error like a `PDOException` try editing your `.env` file and change `DB_HOST=127.0.0.1` to `DB_HOST=localhost` or `DB_HOST=mysql` (for *docker-compose* environment).\n\n- If you get a password error try this command:\n```\n# ALTER USER \'root\'@\'%\' IDENTIFIED WITH mysql_native_password BY \'root\'; \n```\n\n## Run\n\nTo start the PHP built-in server\n```\n$ php artisan serve --port=8080\nor\n$ php -S localhost:8080 -t public/\n```\n\nNow you can browse the site at [http://localhost:8080](http://localhost:8080)  \xf0\x9f\x99\x8c\n\n## Docker\n\nHere is a Docker based local development environment prepared, which provides a very flexible and extensible way of building your custom Laravel applications.\n\n### What\'s Inside\nThis project is based on [docker-compose](https://docs.docker.com/compose/). By default, the following containers are started: _laravel-env (centos:7 based), mysql, nginx_. Additional containers (_phpmyadmin, mailhog_) are externalized into `docker-compose.utils.yml`. The `/var/www/laravel-boilerplate` directory is the web root which is mapped to the nginx container.\nYou can directly edit configuration files from within the repo as they are mapped to the correct locations in containers.\n\n<p align="center"><img src="https://raw.githubusercontent.com/Labs64/laravel-boilerplate/master/dockerfiles/img/laravel-boilerplate-docker.png" alt="Laravel Boilerplate Docker"></p>\n\n### System Requirements\nTo be able to run Laravel Boilerplate you have to meet the following requirements:\n* [docker](https://www.docker.com)\n* [docker-compose](https://docs.docker.com/compose/)\n\n### Run\n\n1. Clone repository\n```\n$ git clone https://github.com/Labs64/laravel-boilerplate.git\n```\n\n2. Copy `.env.example` to `.env` and modify according to your environment (make sure database host set to `DB_HOST=mysql`)\n```\n$ cp .env.example .env\n```\n\n3. Start environment\n```\n$ docker-compose up -d  # to start base containers\nor\n$ docker-compose -f docker-compose.yml -f docker-compose.utils.yml up -d  # to start base and utils containers\n```\n\n4. Build project\n```\n$ docker exec laravel-boilerplate_laravel-env_1 ./dockerfiles/bin/prj-build.sh\n or\n$ docker-compose run --rm laravel-boilerplate_laravel-env_1 ./dockerfiles/bin/prj-build.sh\n```\n\nNow you can browse the site at [http://localhost:80](http://localhost:80)  \xf0\x9f\x99\x8c\n\n---\n\n5. Stop environment\n```\n$ docker-compose down\n or\n$ docker-compose -f docker-compose.yml -f docker-compose.utils.yml down\n```\n\n## How To\'s & Modules configuration\n\nCheck out project [wiki](https://github.com/Labs64/laravel-boilerplate/wiki) pages for modules configuration and troubleshooting.\nFor more detailed instructions on how to use Laravel and it\'s extensions, check out the full Laravel [documentation](https://laravel.com/docs/).\n\n## How to contribute\n\nFork the repository, read the [CONTRIBUTE](CONTRIBUTE.md) file and make some changes.\nOnce you\'re done with your changes send a pull request and check [CI validation status](https://travis-ci.org/Labs64/laravel-boilerplate).\nThanks!\n\n### Contributors and Supporters\n\nThank you to all the [contributors](https://github.com/Labs64/laravel-boilerplate/graphs/contributors) on this project.
