About\nSockethub is a translation layer for web applications to communicate with other protocols and services that are traditionally either inaccessible or impractical to use from in-browser JavaScript.\n\nUsing [ActivityStream](http://activitystrea.ms/) (AS) objects to pass messages to and from the web app, Sockethub acts as a smart proxy server/agent, which can maintain state, and connect to sockets, endpoints and networks that would otherwise be restricted from an application running in the browser.\n\nOriginally inspired as a sister project to [RemoteStorage](https://remotestorage.io), and assisting in the development of [unhosted](http://unhosted.org) and [noBackend](http://nobackend.org) applications, Sockethub's functionality can also fit into a more traditional development stack, removing the need for custom code to handle various protocol specifics at the application layer.\n\nExample uses of Sockethub are:\n\n* Writing and receiving messages (SMTP, IMAP, Facebook, Twitter, ...)\n* Instant messaging (XMPP, IRC, MSN, FB Messenger, Hangouts, ...)\n* Discovery (WebFinger, RDF(a), ...)\n\nThe architecture of Sockethub is extensible and supports easy implementation of additional 'platforms' to carry out tasks.\n\n# Docs\n\nSee the [Sockethub wiki](https://github.com/sockethub/sockethub/wiki) for documentation.\n\n# Features\n\nWe use ActivityStreams to map the various actions of a platform to a set of AS '@type's which identify the underlying action. For example, using the XMPP platform, a friend request/accept cycle would use the activity stream types 'request-friend', 'remove-friend', 'make-friend'.\n\nBelow is a list of platform contexts we're currently working on and their types, both the completed and not yet implemented ones. examples.\n\n# Running using Docker Compose\n\nRequires [Docker Compose](https://docs.docker.com/compose/) 1.10.0+\n\n`$ docker-compose up`\n\n> If you\xe2\x80\x99re using Docker natively on Linux, Docker for Mac, or Docker for\n> Windows, then sockethub should now be listening on port 10550 on your Docker\n> daemon host. Point your web browser to http://localhost:10550 to find\n> sockethub. If this doesn\xe2\x80\x99t resolve, you can also try\n> http://0.0.0.0:10550.\n\n> If you\xe2\x80\x99re using Docker Machine on a Mac or Windows, use docker-machine ip\n> MACHINE_VM to get the IP address of your Docker host. Then, open\n> http://MACHINE_VM_IP:10550 in a browser.\n\n# Environment Variables\n\n* PORT\nDefaults to `10550`\n* HOST\nDefaults to `localhost`\n* DEBUG\nSpecify the namespace to console log, ie. `sockethub*` will print all sockethub related debug statements, whereas `*` will also print any other modules debug statements that use the `debug` module.\n\n* REDIS_PORT\nDefaults to `6379`\n* REDIS_HOST\nDefaults to `localhost`\n\n***OR***\n\n* REDIS_URL\nOverrides `REDIS_HOST` and `REDIS_PORT`, can specify a full redis connect URL (eq. `redis://username:password@host:port`)\n\n# Command-line params\n```\n  --help       : this help screen\n  --info       : displays some basic runtime info\n\n  --examples   : enabled examples page and serves helper files like jquery\n\n  --host       : hostname to bind to\n  --port       : port to bind to\n```\n\n# License\n\nSockethub is licensed under the [LGPL](https://github.com/sockethub/sockethub/blob/master/LICENSE)\n\n# Credits\n\nProject created and maintained by [Nick Jennings](http://github.com/silverbucket)\n\nLogo design by [Jan-Christoph Borchardt](http://jancborchardt.net)\n\nSponsored by [NLNET](http://nlnet.nl)\n\n[![NLNET Logo](http://sockethub.org/res/img/nlnet-logo.svg)](http://nlnet.nl)\n\n"
