b"# Nitter (WIP)\n\n[![Build Status](https://travis-ci.org/zedeus/nitter.svg?branch=master)](https://travis-ci.org/zedeus/nitter)\n\nA free and open source alternative Twitter front-end focused on privacy. \\\nInspired by the [invidio.us](https://github.com/omarroth/invidious) project.\n\n- No JavaScript or ads\n- All requests go through the backend, client never talks to Twitter\n- Prevents Twitter from tracking your IP or JavaScript fingerprint\n- Unofficial API (no rate limits or developer account required)\n- AGPLv3 licensed, no proprietary instances permitted\n- Themes\n- Lightweight (for [@nim_lang](https://twitter.com/nim_lang), 36KB vs 580KB from twitter.com)\n- Native RSS feeds\n- Mobile support (responsive design)\n\n## Todo (roughly in this order)\n\n- Embeds\n- More caching (waiting for [moigagoo/norm#19](https://github.com/moigagoo/norm/pull/19))\n- Simple account system with customizable feed\n- Json API endpoints\n- Nitter logo\n- Emoji support (WIP, uses native font for now)\n\n## Resources\n\nThe wiki contains a list of\n[Nitter instances](https://github.com/zedeus/nitter/wiki/Instances) and\na list of [browser extensions](https://github.com/zedeus/nitter/wiki/Extensions)\nmaintained by the community.\n\n## Why?\n\nIt's basically impossible to use Twitter without JavaScript enabled. If you try,\nyou're redirected to the legacy mobile version which is awful both functionally\nand aesthetically. For privacy-minded folks, preventing JavaScript analytics and\npotential IP-based tracking is important, but apart from using the legacy mobile\nversion and a VPN, it's impossible.\n\nUsing an instance of Nitter (hosted on a VPS for example), you can browse\nTwitter without JavaScript while retaining your privacy. In addition to\nrespecting your privacy, Nitter is on average around 15 times lighter than\nTwitter, and in some cases serves pages faster.\n\nIn the future a simple account system will be added that lets you follow Twitter\nusers, allowing you to have a clean chronological timeline without needing a\nTwitter account.\n\n## Screenshot\n\n![nitter](/screenshot.png)\n\n## Installation\n\nTo compile Nitter you need a Nim installation, see\n[nim-lang.org](https://nim-lang.org/install.html) for details. It is possible to\ninstall it system-wide or in the user directory you create below.\n\nYou also need to install `libsass` to compile the scss files. On Ubuntu and\nDebian, you can use `libsass-dev`.\n\n```bash\n# useradd -m nitter\n# su nitter\n$ git clone https://github.com/zedeus/nitter\n$ cd nitter\n$ nimble build -d:release\n$ nimble scss\n$ mkdir ./tmp\n```\n\nSet your hostname, port, page title and HMAC key in `nitter.conf`, then run\nNitter by executing `./nitter`. You should run Nitter behind a reverse proxy\nsuch as [Nginx](https://github.com/zedeus/nitter/wiki/Nginx) or Apache for\nbetter security.\n\nTo build and run Nitter in Docker:\n```bash\ndocker build -t nitter:latest .\ndocker run -v $(pwd)/nitter.conf:/src/nitter.conf -d -p 8080:8080 nitter:latest\n```\n\nA prebuilt Docker image is provided as well:\n```bash\ndocker run -v $(pwd)/nitter.conf:/src/nitter.conf -d -p 8080:8080 zedeus/nitter:latest\n```\n\nTo run Nitter via systemd you can use this service file:\n\n```bash\n[Unit]\nDescription=Nitter (An alternative Twitter front-end)\nAfter=syslog.target\nAfter=network.target\n\n[Service]\nType=simple\n\n# set user and group\nUser=nitter\nGroup=nitter\n\n# configure location\nWorkingDirectory=/home/nitter/nitter\nExecStart=/home/nitter/nitter/nitter\n\nRestart=always\nRestartSec=15\n\n[Install]\nWantedBy=multi-user.target\n```\n\nThen enable and run the service:\n`systemctl enable --now nitter.service`\n\n## Contact\n\nFeel free to join our Freenode IRC channel at #nitter, or our\n[Matrix server](https://riot.im/app/#/room/#nitter:matrix.org).\nYou can email me at zedeus@pm.me if you wish to contact me personally.\n"
