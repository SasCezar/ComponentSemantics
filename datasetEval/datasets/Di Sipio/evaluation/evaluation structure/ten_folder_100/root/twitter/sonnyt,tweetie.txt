b"Tweetie [![Build Status](https://travis-ci.org/sonnyt/tweetie.svg?branch=master)](https://travis-ci.org/sonnyt/tweetie)\n=======\n\njQuery Tweetie - Simple Twitter Feed Plugin that works with new Twitter API.\n\n# Install\n\nDownload the lastest version of [jQuery Tweetie](https://github.com/sonnyt/tweetie/releases/latest).\n\nIn order to successfully use jQuery Tweetie, you have to have server-side/backend API. The plugin comes with very simple PHP backend that you can easily setup and get going.\n\n# Use\nInitialize just like any other plugin.\n\n```js\n$('.tweets').tweetie();\n```\n\n# Settings\n\nProperty | Type | Description\n--- | --- | ---\nurl | String | ***Required.*** API URL.\ntype | String | ***Required.*** Fetch tweets via hashtags, search terms, username and lists. Available options [timeline](https://dev.twitter.com/rest/reference/get/statuses/user_timeline), [list](https://dev.twitter.com/rest/reference/get/lists/statuses), [search](https://dev.twitter.com/rest/reference/get/search/tweets) and [hashtag](https://dev.twitter.com/rest/reference/get/search/tweets).\ntemplate | String | ***Required.*** Template for each individual tweet. Learn more [here](#templating).\nparams | Object | ***Required.*** Parameters required to fetch collection of relevant tweets matching a specified query.  Only [timeline](https://developer.twitter.com/en/docs/tweets/timelines/api-reference/get-statuses-user_timeline), [list](https://developer.twitter.com/en/docs/accounts-and-users/create-manage-lists/api-reference/get-lists-statuses) and [search/hashtag](https://developer.twitter.com/en/docs/tweets/timelines/api-reference/get-statuses-home_timeline.html) params are accepted.\ndateFormat | String | Formating for `created_at` attribute. Learn more [here](#date-format).\n\n## Templating\n\nTweetie uses curly brace templating syntax. You can access any tweet properties using the `{{}}` double curly brase. For your convinece every `link`, `@username` and `#hashtag` in the tweet body is automatically hyper-linked.\n\n**Example**\n\n```handlebars\n{{tweet.text}} tweeted by {{tweet.user.screen_name}} at {{tweet.created_at}}\n```\n\n## Date Format\nFormat | Example | Description\n--- | --- | ---\n`%d` | `1..31` | Day of month\n`%m` | `1..12` |\tMonth number\n`%B` | `January..December` | Month name\n`%b` | `Jan..Dec`| Month name abbreviated\n`%Y` | `2018` | 4 digit year\n`%y` | `18` | 2 digit year\n\nDate format automatically applies to `created_at` template attribute.\n"
