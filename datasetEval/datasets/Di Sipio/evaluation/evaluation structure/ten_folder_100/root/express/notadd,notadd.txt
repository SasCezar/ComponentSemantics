b'<p align="center"><img src="https://www.notadd.com/src/notado_logo420x96.svg" alt="Notadd Logo."></p>\n\n# Overview\n\n[\xe4\xb8\xad\xe6\x96\x87\xe6\x96\x87\xe6\xa1\xa3](./README_zh.md)\n\nNotadd is an open source, Nest.js framework-based microservice development architecture that allows you to build a microservices system using the right modules and addons for different business needs. Notadd officially provides an abstract public service layer. Within the service layer, each module provides the Grpc interface for the Notadd main program to call. For example, a CMS system, you can use the officially provided `nt-module-cms` and `nt-module-user` modules as the underlying service layer. Then use the Notadd main program to write your API layer code according to the `protobuf` message protocol defined by the service layer.\n\n# Features\n\n- **[Microservice]** Supports stand-alone deployment and microservice\n- **[High Performance]** Asynchronous high-performance applications, tens of thousands of concurrent\n- **[Easy to maintain]** Developed with Typescript, intelligent code hints and compile-time code checking mechanisms\n- **[Pluggable]** modular development system, according to business needs, select the appropriate module, build the API layer\n\n## Technology stack\n\n- Typescript\n- Nest.js\n- GraphQL\n- TypeORM\n- Grpc\n- Redis\n\n# System Architecture\n\n![notadd-arch](https://www.notadd.com/src/notadd-arch.svg)\n\n## Modular design\n\nEnterprise Official Website: CMS module + neditor plug-in, message board plugin\n\nInformation release: CMS module, user module + CMS multi-user plugin, Neditor plugin\n\nWeChat Mall: User module, Mall module, WeChat module + WeChat big turntable, payment plug-in, offline verification plug-in\n\nDining plan: User module, Mall module, WeChat module + ordering plugin, scan code payment plug-in, passenger flow monitoring plug-in ... + infrared sensor development, WiFi probe expansion\n\nHotel Program: User module, Hotel module, WeChat module + booking plugin, payment plug-in, smart WiFi plugin + WiFi probe expansion, door card system expansion\n\nCRM system: User module, CRM module ...\n\nMore to imagine ...\n\n# Quick Start\n\n1. Clone Rpc sample service to the local [nt-rpc-demo](https://github.com/notadd/nt-rpc-demo)\n2. Clone the user service to the local [nt-module-user](https://github.com/notadd/nt-module-user)\n3. Start the microservice according to the instructions of `nt-rpc-demo` and `nt-module-user`\n4. Clone this project to your local\n5. Installation depends on `yarn install`\n6. Start `yarn start`\n7. Open a browser and go to `localhost:5000/graphql`\n8. Test GraphQL API\n\n> Note: The Notadd main program provides demo code at this stage, and does not rule out the removal of all graphql api code later.\n\n## Module list\n\n- [nt-module-user](https://github.com/notadd/nt-module-user) user module\n- [nt-module-cms](https://github.com/notadd/nt-module-cms)  CMS module\n\n## Addon list\n\n- [nt-addon-pay](https://github.com/notadd/nt-addon-pay) payment addon\n- [nt-addon-wechatapi](https://github.com/notadd/nt-addon-wechatapi) wechatapi addon\n\n# Contribution\n\nWelcome to Pull requests.
