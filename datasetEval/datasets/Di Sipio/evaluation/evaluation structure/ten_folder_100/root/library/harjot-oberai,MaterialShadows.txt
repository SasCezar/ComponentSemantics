b'<div align="center"><img src="/screens/cover.png"/></div>\n\n# MaterialShadows\n[![Platform](https://img.shields.io/badge/platform-Android-yellow.svg)](https://www.android.com)\n[![API](https://img.shields.io/badge/API-21%2B-brightgreen.svg?style=flat)](https://android-arsenal.com/api?level=21)\n[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)\n\nA library for seamlessly integrating Material shadows. \nThe library takes existing material shadows to next level by adding the following features :\n\n- <b>Convex shadows</b> : The shadows are not only rectangular or circular, they can take any convex shape depending on the view and its content.\n- <b>Support for shadow offsets</b> : The library allows developers to set <b>X</b> and <b>Y</b> offset for the shadows.\n- <b>Support for shadow intensity</b> : The library also has support for setting shadow intensity via `shadowAlpha` attribute.\n- <b>Shadows for semi-transparent views</b> : The library allows shadows for semi-transparent views.\n- <b>Support for Async Shadow calculations</b> : The library allows the operations to be async to avoid blocking the UI thread for long calculations.\n- <b>Shadow animations</b> : The library supports fade out animation for shadow.\n\n# Usage\nJust add the following dependency in your app\'s `build.gradle`\n```groovy\ndependencies {\n      implementation \'com.sdsmdg.harjot:materialshadows:1.2.5\'\n}\n```\n\n# Third Party Bindings\n<b>[React Native](https://github.com/facebook/react-native)</b><br>\nYou may now use this library with `react-native` via the module [here](https://github.com/prscX/react-native-material-shadows), built by [Pranav Raj Singh Chauhan](https://github.com/prscX).\n\n# How does this work ?\nThe `MaterialShadowViewWrapper` is an extension of `Relative Layout`. The `MaterialShadowFrameLayoutWrapper` is an extension of `FrameLayout`. Use any one of them as per your convenience.\n\nAll the child views go through the same process of generating shadow as given below : \n1. First a bitmap is generated from the drawing cache of the view.\n2. The bitmap is traversed pixel by pixel to remove all transparent pixels and get a list of points corresponding to the actual outline of the content of the view.\n3. Since the points corresponding to outline may give a concave path, hence <b>GrahamScan algorithm</b> is used to generate a convex hull of the outline points.\n4. A path is created from the points of the resulting convex hull.\n5. This path is passed to a `CustomViewOutlineProvider` object that is later attached to the view itself.\n6. Hence we get a convex shadow for any type of view based on its content.\n\n<b>P.S. : </b> All the calculations related to graham scan are done asynchronously by default. This behavior can be controlled by `calculateAsync` parameter. (Thanks [Yaroslav!](https://github.com/yarolegovich))\n\n# Example Usage 1 (Simple)\n#### XML\n```xml\n<com.sdsmdg.harjot.materialshadows.MaterialShadowViewWrapper\n `ShadowGenerator.java` encapsulates all the code related to the generation of convex shadows, it is really easy to plug in convex shadows with any custom ViewGroup or some platform ViewGroup like `LinearLayout` etc.\n\n```java\npublic class CustomShadowWrapper extends CustomViewGroup {\n      \n      ShadowGenerator shadowGenerator;\n      \n      @Override\n      protected void onLayout(boolean changed, int l, int t, int r, int b) {\n          super.onLayout(changed, l, t, r, b);\n          if (shadowGenerator == null) {\n              shadowGenerator = new ShadowGenerator(this, offsetX, offsetY, shadowAlpha, shouldShowWhenAllReady, shouldCalculateAsync, shouldAnimateShadow, animationDuration);\n          }\n          shadowGenerator.generate();\n      }\n    \n      @Override\n      protected void onDetachedFromWindow() {\n          super.onDetachedFromWindow();\n          if (shadowGenerator != null) {\n              shadowGenerator.releaseResources();\n          }\n      }\n      \n}\n```\n\n<b>Note : </b> Make sure to define all the 7 parameters required for `ShadowGenerator`, namely `offsetX`, `offsetY`, `shadowAlpha`, `shouldShowWhenAllReady`, `shouldCalculateAsync`, `shouldAnimateShadow`, `animationDuration` in the custom wrapper and handle changes in their values.\n\nIn case any parameter value changes, say `OffsetX`, add the following code inside the setter method for `OffsetX` :\n```java\npublic void setOffsetX(float offsetX) {\n      this.offsetX = offsetX;\n      if (shadowGenerator != null) {\n            shadowGenerator.setOffsetX(offsetX);\n      }\n}\n```\n\nIf you want to add custom XML attributes with your `CustomShadowWrapper` class, add this to `attrs.xml` ([here](/materialshadows/src/main/res/values/attrs.xml)) and handle them in your `CustomShadowWrapper` class accordingly.\n```xml\n<declare-styleable name="CustomShadowWrapper">\n        <attr name="shadowOffsetX" />\n        <attr name="shadowOffsetY" />\n        <attr name="shadowAlpha" />\n\n        <attr name="calculateAsync" />\n        <attr name="showWhenAllReady" />\n        <attr name="animateShadow" />\n        <attr name="animationDuration" />\n    </declare-styleable>\n```\n\nSee [MaterialShadowViewWrapper](/materialshadows/src/main/java/com/sdsmdg/harjot/materialshadows/MaterialShadowViewWrapper.java) for more details.\n\n# Documentation\n|XML attribute         |Java set Since the bitmap is traversed pixel by pixel, the performance for large views is bad. Hence the use of the library is limited to small views.\n2. Currently the shadow is generated only for direct children of the `MaterialShadowViewWrapper`. Hence if the desired views are placed inside a Linear Layout or some other view group, then each view must be wrapped by a seperate `MaterialShadowViewWrapper` or `MaterialShadowFrameLayoutWrapper` or a custom view group wrapper may be implemented.\n3. Each child of `MaterialShadowViewWrapper` or custom view group wrapper is assigned the same offset and shadow intensity. If fine control over every view\'s shadow is required then it must be wrapped inside its own `MaterialShadowViewWrapper` or `MaterialShadowFrameLayoutWrapper`.\n\n# Credits\n1.
