So delete any extra sheet in an Excel file.\n   * Following excel column  names supported for both Teachers and Students:\n\n      * `name, email, password, address, about, phone_number, blood_group, nationality, gender`.\n   * Other columns:\n\n      * For Teachers: `department`, (`class, section`) if assigned as class teacher.\n      * For Students: `class, section, session, version, group, birthday, religion, father_name, father_phone_number, father_national_id, father_occupation, father_designation, father_annual_income, mother_name, mother_phone_number, mother_national_id, mother_occupation, mother_designation, mother_annual_income`\n   * For any number(e.g: phone_number) starts with zero, put (\') before zero.\n* Supported Languages (**English, Spanish**)\n\n   * To set default Language and Timezone, Edit as following in `config/app.php`:\n\n      ```php\n      \'timezone\' => \'Asia/Dhaka\',//\'UTC\',\n      \'locale\' => \'en\',//\'es-MX\' for Spanish\n      ```\n\n\n## Framework used\n\n- Laravel 5.5\n- Bootstrap 3.3.7\n\n## Server Requirements\n\n- PHP >= 7.1.0\n- OpenSSL PHP Extension\n- PDO PHP Extension\n- Mbstring PHP Extension\n- Tokenizer PHP Extension\n- XML PHP Extension\n\n## How to Start\n### Using a Container:\n\n**Anyone having trouble related to `mysql-client`, PHP 7.3 needs mariadb instead of mysql.** See issue [#192](https://github.com/changeweb/Unifiedtransform/issues/192)\n\n**[Docker](https://www.docker.com/)** is now supported.\n\nYou need to change Docker configuration files according to your need.\n\n- Change following lines in `docker-compose.yml`\n\n      MYSQL_ROOT_PASSWORD: your password\n      MYSQL_USER: root\n      MYSQL_PASSWORD: your password\n\n- To run this software in Docker containers run `sudo docker-compose up -d`.\n- Then run `sudo docker container ls --all`. Copy **Nginx** Container ID.\n- Then run `sudo docker exec -it <container id> bash`\n- Run `cp .env.example .env` and change following lines in `.env` file\n   ```sh\n   DB_HOST=db\n   DB_PORT=3306\n   DB_DATABASE=school\n   DB_USERNAME=root\n   DB_PASSWORD=your password\n   ```\n- Run `composer install`\n- Run `php artisan key:generate`\n- Run `php artisan migrate:fresh --seed`\n- Visit `http://localhost:80`.\n\n### Not using a Container:\n\nHere are some basic steps to start using this application\n\n**Note:** Instruction on cached data for Dashboard is given in **Good to know** Important: only a `master` can create a new school and its admins!\n* Login at `example.com/login` using your `Master` account credentials\n* Create a new `school`\n* Create a new `admin` for the newly created school\n\n## Manage a school\n\n* Important: A `master` CANNOT manage a school\'s data!\n* Login as `admin` at `example.com/login`\n* Now add data to the school as required.\n\n## Basic Steps by Serial\n\n* Create Classes\n* Create Sections\n* Create Exam\n* Add Students\n* Add Teachers\n* Add Courses\n* Then teacher can take attendance, give marks\n\n## Manage Exam (In exam manage page) by Admin\n\n1. Check Notice published checkbox for an Exam after uploading Exam Notice.\n2. Check Result published checkbox for an Exam after all teachers updated their courses\' marks.\n\n   * Checking result as published sets the Exam as completed.\n3. Exam is set as Active by default while created. You can deactivate the exam by checking related checkbox.\n\n## Manage GPA and Grade\n\n1. Admin adds GPAs for respective mark ranges.\n2. For giving marks, Teacher clicks on Submit Grade button and do the following:\n\n   1. Select a GPA by name from dropdown\n   2. Configure Class Test, Quiz, ...etc. count, percentage (Optional)\n   3. Give marks\n   4. To get Grade of students of a course for given marks, Teacher clicks the Get Total Marks button.\n      (Usually this is done at the end of the semester)\n\n## Good to know\n\n* Setup your Mail configuration in `.env` file if you want to send email. Currently registered users are notified by invitation mail if Mail is configured properly.\n* This project uses [Laravel Impersonate](https://github.com/404labfr/laravel-impersonate) in development and staging environments, so you can view the application through another user\'s eyes, which is useful for testing.
