Installation\n\n### Easy install\n\n```bash\ncurl -fsSL https://sh.poac.pm | bash\n```\n*When your OS is macOS, use [Homebrew](https://github.com/Homebrew/brew)*\n\n### Manual install (Build)\n\nPoac requires the following tools and packages to build:\n* [`cmake`](https://github.com/Kitware/CMake): `3.0` or higher\n* [`boost`](https://github.com/boostorg): `1.66.0` or higher\n* [`openssl`](https://github.com/openssl/openssl): as new as possible\n* [`yaml-cpp`](https://github.com/jbeder/yaml-cpp): `0.6.0` or higher\n\n```bash\n$ git clone https://github.com/poacpm/poac.git\n$ cd poac\n$ mkdir build && cd $_\n$ cmake ..\n$ make\n$ make install\n```\n\n<!--\nIf poac is already installed, you can build using poac:\n```bash\n$ poac build\n```\n-->\n\n### Runtime requirements\n\n* compiler (GCC | Clang | MSVC | ICC)\n* `tar`: in publish command\n* `dot(graphviz)`: in graph command\n* `git`: in install command\n* `cmake`: optional\n* `make`: optional\n\n## Why poac?\n\nI often see that C++ was said to be a difficult language, and I saw many people who shun it just because of C++.\nIt was thought that it is hard to construct an environment, there is no definitive package manager, and the strange syntax of the build system [CMake](https://cmake.org), etc. is the cause that makes us feel hesitant.\n\nBy developing a package manager and build system that has an intuitive and easy-to-use interface like [npm](https://www.npmjs.com) and [Cargo](https://github.com/rust-lang/cargo), and users can develop applications and libraries without being aware of [CMake](https://cmake.org), you can focus on learning C++ without being stumble.\nI also plan to implement integration with many other build systems and package managers, so you should be able to switch seamlessly.\n\n## Contributing\nPlease see [CONTRIBUTING.md](.github/CONTRIBUTING.md).\nYou can also find the useful [architecture documentation](https://doc.poac.pm/en/architecture/).\n\n##
