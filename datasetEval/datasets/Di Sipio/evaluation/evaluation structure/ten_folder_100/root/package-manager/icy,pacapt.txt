b"## `pacapt` - An `ArchLinux`'s pacman-like wrapper for many package managers\n\n[![Build Status](https://travis-ci.org/icy/pacapt.svg?branch=ng)](https://travis-ci.org/icy/pacapt)\n\n`pacapt` is a wrapper for many package managers.\nSimply install package with `pacapt -S htop` or `pacapt install htop`\non any `Linux`, `BSD`, `Mac OS` machines.\nIt supports the following package managers:\n\n* `pacman`        by `Arch Linux`, `ArchBang`, `Manjaro`, etc.\n* `dpkg/apt-get`  by `Debian`, `Ubuntu`, etc.\n* `homebrew`      by `Mac OS X`\n* `macports`      by `Mac OS X`\n* `yum/rpm`       by `Redhat`, `CentOS`, `Fedora`, etc.\n* `portage`       by `Gentoo`\n* `zypper`        by `OpenSUSE`\n* `pkgng`         by `FreeBSD`\n* `cave`          by `Exherbo Linux`\n* `pkg_tools`     by `OpenBSD`\n* `sun_tools`     by `Solaris(SunOS)`\n* `apk`           by `Alpine Linux`\n* `tazpkg`        by `SliTaz Linux`\n* `swupd`         by `Clear Linux`\n* `tlmgr`         by `TeX Live`\n* `conda`         by [`Conda`](https://conda.io/docs/)\n\n## TOC\n\n* [Description](#pacapt---an-archlinuxs-pacman-like-wrapper-for-many-package-managers)\n* [Installation](#installation)\n  * [Install stable `Bash` script 2.4.x from Github](#install-stable-bash-script-24x-from-github)\n* [Usage](#usage)\n  * [Basic options](#basic-options)\n  * [Basic operations](#basic-operations)\n  * [Implemented Operations](#implemented-operations)\n* [Related projects](#related-projects)\n* [Similar projects](#similar-projects)\n* [Development](#development)\n* [License](#license)\n* [Authors](#authors-contributors)\n\n## Installation\n\n1. This script shouldn't be installed on an Arch-based system;\n2. On `FreeBSD` and `Alpine Linux`, please install `bash` package first.\n\n### Install stable `Bash` script 2.4.x from Github\n\nYou can download the stable script and make it executable.\nOn non-Arch-based system, you may use `pacman` as script name instead of `pacapt`.\n\n````\n$ sudo wget -O /usr/local/bin/pacapt \\\nhttps://github.com/icy/pacapt/raw/ng/pacapt\n\n$ sudo chmod 755 /usr/local/bin/pacapt\n\n$ sudo ln -sv /usr/local/bin/pacapt /usr/local/bin/pacman || true\n````\n\nFor non-system package manager, you need to create symbolic links\n\n```\n$ ln -s /usr/local/bin/pacapt /usr/local/bin/pacapt-tlmgr\n$ ln -s /usr/local/bin/pacapt /usr/local/bin/pacapt-conda\n```\n\nYou can also use shorter links:\n\n```\n$ ln -s /usr/local/bin/pacapt /usr/local/bin/p-tlmgr\n$ ln -s /usr/local/bin/pacapt /usr/local/bin/p-conda\n```\n\nnoting the suffix (e.g., `-tlmgr`, `-conda`) is mandatory.\n\n## Usage\n\n### Basic operations\n\nFor system package manager\n\n* Update package database: `pacapt -Sy`, or `pacapt update`\n* Install a package: `pacapt -S foo`, or `pacapt install foo`\n* Search a package: `pacapt -Ss foo`, or `pacapt search foo`\n* Remove a package: `pacapt -R foo`, or `pacapt remove foo`\n* Upgrade system: `pacapt -Su`, or `pacapt upgrade`\n* Remove orphans: `pacapt -Sc`, or `pacapt autoremove foo`\n* Clean up: `pacapt -Scc` or `pacapt -Sccc`, or `pacapt clean`\n\nFor non-system package manager: Similar as above, however you need to call correct script name, e.g.,\n\n* Intall a Conda package: `pacapt-conda -S foo`\n* Remove a Conda package: `pacapt-conda -R foo`\n\n### Basic options\n\nSee also https://github.com/icy/pacapt/blob/ng/lib/help.txt.\n\nSome basic command line options\n\n* `-h` (`--help`): Print help message;\n* `-P`: Print list of supported operations;\n* `-V`: Print script version\n\nSome popular options of the original `ArchLinux`'s `pacman` program\nare supported and listed in the table in the next section.\n\nA long list of options and operations can be found from [`ArchLinux`'s wiki](https://wiki.archlinux.org/index.php/Pacman/Rosetta).\n\n### Implemented operations\n\n```\n
