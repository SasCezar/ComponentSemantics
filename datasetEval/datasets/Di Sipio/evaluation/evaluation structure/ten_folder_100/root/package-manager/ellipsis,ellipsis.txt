b'# Ellipsis [![Build Status][travis-image]][travis-url] [![Documentation status][docs-image]][docs-url] [![Latest tag][tag-image]][tag-url] [![Gitter chat][gitter-image]][gitter-url]\n\n```\n _    _    _\n/\\_\\ /\\_\\ /\\_\\\n\\/_/ \\/_/ \\/_/   \xe2\x80\xa6because $HOME is where the <3 is!\n```\n\nEllipsis is a package manager for dotfiles.\n\n### Features\n- Creating new packages is trivial (any git repository is already a package).\n- Modular configuration files are easier to maintain and share with others.\n- Allows you to quickly see which dotfiles have been modified, and keep them\n  updated and in-sync across systems.\n- Adding new dotfiles to your collection can be automated with `ellipsis add`.\n- Cross platform, known to work on Mac OS X, Linux, FreeBSD and even Cygwin.\n- Large test suite to ensure your `$HOME` doesn\'t get ravaged.\n- Completely customizable.\n- [Works with existing dotfiles!][docs-upgrading]\n\n### Install\n**Requirements:** bash, curl, git\n\nClone and symlink or use handy-dandy installer:\n\n```bash\n# Manual install\n$ git clone https://github.com/ellipsis/ellipsis .ellipsis\n\n# Using installer\n$ curl ellipsis.sh | sh\n```\n\n<sup>...no you didn\'t read that wrong, [the ellipsis.sh website also doubles as the installer][installer]</sup>\n\nWith the installer you can also specify which packages to install by setting\nthe `PACKAGES` variable, i.e.:\n\n```bash\n$ curl https://ellipsis.sh | PACKAGES=\'vim zsh\' sh\n```\n\nAdd `~/.ellipsis/bin` to your `$PATH` (or symlink somewhere convenient) and\nstart managing your dotfiles in style :)\n\nAs of version `1.7.3` you can also use [the init system][docs-init] to\nautomatically setup you environment. As a bonus it will allow you to use the\npowerful `pkg.init` hook to do the same for your packages.\n\n### Usage\nEllipsis comes with no dotfiles out of the box. To install packages, use\n`ellipsis install`. Packages can be specified by github-user/repo or full\nssh/git/http(s) urls:\n\n```bash\n$ ellipsis install ssh://github.com/zeekay/private.git\n$ ellipsis install zeekay/vim\n$ ellipsis install zsh\n```\n\n...all work. By convention `username/package` and `package` are aliases for\nhttps://github.com/username/dot-package. (customizable using `ELLIPSIS_PREFIX`)\n\nFor full usage information you can read the [docs][docs-usage] or ask help from\nthe command line with the `-h` option.\n\n### Configuration\nYou can customize ellipsis by exporting a few different variables:\n\n| Variable                        | Description                                                                                                                                                          |\n|---------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|\n| `GITHUB_USER` / `ELLIPSIS_USER` | Customizes whose dotfiles are installed when you `ellipsis install` without specifying user or a full repo url. Defaults to `$(git config github.user)` or `whoami`. |\n| `ELLIPSIS_REPO`                 | Customize location of ellipsis repo cloned during a curl-based install. Defaults to `https://github.com/ellipsis/ellipsis`.                                          |\n| `ELLIPSIS_PROTO`                | Customizes which protocol new packages are cloned with, you can specify `https`,`ssh`, `git`. Defaults to `https`.                                                   |\n| `ELLIPSIS_PREFIX`               | Customizes the prefix of ellipsis configuration packages (default: `dot-`).                                                                                                            |\n| `ELLIPSIS_HOME`                 | Customize which folder files are symlinked into, defaults to `$HOME`. (Mostly useful for testing)                                                                    |\n| `ELLIPSIS_PATH`                 | Customize where ellipsis lives on your filesystem, defaults to `~/.ellipsis`.                                                                                        |\n| `ELLIPSIS_PACKAGES`             | Customize where ellipsis installs packages on your filesystem, defaults to `~/.ellipsis/packages`.                                                                   |\n| `ELLIPSIS_LOGFILE`              | Customize location of the logfile, defaults to `/tmp/ellipsis.log`.                                                                                                  |\n\n```bash\nexport ELLIPSIS_USER="zeekay"\nexport ELLIPSIS_PROTO="ssh"\nexport ELLIPSIS_PATH="~/.el"\n```\n\n### Packages\nA package is any repo with files you want to symlink into `$ELLIPSIS_HOME`\n(typically `$HOME`). By default all of a repository\'s non-hidden files (read:\nnot beginning with a `.`) will naively be linked into place, with the exception\nof a few common text files (`README`, `LICENSE`, etc).\n\nYou can customize how ellipsis interacts with your package by adding an\n`ellipsis.sh` file to the root of your project. Here\'s an example of a complete\n`ellipsis.sh` file:\n\n```bash\n#!/usr/bin/env bash\n```\n\nYep, that\'s it :) If all you want to do is symlink some files into `$HOME`,\nadding an `ellipsis.sh` to your package is completely optional. But what if you\nneed more? [That\'s where hooks come in...][docs-hooks]\n\n### Docs\nPlease consult the [docs][docs-url] for more information.\n\nSpecific parts that could be off interest:\n- [Hooks][docs-hooks]\n- [Init system][docs-init]\n- [API][docs-api]\n- [Package index][docs-pkgindex]\n- [Upgrading to Ellipsis][docs-upgrading]\n- [Zsh completion][docs-completion]\n\n### Development\nPull requests welcome!
