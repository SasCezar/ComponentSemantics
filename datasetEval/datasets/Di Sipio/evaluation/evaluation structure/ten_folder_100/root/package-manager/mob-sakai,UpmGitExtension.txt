Click ![giticon](https://user-images.githubusercontent.com/12690315/60764763-7fd4c800-a0ca-11e9-957b-ca68e3ca6123.png) button in package manager UI to open `Install Package Window`  \n![](https://user-images.githubusercontent.com/12690315/60766233-dbf71680-a0e1-11e9-8303-fbd790e9e35b.png)  \n![](https://user-images.githubusercontent.com/12690315/60764768-91b66b00-a0ca-11e9-9ccd-9fef88c77d5e.png)\n1. Input a git repository url and click `Find Versions` button  \n![](https://user-images.githubusercontent.com/12690315/60766258-4314cb00-a0e2-11e9-91f8-3aad514450bc.png)\n2. Select a tag or branch and click `Find Package` button  \n![](https://user-images.githubusercontent.com/12690315/60766257-4314cb00-a0e2-11e9-8b2e-23efc50ded72.png)\n3. Wait a few seconds for validation\n4. Click `Add Package` button to install the package  \n![](https://user-images.githubusercontent.com/12690315/60766259-4314cb00-a0e2-11e9-9b89-0bc0d4f71517.png)\n\n\n### Update package with a specific tag or branch as version\n\nYou can update the package in your project, _just like official packages._\n\n#### For Unity 2019.1 or later\n\n1. Select the version of the package\n2. Click `Update To ***` button  \n![](https://user-images.githubusercontent.com/12690315/60766318-fc73a080-a0e2-11e9-9020-23dfc05939a0.png)\n\n#### For Unity 2018.3\n\n1. Click version popup and select a tag or branch in repository  \n![](https://user-images.githubusercontent.com/12690315/60766391-1cf02a80-a0e4-11e9-8c3f-d420b7e84b46.png)\n2. Click `Update To` button\n\n\n### Remove package\n\nYou can update the package from your project, just like official packages.\n\n1. Click `Remove` button  \n![](https://user-images.githubusercontent.com/12690315/60766319-fd0c3700-a0e2-11e9-9154-b88161496a3e.png)\n\n\n\n<br><br><br><br>\n## Development Note\n\n### Develop a package for UPM\n\nThe branching strategy when I develop a package for UPM is as follows.\n\n|Branch|Description|\'Assets\' directory|\n|-|-|-|\n|develop|Develop, Test|Included|\n|master|Publish with legacy way (.unitypackage)|Included|\n|upm(default)|Subtree to publish for UPM|Excluded|\n|{tags}|Tags to install using UPM|Excluded|\n\n**Steps to release a package:**\n1. Update version in `package.json`.\n2. Develop package project on develop branch.\n3. Close all issues on GitHub for new version.\n4. Generate `CHANGELOG.md` using `github_changelog_generator` and commit it.\n5. Merge into master branch and publish as new version.\n6. Split subtree into ump branch.\n7. Tag on ump branch as new version.\n8. Release.\n\nFor details, see https://www.patreon.com/posts/25070968.\n\n\n\n<br><br><br><br>\n## License\n\n* MIT\n\n\n\n## Author\n\n[mob-sakai](https://github.com/mob-sakai)\n[![](https://img.shields.io/twitter/follow/mob_sakai.svg?label=Follow&style=social)](https://twitter.com/intent/follow?screen_name=mob_sakai)  \n[![become_a_sponsor_on_github](https://user-images.githubusercontent.com/12690315/66942881-03686280-f085-11e9-9586-fc0b6011029f.png)](https://github.com/users/mob-sakai/sponsorship)\n\n\n\n## See Also\n\n* GitHub page : https://github.com/mob-sakai/UpmGitExtension\n* Releases : https://github.com/mob-sakai/UpmGitExtension/releases\n* Issue tracker : https://github.com/mob-sakai/UpmGitExtension/issues\n* Current project : https://github.com/mob-sakai/UpmGitExtension/projects/1\n* Change log : https://github.com/mob-sakai/UpmGitExtension/blob/upm/CHANGELOG.md'
