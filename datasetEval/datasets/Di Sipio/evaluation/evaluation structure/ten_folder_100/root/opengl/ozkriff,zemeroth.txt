player\xe2\x80\x99s control;\n- Small unscrollable maps;\n- Relatively short game session (under an hour);\n- Simple vector 2d graphics with just 3-5 sprites per unit;\n- Reaction attacks and system dependencies][ggez_sys_deps].\n\n```bash\n# Clone this repo\ngit clone https://github.com/ozkriff/zemeroth\ncd zemeroth\n\n# Assets are stored in a separate repo.\n# Zemeroth expects them to be in `assets` directory.\ngit clone https://github.com/ozkriff/zemeroth_assets assets\n\n# Compile a release version (debug builds give low FPS at the moment)\ncargo build --release\n\n# Run it\ncargo run --release\n```\n\n[ggez_sys_deps]: https://github.com/ggez/ggez/blob/a62acaa24/docs/BuildingForEveryPlatform.md\n\n## WebAssembly\n\n```bash\ncargo install cargo-web\n./utils/wasm/build.sh\ncargo web start\n```\n\nThen open `http://localhost:8000` in your browser.\n\nThe WASM version of the game uses\n[not-fl3/good-web-game](https://github.com/not-fl3/good-web-game):\n\n> [Note](https://github.com/ggez/ggez/issues/71#issuecomment-459875258)\n> that good-web-game is not really GGEZ\'s backend,\n> but a separate web-targeted engine with a similar API\n> that @not-fl3 uses for his prototypes.\n>\n> Zemeroth uses good-web-game for its web version as a quick-n-dirty\n> immediate solution until a proper WASM support arrives to GGEZ\n> (there\'re no plans of making good-web-game some kind of official GGEZ backend).\n>\n> The currently implemented subset of GGEZ API is quite limited\n> and while it may be used for something else that Zemeroth,\n> it will probably require a lot of work to do (contributions are welcome ;) ).\n\n## Dependencies\n\nThe key external dependency of Zemeroth is [ggez] game engine.\n\nThis repo contains a bunch of helper crates:\n\n- [zcomponents] is a simple component storage\n- [ggwp-zgui] is a simple and opinionated ggez-based GUI library\n- [ggwp-zscene] is a simple scene and declarative animation manager\n\n[ggez]:
