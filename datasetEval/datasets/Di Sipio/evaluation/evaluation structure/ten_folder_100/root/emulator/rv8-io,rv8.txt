b"rv8\n===========\n\nRISC-V simulator for x86-64\n\n[![Build Status](https://travis-ci.org/rv8-io/rv8.svg?branch=master)](https://travis-ci.org/rv8-io/rv8)\n[![Coverity Status](https://img.shields.io/coverity/scan/13148.svg)](https://scan.coverity.com/projects/rv8-io-rv8)\n[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](https://opensource.org/licenses/MIT)\n[![Donate](https://img.shields.io/badge/Donate-PayPal-green.svg)](https://www.paypal.com/cgi-bin/webscr?cmd=_s-xclick&hosted_button_id=TKDFDG65GCXD2)\n\n**rv8** is a RISC-V simulation suite comprising a high performance x86-64 binary translator, a user mode simulator, a full system emulator, an ELF binary analysis tool and ISA metadata:\n\n* **rv-jit** - _user mode x86-64 binary translator_\n* **rv-sim** - _user mode system call proxy simulator_\n* **rv-sys** - _full system emulator with soft MMU_\n* **rv-bin** - _ELF disassembler and histogram tool_\n* **rv-meta** - _code and documentation generator_\n\nThe rv8 simulator suite contains libraries and command line tools for creating instruction opcode maps, C headers and source containing instruction set metadata, instruction decoders, a JIT assembler, LaTeX documentation, a metadata based RISC-V disassembler, a histogram tool for generating statistics on RISC-V ELF executables, a RISC-V proxy syscall simulator, a RISC-V full system emulator that implements the RISC-V 1.9.1 privileged specification and an x86-64 binary translator.\n\n![rv8 binary translation](/doc/images/bintrans.png)\n\n_**RISC-V to x86-64 binary translator**_\n\nThe rv8 binary translation engine works by interpreting code while profiling it for hot paths. Hot paths are translated on the fly to native code. The translation engine maintains a call stack to allow runtime inlining of hot functions. A jump target cache is used to accelerate returns and indirect calls through function pointers. The translator supports hybrid binary translation and interpretation to handle instructions that do not have native translations. Currently \xe2\x80\x98IM\xe2\x80\x99 code is translated and \xe2\x80\x98AFD\xe2\x80\x99 is interpreted. The translator supports RVC compressed code.\n\nThe rv8 binary translator supports a number of simple optimisations:\n\n- Hybrid interpretation and compilation of hot code paths\n- Incremental translation with dynamic trace linking\n- Inline caching of function calls in hot code paths\n- L1 jump target cache for indirect calls and returns\n- Macro-op fusion for common RISC-V instruction sequences\n\n_**RISC-V full system emulator**_\n\nThe rv8 suite includes a full system emulator that implements the RISC-V privileged ISA with support for interrupts, MMIO (memory mapped input output) devices, a soft MMU (memory management unit) with separate instruction and data TLBs (translation lookaside buffers). The full system emulator has a simple integrated debugger that allows setting breakpoints, single stepping and disassembling instructions as they are executed.\n\nThe rv8 full system emulator has the following features:\n\n- RISC-V IMAFD Base plus Privileged ISA (riscv32 and riscv64)\n- Simple integrated debug command line interface\n- Histograms (registers, instruction and program counter frequency)\n- Soft MMU supporting sv32, sv39, sv48 page translation modes\n- Abstract MMIO device interface for device emulation\n- Extensible interpreter generated from ISA metadata\n- Protected address space\n\n_**RISC-V user mode simulator**_\n\nThe rv8 user mode simulator is a single address space implementation of the RISC-V ISA that implements a subset of the RISC-V Linux syscall ABI (application binary interface) and delegates system calls to the underlying native host operating system. The user mode simulator can run RISC-V Linux binaries on non-Linux operating systems via system call emulation. The current user mode simulator implements a small number of system calls to allow running RISC-V Linux ELF static binaries.\n\nThe rv8 user mode simulator has the following features:\n\n- RISC-V IMAFD Base ISA (riscv32 and riscv64)\n- Simple integrated debug command line interface\n- A small set of emulated Linux system calls for simple file IO\n- Extensible interpreter generated from ISA metadata\n- Instruction logging mode for tracing program execution\n- Shared address space\n  - `0x000000000000 - 0x000000000fff` (zero)\n  - `0x000000001000 - 0x7ffdffffffff` (guest)\n  - `0x7ffe00000000 - 0x7fffffffffff` (host)\n\n## Project Goals\n\nThe future goals of the rv8 project are:\n\n- Concise metadata representing the RISC-V ISA\n- Tools for metadata-based generation of source and documentation \n- High performance emulation, sandboxing and binary translation\n- RISC-V-(n) \xe2\x86\x92 RISC-V-(n+1)\n- RISC-V-(n) \xe2\x86\x92 Intel i7 / AMD64 + AVX-512\n- RISC-V Linux ABI emulation on MacOS, Windows, Linux and *BSD\n- RISC-V Linux ABI randomisation and entropy coding\n- RISC-V Specification undefined behaviour investigation\n- RISC-V Virtualization and memory protection investigation\n\n\n## Supported Platforms\n\n- Target\n  - RV32IMAFDC\n  - RV64IMAFDC\n  - Privilged ISA 1.9.1\n- Host\n  - Linux (Debian 9.0 x86-64, Ubuntu 16.04 x86-64, Fedora 25 x86-64) _(stable)_\n  - macOS (Sierra 10.11 x86-64) _(stable)_\n  - FreeBSD (11 x86-64) _(alpha)_\n\n\n## Getting Started\n\n_Building riscv-gnu-toolchain_\n\n```\n$ sudo apt-get install autoconf automake autotools-dev curl libmpc-dev libmpfr-dev libgmp-dev gawk build-essential bison flex texinfo gperf libtool patchutils bc zlib1g-dev\n$ git clone https://github.com/riscv/riscv-gnu-toolchain.git\n$ cd riscv-gnu-toolchain\n$ git submodule update --init --recursive\n$ ./configure --prefix=/opt/riscv/toolchain\n$ make\n```\n\n_Building rv8_\n\n```\n$ export RISCV=/opt/riscv/toolchain\n$ git clone https://github.com/rv8-io/rv8.git\n$ cd rv8\n$ git submodule update --init --recursive\n$ make\n$ sudo make install\n```\n\n_Running rv8_\n\n```\n$ make test-build\n$ rv-jit build/riscv64-unknown-elf/bin/test-dhrystone\n```\n\n\n## Build Dependencies\n\n- gmake\n- gcc-5.4 or clang-3.4 (Linux minimum)\n- gcc-6.3 or clang-3.8 (Linux recommended)\n- RISC-V GNU Toolchain\n- RISCV environment variable\n\n\n### Ubuntu 14.04LTS Dependencies\n\nThe compiler in Ubuntu 14.04LTS doesn't support C++14. apt-get install g++-6\nmake CXX=g++-6 CC=gcc-6\n```\n\n\n## Screenshots\n\n![ASCII map screenshot](/doc/images/screenshot-1.png)\n\n*Example ASCII map output from make map*\n\n![Histogram <elf_file>\n                        --help, -h            Show help\n                        --char, -c <string>   Character to use in bars\n                        --bars, -b            Print bars next to counts\n                --instructions, -I            Instruction Usage Histogram\n                   --registers, -R            Register Usage Histogram\n          --registers-operands, -P            Register Usage Histogram (with operand positions)\n                   --max-chars, -m <string>   Maximum number of characters for bars\n                --reverse-sort, -r            Sort in Reverse\n```\n\nTo print the top 20 instructions in a RISC-V ELF binary:\n\n```\nrv-bin histogram -I -b -c \xe2\x96\x88 linux/vmlinux | head -20\n```\n\nTo print the top 20 registers in a RISC-V ELF binary:\n\n```\nrv-bin histogram -R -b -c \xe2\x96\x88 linux/vmlinux | head -20\n```\n\n\n### RISC-V Metadata Utility\n\nThe RV source and documentation generator usage command line options:\n\n```\n$
