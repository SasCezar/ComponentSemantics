b'<br>\n<p align="center">\n<img width="350" title="cubbie" alt="cubbie!" src="https://raw.githubusercontent.com/samueleaton/design/master/sentry.png">\n</p>\n<br>\n\n# Sentry \xf0\x9f\xa4\x96\n\nBuild/Runs your crystal application, watches files, and rebuilds/reruns app on file changes\n\n## Installation\n\nTo install in your project, from the root directory of your project, run:\n```bash\ncurl -fsSLo- https://raw.githubusercontent.com/samueleaton/sentry/master/install.cr | crystal eval\n```\n\nIf using Crystal version `0.24.2` try the following:\n```bash\ncurl -fsSLo- https://raw.githubusercontent.com/samueleaton/sentry/crystal-v0.24.2/install.cr | crystal eval\n```\n\nIf using Crystal version `0.23.1` or lower try the following:\n```bash\ncurl -fsSLo- https://raw.githubusercontent.com/samueleaton/sentry/crystal-v0.23.1/install.cr | crystal eval\n```\n\nThis will install the Sentry CLI tool. To use the Crystal API, see [CRYSTAL_API.md](./CRYSTAL_API.md).\n\n<p align="center">\n  <img width="450" title="sentry" alt="sentry" src="https://raw.githubusercontent.com/samueleaton/design/master/sentry.gif" />\n</p>\n\n**Troubleshooting the install:** This install script is just a convenience. If it does not work, simply: (1) place the files located in the `src` dir into a your project in a `dev/` dir, and (2) compile sentry by doing `crystal build --release dev/sentry_cli.cr -o ./sentry`.\n\n## Usage\n\nAssuming `sentry.cr` was correctly placed in `[your project name]/dev/sentry.cr` and compiled into the root of your app as `sentry`, simply run:\n\n```bash\n./sentry [options]\n```\n\n### Options\n\n#### Show Help Menu\n\n```bash\n./sentry --help\n```\n\nExample\n```bash\n$ ./sentry -h\n\nUsage: ./sentry [options]\n     -n NAME, --name=NAME             Sets the display name of the app process (default name: <your_app_here>)\n     -b COMMAND, --build=COMMAND      Overrides the default build command\n     --build-args=ARGS                Specifies arguments for the build command\n     --no-build                       Skips the build step\n     -r COMMAND, --run=COMMAND        Overrides the default run command\n     --run-args=ARGS                  Specifies arguments for the run command\n     -w FILE, --watch=FILE            Overrides default files and appends to list of watched files\n     -c FILE, --config=FILE           Specifies a file to load for automatic configuration (default: \'.sentry.yml\')\n     --install                        Run \'shards install\' once before running Sentry build and run commands\n     --no-color                       Removes colorization from output\n     -i, --info                       Shows the values for build/run commands, build/run args, and watched files\n     -h, --help                       Show this help\n```\n\n#### Override Default Build Command\n\n```bash\n./sentry -b "crystal build --release ./src/my_app.cr"\n```\n\nThe default build command is `crystal build ./src/[app_name].cr`. The release flag is omitted by default for faster compilation time while you are developing.\n\n#### Override Default Run Command\n\n```bash\n./sentry -r "./my_app"\n```\n\nThe default run command is `./[app_name]`.\n\n#### Override Default Files to Watch\n\n```bash\n./sentry -w "./src/**/*.cr" -w "./lib/**/*.cr"\n```\n\nThe default files being watched are `["./src/**/*.cr", "./src/**/*.ecr"]`.\n\nBy specifying files to watch, the default will be omitted. So if you want to watch all of the file in your `src` directory, you will need to specify that like in the above example.\n\n#### Show Info Before Running\n\nThis shows the values for the build command, run command, and watched files.\n\n```bash\n./sentry -i\n```\n\nExample\n```\n$ ./sentry -i\n\n\xf0\x9f\xa4\x96  Sentry configuration:\n       display name:   my_app\n       shard name:     my_app\n       install shards: true\n       info:           true\n       build:          crystal build ./src/my_app.cr\n       build_args:     []\n       run:            ./my_app\n       run_args:       []\n       watch:          ["./src/**/*.cr", "./src/**/*.ecr"]\n\xf0\x9f\xa4\x96  Your SentryBot is vigilant. beep-boop...\n...\n...\n```\n\n#### Setting Build or Run Arguments\n\nIf you prefer granularity, you can specify arguments to the build or run commands using the `--build-args` or `--run-args` flags followed by a string of arguments.\n\n```bash\n./sentry -r "crystal" --run-args "spec --debug"\n```\n\n#### Running `shards install` Before Starting\n\nThis is especially usefull when initiating Sentry from a `Dockerfile` or `package.json` file. It guarantees all the shards are installed before running.\n\n```bash\n./sentry --install\n```\n\n#### Reading Configurations from a File\n\nSentry will automatically read configurations from `.sentry.yml` if it exists. This can be changed with `-c FILE` or `--config=FILE`.\n\nSee the `YAML.mapping` definition in the `Config` class in [the `/src/sentry.cr` file](src/sentry.cr) for valid file properties.\n\n#### Removing Colorization\n\nSentry output is colorized by default. To remove colorization, pass the `--no-color` argument.\n\nExample\n```bash\n./sentry --no-color\n```\n\n## Sentry Crystal API\n\nSee [CRYSTAL_API.md](./CRYSTAL_API.md)\n\n## Why?\n(1) It is tiring to have to stop and restart an app on every change.\n\n(2) Docker!\n\nStop and restarting your app is especially expensive (and annoying) when running the app in a docker container, where one would need to totally rebuild the docker image for every change.\n\nNow, for development, simply run sentry in your docker container, and it will rebuild the app from the docker container on any changes, without rebuilding the docker image/container.\n\n## Contributing\n\n1.
