b"# FastLogin\n\nChecks if a Minecraft player has a paid account (premium). If so, they can skip offline authentication (auth plugins).\nSo they don't need to enter passwords. This is also called auto login (auto-login).\n\n## Features\n\n* Detect paid accounts from others\n* Automatically login paid accounts (premium)\n* Support various of auth plugins\n* Cauldron support\n* Forge/Sponge message support\n* Premium UUID support\n* Forwards Skins\n* Detect user name changed and will update the existing database record\n* BungeeCord support\n* Auto register new premium players\n* Plugin: ProtocolSupport is supported and can be used as an alternative to ProtocolLib\n* No client modifications needed\n* Good performance by using async non blocking operations\n* Locale messages\n* Import the database from similar plugins\n\n## Development builds\n\nDevelopment builds of this project can be acquired at the provided CI (continuous integration) server. It contains the\nlatest changes from the Source-Code in preparation for the following release. This means they could contain new\nfeatures, bug fixes and other changes since the last release.\n\nNevertheless builds are only tested using a small set of automated and a few manual tests. Therefore they **could**\ncontain new bugs and are likely to be less stable than released versions.\n\nhttps://ci.codemc.org/job/Games647/job/FastLogin/changes\n\n***\n\n## Commands\n\n    /premium [player] Label the invoker or the argument as paid account\n    /cracked [player] Label the invoker or the argument as cracked account\n\n## Permissions\n\n    fastlogin.bukkit.command.premium\n    fastlogin.bukkit.command.cracked\n    fastlogin.command.premium.other\n    fastlogin.command.cracked.other\n\n## Requirements\n\n* Plugin: \n    * [ProtocolLib](https://www.spigotmc.org/resources/protocollib.1997/) or \n    * [ProtocolSupport](https://www.spigotmc.org/resources/protocolsupport.7201/)\n* [Spigot](https://www.spigotmc.org) 1.7.10+\n* Java 8+\n* Run Spigot and/or BungeeCord/Waterfall in offline mode (see server.properties or config.yml)\n* An auth plugin. Supported plugins\n\n### Bukkit/Spigot/Paper\n\n* [AuthMe (5.X)](https://dev.bukkit.org/bukkit-plugins/authme-reloaded/)\n* [xAuth](https://dev.bukkit.org/bukkit-plugins/xauth/)\n* [LogIt](https://github.com/games647/LogIt)\n* [AdvancedLogin (Paid)](https://www.spigotmc.org/resources/advancedlogin.10510/)\n* [CrazyLogin](https://dev.bukkit.org/bukkit-plugins/crazylogin/)\n* [LoginSecurity](https://dev.bukkit.org/bukkit-plugins/loginsecurity/)\n* [UltraAuth](https://dev.bukkit.org/bukkit-plugins/ultraauth-aa/)\n\n### BungeeCord/Waterfall\n\n* [BungeeAuth](https://www.spigotmc.org/resources/bungeeauth.493/)\n\n## Network requests\n\nThis plugin performs network requests to:\n\n* https://api.mojang.com - retrieving uuid data to decide if we should activate premium login\n* https://sessionserver.mojang.com - verify if the player is the owner of that account\n\n***\n\n## How to install\n\n### Bukkit/Spigot/Paper\n\n1. Download and install ProtocolLib/ProtocolSupport\n2. Download and install FastLogin (or FastLoginBukkit for newer versions)\n3. Set your server in offline mode by setting the value onlinemode in your server.properties to false\n\n### BungeeCord/Waterfall\n\n1. Activate BungeeCord in the Spigot configuration\n2. Restart your server\n3. Now there is proxy-whitelist file in the FastLogin folder\nPut your stats id from the BungeeCord config into this file\n4. Activate ipForward in your BungeeCord config\n5. Download and Install FastLogin (or FastLoginBungee in newer versions) on BungeeCord AND Spigot\n(on the servers where your login plugin is or where player should be able to execute the commands of FastLogin)\n6. Check your database settings in the config of FastLogin on BungeeCord\n7. Set your proxy (BungeeCord) in offline mode by setting the value onlinemode in your config.yml to false\n8. You should *always* firewall your Spigot server that it's only accessible through BungeeCord \n    * https://www.spigotmc.org/wiki/bungeecord-installation/#post-installation\n    * BungeeCord doesn't support SQLite per default, so you should change the configuration to MySQL or MariaDB\n"
