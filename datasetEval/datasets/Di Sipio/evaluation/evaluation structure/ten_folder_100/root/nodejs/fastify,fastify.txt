Fastify is a web framework highly focused on providing the best developer experience with the least overhead and a powerful plugin architecture. It is inspired by Hapi and Express and as far as we know, it is one of the fastest web frameworks in town.\n\n### Install\n\nInstall with npm:\n```\nnpm i fastify --save\n```\nInstall with yarn:\n```\nyarn add fastify\n```\n\n### Example\n\n```js\n// Require the framework and instantiate it\nconst fastify = require(\'fastify\')({\n  logger: true\n})\n\n// Declare a route\nfastify.get(\'/\', (request, reply) => {\n  reply.send({ hello: \'world\' })\n})\n\n// Run the server!\nfastify.listen(3000, (err, address) => {\n  if (err) throw err\n  fastify.log.info(`server listening on ${address}`)\n})\n```\n\nwith async-await:\n\n```js\nconst fastify = require(\'fastify\')({\n  logger: true\n})\n\nfastify.get(\'/\', async (request, reply) => {\n  reply.type(\'application/json\').code(200)\n  return { hello: \'world\' }\n})\n\nfastify.listen(3000, (err, address) => {\n  if (err) throw err\n  fastify.log.info(`server listening on ${address}`)\n})\n```\n\nDo you want to know more? Head to the <a href="https://github.com/fastify/fastify/blob/master/docs/Getting-Started.md"><code><b>Getting Started</b></code></a>.\n\n### Quick start with Fastify CLI\n\nGood tools make API development quicker and easier to maintain than doing everything manually.\n\nThe [Fastify CLI](https://github.com/fastify/fastify-cli) is a command line interface tool that can create new projects, manage plugins, and perform a variety of development tasks testing and running the application.\n\nThe goal in this guide is to build and run a simple Fastify project, using the [Fastify CLI](https://github.com/fastify/fastify-cli), while adhering to the Style Guide recommendations that benefit every Fastify project.\n\n### Example\n\nOpen a terminal window.\n\n```\nnpm install fastify-cli --global\n```\n\nGenerate a new project and default app by running the following command:\n\n```\nfastify generate\n```\n\nFor more information, see the [Fastify CLI documentation](https://github.com/fastify/fastify-cli).\n\n### Fastify v1.x\n\nCode for Fastify\'s **v1.x** is in [Branch 1.x](https://github.com/fastify/fastify/tree/1.x), so all Fastify 1.x related changes should be based on **`branch 1.x`**.\n\n> ## Note\n> `.listen` binds to the local host, `localhost`, interface by default (`127.0.0.1` or `::1`, depending on the operating system configuration). If you are running Fastify in a container (Docker, [GCP](https://cloud.google.com/), etc.), you may need to bind to `0.0.0.0`. Be careful when deciding to listen on all interfaces; it comes with inherent [security risks](https://web.archive.org/web/20170711105010/https://snyk.io/blog/mongodb-hack-and-secure-defaults/).\n> See [the documentation](https://github.com/fastify/fastify/blob/master/docs/Server.md#listen) for more information.\n\n### Core features\n\n- **Highly performant:** as far as we know, Fastify is one of the fastest web frameworks in town, depending on the code complexity we can serve up to 30 thousand requests per second.\n- **Extendible:** Fastify is fully extensible via its hooks, plugins and decorators.\n- **Schema based:** even if it is not mandatory we recommend to use [JSON Schema](http://json-schema.org/) to validate your routes and serialize your outputs, internally Fastify compiles the schema in a highly performant function.\n- **Logging:** logs are extremely important but are costly; we chose the best logger to almost remove this cost, [Pino](https://github.com/pinojs/pino)!\n- **Developer friendly:** the framework is built to be very expressive and help the developer in their daily use, without sacrificing performance and security.\n\n### Benchmarks\n\n__Machine:__ EX41S-SSD, Intel Core i7, 4Ghz, 64GB RAM, 4C/8T, SSD.\n\n__Method:__: `autocannon -c 100 -d 40 -p 10 localhost:3000` * 2, taking the second average\n\n| Framework          | Version                    | Router?
