These included files can be read in the application by using `fs.readFile` or `fs.readFileSync`.\n\n## Compiling Node\n\nBy default `nexe` will attempt to download a pre-built executable. However, It may be unavailable ([github releases](https://github.com/nexe/nexe/releases))\nor you may want to customize what is built. See `nexe --help` for a list of options available when passing the [`--build`](#build-boolean) option. You will also need to ensure your environment is setup to [build node](https://github.com/nodejs/node/blob/master/BUILDING.md). Note: the `python` binary in your path should be an acceptable version of python 2. eg. Systems that have python2 will need to create a [symlink](https://github.com/nexe/nexe/issues/354#issuecomment-319874486).\n\n## Node.js API\n\n#### Example\n\n```javascript\nconst { compile } = require(\'nexe\')\n\ncompile({\n  input: \'./my-app.js\',\n  build: true, //required to use patches\n  patches: [\n    async (compiler, next) => {\n      await compiler.setFileContentsAsync(\n        \'lib/new-native-module.js\',\n        \'module.exports = 42\'\n      )\n      return next()\n    }\n  ]\n}).then(() => {\n  console.log(\'success\')\n})\n```\n## NexeOptions\n\n### `options: object`\n\n - #### `input: string`\n    - Input bundle file path\n    - default: stdin or the current directory\'s main file (package.json)\n - #### `output: string`\n    - Output executable file path\n    - default: same as `name` with an OS specific extension.\n - #### `target: string | object`\n    - An object or string describing platform-arch-version. e.g. `\'windows-ia32-10.13.0\'`\n      - each segment is optional, and will be merged with the current environment\n      - Examples: ([full list](https://github.com/nexe/nexe/releases))\n        - `\'win32-x86-10.13.0`\n        - `{ platform: \'alpine\' }`\n        - `darwin-10.13.0`\n        - `linux-x64`\n        - `macos-10.13.0`\n\n        See [test/target.spec.ts](test/target.spec.ts)\n    - If the [`build`](#build-boolean) flag is set, the platform portion of the target is ignored.\n    - default: `process`\n- #### `bundle: string | boolean`\n    - If a string is provided it must be a valid relative module path\n    and should provide an export with the following signature:\n    ```typescript\n    export function createBundle (options: NexeOptions): Promise<string>\n    ```\n    - default: true\n - #### `name: string`\n    - Module friendly name of the application\n    - default: basename of the input file, or `nexe_${Date.now()}`\n - #### `cwd: string`\n    - Directory nexe will operate on as though it is the cwd\n    - default: process.cwd()\n - #### `mangle: boolean`\n    - If set to false, nexe will not include the virtual filesystem (your application and resources) on the output.\n    - This will cause the output to error as an "Invalid Binary" unless a userland patch alters the contents of lib/_third_party_main.js in the nodejs source.\n    - default: true\n - #### `build: boolean`\n    - Build node from source, passing this flag tells nexe to download and build from source. Subsequently using this flag will cause nexe to use the previously built binary. To rebuild, first add [`--clean`](#clean-boolean)\n - #### `asset: string`\n    - Provide a pre-built nexe binary asset, this can either be an http or https URL or a file path.\n    - default: `null`\n - #### `python: string`\n    - On Linux this is the path pointing to your python2 executable\n    - On Windows this is the directory where `python` can be accessed\n    - default: `null`\n - #### `flags: string[]`\n    - Array of node runtime flags to build node with.\n    - Example: `[\'--expose-gc\']`\n    - default: `[]`\n - #### `configure: string[]`\n    - Array of arguments for the node build configure step\n    - Example: `[\'--with-dtrace\', \'--dest-cpu=x64\']`\n    - default: `[]`\n - #### `make: string[]`\n    - Array of arguments for the node build make step\n    - default: `[]`\n - #### `vcBuild: string[]`\n    - Options for windows build\n    - default: `[\'nosign\', \'release\']`\n - #### `snapshot: string`\n    - path to a file to be used as the warmup snapshot for the build\n    - default: `null`\n - #### `resources: string[]`\n    - Array of globs with files to include in the build\n    - Example: `[\'./public/**/*\']`\n    - default: `[]`\n - #### `temp: string`\n    - Path to use for storing nexe\'s build files\n    - Override in the env with `NEXE_TEMP`\n    - default: `~/.nexe`\n - #### `ico: string`\n    - Path to a user provided icon to be used (Windows only). Requires `--build` to be set.\n - #### `rc: object`\n    - Settings for patching the [node.rc](https://github.com/nodejs/node/blob/master/src/res/node.rc) configuration file (Windows only).\n    - Example (keys may vary depending on the version. Reference the file linked above):\n      ```javascript\n        {\n          CompanyName: "ACME Corp",\n          PRODUCTVERSION: "17,3,0,0",\n          FILEVERSION: "1,2,3,4"\n          ...\n        }\n      ```\n    - default: `{}`\n - #### `clean: boolean`\n    - If included, nexe will remove temporary files for the accompanying configuration and exit\n - #### `enableNodeCli: boolean`\n    - Enable the original Node CLI (will prevent application cli from working).\n    - Node CLI arguments passed via the [NODE_OPTIONS](https://nodejs.org/api/cli.html#cli_node_options_options) environment\n      variable will still be processed. NODE_OPTIONS support can be disabled with the `--without-node-options` configure flag.\n    - default: `false`\n  - #### `fakeArgv: boolean`\n    - fake the entry point file name (`process.argv[1]`). If nexe was used with stdin this will be `\'[stdin]\'`.\n - #### `ghToken: string`\n    - Provide a Github Token for accessing nexe releases\n    - This is usually needed in CI environments\n    - default: `process.env.GITHUB_TOKEN`\n - #### `sourceUrl: string`\n    - Provide an alternate url for the node source code\n    - Note: temporary files will still be created for this under the specified version\n - #### `loglevel: string`\n    - Set the loglevel, info, silent, or verbose\n    - default: `\'info\'`\n - #### `patches: NexePatch[]`\n    - Userland patches for patching or modifying node source\n    - default: `[]`\n - #### `plugins: NexePatch[]`\n    - Userland plugins for modifying nexe executable behavior\n    - default: `[]`\n\n### `NexePatch: (compiler: NexeCompiler, next: () => Promise<void>) => Promise<void>`\n\nPatches and Plugins are just a middleware functions that take two arguments, the `compiler`, and `next`. The compiler is described below, and `next` ensures that the pipeline continues. Its invocation should always be awaited or returned to ensure correct behavior. Patches also require that [`--build`](#build-boolean) be set, while plugins do not.\n\nFor examples, see the built in patches: [src/patches](src/patches).\n\n### `NexeCompiler`\n\n - `setFileContentsAsync(filename: string, contents: string): Promise<void>`\n    - Quickly set a file\'s contents within the downloaded Node.js source.\n - `replaceInFileAsync(filename: string, ...replaceArgs): Promise<void>`\n    - Quickly perform a replace in a file within the downloaded Node.js source. The rest arguments are passed along to `String.prototype.replace`\n - `readFileAsync(filename: string): Promise<NexeFile>`\n    - Access (or create) a file within the downloaded Node.js source.\n - `addResource(filename: string, contents: Buffer): void`\n    - Add a resource to the nexe bundle\n - `files: NexeFile[]`\n    - The cache of the currently read, modified, or created files within the downloaded Node.js source.\n\n#### `NexeFile`\n  - `contents: string`\n  - `absPath: string`\n  - `filename: string`\n\nAny modifications made to `NexeFile#contents` will be maintained in the cache _without_ the need to explicitly write them back out, e.g. using `NexeCompiler#setFileContentsAsync`.\n\n## Native Modules\n\nTODO\n\n- [ ] Implement support for `node-pre-gyp#find`.\n\n## Contributing\n\nBuilding\n```\n$ git clone git@github.com:nexe/nexe.git\n$ cd nexe\n$ yarn\n```\n\nTesting\n```\n$ npm test\n```\n\n## Maintainers\n\n[![Jared Allard](https://avatars.githubusercontent.com/u/2391349?s=130)](https://jaredallard.me/) | [![Caleb Boyd](https://avatars.githubusercontent.com/u/5818726?s=130)](https://github.com/calebboyd) | [![Dustin Greif](https://avatars.githubusercontent.com/u/3026298?s=130)](https://github.com/dgreif) |\n---|---|---|\n[Jared Allard](https://github.com/jaredallard) | [Caleb Boyd](http://github.com/calebboyd) |  [Dustin Greif](https://github.com/dgreif) |\n\n### Former\n\n- [Craig Condon](http://crcn.codes/)\n'
