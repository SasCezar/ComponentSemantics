The acronym \xe2\x80\x9cMEVN\xe2\x80\x9d stands for \xe2\x80\x9cMongoDB Express.js VueJS Node.js\xe2\x80\x9d. It offers a super simple boilerplate template and additional utilities for building a MEVN stack based webapp. It takes away the hassle of setting up the local development environment which may become a nightmare especially for beginners who are just starting out.\n\n- [Basic](http://github.com/madlabsinc/mevn-starter-templates/)\n- [Pwa](http://github.com/madlabsinc/mevn-starter-templates/tree/pwa)\n- [GraphQL](http://github.com/madlabsinc/mevn-starter-templates/tree/graphql)\n- [Nuxt.js](http://github.com/madlabsinc/mevn-starter-templates/tree/nuxt)\n\n## Installation\n\n### Prerequisites\n\n- [**npm**](https://www.npmjs.com/) it is a package manager for the JavaScript programming language.\n- [**node.js**](https://nodejs.org/en/) is an open-source, cross-platform JavaScript run-time environment that executes JavaScript code outside of a browser.\n- [**git**](https://git-scm.com/) is a version control system for tracking changes in computer files and coordinating work on those files among multiple people. It is primarily used for source code management in software development.\n\n## Quickstart\n\n``` bash\nnpm install -g mevn-cli\nmevn init <appname>\n```\n\n### Available Commands\n\n `Mevn-CLI` offers the following set of commands:-\n\n| command | description |                                                                                                \n| -------------- |  ---------------- |\n| mevn init | Bootstraps a MEVN stack based boilerplate template of choice |\n| mevn serve | Serves the client/server side template locally |\n| mevn add | Adds additional plugins as required on the go |\n| mevn generate | Generates component, model, route, controller, and DB config files |\n| mevn codesplit | Lazy load components as required |\n| mevn dockerize | Serves the client and server in separate docker containers |\n| mevn deploy | Deploys the webapp to a cloud service of choice |\n| mevn info | Prints debugging information about the local environment |\n\n## Features\n\n- It allows Developers to build webapps with ease in which all the local environment setup is being taken care of. All he/she has to do is to focus on writing actual code.\n- Its written in ES6 syntax, which is a developer-friendly syntax that keeps the code simpler and smaller.\n- The whole project is done in modules(thanks to ES6 syntax) which enhances user readability and much more compact code.  \n- Mevn-CLI simplifies the entire workflow by generating boilerplate code as required, automating redundant tasks etc.\n\n## Contributing\n\nBefore contributing to this repository, please first discuss the change you wish to make via issue, or any other method with the owners of this repository before making a change. Take a look at the [Contributing Guidelines](https://github.com/madlabsinc/mevn-cli/wiki/Contributing-Guidelines) to get a better picture regarding the codebase and project structure.\n\n### How do I contribute?\n1. Ensure you have no "dummy" files left, if you do then add them to the bottom of `.gitignore`.\n2. Fork and clone our repository.\n3. Make your life-changing changes.\n4. Run `npm run build` which generates a `lib` directory with the transpiled `es5` code.\n5. Fire in `sudo npm link` to test everything works fine.\n6. Commit and push your changes.\n7. Make a detailed pull request.\n\n> `npm link` creates a symlink in the global folder making the `mevn` command globally available within your local development environment\n\n### Why should I contribute?\nContributing helps people and makes the world simply a better place, without contributors this project would cease to exist.\n\n### How should I write a commit message?\nThis project uses [Commitlint](https://github.com/conventional-changelog/commitlint/#what-is-commitlint) to check if the commit messages meet the [conventional commit format](https://www.conventionalcommits.org/en/v1.0.0/).\nThe full pattern is:\n```sh\ntype(scope?): subject #scope is optional\n\nbody? #body is optional\n\nfooter? #footer is optional\n```\n\nFollowing that pattern, your commit messages should look like these:\n```sh\nfeat: activate open collective\n```\n\n```sh\nchore: correct typo\n\nIt should be "guest" and not "gest"\n```\n\n```sh\nrefactor(cli): drop support for node 6\n\nBREAKING CHANGE: you will need to update your node version to keep using this CLI\nThis closes #123\n```\n\n### What if I cannot code or do not like it?\nYou can always write documentation, most repositories lack in it.\n\n### What is next?\nNothing! You\'re done and ready to get coding!\n\n\n## Versioning And Help\n\n| option | description\n| --- | --- |\n| -V, --version | Check CLI version |\n| -h, --help | Get help and check usage |\n\n## Contributors\n\n### Code Contributors\n\nThis project exists thanks to all the people who contribute.
