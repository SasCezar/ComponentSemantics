b"# Bookmarks.dev-API\n\nBack-end API for the [www.bookmarks.dev](http://www.bookmarks.dev).\n\n## Built With\n* [MongoDB](https://docs.mongodb.com/manual/)\n* [ExpressJS](https://expressjs.com/en/api.html)\n* [Angular](https://angular.io/docs/ts/latest/)\n* [NodeJS](https://nodejs.org/en/docs/)\n* [Keycloak](http://www.keycloak.org/) for authentication and authorization:\n \n ![components-graph](docs/graphviz/components-graph.png)\n \n> The production setup is can be found here - [components production](https://raw.githubusercontent.com/wiki/CodepediaOrg/bookmarks-api/images/network-diagram.png)\n***\n\n## Getting Started \n\nThese instructions will get you a copy of the project up and running on your local machine for development and testing purposes.\n See deployment for notes on how to deploy the project on a live system.\n\n### Prerequisites\n\nWhat you need to run this app:\n* `node` and `npm` (we recommend using [NVM](https://github.com/creationix/nvm))\n  * Ensure you're running Node at least (`v6.x.x`+) and NPM (`3.x.x`+)\n* [nodemon](https://nodemon.io/) - `npm install -g nodemon`\n* **Docker** - we recommend using [Docker Desktop](https://www.docker.com/products/docker-desktop)\n\n### Installing (development setup)\n\nA step by step series of examples that tell you how to get a development env running\n\n#### Start MongoDB and Keycloak server\n\n```bash\ndocker-compose up\n\nOR\n\ndocker-compose up -d # if you want it to run in the background\n```\n\n#### Create a Keycloak user (2 min)\nTo test with the front-end [bookmarks.dev](https://github.com/CodepediaOrg/bookmarks.dev) create a Keycloak test user.\nFollow the screenshots in [Add a Keycloak user](docs/keycloak/add-keycloak-user.md) to create it.\n\n#### Start the development server (API)\n\n```bash\nnvm use\nnpm install\nnpm run debug\n```\n\nThis will start the API with [nodemon](http://nodemon.io) and will watch for code changes and automatically redeploy. \n\n> To be able to automatically add youtube videos published date and duration to the title you need to \ncreate a _nodemon.json_ file based on the [nodemon.json.example](nodemon.json.example) and a real youtube api key\n\n## Testing\n\n### Integration tests\n\n> Start Keycloak and mongodb by issuing the `docker-compose up` command\n\nRun the integration test by issuing the following command.\n\n```bash\nnpm run integration-tests\n```\n\nA report will be generated.\n\n## Swagger docs\n\n* [local](http://localhost:3000/api/docs)\n* [production](https://www.bookmarks.dev/api/docs)\n\n## Deployment\n\nWe currently use [pm2](https://pm2.keymetrics.io/) to start the project in production.\n\n### Start\n```bash\npm2 start pm2-process.json --env production\n```\n\n> Commited is a [pm2-process.exammple.json](pm2-process.exammple.json) example file\n\n### Restart\n```bash\npm2 restart pm2-process.json --env production\n```\n\n### Stop\n```bash\npm2 stop pm2-process.json --env production\n```\n\n## Contributing  \nPlease read [CONTRIBUTING.md](https://gist.github.com/PurpleBooth/b24679402957c63ec426) for details on our code of conduct, and the process for submitting pull requests to us.\n\n## Versioning\n\nWe use [SemVer](http://semver.org/) for versioning.
