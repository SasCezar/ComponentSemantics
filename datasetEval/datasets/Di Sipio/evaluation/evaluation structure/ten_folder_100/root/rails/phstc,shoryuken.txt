b"# Shoryuken\n\n![Shoryuken](shoryuken.jpg)\n\nShoryuken _sho-ryu-ken_ is a super-efficient [Amazon SQS](https://aws.amazon.com/sqs/) thread-based message processor.\n\n[![Build Status](https://travis-ci.org/phstc/shoryuken.svg)](https://travis-ci.org/phstc/shoryuken)\n[![Code Climate](https://codeclimate.com/github/phstc/shoryuken/badges/gpa.svg)](https://codeclimate.com/github/phstc/shoryuken)\n\n## Key features\n\n- [Rails Active Job](https://github.com/phstc/shoryuken/wiki/Rails-Integration-Active-Job)\n- [Queue Load balancing](https://github.com/phstc/shoryuken/wiki/Shoryuken-options#load-balancing)\n- [Concurrency per queue](https://github.com/phstc/shoryuken/wiki/Processing-Groups)\n- [Long Polling](https://github.com/phstc/shoryuken/wiki/Long-Polling)\n- [Batch processing](https://github.com/phstc/shoryuken/wiki/Worker-options#batch)\n- [Auto extend visibility timeout](https://github.com/phstc/shoryuken/wiki/Worker-options#auto_visibility_timeout)\n- [Exponential backoff](https://github.com/phstc/shoryuken/wiki/Worker-options#retry_intervals)\n- [Middleware support](https://github.com/phstc/shoryuken/wiki/Middleware)\n- Amazon SQS CLI. See `shoryuken help sqs`\n\n## Requirements\n\nRuby 2.0 or greater.\n\n## Installation\n\nAdd this line to your application's Gemfile:\n\n```ruby\ngem 'shoryuken'\n```\n\nIf you are using AWS SDK version 3, please also add this line:\n\n```ruby\ngem 'aws-sdk-sqs'\n```\n\nThe extra gem `aws-sdk-sqs` is required in order to keep Shoryuken compatible with AWS SDK version 2 and 3. \n\nAnd then execute:\n\n```shell\n$ bundle\n```\n\n## Usage\n\nCheck the [Getting Started](https://github.com/phstc/shoryuken/wiki/Getting-Started) page.\n\n## More Information\n\nFor more information check the [wiki page](https://github.com/phstc/shoryuken/wiki).\n\n## Credits\n\n[Mike Perham](https://github.com/mperham), creator of [Sidekiq](https://github.com/mperham/sidekiq), and [everybody who contributed to it](https://github.com/mperham/sidekiq/graphs/contributors).
