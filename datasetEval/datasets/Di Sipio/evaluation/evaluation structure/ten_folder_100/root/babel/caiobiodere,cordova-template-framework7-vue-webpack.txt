But you can develop your app with this way:\n\n1. `cordova platform add ios browser` (browser needs for development in live-reload mode.)\n2. `cordova run ios -- --lr` (wait till app opens in your ios emulator or phone. it will close console output after publish, so live-reload will not work. don\'t close the app and go to next step.)\n3. `cordova run browser -- --lr` (you can use live-reload in your phone-emulator and browser at same time. you can edit your files in live-reload mode now.)\n\n---\n\n## Features\n\nHooks are smart. They can fix some problems for you. Fix list:\n* `npm install` Automatically checks node js dependencies.\n* `package.json` Renames `name` variable if it has a space characters and auto saves. (It needed for npm install)\n* `www` Automatically manages `www` folder. You don\'t need to think about www folder. Your target is always `src` folder.\n* `static` Static assets automatically sync on live reload!\n* `CordovaHtmlOutputPlugin` Automatically adds `cordova.js` to html. You don\'t need to add to your file manually. It\'s helpful for webpack.\n* `manifest.json` Some cordova plugins needs manifest.json in root folder. If you add manifest.json file to your `src` folder, our smart hooks automagically copy it to `www` folder!\n\n###### Live Reload Related\n* `config.xml` Live reload needs `<allow-navigation href="*"/>` in development mode. So our smart hooks manages this too. You don\'t need to think about it.\n* `live-reload` Manages live-reload dependencies automatically. Just write your code, and don\'t think about dependencies.\n* `device_router.html` Smart router in live-reload mode. It searches for best available ip for connect server. if it can\'t find, you can write ip:port manually.\n* `CordovaDeviceRouter.js` In live-reload mode, you can connect to server from multiple devices. This file inject right `cordova.js` file to page. So you can connect to webpack-dev-server from multiple devices at same time.\n\n## Installation\n### IMPORTANT: Phonegap build tools not supported currently. I suggest to use cordova with this template.\n\nThis template need cordova or phonegap, for more information [cordova installation](https://cordova.apache.org/docs/en/latest/guide/cli/) or [phonegap installation](http://docs.phonegap.com/getting-started/1-install-phonegap/desktop/).\n\nOur Magic words:\n\n``` bash\ncordova create <project_create_dir> [com.example.projectname] [ProjectClassName] --template cordova-template-framework7-vue-webpack\nphonegap create <project_create_dir> [com.example.projectname] [ProjectClassName] --template cordova-template-framework7-vue-webpack\n```\n\nboom! :boom: you have your brand new cordova / phonegap project with framework7 - vue 2 and webpack 4!\n\n![such a wow!](https://cloud.githubusercontent.com/assets/296796/3511506/4042665c-06b0-11e4-953c-4f14c11f81ec.png "such a wow!")\n\n## Installation using pre-defined templates\n\nFramework v4 version\n\n``` bash\ncordova create cordova-template com.template DefaultTemplate --template git://github.com/caiobiodere/cordova-template-framework7-vue-webpack.git#master\n```\n\n[Framework Single View v4 version](https://github.com/framework7io/framework7-template-single-view)\n\n``` bash\nStill under development\n```\n\n[Framework Tabbed Views v4 version](https://github.com/framework7io/framework7-template-tabs)\n\n``` bash\nStill under development\n```\n\n[Framework Split View v4 version](https://github.com/framework7io/framework7-template-split-view)\n\n``` bash\nStill under development\n```\n\nFramework v3 version\n\n``` bash\ncordova create cordova-template com.template DefaultTemplate --template git://github.com/caiobiodere/cordova-template-framework7-vue-webpack.git#feature/v3-default-template\n```\n\n---\n\n## Usage\n\nYou can use every cordova | phonegap commands.\nYou just have one more command option: `-- --lr`.
