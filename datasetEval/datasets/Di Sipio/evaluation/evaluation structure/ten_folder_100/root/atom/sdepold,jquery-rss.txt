b'# jquery.rss [![Build Status](https://travis-ci.org/sdepold/jquery-rss.svg?branch=master)](https://travis-ci.org/sdepold/jquery-rss)\n\nThis plugin can be used to read a RSS feed and transform it into a custom piece of HTML.\n\n## Alternatives\n\nA vanilla JavaScript version of this library can be found here: [Vanilla RSS](https://github.com/sdepold/vanilla-rss).\nThis plugin uses [Feedr](https://github.com/sdepold/feedr), a backend server that parses and converts RSS feeds into its JSON representation. The server was built as a drop-in replacement for Google\'s former Feed API.\n\n## Support\n\nSince version 3.4.0 of jquery.rss, users have the chance to support funding future developments and\ncovering the costs for the hosting of jquery.rss\' respective server side companion app [feedr](https://github.com/sdepold/feedr).\n\nEvery once in a while supporters will get affiliate links instead of one of the feed\'s entries.\n\nIf you are not interested in supporting the authors of the plugin, then you can easily opt-out of it by setting the respective\n`support` option. See below for further details.\n\nThanks in advance!\n\n## Installation\n\nThrough npm:\n\n```\n$ npm install jquery\n$ npm install jquery-rss\n\nconst $ = require(\'jquery\');\nrequire(\'jquery-rss); // This will add the plugin to the jQuery namespace\n```\n\nThrough cdnjs:\n\n```\n<script src="http://code.jquery.com/jquery-1.11.0.js"></script>\n<script src="https://cdnjs.cloudflare.com/ajax/libs/jquery-rss/3.3.0/jquery.rss.min.js"></script>\n```\n\n## Setup\n\n```html\n<!DOCTYPE html>\n<html>\n  <head>\n    <title>jquery.rss example</title>\n    <script src="lib/jquery-1.6.4.min.js"></script>\n    <script src="https://cdnjs.cloudflare.com/ajax/libs/moment.js/2.8.4/moment.min.js"></script>\n    <script src="dist/jquery.rss.min.js"></script>\n    <script>\n      jQuery(function($) {\n        $("#rss-feeds").rss("http://feeds.feedburner.com/premiumpixels");\n      });\n    </script>\n  </head>\n  <body>\n    <div id="rss-feeds"></div>\n  </body>\n</html>\n```\n\nDemo link for above code: http://embed.plnkr.co/WQRoCYLld162uplnz1rc/preview\n\nNote: Moment.js is _optional_. If you include it, jquery.rss will use it to format dates.\nIf you do not want to include Moment.js, you may opt for providing your own date formatting function, or for not formatting dates at all.\n\n## Options\n\n```javascript\n$("#rss-feeds").rss(\n  // You can either provide a single feed URL or a list of URLs (via an array)\n  "http://feeds.feedburner.com/premiumpixels",\n  {\n    // That server is currently running on Heroku and might have some downtimes, interruptions or unexpected issues. While I will try to keep those problems as rare as possible, it can totally happen from time to time. I might move the service to some other provide or even improve the infrastructure.\n\nIf you don\'t want to rely on the [provided server](http://feedrapp.info) and instead run your own version, you can just download feedr, install the dependencies and run it. As written above, you can specify the host which is used to parse the feeds with the `host` option.\n\n## Templating\n\nAs seen in the options, you can specify a template in order to transform the json objects into HTML. In order to that, you can either define the outer template (which describes the html around the entries) or the entry template (which describes the html of an entry).\n\nThe basic format of those templates are:\n\n```html\n<!-- layoutTemplate: -->\n"<outer-html>{entries}</outer-html>"\n\n<!-- entryTemplate: -->\n"<any-html>{token1}{token2}</any-html>"\n```\n\nSo, let\'s say you have specified a limit of 2, using the upper pseudo html. This will result in the following:\n\n```html\n<outer-html>\n  <any-html>{token1}{token2}</any-html>\n  <any-html>{token1}{token2}</any-html>\n</outer-html>\n```\n\nThere are some predefined tokens:\n\n- url: the url to the post\n- author: the author of the post\n- date: the publishing date\n- title: the title of the post\n- body: the complete content of the post\n- shortBody: the shortened content of the post\n- bodyPlain: the complete content of the post without html\n- shortBodyPlain: the shortened content of the post without html\n- teaserImage: the first image in the post\'s body\n- teaserImageUrl: the url of the first image in the post\'s body\n- index: the index of the current entry\n- totalEntries: the total count of the entries\n- feed: contains high level information of the feed (e.g. title of the website)\n\nYou can also define custom tokens using the `tokens` option:\n\n```javascript\n$("#foo").rss(url, {\n  entryTemplate: "{dynamic}, {static}, {re-use}",\n  tokens: {\n    dynamic: function(entry, tokens) {\n      return "dynamic-stuff: " + entry.title;\n    },\n    "re-use": function(entry, tokens) {\n      return encodeURIComponent(tokens.teaserImageUrl);\n    },\n    static: "static"\n  }\n});\n```\n\nPlease make sure to NOT define infinite loops. The following example is really BAD:\n\n```javascript\n$(\'#foo\').rss(url, {\n  entryTemplate: "{loop}",\n  tokens: {\n    whoops: function(entry, tokens) { return tokens.loop() }\n    loop: function(entry, tokens) { return tokens.whoops() }\n  }\n})\n```\n\nHere is a real-world example:\n\n```javascript\n$("#foo").rss(url, {\n  layoutTemplate: "<table><tr><th>Title</th></tr>{entries}</table>",\n  entryTemplate: "<tr><td>{title}</td></tr>"\n});\n```\n\n## Filtering\n\nThe plugin also allows you to filter specific entries in order to only print them:\n\n```javascript\n$("#foo").rss(url, {\n  limit: 100,\n  filterLimit: 10,\n  filter: function(entry, tokens) {\n    return tokens.title.indexOf("my filter") > -1;\n  }\n});\n```\n\nThis will request 100 entries via the Feed API and renders the first 10 matching entries.\n\n## Testing\n\nThe test suite is using BusterJS.
