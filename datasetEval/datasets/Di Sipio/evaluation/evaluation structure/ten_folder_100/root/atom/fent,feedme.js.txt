b'# FeedMe.js\n\n\nfeedme.js is an RSS/Atom/JSON feed parser. How is this different from the other few feed parsers? It uses [sax-js](https://github.com/isaacs/sax-js) for xml parsing and [clarinet](https://github.com/dscape/clarinet) for json parsing. That means it is coded in pure Javascript and thus more deployable. I needed a parser that wouldn\'t require me to install external dependencies or to compile anything.\n\n[![Build Status](https://secure.travis-ci.org/fent/feedme.js.svg)](http://travis-ci.org/fent/feedme.js)\n[![Dependency Status](https://david-dm.org/fent/feedme.js.svg)](https://david-dm.org/fent/feedme.js)\n[![codecov](https://codecov.io/gh/fent/feedme.js/branch/master/graph/badge.svg)](https://codecov.io/gh/fent/feedme.js)\n\n\n# Usage\n\n```js\nconst FeedMe = require(\'feedme\');\nconst http = require(\'http\');\n\nhttp.get(\'http://www.npr.org/rss/rss.php?id=1001\', (res) => {\n  if (res.statusCode != 200) {\n    console.error(new Error(`status code ${res.statusCode}`));\n    return;\n  }\n  let parser = new FeedMe();\n  parser.on(\'title\', (title) => {\n    console.log(\'title of feed is\', title);\n  });\n  parser.on(\'item\', (item) => {\n    console.log();\n    console.log(\'news:\', item.title);\n    console.log(item.description);\n  });\n  res.pipe(parser);\n});\n```\n\n\n# API\n\n### new FeedMe([buffer])\nCreates a new instance of the FeedMe parser. `buffer` can be `true` if you want the parser to buffer the entire feed document as a JSON object, letting you use the `FeedMe#done()` method.\n\n### parser.write(xml)\nWrite to the parser.\n\n### parser.done()\nCan only be used if `buffer` is `true`. It returns the feed as a Javascript object, should be called after `end` is emitted from the parser. Subelements are put as children objects with their names as keys. When one object has more than one child of the same name, they are put into an array. Items are always put into an array.\n\n```js\nconst FeedMe = require(\'feedme\');\nconst http = require(\'http\');\n\nhttp.get(\'https://nodejs.org/en/feed/blog.xml\', (res) => {\n  let parser = new FeedMe(true);\n  res.pipe(parser);\n  parser.on(\'end\', () => {\n    console.log(parser.done());\n  });\n});\n```\n\nAn example of what `parser.done()` could return.\n\n```js\n{\n  type: \'rss 2.0\',\n  title: \'Liftoff News\',\n  link: \'http://liftoff.msfc.nasa.gov/\',\n  description: \'Liftoff to Space Exploration.\',\n  language: \'en-us\',\n  pubdate: \'Tue, 10 Jun 2003 04:00:00 GMT\',\n  lastbuilddate: \'Tue, 10 Jun 2003 09:41:01 GMT\',\n  docs: \'http://blogs.law.harvard.edu/tech/rss\',\n  generator: \'Weblog Editor 2.0\',\n  managingeditor: \'editor@example.com\',\n  webmaster: \'webmaster@example.com\',\n  items:  [\n    {\n      title: \'Star City\',\n      link: \'http://liftoff.msfc.nasa.gov/news/2003/news-starcity.asp\',\n      description: \'How do Americans get ready to work with Russians aboard the International Space Station? The proposed VASIMR engine would do that.\',\n      pubdate: \'Tue, 27 May 2003 08:37:32 GMT\',\n      guid: \'http://liftoff.msfc.nasa.gov/2003/05/27.html#item571\'    },\n    {\n      title: \'Astronauts\\\' Dirty Laundry\',\n      link: \'http://liftoff.msfc.nasa.gov/news/2003/news-laundry.asp\',\n      description: \'Compared to earlier spacecraft, the International Space Station has many luxuries, but laundry facilities are not one of them.  Instead, astronauts have other options.\',\n      pubdate: \'Tue, 20 May 2003 08:56:02 GMT\',\n      guid: \'http://liftoff.msfc.nasa.gov/2003/05/20.html#item570\'    }\n  ]\n}\n```\n\n\n### Event: \'item\'\n* `Object` - Item from the feed.\n\nEmitted whenever the parser finds a new feed item. An item can be inside an `<item>` or <entry>` tag.\n\n### Event: `tagname`\n* `Object` - The object containing the value of the tag found.\n\nEmitted whenever a tag on the root of the document is finished parsing. The root being the `<channel>` or `<feed>` tag. Example:\n\n```js\nparser.on(\'description\', (d) => {\n  // do something\n});\n```\n\n### Event: \'type\'\n* `string` - Type of feed. Example: atom, rss 2.0, json.\n\n### Event: \'error\'\n* `Error`\n\nEmitted when there is an error parsing the document.\n\n\n# Install\n\n    npm install feedme\n\n\n# Tests\nTests are written with [mocha](https://mochajs.org)\n\n```bash\nnpm test\n```\n'
