b'Partisan\n=======================================================\n\n[![Build Status](https://travis-ci.org/lasp-lang/partisan.svg?branch=master)](https://travis-ci.org/lasp-lang/partisan)\n\nPartisan is a flexible, TCP-based membership system for Erlang/Elixir.\n\nPartisan features:\n\n* Single node testing, facilitated by a disterl control channel for figuring out which ports the peer service is operating at.\n* Messages are sent via TCP connections that are maintained to all cluster members.\n* Failure detection is performed TCP.\n* Connections are verified at each gossip round.\n* Configurable fanout.\n* On join, gossip is performed immediately, instead of having to wait for the next gossip round.\n* HyParView implementation.\n\nPartisan has many available peer service managers:\n\n* Full membership with TCP-based failure detection: `partisan_default_peer_service_manager.`\n* Client/server topology: `partisan_client_server_peer_service_manager.`\n* HyParView, hybrid partial view membership protocol, with TCP-based failure detection: `partisan_hyparview_peer_service_manager.`\n* Static topology: `partisan_static_peer_service_manager`.'
