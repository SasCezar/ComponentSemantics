b'# Nerves\n\nCraft and deploy bulletproof embedded software in Elixir\n\n[![Backers on Open Collective](https://opencollective.com/nerves-project/backers/badge.svg)](#backers)\n[![Sponsors on Open Collective](https://opencollective.com/nerves-project/sponsors/badge.svg)](#sponsors)\n[![CircleCI](https://circleci.com/gh/nerves-project/nerves/tree/master.svg?style=svg)](https://circleci.com/gh/nerves-project/nerves/tree/master)\n[![Hex version](https://img.shields.io/hexpm/v/nerves.svg "Hex version")](https://hex.pm/packages/nerves)\n\n## Host Requirements\n\n* Mac OS 10.10+\n* 64-bit Linux (tested on Debian / Ubuntu / Redhat / CentOS)\n* Windows 10 with [Windows Subsystem for Linux](https://msdn.microsoft.com/en-us/commandline/wsl/install_guide) (experimental)\n* Raspberry Pi 3 (experimental)\n* Elixir ~> 1.4\n\nSee [Installation Docs](https://hexdocs.pm/nerves/installation.html) for\nsoftware dependencies.\n\n## Quick-Reference\n\n### Generating a New Nerves Application\n\n```bash\nmix nerves.new my_app\n```\n\n### Building Firmware\n\n```bash\nexport MIX_TARGET=rpi3\nmix deps.get      # Fetch the dependencies\nmix firmware      # Cross-compile dependencies and create a .fw file\nmix firmware.burn # Burn firmware to an inserted SD card\n```\n\n**Note:** The `mix firmware.burn` target relies on the presence of `ssh-askpass`. Some\nusers may need to export the `SUDO_ASKPASS` environment variable to point to their askpass\nbinary.
