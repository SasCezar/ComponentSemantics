b'![](https://i.imgur.com/qlW1eEG.png)\n\n# pnpm\n\n> Fast, disk space efficient package manager\n\n[![npm version](https://img.shields.io/npm/v/pnpm.svg)](https://www.npmjs.com/package/pnpm)\n[![Status](https://travis-ci.org/pnpm/pnpm.svg?branch=master)](https://travis-ci.org/pnpm/pnpm "See test builds")\n[![Windows build status](https://ci.appveyor.com/api/projects/status/f7437jbcml04x750/branch/master?svg=true)](https://ci.appveyor.com/project/zkochan/pnpm-17nv8/branch/master)\n[![Join the chat at https://gitter.im/pnpm/pnpm](https://badges.gitter.im/pnpm/pnpm.svg)](https://gitter.im/pnpm/pnpm?utm_source=badge&utm_medium=badge&utm_campaign=pr-badge&utm_content=badge)\n[![OpenCollective](https://opencollective.com/pnpm/backers/badge.svg)](#backers)\n[![OpenCollective](https://opencollective.com/pnpm/sponsors/badge.svg)](#sponsors)\n[![Twitter Follow](https://img.shields.io/twitter/follow/pnpmjs.svg?style=social&label=Follow)](https://twitter.com/pnpmjs)\n\nFeatures:\n\n* **Fast.** As fast as npm and Yarn.\n* **Efficient.** One version of a package is saved only ever once on a disk.\n* **Great for [multi-package repositories](https://pnpm.js.org/en/workspaces) (a.k.a. monorepos).**\n* **Strict.** A package can access only dependencies that are specified in its `package.json`.\n* **Deterministic.** Has a lockfile called `pnpm-lock.yaml`.\n* **Works everywhere.** Works on Windows, Linux and OS X.\n\nLike this project? Let people know with a [tweet](https://bit.ly/tweet-pnpm).\n\n## Table of Contents\n\n* [Background](#background)\n* [Install](#install)\n* [Usage](#usage)\n* [Benchmark](#benchmark)\n* [Support](#support)\n* [Contributors](#contributors)\n\n## Background\n\npnpm uses hard links and symlinks to save one version of a module only ever once on a disk.\nWhen using npm or Yarn for example, if you have 100 projects using the same version\nof lodash, you will have 100 copies of lodash on disk. With pnpm, lodash will be saved in a\nsingle place on the disk and a hard link will put it into the `node_modules` where it should\nbe installed.\n\nAs a result, you save gigabytes of space on your disk and you have a lot faster installations!\nIf you\'d like more details about the unique `node_modules` structure that pnpm creates and\nwhy it works fine with the Node.js ecosystem, read this small article: [Flat node_modules is not the only way](https://medium.com/pnpm/flat-node-modules-is-not-the-only-way-d2e40f7296a3).\n\n## Install\n\nUsing a [standalone script](https://github.com/pnpm/self-installer#readme):\n\n```\ncurl -L https://unpkg.com/@pnpm/self-installer | node\n```\n\nVia npm:\n\n```\nnpm add -g pnpm\n```\n\nOnce you first installed pnpm, you can upgrade it using pnpm:\n\n```\npnpm add -g pnpm\n```\n\n> Do you wanna use pnpm on CI servers? See: [Continuous Integration](https://pnpm.js.org/en/continuous-integration).\n\n## Usage\n\n### pnpm CLI\n\nJust use pnpm in place of npm. For instance, to install run:\n\n```\npnpm install\n```\n\nFor more advanced usage, read [pnpm CLI](https://pnpm.js.org/en/pnpm-cli) on our website.\n\nFor using the programmatic API, use pnpm\'s engine: [supi](packages/supi).\n\n### pnpx CLI\n\nnpm has a great package runner called [npx](https://medium.com/@maybekatz/introducing-npx-an-npm-package-runner-55f7d4bd282b).\npnpm offers the same tool via the `pnpx` command. The only difference is that `pnpx` uses pnpm for installing packages.\n\nThe following command installs a temporary create-react-app and calls it,\nwithout polluting global installs or requiring more than one step!\n\n```\npnpx create-react-app my-cool-new-app\n```\n\n## Benchmark\n\npnpm is as fast as npm and Yarn.
