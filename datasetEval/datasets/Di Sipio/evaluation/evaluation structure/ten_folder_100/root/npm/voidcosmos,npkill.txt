You can then select which ones you want to erase to free up space. Yay!\n\n## Table of Contents\n\n- [Features](#features)\n- [Installation](#installation)\n- [Usage](#usage)\n  - [Options](#options)\n  - [Examples](#examples)\n- [Set Up Locally](#setup-locally)\n- [Roadmap](#roadmap)\n- [Known bugs](#known-bugs)\n- [Contributing](#contributing)\n- [Buy us a coffee](#donations)\n- [License](#license)\n\n<a name="features"></a>\n\n# :heavy_check_mark: Features\n\n- **Clear space:** Get rid of old and dusty node_modules cluttering up your machine.\n\n- **Very fast:** NPKILL is written in TypeScript, but searches are performed at a low level, improving performance greatly.\n\n- **Easy to use:** Say goodbye to lengthy commands. Using npkill is as simple as reading a list of your node_modules, and pressing Del to get rid of them. Could it be any easier? ;)\n\n- **Minified:** It barely has any dependencies.\n\n<a name="installation"></a>\n\n# :cloud: Installation\n\nYou don\'t really need to install it to use it!\nSimply use the following command:\n\n```bash\n$ npx npkill\n```\n\nOr if for some reason you really want to install it:\n\n```bash\n$ npm i -g npkill\n# Unix users may need to run the command with sudo. Go carefully\n```\n\n<a name="usage"></a>\n\n# :clipboard: Usage\n\n```bash\n$ npx npkill\n# or just npkill if installed globally\n```\n\nBy default, npkill will scan for node_modules starting at the path where `npkill` command is executed.\n\nMove between the listed folders with <kbd>\xe2\x86\x93</kbd> <kbd>\xe2\x86\x91</kbd>, and use <kbd>Space</kbd> to delete the selected folder.\nYou can also use <kbd>j</kbd> and <kbd>k</kbd> to move between the results\n\nTo exit, <kbd>Q</kbd> or <kbd>Ctrl</kbd> + <kbd>c</kbd> if you\'re brave.\n\n<a name="options"></a>\n\n## Options\n\n| ARGUMENT               | DESCRIPTION                                                                                                                                    |\n| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |\n| -c, --bg-color         | Change row highlight color. _(Available: **blue**, cyan, magenta, white, red and yellow)_                                                      |\n| -d, --directory        | Set the directory from which to begin searching. By default, starting-point is .                                                               |\n| -D, --delete-all       | CURRENTLY DISABLED. Automatically delete all node_modules folders that are found                                                               |\n| -e, --show-errors      | Show error messages related to the search if any                                                                                               |\n| -E, --exclude          | Exclude directories from search (directory list must be inside double quotes "", each directory separated by \',\' ) Example: "ignore1, ignore2" |\n| -f, --full             | Start searching from the home of the user (example: "/home/user" in linux)                                                                     |\n| -gb                    | Show folders in Gigabytes instead of Megabytes.                                                                                                |\n| -h, --help, ?          | Show this help page and exit                                                                                                                   |\n| -nu, --no-check-update | Don\'t check for updates on startup                                                                                                              |\n| -s, --sort             | Sort results by: size or path _[ beta ]_                                                                                                       |\n| -t, --target           | Specify the name of the directories you want to search (by default, is node_modules)                                                           |\n| -v, --version          | Show npkill version                                                                                                                            |\n\n**Warning:** _In future versions some commands may change_\n\n<a name="examples"></a>\n\n## Examples\n\n- Search **node_modules** directories in your _projects_ directory:\n\n```bash\nnpkill -d ~/projects\n\n# other alternative:\ncd ~/projects\nnpkill\n```\n\n- List directories named "dist" and show errors if any occur:\n\n```bash\nnpkill --target dist -e\n```\n\n- Displays the magenta color cursor... because I like magenta!\n\n```bash\nnpkill --color magenta\n```\n\n- List **vendor** directories in your _projects_ directory, sort by size, and show size in gb:\n\n```bash\nnpkill -d \'~/more projects\' -gb --sort size --target vendor\n```\n\n- List **node_modules** in your _projects_ directory, excluding the ones in _progress_ and _ignore-this_ directories:\n\n```bash\nnpkill -d \'projects\' --exclude "progress, ignore-this"\n```\n\n- Automatically delete all node_modules that have sneaked into your backups:\n\n```bash\n# Disabled for security reasons (you can use it in version 0.2.4 at your risk)\nnpkill -d ~/backups/ --delete-all\n```\n\n<a
