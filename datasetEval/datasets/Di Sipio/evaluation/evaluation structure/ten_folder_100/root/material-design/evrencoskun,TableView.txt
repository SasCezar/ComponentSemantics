TableView does not store or copy the data in its TableModel;\ninstead it maintains a map from the row indexes of the view to the row indexes of the model. \n\n### 1. ISortableModel to your Cell Model \n\nTo be able use this feature on your TableView. You have to implement **ISortableModel** to your Cell Model. This interface\nwants two methods from your cell model. These are:\n\n- To compare sorted items ordered by normal items in terms of "**are _items_ the same**":\n```java\n    String getId()\n```\n\n- To compare sorted items ordered by normal items in terms of  "**are _contents_ the same**":\n```java\n    Object getContent()\n```\n\nAs you seen getContent value returns Object. TableView controls the type of the object. And It sorts by considering to the type of class.\nSo you can sort any type of value. Such as; \n\n- **Number**\n- **String**\n- **Date**\n- **Boolean**\n- **Comparable**\n\n### 2. AbstractSorterViewHolder to your Column Header ViewHolder\n \nAbstractSorterViewHolder helps to listen to change of sorting actions. So you can do whatever you want on any sorting state.\n\n- This interface method will be called after each sorting process. *Note* : It will be also called every recycling process.\n```java\n    onSortingStatusChanged(SortState sortState)\n```\n- This method gives current Sorting state:\n```java\n    SortState getSortState()\n```\n\n#### Sorting States\n\n```java\n    /**\n     * Enumeration value indicating the items are sorted in increasing order.\n     * For example, the set <code>1, 4, 0</code> sorted in\n     * <code>ASCENDING</code> order is <code>0, 1, 4</code>.\n     */\n     ASCENDING,\n    \n    /**\n     * Enumeration value indicating the items are sorted in decreasing order.\n     * For example, the set <code>1, 4, 0</code> sorted in\n     * <code>DESCENDING</code> order is <code>4, 1, 0</code>.\n     */\n    DESCENDING,\n    \n    /**\n     * Enumeration value indicating the items are unordered.\n     * For example, the set <code>1, 4, 0</code> in\n     * <code>UNSORTED</code> order is <code>1, 4, 0</code>.\n     */\n    UNSORTED\n```\n\n### 3. Helper methods for sorting process\n\nSeveral helper methods have been inserted on TableView. These are:\n- To **sort the TableView according to a specified column**:\n```java\n    sortColumn(int column, SortState sortState)\n```\n- To **get the current sorting state of a column**:\n```java\n    SortState getSortingStatus(int column)\n```\n\n## Filtering\n\nAs of version **0.8.6**, a **filtering** functionality has been added.\n>Filtering, by definition and usage in this context, is displaying a subset of data into the TableView based on a given filter globally. on a specified column or combination.\n\n### Steps to implement filtering functionality into TableView\n\n1. Implement the **IFilterableModel** interface to your Cell Item Model. This interface would require you to implement the method ```getFilterableKeyword()```.\n2. The ```getFilterableKeyword()``` requires a **String** value to be returned. Make sure that this item is unique for a specific filter and without conflict to other filter Strings.\n3. Create a **Filter** instance passing the created TableView:\n```java\n    ...\n    private TableView   tableView;\n    private Filter      tableViewFilter;\n    ...    \n    @Override\n    protected void onCreate(Bundle savedInstanceState) {\n        ...\n        tableView = setUpTableView();\n        tableViewFilter = new Filter(tableView);\n        ...\n    }\n    ...\n```\n4. Filtering can be done for the whole table (global), for a specified column or combination. The filtering call is very easy to use once the **Filter** object is created:\n```java\n    ...\n    public void filterWholeTable(String filterKeyword) {\n        tableViewFilter.set(filterKeyword);\n    }\n    \n    // assuming that Gender column is the 4th column in the TableView\n    public void filterTableForGender(String genderFilterKeyword) {\n        tableViewFilter.set(3, genderFilterKeyword);\n    }\n    \n    public void filterTableColumnForKeyword(int column, String keyword) {\n        tableViewFilter.set(column, keyword);\n    }\n    ...\n```\n\n#### Filtering notes\n\n1. For clearing a filter, just pass an **empty** String ("" **AND NOT ```null```**) to the ```set``` method.\n```java\n    ...\n    public void clearWholeTableFilter() {\n        tableViewFilter.set("");\n    }\n    \n    // assuming that Gender column is the 4th column in the TableView\n    public void clearFilterForGender() {\n        tableViewFilter.set(3, "");\n    }\n    ...\n```\n2. Multiple filtering combinations are supported such as COLUMN + WHOLE TABLE filter or MULTIPLE COLUMNS filter: e.g. "Happy" + "Boy" + all strings with a \'-\' character.\n3. Based on step 2 of the implementation steps above, **FilterableKeyword** for different types of filters must be unique Strings **AND** no common substring. For instance, "**_male_**" and "**_female_**" should not be used together as filter keywords, since the method for processing filters uses ```String.contains(CharSequence s)```, the filtering process will return all data with **male** keyword, thus, **female** is also included in the filtered data set. Better use "boy" and "girl" or the _hashed_ values of your keyword Strings.\n4. Advanced usage: **FilterChangedListener** is also available and can be implemented if you want to do something whenever the TableView is filtered. Public listener methods include:\n```java\n    // FilterChangedListener implementation:\n    ...\n    tableView.getFilterHandler().addFilterChangedListener(filterChangedListener);\n    ...\n    \n    // The filterChangedListener variable:\n    private FilterChangedListener filterChangedListener =\n            new FilterChangedListener() {\n                /**\n                 * Called when a filter has been changed.\n                 *\n                 * @param filteredCellItems      The list of filtered cell items.\n                 * @param filteredRowHeaderItems The list of filtered row items.\n                 */\n                @Override\n                public void onFilterChanged(List<List<T>> filteredCellItems, List<T> filteredRowHeaderItems) {\n                    // do something here...\n                }\n                \n                /**\n                 * Called when the TableView filters are cleared.\n                 *\n                 * @param originalCellItems      The unfiltered cell item list.\n                 * @param originalRowHeaderItems The unfiltered row header list.\n                 */\n                @Override\n                public void onFilterCleared(List<List<T>> originalCellItems, List<T> originalRowHeaderItems) {\n                    // do something here...\n                }\n    };\n```\n\n## Pagination\n\nAs of version **0.8.6**, a **pagination** functionality has been added.\n>Pagination, by definition and usage in this context, is the division of the whole set of data into subsets called pages and loading the data into the TableView page-by-page and not the whole data directly. This is useful if you have a large amount of data to be displayed.\n\n### Steps to implement pagination functionality into TableView\n\n##### Creating views to control the Pagination\n\n_Depending on your preference, you may not follow the following and create your own implementation._\n1. Create a layout with the following components: Two **Button** views to control next and previous page, a **Spinner** if you want to have a customized number of pagination (e.g. 10, 20, 50, All), an **EditText** to have a user input on which page s/he wants to go to, a **TextView** to display details (e.g. _Showing page X, items Y-Z_)\n2. Asign the views with the controls and methods which are discussed below.\n\n##### Implementation of the pagination\n\n1. Create a **Pagination** instance passing the created TableView.\n```java\n    ...\n    private TableView   tableView;\n    private Pagination  mPagination;\n    ...    \n    @Override\n    protected void onCreate(Bundle savedInstanceState) {\n        ...\n        tableView = setUpTableView();\n        mPagination = new Pagination(mTableView);\n        ...\n    }\n    ...\n\n```\n\n- The Pagination class has three possible constructors: (1) passing the TableView instance only, (2) TableView and the initial ITEMS_PER_PAGE and (3) TableView, initial ITEMS_PER_PAGE and the OnTableViewPageTurnedListener. By default, if no ITEMS_PER_PAGE specified, the TableView will be paginated into **10** items per page.\n```java\n    /**\n     * Basic constructor, TableView instance is required.\n     *\n     * @param tableView The TableView to be paginated.\n     */\n    public Pagination(ITableView tableView) { ... }\n    \n    /**\n     * Applies pagination to the supplied TableView with number of items per page.\n     *\n     * @param tableView    The TableView to be paginated.\n     * @param itemsPerPage The number of items per page.\n     */\n    public Pagination(ITableView tableView, int itemsPerPage) { ... }\n    \n    /**\n     * Applies pagination to the supplied TableView with number of items per page and an\n     * OnTableViewPageTurnedListener for handling changes in the TableView pagination.\n     *\n     * @param tableView    The TableView to be paginated.\n     * @param itemsPerPage The number of items per page.\n     * @param listener     The OnTableViewPageTurnedListener for the TableView.\n     */\n    public Pagination(ITableView tableView, int itemsPerPage, OnTableViewPageTurnedListener listener) { ... }\n```\n2. **Loading the next page of items** into the TableView using the ```nextPage()``` method. You can assign this to your implementation of nextPageButton onClick action:\n```java\n    ...\n    public void nextTablePage() {\n        mPagination.nextPage();\n    }\n    ...\n```\n\n2. **Loading the previous page of items** into the TableView using the ```previousPage()``` method. You can assign this to your implementation of previousPageButton onClick action:\n```java\n    ...\n    public void previousTablePage() {\n        mPagination.previousPage();\n    }\n    ...\n```\n\n3. You can navigate through the pages by **going to a specific page directly** using the ```goToPage(int page)``` method. You can assign this to the EditText field TextChanged action (using TextWatcher):\n```java\n    ...\n    public void goToTablePage(int page) {\n        mPagination.goToPage(page);\n    }\n    ...\n```\n\n4. You can customize and **set the number of items to be displayed per page** of the TableView using the ```setItemsPerPage(int itemsPerPage)``` method. You can assign this to your Spinner with the number of items per page list:\n```java\n    ...\n    public void setTableItemsPerPage(int itemsPerPage) {\n        mPagination.setItemsPerPage(itemsPerPage);\n    }\n    ...\n```\n\n5. Advanced usage: A listener interface (**Pagination.OnTableViewPageTurnedListener**) can also be implemented if you want to do something everytime _a page is turned_ (e.g. previous, next, goToPage or change items per page action is called):\n```java\n    ...\n        mPagination = new Pagination(mTableView);\n        mPagination.setOnTableViewPageTurnedListener(onTableViewPageTurnedListener);\n    ...\n    \n    private Pagination.OnTableViewPageTurnedListener onTableViewPageTurnedListener =\n            new Pagination.OnTableViewPageTurnedListener() {\n                /**\n                 * Called when the page is changed in the TableView.\n                 *\n                 * @param numItems   The number of items currently being displayed in the TableView.\n                 * @param itemsStart The starting item currently being displayed in the TableView.\n                 * @param itemsEnd   The ending item currently being displayed in the TableView.\n                 */\n                @Override\n                public void onPageTurned(int numItems, int itemsStart, int itemsEnd) {\n                    // Do something here...\n                    // You can update a TextView to display details (e.g. Showing page X, items Y-Z)\n                }\n            };\n    ...\n```\n\n#### Pagination notes\n1. Other methods which can be used from Pagination:\n```java\n    /**\n     * Removes the OnTableViewPageTurnedListener for this Pagination.\n     */\n    void removeOnTableViewPageTurnedListener();\n    \n    /**\n     * @return The current page loaded to the table view.\n     */\n    int getCurrentPage();\n    \n    /**\n     * @return The number of items per page loaded to the table view.\n     */\n    int getItemsPerPage();\n    \n    /**\n     * @return The number of pages in the pagination.\n     */\n    int getPageCount();\n    \n    /**\n     * @return Current pagination state of the table view.\n     */\n    boolean isPaginated();\n```\n\n2. Pagination and Filtering works seamlessly: Filter action <---> Paginate action, e.g. You filter for all "Boy" then paginate by 50 items per page OR You paginate by 25 items per page and go to a specific page and then filter all "Sad", etc.\n\n## Change your TableView model \n\nAs of version **0.8.5.1**, TableView has some helper functions to change desired cell item model easily. These are the following:\n\n- To **add a row**:\n```java\n    addRow(int rowPosition, YourRowHeaderModel rowHeaderItem, List<YourCellItemModel> cellItems)\n```\n\n- To **add a multiple rows**:\n```java\n    addRowRange(int rowPositionStart, List<YourRowHeaderModel> rowHeaderItem, List<List<YourCellItemModel>> cellItems)\n```\n\n- To **remove a row**:\n```java\n    removeRow(int rowPosition)\n``` \n\n- To **remove multiple rows**:\n```java\n    removeRowRange(int rowPositionStart, int itemCount)\n``` \n\n- To **update a row header**:\n```java\n    changeRowHeaderItem(int rowPosition, YourRowHeaderModel rowHeaderModel)\n```\n\n- To **update multiple row headers**:\n```java\n    changeRowHeaderItemRange(int rowPositionStart, List<YourRowHeaderModel> rowHeaderModelList)\n``` \n\n- To **update a cell item**:\n```java\n    changeCellItem(int columnPosition, int rowPosition, YourCellItemModel cellModel)\n``` \n\n- To **update a column header**:\n```java\n    changeColumnHeader(int columnPosition, YourColumnHeaderModel columnHeaderModel)\n``` \n\n- To **update multiple column headers**:\n```java\n    changeColumnHeaderRange(int columnPositionStart, List<YourColumnHeaderModel> columnHeaderModelList)\n```\n\n*Note:*[TableViewSample 2 app](https://github.com/evrencoskun/TableViewSample2) also shows usage of these helper methods.\n\n## Hiding & Showing the Row\n\nWith 0.8.5.1 version, hiding and showing any of row is pretty easy for TableView.  For that several helper methods have been inserted on TableView. \n\n- To **show** a row:\n```java\n    showRow(int row)\n```\n\n- To **hide** the row:\n```java\n    hideRow(int row)\n```\n\n- To **show all hidden** rows:\n```java\n    showAllHiddenRows()\n```\n\n- TableView store a map that contains all hidden rows. This method for the time that is necessary to **clear the list of hidden** rows. \n```java\n    clearHiddenRowList()\n```\n\n- To **check the visibility state** of a row:\n```java\n    isRowVisible(int row)\n```\n\n## Hiding & Showing the Column\n\nWith 0.8.5.5 version, hiding and showing any of column is pretty easy for TableView.  For that several helper methods have been inserted on TableView. \n\n- To **show** a column:\n```java\n    showColumn(int column)\n```\n\n- To **hide** a column:\n```java\n    hideColumn(int column)\n```\n\n- To **show all hidden** columns\n```java\n    showAllHiddenColumns()\n```\n\n- TableView store a map that contains all hidden columns. This method for the time that is necessary to **clear the list of hidden columns**.\n```java\n    clearHiddenColumnList()\n```\n\n- To **check the visibility state** of a column:\n```java\n    isColumnVisible(int column)\n```\n## Advanced Usage\n\n- To recalculate the desired column. Sample app shows also its usage as well.\n\n```java\n    remeasureColumnWidth(int column);\n``` \n\n- To ignore column width calculation for better performance, the below line can be used.\n  \n```java\n    tableView.setHasFixedWidth(false);\n``` \n\n- To ignore setting selection colors that are displayed by user interaction, the below line can be used.\n\n```java\n    tableView.setIgnoreSelectionColors(false);\n``` \n\n- To show or hide separators of the TableView, you can simply use these helper methods.\n\n```java\n    setShowHorizontalSeparators(boolean showSeparators)\n```\n\n```java\n    setShowVerticalSeparators(boolean showSeparators)\n```\n\n- There are 2 new helper methods to scroll desired column or row position programmatically.\n\n```java\n    scrollToColumnPosition(int column)\n```\n\n```java\n    scrollToRowPosition(int row)\n```\n\n- With 0.8.8 version, Column width values can be set programatically using the below function. \n \n```java\n    setColumnWidth(int columnPosition, int width)\n```\n\n## Sample Apps\n\n- This repository has an Sample Application of TableView.\n- [TableViewSample 2](https://github.com/evrencoskun/TableViewSample2)\n- (contact me to mention your app on this page)\n\n## Article\n\n- Coming soon.\n\n## Communication\n\n- If you **need help**, please use [Stack Overflow](https://stackoverflow.com/questions/tagged/tableview+android).
