b'<img src="https://mike-schultz.github.io/materialette/assets/banner-sm.png" alt="materialette" description="Material design">\n<p align="center"> <img src="https://mike-schultz.github.io/materialette/assets/app.gif" alt="materialette" description="Material design"></p>\n# Materialette\nEasily access every color in [Google\'s material design palette](https://material.google.com/style/color.html) from your OS dock.\n\nMade using [electron](http://electron.atom.io/).\n\n[![Github All Releases](https://img.shields.io/github/downloads/mike-schultz/materialette/total.svg)]()\n\n## Install\nDownload the latest build from the [releases](https://github.com/mike-schultz/materialette/releases) section. Make sure to choose your operating system appropriately.\n### macOS\n1. Unzip the Materialette.zip.\n2. Move the Materialette.app file to the `/Applications` directory\n\n### Windows 7.1+\n1. Unzip to a desired location and run the exe\n\n### Linux (Some distros)\n1. Unzip to a desired location\n2. open terminal\n3. `cd` to the unzipped directory\n4. run `./Materialette`\n\nNote: There are some known issues with using electron + Tray with Linux.\n\n#### Confirmed:\n* **Ubuntu 16.04**\n  * Requires libappindicator1. Install with `sudo apt-get install libappindicator1`.\n  * Upon opening, a blank button is presented. Clicking it will load Materialette.\n* **Fedora 24**\n  * Requires libappindicator1. Install with `sudo dnf install libappindicator1`.\n\nIf you are able to run the app successfully in a different distro, please update this README!\n\n## Develop\n\nIf you\'re using [nvm](https://github.com/creationix/nvm#nvmrc), run the command `$ nvm use` to use the version of Node specified in the\n[.nvmrc](https://github.com/creationix/nvm#nvmrc).\n\n* Install dependencies with `$ npm install` or `$ yarn`. A yarn.lock is included.\n* Compile SCSS -> CSS `$ gulp sass` or `$ gulp sass:watch`\n* Start the application with `$ npm start `\n* Build the application with  `$ npm run build`'
