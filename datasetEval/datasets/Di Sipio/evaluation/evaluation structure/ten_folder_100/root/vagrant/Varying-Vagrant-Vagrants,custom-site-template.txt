b'# VVV Custom site template\nFor when you just need a simple dev site\n\n## Overview\nThis template will allow you to create a WordPress dev environment using only `vvv-custom.yml`.\n\nThe supported environments are:\n- A single site\n- A subdomain multisite\n- A subdirectory multisite\n\nThe Nginx configuration for this site can be overriden by creating a `vvv-nginx-custom.conf`.\n\n# Configuration\n\n### The minimum required configuration:\n\n```yaml\nmy-site:\n  repo: https://github.com/Varying-Vagrant-Vagrants/custom-site-template\n  hosts:\n    - my-site.test\n```\n| Setting    | Value       |\n|------------|-------------|\n| Domain     | my-site.test |\n| Site Title | my-site.test |\n| DB Name    | my-site     |\n| Site Type  | Single      |\n| WP Version | Latest      |\n\n### Minimal configuration with custom domain and WordPress Nightly:\n\n```yaml\nmy-site:\n  repo: https://github.com/Varying-Vagrant-Vagrants/custom-site-template\n  hosts:\n    - foo.test\n  custom:\n    wp_version: nightly\n```\n| Setting    | Value       |\n|------------|-------------|\n| Domain     | foo.test     |\n| Site Title | foo.test     |\n| DB Name    | my-site     |\n| Site Type  | Single      |\n| WP Version | Nightly     |\n\n### WordPress Multisite with Subdomains:\n\n```yaml\nmy-site:\n  repo: https://github.com/Varying-Vagrant-Vagrants/custom-site-template\n  hosts:\n    - multisite.test\n    - site1.multisite.test\n    - site2.multisite.test\n  custom:\n    wp_type: subdomain\n```\n| Setting    | Value               |\n|------------|---------------------|\n| Domain     | multisite.test      |\n| Site Title | multisite.test      |\n| DB Name    | my-site             |\n| Site Type  | Subdomain Multisite |\n\n### WordPress Multisite with Subdirectory:\n\n```yaml\nmy-site:\n  repo: https://github.com/Varying-Vagrant-Vagrants/custom-site-template\n  hosts:\n    - multisite.test\n  custom:\n    wp_type: subdirectory\n```\n| Setting    | Value                  |\n|------------|------------------------|\n| Domain     | multisite.test         |\n| Site Title | multisite.test         |\n| DB Name    | my-site                |\n| Site Type  | Subdirectory Multisite |\n\n## Configuration Options\n\n```yaml\nhosts:\n    - foo.test\n    - bar.test\n    - baz.test\n```\nDefines the domains and hosts for VVV to listen on. \nThe first domain in this list is your sites primary domain.\n\n```yaml\ncustom:\n    site_title: My Awesome Dev Site\n```\nDefines the site title to be set upon installing WordPress.\n\n```yaml\ncustom:\n    wp_version: 4.6.4\n```\nDefines the WordPress version you wish to install.\nValid values are:\n- nightly\n- latest\n- a version number\n\nOlder versions of WordPress will not run on PHP7, see this page on [how to change PHP version per site](https://varyingvagrantvagrants.org/docs/en-US/adding-a-new-site/changing-php-version/).\n\n```yaml\ncustom:\n    wp_type: single\n```\nDefines the type of install you are creating.\nValid values are:\n- single\n- subdomain\n- subdirectory\n- none\n\n```yaml\ncustom:\n    db_name: super_secet_db_name\n```\nDefines the DB name for the installation.\n\nOther parameters available:\n\n```yaml\ncustom:\n    delete_default_plugins: true # Only on install of WordPress\n    install_test_content: true # Install test content. Only on install of WordPress\n    install_plugins: # Various way to install a plugin\n         - query-monitor\n         - https://github.com/crstauf/query-monitor-extend/archive/version/1.0.zip\n         - https://github.com/norcross/airplane-mode/archive/master.zip\n    install_themes: # Various way to install a theme\n         - understrap\n         - https://github.com/understrap/understrap-child.git\n    wpconfig_constants:\n         WP_DEBUG: true\n         WP_DEBUG_LOG: true\n         WP_DISABLE_FATAL_ERROR_HANDLER: true # To disable in WP 5.2 the FER mode\n    locale: it_IT\n    admin_user: admin # Only on install of WordPress\n    admin_password: password # Only on install of WordPress\n    admin_email: admin@local.test # Only on install of WordPress\n    live_url: http://example.com # Redirect any uploads not found locally to this domain\n```\n'
