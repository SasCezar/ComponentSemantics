b'NixOS boxes for Vagrant\n=======================\n\n[NixOS](http://nixos.org) is a linux distribution based on a purely functional\npackage manager. This project builds [vagrant](http://vagrantup.com) .box\nimages.\n\nStatus\n------\n\nstable\n\nUsage\n-----\n\n```\nvagrant init nixos/nixos-18.09-i686\n# or\nvagrant init nixos/nixos-18.09-x86_64\n```\n\nAlso have a look at the accompanying nixos vagrant plugin:\nhttps://github.com/nix-community/vagrant-nixos-plugin\n\nBuilding the images\n-------------------\n\nFirst install [packer](http://packer.io) and\n[virtualbox](https://www.virtualbox.org/).\n\nThree packer builders are currently supported:\n- Virtualbox\n- qemu / libvirt\n- VMware\n\nHave a look at the different `make build` target to build your image.\n\nIf you build on a host that does not support Makefile, here are some examples:\n```\npacker build --only=virtualbox-iso nixos-i686.json\npacker build --only=qemu nixos-x86_64.json\npacker build --only=vmware-iso nixos-x86_64.json\n```\n\nThe vagrant .box image is now ready to go and you can use it in vagrant:\n\n```\nvagrant box add nixbox32 packer_virtualbox-iso_virtualbox.box\n# or\nvagrant box add nixbox64 packer_virtualbox-iso_virtualbox.box\n```\n\nUpdating the ISO urls\n---------------------\n\nTo update the ISO urls to the latest release run: `make update`\n\nTroubleshooting\n-----------------\n\nIf you build on a Windows OS, please make sure you keep the unix file encoding of the generated configuration files\n(see [issue \\#30](https://github.com/nix-community/nixbox/issues/30)\n\nLicense\n-------\n\nCopyright 2015 under the MIT\n'
