The plugin uses Bootstrap markup and styling by default, but it \ncan be overridden with any other CSS markup. View the [documentation](http://plugins.krajee.com/star-rating) or a [complete demo](http://plugins.krajee.com/star-rating/demo) \nat Krajee JQuery plugins. \n\n> NOTE: Release v4.0 is a modified rewrite with various new enhancements and BC breaking features. It allows rendering richer markup for star symbols and offers theming support.\n\n![Star Rating Screenshot](https://lh3.googleusercontent.com/puCbNL9LlBMty8DmaZxAq0yM8teuhM_hEvox-NuJ2x7xWedNhs8nwSk1Zo8FISFAsyt8=w1366-h768-rw-no)\n\n## Features  \n\n- Convert any HTML input to a star rating control. \n- The plugin automatically converts an input to a star rating control if you set its `class = rating`. \n  All options to the input can be passed as HTML5 `data` attributes.\n- You can use the HTML 5 number input for polyfill and the plugin will automatically use the number attributes like `min`, `max`, and `step`.\n  However, number inputs have a problem with decimal values on the Chrome Browser. Read the Browser Support section in the documentation.\n- Involves pure CSS3 styling of the stars. Say goodbye to image sprites or playing with image backgrounds. Offers clean scalable vector \n  icons for consistent display across devices. Optionally one can use the Unicode character set to override symbols.\n- Use any of your favorite font icon frameworks to render your star symbols (for example you can easily use the icons from the FontAwesome library).\n- Render and display fractional star ratings. Configure number of stars, min, max, step, and theoretically  support any fractional rating.\n- Uses Bootstrap 3.x styles & glyphs by default. But this can be overridden with plugin parameters and your own CSS styles.\n- Support for RIGHT TO LEFT (RTL) input. Automatically changes star styling for RTL.\n- Ability to clear values and options for the stars. Control where the clear button element can be shown.\n- Reset star rating to the initial value when the form is reset.\n- Ability to control and display caption of the selected stars. Each rated star can have its own caption. Control where the \n  caption element can be shown.\n- Ability to size the rating control including the stars, caption, and clear button. Five prebuilt size templates are \n  available `xl`, `lg`, `md`, `sm`, and `xs`.\n- Triggers JQuery events for advanced development. Events currently available are `rating.change`, `rating.clear`, `rating.reset`, `rating.refresh`, `rating.hover`, and `rating.hoverleave`.\n- Disabled and readonly input star rating support.\n- Change stars and caption on mouse hover (new feature since v3.0.0).\n- Change stars and caption on slide and drag for mobile/touch devices (new feature since v3.1.0).\n- Support for translations and locales.\n\n### New features/changes since release v4.0\n\n- **BC Breaking Change**: The `symbol`, `glyphicon`, `ratingClass` properties have been removed. The functionality is replaced with the `theme` property (and can also be complemented or implemented separately using the `containerClass` property).\n- New property `theme` will assign a CSS class with the `rating-<theme-name>` to the rating container.\n    - Themes included\n        - The default (blank) theme (for displaying bootstrap glyphicons)\n        - `krajee-svg` (for displaying svg icons)\n        - `krajee-uni` (for displaying unicode symbols as stars)\n        - `krajee-fa` (for displaying font awesome icons)\n    - Add ability to override and add one\'s own themes\n- Stars now have a better padding and spacing that can be configured via CSS and themes\n- New property `filledStar` - will allow one to set the markup for `filledStar` - will default to \n    - `<i class="glyphicon glyphicon-star"></i>`\n- New property `emptyStar` - will allow one to set the markup for `emptyStar` - will default to \n    - `<i class="glyphicon glyphicon-star-empty"></i>`\n- Exclusive support for SVG (and a prebuilt `krajee-svg` theme that contains two different ready to use SVG icons).\n- Ability to easily set the widget as a "display only" rating via `displayOnly` property.\n- New property `animate` to control animation of highlighted stars on hover or click.\n\n> NOTE: Refer [change log](https://github.com/kartik-v/bootstrap-star-rating/blob/master/CHANGE.md) for details on plugin enhancements, fixes, and changes.\n\n## Documentation and Demo\n\nView the [plugin documentation](http://plugins.krajee.com/star-rating) and [plugin demos](http://plugins.krajee.com/star-rating/demo) at Krajee JQuery plugins. \n\n## Pre-requisites  \n\n1. [Bootstrap 3.x](http://getbootstrap.com/)\n2. Latest [JQuery](http://jquery.com/)\n3. Most browsers supporting CSS3 & JQuery. \n\n## Installation\n\n### Using Bower\nYou can use the `bower` package manager to install. Run:\n\n    bower install bootstrap-star-rating\n\n### Using Composer\nYou can use the `composer` package manager to install. Either run:\n\n    $ php composer.phar require kartik-v/bootstrap-star-rating "@dev"\n\nor add:\n\n    "kartik-v/bootstrap-star-rating": "@dev"\n\nto your composer.json file\n\n> NOTE: You can use the [sass branch](https://github.com/kartik-v/bootstrap-star-rating/tree/sass) for installation using `bootstrap-sass` dependency.\nThe [master branch](https://github.com/kartik-v/bootstrap-star-rating/tree/master) can be used for installation using plain `bootstrap` dependency.\n\n### Manual Install\n\nYou can also manually install the plugin easily to your project. Just download the source [ZIP](https://github.com/kartik-v/bootstrap-star-rating/zipball/master) or [TAR ball](https://github.com/kartik-v/bootstrap-star-rating/tarball/master) and extract the plugin assets (css and js folders) into your project.\n\n## Usage\n\nStep 1: Load the following assets in your header. \n\n```html\n<!-- default styles -->\n<link href="https://maxcdn.bootstrapcdn.com/bootstrap/4.3.1/js/bootstrap.min.js" rel="stylesheet">\n<link href="https://cdnjs.cloudflare.com/ajax/libs/bootstrap-star-rating/4.0.6/css/star-rating.min.css" media="all" rel="stylesheet" type="text/css" />\n\n<!-- optionally if you need to use a theme, then include the theme CSS file as mentioned below -->\n<link href="https://cdnjs.cloudflare.com/ajax/libs/bootstrap-star-rating/4.0.6/themes/krajee-svg/theme.css" media="all" rel="stylesheet" type="text/css" />\n\n<!-- important mandatory libraries -->\n<script src="https://code.jquery.com/jquery-3.3.1.min.js"></script>\n<script src="https://cdnjs.cloudflare.com/ajax/libs/bootstrap-star-rating/4.0.6/js/star-rating.min.js" type="text/javascript"></script>\n\n<!-- optionally if you need to use a theme, then include the theme JS file as mentioned below -->\n<script src="https://cdnjs.cloudflare.com/ajax/libs/bootstrap-star-rating/4.0.6/themes/krajee-svg/theme.js"></script>\n\n<!-- optionally if you need translation for your language then include locale file as mentioned below -->\n<script src="https://cdnjs.cloudflare.com/ajax/libs/bootstrap-star-rating/4.0.6/js/locales/<lang>.js"></script>\n```\n\nIf you noticed, you need to load the `jquery.min.js` and `bootstrap.min.css` in addition to the `star-rating.min.css` and `star-rating.min.js` for\nthe plugin to work with default settings.\n\nStep 2: Setup your input markup to automatically initialize the rating\n```html\n<input id="input-id" type="text" class="rating" data-size="lg" >\n```\n\nStep 3: Initialize the plugin on your page for other input types. For example,\n\n```js\n// initialize with defaults\n$("#input-id").rating();\n\n// with plugin options (do not attach the CSS class "rating" to your input if using this approach)\n$("#input-id").rating({\'size\':\'lg\'});\n```\n\nThe `#input-id` is the identifier for the input on your page (that you used to initialize the rating), and this input is hidden automatically by the plugin (by adding the bootstrap CSS class `hide`). \n\nAlternatively, you can directly call the plugin options by setting data attributes to your input field.\n\n## Contributors\n\n### Code Contributors\n\nThis project exists thanks to all the people who contribute.
