b'# hideShowPassword\n\nBecause life\'s too short to waste time re-typing passwords.\n\nInspired by a pattern seen in [Polar](http://www.polarb.com/), [IE 10+](http://ie.microsoft.com) and [LinkedIn](http://www.linkedin.com/mobile) and [documented by Luke W](http://www.lukew.com/ff/entry.asp?1653), hideShowPassword lets you easily hide and show passwords via JavaScript or a nifty inset toggle.\n\n* [Explanatory blog post](http://blog.cloudfour.com/hide-show-passwords-plugin/)\n* [What\'s new in Version 2](http://blog.cloudfour.com/hideshowpassword-2/)\n* [Live demo](http://cloudfour.github.io/hideShowPassword/)\n\nThe plugin works in any browser that supports resetting the `type` attribute of `<input>` elements (pretty much everything newer than IE8). The plugin should fall back gracefully in cases where this is not supported.\n\n## Installation\n\nInclude the plugin after you\'ve included [jQuery](http://jquery.com/):\n\n```html\n<script src="//code.jquery.com/jquery-2.1.3.min.js"></script>\n<script src="path/to/hideShowPassword.min.js"></script>\n```\n\nIf [Modernizr](http://modernizr.com/) is also included, the plugin\'s touch enhancements will default to the value of `Modernizr.touchevents`.\n\n### Using [npm](https://www.npmjs.com/) and [Browserify](http://browserify.org/)\n\n```\nnpm install --save jquery\nnpm install --save hideshowpassword\n```\n\n```javascript\nvar $ = require(\'jquery\');\nrequire(\'hideshowpassword\');\n```\n\n### Using [Bower](http://bower.io/)\n\n```\nbower install hideshowpassword\n```\n```html\n<script src="//code.jquery.com/jquery-2.1.3.min.js"></script>\n<script src="bower_components/hideShowPassword/hideShowPassword.min.js"></script>\n```\n\n## Usage\n\nThe plugin acts on `<input>` elements (typically password fields):\n\n```html\n<input id="password" type="password">\n```\n\n### Showing, hiding and toggling\n\nYou can quickly show, hide or toggle the visibility of the field\'s contents:\n\n```javascript\n$(\'#password\').showPassword();    // Reveal\n$(\'#password\').hidePassword();    // Hide\n$(\'#password\').togglePassword();  // Toggle\n```\n\nThese are all shorthand versions of the `hideShowPassword` method:\n\n```javascript\n$(\'#password\').hideShowPassword(true);      // Reveal\n$(\'#password\').hideShowPassword(false);     // Hide\n$(\'#password\').hideShowPassword(\'toggle\');  // Toggle\n```\n\n### Enabling the inner toggle button\n\nThe inner toggle functionality tends to steal the show for this plugin. You can pass it along as the second argument of the `hideShowPassword` method:\n\n```javascript\n// Reveal password by default and create an inner toggle\n$(\'#password\').hideShowPassword(true, true);\n```\n\nOr as the first argument of any of the shorthand methods:\n\n``` javascript\n// Reveal password by default and create an inner toggle\n$(\'#password\').showPassword(true);\n```\n\nIf you would like the inner toggle to be hidden until a specific event, you can pass along that event as a string instead:\n\n``` javascript\n// Hide the inner toggle till the #password element is focused\n$(\'#password\').showPassword(\'focus\');\n```\n\n### Specifying more options\n\nAny additional options may be modified by passing along an object as the last argument of any of the aformentioned methods. Here\'s the previous example, but with a custom class for the toggle:\n\n```javascript\n$(\'#password\').hideShowPassword(true, \'focus\', {\n  toggle: {\n    className: \'my-toggle\'\n  }\n});\n```\n\nIn fact, we could pass along all of these arguments as a single object if we want:\n\n```javascript\n$(\'#password\').hideShowPassword({\n  show: true,\n  innerToggle: \'focus\',\n  toggle: {\n    className: \'my-toggle\'\n  }\n});\n```\n\nThere are [many options](#options) available if your project\'s needs are particularly unique.\n\n### Events\n\nIf you need to respond to changes to the password field\'s visibility, you can use any of the following events:\n\n```javascript\n$(\'#password\')\n  .on(\'hideShowPasswordInit\', function(){\n    console.log(\'plugin initialized\');\n  })\n  .on(\'passwordVisibilityChange\', function(){\n    console.log(\'password shown or hidden\');\n  })\n  .on(\'passwordShown\', function(){\n    console.log(\'password shown\');\n  })\n  .on(\'passwordHidden\', function(){\n    console.log(\'password hidden\');\n  });\n```\n\n## Options\n\nHere are all of the available options and their defaults:\n\n```javascript\n.hideShowPassword({\n  // Visibility of the password text. Can be true, false, \'toggle\'\n  // or \'infer\'. If \'toggle\', it will be the opposite of whatever\n  // it currently is. If \'infer\', it will be based on the input\n  // type (false if \'password\', otherwise true).\n  show: \'infer\',\n\n  // Set to true to create an inner toggle for this input. Can\n  // also be sent to an event name to delay visibility of toggle\n  // until that event is triggered on the input element.\n  innerToggle: false,\n\n  // If false, the plugin will be disabled entirely. Set to\n  // the outcome of a test to insure input attributes can be\n  // set after input has been inserted into the DOM.\n  enable: canSetInputAttribute,\n\n  // Event to trigger whenever the element is toggled.\n  // For example, if \'focus\' it will focus the cursor in the\n  // input element after toggling.\n  triggerOnToggle: false,\n\n  // Class to add to input element when the plugin is enabled.\n  className: \'hideShowPassword-field\',\n\n  // Event to trigger when the plugin is initialized and enabled.\n  initEvent: \'hideShowPasswordInit\',\n\n  // Event to trigger whenever the visibility changes.\n  changeEvent: \'passwordVisibilityChange\',\n\n  // Properties to add to the input element.\n  props: {\n    autocapitalize: \'off\',\n    autocomplete: \'off\',\n    autocorrect: \'off\',\n    spellcheck: \'false\'\n  },\n\n  // Options specific to the inner toggle.\n  toggle: {\n    // The element to create.\n    element: \'<button type="button">\',\n    // Class name of element.\n    className: \'hideShowPassword-toggle\',\n    // Whether or not to support touch-specific enhancements.\n    // Defaults to the value of Modernizr.touchevents if available,\n    // otherwise false.\n    touchSupport: (typeof Modernizr === \'undefined\') ? false : Modernizr.touchevents,\n    // Non-touch event to bind to.\n    attachToEvent: \'click.hideShowPassword\',\n    // Event to bind to when touchSupport is true.\n    attachToTouchEvent: \'touchstart.hideShowPassword mousedown.hideShowPassword\',\n    // Key event to bind to if attachToKeyCodes is an array\n    // of at least one keycode.\n    attachToKeyEvent: \'keyup\',\n    // Key codes to bind the toggle event to for accessibility.\n    // If false, this feature is disabled entirely.\n    // If true, the array of key codes will be determined based\n    // on the value of the element option.\n    attachToKeyCodes: true,\n    // Styles to add to the toggle element. Does not include\n    // positioning styles.\n    styles: { position: \'absolute\' },\n    // Styles to add only when touchSupport is true.\n    touchStyles: { pointerEvents: \'none\' },\n    // Where to position the inner toggle relative to the\n    // input element. Can be \'right\', \'left\' or \'infer\'. If\n    // \'infer\', it will be based on the text-direction of the\n    // input element.\n    position: \'infer\',\n    // Where to position the inner toggle on the y-axis\n    // relative to the input element. Can be \'top\', \'bottom\'\n    // or \'middle\'.\n    verticalAlign: \'middle\',\n    // Amount by which to "offset" the toggle from the edge\n    // of the input element.\n    offset: 0,\n    // Attributes to add to the toggle element.\n    attr: {\n      role: \'button\',\n      \'aria-label\': \'Show Password\',\n      title: \'Show Password\',\n      tabIndex: 0\n    }\n  },\n\n  // Options specific to the wrapper element, created\n  // when the innerToggle is initialized to help with\n  // positioning of that element.\n  wrapper: {\n    // The element to create.\n    element: \'<div>\',\n    // Class name of element.\n    className: \'hideShowPassword-wrapper\',\n    // If true, the width of the wrapper will be set\n    // unless it is already the same width as the inner\n    // element. If false, the width will never be set. Any\n    // other value will be used as the width.\n    enforceWidth: true,\n    // Styles to add to the wrapper element. Does not\n    // include inherited styles or width if enforceWidth\n    // is not false.\n    styles: { position: \'relative\' },\n    // Styles to "inherit" from the input element, allowing\n    // the wrapper to avoid disrupting page styles.\n    inheritStyles: [\n      \'display\',\n      \'verticalAlign\',\n      \'marginTop\',\n      \'marginRight\',\n      \'marginBottom\',\n      \'marginLeft\'\n    ],\n    // Styles for the input element when wrapped.\n    innerElementStyles: {\n      marginTop: 0,\n      marginRight: 0,\n      marginBottom: 0,\n      marginLeft: 0\n    }\n  },\n\n  // Options specific to the \'shown\' or \'hidden\'\n  // states of the input element.\n  states: {\n    shown: {\n      className: \'hideShowPassword-shown\',\n      changeEvent: \'passwordShown\',\n      props: { type: \'text\' },\n      toggle: {\n        className: \'hideShowPassword-toggle-hide\',\n        content: \'Hide\',\n        attr: {\n          \'aria-pressed\': \'true\'\n          title: \'Hide Password\',\n        }\n      }\n    },\n    hidden: {\n      className: \'hideShowPassword-hidden\',\n      changeEvent: \'passwordHidden\',\n      props: { type: \'password\' },\n      toggle: {\n        className: \'hideShowPassword-toggle-show\',\n        content: \'Show\',\n        attr: {\n          \'aria-pressed\': \'false\',\n          title: \'Show Password\',\n        }\n      }\n    }\n  }\n\n});\n```\n\n## Known Issues\n\n### Competing controls in IE10+ (Windows 8)\n\nInternet Explorer 10 introduced its own controls for password and text input fields that sometimes compete with the inner toggle functionality of this plugin. Thankfully, they are easily overwritten using CSS:\n\n```css\n::-ms-reveal,\n::-ms-clear {\n  display: none !important;\n}\n```\n\n### Error when debugging IE8 or earlier in IE9+ Developer Tools\n\nFor some reason the plugin returns a false positive when feature-testing unless honest-to-goodness IE8 or earlier is used.\n\n### Toggle quirks in invisible elements\n\nIf you use the inner toggle feature on an invisible element, it may not have enough information to correctly style the wrapper and toggle elements. It\'s recommended that you delay instantiation of the plugin until the elements are visible.\n\nHere\'s a hypothetical example using a [Bootstrap modal](http://getbootstrap.com/javascript/#modals):\n\n```javascript\n$(\'#my-modal\').on(\'shown.bs.modal\', function (event) {\n  $(\'#password\').showPassword(true);\n});\n```\n\n## History\n\n*
