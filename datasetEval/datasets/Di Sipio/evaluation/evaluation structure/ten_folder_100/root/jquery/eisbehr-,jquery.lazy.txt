page loading times and decrease traffic to your users by only loading the content in view. \nYou can use Lazy in all vertical and horizontal scroll ways.\nIt supports images in `<img />` tags and backgrounds, supplied with css like `background-image`, by default.\nOn those elements Lazy can set an default image or a placeholder while loading and supports retina displays as well.\nBut Lazy is even able to load any other content you want by [plugins](#loader-plugins) and [custom loaders](#custom-content-loaders). \n\n\n## Compatibility\nLazy will work with a wide range of browsers and support jQuery versions for years backwards and Zepto as alternative. \nYou can pick any version since jQuery 1.7.2 or Zepto 1.1.6 or greater.\nThere is no way to guarantee, that Lazy will work with all browsers, but all I\'ve tested worked great so far.\nIf you find any problems in specific browsers, [please let me know](http://github.com/eisbehr-/jquery.lazy/issues). \n\n**Tested in:** IE, Chrome (+ mobile), Firefox (+ mobile), Safari (+ mobile) and Android Browser.\n\n\n## Documentation / Examples\nFor [documentation](http://jquery.eisbehr.de/lazy/#parameter), [examples](http://jquery.eisbehr.de/lazy/#examples) and other information take a look on the [project page](http://jquery.eisbehr.de/lazy/).\n\n\n## Installation\nFirst of all, you will need a copy of [jQuery](http://jquery.com) or [Zepto](http://zeptojs.com) to use Lazy successfully on your project.\nIf you get this you can install Lazy by different ways.\nSome examples below:\n\n#### CDN\nLazy and all plugins are available over [cdnjs](http://cdnjs.com) and [jsDelivr](http://jsdelivr.com) CDN and can directly included to every page.\n```HTML\n<!-- jsDeliver -->\n<script type="text/javascript" src="//cdn.jsdelivr.net/gh/eisbehr-/jquery.lazy@1.7.10/jquery.lazy.min.js"></script>\n<script type="text/javascript" src="//cdn.jsdelivr.net/gh/eisbehr-/jquery.lazy@1.7.10/jquery.lazy.plugins.min.js"></script>\n\n<!-- cdnjs -->\n<script type="text/javascript" src="//cdnjs.cloudflare.com/ajax/libs/jquery.lazy/1.7.10/jquery.lazy.min.js"></script>\n<script type="text/javascript" src="//cdnjs.cloudflare.com/ajax/libs/jquery.lazy/1.7.10/jquery.lazy.plugins.min.js"></script>\n```\n\n#### Self-Hosted\n[Download](https://github.com/eisbehr-/jquery.lazy/archive/master.zip) and save one of two available files to include Lazy to your page, either the [development](http://raw.githubusercontent.com/eisbehr-/jquery.lazy/master/jquery.lazy.js) or the [minified](http://raw.githubusercontent.com/eisbehr-/jquery.lazy/master/jquery.lazy.min.js) version.\n```HTML\n<script type="text/javascript" src="jquery.lazy.min.js"></script>\n```\n\n#### Package Managers\nLazy is even available through [NPM](http://npmjs.org) and [Bower](http://bower.io).\nJust use one of the following commands below:\n\n[![NPM](https://nodei.co/npm/jquery-lazy.png?compact=true)](https://nodei.co/npm/jquery-lazy/)\n```sh\n$ npm install jquery-lazy\n$ bower install jquery-lazy\n```\n\n\n## Basic Usage\n1.) The basic usage of Lazy is pretty easy.\nFirst of all you need to prepare all elements you want to lazy load. \nBy default add a `data-src` attribute to images containing the loadable image and/or a `data-loader` attribute to elements witch shall use a [plugin](#loader-plugins) or [custom loaders](#custom-content-loaders).\n```HTML\n<img class="lazy" data-src="path/to/image_to_load.jpg" src="" />\n```\n\n2.) Start using Lazy by calling it after page load.\nYou don\'t have to specify your elements exactly, but for better performance, or different options, load your elements over unique classes or any other selector. \n```JS\n$(function($) {\n    $("img.lazy").Lazy();\n});\n```\nTake a look at the [documentation](http://jquery.eisbehr.de/lazy/) to get an idea what Lazy is capable of.\n\n\n## Callbacks / Events\nLazy comes with a bunch of [callbacks and events](http://jquery.eisbehr.de/lazy/index.php?c=callback) you can assign to.\nJust add them by initialization settings:\n* `beforeLoad` - before item is about to be loaded\n* `afterLoad` - after the item was loaded successfully\n* `onError` - whenever an item could not be loaded\n* `onFinishedAll` - after all items in instance was loaded or returned an error\n\n\n## Instances and public Functions\nLazy supports multiple parallel instances.\nJust initialize them with different selectors.\nTo access an instances public functions you can initialize them in an object oriented manner or grab the instance bind to every element by default:\n```JS\n// object oriented way\nvar instance = $("img.lazy").Lazy({chainable: false});\n\n// grab from elements (only works well if you use same selectors)\n$("img.lazy").Lazy();\nvar instance = $("img.lazy").data("plugin_lazy");\n```\n\nEvery instance has some public available functions to control it\'s behavior.\nThere are currently six available:\n```JS\ninstance.config(entryName[, newValue]); // get or set an configuration entry\ninstance.addItems(items); // add new items to current instance\ninstance.getItems(); // get all unhandled items left of current instance\ninstance.update([useThrottle]); // loads all elements in current viewport\ninstance.force(items); // force loading specific items, ignoring the viewport\ninstance.loadAll(); // loads all remaining available elements from this instance\ninstance.destroy(); // unbinds all events and stop execution directly\n```\n\n\n## Custom Content Loaders\nWith the custom loaders option there is a powerful solution to load every contents the Lazy way.\nLazy will handle everything, you just create a loading method witch got triggered whenever the element hits the visibility threshold.\nIt is still possible to load images and custom loaders in the same Lazy instance.\n\nTo use this just define a loader function inside the Lazy initialisation and pass the loader name to the `data-loader` attribute of the elements witch should be lazy loaded.\n```HTML\n<div class="lazy" data-loader="customLoaderName"></div>\n<img class="lazy" data-src="path/to/image_to_load.jpg" src="" />\n<div class="lazy" data-loader="customLoaderName"></div>\n<div class="lazy" data-loader="asyncLoader"></div>\n```\n```JS\n$(".lazy").Lazy({\n    // callback\n    beforeLoad: function(element) {\n        console.log("start loading " + element.prop("tagName"));\n    },\n\n    // custom loaders\n    customLoaderName: function(element) {\n        element.html("element handled by custom loader");\n        element.load();\n    },\n    asyncLoader: function(element, response) {\n        setTimeout(function() {\n            element.html("element handled by async loader");\n            response(true);\n        }, 1000);\n    }\n});\n```\n\n\n## Loader Plugins\nThe loader plugins can extend the functionality of Lazy, like loading other elements and data.\nIt is basically the same as the [custom content loaders](#custom-content-loaders), with the difference, that plugins can extend all further instances globally at once permanently and let them handle specific elements like `<video>` by default, without `data-loader` attribute set. \nWith custom content loaders you have to initialize each instance on setup with the loader.\nWith plugins you only load the plugin file and you\'re done for all instances from now on. \n\nFor more information and examples, take a look at the [existing plugins](https://github.com/eisbehr-/jquery.lazy/tree/master/plugins) or the [readme.md](https://github.com/eisbehr-/jquery.lazy/tree/master/plugins/README.md) in there.\n\n\n## Configuration Parameters\nThe following configurations is available by default:\n\nName               | Type       | Default            | Description\n------------------ | ---------- | ------------------ | -----------\nname               | *string*   | *\'lazy\'*           | Internal name, used for namespaces and bindings.\nchainable          | *boolean*  | *true*             | By default Lazy is chainable and will return all elements. If set to `false` Lazy will return the created plugin instance itself for further use.\nautoDestroy        | *boolean*  | *true*             | Will automatically destroy the instance when no further elements are available to handle.\nbind               | *string*   | *\'load\'*           | If set to `load`\' Lazy starts working directly after page load. If you want to use Lazy on own events set it to `event`\'.\nthreshold          | *integer*  | *500*              | Amount of pixels below the viewport, in which all images gets loaded before the user sees them.\nvisibleOnly        | *boolean*  | *false*            | Determine if only visible elements should be load.\nappendScroll       | *integer*  | *window*           | An element to listen on for scroll events, useful when images are stored in a container.\nscrollDirection    | *string*   | *\'both\'*           | Determines the handles scroll direction. Possible values are `both`, `vertical` and `horizontal`.\nimageBase          | *string*   | *null*             | If defined this will be used as base path for all images loaded by this instance.\ndefaultImage       | *string*   | *blank image*      | Base64 image string, set as default image source for every image without a predefined source attribute.\nplaceholder        | *string*   | *null*             | Base64 image string, set a background on every element as loading placeholder.\ndelay              | *integer*  | *-1*               | If you want to load all elements at once after page load, then you can specify a delay time in milliseconds.\ncombined           | *boolean*  | *false*            | With this parameter, Lazy will combine the event driven and delayed element loading.\n**attributes**     |            |                    |\nattribute          | *string*   | *\'data-src\'*       | Name of the image tag `src` attribute, where the image path is stored.\nsrcsetAttribute    | *string*   | *\'data-srcset\'*    | Name of the image tag `srcset` attribute, where the source set is stored.\nsizesAttribute     | *string*   | *\'data-sizes\'*     | Name of the image tag `sizes` attribute, where the size definition for source set is stored.\nretinaAttribute    | *string*   | *\'data-retina\'*    | Name of the image tag attribute, where the path for optional retina image is stored.\nloaderAttribute    | *string*   | *\'data-loader\'*    | Name or the element attribute, where the identifier of the plugin or customer loader is sored.\nimageBaseAttribute | *string*   | *\'data-imagebase\'* | Name ot the image tag element, where the specific image base is stored. This will overwrite the global `imageBase` config.\nremoveAttribute    | *boolean*  | *true*             | Determine if the attribute should be removed from the element after loading.\nhandledName        | *string*   | *\'handled\'*        | Name of the element tag data attribute, to determine if element is already handled.\nloadedName         | *string*   | *\'loaded\'*         | Name of the element tag data attribute, to determine if element is already loaded.\n**effect**         |            |                    |\neffect             | *string*   | *\'show\'*           | Function name of the effect you want to use to show the loaded images, like `show` or `fadeIn`.\neffectTime         | *integer*  | *0*                | Time in milliseconds the effect should use to view the image.\n**throttle**       |            |                    |\nenableThrottle     | *boolean*  | *true*             | Throttle down the loading calls on scrolling event.\nthrottle           | *integer*  | *250*              | Time in milliseconds the throttle will use to limit the loading calls.\n**callbacks**      |            |                    |\nbeforeLoad         | *function* | *undefined*        | Callback function, which will be called before the element gets loaded. Has current element and response function as parameters. `this` is the current Lazy instance.\nafterLoad          | *function* | *undefined*        | Callback function, which will be called after the element was loaded. Has current element and response function as parameters. `this` is the current Lazy instance.\nonError            | *function* | *undefined*        | Callback function, which will be called if the element could not be loaded. Has current element and response function as parameters. `this` is the current Lazy instance.\nonFinishedAll      | *function* | *undefined*        | Callback function, which will be called after all elements was loaded or returned an error. The callback has no parameters. `this` is the current Lazy instance.\n\n\n## Build and Validation\nThis project includes an automated build script using `gulp`.\nTo build your own versions of Lazy you need to [install it](#package-managers) via [npm](https://www.npmjs.com/package/jquery-lazy) first.\nAfterwards you can use the following command in your console to automatically generate all productive files.\nWhile building these files everything will be checked with `jshint` for validity too.\n```sh\n$ gulp build\n```\n\n### Available `gulp` Tasks:\n\nName           | Description\n-------------- | -----------\nbuild          | check & build everything\nbuild-main     | check & build main project file\nbuild-plugins  | check & build single plugin files\nconcat-plugins | build concatenated plugins file\nvalidate       | check all files with `jshint`\nwatch          | watches all files to check & build everything on change\n\n\n## Bugs / Feature request\nPlease [report](http://github.com/eisbehr-/jquery.lazy/issues) bugs and feel free to [ask](http://github.com/eisbehr-/jquery.lazy/issues) for new features directly on GitHub.\n\n\n## License\nLazy is dual-licensed under [MIT](http://www.opensource.org/licenses/mit-license.php) and [GPL-2.0](http://www.gnu.org/licenses/gpl-2.0.html) license.\n\n\n## Donation\n_You like to support me?_  \n_You appreciate my work?_  \n_You use it in commercial projects?_  \n  \nFeel free to make a little [donation](https://www.paypal.com/cgi-bin/webscr?cmd=_s-xclick&hosted_button_id=FFL6VQJCUZMXC)! :wink:\n'
