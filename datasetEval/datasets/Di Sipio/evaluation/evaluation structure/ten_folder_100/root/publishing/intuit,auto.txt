labels. Streamline your release workflow and publish constantly! `auto` is meant to be run in a continuous integration (CI) environment, but all the commands work locally as well.\n\nRelease Features:\n\n- Calculate semantic version bumps from PRs\n- Skip a release with the `skip-release` label\n- Publish canary releases from PRs or locally\n- Generate changelogs with fancy headers, authors, and monorepo package association\n- Use labels to create new changelog sections\n- Generate a GitHub release\n\nPull Request Interaction Features:\n\n- Get the labels for a PR\n- Set the status of a PR\n- Check that a pull request has a SemVer label\n- Comment on a PR with markdown\n- Update the PR body with contextual build metadata\n\nVisit [the docs](https://intuit.github.io/auto/) for more information.\n\n## Plugins\n\nAuto has an extensive plugin system and wide variety of official plugins. Make a PR to add yours!\n\n**Package Managers:**\n\n- [chrome](./plugins/chrome) - Publish code to Chrome Web Store\n- [crates](./plugins/crates) - Publish Rust crates\n- [git-tag](./plugins/git-tag) - Manage your projects version through just a git tag (`default` when used with binary)\n- [maven](./plugins/maven) - Publish code with maven\n- [npm](./plugins/npm) - Publish code to npm (`default` when installed through `npm`)\n\n**Extra Functionality:**\n\n- [all-contributors](./plugins/all-contributors) - Automatically add contributors as changelogs are produced using [`all-contributors-cli](https://www.npmjs.com/package/all-contributors-cli)\n- [conventional-commits](./plugins/conventional-commits) - Parse conventional commit messages for version bumps\n- [first-time-contributor](./plugins/first-time-contributor) - Thank first time contributors for their work right in your release notes.\n- [jira](./plugins/jira) - Include Jira story links in the changelog\n- [omit-commits](./plugins/omit-commits) - Ignore commits base on name, email, subject, labels, and username\n- [omit-release-notes](./plugins/omit-release-notes) - Ignore release notes in PRs made by certain accounts\n- [released](./plugins/released) - Add a `released` label to published PRs, comment with the version it\'s included in and comment on the issues the PR closes\n- [s3](./plugins/s3) - Post your built artifacts to amazon s3\n- [slack](./plugins/slack) - Post release notes to slack\n- [twitter](./plugins/twitter) - Post release notes to twitter\n- [upload-assets](./plugins/upload-assets) - Add extra assets to the release\n\n## Start Developing\n\nTo get set up, fork and clone the project then run the following command:\n\n```sh\nyarn\n```\n\n### Build/Typecheck\n\nYou must build at least once before running the tests or lint.\n\n```sh\nyarn build\n```\n\nIn watch mode:\n\n```sh\nyarn start\n```\n\n### Cleaning\n\n```sh\nyarn clean\n```\n\n### Linting\n\n```sh\nyarn lint\n```\n\n### Testing\n\n```sh\nyarn test\n```\n\n### Run the docs\n\n```sh\nyarn docs:watch\n```\n\n### Create a new plugin\n\nGet started developing a new plugin in the monorepo in seconds.\n\nThe two arguments are:\n\n1. A spaced name\n2. A description\n\n```sh\nyarn create:plugin "My Plugin" "Do something really cool"\n```\n\n## Contributing\n\nFeel free to make an [issue](https://github.com/intuit/auto/issues) or open a [pull request](https://github.com/intuit/auto/pulls)!\n\nMake sure to read our [code of conduct](./CODE_OF_CONDUCT.md).\n\n## :rocket: Projects Using `auto` :rocket:\n\n:star: [Storybookjs design-system](https://github.com/storybookjs/design-system) - Storybook\'s official design system\n\n:star: [space-kit](https://github.com/apollographql/space-kit) - Home base for Apollo\'s design system\n\n:star: [react-glider](https://github.com/hipstersmoothie/react-glider) - A react wrapper for glider.js\n\n:star: [Ignite](https://github.com/intuit/Ignite) - Modern markdown documentation generator\n\n:star: [reaction](https://github.com/artsy/reaction) - Artsy\xe2\x80\x99s React Components\n\n:star: [emission](https://github.com/artsy/emission) - Artsy\xe2\x80\x99s React Native Components\n\n:star: [webpack-inject-plugin](https://github.com/adierkens/webpack-inject-plugin) - A webpack plugin to dynamically inject code into the bundle.\n\n:star: [html-webpack-insert-text-plugin](https://github.com/hipstersmoothie/html-webpack-insert-text-plugin) - Insert text into the head or body of your HTML\n\n## :nail_care: `auto` Badge :nail_care:\n\nDoes your project use `auto`?
