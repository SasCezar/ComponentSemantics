b'# About TYPEMILL\r\n\r\nTYPEMILL is a small flat file cms created for editors and writers. It provides a author-friendly dashboard and a visual-block-editor for markdown based on vue.js. Use TYPEMILL for manuals, documentations, web-books and similar publications. The website http://typemill.net itself is an example for TYPEMILL.\r\n\r\n![TYPEMILL Screenshot](https://typemill.net/media/tm-demo.gif)\r\n\r\n## Features\r\n\r\n* Creates a website based on markdown files.\r\n* Provides an author-friendly visual markdown editor (work in progress, based on VUE.js).\r\n* Provides a pure markdown editing mode.\r\n* Markdown supports table of contents (TOC), tables, footnotes, abbreviations and definition lists.\r\n* Create and sort pages with drag & drop in the navigation.\r\n* Configure the system, the themes and the plugins in the dashboard.\r\n* Create and manage users.\r\n* Develop configurable plugins with the Symfony Event Dispatcher.\r\n* Develop configurable themes with TWIG.\r\n* Allows super easy backend and frontend forms with simple YAML-files.\r\n* Ships with a fully responsive standard theme\r\n* Ships with plugins for \r\n  * Search\r\n  * MathJax and KaTeX.\r\n  * Code highlighting.\r\n  * Matomo/Piwik and Google Analytics.\r\n  * Cookie Consent.\r\n\r\n## Requirements\r\n\r\n* PHP 7+\r\n* Apache server\r\n* mod_rewrite and htaccess\r\n\r\nIf you run a linux system, then please double check that mod_rewrite and htaccess are active!!!\r\n\r\n## Installation\r\n\r\nDownload TYPEMILL from the [TYPEMILL website](http://typemill.net), unzip the files and you are done.\r\n\r\nIf you are a developer, you can also clone this repository. To do so, open your command line, go to your project folder (e.g. htdocs) and type:\r\n\r\n    git clone git://github.com/trendschau/typemill.git\r\n\r\nThe GitHub-version has no vendor-folder, so you have to update and include all libraries and dependencies with composer. To do so, open your command line, go to your TYPEMILL folder and type:\r\n\r\n    composer update\r\n\r\nIf you did not use composer before, please go to the [composer website](http://getcomposer.org) and start to learn.\r\n\r\nTo run TYPEMILL on a **live** system, simply upload the files to your server\r\n\r\n## Make Folders Writable.\r\n\r\nMake sure that the following folders and all their files are writable (permission 774 recursively):\r\n\r\n* cache\r\n* content\r\n* media\r\n* settings\r\n\r\nYou can use your ftp-software for that.\r\n\r\n## Setup\r\n\r\nPlease go to `your-typemill-website.com/setup`, create an initial user and configure your system in the author panel. \r\n\r\n## Login\r\n\r\nYou can find your login screen under `/tm/login` or simply go to `/setup` and you will be redirected to the login-page. \r\n\r\n## Documentation\r\n\r\nYou can read the full documentation for writers, for theme developers and for plugin developers on the [TYPEMILL website](http://typemill.net).\r\n\r\n## Licence\r\n\r\nTYPEMILL is published under MIT licence. Please check the licence of the included libraries, too.\r\n\r\n## Contributors & Supporters\r\n\r\n* [Eziquel Bruni](https://github.com/EzequielBruni) edits the typemill documentation.\r\n* [Ricky](https://github.com/rbertram90) developed the discard functionality.\r\n* [vodaris](https://www.vodaris.de) sponsored the development of the search plugin.\r\n\r\n## How to Contribute\r\n\r\nTypemill is still in an early stage and contributions are highly welcome. \r\n\r\nYou can check the [roadmap for Typemill](https://github.com/typemill/typemill/issues/35) and scroll through the issues.
