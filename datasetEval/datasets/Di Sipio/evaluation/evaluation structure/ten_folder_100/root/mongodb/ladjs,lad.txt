configurable.\n\n### Microservices\n\nThese microservices are preconfigured for security, performance, and graceful reloading.\n\n* Webapp server \xe2\x86\x92 [web.js](template/web.js)\n* API server \xe2\x86\x92 [api.js](template/api.js)\n* Job scheduler \xe2\x86\x92 [bull.js](template/bull.js)\n* Proxy server \xe2\x86\x92 [proxy.js](template/proxy.js)\n\n### Front-end\n\n* Browser linting using [eslint-plugin-compat][] and [browserslist][] (see [.browserslistrc](template/.browserslistrc) for the default config)\n* [Pug][] template engine (you can easily use [Moon][], [Vue][], [React][], or [Angular][], though typically [you aren\'t going to need it][yagni])\n* [Gulp][] (latest version 4.x)\n* [Sass][]\n* [PostCSS][] (with [font-magician][], [import-url][], [font-grabber][], [base64][], and [cssnext][] pre-configured)\n* [Bootstrap][]\n* [Font Awesome][font-awesome]\n* [SpinKit][]\n* [SweetAlert2][]\n* [Dense][]\n* [Waypoints][]\n* [LiveReload][]\n* [Frisbee][]\n* \xe2\x80\xa6\n\n### Back-end\n\n* Redis, sessions, and flash toast and modal [SweetAlert2][] messages (uses [ioredis][] which has support for [Cluster][redis-cluster], [Sentinel][redis-sentinel], and more)\n* Koa-based webapp and API servers (uses HTTP/2 for production!)\n* Pagination built-in (using [ctx-paginate][])\n* RESTful API with BasicAuth and versioning\n* Automated job scheduler with cron and human-readable syntax (backed by [Mongoose][] and [Bull][])\n* Passport-based authentication and group-based (Unix-like) permissioning\n* Stripe-inspired error handling\n* Mongoose and MongoDB with common database plugins\n* Email template engine with [Nodemailer][] and local rendering\n* Proxy eliminates need for Nginx reverse-proxy or Apache virtual hosts\n* Multilingual through built-in i18n translation support ([see configuration](#translation-configuration))\n* Automatic phrase translation with Google Translate\n* Sitemap generator for simple SEO\n* \xe2\x80\xa6\n\n### Translation\n\nFinally a framework that solves i18n everywhere; complete with automatic translation.\n\n* Translation constants built-in so you [don\'t repeat yourself][dry]\n* Webapp error messages and templates are translated\n* Emails are translated\n* API responses are translated\n* Database errors are translated\n* Authentication errors are translated\n* \xe2\x80\xa6\n\n### Email Engine\n\nOur beautiful email engine uses [email-templates][] (which is also made by the creator of Lad)!\n\n* Test your emails locally with automatic browser-rendering on the fly\n* Automatically inlines CSS for cross-browser and cross-platform email client support\n* Use [Bootstrap][] in your email template designs\n* Reuse your existing CSS and webapp styling\n* Use any template engine (defaults to Pug)\n* [Render custom fonts in emails with code][custom-fonts-in-emails]\n* [Add icons with Font Awesome with code][font-awesome-assets]\n* [Automatically avoid email client caching][nodemailer-base64-to-s3]\n* Include any image you want and it will be properly rendered\n* Rids the need for awkward embedded image CID attachments\n* \xe2\x80\xa6\n\n### Error This will display to you all the scripts you can run.\n\nThe `start` script (among many others) uses [nps][] and [nps-utils][] under the hood.  This helps to keep scripts very developer-friendly, and rids the need to write in JSON syntax.\n\nThis script accepts a `<script>` argument, whereas a script of `all` will spawn, watch, and re-compile all of the [microservices](#microservices) mentioned above.\n\nIt will also open the browser for you to <http://localhost:3000> automatically for testing the web front-end.\n\n[npm][]:\n\n```sh\nnpm start all\n```\n\n[yarn][]:\n\n```sh\nyarn start all\n```\n\n##### Debugging\n\n* `DEBUG` - debug using [debug][] output (widely adopted package in the community for debugging across all Node packages):\n\n  ```sh\n  DEBUG=* ...\n  ```\n\n* `NODE_DEBUG` - debug [node][] internal modules:\n\n  ```sh\n  NODE_DEBUG=* ...\n  ```\n\n* `MONGOOSE_DEBUG` - debug Mongoose raw database operation output:\n\n  ```sh\n  MONGOOSE_DEBUG=true ...\n  ```\n\n* `TRANSPORT_DEBUG` - debug Nodemailer transport:\n\n  ```sh\n  TRANSPORT_DEBUG=true ...\n  ```\n\n* `REDIS_MONITOR` - debug Redis using `MONITOR` (uses [@ladjs/redis][ladjs-redis] and passes `true` for the `monitor` argument):\n\n  ```sh\n  REDIS_MONITOR=true ...\n  ```\n\n* `REDIS_FRIENDLY_ERROR_STACK` - debug Redis with friendly error stack messages (see [showFriendlyErrorStack][show-friendly-error-stack] option of [ioredis][])\n\n  ```sh\n  REDIS_FRIENDLY_ERROR_STACK=true ...\n  ```\n\n#### Production\n\nWe strongly recommend using [SemaphoreCI][], [PM2][], and [Digital Ocean][do] for production deployment.\n\n1. Contents of the file at `config/index.js` (reads in `process.env` environment variables)\n2. Contents of the files in directories under `config/environments/` (sets defaults per environment, e.g. you can pass `NODE_ENV=staging` and it will load the file at `config/environments/staging.js`)\n3. Environment variables used to override defaults or set required ones (e.g. `NODE_ENV=production`)\n4. Environment configuration in `.env`\n5. Environment configuration in `.env.defaults`\n\nPrecedence is taken by the environment configuration files, environment variables, then the `.env` file.\n\nBasically [dotenv][] won\'t set an environment variable if it already detects it was passed as an environment variable.\n\nTake a look at the [config](template/config) folder contents and also at the defaults at [.env.defaults](template/.env.defaults).\n\n#### SSL Configuration\n\nTo configure SSL for the web or API server simply set them in your `.env` file or pass them as environment variables.\n\n> Web server:\n\n* `WEB_PROTOCOL` - you must set this to `https`\n* `WEB_SSL_KEY_PATH` - file path to your SSL key file (e.g. `/home/deploy/.ssl/web-key.pem`)\n* `WEB_SSL_CERT_PATH` - file path to your SSL certificate file (e.g. `/home/deploy/.ssl/web-cert.pem`)\n* `WEB_SSL_CA_PATH` (optional) - file path to your SSL certificate authority file (e.g. `/home/deploy/.ssl/web-ca-cert.pem`)\n\n> API server:\n\n* `API_PROTOCOL` - you must set this to `https`\n* `API_SSL_KEY_PATH` - file path to your SSL key file (e.g. `/home/deploy/.ssl/api-key.pem`)\n* `API_SSL_CERT_PATH` - file path to your SSL certificate file (e.g. `/home/deploy/.ssl/api-cert.pem`)\n* `API_SSL_CA_PATH` (optional) - file path to your SSL certificate authority file (e.g. `/home/deploy/.ssl/api-ca-cert.pem`)\n\n#### Outbound Email Configuration\n\nBy default in the development environment we simply render the email in your browser.\n\nHowever in other environments such as production, you definitely want emails to be sent.\n\nWe built-in support for Postmark by default (though you can swap in your own `transport` provider in the `jobs/email.js` file):\n\n1.
