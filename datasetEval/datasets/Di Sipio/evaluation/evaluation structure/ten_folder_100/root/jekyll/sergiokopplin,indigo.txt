Fork the project [Indigo](https://github.com/sergiokopplin/indigo/fork)\n3. Edit `_config.yml` with your data (check <a href="README.md#settings">settings</a> section)\n4. Write some posts :bowtie:\n\nIf you want to test locally on your machine, do the following steps also:\n\n1. Install [Jekyll](http://jekyllrb.com), [NodeJS](https://nodejs.org/) and [Bundler](http://bundler.io/).\n2. Clone the forked repo on your machine\n3. Enter the cloned folder via terminal and run `bundle install`\n4. Then run `bundle exec jekyll serve --config _config.yml,_config-dev.yml`\n5. Open it in your browser: `http://localhost:4000`\n6. Test your app with `bundle exec htmlproofer ./_site`\n7. Do you want to use the [jekyll-admin](https://jekyll.github.io/jekyll-admin/) plugin to edit your posts? Go to the admin panel: `http://localhost:4000/admin`. The admin panel will not work on GitHub Pages, [only locally](https://github.com/jekyll/jekyll-admin/issues/341#issuecomment-292739469).\n\n## Settings\n\nYou must fill some informations on `_config.yml` to customize your site.\n\n```\nname: John Doe\nbio: \'A Man who travels the world eating noodles\'\npicture: \'assets/images/profile.jpg\'\n...\n\nand lot of other options, like width, projects, pages, read-time, tags, related posts, animations, multiple-authors, etc.\n```\n\n## How To?\n\nCheck the [FAQ](./FAQ.md) if you have any doubt or problem.\n\n---\n\n[MIT](http://kopplin.mit-license.org/) License \xc2\xa9 S\xc3\xa9rgio Kopplin\n'
