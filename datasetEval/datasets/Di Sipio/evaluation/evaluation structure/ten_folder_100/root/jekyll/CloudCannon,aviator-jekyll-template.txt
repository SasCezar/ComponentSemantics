b"# Aviator\n\nAPI documentation template for Jekyll. Browse through a [live demo](https://tangerine-lemon.cloudvent.net/).\nStart documenting your API with this configurable theme.\n\n![Aviator template screenshot](images/_screenshot.png)\n\nAviator was made by [CloudCannon](http://cloudcannon.com/), the Cloud CMS for Jekyll.\n\nFind more templates, themes and step-by-step Jekyll tutorials at [CloudCannon Academy](https://learn.cloudcannon.com/).\n\n## Features\n\n* Three column layout\n* Fully responsive\n* Full text search\n* Pre-styled components\n* Auto-generated navigation based on category\n* Optimised for editing in [CloudCannon](http://cloudcannon.com/)\n* SEO tags\n* Google Analytics\n\n## Setup\n\n1. Add your site and author details in `_config.yml`.\n2. Get a workflow going to see your site's output (with [CloudCannon](https://app.cloudcannon.com/) or Jekyll locally).\n\n## Develop\n\nAviator was built with [Jekyll](http://jekyllrb.com/) version 3.7.2, but should support newer versions as well.\n\nInstall the dependencies with [Bundler](http://bundler.io/):\n\n~~~bash\n$ bundle install\n~~~\n\nRun `jekyll` commands through Bundler to ensure you're using the right versions:\n\n~~~bash\n$ bundle exec jekyll serve\n~~~\n\n## Editing\n\nAviator is already optimised for adding, updating and removing documentation pages in CloudCannon.\n\n### Usage\n\n* Each section is a different collection, this helps organise your content.\n* Set the order of the collections with the position_number field in collection configuration in `_config.yml`.\n* Set the order of the documents inside a collection by setting the position_number in front matter.\n\n### Search\n\n* Add `excluded_in_search: true` to any documentation page's front matter to exclude that page in the search results.\n"
