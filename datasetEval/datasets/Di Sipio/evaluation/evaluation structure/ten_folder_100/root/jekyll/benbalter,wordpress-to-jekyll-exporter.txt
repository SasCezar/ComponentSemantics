b'# WordPress to Jekyll Exporter\n\nOne-click WordPress plugin that converts all posts, pages, taxonomies, metadata, and settings to Markdown and YAML which can be dropped into Jekyll.\n\n[![Build Status](https://travis-ci.org/benbalter/wordpress-to-jekyll-exporter.svg?branch=master)](https://travis-ci.org/benbalter/wordpress-to-jekyll-exporter) [![PRs Welcome](https://img.shields.io/badge/PRs-welcome-brightgreen.svg?style=flat-square)](http://makeapullrequest.com)\n\nView plugin in [the WordPress plugin directory](https://wordpress.org/plugins/jekyll-exporter/).\n\n## Features\n\n* Converts all posts, pages, and settings from WordPress for use in Jekyll\n* Export what your users see, not what the database stores (runs post content through `the_content` filter prior to export, allowing third-party plugins to modify the output)\n* Converts all `post_content` to Markdown Extra (using Markdownify)\n* Converts all `post_meta` and fields within the `wp_posts` table to YAML front matter for parsing by Jekyll\n* Generates a `_config.yml` with all settings in the `wp_options` table\n* Outputs a single zip file with `_config.yml`, pages, and `_posts` folder containing `.md` files for each post in the proper Jekyll naming convention\n* No settings. Just a single click.\n\n## Usage\n\n1. Place plugin in `/wp-content/plugins/` folder\n2. Activate plugin in WordPress dashboard\n3. Select `Export to Jekyll` from the `Tools` menu\n\n## More information\n\nSee [the full documentation](https://ben.balter.com/wordpress-to-jekyll-exporter):\n\n* [Changelog](/docs/changelog.md)\n* [Command-line-usage](/docs/command-line-usage.md)\n* [Custom post types](/docs/custom-post-types.md)\n* [Developing locally](/docs/developing-locally.md)\n* [Minimum required PHP version](/docs/required-php-version.md)\n'
