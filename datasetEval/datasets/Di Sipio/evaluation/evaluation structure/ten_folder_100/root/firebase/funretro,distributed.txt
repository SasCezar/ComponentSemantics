b'# Fun Retro\n\nEasy to use and beautiful restrospective board.\n\n[![Build](https://travis-ci.org/funretro/distributed.svg?branch=master)](https://travis-ci.org/funretro/distributed)\n<a href="https://codeclimate.com/github/funretro/distributed"><img src="https://codeclimate.com/github/glauberramos/fireideaz/badges/gpa.svg" /></a>&nbsp;[![devDependency Status](https://david-dm.org/funretro/distributed/dev-status.svg)](https://david-dm.org/funretro/distributed#info=devDependencies)\n\n## How to setup local environment\n\n1. Clone the repository\n2. Fun Retro is using [Firebase](http://www.firebase.com). First create an account at firebase. Then create a test project.\n3. Enable sign-up using Email/Password for your project. This can be done\n   through the \'Authentication\' tab for the project.\n4. Change the file on js/vendor/firebaseInitialization.js and replace the values with your project values\n\n```\nvar config = {\n  apiKey: "YOUR_API_KEY",\n  authDomain: "YOUR_PROJECT_ID.firebaseapp.com",\n  databaseURL: "https://YOUR_PROJECT_ID.firebaseio.com",\n  projectId: "YOUR_PROJECT_ID",\n  storageBucket: "YOUR_PROJECT_ID.appspot.com",\n  messagingSenderId: "YOUR_MESSAGE_ID"\n};\n\nfirebase.initializeApp(config);\n```\n\n5. The app uses firebase auth and creates a new user when you attempt to create a new board. So, once the app is running, head over to your firebase console `Authentication` section and enable auth using email & password. After change it should look like this [screenshot](https://drive.google.com/file/d/1u5qzuXeyERstqK0i6zR02uyP8ZdLK8ob/view?usp=sharing)\n6. If you don\'t want to file to be tracked by git just run this command on the project home folder:\n\n```\ngit update-index --assume-unchanged js/vendor/firebaseInitialization.js\n```\n\n7. Install [Node.js](https://nodejs.org/en/)\n8. Run `npm install` (inside the project folder)\n9. Install [Gulp](http://gulpjs.com/)\n10. Run `gulp`\n11. Now you can open the app on [http://localhost:4000/](http://localhost:4000/) and start creating your boards.\n\n## Contribute to Fun Retro\n\nTake a look on our [Contributing](https://github.com/funretro/distributed/blob/master/CONTRIBUTING.md) guide\n\n## Running the application\n\n1. Install [Node.js](https://nodejs.org/en/)\n2. Run `npm install` (inside the project folder)\n3. Run `gulp`\n4. Point your browser to [http://localhost:4000/](http://localhost:4000/) or open dist/index.html\n\n## Continuous Integration\n\nOur tests run on [TravisCI](https://travis-ci.org/funretro/distributed)\n\n## Running the tests\n\n1. Run `gulp test`\n2. Refactor and watch tests re-run.\n3. Fix any red tests\n4. Note: you can open [http://localhost:9876/debug.html](http://localhost:9876/debug.html) to run tests with a browser\n'
