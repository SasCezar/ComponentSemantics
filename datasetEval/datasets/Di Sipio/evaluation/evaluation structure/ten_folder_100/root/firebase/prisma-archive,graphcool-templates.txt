b'# Graphcool Templates \xf0\x9f\x93\x97\n\n**Use Graphcool [templates](https://www.graph.cool/docs/reference/service-definition/templates-zeiv8phail/) to quickly pull in predefined functionality into your own Graphcool service.**\n\n## Overview\n\n###  Officially supported templates\n\nWe maintain a number of officially supported templates:\n\n- [auth](./auth): Templates for different authentication mechanisms, such as [email-password](./auth/email-password) or [facebook](./auth/facebook).\n- [messaging](./messaging): Templates to quickly integrate with different messaging services, such as [mailgun](./messaging/mailgun) (emails), [pusher](./messaging/pusher) (push notifications) or [twilio](./messaging/twilio) (SMS).\n\n### Outdated templates\n\nNotice that the [outdated](./outdated) templates are _not_ officially supported. Right now, most of the templates inside the [outdated](./outdated) directory are not working when used with the [`graphcool-framework add-template`](https://www.graph.cool/docs/reference/graphcool-cli/commands-aiteerae6l/#graphcool-add-template) command!\n\n### Contributions\n\nContributions are _highly_ welcome :green_heart: if you\'d like to add a new template, you can get inspiration from the [help wanted label](https://github.com/graphcool/templates/labels/help%20wanted), or suggest your own template!\n\n## Adding templates to your service\n\nThere are two ways how you can use a template in your service:\n\n1. Manually download and copy the code from a template directory in this repository. \n2. Use the [`add-template`](https://www.graph.cool/docs/reference/graphcool-cli/commands-aiteerae6l/#graphcool-add-template) command from the [CLI](!alias-zboghez5go).\n\n### Option 1: Manually adding templates\n\nThe process of adding a template to a Graphcool service involves several steps. You\'re basically _merging_ your local service definition with the service definition of the template you want to use. In the end, you end up with only _one_ `graphcool.yml` as well as only _one_ `types.graphql`.\n\nAdding a template manually:\n\n1. Download the folder that contains the service definition of the template you want to use.\n2. Copy that folder into the root directory of your Graphcool service.\n3. Copy over the contents from the template\'s `graphcool.yml` into the `graphcool.yml` of your own service. Be sure to adjust any file references, e.g. source files that contain code for [functions](https://www.graph.cool/docs/reference/functions/overview-aiw4aimie9/), if necessary.\n4. Copy over the contents from the template\'s `types.graphql` into the `types.graphql` of your own service. \n5. Deploy your changes with `graphcool deploy`.\n\n### Option 2: Adding templates with the CLI\n\nThe `add-template` command in the Graphcool CLI basically automates the process of [manually adding templates](#option-1-manually-adding-templates). \n\nOfficial templates can be added by providing their relative path in this repository, e.g. for the [`email-password`](https://github.com/graphcool/modules)-template:\n\n```sh\ngraphcool-framework add-template auth/email-password\n```\n\nTo install your own templates, you can provide the path to a GitHub repository, too: \n\n```sh\ngraphcool-framework add-template graphcool/templates/auth/email-password\n```\n\nWhen merging the template\'s `graphcool.yml` and `types.graphql` files with the ones from your local service definition, the CLI will only add the contents from the template files into your local files _as comments_. So **you need to explicitly uncomment the parts form the template files that you actually want to use in your service**.\n\nThe process for using the CLI to add a template thus looks as follows:\n\n1. Use the `add-template <path>` CLI command and specify the `<path>` which points to the template\'s directory in this repository.\n2. Uncomment the lines in `graphcool.yml` and `types.graphql`.\n3. Deploy your changes with `graphcool deploy`.\n\n## Contributors\n\nA big thank you to all contributors and supporters of this repository \xf0\x9f\x92\x9a\n\n<a href="https://github.com/marktani/"
