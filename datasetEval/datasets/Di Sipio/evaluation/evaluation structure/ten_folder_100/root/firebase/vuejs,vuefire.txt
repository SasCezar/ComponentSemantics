In order to keep all\nclients data in-sync with its cloud database, their [js\nSDK](https://firebase.google.com/docs/firestore/quickstart) provides the tools\nto do so. However, it quickly becomes bothersome to _bind_ multiple documents or\ncollections to your application, keep them synchronized as well as handling\n[references](https://firebase.google.com/docs/firestore/manage-data/data-types)\nto other documents or collections, which can contain references themselves and\nmust also be kept up to date. The goal of vuefire and vuexfire is to make this\nas simple as a function call that returns a promise so it is also easy to setup\nSSR and allows you to focus on developing your application.\n\nTo better understand why Vuefire will make it so much easier to develop Vue apps\nwith firebase, please, check [this link in the\ndocumentation](https://vuefire.vuejs.org/vuefire/#why)\n\nNote: _This repository contains the code for `vuefire` and `vuexfire` packages:_\n\n## Packages\n\n| Package                        | version
