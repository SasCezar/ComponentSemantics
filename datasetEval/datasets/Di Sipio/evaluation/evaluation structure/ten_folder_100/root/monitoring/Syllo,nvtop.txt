b'NVTOP\n=====\n\nWhat is NVTOP?\n--------------\n\nNvtop stands for NVidia TOP, a (h)top like task monitor for NVIDIA GPUs. It can\nhandle multiple GPUs and print information about them in a htop familiar way.\n\nBecause a picture is worth a thousand words:\n\n![NVTOP interface](/screenshot/NVTOP_ex1.png)\n\nTable of Contents\n-----------------\n\n1. [Fedora / RedHat / CentOS](#fedora--redhat--centos)\n   1. [OpenSUSE](#opensuse)\n   1. [Arch Linux](#arch-linux)\n   1. [NVTOP Build](#nvtop-build)\n1. [Troubleshoot](#troubleshoot)\n1. [License](#license)\n\nNVTOP Options and Interactive Commands\n--------------------------------------\n\nNVTOP comes with a manpage!\n```bash\nman nvtop\n```\nFor quick command line arguments help\n```bash\nnvtop -h\nnvtop --help\n```\n\nGPU Support\n-----------\n\nThe *NVML library* does not support some of the queries for GPUs coming before the\nKepler microarchitecture. Anything starting at GeForce 600, GeForce 800M and\nsuccessor should work fine. For more information about supported GPUs please\ntake a look at the [NVML documentation](http://docs.nvidia.com/deploy/nvml-api/nvml-api-reference.html#nvml-api-reference).\n\nBuild\n-----\n\nTwo libraries are required:\n\n* The *NVIDIA Management Library* (*NVML*) which comes with the GPU driver.\n  * This queries the GPU for information.\n* The *ncurses* library driving the user interface.\n  * This makes the screen look beautiful.\n\n## Distribution Specific Installation Process\n\n### Ubuntu / Debian\n\n#### Ubuntu disco (19.04) / Debian buster (stable)\n\n- ```bash\n  sudo apt install nvtop\n  ```\n\n#### Older\n\n- NVIDIA drivers (see [Ubuntu Wiki](https://help.ubuntu.com/community/BinaryDriverHowto/Nvidia) or [Ubuntu PPA](https://launchpad.net/~graphics-drivers/+archive/ubuntu/ppa) or [Debian Wiki](https://wiki.debian.org/NvidiaGraphicsDrivers#NVIDIA_Proprietary_Driver))\n- CMake, ncurses and git\n  ```bash\n  sudo apt install cmake libncurses5-dev libncursesw5-dev git\n  ```\n- NVTOP\n  - Follow the [NVTOP Build](#nvtop-build)\n\n### Fedora / RedHat / CentOS\n\n- NVIDIA drivers, **CUDA required for nvml libraries** (see [RPM Fusion](https://rpmfusion.org/Howto/NVIDIA))\n- CMake, ncurses and git\n  ```bash\n  sudo dnf install cmake ncurses-devel git\n  ```\n- NVTOP\n  - Follow the [NVTOP Build](#nvtop-build)\n\n### OpenSUSE\n\n- NVIDIA drivers (see [SUSE Support Database](https://en.opensuse.org/SDB:NVIDIA_drivers))\n- CMake, ncurses and git\n  ```bash\n  sudo zypper install cmake ncurses-devel git\n  ```\n- NVTOP\n  - Follow the [NVTOP Build](#nvtop-build)\n\n### Arch Linux\n\n- NVIDIA drivers (see [Arch Linux wiki](https://wiki.archlinux.org/index.php/NVIDIA))\n- CMake, ncurses and git\n  ```bash\n  sudo pacman -S cmake ncurses git\n  ```\n- NVTOP\n  - The `nvtop` AUR package\n  - Follow the [NVTOP Build](#nvtop-build)\n\n## NVTOP Build\n\n```bash\ngit clone https://github.com/Syllo/nvtop.git\nmkdir -p nvtop/build && cd nvtop/build\ncmake ..\n\n# If it errors with "Could NOT find NVML (missing: NVML_INCLUDE_DIRS)"\n# try the following command instead, otherwise skip to the build with make.\ncmake .. -DNVML_RETRIEVE_HEADER_ONLINE=True\n\nmake\nmake install # You may need sufficient permission for that (root)\n```\n\nIf you use **conda** as environment manager and encounter an error while building nvtop, try `conda deactivate` before invoking `cmake`.\n\nThe build system supports multiple build type (e.g. -DCMAKE_BUILD_TYPE=RelWithDebInfo):\n\n* Release: Binary without debug information\n* RelWithDebInfo: Binary with debug information\n* Debug: Compile with warning flags and address/undefined sanitizers enabled (for development purposes)\n\nTroubleshoot\n------------\n\n- The plot looks bad:\n  - Verify that you installed the wide character version of the NCurses library (libncurses**w**5-dev for Debian / Ubuntu), clean the build directory\n    and restart the build process.\n- **Putty**: Tell putty not to lie about its capabilities (`$TERM`) by setting the field ``Terminal-type string`` to ``putty`` in the menu\n  ``Connection > Data > Terminal Details``.\n\nLicense\n-------\n\nNvtop is licensed under the GPLV3 license or any later version.\nYou will find a copy of the license inside the COPYING file of the repository or\nat the gnu website <[www.gnu.org/licenses/](http://www.gnu.org/licenses/)>.\n'
