*Netcap* (NETwork CAPture) framework efficiently converts a stream of network packets into platform neutral type-safe structured audit records that represent specific protocols or custom abstractions.\nThese audit records can be stored on disk or exchanged over the network,\nand are well suited as a data source for machine learning algorithms.\nSince parsing of untrusted input can be dangerous and network data is potentially malicious,\nimplementation was performed in a programming language that provides a garbage collected memory safe runtime.\n\nIt was developed for a series of experiments in my bachelor thesis: _Implementation and evaluation of secure and scalable anomaly-based network intrusion detection_.\nSlides from my presentation at the Leibniz Supercomputing Centre of the Bavarian Academy of Sciences and Humanities are available on [researchgate](https://www.researchgate.net/project/Anomaly-based-Network-Security-Monitoring).\n\nThe project won the 2nd Place at Kaspersky Labs SecurIT Cup 2018 in Budapest.\n\n*Netcap* uses Google\'s Protocol Buffers to encode its output, which allows accessing it across a wide range of programming languages.\nAlternatively, output can be emitted as comma separated values, which is a common input format for data analysis tools and systems.\nThe tool is extensible and provides multiple ways of adding support for new protocols, \nwhile implementing the parsing logic in a memory safe way.\nIt provides high dimensional data about observed traffic and allows the researcher to focus on experimenting with novel approaches for detecting malicious behavior in network environments,\ninstead of fiddling with data collection mechanisms and post processing steps.\nIt has a concurrent design that makes use of multi-core architectures.\nThe name *Netcap* was chosen to be simple and descriptive.\nThe command-line tool was designed with usability and readability in mind,\nand displays progress when processing packets.\nThe latest version offers 58 audit record types of which 53 are protocol specific and 5 are flow models.\n\nFor more details please refer to the [Documentation](https://docs.netcap.io), visit the [Homepage](https://netcap.io) and read the [thesis](https://github.com/dreadl0ck/netcap/blob/master/mied18.pdf).\n\nA simple demonstration of generating audit records from a PCAP dump file,\nquerying and displaying the collected information in various ways:\n\n[![asciicast](https://asciinema.org/a/217939.svg)](https://asciinema.org/a/217939)\n\nAnd live operation decoding traffic from my wireless network interface, while I am surfing the web:\n\n[![asciicast](https://asciinema.org/a/217941.svg)](https://asciinema.org/a/217941)\n\n## Design Goals\n\n- memory safety when parsing untrusted input\n- ease of extension\n- output format interoperable with many different programming languages\n- concurrent design\n- output with small storage footprint on disk\n- maximum data availability\n- allow implementation of custom abstractions\n- rich platform and architecture support\n\n## Use Cases\n\n- monitoring honeypots\n- monitoring medical / industrial devices\n- research on anomaly-based detection mechanisms\n- Forensic data analysis\n\n## License\n\nNetcap is licensed under the GNU General Public License v3, which is a very permissive open source license, that allows
