an existing [colorscheme](https://github.com/cointop-sh/colors/blob/master/cointop.toml), rename it, and customize the colors.\n\n## Config\n\nThe first time you run cointop, it\'ll create a config from?\n\n  - A: By default, the data is from [CoinGecko](https://www.coingecko.com/). Data from [CoinMarketCap](https://coinmarketcap.com/) is another option.\n\n- Q: What APIs does it support?\n\n  - A: APIs currently supported are [CoinMarketCap](https://coinmarketcap.com/) and [CoinGecko](https://www.coingecko.com/).\n\n- Q: What coins does this support?\n\n  - A: This supports any coin supported by the API being used to fetch coin information.\n\n- Q: How do I set the API to use?\n\n  - A: You can use the `--api` flag, eg. `--api coingecko`. You can also set the API choice in the config file.\n\n    ```toml\n    api = "coingecko"\n    ```\n\n    Options are: `coinmarketcap`, `coingecko`\n\n- Q: How do I change the colorscheme (theme)?\n\n  - A: You can use the `--colorscheme` flag, eg. `--colorscheme matrix`. You can also set the colorscheme choice in the config file.\n\n    ```toml\n    colorscheme = "<colorscheme>"\n    ```\n\n    For more instructions, visit the [colors](https://github.com/cointop-sh/colors) repository.\n\n- Q: How do I create a custom colorscheme?\n\n  - A: Copy an existing [colorscheme](https://github.com/cointop-sh/colors/blob/master/cointop.toml) to `~/.cointop/colors/` and customize the colors. Then run cointop with `--colorscheme <colorscheme>` to use the colorscheme.\n\n- Q: Where is the config file located?\n\n  - A: The default configuration file is located under `~/.cointop/config.toml`\n\n- Q: What format is the configuration file in?\n\n  - A: The configuration file is in [TOML](https://en.wikipedia.org/wiki/TOML) format.\n\n- Q: Will you be supporting more coin API\'s in the future?\n\n  - A: Yes supporting more coin APIs is planned.\n\n- Q: How often is the data polled?\n\n  - A: Data gets polled once 60 seconds by default. You can press <kbd>Ctrl</kbd>+<kbd>r</kbd> to force refresh. You can configure the refresh rate with the flag `--refresh-rate <seconds>`\n\n- Q: How can I change the refresh rate?\n\n  - A: Run cointop with the flag `--refresh-rate 60` where the value is the number of seconds that it will fetch for data. You can also set the refresh rate in the config file:\n\n    ```toml\n    refresh_rate = 60\n    ```\n\n- Q: I ran cointop for the first time and don\'t see any data?\n\n  - A: Running cointop for the first time will fetch the data and populate the cache which may take a few seconds.\n\n- Q: I\'m no longer seeing any data!\n\n  - A: Run cointop with the `--clean` flag to delete the cache. If you\'re still not seeing any data, then please [submit an issue](https://github.com/miguelmota/cointop/issues/new).\n\n- Q: How do I get a CoinMarketCap Pro API key?\n\n- A: Create an account on [CoinMarketCap](https://pro.coinmarketcap.com/signup) and visit the [Account](https://pro.coinmarketcap.com/account) page to copy your Pro API key.\n\n- Q: How do I add my CoinMarketCap Pro API key?\n\n  - A: Add the API key in the cointop config file:\n\n    ```toml\n    [coinmarketcap]\n      pro_api_key = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"\n    ```\n\n    Alternatively, you can export the environment variable `CMC_PRO_API_KEY` containing the API key in your `~/.bashrc`\n\n    ```bash\n    export CMC_PRO_API_KEY=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx\n    ```\n\n    You may also set the API key on start:\n\n    ```bash\n    cointop --coinmarketcap-api-key=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx\n    ```\n\n- Q: I can I add my own API to cointop?\n\n  - A: Fork cointop and add the API that implements the API [interface](https://github.com/miguelmota/cointop/blob/master/cointop/common/api/interface.go) to [`cointop/cointop/common/api/impl/`](https://github.com/miguelmota/cointop/tree/master/cointop/common/api/impl). You can use the CoinGecko [implementation](https://github.com/miguelmota/cointop/blob/master/cointop/common/api/impl/coingecko/coingecko.go) as reference.\n\n- Q: I installed cointop without errors but the command is not found.\n\n  - A: Make sure your `GOPATH` and `PATH` is set correctly.\n\n    ```bash\n    export GOPATH=$HOME/go\n    export PATH=$PATH:$GOPATH/bin\n    ```\n\n- Q: How do I search?\n\n  - A: The default key to open search is <kbd>/</kbd>. Type the search query after the `/` in the field and hit <kbd>Enter</kbd>.\n\n- Q: How do I exit search?\n\n  - A: Press <kbd>ESC</kbd> to exit search.\n\n- Q: Does this work on the Raspberry Pi?\n\n  - A: Yes, cointop works on the Rasperry Pi including the RPi Zero.\n\n- Q: How do I add/remove a favorite?\n\n  - A: Press the <kbd>f</kbd> key to toggle a coin as a favorite.\n\n- Q: How do I view all my favorites?\n\n  - A: Press <kbd>F</kbd> (Shift+f) to toggle view all your favorites.\n\n- Q: How do I save my favorites?\n\n  - A: Favorites are autosaved when setting them. You can also press <kbd>ctrl</kbd>+<kbd>s</kbd> to manually save your favorites to the config file.\n\n- Q: What does the yellow asterisk in the row mean?\n\n  - A: The yellow asterisk or star means that you\'ve selected that coin to be a favorite.\n\n- Q: My favorites aren\'t being saved?\n\n  - A: Try running cointop with `--clean` flag to clear the cache which might be causing the problem.\n\n- Q: How do I add a coin to my portfolio?\n\n  - Press <kbd>e</kbd> on the highlighted coin to enter holdings and add to your portfolio.\n\n- Q: How do I edit the holdings of a coin in my portfolio?\n\n  - Press <kbd>e</kbd> on the highlighted coin to edit the holdings.\n\n- Q: How do I remove a coin in my portfolio?\n\n  - Press <kbd>e</kbd> on the highlighted coin to edit the holdings and set the value to any empty string (blank value). Set it to `0` if you want to keep the coin without a value.\n\n- Q: How do I view my portfolio?\n\n  - A: Press <kbd>P</kbd> (Shift+p) to toggle view your portfolio.\n\n- Q: How do I save my portfolio?\n\n  - A: Your portfolio is autosaved after you edit holdings. You can also press <kbd>ctrl</kbd>+<kbd>s</kbd> to manually save your portfolio holdings to the config file.\n\n- Q: I\'m getting question marks or weird symbols instead of the correct characters.\n\n  - A: Make sure that your terminal has the encoding set to UTF-8 and that your terminal font supports UTF-8.\n\n    You can also try running cointop with the following environment variables:\n\n    ```bash\n    LANG=en_US.utf8 TERM=xterm-256color cointop\n    ```\n\n    If you\'re on Windows (PowerShell, Command Prompt, or WSL), please see the [wiki](https://github.com/miguelmota/cointop/wiki/Windows-Command-Prompt-and-WSL-Font-Support) for font support instructions.\n\n- Q: How do I install Go on Ubuntu?\n\n  - A: There\'s instructions on installing Go on Ubuntu in the [wiki](https://github.com/miguelmota/cointop/wiki/Installing-Go-on-Ubuntu).\n\n- Q: I\'m getting errors installing the snap in Windows WSL.\n\n  - A: Unfortunately Windows WSL doesn\'t support `snapd` which is required for snaps to run. See this [issue thread](https://forum.snapcraft.io/t/windows-subsystem-for-linux/216).\n\n- Q: How do I fix my GOPATH on Windows?\n\n    - A: Go to Control Panel -> Under _System_ click _Edit the system environment variables_ -> then click the _Environment Variables..._ button -> check the `GOPATH` variable.\n\n      Check the environment variable in PowerShell:\n\n      ```bash\n      $ Get-ChildItem Env:GOPATH\n\n      Name                           Value\n      ----                           -----\n      GOPATH                         C:\\Users\\alice\\go\n      ```\n\n- Q: How do I manually build the cointop executable on Windows?\n\n    - A: Here\'s how to build the executable and run it:\n\n      ```powershell\n      > md C:\\Users\\Josem\\go\\src\\github.com\\miguelmota -ea 0\n      > git clone https://github.com/miguelmota/cointop.git\n      > go build -o cointop.exe main.go\n      > cointop.exe\n      ```\n\n- Q: How do I show the help menu?\n\n  - A: Press <kbd>?</kbd> to toggle the help menu. You can also press <kbd>ctrl</kbd>+<kbd>s</kbd> to manually save the selected currency conversion.\n\n- Q: What does saving do?\n\n  - A: The save command (<kbd>ctrl</kbd>+<kbd>s</kbd>) saves your selected currency, selected favorite coins, and portfolio coins to the cointop config file.\n\n- Q: The data isn\'t refreshing!\n\n  - A: The coin APIs have rate limits, so make sure to keep manual refreshes to a minimum. If you\'ve hit the rate limit then wait about half an hour to be able to fetch the data again. Keep in mind that some coin APIs, such as CoinMarketCap, update prices every 5 minutes so constant refreshes aren\'t necessary.\n\n- Q: How do I quit the application?\n\n  - A: Press <kbd>ctrl</kbd>+<kbd>c</kbd> to quit the application.\n\n- Q: How do I quit the open view/window?\n\n  - A: Press <kbd>q</kbd> to quit the open view/window.\n\n- Q: How do I set the favorites view to be the default view?\n\n  - A: In the config file, set `default_view = "favorites"`\n\n- Q: How do I set the portfolio view to be the default view?\n\n  - A: In the config file, set `default_view = "portfolio"`\n\n- Q: How do I set the table view to be the default view?\n\n  - A: In the config file, set `default_view = "default"`\n\n- Q: How can use a different config file other than the default?\n\n  - A: Run cointop with the `--config` flag, eg `cointop --config="/path/to/config.toml"`, to use the specified file as the config.\n\n- Q: I\'m getting the error `open /dev/tty: no such device or address`.\n\n    -A: Usually this error occurs when cointop is running as a daemon or slave which means that there is no terminal allocated, so `/dev/tty` doesn\'t exist for that process. Cointop differs in that it is interactive and also supports more currencies.\n\n- Q: How can I get just the coin price with cointop?\n\n  - A: Use the `cointop price` command. \xe2\x82\xac245.51\n\n    $ cointop price -c ethereum -f usd --api coinmarketcap\n    $276.37\n    ```\n\n- Q: Does cointop do mining?\n\n  - A: Cointop does not do any kind of mining.\n\n## Mentioned
