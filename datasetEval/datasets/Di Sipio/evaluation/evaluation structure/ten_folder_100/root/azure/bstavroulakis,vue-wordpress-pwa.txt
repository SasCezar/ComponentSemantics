You can use it as a starter pack for your WordPress site as well.\n\nSince WordPress can expose a REST API, this project uses Vue.js, Vue-router, Vuex, Vue-resource & Progressive Web App techniques to create an offline-first web client.\n\nYou can also fork the project and host it on Azure App Services with continuous integration using the deploy.cmd file (included)\n\nLive version: [https://www.fullstackweekly.com/](https://www.fullstackweekly.com/)\n\n## Getting Started\n\n### Prerequisites\n\nMake sure that [Node v11](https://nodejs.org/en/download/releases/) is installed.\n\n## Features\n\n- Modern JavaScript syntax with [ES6](https://github.com/lukehoban/es6features) via [babel](https://babeljs.io/).\n- Module bundler via [webpack](https://webpack.github.io/).\n- Components-based architecture via [Vue.js](https://vuejs.org/).\n- State management via [Vuex](https://vuex.vuejs.org/en/).\n- Cache busting via [file-loader](https://webpack.github.io/)\n- Progressive Web App enabled with [service workers](https://developers.google.com/web/fundamentals/getting-started/primers/service-workers)\n- Offline capable with [service workers, app cache](https://developers.google.com/web/fundamentals/getting-started/primers/service-workers)\n- CSS with [Sass](http://sass-lang.com/)\n\n### Instructions\n\nFirst, install the WordPress plugins to enhance its REST API\n\n- Better REST API Featured Images\n- REST API Enabler\n- WP REST API - filter fields\n- WP REST API - Pure Taxonomies\n\nThen, clone the repo or create a fork\n\n```bash\n$ git clone https://github.com/bstavroulakis/vue-wordpress-pwa.git\n```\n\nThen, edit the config:\n\n```bash\nsrc/app.config.js\n```\n\nThen, edit the theme, routes and assets:\n\n```bash\nsrc/theme/\nsrc/components/routerTypes.js\nsrc/components/router.js\nsrc/assets\n```\n\nFinally, to run the project for development with hot reload:\n\n```bash\n$ npm install\n$ npm run dev\n```\n\nOr, build the project for production:\n\n```bash\n$ npm run build\n```\n\n### Deployment\n\nSetup a new site on Azure App Services https://[fullstackweekly].azurewebsites.net\nand as the path select your forked Github project.\n\nYour site should be running on the path you created (https://[fullstackweekly].azurewebsites.net)!\n\nIt should work offline as well and with every commit it will deploy the changes to Azure.\nOnce the user refreshes the site he will receive the new cached version `immediately`.\n\nLicense\n\nMIT\n'
