b'\xef\xbb\xbf# SQL Index Manager\n\nThis tool lets you quickly and easily find out the status of your indexes and discover which databases need maintenance.   \nYou can do maintenance through the UI, or generate a T-SQL script to run in SSMS.\n\n[SQL Index Manager \xe2\x80\x93 Free GUI Tool for Index Maintenance on SQL Server and Azure](https://www.codeproject.com/Articles/5162340/SQL-Index-Manager-Free-GUI-Tool-for-Index-Maintena)\n\n## Key Features\n\n* An incredibly fast describe engine\n* Multiple databases scanning\n* Advice on rebuilding or reorganizing\n* Analysis of fragmentation results\n* Configurable fragmentation thresholds\n* One-click maintenance\n* Command-line automation\n* Automatic T-SQL script generation\n* Columnstore maintenance support\n* Statistics maintenance\n* Collecting missing indexes\n* Detect duplicate indexes\n* Drop or disable unused indexes\n* Index rebuild with compression options\n* Export of scan results into Excel, CSV, HTML and text files\n* Support for any editions of SQL Server 2008+ and Azure\n* And a lot of other improvements :)\n\n## Latest Version\n\nYou can download .zip file with the latest build of the master branch from [Releases](https://github.com/sergeysyrovatchenko/SQLIndexManager/releases)\n\n## Screenshots\n\n![SQL Index Manager](https://habrastorage.org/webt/jw/8s/vk/jw8svkqqg0ybvtdgt1cdoiulxsm.png)\n![SQL Index Manager](https://habrastorage.org/webt/oq/rf/br/oqrfbrezv3yay64mj4gzpt9cjei.png)\n![SQL Index Manager](https://habrastorage.org/webt/3x/iz/4n/3xiz4nf8-wneuchgmovaceynny0.png)\n\n## Command Line\n\n**/connection "Data Source=HOMEPC\\SQL_2017;Integrated Security=True;User ID=HOMEPC\\PC"**\n\nThis switch is used to specify connection string\n\n**/databases "AdventureWorks2017;AdventureWorks2012;msdb"**\n\nThis switch is used to specify databases to manage indexes in\n\n**/connectiontimeout 30**\n\nThis switch is used to specify connection timeout in seconds. Overrides value specified in the connection string\n\n**/commandtimeout 120**\n\nThis switch is used to specify command execution timeout in seconds. Overrides default option (90 seconds)\n\n**/online**\n\nThis switch is used to rebuild indexes with online option\n\n**/ignoreheap**\n\nThis switch is used to ignore any heaps during maintenance\n\n**/ignorecolumnstore**\n\nThis switch is used to ignore any columnstore indexes during maintenance\n\n**/ignorebtree**\n\nThis switch is used to ignore any b-tree indexes during maintenance\n\n**/missingindex**\n\nIncluding missing indexes into regular maintenance\n\n**/maxdop 8**\n\nConfigure the max degree of parallelism during maintenance. Overrides server option\n\n**/reorganizethreshold 15**\n\nSpecifies reorganize the fragmentation threshold in %. Overrides default option (15%)\n\n**/rebuildthreshold 30**\n\nSpecifies rebuild the fragmentation threshold in %. Overrides default option (30%)\n\n**/minindexsize 32**\n\nThis switch is used to specify the minimum index size in MB. Overrides global option (6 MB)\n\n**/predescribesize 128**\n\nThis switch is used to specify the predescribe index size in MB. Overrides global option (256 MB)\n\n**/maxindexsize 8096**\n\nThis switch is used to specify the maximun index size in MB. Overrides global option (8 GB)\n'
