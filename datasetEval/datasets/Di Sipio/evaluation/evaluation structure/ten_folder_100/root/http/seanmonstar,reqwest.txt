b'# reqwest\n\n[![Travis CI Status](https://travis-ci.org/seanmonstar/reqwest.svg?branch=master)](https://travis-ci.org/seanmonstar/reqwest)\n[![Appveyor CI Status](https://ci.appveyor.com/api/projects/status/9ol6jcamwdcxq9gr/branch/master?svg=true)](https://ci.appveyor.com/project/seanmonstar/reqwest)\n[![crates.io](https://img.shields.io/crates/v/reqwest.svg)](https://crates.io/crates/reqwest)\n[![Documentation](https://docs.rs/reqwest/badge.svg)](https://docs.rs/reqwest)\n\nAn ergonomic, batteries-included HTTP Client for Rust.\n\n- Plain bodies, JSON, urlencoded, multipart\n- Customizable redirect policy\n- HTTP Proxies\n- HTTPS via system-native TLS (or optionally, rustls)\n- Cookie Store\n- [Changelog](CHANGELOG.md)\n\n> *NOTE*: reqwest\'s [master](https://github.com/seanmonstar/reqwest) branch is currently\npreparing breaking changes, for most recently released code, look to the\n[0.9.x](https://github.com/seanmonstar/reqwest/tree/0.9.x) branch.\n\n## Example\n\nAsync:\n\n```rust,no_run\nuse std::collections::HashMap;\n\n#[tokio::main]\nasync fn main() -> Result<(), Box<dyn std::error::Error>> {\n    let resp: HashMap<String, String> = reqwest::get("https://httpbin.org/ip")\n        .await?\n        .json()\n        .await?;\n    println!("{:#?}", resp);\n    Ok(())\n}\n```\n\nBlocking:\n\n```rust,no_run\nuse std::collections::HashMap;\n\nfn main() -> Result<(), Box<dyn std::error::Error>> {\n    let resp: HashMap<String, String> = reqwest::blocking::get("https://httpbin.org/ip")?\n        .json()?;\n    println!("{:#?}", resp);\n    Ok(())\n}\n```\n\n## Requirements\n\nOn Linux:\n\n- OpenSSL 1.0.1, 1.0.2, or 1.1.0 with headers (see https://github.com/sfackler/rust-openssl)\n\nOn Windows and macOS:\n\n- Nothing.\n\nReqwest uses [rust-native-tls](https://github.com/sfackler/rust-native-tls), which will use the operating system TLS framework if available, meaning Windows and macOS. On Linux, it will use OpenSSL 1.1.\n\n\n## License\n\nLicensed under either of\n\n- Apache License, Version 2.0 ([LICENSE-APACHE](LICENSE-APACHE) or http://apache.org/licenses/LICENSE-2.0)\n- MIT license ([LICENSE-MIT](LICENSE-MIT) or http://opensource.org/licenses/MIT)\n\n### Contribution\n\nUnless you explicitly state otherwise, any contribution intentionally submitted for inclusion in the work by you, as defined in the Apache-2.0 license, shall be dual licensed as above, without any additional terms or conditions.\n'
