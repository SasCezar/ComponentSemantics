b"# VersionPress\n\nCombining the best of both worlds, VersionPress is a user-friendly versioning solution for WordPress powered by Git. You can:\n\n- See history of your site\n- Undo anything, incl. database changes\n- Roll back to previous states\n- Create safe staging environments: `wp vp clone --name=staging`\n- Merge databases: `wp vp pull --from=staging`\n- Push to GitHub or implement any workflow you like\n\n![image](https://user-images.githubusercontent.com/101152/27992357-15600e14-6493-11e7-8e19-1e85c858a1b8.png)\n\n[![Gitter](https://img.shields.io/gitter/room/nwjs/nw.js.svg)](https://gitter.im/versionpress/versionpress)\n\n## Developer Preview\n\nVersionPress is young \xf0\x9f\x90\xa3. It works fine on simpler sites and you're encouraged to try it in your development workflows but it is not production-ready yet. [Learn more](https://docs.versionpress.net/en/getting-started/about-eap).\n\n## Get started\n\nTo **use** VersionPress:\n\n1. Make sure you understand the [Developer Preview project status](https://docs.versionpress.net/en/getting-started/about-eap).\n2. Download the latest version from [GitHub releases](https://github.com/versionpress/versionpress/releases).\n3. Install to your WordPress site.\n4. Read the [quick start tutorial](https://blog.versionpress.net/2015/05/versionpress-1-0-walkthrough/) and [part 2 on staging](https://blog.versionpress.net/2015/09/versionpress-2-0-staging/).\n5. Enjoy!\n\nTo **develop** VersionPress:\n\n1. Set up your development environment as per [Dev-Setup.md](docs/content/en/developer/dev-setup.md).\n2. Send pull requests.\n\nSee [CONTRIBUTING.md](./CONTRIBUTING.md) for more.\n\n## Useful resources\n\n- Homepage: <https://versionpress.net/>\n- [User documentation](https://docs.versionpress.net/)\n- [Blog](https://blog.versionpress.net/)\n- Support repo: [`versionpress/support`](https://github.com/versionpress/support)\n\n## Contributing\n\nPlease see [CONTRIBUTING.md](./CONTRIBUTING.md), thank you!\n"
