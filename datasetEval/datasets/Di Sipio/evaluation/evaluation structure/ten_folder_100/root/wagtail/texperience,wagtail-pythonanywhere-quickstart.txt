b'wagtail-pythonanywhere-quickstart\n=================================\n\n.. image:: https://img.shields.io/badge/version-v1.0.1-blue.svg\n\n.. image:: https://img.shields.io/badge/license-ISC%20License%20(ISCL)-blue.svg\n    :target: http://en.wikipedia.org/wiki/ISC_license\n\n`Wagtail CMS`_ quickstart for deployment on `PythonAnywhere`_\n\n.. _Wagtail CMS: https://wagtail.io\n.. _PythonAnywhere: https://www.pythonanywhere.com\n\nPrerequisites\n-------------\n\n* You have an account at `PythonAnywhere`_\n\n.. _PythonAnywhere: https://www.pythonanywhere.com\n\nRecommendations\n---------------\n\n* You use `virtualenv`_ for isolated local python development\n* You are familiar with the `Django`_ basics, at least visited their great `online tutorial`_\n\n.. _virtualenv: https://virtualenv.pypa.io\n.. _Django: https://www.djangoproject.com\n.. _online tutorial: https://docs.djangoproject.com/en/dev/intro/tutorial01\n\nTechnical requirements\n----------------------\n\nThis quickstart uses Wagtail 2.3 LTS and Django 1.11 LTS on Python 3.6, so that you can get along without updates for as long as possible. You may change this requirements to your needs. For all supported combinations of Wagtail, Django and Python please refer to the `Wagtail documentation`_.\n\n.. _Wagtail documentation: https://docs.wagtail.io/en/stable/releases/upgrading.html\n\nSet up the python application\n-----------------------------\n\n* Open a bash console on PythonAnywhere\n* Create a virtualenv\n\n  ``mkvirtualenv wagtail-pythonanywhere-quickstart-py36 --python=/usr/bin/python3.6``\n\n* Clone the repository\n\n  ``git clone https://github.com/texperience/wagtail-pythonanywhere-quickstart.git``\n\n* Change into the project directory\n\n  ``cd wagtail-pythonanywhere-quickstart``\n\n* Install the python packages\n\n  ``pip install -r requirements.txt``\n\n* Prepare a local configuration file\n\n  * Create a new file called ``local.py`` under ``mysite/settings``\n  * Define at least the ``SECRET_KEY`` and ``ALLOWED_HOSTS`` settings\n  * Typically you may want to define ``DATABASES`` and point ``MEDIA_ROOT`` as well as ``STATIC_ROOT`` to a persistent directory\n\n* Initialize the database\n\n  ``python manage.py migrate``\n\n* Create a superuser account\n\n  ``python manage.py createsuperuser``\n\n* Collect static files\n\n  ``python manage.py collectstatic``\n\nSet up the web server\n---------------------\n\n* Open the PythonAnywhere "Web" tab\n* Select ``Add a new web app``\n* Choose ``Manual configuration (including virtualenvs)``\n* Choose ``Python 3.6``\n\nAfter your web app is created:\n\n* Set the source code location to your projects home\n* Change the WSGI configuration file\n* Set the virtualenv path to the environment created above\n* Set static and media urls and location\n* Restart your application\n\nCode and contribution\n---------------------\n\nThe code is open source and released under the `ISC License (ISCL)`_.
