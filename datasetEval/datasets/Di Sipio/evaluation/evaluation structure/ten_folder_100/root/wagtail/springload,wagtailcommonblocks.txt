b"# wagtailcommonblocks [![PyPI](https://img.shields.io/pypi/v/wagtailcommonblocks.svg)](https://pypi.python.org/pypi/wagtailcommonblocks)\n\n> Common StreamField blocks for Wagtail.\n\n*Check out [Awesome Wagtail](https://github.com/springload/awesome-wagtail) for more awesome packages and resources from the Wagtail community.*\n\n## Quickstart\n\nAssuming you have a [Wagtail](https://wagtail.io/) project up and running:\n\n```sh\npip install wagtailcommonblocks\n```\n\nAdd commonblocks to your `settings.py` in the INSTALLED_APPS section, before the core wagtail packages:\n\n```python\n...\n    'commonblocks',\n    'wagtail.contrib.wagtailsitemaps',\n...\n```\n# Available blocks\n\n* CommonPageChooserBlock\n* SimpleRichTextBlock\n* CommonImageBlock\n* CommonQuoteBlock\n* CommonHeadingBlock\n* CommonVideoBlock\n* CommonInternalLink\n* CommonExternalLink\n* CommonLinksBlock\n\nYou can override the headings of the `CommonHeadingBlock` block:\n\n```python\n...\nCOMMONBLOCKS_HEADING = (\n    ('h1', 'h1'),\n    ('h2', 'h2'),\n)\n```\n\n## Development\n\n### Releases\n\n- Make a new branch for the release of the new version.\n- Update the [CHANGELOG](https://github.com/springload/wagtailcommonblocks/CHANGELOG.md).\n- Update the version number in `setup.py`, following semver.\n- Make a PR and squash merge it.\n- Back on master with the PR merged, use `make publish` (confirm, and enter your password).\n- Finally, go to GitHub and create a release and a tag for the new version.\n- Done!\n"
