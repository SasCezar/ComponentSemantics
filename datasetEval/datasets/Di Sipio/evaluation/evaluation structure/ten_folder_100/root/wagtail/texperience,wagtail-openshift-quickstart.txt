http://en.wikipedia.org/wiki/ISC_license\n\n`Wagtail CMS`_ quickstart for deployment on OpenShift Online\n\n.. _Wagtail CMS: http://wagtail.io\n\nPrerequisites\n-------------\n* You have an `OpenShift account`_\n* You have installed the `rhc` command line tools for `remote OpenShift administration`_\n\n.. _OpenShift account: https://www.openshift.com\n.. _remote OpenShift administration: https://developers.openshift.com/en/getting-started-client-tools.html\n\nRecommendations\n---------------\n* You use `virtualenv`_ for isolated local python development\n* You are familiar with `Django`_ basics, at least visited their great `online tutorial`_\n\n.. _virtualenv: http://virtualenv.readthedocs.org/en/latest/virtualenv.html\n.. _Django: https://www.djangoproject.com\n.. _online tutorial: https://docs.djangoproject.com/en/dev/intro/tutorial01\n\nGetting started\n---------------\n\nInitialize OpenShift application and local codebase\n***************************************************\n* Open a terminal and change into your workspace where your code will live\n* Create your `Python-based`_ (``python-2.7`` or ``python-3.3``) and `PostgreSQL-backed`_ OpenShift `application`_ with a single command\n\n  ``rhc app create yourwagtaildemoapp python-2.7 postgresql-9.2 --from-code https://github.com/texperience/wagtail-openshift-quickstart``\n\n.. _Python-based: https://www.python.org\n.. _PostgreSQL-backed: http://www.postgresql.org\n.. _application: https://developers.openshift.com/en/getting-started-creating-applications.html\n\n**Great!** Your application is deployed and the git repository is cloned locally into ``yourwagtaildemoapp``\n\nPrepare OpenShift environment\n*****************************\n* Open a terminal session on your application server\n\n  ``rhc ssh -a yourwagtaildemoapp``\n* Create a superuser account\n\n  ``python $OPENSHIFT_REPO_DIR/wsgi/wagtail-openshift-quickstart/manage.py createsuperuser``\n* Close terminal session\n\n  ``exit``\n* Enjoy your remote production site at\n\n  ``http://yourwagtaildemoapp-yourdomain.rhcloud.com`` (Homepage)\n\n  ``http://yourwagtaildemoapp-yourdomain.rhcloud.com/wagtail`` (Wagtail CMS administration)\n\n  ``http://yourwagtaildemoapp-yourdomain.rhcloud.com/django`` (Django administration)\n\n**Congratulations!** Now you can manage and publicly view content in your production site!\n\nInitialize local development environment\n****************************************\n* Prepare your system as documented in the Wagtail docs under `Getting started`_, stop before ``pip install wagtail``\n* Change directory to your local repository root\n\n  ``cd yourwagtaildemoapp``\n* Install necessary python packages (includes all relevant dependencies)\n\n  ``pip install -e .``\n* Create and configure a ``local_settings.py`` (copy template from ``data/local_settings.py``) located at ``$HOME/wagtail-openshift-quickstart/conf``\n* Change directory to your application root\n\n  ``cd wsgi/wagtail-openshift-quickstart``\n* Initialize the database structure and data\n\n  ``python manage.py migrate``\n* Create a superuser account\n\n  ``python manage.py createsuperuser``\n* Run development server\n\n  ``python manage.py runserver``\n* Enjoy your local development site at\n\n  ``http://localhost:8000`` (Homepage)\n\n  ``http://localhost:8000/wagtail`` (Wagtail CMS administration)\n\n  ``http://localhost:8000/django`` (Django administration)\n\n**Let's code!** You're ready to develop your Wagtail powered webpages.\n\n.. _Getting started: http://docs.wagtail.io/en/stable/getting_started/index.html\n\n**That's it! Easy, right?** You've set up an OpenShift hosted Wagtail CMS and your local development environment in 10 Minutes! Awesome!\n"
