b'# Scylla\n\n## Quick-start\n\nTo get the build going quickly, Scylla offers a [frozen toolchain](tools/toolchain/README.md)\nwhich would build and run Scylla using a pre-configured Docker image.\nUsing the frozen toolchain will also isolate all of the installed\ndependencies in a Docker container.\nAssuming you have met the toolchain prerequisites, which is running\nDocker in user mode, building and running is as easy as:\n\n```bash\n$ ./tools/toolchain/dbuild ./configure.py\n$ ./tools/toolchain/dbuild ninja build/release/scylla\n$ ./tools/toolchain/dbuild ./build/release/scylla --developer-mode 1\n ```\n\nPlease see [HACKING.md](HACKING.md) for detailed information on building and developing Scylla.\n\n**Note**: GCC >= 8.1.1 is required to compile Scylla.\n\n## Running Scylla\n\n* Run Scylla\n```\n./build/release/scylla\n\n```\n\n* run Scylla with one CPU and ./tmp as work directory\n\n```\n./build/release/scylla --workdir tmp --smp 1\n```\n\n* For more run options:\n```\n./build/release/scylla --help\n```\n\n## Scylla APIs and compatibility\nBy default, Scylla is compatible with Apache Cassandra and its APIs - CQL and\nThrift. There is also experimental support for the API of Amazon DynamoDB,\nbut being experimental it needs to be explicitly enabled to be used. For more\ninformation on how to enable the experimental DynamoDB compatibility in Scylla,\nand the current limitations of this feature, see\n[Alternator](docs/alternator/alternator.md) and\n[Getting started with Alternator](docs/alternator/getting-started.md).\n\n## Documentation\n\nDocumentation can be found in [./docs](./docs) and on the\n[wiki](https://github.com/scylladb/scylla/wiki). There is currently no clear\ndefinition of what goes where, so when looking for something be sure to check\nboth.\nSeastar documentation can be found [here](http://docs.seastar.io/master/index.html).\nUser documentation can be found [here](https://docs.scylladb.com/).\n\n## Building Fedora RPM\n\nAs a pre-requisite, you need to install [Mock](https://fedoraproject.org/wiki/Mock) on your machine:\n\n```\n# Install mock:\nsudo yum install mock\n\n# Add user to the "mock" group:\nusermod -a -G mock $USER && newgrp mock\n```\n\nThen, to build an RPM, run:\n\n```\n./dist/redhat/build_rpm.sh\n```\n\nThe built RPM is stored in ``/var/lib/mock/<configuration>/result`` directory.\nFor example, on Fedora 21 mock reports the following:\n\n```\nINFO: Done(scylla-server-0.00-1.fc21.src.rpm) Config(default) 20 minutes 7 seconds\nINFO: Results and/or logs in: /var/lib/mock/fedora-21-x86_64/result\n```\n\n## Building Fedora-based Docker image\n\nBuild a Docker image with:\n\n```\ncd dist/docker\ndocker build -t <image-name> .\n```\n\nRun the image with:\n\n```\ndocker run -p $(hostname -i):9042:9042 -i -t <image name>\n```\n\n## Contributing to Scylla\n\n[Hacking howto](HACKING.md)\n[Guidelines for contributing](CONTRIBUTING.md)\n'
