It can run on the server-side with Node.js, or on the client-side within web browsers\n\n<!-- /DESCRIPTION -->\n\n\nQueryEngine provides extensive Querying, Filtering, and Searching abilities for [Backbone.js Collections](http://documentcloud.github.com/backbone/#Collection) as well as JavaScript arrays and objects. The Backbone.js and Underscore dependencies are optional.\n\n\n## Features\n\n* runs on [node.js](http://nodejs.org/) and in the browser\n* supports [NoSQL](http://www.mongodb.org/display/DOCS/Advanced+Queries) queries (like [MongoDB](http://www.mongodb.org/))\n* supports filters (applying a filter function to a collection)\n* supports search strings (useful for turning search input fields into useful queries)\n* supports pills for search strings (e.g. `author:ben priority:important`)\n* supports optional live collections (when a model is changed, added or removed, it can automatically be tested against the collections queries, filters, and search string, if it fails, remove it from the collection)\n* supports parent and child collections (when a parent collection has a model removed, it is removed from the child collection too, when a parent collection has a model added or changed, it is retested against the child collection)\n* actively maintained, supported, and implemented by several companies\n\n\n## Compatability\n\nTested and working against:\n\n- No library\n- [Backbone](http://backbonejs.org) v0.9.2, v0.9.9, v1.0.0, v1.1.0, v1.1.2\n- [Exoskeleton](http://exosjs.com/) v0.5.1, v0.7.0\n\n\n## Using\n\n- [Interactive Demos](http://bevry.github.io/query-engine/)\n- [Complete
