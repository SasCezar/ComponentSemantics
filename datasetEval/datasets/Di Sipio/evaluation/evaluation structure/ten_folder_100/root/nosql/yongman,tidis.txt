b'[![Build Status](https://travis-ci.org/yongman/tidis.svg?branch=master)](https://travis-ci.org/yongman/tidis)\n[![Go Report Card](https://goreportcard.com/badge/github.com/yongman/tidis)](https://goreportcard.com/report/github.com/yongman/tidis)\n![Project Status](https://img.shields.io/badge/status-alpha-yellow.svg)\n\n# What is Tidis?\n\nTidis is a Distributed NoSQL database, providing a Redis protocol API (string, list, hash, set, sorted set), written in Go.\n\nTidis is like [TiDB](https://github.com/pingcap/tidb) layer, providing protocol transform and data structure compute, powered by [TiKV](https://github.com/pingcap/tikv) backend distributed storage which use Raft for data replication and 2PC for distributed transaction.\n\n## Features\n\n* Redis protocol compatible\n* Linear scale-out ability\n* Storage and computation separation\n* Data safety, no data loss, Raft replication\n* Transaction support\n\nAny pull requests are welcomed.\n\n## Architecture\n\n*Architechture of tidis*\n\n![architecture](docs/tidis-arch.png)\n\n*Architechture of tikv*\n\n![](https://pingcap.com/images/blog/TiKV_%20Architecture.png)\n- Placement Driver (PD): PD is the brain of the TiKV system which manages the metadata about Nodes, Stores, Regions mapping, and makes decisions for data placement and load balancing. PD periodically checks replication constraints to balance load and data automatically.\n- Node: A physical node in the cluster. Within each node, there are one or more Stores. Within each Store, there are many Regions.\n- Store: There is a RocksDB within each Store and it stores data in local disks.\n- Region: Region is the basic unit of Key-Value data movement and corresponds to a data range in a Store. Each Region is replicated to multiple Nodes. These multiple replicas form a Raft group. A replica of a Region is called a Peer.\n\n\n## 1. Run tidis with docker-compose in one command\n\n```\ngit clone https://github.com/yongman/tidis-docker-compose.git\ncd tidis-docker-compose/\ndocker-compose up -d\n```\n\nOr follow [tidis-docker-compose](https://github.com/yongman/tidis-docker-compose) guide to run with `docker-compose`\n\n## 2. Build or Docker mannually\n\n### Build from source\n\n```\ngit clone https://github.com/yongman/tidis.git\ncd tidis && make\n```\n\n### Pull from docker\n\n```\ndocker pull yongman/tidis\n```\n\n### Run TiKV cluster for test\n\nUse `docker run tikv` for test, just follow [PingCAP official guide](https://github.com/pingcap/docs/blob/master/op-guide/docker-deployment.md), you just need to deploy PD and TiKV servers, Tidis will take the role of TiDB.\n\n### Run Tidis or docker\n\n#### Run tidis from executable file\n\n```\nbin/tidis-server -conf config.toml\n```\n\n#### Run tidis from docker\n\n```\ndocker run  -d --name tidis -p 5379:5379 -v {your_config_dir}:/data yongman/tidis -conf="/data/config.toml"\n```\n\n## 3. Client request\n\n```\nredis-cli -p 5379\n127.0.0.1:5379> get a\n"1"\n127.0.0.1:5379> lrange l 0 -1\n1) "6"\n2) "5"\n3) "4"\n127.0.0.1:5379> zadd zzz 1 1 2 2 3 3 4 4\n(integer) 4\n127.0.0.1:5379> zcard zzz\n(integer) 4\n127.0.0.1:5379> zincrby zzz 10 1\n(integer) 11\n127.0.0.1:5379> zrange zzz 0 -1 withscores\n1) "2"\n2) "2"\n3) "3"\n4) "3"\n5) "4"\n6) "4"\n7) "1"\n8) "11"\n```\n\n\n## Already
