It offers support for natively storing 17 different formats of data, including JSON, XML, CSV, PDF, Word, Excel, Log, GIS, Image amongst others. It run two full feature storage engines. One that stores data in memory and the other that stores data on disk. In-memory storage offers sheer performance for real-time analytics, while the disk storage make BlobCity an excellent alternative for DataLakes.\n\n<a href="https://www.gartner.com/doc/3288923"><img src="https://blobcity.com/assets/img/Gartner-CoolVendor-2016.jpg" height="80"/></a>\n<a href="https://hub.docker.com/_/blobcity-db"><img src="https://blobcity.com/assets/img/Docker_Container_white_icon%202@2x.png" height="80"/></a>\n\n# Supported Data Formats\n**Push data in any of these 17 formats:** JSON, XML, CSV, SQL, Plaintext, PDF, Excel, Word, RTF, ZIP, Log, Powerpoint, syslog, audio files, video files, image files, GIS\n\n# Multi-Model Example\n<img src="http://blobcity.com/assets/img/JSON-XML.gif"/>\n\n**JSON Record**\n```JSON\n{"col1": 1, "col2": 2}\n```\n\n**XML Record**\n```XML\n<col1>3</col1></col2>4</col2>\n```\n**Auto created table schema and data**\n\n| col1 | col2 |\n|:----:|:----:|\n|   1  |   2  |\n|   3  |   4  |\n\nPush variety of data into a single collection within BlobCity, and get back a standardised response.\n\n```shell\nuser$ nc localhost 10113\nusername>root\npassword>9a371c6445\nYou are now inside the BlobCity DB console\nType \'help\' for assistance and \'exit\' to quit\nblobcity>create-ds test\nDatastore successfully created\nblobcity>create-collection test.test\nCollection successfully created\n\nblobcity>insert into test.test JSON\nIn insert mode. Type 1 JSON per line and press enter to insert\n{"col1": 1, "col2": 2}\nInserted\nexit\nExited insert mode\n\nblobcity>insert into test.test XML\nIn insert mode. Type 1 XML per line and press enter to insert\n<col1>3</col1><col2>4</col2>\nInserted\nexit\nExited insert mode\n\nblobcity>sql test: select * from test.test\n{"p":[{"_id":"5cb30531-dde1-493c-9c67-86b5f4dce36c","col2":2,"col1":1},{"_id":"57f653e3-de68-4591-9563-af9ad66af56b","col2":4,"col1":3}],"time(ms)":2,"ack":"1","rows":2}\n\nblobcity>sql test: select col1 from test.test\n{"p":[{"col1":"1"},{"col1":"3"}],"time(ms)":18,"ack":"1","rows":2}\n\nblobcity>set-column-type test.test col1 integer \nColumn type successfully updated in schema\n\nblobcity>sql test: select SUM(col1) from test.test\n{"p":[{"SUM(col1)":4}],"time(ms)":27,"ack":"1","rows":1}\n```\n\nThe above example shows inserting both JSON and XML recoreds into the same collection. The DB seamlessly creates columns and merges the columns to allow querying of both records using SQL. \n\n# Features\n* **Full SQL:** Run SQL queries over REST, ODBC & JDBC connectivity\n* **DataLake:** On-disk storage engine optimised for DataLake scale with low latency query response\n* **DML Support:** Designed like a DataLake, but works like a database. Full support for `UPDATE` & `DELETE` queries\n* **Realtime:** HIgh speed in-memory storage optimised for real-time analytics\n* **17 Data Formats:** Stores 17 formats of data such as JSON, XML, PDF, Excel, Word amongst others for collective analytics\n* **ACID:** Full ACID compliant transactions on individual records\n* **Stored Procedures:** Run Java & Scala code within the database for complex operations on data without moving the data out of the database\n* **Fine-grained Access Control:** Control data access across users and departments, with column level control on user access\n* **On-Cloud:** Fully managed virtually infinte scale, multi-tenant cloud with unlimited free storae and pay only for what you analyse\n\n# Get Started\n<a href="http://www.blobcity.com"><img src="https://www.blobcity.com/assets/img/blobcity-favicon.png" height="50"/></a>\n&nbsp;&nbsp;\n<a href="https://docs.blobcity.com/docs/getting-started"><img src="https://www.blobcity.com/assets/img/docker.png" height="50"/></a>\n&nbsp;&nbsp;\n<a href="https://docs.blobcity.com/docs/blobcity-on-aws-marketplace"><img src="https://blobcity.com/assets/img/aws.png" height="50"/></a>\n&nbsp;&nbsp;\n<a href="https://docs.blobcity.com/docs/blobcity-on-digital-ocean-marketplace"><img src="https://www.blobcity.com/assets/img/do.png" height="50"/></a>\n\n### On BlobCity Cloud\nStore unlimited data for free and pay only for what you analyse. Delivers ultra high speed analytics over multi-tenant infrastructure, starting at $10/month. \n\n[Start Now](https://blobcity.com/getstarted.html)\n\n### Using Docker\n`docker run -i -p 10111:10111 -p 10113:10113 blobcity/db`\n\nOnce container is started, open a telnet connection on port `10113` to connect to over network CLI. \n```shell\nnc localhost 10113\nTrying 127.0.0.1...\nConnected to localhost.\nEscape character is \'^]\'.\nusername>root\npassword>xxxxx\nYou are now inside the BlobCity DB console\nType \'help\' for assistance and \'exit\' to quit\nblobcity>\n```\n\nA random auto-generated password is placed at `/mnt/data/root-pass.txt`. This file can be found within the container. It can be fetched from within the container, or by mounting this folder to an external mount point. \n\n`docker run -i -v /my-folder:/mnt/data -p 10111:10111 -p 10113:10113 blobcity/db`\n\nThe password file can now be found at `/my-folder/root-pass.txt` on your computer.\n\n### Using Binary Distribution\n*Supported only on Unix & MacOS distributions*\n\nDownload latest tar.gz archive from the [releases](https://github.com/blobcity/db/releases).\n\nDecompress the download file, then run `blobcity.sh` from inside the `bin` folder.\n\n```shell\nuser>tar -xvf blobcity-db-x.x.x.tar.gz\nuser>cd blobcity-db-x.x.x/bin\nuser>sh ./blobcity.sh\n```\n\n`JAVA_HOME` must be set to a JDK / JRE version 8 or higher for the DB to be booted. \n\nThe database will create a folder called `data` at `blobcity-db-x.x.x/data`. The randomly generated `root` user password can be found inside a text file at `blobcity-db-x.x.x/data/root-pass.txt`. \n\nUse this password to connect to the CLI console to start using the DB. It is recommended that the `data` folder be stored at a difference location than the boot volume, and the volume be XFS formatted. \n\nThe location of the data folder can be set by editing the `blobcity.sh` file and uncommenting the following line and setting a folder path of your choice.\n\n```sh\n#export BLOBCITY_DATA=\n```\n\nLook at some of the [best practices](https://docs.blobcity.com/docs/disk-storage-performance-considerations) for optimal disk storage performance. \n\n# Acceleration\nBlobCity is a winner of Economic Times Power of Ideas (Season 2), is funded by CIIE IIM-Ahmedabad and is a graduate from NetApp Excellerator (Cohort #2). \n\n<a href="https://ciie.co"><img src="https://www.blobcity.com/assets/img/ciie-logo.png" height="60"/></a>\n&nbsp;&nbsp;\n<a href="https://startup.netapp.in"><img src="https://www.blobcity.com/assets/img/netapp-excellerator.png" height="60"/></a>\n\n# Docs\n\n[https://docs.blobcity.com](https://docs.blobcity.com)\n\n# Contribute\n\n[Join our Slack community](https://slack.blobcity.com) and request to become a contributor.
