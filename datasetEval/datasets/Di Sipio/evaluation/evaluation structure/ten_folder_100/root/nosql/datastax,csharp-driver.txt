b'# DataStax C# Driver for Apache Cassandra\n\nA modern, [feature-rich][features] and highly tunable C# client library for Apache Cassandra (1.2+) using exclusively Cassandra\'s binary protocol and Cassandra Query Language v3. _Use the [DSE C# driver][dse-driver] for better compatibility and support for DataStax Enterprise_.\n\nThe driver supports .NET Framework 4.5+ and .NET Core 1+.\n\n## Installation\n\n[Get it on Nuget][nuget]\n\n```bash\nPM> Install-Package CassandraCSharpDriver\n```\n\n[![Build status](https://travis-ci.org/datastax/csharp-driver.svg?branch=master)](https://travis-ci.org/datastax/csharp-driver)\n[![Windows Build status](https://ci.appveyor.com/api/projects/status/ri1olv8bl7b7yk7y/branch/master?svg=true)](https://ci.appveyor.com/project/DataStax/csharp-driver/branch/master)\n[![Latest stable](https://img.shields.io/nuget/v/CassandraCSharpDriver.svg)](https://www.nuget.org/packages/CassandraCSharpDriver)\n\n## Features\n\n- Sync and [Async](#asynchronous-api) API\n- Simple, [Prepared](#prepared-statements), and [Batch](#batching-statements) statements\n- Asynchronous IO, parallel execution, request pipelining\n- Connection pooling\n- Auto node discovery\n- Automatic reconnection\n- Configurable [load balancing][policies] and [retry policies][policies]\n- Works with any cluster size\n- [Linq2Cql][linq] and Ado.Net support\n\n## Documentation\n\n- [Documentation index][docindex]\n- [Getting started guide][getting-started]\n- [API docs][apidocs]\n\n## Getting Help\n\nYou can use the project [Mailing list][mailinglist] or create a ticket on the [Jira issue tracker][jira]. Additionally, you can ask questions on [DataStax Community][community].\n\n## Upgrading from previous versions\n\nIf you are upgrading from previous versions of the driver, [visit the Upgrade Guide][upgrade-guide].\n\nIf you are upgrading to [DataStax Enterprise][dse], use the [DSE C# driver][dse-driver] for more features and better\ncompatibility.\n\n## Basic Usage\n\n```csharp\nvar cluster = Cluster.Builder()\n                     .AddContactPoints("host1")\n                     .Build();\n// Connect to the nodes using a keyspace\nvar session = cluster.Connect("sample_keyspace");\n// Execute a query on a connection synchronously\nvar rs = session.Execute("SELECT * FROM sample_table");\n// Iterate through the RowSet\nforeach (var row in rs)\n{\n  var value = row.GetValue<int>("sample_int_column");\n  // Do something with the value\n}\n```\n\n### Prepared statements\n\nPrepare your query **once** and bind different parameters to obtain best performance.\n\n```csharp\n// Prepare a statement once\nvar ps = session.Prepare("UPDATE user_profiles SET birth=? WHERE key=?");\n\n// ...bind different parameters every time you need to execute\nvar statement = ps.Bind(new DateTime(1942, 11, 27), "hendrix");\n// Execute the bound statement with the provided parameters\nsession.Execute(statement);\n```\n\n### Batching statements\n\nYou can execute multiple statements (prepared or unprepared) in a batch to update/insert several rows atomically even in different column families.\n\n```csharp\n// Prepare the statements involved in a profile update once\nvar profileStmt = session.Prepare("UPDATE user_profiles SET email=? WHERE key=?");\nvar userTrackStmt = session.Prepare("INSERT INTO user_track (key, text, date) VALUES (?, ?, ?)");\n// ...you should reuse the prepared statement\n// Bind the parameters and add the statement to the batch batch\nvar batch = new BatchStatement()\n  .Add(profileStmt.Bind(emailAddress, "hendrix"))\n  .Add(userTrackStmt.Bind("hendrix", "You changed your email", DateTime.Now));\n// Execute the batch\nsession.Execute(batch);\n```\n\n### Asynchronous API\n\nSession allows asynchronous execution of statements (for any type of statement: simple, bound or batch) by exposing the `ExecuteAsync` method.\n\n```csharp\n// Execute a statement asynchronously using await\nvar rs = await session.ExecuteAsync(statement);\n```\n\n### Avoid boilerplate mapping code\n\nThe driver features a built-in [Mapper][mapper] and [Linq][linq] components that can use to avoid boilerplate mapping code between cql rows and your application entities.\n\n```csharp\nUser user = mapper.Single<User>("SELECT name, email FROM users WHERE id = ?", userId);\n```\n\nSee the [driver components documentation][components] for more information.\n\n### Automatic pagination of results\n\nYou can iterate indefinitely over the `RowSet`, having the rows fetched block by block until the rows available on the client side are exhausted.\n\n```csharp\nvar statement = new SimpleStatement("SELECT * from large_table");\n// Set the page size, in this case the RowSet will not contain more than 1000 at any time\nstatement.SetPageSize(1000);\nvar rs = session.Execute(statement);\nforeach (var row in rs)\n{\n  // The enumerator will yield all the rows from Cassandra\n  // Retrieving them in the back in blocks of 1000.\n}\n```\n\n### User defined types mapping\n\nYou can map your [Cassandra User Defined Types][udt] to your application entities.\n\nFor a given udt\n\n```cql\nCREATE TYPE address (\n  street text,\n  city text,\n  zip_code int,\n  phones set<text>\n);\n```\n\nFor a given class\n\n```csharp\npublic class Address\n{\n  public string Street { get; set; }\n  public string City { get; set; }\n  public int ZipCode { get; set; }\n  public IEnumerable<string> Phones { get; set;}\n}\n```\n\nYou can either map the properties by name\n\n```csharp\n// Map the properties by name automatically\nsession.UserDefinedTypes.Define(\n  UdtMap.For<Address>()\n);\n```\n\nOr you can define the properties manually\n\n```csharp\nsession.UserDefinedTypes.Define(\n  UdtMap.For<Address>()\n    .Map(a => a.Street, "street")\n    .Map(a => a.City, "city")\n    .Map(a => a.ZipCode, "zip_code")\n    .Map(a => a.Phones, "phones")\n);\n```\n\nYou should **map your [UDT][udt] to your entity once** and you will be able to use that mapping during all your application lifetime.\n\n```csharp\nvar rs = session.Execute("SELECT id, name, address FROM users where id = x");\nvar row = rs.First();\n// You can retrieve the field as a value of type Address\nvar userAddress = row.GetValue<Address>("address");\nConsole.WriteLine("user lives on {0} Street", userAddress.Street);\n```\n\n\n### Setting cluster and statement execution options\n\nYou can set the options on how the driver connects to the nodes and the execution options.\n\n```csharp\n// Example at cluster level\nvar cluster = Cluster\n  .Builder()\n  .AddContactPoints(hosts)\n  .WithCompression(CompressionType.LZ4)\n  .WithLoadBalancingPolicy(new DCAwareRoundRobinPolicy("west"));\n\n// Example at statement (simple, bound, batch) level\nvar statement = new SimpleStatement(query)\n  .SetConsistencyLevel(ConsistencyLevel.Quorum)\n  .SetRetryPolicy(DowngradingConsistencyRetryPolicy.Instance)\n  .SetPageSize(1000);\n```\n\n##
