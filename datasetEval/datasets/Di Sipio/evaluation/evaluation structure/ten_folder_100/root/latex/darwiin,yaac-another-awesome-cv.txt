Feel free to use my [referal link](https://www.overleaf.com/signup?ref=54c221604cd6) if you want to create your account.\n\n## About\n\n**YAAC: Another Awesome CV** was first based on a CV template from Alessandro Plasmati. This template used _XeLaTeX_ engine and _[Fontin Sans](http://www.exljbris.com/fontinsans.html)_ font. \n\n**Personal data** has moved on top of the first page just before the position and _[Fontin Sans](http://www.exljbris.com/fontinsans.html)_ font has been replaced by _[Source Sans Pro Font](https://github.com/adobe-fonts/source-sans-pro)_ from Adobe. _[Font Awesome](http://fontawesome.io/)_ icons highlight important elements.\n\nUnlike _Alessandro Plasmati_ CV template, all layout stuff in **YAAC: Another Awesome CV** has moved in the Latex class file _yaac-another-awesome-cv.cls_ to improve source code readability and style reusability.\n\nMore informations about the original Alessandro Plasmati template can be found here :\n\n   -  [ Scribd ](http://fr.scribd.com/doc/16335667/Writing-your-Professional-CV-with-LaTeX)\n   -  [ LaTeX Templates ](http://www.latextemplates.com/template/plasmati-graduate-cv)\n   -  [ ShareLatex `\\documentclass` you can use option(s) to customize your CV rendering.\n1. **localFont** to use the fonts included in the _fonts_ folder\n2. **alternative** to use the _alternative header_ layout\n3. **10pt** or **11pt** or **12pt** to change the font size\n4. **compact** to reduce vertical space between you experience entries\n\n\n```latex\n% Used with no option, the template will use the 'traditional' \n% header layout your system fonts and the default color scheme ie. blue\n\\documentclass{yaac-another-awesome-cv}\n\n% Used with localFont option the template will use the 'traditional' \n% header layout, fonts included in the fonts directory and the default color scheme\n\\documentclass[localFont]{yaac-another-awesome-cv}\n\n% Used with alternative option, the template will use the 'alternative' \n% header layout, your system fonts and the default color scheme\n\\documentclass[alternative]{yaac-another-awesome-cv}\n\n% Used with localFont, alternative and 10pt options the template will use the 'alternative' \n% header layout, fonts included in the fonts directory, the default color scheme and the '10pt' font size.\n% Based on the article document class, size options are 10pt, 11pt and 12pt\n\\documentclass[localFont,alternative,10pt]{yaac-another-awesome-cv}\n\n% Used with 'compact' option the template will use the compact experience layout \n% Feel free to use this option with the others that are available\n\\documentclass[compact]{yaac-another-awesome-cv}\n\n```\n\n### Construct the header\n\nOutside of the `\\socialinfo` wrapper you have to define the mandatory parameters `\\name` and `\\tagline`.\n\n```latex\n% Define author's name\n% Usage: \\name{<firstname>}{<lastname>}\n% Mandatory\n\\name{Christophe}{ROGER}\n\n% Define author's photo (optional)\n% Usage: \\photo[<shape: circular, square, roundedsquare, squircle>]{<diameter>}{<photo>}\n% The shape of the author's photo is circular by default.\n\\photo{2.5cm}{darwiin}\n\n% Define author's tagline\n% Usage: \\tagline{<tag line>} \n% Mandatory\n\\tagline{Chef de projet IT}\n```\n\nMost social network have their command to render a clickable link or a simple text entry.\n\n```latex\n% Render author's linked-in (optional)\n% Usage: \\linkedin{<linked-in-nick>}\n\\linkedin{christopheroger}\n\n% Render author's viadeo(optional)\n% Usage: \\viadeo{<viadeo-nick>}\n\\viadeo{christopheroger}\n\n% Render author's github (optional)\n% Usage: \\github{<github-nick>}\n\\github{darwiin}\n\n% Render author's stackoverflow profile (optional)\n% Usage: \\stackoverflow{<stackoverflow-user-id>}\n\\stackoverflow{759643}\n\n% Render author's stackexchange profile (optional)\n% Usage: \\stackexchange{<stackexchange-user-id>}\n\\stackexchange{396216}\n\n% Render author's email (optional)\n% Usage: \\email{<email adress>}\n\\email{christophe.roger@mail.com}\n```\n\nPut these command in the `\\socialinfo` wrapper. Feel free to add `\\\\` when you want to force a new line.\n\n```latex\n\\socialinfo{\n  \\linkedin{christopheroger}\n  \\viadeo{christopheroger}\n  \\github{darwiin}\\\\\n  \\smartphone{+687 123 456}\n  \\email{christophe.roger@mail.com}\\\\\n  \\address{2 Rue du quartier, 98765 Ville, Pays}\\\\\n  \\infos{N\xc3\xa9 le 23 septembre 1982 (35 ans) \xc3\xa0 Noum\xc3\xa9a, Nouvelle-Cal\xc3\xa9donie}\n}\n```\n\nUse the `\\makecvheader`command to generate the header.\n\n```latex\n\\makecvheader\n```\n\n### Set the left column size\n\nSections share the left column size (2.5cm by default). You can change this default size using the `\\setleftcolumnlength` command.\n\n```latex\n% Override default left column length of (2.5cm) (optional)\n% Usage: \\setleftcolumnlength{1.5cm} to change the left column lengtyh to 1.5cm\n% I recommend to use a size between 1.5cm to 2.5cm\n\\setleftcolumnlength{1.5cm}\n``\n\n\n### Construct the _comptences_ or _skills_ section\n\nTo list your awesome skills as keyword lists you first have to declare the `keywords` environment and then fill it with a list of keywordsentry\n\n```latex\n% Begin a new keywords environment to show us your awesome keywords\n  \\begin{keywords}\n    \n    % Here's go your keyword lists group by categories\n    \\keywordsentry{Frameworks}{Spring, Spring Boot, \\textbf{Ionic 3}, Angular 2, Angular 4, ZK, JBoss RichFaces}\n  \\end{keywords}\n```\n\n### Construct the _experiences_ section\n\nTo describe your experiences you have first to declare the `experiences` environment\n\n```latex\n% Begin a new experiences environment to use experience and consultantexperience macro\n\\begin{experiences}\n\n% Here's go your experiences\n\n\\end{experiences}\n```\n\nThen you can describe your experiences using **\\experience** and **\\consultantexperience** entries. Each\nentry must be separated by the **\\emptyseparator** \n\n```latex\n% Begin a new experiences environment to use experience and consultantexperience macro\n\\begin{experiences}\n\n% The experience entry work as below and can be used to describe a job experience\n  \\experience\n    {End date}      {Experience title}{Enterprise}{Country}\n    {Begin date}    {\n    \t\t\t\t  experience details\n                      \\begin{itemize}\n                        \\item Item 1: _Item 1 description_\n                        \\item Item 2: _Item 2 description_\n                        \\item Item 3: _Item 3 description_\n                      \\end{itemize}\n                    }\n                    {Technology highlights}\n\n% The emptyseparator macro is used to create white space in your experience\n  \\emptySeparator\n\n% The consultantexperience macro is very similar to the experience macro, but offer you \n% the possibility tu put client details\n  \\consultantexperience\n    {End date}        {Experience title}{Enterprise}{Country}\n    {Begin date}      {Client job title}{Clent enterprise}\n                    {\n                      experience details\n                      \\begin{itemize}\n                        \\item Item 1: _Item 1 description_\n                        \\item Item 2: _Item 2 description_\n                        \\item Item 3: _Item 3 description_\n                      \\end{itemize}\n                    }\n                    {Technology highlights}\n\\end{experiences}\n```\n\n### Construct the _languages_ section\n\nThe _languages_ section use the **skills** environment. \n\n```latex\n% Begin a new skills environment and fill it with skill entries\n  \\begin{skills}\n\n% Render a skill in the skills environment\n% Usage: \\skill{<skill>}{<level between 1 and 5>}\n    \\skill{Fran\xc3\xa7ais}{5}\n    \\skill{Anglais}{4}\n\n% End the skills environment    \n  \\end{skills}\n```\n\n### Construct the _scolarship_ section\n\nThe _scolarship_ section use the **scolarship** environment.\n\n```latex\n\\begin{scholarship}\n\n  % Render a scholarshipentry in the scolarship environment\n  % Usage: \\scholarshipentry{<date>}{<description>}\n  \\scholarshipentry{2007}\n    {Master STIC Professionel fili\xc3\xa8re MBDS de l'Universit\xc3\xa9 de Nice Sophia Antipolis (Master Informatique sp\xc3\xa9cialit\xc3\xa9 Multim\xc3\xa9dia, Base de Donn\xc3\xa9es et int\xc3\xa9gration de Syst\xc3\xa8mes)}\n  \\scholarshipentry{2005}\n    {Licence Sciences et Technologies, Mention Informatique, de l'Universit\xc3\xa9 de Nouvelle-Cal\xc3\xa9donie}\n\\end{scholarship}\n```\n\n### Construct the _projects_ section\n\nThe _projects_ section use the **projects** environment.\n\n```latex\n\\begin{projects}\n  \\project\n    {Simply City}{2017 - 2018}\n    {\\website{https://www.simplycity.nc}{https://www.simplycity.nc} \\website{https://innovation.engie.com/fr/news/actus/territoires/simply-city-lappli-qui-simplifie-la-ville-au-ces-2018-avec-engie/8156}{Pr\xc3\xa9sentation CES 2018} }\n    {Simply City est une application mobile, gratuite et participative destin\xc3\xa9e \xc3\xa0 tous les habitants, visiteurs et touristes qui s\xc3\xa9journent dans une ville. L\xe2\x80\x99application permet de conna\xc3\xaetre toutes les informations et services utiles en temps r\xc3\xa9el.}\n    {Ionic 3,Typescript,Javascript,Visual Studio Code}\n        \n\\end{projects}\n```\n\n### Construct the _references_ section\n\nThe _references_ section use the **references** environment.\n\n```latex\n  \\begin{references}\n\n    % Here's go your references\n\n  \\end{references}\n```\n\nThen you can list your references as follow:\n\n```latex\n  \\begin{referees}\n    \\referee\n      {John Snow}\n      {Lord Commander of the Night Watch}\n      {Night Watch}\n      {john.snow@nightwatch.org}\n      {+687 987 654}\n\n    \\referee\n      {G\xc3\xa9ry Loutre}\n      {Architecte logiciel}\n      {Cafat}\n      {ref1@cafat.nc}\n      {+687 987 654}\n\n  \\begin{referees}\n```\n\n\n## License\n\nLatex class file _yaac-another-awesome-cv_ is published under the term of the [LPPL Version 1.3c](https://www.latex-project.org/lppl.txt).\n\nAll content files are published under the term of the [CC BY-SA 4.0 License](https://creativecommons.org/licenses/by-sa/4.0/legalcode).\n"
