b'<p align="center">\n  <img alt="betterbib" src="https://nschloe.github.io/betterbib/betterbib.svg" width="60%">\n</p>\n\n[![CircleCI](https://img.shields.io/circleci/project/github/nschloe/betterbib/master.svg?style=flat-square)](https://circleci.com/gh/nschloe/betterbib)\n[![codecov](https://img.shields.io/codecov/c/github/nschloe/betterbib.svg?style=flat-square)](https://codecov.io/gh/nschloe/betterbib)\n[![Code style: black](https://img.shields.io/badge/code%20style-black-000000.svg?style=flat-square)](https://github.com/psf/black)\n[![awesome](https://img.shields.io/badge/awesome-yes-ff69b4.svg?style=flat-square)](https://github.com/nschloe/betterbib)\n[![PyPi Version](https://img.shields.io/pypi/v/betterbib.svg?style=flat-square)](https://pypi.org/project/betterbib)\n[![GitHub stars](https://img.shields.io/github/stars/nschloe/betterbib.svg?style=flat-square&logo=github&label=Stars&logoColor=white)](https://github.com/nschloe/betterbib)\n[![PyPi downloads](https://img.shields.io/pypi/dm/betterbib.svg?style=flat-square)](https://pypistats.org/packages/betterbib)\n\nBibTeX files are typically manually maintained and thus often contain inconsistencies,\nmistakes, or are missing information. betterbib helps maintaining your BibTeX files by\ncomparing them with online sources and correcting whatever entries are found faulty.\n\nSimply run\n```\nbetterbib in.bib out.bib\n```\nto improve your BibTeX file with default settings. For example, the input BibTeX\n```\n@article {krylov,\n  author = {Liesen and Gaul and Nabben},\n  title = {Framework Deflation Krylov Augmented}\n}\n```\nis converted to\n```\n@article{krylov,\n author = {Gaul, Andr\xc3\xa9 and Gutknecht, Martin H. and Liesen, J\xc3\xb6rg and Nabben, Reinhard},\n title = {A Framework for Deflated and Augmented {Krylov} Subspace Methods},\n doi = {10.1137/110820713},\n number = {2},\n pages = {495-518},\n source = {Crossref},\n url = {https://doi.org/10.1137/110820713},\n volume = {34},\n journal = {SIAM J. Matrix Anal. \\& Appl.},\n publisher = {Society for Industrial \\& Applied Mathematics (SIAM)},\n issn = {0895-4798, 1095-7162},\n year = {2013},\n month = jan,\n}\n```\n\n### Tools\n\nAll of the following tools can read from standard input and write to standard output, so\nyou can concatenate them to get exactly what you want. For example, the above\n`betterbib` command is equivalent to\n```\nbetterbib-sync in.bib | betterbib-journal-abbrev | betterbib-format -b - out.bib\n```\n\n\n#### Sync\n\nbetterbib fetches data from\n\n   * [Crossref](http://www.crossref.org/) (default) or\n   * [DBLP](http://dblp.uni-trier.de/) (`--source dblp`).\n\nAll betterbib-sync command-line options are explained in `betterbib-sync -h`.\n\n#### Format\n\nThe tool\n```\nbetterbib-format in.bib out.bib\n```\nallows you to apply consistent formatting to you BibTeX file. See `-h`/`--help` for\noptions.\n\n#### (Un)abbreviate journal names\n\nThe tool\n```\nbetterbib-journal-abbrev in.bib out.bib\n```\nallows you to apply consistent abbreviation of journal names. See `-h`/`--help` for\noptions.\n\n#### Custom journal abbreviations\n\nTo use custom abbrebiations for journal names, create a file as a JSON dictionary, and\nprovide that as a command line argument with `--extra-abbrev-file`. For example, if the\nfile `correct_pnas.json` is:\n```json\n{"PNAS": "Proc. Natl. Acad. Sci.  U.S.A."}\n```\nand you call `betterbib-journal-abbrev --extra-abbrev-file=correct_pnas.json`, this will\nreplace any bibtex entries listed with journal "PNAS" with the correct abbreviation.\n\nThis option is included in the `betterbib` and `betterbib-journal-abbrev` commands.\n\nWhen combined with the `--long-journal-names` option, this will override default options\nonly if both have the same abbreviation.\n\n\n### Configuration\n\nIn BibTeX titles, some words need to be protected by curly brackets such that they are\ncapitalized correctly, e.g., `{Einstein}`.  betterbib automatically recognizes some of\nthem (if they are in the default dictionary, like `Einstein`), but you might want to add\nsome. To this end, create the config file\n```\n~/.config/betterbib/config.ini\n```\nand fill it with, e.g.,\n```\n[DICTIONARY]\nadd=Arnoldi,\n    Bernstein,\n    Boolean,\n    Chebyshev,\n    Hermitian\n\nremove=hermitian,\n   boolean\n```\n\n\n### Installation\n\nbetterbib is [available from the Python Package\nIndex](https://pypi.org/project/betterbib/), so simply do\n```\npip3 install betterbib --user\n```\nto install.\n\n### Testing\n\nTo run the betterbib unit tests, check out this repository and type\n```\npytest\n```\n\n### License\n\nbetterbib is published under the [MIT license](https://en.wikipedia.org/wiki/MIT_License).\n'
