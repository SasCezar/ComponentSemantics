b'# Homebridge-Synology\n\n[![Package Quality](http://npm.packagequality.com/shield/homebridge-synology.svg)](http://packagequality.com/#?package=homebridge-synology)\n\nhomebridge-plugin. Control your Synology Diskstation with Apple-Homekit.\n\n> supports DSM 5.x and 6.x\n\n## Installation\nFollow the instruction in [NPM](https://www.npmjs.com/package/homebridge) for the homebridge server installation. The plugin is published through [NPM](https://www.npmjs.com/package/homebridge-synology) and should be installed "globally" by typing:\n\n    sudo npm install -g homebridge-synology\n\n## Configuration\n\nconfig.json\n\nExample:\n\n    {\n        "bridge": {\n            "name": "Homebridge",\n            "username": "CC:22:3D:E3:CE:51",\n            "port": 51826,\n            "pin": "031-45-154"\n        },\n        "description": "This is an example configuration file for homebridge synology plugin",\n        "hint": "Always paste into jsonlint.com validation page before starting your homebridge, saves a lot of frustration",\n        "accessories": [\n            {\n                "accessory": "Synology",\n                "name": "Diskstation", // the name is displayed in homekit\n                "ip": "192.168.178.1", // ip of your diskstation\n                "mac": "A1:B3:C3:D4:E5:EX", // mac of your diskstation\n                "port": "port number", // (optional) port number of the webinterface, default 5000 (or 5001 if you set secure=true)\n                "secure": false, // set this to true if you use a secure connection (https)\n                "account": "admin",\n                "password": "supersecret",\n                "version": 5, // (optional) DSM Version, default is 6\n                "timeout": 5000, // (optional) in ms, increase this value for slow diskstations\n                "disabled": ["temp"], // (optional) see "disable services"\n                "doPolling": true, // (optional) default is false\n                "pollingInterval": 60 // (optional) in s, default is 60\n            }\n        ]\n    }\n\n\n### Disable services\nYou can disable services of your Synology accessory. Add a `disabled` property with an array to your config.json. You can add the following parameters:\n- `temp` to disable the temperature\n- `stats` to disable the custom characeristics cpu load and disk usage quote.\n**Note** This accessory will only appear as switch at the Apple Home App. Use the EVE App instead to get all services.\n\n### Two factor authentification (2FA)\nThis plugin does not support 2FA. If you have enabled 2FA, you can\'t use this plugin.\n\n## Functions\n- wake up (wake-on-lan has to be active) your diskstation\n- shutdown your diskstation\n- get the current system or average disk temperature\n- get the current cpu load\n- get the disk usage quote (it is the average usage if you have more than one volume)\n\n## Issues\nPlease double check your config.json before opening an issue.\nWhen you open an issue provide a detailed description of your problem and add your config.json (without password).\n\n## Roadmap\n- ~~Polling feature with ping~~\n\n## support\nPRs are always welcome.
