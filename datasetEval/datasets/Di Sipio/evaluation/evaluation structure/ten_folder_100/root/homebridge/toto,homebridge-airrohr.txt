b"# homebridge-airrohr\n\n![Tavis CI build status](https://travis-ci.org/toto/homebridge-airrohr.svg?branch=master)\n\n[HomeBridge](http://github.com/nfarina/homebridge) module for the DIY [luftdaten.info](https://luftdaten.info) [air particulates](https://en.wikipedia.org/wiki/Particulates) sensor from [OpenData Stuttgart](ttps://github.com/opendata-stuttgart/).\n\nIt can be used to see the status of your own sensor in HomeKit. You can also use it to see another sensor if its data is published to api.luftdaten.info (see [here](http://luftdaten.info/faq/#toggle-id-8)).\n\n## Features\n\n- See air quality (incl. particulates values for PM10 and PM2.5), temperature and humidity in HomeKit and the Home app\n- Get the data from a sensor on your local network\n- Get the data from public sensor on `api.luftdaten.info`\n- _Soon:_ See history for temperature, etc. using the Elgato Eve app \n\n![homebridge-airrohr sensor in the wild](img/sensor.jpeg)\n\n## Setup\n\nFirst follow the instructions for [HomeBridge](http://github.com/nfarina/homebridge). \n\nInstall `homebridge-airrohr` using `(sudo) npm install -g homebridge-airrohr`.\n\nConfigure your AirRohr sensor in the `homebridge` settings file. See [config.sample.json](config.sample.json). All settings except `update_interval_seconds` (defaults to 120 seconds) and `history` (defaults to persisting to memory) are required.\n\nSee the documentation of the [fakegato-history](https://github.com/simont77/fakegato-history/blob/master/README.md#history-persistence) module for the list of supported options.\n\nTo run this project you have two options\n\n### Option A: The sensor & homebridge in your (home) network\n\nTo do this set `json_data` to\n\n`http://feinstaubsensor-<YOUR_SENSOR_ID>.local/data.json`\n\nusing the same local network as your sensor. \n\n*Note:* If you use a HomeKit hub (like a Apple TV or a iPad) you can still access your data from outside your home. See [Apple's documentation for details](https://support.apple.com/en-us/HT207057).\n\nDon't set `public_temperature_json_data` or `public_airquality_json_data` in this scenario.\n\n### Option B: Use the luftdaten.info API\n\nThis allows you to add any sensor not only your own. To get the same functionality as above you need to set two IDs/URLs:\n\n- Set `public_temperature_json_data` to the URL where to find the temperature data:  `http://api.luftdaten.info/v1/sensor/<YOUR_TEMPERATURE_SENSOR_ID>/` \n\n- Set `public_airquality_json_data` to the URL where to find the air particulates (PM10, PM25) data:  `http://api.luftdaten.info/v1/sensor/<YOUR_AIRQUALITY_SENSOR_ID>/` \n\nDon't set `json_data` in this scenario.\n\n*Note:* If you set `json_data` the `public_\xe2\x80\xa6` settings will be ignored.\n\n## Demo\n\n![homebridge-airrohr in apple home app](img/screenshot.jpeg)\n"
