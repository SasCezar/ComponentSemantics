[![Donate](https://img.shields.io/badge/donate-paypal-yellowgreen.svg)](https://paypal.me/oznu)\n\nControl your supported [Homebridge](https://github.com/nfarina/homebridge) accessories from any Google Home speaker or the Google Home mobile app. Inspired by [homebridge-alexa](https://github.com/NorthernMan54/homebridge-alexa).\n\n* [Supported Device Types](#supported-device-types)\n* [License](#license)\n* [Sponsors](#sponsors)\n\n## Supported Device Types\n\n* Switch\n* Outlet\n* Light Bulb\n    * On / Off\n    * Brightness\n    * Color (Hue/Saturation)\n* Fan (On / Off)\n* Fan v2 (On / Off)\n* Window\n* Window Coverings\n* Door\n* Garage Door\n* Thermostat\n* Television (On / Off)\n* Lock Mechanism (2FA support)\n\n*Note: Google Smart Home does not currently support "sensor" devices such as Temperature Sensors, Motion Sensors, Occupancy Sensors etc.*\n\n## Installation Instructions\n\n#### Option 1: Install via Homebridge Config UI X:\n\nSearch for "Google Home" in [homebridge-config-ui-x](https://github.com/oznu/homebridge-config-ui-x) and install `homebridge-gsh`.\n\n#### Option 2: Manually Install:\n\n```\nsudo npm install -g homebridge-gsh\n```\n\n## Configuration\n\nTo configure `homebridge-gsh` you must also be running [homebridge-config-ui-x](https://github.com/oznu/homebridge-config-ui-x).\n\n1. Navigate to the Plugins page in [homebridge-config-ui-x](https://github.com/oznu/homebridge-config-ui-x).\n2. Click the **Settings** button for the Google Smart Home plugin.\n3. Click the **Link Account** button.\n4. Sign in with your Google or GitHub account.\n5. Your account is now linked.\n6. Restart Homebridge for the changes to take effect.\n7. Add the [Homebridge Action](https://assistant.google.com/services/a/uid/000000b558f0d5d1?hl=en) using the Google Home mobile app. [See Wiki](https://github.com/oznu/homebridge-gsh/wiki#add-homebridge-to-google-home-app) for detailed instructions.\n\n![homebridge-gsh-signup](https://user-images.githubusercontent.com/3979615/62948031-ff228d80-be26-11e9-9e07-ef1023f28fa8.gif)\n\n### Enabling Accessory Control\n\nHomebridge must be running in insecure mode to allow accessory control via this plugin. See [Enabling Accessory Control](https://github.com/oznu/homebridge-config-ui-x/wiki/Enabling-Accessory-Control) for instructions.\n\n### Multiple Homebridge Instances\n\nThis plugin **must** only be configured on one Homebridge instance on your network as the plugin will discover all your other Homebridge instances and be able to control them. For this to work:\n\n* all instances must be running [in insecure mode](https://github.com/oznu/homebridge-config-ui-x/wiki/Enabling-Accessory-Control)\n* all instances must have the same PIN defined in the `config.json`\n\n## Known Issues\n\n1. Only one Homebridge instance can be linked to an account (even across different local networks). You will experience unintended results if you try and link more than one instance to the same account.\n\n## Troubleshooting\n\n#### 1. Errors during installation\n\nMake sure you installed the package with `sudo` flag. Most installation errors can be fixed by removing the plugin and reinstalling:\n\n```shell\n# cleanup\nsudo npm uninstall -g homebridge-gsh\n\n# reinstall\nsudo npm install -g homebridge-gsh\n```\n\n#### 2. Cannot control accessories\n\nSee [Enabling Accessory Control](https://github.com/oznu/homebridge-config-ui-x/wiki/Enabling-Accessory-Control) and [Multiple Homebridge Instances](#multiple-homebridge-instances).\n\n#### 3.
