vscode-terraform\n  <br>\n  <br>\n</h1>\n\n<h4 align="center">Adds syntax highlighting and other editing features for <a href="https://www.terraform.io/">Terraform</a> files</h4>\n\n<p <img src="https://img.shields.io/github/release/mauve/vscode-terraform.svg" alt="Release">\n  </a>\n</p>\n<!-- markdownlint-enable -->\n<!-- markdownlint-disable MD002 MD013 MD041 -->\n\n## Features at a glance\n\n- Auto-completion support\n  1. `Terraform` high level types `(variable, resource, data, module ...)` auto completion support.\n  2. `resource` types `(aws_dynamodb_table ...)` auto completion support for `aws`, `azure`, `datadog`, `google`, `oci` and `openstack`\n  3. `resource | data | module | output | variable` property types auto completion support\n  4. `resource | data` 1-level nested block property types auto completion support\n- Syntax highlighting for `.tf` and `.tfvars` files (and `.hcl`)\n- Automatic formatting using `terraform fmt`\n- Automatically closes braces and quotes\n- Adds a command for running `terraform validate`\n- Linting support with the help of [tflint](https://github.com/wata727/tflint)\n- Browse document symbols\n- Browse workspace symbols\n- Peek definition\n- Goto definition\n- Find references\n- Completion for variables and outputs\n- Rename variables, resource, and data types and all references\n- Show variable values on hover\n- CodeLens shows if a section is referenced\n\n## News in 1.0.0\n\n- Indexing support is no longer experimental\n- Indexing support no longer requires a separate tool (HCL parsing is now built directly into the plugin)\n- *Rename refactoring* now supports all types\n- Hovering a variable shows the default value\n- A code lens shows how often a resource (or variable, or data) is referenced\n- Terraform now works with *Visual Studio LiveShare* (thanks to [@lostintangent](https://github.com/lostintangent))\n\n## Syntax Highlighting\n\n![Syntax Highlighting](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/screenshot.png)\n\n## Auto-completion support\n\n![Auto completion](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-auto-completion.gif)\n\n## Formatting support\n\nThe plugin also ships a formatter integration which uses `terraform fmt` for\nformatting files. Formatting can be enabled or disabled by setting `editor.formatOnSave` or `[terraform]editor.formatOnSave`\nto `true` or `false`.\n\n## Linting\n\n![Linting support](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/screenshot-tflint.png)\n\nYou can lint your terraform templates by invoking the command `Terraform: Lint` from within VSCode.\n\nLinting is supported with the help of [tflint](https://github.com/wata727/tflint), either download it and add it\nto your path or can configure `terraform.lintPath` to point to your tflint executable. `tflint` will be executed\nwith the workspace directory as the current working directory so you can configure `tflint` by dropping a `.tflint.hcl`\nfile in your workspace root.\nYou can change the configuration file path by supplying the configuration option `terraform.lintConfig`.\n\n## Validation\n\nYou can `terraform validate` your project by invoking the command `Terraform: Validate` from within VSCode.\n\n`terraform validate` is invoked with the workspace root as current working directory and with the setting\n`terraform.templateDirectory` as template directory, by default this setting is "templates" (relative to\nworkspace root).\n\nBecause the output of `terraform validate` is not parseable the output is just dumped into the output tab.\n\n### Indexing support\n\nThe plugin indexes all the terraform files in the workspace by default and enables some advanced features,\nlike *Goto Definition* and *Show references* based on information gathered during indexing.\n\nIndexing is enabled by default but can be disabled using `terraform.indexing.enable`.\n\n### Browse Document Symbols\n\nPress `Ctrl+Shift+O` or `\xe2\x87\xa7\xe2\x8c\x98O` to browse symbols in the current file.\n\n![Browse Document Symbols](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-browse-document-symbols.png)\n\n### Browse Workspace Symbols\n\nPress `Ctrl+T` or `\xe2\x8c\x98T` to quickly jump to any symbol.\n\n![Browse Workspace Symbols](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-browse-workspace-symbols.png)\n\n### Peek and Go To definition\n\nPress `Alt+F12` or `\xe2\x8c\xa5F12` to peek definition (currently only some types supported).\n\n![Peek definition](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-peek-definition.png)\n\n### Find All References\n\nPress `Shift+F12` or `\xe2\x87\xa7F12` to find all references (currently only variables).\n\n![Find all references](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-find-references.png)\n\n### Code completion\n\nPress `Ctrl+Space` (also triggered automatically after `.` and `"`) for simple code completion (currently only variables and outputs).\n\n![Complete variables and outputs](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-complete-variables-and-outputs.png)\n\n![Complete variables](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-complete-variables.png)\n\n### Rename variables, resources and its references\n\nPress `F2` to automatically rename a variable and its usages.\n\n![Rename variable (before)](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-rename-variable-before.png)\n![Rename variable (after)](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-rename-variable-after.png)\n\n### Show variable default on hover\n\n![Hover over variable](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-hover-variable.png)\n\n### CodeLens support\n\n![CodeLens shows number of references](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-codelens-references.png)\n\n### Document Link support\n\n![Click on resource type to go to documentation](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-document-link-2.png)\n\n### Outline View support\n\n![An outline of the currently open document is automatically showed](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-outline-view.png)\n\n### Module Overview\n\n![A custom view is contributed which shows an overview of all modules](https://raw.githubusercontent.com/mauve/vscode-terraform/master/images/terraform-module-overview.png)\n'
