b'# Vue-ECharts\n\n> ECharts component for Vue.js.\n\n> [\xf0\x9f\x87\xa8\xf0\x9f\x87\xb3 \xe4\xb8\xad\xe6\x96\x87\xe7\x89\x88](./README.zh_CN.md)\n\nBuilt upon [ECharts](http://echarts.baidu.com/index.html) `v4.1.0`+ and depends on [Vue.js](https://vuejs.org/) `v2.2.6`+.\n\n## Installation\n\n### npm (Recommended)\n\n```bash\n$ npm install echarts vue-echarts\n```\n\n### CDN\n\nInclude `echarts` and `vue-echarts` in your HTML file like this:\n\n```html\n<script src="https://cdn.jsdelivr.net/npm/echarts@4.1.0/dist/echarts.js"></script>\n<script src="https://cdn.jsdelivr.net/npm/vue-echarts@4.0.2"></script>\n```\n\n## Usage\n\n### ES Modules with npm & Vue Loader (Recommended)\n\n```js\nimport Vue from \'vue\'\nimport ECharts from \'vue-echarts\' // refers to components/ECharts.vue in webpack\n\n// import ECharts modules manually to reduce bundle size\nimport \'echarts/lib/chart/bar\'\nimport \'echarts/lib/component/tooltip\'\n\n// If you want to use ECharts extensions, just import the extension package and it will work\n// Taking ECharts-GL as an example:\n// You only need to install the package with `npm install --save echarts-gl` and import it as follows\nimport \'echarts-gl\'\n\n// register component to use\nVue.component(\'v-chart\', ECharts)\n```\n\n#### \xe2\x9a\xa0\xef\xb8\x8f Heads up\n\n##### Importing the source version\n\nVue-ECharts offers an untranspiled version for webpack by default. If you are using official Vue CLI to create your project, you may encounter the problem that the default configuration will exclude `node_modules` from files to be transpiled by Babel. You need to modify the configuration as follows:\n\nFor **Vue CLI 3+**, add `vue-echarts` and `resize-detector` into `transpileDependencies` in `vue.config.js` like this:\n\n```js\n// vue.config.js\nmodule.exports = {\n  transpileDependencies: [\n    \'vue-echarts\',\n    \'resize-detector\'\n  ]\n}\n```\n\nFor **Vue CLI 2** with the `webpack` template, modify `build/webpack.base.conf.js` like this:\n\n```diff\n      {\n        test: /\\.js$/,\n        loader: \'babel-loader\',\n-       include: [resolve(\'src\'), resolve(\'test\')]\n+       include: [\n+         resolve(\'src\'),\n+         resolve(\'test\'),\n+         resolve(\'node_modules/vue-echarts\'),\n+         resolve(\'node_modules/resize-detector\')\n+       ]\n      }\n```\n\nIf you are using bare webpack config, just do similar modifications make it work.\n\n##### Using with Nuxt.js\n\nWhen using Vue-ECharts on the server side with Nuxt.js, it may be not properly transpiled because Nuxt.js prevents files under `node_modules` from being bundled into the server bundle with only a few exceptions by default. We need to whitelist `vue-echarts` manually.\n\nFor **Nuxt.js v2**, modify `nuxt.config.js` as follows:\n\n```js\nmodule.exports = {\n  build: {\n    transpile: [\'vue-echarts\', \'resize-detector\']\n  }\n}\n```\n\nFor **Nuxt.js v1**, modify `nuxt.config.js` as follows:\n\n```js\n// Don\'t forget to\n// npm i --save-dev webpack-node-externals\nconst nodeExternals = require(\'webpack-node-externals\')\n\nmodule.exports = {\n  // ...\n  build: {\n    extend (config, { isServer }) {\n      // ...\n      if (isServer) {\n        config.externals = [\n          nodeExternals({\n            // default value for `whitelist` is\n            // [/es6-promise|\\.(?!(?:js|json)$).{1,5}$/i]\n            whitelist: [/es6-promise|\\.(?!(?:js|json)$).{1,5}$/i, /^vue-echarts/]\n          })\n        ]\n      }\n    }\n  }\n}\n```\n\n### AMD\n\n```js\nrequire.config({\n  paths: {\n    \'vue\': \'path/to/vue\',\n    \'echarts\': \'path/to/echarts\',\n    \'vue-echarts\': \'path/to/vue-ehcarts\'\n  }\n})\n\nrequire([\'vue\', \'vue-echarts\'], function (Vue, ECharts) {\n  // register component to use...\n  Vue.component(\'v-chart\', ECharts)\n})\n```\n\n### Global variable\n\nWithout any module system, the component is exposed as `window.VueECharts`.\n\n```js\n// register component to use\nVue.component(\'v-chart\', VueECharts)\n```\n\n## Using the component\n\n```vue\n<template>\n<v-chart :options="polar"/>\n</template>\n\n<style>\n/**\n * The default size is 600px\xc3\x97400px, for responsive charts\n * you may need to set percentage values as follows (also\n * don\'t forget to provide a size for the container).\n */\n.echarts {\n  width: 100%;\n  height: 100%;\n}\n</style>\n\n<script>\nimport ECharts from \'vue-echarts\'\nimport \'echarts/lib/chart/line\'\nimport \'echarts/lib/component/polar\'\n\nexport default {\n  components: {\n    \'v-chart\': ECharts\n  },\n  data () {\n    let data = []\n\n    for (let i = 0; i <= 360; i++) {\n        let t = i / 180 * Math.PI\n        let r = Math.sin(2 * t) * Math.cos(2 * t)\n        data.push([r, i])\n    }\n\n    return {\n      polar: {\n        title: {\n          text: \'\xe6\x9e\x81\xe5\x9d\x90\xe6\xa0\x87\xe5\x8f\x8c\xe6\x95\xb0\xe5\x80\xbc\xe8\xbd\xb4\'\n        },\n        legend: {\n          data: [\'line\']\n        },\n        polar: {\n          center: [\'50%\', \'54%\']\n        },\n        tooltip: {\n          trigger: \'axis\',\n          axisPointer: {\n            type: \'cross\'\n          }\n        },\n        angleAxis: {\n          type: \'value\',\n          startAngle: 0\n        },\n        radiusAxis: {\n          min: 0\n        },\n        series: [\n          {\n            coordinateSystem: \'polar\',\n            name: \'line\',\n            type: \'line\',\n            showSymbol: false,\n            data: data\n          }\n        ],\n        animationDuration: 2000\n      }\n    }\n  }\n}\n</script>\n```\n\nSee more examples [here](https://github.com/ecomfe/vue-echarts/tree/master/src/demo).\n\n### Props *(all reactive)*\n\n* `initOptions`\n\n  Used to initialize ECharts instance.\n\n* `theme`\n\n  The theme used for current ECharts instance.\n\n* `options`\n\n  Used to update data for ECharts instance. Modifying this prop will trigger ECharts\' `setOption` method.\n\n  If you mutate the data bound to `options` while retaining the object reference, `setOption` will be called with `notMerge: false`. Otherwise, if you bind a new object to `options`, `setOption` will be called with `notMerge: true`.\n\n  For example, if we have the following template:\n\n  ```html\n  <v-chart :options="data"/>\n  ```\n\n  Then:\n\n  ```\n  this.data = newObject // setOption(this.options, true)\n  this.data.title.text = \'Trends\' // setOption(this.options, false)\n  ```\n\n* `group`\n\n  This prop is automatically bound to the same prop of the ECharts instance.\n\n* `autoresize` (default: `false`)\n\n  This prop indicates ECharts instance should be resized automatically whenever its root is resized.\n\n* `manual-update` (default: `false`)\n\n  For performance critical scenarios (having a large dataset) we\'d better bypass Vue\'s reactivity system for `options` prop. By specifying `manual-update` prop with `true` and not providing `options` prop, the dataset won\'t be watched any more. After doing so, you need to retrieve the component instance with `ref` and manually call `mergeOptions` method to update the chart.\n\n### Computed\n\n* `width` **[readonly]**\n\n  Used to retrieve the current width of the ECharts instance.\n\n* `height` **[readonly]**\n\n  Used to retrieve the current height of the ECharts instance.\n\n* `computedOptions` **[readonly]**\n\n  Used to retrive the actual options calculated by ECharts after updating `options`.\n\n### Methods\n\n* `mergeOptions` (use `setOption` in ECharts under the hood)\n\n  *Provides a better method name to describe the actual behavior of `setOption`.*\n\n* `appendData`\n* `resize`\n* `dispatchAction`\n* `showLoading`\n* `hideLoading`\n* `convertToPixel`\n* `convertFromPixel`\n* `containPixel`\n* `getDataURL`\n* `getConnectedDataURL`\n* `clear`\n* `dispose`\n\n### Static Methods\n\n* `connect`\n* `disconnect`\n* `registerMap`\n* `registerTheme`\n* `graphic.clipPointsByRect`\n* `graphic.clipRectByRect`\n\n### Events\n\nVue-ECharts support the following events:\n\n* `legendselectchanged`\n* `legendselected`\n* `legendunselected`\n* `legendscroll`\n* `datazoom`\n* `datarangeselected`\n* `timelinechanged`\n* `timelineplaychanged`\n* `restore`\n* `dataviewchanged`\n* `magictypechanged`\n* `geoselectchanged`\n* `geoselected`\n* `geounselected`\n* `pieselectchanged`\n* `pieselected`\n* `pieunselected`\n* `mapselectchanged`\n* `mapselected`\n* `mapunselected`\n* `axisareaselected`\n* `focusnodeadjacency`\n* `unfocusnodeadjacency`\n* `brush`\n* `brushselected`\n* `rendered`\n* `finished`\n* Mouse events\n  * `click`\n  * `dblclick`\n  * `mouseover`\n  * `mouseout`\n  * `mousemove`\n  * `mousedown`\n  * `mouseup`\n  * `globalout`\n  * `contextmenu`\n\nFor further details, see [ECharts\' API documentation](https://echarts.apache.org/en/api.html).\n\n## Local development\n\n```bash\n$ npm i\n$ npm run dev\n```\n\nOpen `http://localhost:8080/demo` to see the demo.\n'
