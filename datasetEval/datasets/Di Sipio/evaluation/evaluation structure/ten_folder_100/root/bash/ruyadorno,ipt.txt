It takes any kind of list as an input and uses that list to build an interactive interface to let you select an element from it.\n\nStop manually dragging your mouse around to copy output data from a terminal, using the **ipt** workflow you can pipe data from a command and select what to copy to clipboard from a convenient visual menu.\n\nSelected data is also output to _stdout_ allowing for easily composing various workflows - Just create your custom alias!\n\n<br />\n<br />\n\n<p align="center">\n<img alt="demo animation" width="600" src="https://cdn.rawgit.com/ruyadorno/ipt/master/demo.svg" />\n</p>\n\n<br />\n\n## :newspaper: News\n\n- **v2.1.0:**\n  - Added `-D` option for defining a pre-selected item in the list\n- **v2.0.0:**\n  - Added `-p` option for extracting path out of selected items\n  - Added `-M` option for customizing interface message\n  - Added `-0` option for better null char compatibility\n  - Added `-S` option for setting number of lines to be displayed\n  - Added a proper programmatic API\n  - Updated all dependencies to their latest versions\n  - **ipt** now requires node@8+\n- **v1.1.0:** Added autocomplete (or fuzzy finder) mode `ipt -a`\n\n<br />\n\n## :heart_eyes: Features\n\n### Reads from standard input, prints to standard output\n\n**iPipeTo** is inspired by the [Unix Pipeline](https://en.wikipedia.org/wiki/Pipeline_\\(Unix\\)) and is composable with any command line utility you already use and love.\n\n### Select multiple items\n\nThe `-m` or `--multiple` flag allows you to select many items out of the interactive list instead of the standard "pick one" behavior.\n\n### Fuzzysearch (autocomplete mode)\n\nUsing `-a` or `--autocomplete` option switchs the behavior of the interactive list to that of a fuzzysearch (or autocomplete) where options are narrowed as you type.\n\n### Extract path from result\n\nA convenient option that helps extract file system path values out of the selected item, very useful when manipulating verbose output. Use `-p` or `--extract-path` options.\n\n### Copy to clipboard\n\nThe `-c` or `--copy` option allows you to copy the selected item value to clipboard. Makes for useful workflows where you may need that value somewhere else such as out of the terminal or in a manual command to type later.\n\n### Customization\n\nCustomize your workflow by defining the separator to be used to generate the list (`-s` or `--separator` options), a custom message to display on the interactive interface (`-M` or `--message`) and much more. Make sure to take a look at the [Help](#help) section to learn about all the available options.\n\n### Default selected items\n\nUse the `-D` or `--default` option to determine which item(s) should be selected by default on the interactive list. The `-P` or `--default-separator` also allows for a custom char to be used to split these items (similar to `-s` option).\n\n### Node.js based\n\nAll you need in order to run **ipt** is the [Node.js](https://nodejs.org/en/) runtime and [npm](https://www.npmjs.com/), if you have those you\'re already all set!\n\n<br />\n\n## :mag_right: Examples\n\nThe default behavior of **ipt** is to allow for the selection of one item from the interactive list, once selected this item will be output to _stdout_, you can also use `-c` option to copy the result to your clipboard.\n\n\n### Using Unix pipes to send selected value to next command\n\nHere we get a simple list of branchs `git branch -a`, pipe into `ipt` and pipe the selected item value to `git checkout` to checkout into the selected branch. `xargs` is needed to get the data from standard input and read it as an argument.\n\n<br />\n\n<p align="center">\n<img alt="`git branch -a | ipt | xargs git checkout` selects a branch name from menu and that branch gets checked out by git" width="600" src="https://cdn.rawgit.com/ruyadorno/ipt/master/example/git-checkout.svg" />\n</p>\n\n<br/>\n<br/>\n\n### Using the multiple choices option\n\nIn the following example we list all the files from the folder `ls` and pipe that list into `ipt` only that this time we use the "multiple" flag `-m` that allows for selecting multiple items from a list. The selected items get piped to `trash` that deletes them.\n\n<br />\n\n<p align="center">\n<img alt="`ls | ipt -m | xargs rm` selects multiple items from the menu and deletes them" width="600" src="https://cdn.rawgit.com/ruyadorno/ipt/master/example/rm.svg" />\n</p>\n\n<br/>\n<br/>\n\n### Using file as input data\n\nYou can also read a file as source of input data instead of reading from the standard input, here we read from a `TODO` file and redirect the selected items to be written in a `DONE` file.\n\n<br />\n\n<p align="center">\n<img alt="Selects multiple lines from a file and append them to another one" width="600" src="https://cdn.rawgit.com/ruyadorno/ipt/master/example/input.svg" />\n</p>\n\n<br/>\n<br/>\n\n## :arrow_down: Install\n\nAvailable on **npm**:\n\n```sh\nnpm install -g ipt\n```\n\n_Keep in mind that you\'ll need the latest **Node.js** LTS installed!_\n\n<br />\n\n## :sunrise: Awesome workflows\n\nShowcases some useful predefined workflow scripts for using **iPipeTo**:\n\n- [ntl](https://github.com/ruyadorno/ntl): Interactive cli menu to list/run npm tasks\n- [itrash](https://github.com/ruyadorno/itrash): Selects files to delete from current folder\n- [git-iadd](https://github.com/ruyadorno/git-iadd): Interactive staging of selected changed files\n- [git-ishow](https://github.com/ruyadorno/git-ishow): Choose one git stash item to show\n- [npmlist](https://github.com/hankchanocd/npmlist/tree/master): Fuzzy list everything with npm module\n\nFound an awesome workflow idea?
