b'<p align="center">\n  <img src="https://raw.github.com/clojure-emacs/cider/master/logo/cider-logo-w640.png" alt="CIDER Logo"/>\n</p>\n\n-----------\n[![License GPL 3][badge-license]](http://www.gnu.org/licenses/gpl-3.0.txt)\n[![MELPA](http://melpa.org/packages/cider-badge.svg)](http://melpa.org/#/cider)\n[![MELPA Stable](http://stable.melpa.org/packages/cider-badge.svg)](http://stable.melpa.org/#/cider)\n[![CircleCI](https://circleci.com/gh/clojure-emacs/cider.svg?style=svg)](https://circleci.com/gh/clojure-emacs/cider)\n\nCIDER is the **C**lojure(Script) **I**nteractive **D**evelopment **E**nvironment\nthat **R**ocks!\n\nCIDER extends Emacs with support for interactive programming in Clojure. The\nfeatures are centered around `cider-mode`, an Emacs minor-mode that complements\n[clojure-mode][]. While `clojure-mode` supports editing Clojure source files,\n`cider-mode` adds support for interacting with a running Clojure process for\ncompilation, debugging, definition and documentation lookup, running tests and\nso on.\n\n----------\n[![OpenCollective](https://opencollective.com/cider/backers/badge.svg)](#open-collective-backers)\n[![OpenCollective](https://opencollective.com/cider/sponsors/badge.svg)](#open-collective-sponsors)\n[![Patreon](https://img.shields.io/badge/patreon-donate-orange.svg)](https://www.patreon.com/bbatsov)\n[![Paypal](https://www.paypalobjects.com/en_US/i/btn/btn_donate_SM.gif)](https://www.paypal.com/cgi-bin/webscr?cmd=_s-xclick&hosted_button_id=GRQKNBM6P8VRQ)\n\n**Please consider [supporting financially its ongoing development](#funding).**\n\n## Quickstart\n\nThe instructions that follow are meant to get you from zero to a running CIDER\nREPL in under 5 minutes.  See the\n[user manual](https://docs.cider.mx) for (way) more\ndetails.\n\n### Installation\n\nThe recommended way to install CIDER is via `package.el` - the built-in package\nmanager in Emacs.\n\nCIDER is available on the two major `package.el` community\nmaintained repos -\n[MELPA Stable](http://stable.melpa.org)\nand [MELPA](http://melpa.org).\n\nYou can install CIDER with the following command:\n\n<kbd>M-x</kbd> `package-install` <kbd>RET</kbd> `cider` <kbd>RET</kbd>\n\n### Launch an nREPL server and client from Emacs\n\nSimply open in Emacs a file belonging to your `lein` or `boot` project (like\n`foo.clj`) and type <kbd>M-x</kbd> `cider-jack-in`. This will start an nREPL\nserver with all the project dependencies loaded in and CIDER will automatically\nconnect to it.\n\nAlternatively you can use <kbd>C-u M-x</kbd> `cider-jack-in` to specify the name\nof a `lein` or `boot` project, without having to visit any file in it.\n\nIn Clojure(Script) buffers the command `cider-jack-in` is bound to\n<kbd>C-c C-x (C-)j</kbd>.\n\n### Connect to a running nREPL server\n\nYou can go to your project\'s directory in a terminal and type there\n(assuming you\'re using Leiningen that is):\n\n```\n$ lein repl\n```\n\nOr with Boot:\n\n```\n$ boot repl -s wait\n```\n\nAlternatively you can start nREPL either manually or by the facilities provided\nby your project\'s build tool (Gradle, Maven, etc).\n\nAfter you get your nREPL server running go back to Emacs.  Typing there <kbd>M-x</kbd>\n`cider-connect` will allow you to connect to the running nREPL server.\n\nIn Clojure(Script) buffers the command `cider-connect` is bound to\n<kbd>C-c C-x (C-)c (C-)j</kbd> and the command `cider-connect-cljs` is bound to\n<kbd>C-c C-x (C-)c (C-)s</kbd>.\n\n## Diving Deeper\n\nCIDER packs a ton of functionality and you really want to be familiar with it,\nso you can fully empower your workflow. The best way to get acquainted with all\navailable features is to go over the entire\n[CIDER manual](https://docs.cider.mx/).\n\nIf you\'re into video lessons, you might also check out\nthis [intro to CIDER demo](https://www.youtube.com/watch?v=aYA4AAjLfT0) as well.\n\n## Changelog\n\nAn extensive changelog is available [here](CHANGELOG.md).\n\n## Team\n\n### The Core Team\n\nThe direction of the project is being stewarded by the CIDER core team. At this point bumps of the\nminor (second) version number are considered major releases and always\ninclude new features or significant changes to existing features. API\ncompatibility between major releases is not a (big) concern (although we try\nto break the API rarely and only for a good reason).\n\nThe development cycle for the next major\nrelease starts immediately after the previous one has been\nshipped. Bugfix/point releases (if any) address only serious bugs and\nnever contain new features.\n\nThe versions of CIDER and `cider-nrepl` are always kept in sync. If you\'re\ntracking the `master` branch of CIDER, you should also be tracking the `master`\nbranch of `cider-nrepl`.\n\n## Logo\n\nCIDER\'s logo was created by [@tapeinosyne](https://github.com/tapeinosyne).
