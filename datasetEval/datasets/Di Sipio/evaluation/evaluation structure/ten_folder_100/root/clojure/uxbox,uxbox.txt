UXBOX is currently at an early development stage but we are working hard to bring you the beta version as soon as possible. Follow the project progress in Twitter or Github and stay tuned!\n\n[See SVG specification](https://www.w3.org/Graphics/SVG/)\n\n## SVG based ##\n\nUXBOX works with SVG, a standard format, for all your designs and prototypes . This means that all your stuff in UXBOX is portable and editable in many other vector tools and easy to use on the web.\n\n## Development ##\n\nMost of the main operations can be done through the helper script `manage.sh`.\n\nThe development requires of UXBOX is done through a single docker container. Each main service is opened in a different [tmux](https://github.com/tmux/tmux) sessions.\n\n## Docker\n\nDocker is also used to build release images for backend and frontend. Use the helper script `manage.sh` to build the images.\nYou can run locally UXBOX through a docker-compose or by manually running the containers.\n\nComplementary to the docker images you can build locally from this repository, you can find additionnal flavors for backend and frontend on external repositories:\n* [Monogramm/docker-uxbox-frontend](https://github.com/Monogramm/docker-uxbox-frontend)\n* [Monogramm/docker-uxbox-backend](https://github.com/Monogramm/docker-uxbox-backend)\n\n\n### Persistent data\nThe UXBOX installation and all data are stored in the database (file uploads, etc). The docker daemon will store that data within the docker directory `/var/lib/docker/volumes/...`. That means your data is saved even if the container crashes, is stopped or deleted.\n\nTo make your data persistent to upgrading and get access for backups is using named docker volume or mount a host folder. To achieve this you need one volume for your database container.\n\nDatabase:\n- `/var/lib/postgresql/data` PostgreSQL Data\n```console\n$ docker run -d \\\n    -v db:/var/lib/postgresql/data \\\n    postgresql\n```\n\nYou also need to persist the UXBOX backend public resources (media and assets) to not lose images uploaded and allow the frontend to expose assets.\n- `/srv/uxbox/resources/public` UXBOX backend public resources\n```console\n$ docker run -d \\\n    -v db:/srv/uxbox/resources/public \\\n    monogramm/docker-uxbox-backend\n```\n\n### Auto configuration via environment variables\n\nThe following environment variables are also honored for configuring your UXBOX instance:\n\n#### Frontend\n\n**Only available at build time!**\n-\t`-e UXBOX_API_URL=...` (defaults to `/api`)\n-\t`-e UXBOX_VIEW_URL=...` (defaults to `/view/`)\n-\t`-e UXBOX_DEMO=...` (not defined, setting any value will activate demo mode)\n-\t`-e UXBOX_DEBUG=...` (not defined, setting any value will activate debug mode)\n\nAvailable at runtime:\n-\t`-e LANG=...` (defaults to `en_US.UTF-8`)\n-\t`-e LC_ALL=...` (defaults to `C.UTF-8`)\n\n#### Backend\n\nAvailable at runtime:\n-\t`-e LANG=...` (defaults to `en_US.UTF-8`)\n-\t`-e LC_ALL=...` (defaults to `C.UTF-8`)\n-\t`-e UXBOX_HTTP_SERVER_PORT=...` (defaults to `6060`)\n-\t`-e UXBOX_HTTP_SERVER_DEBUG=...` (defaults to `true`)\n-\t`-e UXBOX_DATABASE_USERNAME="..."` (defaults to `nil`)\n-\t`-e UXBOX_DATABASE_PASSWORD="..."` (defaults to `nil`)\n-\t`-e UXBOX_DATABASE_NAME="..."` (defaults to `"uxbox"`)\n-\t`-e UXBOX_DATABASE_SERVER="..."` (defaults to `"localhost"`)\n-\t`-e UXBOX_DATABASE_PORT=...` (defaults to `5432`)\n-\t`-e UXBOX_MEDIA_DIRECTORY=...` (defaults to `resources/public/media`)\n-\t`-e UXBOX_MEDIA_URI=...` (defaults to `http://localhost:6060/media/`)\n-\t`-e UXBOX_ASSETS_DIRECTORY=...` (defaults to `resources/public/static`)\n-\t`-e UXBOX_ASSETS_URI=...` (defaults to `http://localhost:6060/static/`)\n-\t`-e UXBOX_EMAIL_REPLY_TO="..."` (defaults to `no-reply@uxbox.io`)\n-\t`-e UXBOX_EMAIL_FROM="..."` (defaults to `no-reply@uxbox.io`)\n-\t`-e UXBOX_SUPPORT_EMAIL="..."` (defaults to `support@uxbox.io`)\n-\t`-e UXBOX_SMTP_HOST="..."` (defaults to `"localhost"`)\n-\t`-e UXBOX_SMTP_PORT=...` (defaults to `25`)\n-\t`-e UXBOX_SMTP_USER="..."` (defaults to `nil`)\n-\t`-e UXBOX_SMTP_PASSWORD="..."` (defaults to `nil`)\n-\t`-e UXBOX_SMTP_SSL=...` (defaults to `false`)\n-\t`-e UXBOX_SMTP_TLS=...` (defaults to `false`)\n-\t`-e UXBOX_SMTP_ENABLED=...` (defaults to `false`)\n-\t`-e UXBOX_REGISTRATION_ENABLED=...` (defaults to `true`)\n-\t`-e UXBOX_SECRET="..."` (defaults to `"5qjiAndGY3"`)\n\n**Important note:** make sure to use quotation marks for string variables or the backend might try to interpret the values as symbols and have weird issues.\n\n## Contributing ##\n\n**Open to you!**\n\nWe love the open source
