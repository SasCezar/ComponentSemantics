`config.xml` file instead of `--variable`.\n  This allows you to change your API keys for anytime without reinstallation.\n\n  Please pay attention the variable names are changed.\n\n  ```xml\n  <widget ...>\n    <preference name="GOOGLE_MAPS_ANDROID_API_KEY" value="(api key)" />\n    <preference name="GOOGLE_MAPS_IOS_API_KEY" value="(api key)" />\n  </widget>\n  ```\n\n### API key (Browser platform)\n\n  In the browser platform, the maps plugin uses [Google Maps JavaScript API v3](https://developers.google.com/maps/documentation/javascript/)\n\n  You need to set **two API keys for Google Maps JavaScript API v3**.\n\n  ```js\n  // If your app runs this program on browser,\n  // you need to set `API_KEY_FOR_BROWSER_RELEASE` and `API_KEY_FOR_BROWSER_DEBUG`\n  // before `plugin.google.maps.Map.getMap()`\n  //\n  //   API_KEY_FOR_BROWSER_RELEASE for `https:` protocol\n  //   API_KEY_FOR_BROWSER_DEBUG for `http:` protocol\n  //\n  plugin.google.maps.environment.setEnv({\n    \'API_KEY_FOR_BROWSER_RELEASE\': \'(YOUR_API_KEY_IS_HERE)\',\n    \'API_KEY_FOR_BROWSER_DEBUG\': \'\'\n  });\n\n  // Create a Google Maps native view under the map_canvas div.\n  var map = plugin.google.maps.Map.getMap(div);\n\n  ```\n\n### Why **two API keys**?\n\n  `JavaScript` code is `text code`. Even if you do obfuscation, it\'s still readable finally.\n  In order to protect your API key, you need to set `Key restriction` for these keys.\n\n  ![](https://raw.githubusercontent.com/mapsplugin/cordova-plugin-googlemaps/master/images/api_key_restrictions.png)\n\n  If you don\'t set API key, the maps plugin still work with `development mode`.\n\n  ```js\n  plugin.google.maps.environment.setEnv({\n    \'API_KEY_FOR_BROWSER_RELEASE\': \'(YOUR_API_KEY_IS_HERE)\',\n    \'API_KEY_FOR_BROWSER_DEBUG\': \'\' // If key is empty or unset,\n                                    // the maps plugin runs under the development mode.\n  });\n  ```\n  <img src="https://raw.githubusercontent.com/mapsplugin/cordova-plugin-googlemaps/master/images/development_mode.png" width="300">\n\n### Which browser supported?\n\n  Modern browsers should work without any problem.\n\n  ![](https://raw.githubusercontent.com/mapsplugin/cordova-plugin-googlemaps/master/images/modern_browsers.png)\n\n  Internet Explorer 11 might work. We don\'t confirm all features, but basic features work.\n\n### Behavior differences\n\n  `Google Maps JavaScript API v3` is completely different ecosystem with `Google Maps Android API` and `Google Maps SDK for iOS`.\n\n  `Google Maps JavaScript API v3` :\n  - **can\'t** draw 3D building,\n  - **does\'t work** if offline,\n  - **can\'t** map rotation,\n  - etc...\n\n  In the browser platform, the maps plugin works almost the same behaviors as native platforms(Android, and iOS),\n  but not exactly the same behaviors.\n  So don\'t expect too much.\n\n### Touch mechanism difference\n\n  As you may know, [this plugin displays native Google Maps view under the browser in Android and iOS](#how-does-this-plugin-work-android-ios).\n  However this plugin displays as `normal HTML element` in browser platform.\n\n  Because of this, touch behavior is different.\n  The maps plugin does not hook the touch position, do not set `background: transparent`, ... etc.\n  But if you use this plugin as normal behavior, you don\'t need to consider about this.\n\n---------------------------------------------------------------------------------------------------------\n\n## Please support this plugin activity.\n\n  In order to keep this plugin as free, please consider to donate little amount for this project.\n\n  [![Donate](https://img.shields.io/badge/Donate-PayPal-green.svg)](https://www.paypal.com/cgi-bin/webscr?cmd=_donations&business=SQPLZJ672HJ9N&lc=US&item_name=cordova%2dgooglemaps%2dplugin&currency_code=USD&bn=PP%2dDonationsBF%3abtn_donate_SM%2egif%3aNonHosted)\n\n---------------------------------------------------------------------------------------------------------\n\n## Release Notes\n  - Google Maps JavaScript API v3?\n\nGoogle Maps JavaScript API v3 works on any platforms,\nbut it does not work if device is **offline**.\n\nThis plugin uses three different APIs:\n- Android : [Google Maps Android API](https://developers.google.com/maps/documentation/android/)\n- iOS : [Google Maps SDK for iOS](https://developers.google.com/maps/documentation/ios/)\n- Browser : [Google Maps JavaScript API v3](https://developers.google.com/maps/documentation/javascript/)\n\nIn Android and iOS applications, this plugin displays native Google Maps views, which is **faster** than Google Maps JavaScript API v3.\nAnd it even works if the device is **offline**.\n\nIn Browser platform, this plugin displays JS map views (Google Maps JavaScript API v3).\nIt should work as PWA (progressive web application), but the device has to be **online**.\n\nIn order to work for all platforms, this plugin provides **own API** instead of each original APIs.\nYou can write your code `similar to` the Google Maps JavaScript API v3.\n\n**Feature comparison table**\n\n|                | Google Maps JavaScript API v3     | Cordova-Plugin-GoogleMaps(Android,iOS)| Cordova-Plugin-GoogleMaps(Browser) google.maps.visualization.*       | (not available)                       |\n\n### How does this plugin work (Android, iOS)?\n\nThis plugin generates native map views, and puts them **under the browser**.\n\nThe map views are not HTML elements. This means that they are not a `<div>` or anything HTML related.\nBut you can specify the size and position of the map view using its containing `<div>`.\n\nThis plugin changes the background to `transparent` in your application.\nThen the plugin detects your touch position, which is either meant for the `native map` or an `html element`\n(which can be on top of your map, or anywhere else on the screen).\n\n![](https://github.com/mapsplugin/cordova-plugin-googlemaps-doc/raw/master/v1.4.0/class/Map/mechanism.png)\n\nThe benefit of this plugin is the ability to automatically detect which HTML elements are over the map or not.\n\nFor instance, in the image below, say you tap on the header div (which is over the map view).\nThe plugin will detect whether your tap is for the header div or for the map view and then pass the touch event appropriately.\n\nThis means **you can use the native Google Maps views similar to HTML elements**.\n\n![](https://raw.githubusercontent.com/mapsplugin/cordova-plugin-googlemaps/master/images/touch.png)\n\n\n---------------------------------------------------------------------------------------------------------\n## Official Communities\n\n- Gitter : (managed by @Hirbod)\n\n  https://gitter.im/nightstomp/cordova-plugin-googlemaps\n'
