Access denied page\n- **404:** provide a 404 - Page not found page\n- **[Custom Search](https://developers.google.com/structured-data/slsb-overview):** with Google Sitelink search box, people can reach your content more quickly\n- **File not found:** avoid `404 FILE_NOT_FOUND` errors\n- **Layout:** use `divs` instead of `tables` for layout. Using `tables` is not semantically correct.\n- **Moving a website:** redirect all your links to the new location via `.htaccess`\n- **[Pagination](https://support.google.com/webmasters/answer/1663744):** implement the `rel="next"` and `rel="prev"` attributes to links\n- **[Performance](https://developers.google.com/webmasters/mobile-sites/mobile-seo/common-mistakes/slow-mobile-pages):** performance and loading time is important\n- **Redirects:** avoid redirects if possible. Use 301 redirect instead of 302\n- **[RichSnippets](https://schema.org/):** markup your code with rich snippets, they show up on the search results page\n- **Robots:** block pages which should not be indexed via the `robots.txt` file or\n  `<meta name="robots" content="">`\n- **Validation:** write valid code ([HTML Validator](https://validator.w3.org/) [CSS Validator](https://jigsaw.w3.org/css-validator/))\n- **[WAI-Aria](https://www.w3.org/TR/wai-aria/):** use WAI-Aria tags to help machines understand your code\n\n## Meta Information\n\n- **[Description](https://www.youtube.com/watch?v=W4gr88oHb-k):** each page should have a unique description (max. 160 characters)\n  `<meta name="description" content="">`\n- **Title:** each page should have a unique speaking title (60 - 100 characters)\n  `<title>Website Title</title>`\n\n## Keywords\n\n- **Content:** keyword should appear in ~3% of article length\n- **Heading:** keyword should appear in headings\n- **[Meta Tag](https://www.youtube.com/watch?v=jK7IPbnmvVU):** you can ommit the `<meta name="keywords" content="">`,\n  search engines do not use this meta tag\n- **Research:** rank for keywords with high traffic and less competition\n- **Single:** every page should have a single unique targeted keyword\n- **Title:** keyword should appear in page title\n- **[URL](https://www.youtube.com/watch?v=rAWFv43qubI):** keyword should appear in URL name\n\n## Content\n\n- **Content:** content matters the most in SEO\n- **Flash:** avoid Flash content and Flash pages. They are not accessible on mobile phones and will be ranked lower\n- **Freshness:** new content is important. Updating pages or posting regularly is recommended\n- **Headings:** clear structure `H1` -`H6` max. 70 characters long\n- **Length:** article should be at least 300 words\n- **Strong:** use `strong` tag to highlight your targeted keyword\n- **[Uniqueness](https://www.youtube.com/watch?v=mQZY7EmjbMA):** do not provide duplicated content, use unique content types\n\n## Images\n\n- **[Alt tag](https://support.google.com/webmasters/answer/114016):** add an alt-tag this a description of the image (60 - 70 characters)\n- **Dimensions:** add the `width=""` and `height=""` attributes to the image\n- **[File name](https://www.youtube.com/watch?v=h2SWuUobbr0):** use a short descriptive name\n- **[Optimization](https://imageoptim.com/):** Optimize images by removing some meta information\n- **[Responsive Images](https://www.w3.org/TR/html-picture-element/):** serve the most optimized image corresponding to the window size\n- **Size:** keep the filesize as low as possible\n\n## Videos\n\n- **Controls:** add controls to playback and control your video\n- **Embed:** allow others to embed your videos\n- **Transcriptions:** use transcriptions for indexing, usability & content\n- **[Unplayable content](https://developers.google.com/webmasters/mobile-sites/mobile-seo/common-mistakes/unplayable-content):** avoid unplayable video content. Use HTML5 `<video>` tag instead of Flash\n\n## Links\n\n- **Backlinks:** only add external links if you got a backlink to your site\n- **Internal links:** add ~3 internal links to your content\n- **[Languages](https://moz.com/learn/seo/hreflang-tag):** the hreflang tag tells Google which language you are using on a specific page, so the search engine can serve that result to users searching in that language\n  `<link rel="alternate" href="example.com/fr/" hreflang="fr-fr" />`\n- **Naming:** use a descriptive link name: \xe2\x80\x9cClick here\xe2\x80\x9d or \xe2\x80\x9cRead more\xe2\x80\x9d are bad link text. Better \xe2\x80\x9cRead more about SEO and Web Accessibility\xe2\x80\x9d\n- **[nofollow](https://support.google.com/webmasters/answer/96569):** add `rel="nofollow"` attribute to external links only to prevent spam and bad links\n- **Title:** add the title attribute to links\n\n## Mobile\n\n- **[AppLinks](http://applinks.org/documentation/):** apps that link to your content can then use this metadata to deep-link into your app\n- **[mobile friendly](https://googlewebmastercentral.blogspot.be/2014/11/helping-users-find-mobile-friendly-pages.html):** mobile optimized sites are marked in search results. Test for [mobile friendly site](https://www.google.com/webmasters/tools/mobile-friendly/)\n- **[Smart App Banner](https://developer.apple.com/library/ios/documentation/AppleApplications/Reference/SafariWebContent/PromotingAppswithAppBanners/PromotingAppswithAppBanners.html):** Safari has a Smart App Banner feature that provides a standardized method of promoting apps on the App Store from a website\n- **[Tap targets](https://developers.google.com/speed/docs/insights/SizeTapTargetsAppropriately):** clickable links should not be too small\n- **Viewport:** tell browsers how to adjust the page\'s dimensions and scaling to suit the device\n  `<meta name="viewport" content="width=device-width, initial-scale=1">`\n\n## Sitemap\n\n- **[HTML sitemap](https://www.youtube.com/watch?v=hi5DGOu1uA0):** an HTML
