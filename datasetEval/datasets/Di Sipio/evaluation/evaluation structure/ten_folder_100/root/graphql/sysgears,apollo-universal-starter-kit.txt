You can also test a [demo application] deployed on\nHeroku.\n\n## Description\n\n![](Technologies.png)\n\nApollo Universal Starter Kit is an SEO-friendly, fully configured, modular starter project for developing [Universal\nJavaScript] applications. You can use this kit to create your applications in JavaScript or TypeScript for all major\nplatforms &ndash; mobile, web, and server.\n\nApollo Universal Starter Kit is built with [Apollo], [GraphQL], [React], [Angular], [React Native], [Expo], [Knex.js],\nand [Express] with support for relational databases such as PostgreSQL, MySQL, and SQLite.\n\n[TypeScript] is our language of choice and we use it across the entire project. However, you can freely mix vanilla\nJavaScript (the ES6 and ES7 syntax) and TypeScript when creating your modules.\n\nThe starter kit also integrates [Twitter Bootstrap], [Ant Design], and [NativeBase]\nto provide great possibilities for styling for your web and mobile applications.\n\n## Table of Contents\n\n* [Overview](#overview)\n    * [Why Use Apollo Universal Starter productivity thanks to live reload and (partial) hot code reload\n* Fractal modular architecture that\'s easy to support and extend\n* The possibility to create modules in TypeScript _and_ JavaScript at the same time\n* No need to develop [typical features](#architecture-and-implemented-modules) for your applications\n\n#### I am a Project Manager\n\n* Your application will work faster thanks to GraphQL\n* Your team will be able to reuse the code for all the platforms\n* Your team can create client, server, and mobile JavaScript applications\n* Your application will be easier to support and evolve thanks to the fractal modular architecture\n* Your application will be based on a widely-used JavaScript ecosystem (it\'s easy to find JS developers)\n* Your team can develop new features straightaway instead of creating the basic features\n* Your application will be integrated with Stripe, one of the top payment processors\n\n### Concept\n\nDeveloping client-server-mobile projects in JavaScript never was a trivial task. Not only do you have to spend time\ninstalling the application dependencies and configuring them, but you\'re also constrained to implement many basic\nfunctionalities over and over again. And you never have time to develop a starter codebase that you can reuse across\nall of your projects.\n\nTo relieve you from the burden of configuring the project, developing the application structure, and implementing\ntypical features, we created Apollo Universal Starter Kit.\n\nApollo Universal Starter Kit provides you with a client-server-mobile application that you can employ as a foundation\nfor developing new web or mobile projects using popular tools from the JavaScript ecosystem. But our starter kit does so\nmuch more than just creating a mix of popular JS technologies &mdash; it\'s powered by a few **custom** libraries and\nsolutions to simplify managing project configurations, creating new modules, building GraphQL queries, and perform many\nother tasks.\n\nThe starter kit also consists of many modules that you can augment and adapt to develop your specific application. Also,\nyou can use those prebuilt modules as a reference when implementing basic features for your applications even if you\ncreate them using other technologies.\n\n### Architecture and Implemented Modules\n\nAmong all the approaches to building the application architecture, we opt for the _disposable fractal-based modular\narchitecture_. Thanks to it, it\'s possible to remove any built-in module from Apollo Universal Starter Kit without\nbreaking the application. We recommend that you develop your custom modules with the same idea in mind when using our\nstarter kit.\n\nApollo Universal Starter Kit comes with the following modules:\n\n* **Authentication**. Authentication via social networks (Facebook, GitHub, LinkedIn, and Google using OAuth) and\npassword-based authentication; refreshing a forgotten password\n* **Authorization**. Permission-based authorization with various user roles\n* **Contact Us Form**. Functionality to send messages to the server\n* **Internationalization**. A complete internationalization solution for the client and server\n* **Mobile Chat**. A live chat based on the [React Native Gifted Chat] and powered by GraphQL subscriptions\n* **Pagination**. Navigation between pages and presentation of entities\n* **Payments**. Functionality for recurring payments based on Stripe\n* **Posts and Comments**. Functionality to add, delete, and update posts and comments\n* **State Management**. The application state stored in the database and on the client using different approaches\n* **404 Not Found Page**. A minimalistic module for handling 404 requests\n\nIf you don\'t want to use the pre-built modules in your project, you can remove them using a [dedicated CLI]. For module\nnames, see the names of directories under `modules`.\n\nTo learn more about the features and modules available in Apollo Universal Starter Kit, follow to the dedicated section\n[Features and Modules].\n\n## Demo\n\nHere\'s a demo of Apollo Universal Starter Kit in action:\n\n![screencast](https://user-images.githubusercontent.com/1259926/27387579-c6799ada-56a1-11e7-93fc-d08e9970640d.gif)\n\nYou can try out the latest version of Apollo Universal Starter Kit [deployed on Heroku]. Clone the stable branch of Apollo Universal Starter Kit.\n\n```\ngit clone -b stable https://github.com/sysgears/apollo-universal-starter-kit.git\ncd apollo-universal-starter-kit\n```\n\n**NOTE**: The master branch is not recommended for development. Use it at your own risk.\n\n**NOTE**: If you\'re going to use Windows to develop with Apollo Universal Starter Kit, you need to additionally enable\nsymlinks _before_ you run the project.\n\nFor Windows 10:\n\n* Press `Win` + `I` to open **Settings**\n* Click **Update & Security**\n* Click the **For Developers** tab\n* In the **Use developer features** window, switch to **Developer Mode**\n\n**NOTE**: You can remove the unnecessary stacks from Apollo Universal Starter Kit by using the CLI. Consult a\n[respective CLI section]. Alternatively, you can use the capabilities of your operating system. For example, by running\nthe command below, you can remove all Scala server files:\n\n```bash\nfind -name server-scala | xargs rm -rf\n```\n\nIf you don\'t need the ready-made modules, you can also remove them using the [custom CLI].\n\n2. Install the dependencies. Make sure that you use Yarn 1.0.0 or higher.\n\n```\nyarn\n```\n\nYou can use NPM instead of Yarn to handle the starter kit dependencies and to run scripts. Throughout the Apollo\nUniversal Starter Kit documentation, we\'ll always use Yarn.\n\n3. Seed sample data to the database. The command below will create new tables with sample data in SQLite:\n\n```\nyarn seed\n```\n\nSQLite is a typical default relational database installed in most Linux distributions including Mac OS X; otherwise,\nconsult [SQLite installation guide].\n\n4. Run the starter kit in development mode:\n\n```\nyarn watch\n```\n\nThe server application will be running on [http://localhost:3000], while the client application will be running on\n[http://localhost:8080]. The terminal will tell your the exact ports.\n\nFor more information about running this starter kit for mobile development or Docker, consult the [Getting Started]\nguide.\n\n## Project Structure\n\nThe project structure presents generally accepted guidelines and patterns for building scalable web and mobile\napplications.\n\nThe structure is _fractal_ meaning the available functionality is grouped primarily by feature rather than by file type.\nBut the current structure isn\'t prescriptive, and you can change it however you like.\n\n```\napollo-universal-starter-kit\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 config                      # Various application configurations\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 docs                        # Documentation\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 node_modules                # Global Node.js modules\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 modules                     # All the prebuilt project modules\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 packages                    # Available packages\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 client                  # React client\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 client-angular          # Angular client\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 client-vue              # Vue client\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 common                  # Common code\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 mobile                  # React Native mobile client\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 server                  # Node.js and Express server\n\xe2\x94\x82   \xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 server-scala            # Scala server\n\xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 tools                       # All build and CLI-related files\n```\n\nInside `modules`, you\'ll find all the prebuilt modules that Apollo Universal Starter Kit comes with. Each module under\n`modules` contains sub-directories with module implementations for different technologies. For example, if you look up\nthe module `modules/core`, you\'ll see the following sub-modules:\n\n```\napollo-universal-starter-kit\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 modules                       # Available packages\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 core                      # The core module\n\xe2\x94\x82       \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 client-angular        # Core functionality for Angular app\n\xe2\x94\x82       \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 client-react          # Core functionality for React app\n\xe2\x94\x82       \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 client-react-native   # Core functionality for React Native app\n\xe2\x94\x82       \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 client-vue            # Core functionality for Vue app\n\xe2\x94\x82       \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 common                # React Native mobile client\n\xe2\x94\x82       \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 server-scala          # Core functionality for Scala server\n\xe2\x94\x82       \xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 server-ts             # Core functionality for Express server\n\xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 tools                         # All build and CLI-related files\n```\n\n\n## Apollo Universal Starter Kit Documentation\n\nFollow to the documentation concerning different aspects of how to run, configure, and develop with Apollo Universal\nStarter Kit.\n\n* [Getting Started]\n    * [Installing and Running Apollo Universal Starter Kit]\n    * [Running the Mobile App with Expo]\n    * [Running the Starter Kit in a Mobile Simulator]\n* [Running Apollo Universal Starter Kit with Docker]\n* [Deploying Apollo Universal Starter Kit to Production]\n* [Configuring Apollo Universal Starter Kit]\n* [Features and Modules]\n* [Writing Code]\n* [Debugging Code]\n* [Available Scripts]\n* [Frequently Asked Questions]\n* [Project Structure]\n* [Importing Modules]\n\nTools\n\n* [Apollo Universal Starter Kit CLI]\n\nModules\n\n* [Stripe Payments]\n* [Mobile Chat]\n\n## Support\n\n### Community Support\n\n* [Gitter channel] &ndash; ask questions, find answers, and participate in general discussions\n* [GitHub issues] &ndash; submit issues and send feature requests\n* [Wiki] &ndash; read documentation for the usage scenarios of the starter kit; edit the documentation\n* [FAQ] &ndash; consult the Frequently Asked Questions section\n\n### Commercial Support\n\nThe [SysGears] team provides comprehensive support for commercial partners.
