https://altair-gql-translate.surge.sh/ [Click here](https://crwd.in/altair-gql).\n\n![set url](src/assets/img/readme/app-shot.png "Altair GraphQL Client")\n\nA graphQL client for making graphQL queries to a graphQL server - similar to Postman but for GraphQL.\n\nWeb app: https://altair-gql.sirmuel.design/\n\nThere\'s a [chrome extension](https://chrome.google.com/webstore/detail/altair-graphql-client/flnheeellpciglgpaodhkhmapeljopja) and a [firefox add-on](https://addons.mozilla.org/en-US/firefox/addon/altair-graphql-client/).\n\nThere are apps for [mac, windows and linux users](https://altair.sirmuel.design/) as well.\n\nYou can also install using cask:\n\n```\n$ brew cask install altair-graphql-client\n```\n\nFor linux users, you can also install using [snap](https://snapcraft.io/altair):\n\n```\n$ snap install altair\n```\n\nFor arch linux users, an AUR package [aur/altair](https://aur.archlinux.org/packages/altair/) exists:\n\n```\n$ yay -S altair\n```\n\n### Usage with express\nYou can use altair with an express server using [altair-express-middleware](https://www.npmjs.com/package/altair-express-middleware). Read more about how to use this [here](packages/altair-express-middleware/README.md).\n\n### Usage with koa\nYou can use altair with a koa server using [altair-koa-middleware](https://www.npmjs.com/package/altair-koa-middleware). Read more about how to use this [here](https://altair.sirmuel.design/docs/integrations/altair-koa-middleware).\n\n## Features\n\n- Headers\n- Variables\n- Response Stats\n- Documentation (via introspection)\n- Documentation search\n- Syntax highlighting\n- Error highlighting\n- Add queries and fragments from documentation\n![add fragment](src/assets/img/readme/add-fragment.gif "Altair GraphQL Client")\n- Upload files via GraphQL\n- Autocompletion of fields, arguments, types, etc\n- Autofill all fields at cursor\n- Schema auto refresh\n- Prettify Query\n- Compress Query\n- History of recent queries\n- Multiple Windows\n- GraphQL Subscriptions (with desktop notifications)\n- Import/Export Queries\n- Multiple Languages\n- Light and Dark themes\n\nSee more here: https://altair.sirmuel.design/docs/features\n\n### Missing feature?\n\nIs there a feature that is missing? You can let us know by creating a [new issue](https://github.com/imolorhe/altair/issues/new) or you can add the feature by creating a [Pull Request](https://github.com/imolorhe/altair/blob/staging/.github/CONTRIBUTING.md).\n\n## Development server\n\nRun `yarn start` for a dev server. Navigate to `http://localhost:4200/`. The app will automatically reload if you change any of the source files.\n\n## Build\n\nRun `yarn build` to build the project. The build artifacts will be stored in the `dist/` directory. Use the `-prod` flag for a production build.\n\n## Generate chrome extension files\n\nRun `yarn build-ext` to build the chrome extension files. The extension files will be stored in the `chrome-extension/` directory.\n\n## Generate electron app\n\nRun `yarn build-electron` to build the electron apps. The apps will be stored in the `electron-builds/` directory.\n\n## Updating docs\n\nRun `bundle exec jekyll serve`.\n\n## Running tests\n\nRun `yarn test` to execute the unit tests via [Karma](https://karma-runner.github.io) and the end-to-end tests via [Protractor](http://www.protractortest.org/).\n\n\n## Further help\n\nTo get more help on working with the project, check out the [Angular CLI README](https://github.com/angular/angular-cli/blob/master/README.md).\n\n### Contributors\n\nThis project exists thanks to all the people who contribute.
