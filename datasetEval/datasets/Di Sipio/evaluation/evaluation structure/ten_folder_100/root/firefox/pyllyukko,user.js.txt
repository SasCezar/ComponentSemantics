b'# user.js\n\n**Firefox configuration hardening**\n\nA [user.js](http://kb.mozillazine.org/User.js_file) configuration file for [Mozilla Firefox](https://www.mozilla.org/en-US/firefox/new/) designed to harden browser settings and make it more secure.\n\n**This is a default template with every possible hardening measure enforced. See the [relaxed branch](https://github.com/pyllyukko/user.js/tree/relaxed) for a variant providing more usability**\n\n[![Build Status](https://travis-ci.org/pyllyukko/user.js.svg?branch=master)](https://travis-ci.org/pyllyukko/user.js)\n\n### Main goals\n\n* Limit the possibilities to track the user through [web analytics](https://en.wikipedia.org/wiki/Web_analytics).\n* Harden the browser against known data disclosure or code execution vulnerabilities.\n* Limit the browser from storing anything even remotely sensitive persistently.\n* Make sure the browser doesn\'t reveal too much information to [shoulder surfers](https://en.wikipedia.org/wiki/Shoulder_surfing_%28computer_security%29).\n* Harden the browser\'s encryption (cipher suites, protocols, trusted CAs).\n* Limit possibilities to uniquely identify the browser/device using [browser fingerprinting](https://en.wikipedia.org/wiki/Device_fingerprint).\n* Hopefully limit the attack surface by disabling various features.\n* Still be usable in daily use.\n\n### How to achieve this?\n\nThere are several parts to all this and they are:\n\n* [Downloading](#download) and [installing](#installation) the `user.js` file.\n* Reading about and applying [further hardening](#further-hardening) techniques.\n* _Optional:_ Modifying `user.js` to adapt it to your web browser usage.\n\n----------------------------------------------\n\n* [Download](#download)\n* [Installation](#installation)\n* [What does it do?](#what-does-it-do)\n* [Further hardening](#further-hardening)\n* [Known problems and limitations](#known-problems-and-limitations)\n* [FAQ](#faq)\n* [Contributing](#contributing)\n* [Online tests](#online-tests)\n* [References](#references)\n\n\n----------------------------------------------\n\n## Download\n\nDifferent download methods are available:\n\n * Clone using git: `git clone https://github.com/pyllyukko/user.js`\n * Download and extract the [ZIP file](https://github.com/pyllyukko/user.js/archive/master.zip) containing the latest version.\n * Download the latest `user.js` [directly](https://raw.githubusercontent.com/pyllyukko/user.js/master/user.js)\n\n## Installation\n\n### Backups\n\nDo note that these settings alter your browser behaviour quite a bit, so it is recommended to either create a completely new [profile][15] for Firefox or backup your existing [profile directory](http://kb.mozillazine.org/Profile_folder_-_Firefox) before putting the `user.js` file in place.\n\nTo enable the Profile Manager, run Firefox with [command-line arguments](http://kb.mozillazine.org/Command_line_arguments): `firefox --no-remote -P`\n\n### Single profile installation\n\nCopy `user.js` in your current user profile directory, or (recommended) to a fresh, newly created Firefox profile directory.\n\nThe file should be located at:\n\n| OS                         | Path                                                                                                                                          |\n| -------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |\n| Windows 7                  | `%APPDATA%\\Mozilla\\Firefox\\Profiles\\XXXXXXXX.your_profile_name\\user.js`                                                                       |\n| Linux                      | `~/.mozilla/firefox/XXXXXXXX.your_profile_name/user.js`                                                                                       |\n| OS X                       | `~/Library/Application Support/Firefox/Profiles/XXXXXXXX.your_profile_name`                                                                   |\n| Android                    | `/data/data/org.mozilla.firefox/files/mozilla/XXXXXXXX.your_profile_name` and see [issue #14](https://github.com/pyllyukko/user.js/issues/14) |\n| Sailfish OS + Alien Dalvik | `/opt/alien/data/data/org.mozilla.firefox/files/mozilla/XXXXXXXX.your_profile_name`                                                           |\n| Windows (portable)         | `[firefox directory]\\Data\\profile\\`                                                                                                           |\n\nWith this installation method, if you change any of `user.js` settings through [`about:config`](http://kb.mozillazine.org/About:config) or Firefox preferences dialogs, they will be reset to the `user.js` defined values after you restart Firefox. This makes sure they\'re always back to secure defaults when starting the browser. However this prevents persistently changing settings you don\'t consider appropriate. Either edit `user.js` directly, or use the system-wide installation method described below.\n\n### System-wide installation (all platforms)\n\nGenerate a file suitable for system-wide installation, by running `make` with one of the following targets:\n\n* `systemwide_user.js`: (the value will be used as default value for all Firefox Profiles where it is not explicitly set, it can be changed in `about:config` and is kept across browser sessions)\n* `locked_user.js`: (the value will be used as default value on Firefox profile creation, will be locked and can\'t be changed) in `user.js` or in Firefox\'s `about:config` or settings.\n* `debian_locked.js`: Debian specific. Users are not able to override preferences. See [#415](https://github.com/pyllyukko/user.js/issues/415).\n\nCopy the produced file to the Firefox installation directory. The file should be located at:\n\n| OS                        | Path                                                                         |\n| ------------------------- | ---------------------------------------------------------------------------- |\n| Windows                   | `C:\\Program Files (x86)\\Mozilla Firefox\\mozilla.cfg`                         |\n| Linux                     | `/etc/firefox/syspref.js`, for older versions: `/etc/firefox/firefox.js`     |\n| Linux (Debian)            | `/etc/firefox-esr/firefox-esr.js`                                            |\n| Linux (Gentoo, Archlinux) | `/usr/lib/firefox/mozilla.cfg`, might also be `/usr/lib32/` or `/usr/lib64/` |\n| OS X                      | `/Applications/Firefox.app/Contents/Resources/mozilla.cfg`                   |\n\n#### Additional installation steps for Windows / OS X / Gentoo / Archlinux\n\nCreate `local-settings.js` in Firefox installation directory, with the following contents:\n\n```\npref("general.config.obscure_value", 0);\npref("general.config.filename", "mozilla.cfg");\n```\n\nThis file should be located at:\n\n| OS                        | Path                                                                            |\n| ------------------------- | ------------------------------------------------------------------------------- |\n| Windows                   | `C:\\Program Files (x86)\\Mozilla Firefox\\defaults\\pref\\`                         |\n| OS X                      | `/Applications/Firefox.app/Contents/Resources/defaults/pref`                    |\n| Linux (Gentoo, Archlinux) | `/usr/lib/firefox/defaults/pref/`, might also be `/usr/lib32/` or `/usr/lib64/` |\n\nIf `mozilla.cfg` still fails to load, you must add a blank comment to the top of `mozilla.cfg` like so:\n```\n//\n```\n\n### Updating using git\n\nFor any of the above methods, you can keep your browser\'s `user.js` with the latest version available here: Clone the repository, and create a symbolic link from the appropriate location to the `user.js` file in the repository. Just run `git pull` in the repository when you want to update, then restart Firefox:\n\n```\ncd ~/.mozilla/firefox\ngit clone \'https://github.com/pyllyukko/user.js.git\'\ncd XXXXXXXX.your_profile_name\nln -s ../user.js/user.js user.js\n```\n\n### Verifying\n\nVerify that the settings are effective from [about:support](http://kb.mozillazine.org/Troubleshooting_Information_report#Modified_Preferences) (check the "Important Modified Preferences" and "user.js Preferences" sections).\n\n--------------------------------------------\n\n## What does it do?\n\nThere\'s a whole lot of settings that this modifies and they are divided in the following sections.\n\nSome of the settings in this `user.js` file might seem redundant, as some of them are already set to the same values by default. Some CAs have been known for misusing or deliberately abusing this power in the past, and **a single malicious CA can compromise all** your encrypted communications! Follow [this document](CAs.md) to only trust a selected, trimmed-down list of CAs.\n* Keep your browser updated! If you check [Firefox\'s security advisories](https://www.mozilla.org/security/known-vulnerabilities/firefox.html), you\'ll see that pretty much every new version of Firefox contains some security updates. If you don\'t keep your browser updated, you\'ve already lost the game.\n* Disable/uninstall all unnecessary extensions and plugins!\n* Use long and **unique** passwords/passphrases for each website/service.\n* Prefer open-source, reviewed and audited software and operating systems whenever possible.\n* Do not transmit information meant to be private over unencrypted communication channels.\n* Use a search engine that doesn\'t track its users, and set it as default search engine.\n* If a plugin is absolutely required, [check for plugin updates](https://www.mozilla.org/en-US/plugincheck/)\n* Create different [profiles][15] for different purposes\n* Change the Firefox\'s built-in tracking protection to use the [strict list](https://support.mozilla.org/en-US/kb/tracking-protection-pbm?as=u#w_change-your-block-list)\n* Change the timezone for Firefox by using the `TZ` environment variable (see [here](https://wiki.archlinux.org/index.php/Firefox_privacy#Change_browser_time_zone)) to reduce it\'s value in browser fingerprinting\n* If you are concerned about more advanced threats, use specialized hardened operating systems and browsers such as [Tails](https://tails.boum.org/) or [Tor Brower Bundle](https://www.torproject.org/projects/torbrowser.html.en)\n\n\n### Add-ons\n\nHere is a list of the most essential security and privacy enhancing add-ons that you should consider using:\n\n* [uBlock Origin](https://addons.mozilla.org/firefox/addon/ublock-origin/)\n  * For additional protection, enable more blocklists in the addon dashboard.\n  * For additional protection, set it to [Hard mode](https://github.com/gorhill/uBlock/wiki/Blocking-mode:-hard-mode) (experienced users) - the default is [Easy mode](https://github.com/gorhill/uBlock/wiki/Blocking-mode:-easy-mode)\n* [HTTPS Everywhere](https://www.eff.org/https-everywhere)\n  * For additional protection, enable `Block all unencrypted requests` in the toolbar button menu.
