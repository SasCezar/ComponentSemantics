Installing from the Web Store/Addons page\n\n__Chrome__\nhttps://chrome.google.com/webstore/detail/toggl-button/oejgccbfbmkkpaidnkphaiaecficdnfn\n\n__Firefox__\nhttps://addons.mozilla.org/en-US/firefox/addon/toggl-button-time-tracker/\n\n## Installing from Source\n\n- Clone the repository `git clone git@github.com:toggl/toggl-button.git`\n- Install Node.js and NPM https://nodejs.org/en/\n- Run `npm install`\n- Run `npm start`\n\nThen depending on your browser, continue with the following steps.\n\n#### On Chrome:\n\n1.  Navigate to `chrome://extensions/` and enable "Developer Mode".\n2.  Choose "Load unpacked extension..."\n3.  Select the `dist/chrome/` directory\n\n**Note** that you will sometimes need to manually reload the unpacked extension, depending which files you\'re working on.\n\n#### On Firefox:\n\n1. Navigate to `about:debugging`\n2. Choose `Load Temporary Add-on`\n3. Select the `dist/firefox/manifest.json` file\n\n**Note** that in Firefox the add-on needs to be manually loaded again when the browser restarts.\n\nIf you want to inspect the distribution version, run `npm run build`. A zipped distribution can then be found in `dist/`.\n\n## Changelog\n\nList of all the changes and added features can be found at http://toggl.github.io/toggl-button.\n\n## Using the Button\n\n1. Log in to your [Toggl] account from the extension popup.\n2. Navigate to your [service](#compatible-services) and start your Toggl timer there.\n\n_See this article for reference where the start timer link is located in all the tools: [Where can I find the Button?]_\n\nOr start entry from the extension icon menu.\n\n3. To edit the running time entry:\n  - Edit entry details from the post start popup that is shown right after you click the "Start timer" button.\n  - Edit entry details from the extension icon menu by clicking the running time entry name.\n\n4. To stop the current running timer:\n  - Press the button again.\n  - Stop the entry from the extension icon menu.\n  - Start another time entry inside your account.\n\n## Custom domains\n\nIf you use a setup, where one of the supported services is on a custom domain you can define the custom domains in settings "Permissions" tab.\n\n## Contributing\n\nWant to contribute?
