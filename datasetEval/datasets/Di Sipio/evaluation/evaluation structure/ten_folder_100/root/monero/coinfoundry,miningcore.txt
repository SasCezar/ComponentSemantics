Refer to the section further down below for instructions.\r\n\r\n### Running pre-built Release Binaries on Windows\r\n\r\n- Download miningcore-win-x64.zip from the latest [Release](https://github.com/coinfoundry/miningcore/releases)\r\n- Extract the Archive\r\n- Setup the database as outlined below\r\n- Create a configuration file <code>config.json</code> as described [here](https://github.com/coinfoundry/miningcore/wiki/Configuration)\r\n- Run <code>dotnet Miningcore.dll -c config.json</code>\r\n\r\n### Basic PostgreSQL Database setup\r\n\r\nCreate the database:\r\n\r\n```console\r\n$ createuser miningcore\r\n$ createdb miningcore\r\n$ psql (enter the password for postgres)\r\n```\r\n\r\nInside psql execute:\r\n\r\n```sql\r\nalter user miningcore with encrypted password \'some-secure-password\';\r\ngrant all privileges on database miningcore to miningcore;\r\n```\r\n\r\nImport the database schema:\r\n\r\n```console\r\n$ wget https://raw.githubusercontent.com/coinfoundry/miningcore/master/src/Miningcore/Persistence/Postgres/Scripts/createdb.sql\r\n$ psql -d miningcore -U miningcore -f createdb.sql\r\n```\r\n\r\n### Advanced PostgreSQL Database setup\r\n\r\nIf you are planning to run a Multipool-Cluster, the simple setup might not perform well enough under high load. In this case you are strongly advised to use PostgreSQL 11 or higher. After performing the steps outlined in the basic setup above, perform these additional steps:\r\n\r\n**WARNING**: The following step will delete all recorded shares. Do **NOT** do this on a production pool unless you backup your <code>shares</code> table using <code>pg_backup</code> first!\r\n\r\n```console\r\n$ wget https://raw.githubusercontent.com/coinfoundry/miningcore/master/src/Miningcore/Persistence/Postgres/Scripts/createdb_postgresql_11_appendix.sql\r\n$ psql -d miningcore -U miningcore -f createdb_postgresql_11_appendix.sql\r\n```\r\n\r\nAfter executing the command, your <code>shares</code> table is now a [list-partitioned table](https://www.postgresql.org/docs/11/ddl-partitioning.html) which dramatically improves query performance, since almost all database operations Miningcore performs are scoped to a certain pool. \r\n\r\nThe following step needs to performed **once for every new pool** you add to your cluster. Be sure to **replace all occurences** of <code>mypool1</code> in the statement below with the id of your pool from your Miningcore configuration file:\r\n\r\n```sql\r\nCREATE TABLE shares_mypool1 PARTITION OF shares FOR VALUES IN (\'mypool1\');\r\n```\r\n\r\nOnce you have done this for all of your existing pools you should now restore your shares from backup.\r\n\r\n### [Configuration](https://github.com/coinfoundry/miningcore/wiki/Configuration)\r\n\r\n### [API](https://github.com/coinfoundry/miningcore/wiki/API)\r\n\r\n### Building from Source\r\n\r\n#### Building on Ubuntu 16.04\r\n\r\n```console\r\n$ wget -q https://packages.microsoft.com/config/ubuntu/16.04/packages-microsoft-prod.deb\r\n$ sudo dpkg -i packages-microsoft-prod.deb\r\n$ sudo apt-get update -y\r\n$ sudo apt-get install apt-transport-https -y\r\n$ sudo apt-get update -y\r\n$ sudo apt-get -y install dotnet-sdk-2.2 git cmake build-essential libssl-dev pkg-config libboost-all-dev libsodium-dev libzmq5\r\n$ git clone https://github.com/coinfoundry/miningcore\r\n$ cd miningcore/src/Miningcore\r\n$ dotnet publish -c Release --framework netcoreapp2.2  -o ../../build\r\n```\r\n\r\n#### Building on Windows\r\n\r\nDownload and install the [.Net Core 2.2 SDK](https://www.microsoft.com/net/download/core)\r\n\r\n```dosbatch\r\n> git clone https://github.com/coinfoundry/miningcore\r\n> cd miningcore/src/Miningcore\r\n> dotnet publish -c Release --framework netcoreapp2.2  -o ..\\..\\build\r\n```\r\n\r\n#### Building on Windows - Visual Studio\r\n\r\n- Download and install the [.Net Core 2.2 SDK](https://www.microsoft.com/net/download/core)\r\n- Install [Visual Studio 2017](https://www.visualstudio.com/vs/).
