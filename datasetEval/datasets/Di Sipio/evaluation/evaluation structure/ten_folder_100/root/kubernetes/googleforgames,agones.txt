b'<img src="./docs/agones.png" alt="Agones logo" width="250px" height="250px" />\n\n[![GoDoc](https://godoc.org/agones.dev/agones?status.svg)](https://godoc.org/agones.dev/agones)\n[![Go Report Card](https://goreportcard.com/badge/github.com/googleforgames/agones)](https://goreportcard.com/report/github.com/googleforgames/agones)\n[![GitHub release](https://img.shields.io/github/release/googleforgames/agones.svg)](https://github.com/googleforgames/agones/releases)\n[![Follow on Twitter](https://img.shields.io/twitter/follow/agonesdev.svg?style=social&logo=twitter)](https://twitter.com/intent/follow?screen_name=agonesdev)\n\nAgones is a library for hosting, running and scaling [dedicated game servers](https://en.wikipedia.org/wiki/Game_server#Dedicated_server) on [Kubernetes](https://kubernetes.io).\n\n_Agones, is derived from the Greek word ag\xc5\x8dn which roughly translates to \xe2\x80\x9ccontest\xe2\x80\x9d, \xe2\x80\x9ccompetition at games\xe2\x80\x9d and \xe2\x80\x9cgathering\xe2\x80\x9d.\n([source](https://www.merriam-webster.com/dictionary/agones))_\n\n## Why does this project exist?\nAgones replaces usual bespoke or proprietary cluster management and game server scaling solutions with a [Kubernetes](https://kubernetes.io/) cluster\nthat includes the Agones custom _[Kubernetes Controller](https://kubernetes.io/docs/concepts/api-extension/custom-resources/#custom-controllers)_ and matching [Custom Resource Definitions](https://kubernetes.io/docs/concepts/api-extension/custom-resources/#customresourcedefinitions) for _GameServers_, _Fleets_ and more.\n\nWith Agones, Kubernetes gets native abilities to create, run, manage and scale dedicated game server processes within Kubernetes clusters using standard Kubernetes tooling and APIs. This model also allows any matchmaker to interact directly with Agones via the Kubernetes API to provision a dedicated a game server.\n\nFor more details on why this project was written, read the\n[announcement blog post](https://cloudplatform.googleblog.com/2018/03/introducing-Agones-open-source-multiplayer-dedicated-game-server-hosting-built-on-Kubernetes.html).\n\n## Major Features\n- Define a single `GameServer`, and/or large game server `Fleets` within Kubernetes - either through yaml or via the API\n- Manage GameServer lifecycles - including health checking and connection information.\n- `Fleet` Autoscaling capabilities that integrate with Kubernetes\' native cluster autoscaling\n- Gameserver specific metric exports and dashboards for ops teams\n\n## Usage\n\nDocumentation can be found on the [Agones website](https://agones.dev/site/docs/).\n\n## Get involved\n\n- [Slack](https://join.slack.com/t/agones/shared_invite/enQtMzE5NTE0NzkyOTk1LWU3ODAyZjdjMjNlYWIxZTAwODkxMGY3YWEyZjNjMjc4YWM1Zjk0OThlMGU2ZmUyMzRlMDljNDJiNmZlMGQ1M2U)\n- [Twitter](https://twitter.com/agonesdev)\n- [Mailing List](https://groups.google.com/forum/#!forum/agones-discuss)\n\n## Code of Conduct\n\nParticipation in this project comes under the [Contributor Covenant Code of Conduct](code-of-conduct.md)\n\n## Development and Contribution\n\nPlease read the [contributing](CONTRIBUTING.md) guide for directions on submitting Pull Requests to Agones, and community membership governance.\n\nSee the [Developing, Testing and Building Agones](build/README.md) documentation for developing, testing and building Agones from source.\n\nThe [Release Process](docs/governance/release_process.md) documentation displays the project\'s upcoming release calendar and release process.\n\nAgones is in active development -
