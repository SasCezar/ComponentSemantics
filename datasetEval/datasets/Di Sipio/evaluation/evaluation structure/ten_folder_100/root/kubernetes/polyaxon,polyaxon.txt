And it turns GPU servers into shared, self-service resources for your team or organization.\n\n# Install\n\n#### TL;DR;\n\n * Create a deployment\n\n    ```bash\n    # Create a namespace\n    $ kubectl create namespace polyaxon\n\n    # Add Polyaxon charts repo\n    $ helm repo add polyaxon https://charts.polyaxon.com\n\n    # Deploy Polyaxon\n    $ helm install polyaxon/polyaxon \\\n        --name=polyaxon \\\n        --namespace=polyaxon \\\n        -f config.yaml\n    ```\n\n * Install CLI\n\n    ```bash\n    # Install Polyaxon CLI\n    $ pip install -U polyaxon-cli\n\n    # Config Polyaxon CLI\n    $ polyaxon config ...\n\n    # Login to your account\n    $ polyaxon login\n    ```\n\nPlease check [polyaxon installation guide](https://docs.polyaxon.com/setup/)\n\n> If you are looking for the documentation of Polyaxon v0.4, please [go here](https://v0-4.polyaxon.com/). \n\n# Quick start\n\n#### TL;DR;\n\n * Start a project\n\n    ```bash\n    # Create a project\n    $ polyaxon project create --name=quick-start --description='Polyaxon quick start.'\n\n    # Clone the quick start repo\n    $ git clone https://github.com/polyaxon/polyaxon-quick-start.git\n    $ cd polyaxon-quick-start\n\n    # initialize it\n    $ polyaxon init quick-start\n    ```\n\n * Train and track logs & resources\n\n    ```bash\n    # Upload code and start experiments\n    $ polyaxon run -u\n\n    # Stream logs\n    $ polyaxon experiment -xp 1 logs\n\n    # Stream resources\n    $ polyaxon experiment -xp 1 resources\n    ```\n\n * Dashboard\n\n    ```bash\n    # Start Polyaxon dashboard\n    $ polyaxon dashboard\n\n    Dashboard page will now open in your browser. framework you are using, you need to adapt your code to enable the distributed training,\nand update your polyaxonfile with an environment section.\n\nHere are some examples of using distributed training: \n\n * [Tensorflow distributed example](https://github.com/polyaxon/polyaxon-examples/blob/master/in_cluster/tensorflow/cifar10/polyaxonfile_distributed.yml)\n * [MXNet distributed example](https://github.com/polyaxon/polyaxon-examples/blob/master/in_cluster/mxnet/cifar10/polyaxonfile_distributed.yml)\n * [Pytorch distributed example](https://github.com/polyaxon/polyaxon-examples/blob/master/in_cluster/pytorch/mnist/polyaxonfile_distributed.yml)\n * [Horovod distributed example](https://github.com/polyaxon/polyaxon-examples/blob/master/in_cluster/horovod/tensorflow/polyaxonfile.yml)\n * [MPI distributed example](https://github.com/polyaxon/polyaxon-examples/blob/master/in_cluster/mpijob/polyaxonfile.yaml)\n\n# Hyperparameters tuning\n\nPolyaxon has a concept for suggesting hyperparameters and managing their results very similar to Google Vizier called experiment groups.\nAn experiment group in Polyaxon defines a search algorithm, a search space, and a model to train.\n\n * [Grid search](https://github.com/polyaxon/polyaxon-quick-start/blob/master/polyaxonfile_hyperparams_grid.yml)\n * [Random search with early stopping](https://github.com/polyaxon/polyaxon-quick-start/blob/master/polyaxonfile_hyperparams_early_stopping.yml)\n * [Hyperband](https://github.com/polyaxon/polyaxon-quick-start/blob/master/polyaxonfile_hyperparams_hyperband.yml)\n * [Bayesian Optimization](https://github.com/polyaxon/polyaxon-quick-start/blob/master/polyaxonfile_hyperparams_bo.yml)\n\n# Architecture\n\n![Polyaxon architecture](artifacts/polyaxon_architecture.png)\n\n#
