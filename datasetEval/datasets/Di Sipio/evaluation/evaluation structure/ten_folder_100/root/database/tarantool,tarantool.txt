development discussions https://lists.tarantool.org/mailman/listinfo/tarantool-discussions/\n\nTarantool is an in-memory database and application server.\n\nKey features of the application server:\n * 100% compatible drop-in replacement for Lua 5.1,\n   based on LuaJIT 2.1.\n   Simply use `#!/usr/bin/tarantool` instead of\n   `#!/usr/bin/lua` in your script.\n * full support for Lua modules and a rich set of\n   own modules, including cooperative multitasking,\n   non-blocking I/O, access to external databases, etc\n\nKey features of the database:\n * ANSI SQL, including views, joins, referential\n   and check constraints\n * MsgPack data format and MsgPack based\n   client-server protocol\n * two data engines: 100% in-memory with\n   optional persistence and an own implementation of LSM-tree, \n   to use with large data sets\n * multiple index types: HASH, TREE, RTREE, BITSET\n * asynchronous master-master replication\n * authentication and access control\n * the database is just a C extension to the\n   application server and can be turned off\n\nSupported platforms are Linux/x86 and FreeBSD/x86, Mac OS X.\n\nTarantool is ideal for data-enriched components of\nscalable Web architecture: queue servers, caches,\nstateful Web applications.\n\nTo download and install Tarantool as a binary package for your OS, please visit\nhttps://tarantool.io/en/download/.\n\nTo build
