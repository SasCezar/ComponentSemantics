options\n\n- `--packageManager [npm|yarn]`: specify whether to use [Yarn](https://yarnpkg.com) or NPM as your package manager\n (default is NPM).\n You can also use the environment variable `NGX_PACKAGE_MANAGER` to set this option globally.\n- `--automate <json_file>`: automate prompt answers using specified JSON file (see\n  [here](https://github.com/ngx-rocket/generator-ngx-rocket/tree/master/scripts/tests) for examples).\n- `--addons <addon_name> [<addon_name>] ...`: space-separated list of add-ons to use.\n- `--no-update`: do no update existing project (see also [updating generated projects](#updating-generated-projects)).\n- `--no-analytics`: do not report anonymous usage analytics.\n  You can also use the environment variable `NGX_DISABLE_ANALYTICS` to set this option globally.\n- `--external-chrome`: use external Chrome executable and avoid downloading a Chromium binary via\n  [puppeteer](https://github.com/GoogleChrome/puppeteer) to run unit tests.\n  **Note:** You need a Chrome version `>= 59` that can run in\n  [headless mode](https://developers.google.com/web/updates/2017/04/headless-chrome) or you will have errors.\n- `--raw`: do not use any UI library for templates.\n- `--tools`: generate only the toolchain, without application template.\n- `--location-strategy [hash|path]`: [location strategy](https://angular.io/api/common/LocationStrategy) to use in\n  Angular router (default is `path`).\n- `--no-git`: do not initialize git repository.\n- `--strict`: enable all TypeScript strict type checking options.\n- `--skip-quickstart`: disable quick start message after project generation.\n  \nWhen generating a *fullstack* project (with both client and server code), you can use the environment variables\n`NGX_CLIENT_PATH` and `NGX_SERVER_PATH` to customize the paths for client and server code. Be aware though that some\nadd-ons may force specific paths that will preempt your changes.\n\n## Updating generated projects\n\nAs new features and newer libraries and tools are added to the generator, you may want to update your project at some\npoint. Here is how we suggest you to do it:\n\n1. Make sure your working directory is clean (no pending / uncommited changes).\n2. Run `ngx update` using the [CLI](cli/README) inside your project folder.\n3. The generator will then run again using the same options you used initially, prompting you for each file change.\n   From there the recommended approach is simply to overwrite everything.\n4. Finally, use your source control to see the diff for each file and merge the changes manually.\n\n> Note: you can use the `--tools` option to generate only the toolchain and not application templates, thus reducing\n> the number of changes to merge.\n\n## Customize project generation\n\nYou can customize the generator output to change or enhance it to better suit your needs using\n[add-ons](https://www.npmjs.com/search?q=ngx-rocket-addon).\n\nTo create a new add-on, you can use the `addon` sub-generator of the [CLI](https://github.com/ngx-rocket/generator-ngx-rocket/tree/master/cli):\n```sh\nngx new --addon\n```\n\nSee the [add-on generator documentation](https://github.com/ngx-rocket/generator-ngx-rocket/tree/master/generators/addon)\nfor more information about add-on creation.\n\n# Contributing\n\nSee [contributing guide](CONTRIBUTING.md)\n\n# License\n\n[MIT](LICENSE)\n"
