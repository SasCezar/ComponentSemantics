Since the whole source package\nis under development, you might even encounter unresolved symbols etc. that\nhalt the compiler immediately. If such a thing occurs, you should do a "make\ndistclean" on qlcplus (top-most source directory) and then "qmake" and "make"\nagain. We attempt to keep the GIT master free of fatal errors and it should\ncompile all the time. However, some inter-object dependencies do get mixed up\nsometimes and you need to compile the whole package instead of just the latest\nchanges. Sometimes even that doesn\'t work, because QLC+ installs its common\nlibraries to system directories, where (at least unixes) fetch them instead\nof the source directory. In those cases, you might try going to the libs\ndirectory, compile it with "make" and install with "make install" and then\nattempt to re-compile the whole package with "make".\n\nApache 2.0
