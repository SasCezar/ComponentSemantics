Usage\n\nExample commands:\n\n- Capture with GUI:\n\n    ```shell\n    flameshot gui\n    ```\n\n- Capture with GUI with custom save path:\n\n    ```shell\n    flameshot gui -p ~/myStuff/captures\n    ```\n\n- Capture with GUI after 2 seconds delay (can be useful to take screenshots of mouse hover tooltips, etc.):\n\n    ```shell\n    flameshot gui -d 2000\n    ```\n\n- Fullscreen capture with custom save path (no GUI) and delayed:\n\n    ```shell\n    flameshot full -p ~/myStuff/captures -d 5000\n    ```\n\n- Fullscreen capture with custom save path copying to clipboard:\n\n    ```shell\n    flameshot full -c -p ~/myStuff/captures\n    ```\n\n- Capture the screen containing the mouse and print the image (bytes) in PNG format:\n\n    ```shell\n    flameshot screen -r\n    ```\n\n- Capture the screen number 1 and copy it to the clipboard:\n\n    ```shell\n    flameshot screen -n 1 -c\n    ```\n\nIn case of doubt choose the first or the second command as shortcut in your favorite desktop environment.\n\nA systray icon will be in your system\'s panel while Flameshot is running.\nDo a right click on the tray icon and you\'ll see some menu items to open the configuration window and the information window.\nCheck out the information window to see all the available shortcuts in the graphical capture mode.\n\n### CLI configuration\n\nYou can use the graphical menu to configure Flameshot, but alternatively you can use your terminal or scripts to do so.\n\n- Open the configuration menu:\n\n    ```shell\n    flameshot config\n    ```\n\n- Show the initial help message in the capture mode:\n\n    ```shell\n    flameshot config --showhelp true\n    ```\n\n- For more information about the available options use the help flag:\n\n    ```shell\n    flameshot config -h\n    ```\n\n## Keyboard shortcuts\n\n### Local\n\nThese shortcuts are available in GUI mode:\n\n|  Keys                                                                     |  Description                          |\n| <kbd>Ctrl</kbd> + <kbd>Z</kbd>                                            | Undo the last modification                                     |\n| <kbd>Spacebar</kbd> Flameshot as a default screenshot utility, chances are you want to launch it using the <kbd>Prt Sc</kbd> key. Flameshot doesn\'t yet offer a fully-automated option to do so, but you can configure your system to do so.\n\n#### On KDE Plasma desktop\n\nTo make configuration easier, there\'s a [file](docs/shortcuts-config/flameshot-shortcuts-kde) in the repository that more or less automates this process. This file will assign the following keys to the following actions by default:\n\n|  Keys                                                  |  Description                                                                       |\n|---                                                     |---                                                                                 |\n| <kbd>Prt Sc</kbd>                                      | Start the Flameshot screenshot tool and take a screenshot                          |\n| <kbd>Ctrl</kbd> + <kbd>Prt Sc</kbd>                    | Wait for 3 seconds, then start the Flameshot screenshot tool and take a screenshot |\n| <kbd>Shift</kbd> + <kbd>Prt Sc</kbd>                   | Take a full-screen (all monitors) screenshot and save it                           |\n| <kbd>Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>Prt Sc</kbd> | Take a full-screen (all monitors) screenshot and copy it to the clipboard          |\n\nIf you don\'t like the defaults, you can change them manually later.\n\nSteps for using the configuration:\n\n1. The configuration file configures shortcuts so that Flameshot automatically saves (without opening the save dialog) screenshots to `~/Pictures/Screenshots` folder. Make sure you have that folder by running the following command:\n\n    ```shell\n    mkdir -p ~/Pictures/Screenshots\n    ```\n\n    (If you don\'t like the default location, you can skip this step and configure your preferred directory later.)\n2. Download the configuration file:\n\n    ```shell\n    cd ~/Desktop\n    wget https://raw.githubusercontent.com/lupoDharkael/flameshot/master/docs/shortcuts-config/flameshot-shortcuts-kde\n    ```\n3. Go to _System Settings_ \xe2\x86\x92 _Shortcuts_ \xe2\x86\x92 _Custom Shortcuts_.\n4. If there\'s one, you\'ll need to disable an entry for Spectacle, the default KDE screenshot utility first because its shortcuts might collide with Flameshot\'s ones; so, just uncheck the _Spectacle_ entry.\n5. Click _Edit_ \xe2\x86\x92 _Import..._, navigate to the Desktop folder (or wherever you saved the configuration file) and open the configuration file.\n6. Now the Flameshot entry should appear in the list. Click _Apply_ to apply the changes.\n7. If you want to change the defaults, you can expand the entry, select the appropriate action and modify it as you wish; the process is pretty mush self-explanatory.\n\n## Considerations\n\n- Experimental Gnome Wayland and Plasma Wayland support.\n\n- If you are using Gnome you need to install the [Gnome Shell Extension Appindicator](https://github.com/Ubuntu/gnome-shell-extension-appindicator) extension in order to see the systemtray icon.\n\n- In order to speed up the first launch of Flameshot (DBus init of the app can be slow), consider starting the application automatically on boot.\n\n- Press <kbd>Enter</kbd> or <kbd>Ctrl</kbd> + <kbd>C</kbd> when you are in a capture mode and you don\'t have an active selection and the whole desktop will be copied to your clipboard! Pressing <kbd>Ctrl</kbd> + <kbd>S</kbd> will save your capture in a file! Check the [Shortcuts](#shortcuts) for more information.\n\n- Execute the command `flameshot` without parameters to launch a running instance of the program without taking actions.\n\n## distribution, see [Dependencies](#dependencies) below for more information.\n\n### Dependencies\n\n#### Compile-time\n\n- Qt >= 5.3\n  + Development tools\n- GCC >= 4.9.2\n\n#### Run-time\n\n- Qt\n  + SVG\n\n#### Optional\n\n- Git\n- OpenSSL\n- CA Certificates\n\n#### Debian\n\n```shell\n# Compile-time\napt install g++ build-essential qt5-default qt5-qmake qttools5-dev-tools\n\n# Run-time\napt install libqt5dbus5 libqt5network5 libqt5core5a libqt5widgets5 libqt5gui5 libqt5svg5-dev\n\n# Optional\napt install git openssl ca-certificates\n```\n\n#### Fedora\n\n```shell\n# Compile-time\ndnf install gcc-c++ qt5-devel qt5-qtbase-devel qt5-linguist\n\n# Run-time\ndnf install qt5-qtbase qt5-qtsvg-devel\n\n# Optional\ndnf install git openssl ca-certificates\n```\n\n#### Arch\n\n```shell\n# Compile-time\npacman -S base-devel git qt5-base qt5-tools\n\n# Run-time\npacman -S qt5-svg\n\n# Optional\npacman -S openssl ca-certificates\n```\n\n### Build\n\nAfter installing all the dependencies, finally run the following commands in the sources root directory:\n\n```shell\nmkdir build\ncd build\nqmake ../\nmake\n```\n\n### Install\n\nSimply use `make install` with privileges.\n\n## Packaging\n\nHaving `git` installed is required if you\'re building Flameshot from a snapshot to have precise version information.\n\nIn order to generate the instructions in the `Makefile` to install the application in `/usr` instead of in `/usr/local` you can pass the `packaging` option to `qmake` (`qmake CONFIG+=packaging`).\n\nIf you want to install in a custom directory you can use the `INSTALL_ROOT` variable.\n\n**Example**:\n\nIf you want to install Flameshot in `~/myBuilds/test`, you can execute the following to do so:\n\n```shell\nqmake CONFIG+=packaging\nmake INSTALL_ROOT=~/myBuilds/test install\n```\n\n## License\n- The main code is licensed under [GPLv3](LICENSE)\n- The logo of Flameshot is licensed under [Free Art License
