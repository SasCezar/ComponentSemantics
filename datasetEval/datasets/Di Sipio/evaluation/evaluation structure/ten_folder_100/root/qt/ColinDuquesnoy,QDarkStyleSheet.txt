b'# QDarkStylesheet\n\n[![Build Status](https://travis-ci.org/ColinDuquesnoy/QDarkStyleSheet.png?branch=master)](https://travis-ci.org/ColinDuquesnoy/QDarkStyleSheet)\n[![Docs Status](https://readthedocs.org/projects/qdarkstylesheet/badge/?version=latest&style=flat)](https://qdarkstylesheet.readthedocs.io)\n[![Latest PyPI version](https://img.shields.io/pypi/v/QDarkStyle.svg)](https://pypi.python.org/pypi/QDarkStyle)\n[![License: MIT](https://img.shields.io/dub/l/vibe-d.svg?color=lightgrey)](https://opensource.org/licenses/MIT)\n[![License: CC BY 4.0](https://img.shields.io/badge/License-CC%20BY%204.0-lightgrey.svg)](https://creativecommons.org/licenses/by/4.0/)\n[![conduct](https://img.shields.io/badge/code%20of%20conduct-contributor%20covenant-green.svg?style=flat&color=lightgrey)](http://contributor-covenant.org/version/1/4/)\n\nA dark stylesheet for Qt applications (Qt4, Qt5, PySide, PySide2, PyQt4, PyQt5, QtPy,\nPyQtGraph).\n\n## Installation\n\n### Python\n\nFrom PyPI: Get the latest stable version of `qdarkstyle` package\nusing _pip_ (preferable):\n\n```bash\npip install qdarkstyle\n```\n\nFrom code: Download/clone the project, go to `qdarkstyle` folder then:\n\n- You can use the _setup_ script and pip install.\n\n  ```bash\n  pip install .\n  ```\n\n- Or, you can use the _setup_ script with Python:\n  ```bash\n  python setup.py install\n  ```\n\n### C++\n\n- Download/clone the project and copy the following files to your application\n  directory (keep the existing directory hierarchy):\n\n  - **qdarkstyle/style.qss**\n  - **qdarkstyle/style.qrc**\n  - **qdarkstyle/rc/** (the whole directory)\n\n- Add **qdarkstyle/style.qrc** to your **.pro file** as follows:\n\n  ```cpp\n  RESOURCES += qdarkstyle/style.qrc\n  ```\n\n- Load the stylesheet:\n\n  ```cpp\n  QFile f(":qdarkstyle/style.qss");\n  if (!f.exists())\n  {\n      printf("Unable to set stylesheet, file not found\\n");\n  }\n  else\n  {\n      f.open(QFile::ReadOnly | QFile::Text);\n      QTextStream ts(&f);\n      qApp->setStyleSheet(ts.readAll());\n  }\n  ```\n\n_Note: The ":" in the file name is necessary to define that file as a resource library. For more information see the discussion [here](https://github.com/ColinDuquesnoy/QDarkStyleSheet/pull/87)._\n\n## Usage\n\nHere is an example using PySide\n\n```Python\nimport sys\nimport qdarkstyle\nfrom PySide import QtGui\n\n# create the application and the main window\napp = QtGui.QApplication(sys.argv)\nwindow = QtGui.QMainWindow()\n\n# setup stylesheet\napp.setStyleSheet(qdarkstyle.load_stylesheet_pyside())\n\n# run\nwindow.show()\napp.exec_()\n```\n\nTo use another wrapper for Qt, you need to replace some lines.\nSee examples below.\n\nTo use PyQt4, change two lines\n\n```Python\nfrom PySide import QtGui\napp.setStyleSheet(qdarkstyle.load_stylesheet_pyqt())\n```\n\nIf PyQt5, more lines need to be changed because of its API,\nsee the complete example\n\n```Python\nimport sys\nimport qdarkstyle\nfrom PyQt5 import QtWidgets\n\n# create the application and the main window\napp = QtWidgets.QApplication(sys.argv)\nwindow = QtWidgets.QMainWindow()\n\n# setup stylesheet\napp.setStyleSheet(qdarkstyle.load_stylesheet_pyqt5())\n\n# run\nwindow.show()\napp.exec_()\n```\n\nIf your project uses QtPy or you need to set it programmatically,\nit is far more simple\n\n```Python\nimport sys\nimport qdarkstyle\nimport os\n\n# set the environment variable to use a specific wrapper\n# it can be set to pyqt, pyqt5, pyside or pyside2 (not implemented yet)\n# you do not need to use QtPy to set this variable\nos.environ[\'QT_API\'] = \'pyqt\'\n\n# import from QtPy instead of doing it directly\n# note that QtPy always uses PyQt5 API\nfrom qtpy import QtWidgets\n\n# create the application and the main window\napp = QtWidgets.QApplication(sys.argv)\nwindow = QtWidgets.QMainWindow()\n\n# setup stylesheet\napp.setStyleSheet(qdarkstyle.load_stylesheet_from_environment())\n\n# run\nwindow.show()\napp.exec_()\n```\n\nIt is also simple if you use PyQtGraph\n\n```Python\nimport sys\nimport qdarkstyle\nimport os\n\n# set the environment variable to use a specific wrapper\n# it can be set to PyQt, PyQt5, PySide or PySide2 (not implemented yet)\nos.environ[\'PYQTGRAPH_QT_LIB\'] = \'PyQt\'\n\n# import from pyqtgraph instead of doing it directly\n# note that PyQtGraph always uses PyQt4 API\nfrom pyqtgraph.Qt import QtGui\n\n# create the application and the main window\napp = QtGui.QApplication(sys.argv)\nwindow = QtGui.QMainWindow()\n\n# setup stylesheet\napp.setStyleSheet(qdarkstyle.load_stylesheet_from_environment(is_pyqtgraph=True))\n\n# run\nwindow.show()\napp.exec_()\n```\n\nIf you are using Qt.py, which is different from qtpy, you should install\nqtpy then set both to the same binding.\n\n_There is an example included in the *example* folder.\nYou can run the script without installing qdarkstyle. You only need to have\nPySide or PySide2 or PyQt4 or PyQt5 installed on your system._\n\n## What is new?\n\nIn the version 2.6 and later, a reestructure stylesheet is provided.\nThe palette has only 9 colors. Most widgets are revised and their styles\nwere improved. We also provide a command line (script) to get info that\ncould be used when opening issues.
