Its goal is making an advanced, customizable and FOSS reverse-engineering platform while keeping the user experience at mind. Cutter is created by reverse engineers for reverse engineers.  \n\n[![Build Status](https://travis-ci.com/radareorg/cutter.svg?branch=master)](https://travis-ci.com/radareorg/cutter)\n[![Build status](https://ci.appveyor.com/api/projects/status/s9rkx1dn3uy4bfdx/branch/master?svg=true)](https://ci.appveyor.com/project/radareorg/cutter/branch/master)\n[![Total alerts](https://img.shields.io/lgtm/alerts/g/radareorg/cutter.svg?logo=lgtm&logoWidth=18)](https://lgtm.com/projects/g/radareorg/cutter/alerts/)\n\n![Screenshot](https://raw.githubusercontent.com/radareorg/cutter/master/docs/source/images/screenshot.png)\n\n## Learn more at [https://cutter.re](https://cutter.re).\n\n## Downloading a release\n\nCutter is available for all platforms (Linux, macOS, Windows).\nYou can download the latest release [here](https://github.com/radareorg/cutter/releases).\n * macOS: Download the latest `.dmg` file or use [Homebrew Cask](https://github.com/Homebrew/homebrew-cask) `brew cask install cutter`.\n * Windows: Download the latest Zip archive.\n * Linux: Download the latest AppImage file. Then just make it executable and run it:\n   * `chmod +x <appimage_file>`\n   * `./<appimage_file>`\n\n## Building from sources\n\nTo build Cutter on your local machine, please follow this guide: [Building from source](https://cutter.re/docs/building.html)\n\n## Docker\n\nTo deploy *cutter* using a pre-built `Dockerfile`, it\'s possible to use the [provided configuration](docker). The corresponding `README.md` file also contains instructions on how to get started using the docker image with minimal effort.\n\n## Documentation\n\nYou can find our documentation in our [website](https://cutter.re/docs/).\n\n## Plugins\nCutter supports both Python and Native C++ plugins. Want to extend Cutter with Plugins?
