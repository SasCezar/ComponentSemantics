b'Retire.js\n=========\n\n[![Retire Status](http://retire.insecurity.today/api/image?uri=https://raw.githubusercontent.com/RetireJS/retire.js/master/node/package.json)](http://retire.insecurity.today/api/image?uri=https://raw.githubusercontent.com/RetireJS/retire.js/master/node/package.json)\n\n#### What you require you must also retire\n\nThere is a plethora of JavaScript libraries for use on the Web and in Node.JS apps out there. This greatly simplifies development,but we need to stay up-to-date on security fixes. "Using Components with Known Vulnerabilities" is now a part of the [OWASP Top 10](https://www.owasp.org/index.php/Top_10_2013-A9-Using_Components_with_Known_Vulnerabilities) list of security risks and insecure libraries can pose a huge risk to your Web app. The goal of Retire.js is to help you detect the use of JS-library versions with known vulnerabilities.\n\nRetire.js can be used in many ways:\n\n1. [As  command line scanner](https://github.com/RetireJS/retire.js/tree/master/node)\n2. [As a grunt plugin](https://github.com/bekk/grunt-retire)\n2. [As a gulp task](#user-content-gulp-task)\n3. [As a Chrome extension](https://github.com/RetireJS/retire.js/tree/master/chrome)\n4. [As a Firefox extension](https://github.com/RetireJS/retire.js/tree/master/firefox)\n5. [As a Burp and OWASP Zap plugin](https://github.com/h3xstream/burp-retire-js)\n\nCommand line scanner\n--------------------\nScan a web app or node app for use of vulnerable JavaScript libraries and/or Node.JS modules. In the source code folder of the application folder run:\n```\n$ npm install -g retire\n$ retire\n```\n\nGrunt plugin\n------------\nA [Grunt task for running Retire.js](https://github.com/bekk/grunt-retire) as part of your application\'s build routine, or some other automated workflow.\n\nGulp task\n---------\nAn example of a Gulp task which can be used in your gulpfile to watch and scan your project files automatically. You can modify the watch patterns and (optional) Retire.js options as you like.\n\n```javascript\nconst c = require(\'ansi-colors\');\n\nvar gulp = require(\'gulp\');\nvar beeper = require(\'beeper\');\nvar log = require(\'fancy-log\');\nvar spawn = require(\'child_process\').spawn;\n\ngulp.task(\'retire:watch\', [\'retire\'], function (done) {\n    // Watch all javascript files and package.json\n    gulp.watch([\'js/**/*.js\', \'package.json\'], [\'retire\']);\n});\n\ngulp.task(\'retire\', function() {\n    // Spawn Retire.js as a child process\n    // You can optionally add option parameters to the second argument (array)\n    var child = spawn(\'retire\', [], {cwd: process.cwd()});\n    \n    child.stdout.setEncoding(\'utf8\');\n    child.stdout.on(\'data\', function (data) {\n        log(data);\n    });\n\n    child.stderr.setEncoding(\'utf8\');\n    child.stderr.on(\'data\', function (data) {\n        log(c.red(data));\n        beeper();\n    });\n});\n\n```\n\nChrome and firefox extensions \n-------------\nScans visited sites for references to insecure libraries, and puts warnings in the developer console. An icon on the address bar displays will also indicate if vulnerable libraries were loaded.\n\n\nBurp and OWASP ZAP plugin\n-------------------------\n[@h3xstream](https://github.com/h3xstream) has adapted Retire.js as a [plugin](https://github.com/h3xstream/burp-retire-js) for the penetration testing tools [Burp](https://portswigger.net/burp/) and [OWASP ZAP](https://www.owasp.org/index.php/OWASP_Zed_Attack_Proxy_Project).
