b'dirsearch\n=========\n\nCurrent Release: v0.3.9 (2019.11.26)\n\n\nOverview\n--------\ndirsearch is a simple command line tool designed to brute force directories and files in websites.\n\n\nInstallation & Usage\n------------\n\n```\ngit clone https://github.com/maurosoria/dirsearch.git\ncd dirsearch\npython3 dirsearch.py -u <URL> -e <EXTENSION>\n```\n\nyou can also use this alias to send directly to proxy\n`python3 /path/to/dirsearch/dirsearch.py --http-proxy=localhost:8080`\n\n\nOptions\n-------\n\n\n```\nOptions:\n  -h, --help            show this help message and exit\n\n  Mandatory:\n    -u URL, --url=URL   URL target\n    -L URLLIST, --url-list=URLLIST\n                        URL list target\n    -e EXTENSIONS, --extensions=EXTENSIONS\n                        Extension list separated by comma (Example: php,asp)\n    -E, --extensions-list\n                        Use predefined list of common extensions\n\n  Dictionary Settings:\n    -w WORDLIST, --wordlist=WORDLIST\n    -l, --lowercase\n    -f, --force-extensions\n                        Force extensions for every wordlist entry (like in\n                        DirBuster)\n\n  General Settings:\n    -s DELAY, --delay=DELAY\n                        Delay between requests (float number)\n    -r, --recursive     Bruteforce recursively\n    -R RECURSIVE_LEVEL_MAX, --recursive-level-max=RECURSIVE_LEVEL_MAX\n                        Max recursion level (subdirs) (Default: 1 [only\n                        rootdir + 1 dir])\n    --suppress-empty, --suppress-empty\n    --scan-subdir=SCANSUBDIRS, --scan-subdirs=SCANSUBDIRS\n                        Scan subdirectories of the given -u|--url (separated\n                        by comma)\n    --exclude-subdir=EXCLUDESUBDIRS, --exclude-subdirs=EXCLUDESUBDIRS\n                        Exclude the following subdirectories during recursive\n                        scan (separated by comma)\n    -t THREADSCOUNT, --threads=THREADSCOUNT\n                        Number of Threads\n    -x EXCLUDESTATUSCODES, --exclude-status=EXCLUDESTATUSCODES\n                        Exclude status code, separated by comma (example: 301,\n                        500)\n    --exclude-texts=EXCLUDETEXTS\n                        Exclude responses by texts, separated by comma\n                        (example: "Not found", "Error")\n    --exclude-regexps=EXCLUDEREGEXPS\n                        Exclude responses by regexps, separated by comma\n                        (example: "Not foun[a-z]{1}", "^Error$")\n    -c COOKIE, --cookie=COOKIE\n    --ua=USERAGENT, --user-agent=USERAGENT\n    -F, --follow-redirects\n    -H HEADERS, --header=HEADERS\n                        Headers to add (example: --header "Referer:\n                        example.com" --header "User-Agent: IE"\n    --random-agents, --random-user-agents\n\n  Connection Settings:\n    --timeout=TIMEOUT   Connection timeout\n    --ip=IP             Resolve name to IP address\n    --proxy=HTTPPROXY, --http-proxy=HTTPPROXY\n                        Http Proxy (example: localhost:8080\n    --http-method=HTTPMETHOD\n                        Method to use, default: GET, possible also: HEAD;POST\n    --max-retries=MAXRETRIES\n    -b, --request-by-hostname\n                        By default dirsearch will request by IP for speed.\n                        This forces requests by hostname\n\n  Reports:\n    --simple-report=SIMPLEOUTPUTFILE\n                        Only found paths\n    --plain-text-report=PLAINTEXTOUTPUTFILE\n                        Found paths with status codes\n    --json-report=JSONOUTPUTFILE\n```\n\n\nOperating Systems supported\n---------------------------\n- Windows XP/7/8/10\n- GNU/Linux\n- MacOSX\n\nFeatures\n--------\n- Multithreaded\n- Keep alive connections\n- Support for multiple extensions (-e|--extensions asp,php)\n- Reporting (plain text, JSON)\n- Heuristically detects invalid web pages\n- Recursive brute forcing\n- HTTP proxy support\n- User agent randomization\n- Batch processing\n- Request delaying\n\nAbout wordlists\n---------------\nDictionaries must be text files. Each line will be processed as such, except that the special word %EXT% is used, which will generate one entry for each extension (-e | --extension) passed as an argument.\n\nExample:\n- example/\n- example.%EXT%\n\nPassing the extensions "asp" and "aspx" will generate the following dictionary:\n- example/\n- example.asp\n- example.aspx\n\nYou can also use -f | --force-extensions switch to append extensions to every word in the wordlists (like DirBuster).\n\n## Support Docker\n### Install Docker Linux\nInstall Docker\n```sh\ncurl -fsSL https://get.docker.com | bash\n```\n> To use docker you need superuser power\n\n### Build Image dirsearch\nTo create image\n```sh\ndocker build -t "dirsearch:v0.3.8" .\n```\n> **dirsearch** this is name the image and **v0.3.8** is version\n\n### Using dirsearch\nFor using\n```sh\ndocker run -it --rm "dirsearch:v0.3.8" -u target -e php,html,png,js,jpg\n```\n> target is the site or IP\n\nLicense\n-------\nCopyright (C) Mauro Soria (maurosoria at gmail dot com)\n\nLicense: GNU General Public License, version 2\n\n\nContributors\n---------\nSpecial thanks for these people.\n\n- mzfr\n- Damian89\n- Bo0oM\n- liamosaur\n- redshark1802\n- SUHAR1K\n- FireFart\n- k2l8m11n2\n- vlohacks\n- r0p0s3c\n'
