The aim is help the developer teams to use **Git** and **Scrum** on task management of the day-to-day.\n\nAvailable in English, Chinese, Russian, German, Arabic, Spanish, Portuguese, Italian, French, Indonesian, Serbian, Hungarian and Vietnamese.\n\n\nLaravel GitScrum in gitter.im : [https://gitter.im/laravel-gitscrum/Lobby](https://gitter.im/laravel-gitscrum/Lobby?utm_source=share-link&utm_medium=link&utm_campaign=share-link)\n\n### Features\n\nGitScrum can be integrated with **Github** or **Gitlab** or **Bitbucket**.\n\n- **Product Backlog** contains the Product Owner\'s assessment of business value\n\n- **User Story** is a description consisting of one or more sentences in the everyday or business language that captures what a user does or needs to do as part of his or her job function.\n\n\t**Features**: Acceptance criteria, prioritization using MoSCoW, definition of done checklist, pie chart, assign labels, team members, activities, comments and issues.\n\n- **Sprint Backlog** is the property of the development team and all included estimates are provided by development team. Often an accompanying sprint planning is the board used to see and change state of the issues.\n\n\t**Features**: Sprint planning using Kanban board, burndown chart, definition of done checklist, effort, attachments, activities, comments and issues.\n\n- **Issue** is added in user story to one sprint backlog, or directly in sprint backlog. Generally, each issue should be small enough to be easily completed within a single day.\n\n\t**Features**: Progress state (e.g. to do, in progress, done or archived), issue type (e.g. Improvement, Support Request, Feedback, Customer Problem, UX, Infrastructure, Testing Task, etc...), definition of done checklist, assign labels, effort, attachments, comments, activities, team members.\n\n![GitScrum Dashboard](https://i.imgur.com/xFFSha7.png)\n\n![GitScrum Board](https://i.imgur.com/ppiYPMK.png)\n\n![GitScrum Sprint](https://i.imgur.com/ZHrhKPy.png)\n\n## Installation\n\nThe requirements to Laravel GitScrum application is:\n\n- **PHP - Supported Versions**: >= 7.1\n- **Webserver**: Nginx or Apache\n- **Database**: MySQL, or Maria DB\n\n[**Use Docker** - Containers: php7, nginx and mysql57](https://github.com/renatomarinho/Docker-GitScrum)\n\n### Composer Package\n\n```\n$ composer create-project gitscrum-community-edition/laravel-gitscrum --stability=stable --keep-vcs\n$ cd laravel-gitscrum\n```\n**Important**: If you have not yet installed composer: [Installation - Linux / Unix / OSX](https://getcomposer.org/doc/00-intro.md#installation-linux-unix-osx)\n\n\n### Git Clone\n\n```\n$ git clone git@github.com:GitScrum-Community/laravel-gitscrum.git\n$ cd laravel-gitscrum\n$ composer update\n$ composer run-script post-root-package-install\n```\n\n\n## Setup\n\n**Important**: If you have not the .env file in root folder, you must copy or rename the .env.example to .env\n\n#### Application URL\n\n.env file\n\n```\nAPP_URL=http://yourdomain.tld (you must use protocol http or https)\n```\n\n#### Language\n\nOptions: en | zh | zh_cn | ru | de | es | pt | it | id | fr | hu\n\n.env file\n\n```\nAPP_LANG=en\n```\nCan you help us translate a few phrases into different languages? See: https://github.com/GitScrum-Community/laravel-gitscrum/tree/feature/language-pack/resources/lang\n\n\n#### Database\n\n.env file\n\n```\n\nDB_CONNECTION=mysql\nDB_HOST=XXXXXX\nDB_PORT=3306\nDB_DATABASE=XXXXX\nDB_USERNAME=XXXX\nDB_PASSWORD=XXXXX\n```\n\n**Remember**: Create the database for GitScrum before run artisan command.\n\n```\nphp artisan migrate\nphp artisan db:seed --class=SettingSeeder\n```\n\n#### Github\n\nYou must create a new Github App, visit [GitHub\'s New OAuth Application page](https://github.com/settings/applications/new), fill out the form, and grab your Client ID and Secret.\n\n```\nApplication name: gitscrum\nHomepage URL: URL (Same as APP_URL at .env)\nApplication description: gitscrum\nAuthorization callback URL: http://{URL is the SAME APP_URL}/auth/provider/github/callback\n```\n\n.env file\n\n```\nGITHUB_CLIENT_ID=XXXXX\nGITHUB_CLIENT_SECRET=XXXXXXXXXXXXXXXXXX\n```\n\n#### Gitlab\n\nYou must create a new Gitlab App, visit [Gitlab new application](https://gitlab.com/profile/applications), fill out the form, and grab your Application ID and Secret.\n\n```\nname: gitscrum\nRedirect URI: http://{URL is the SAME APP_URL}/auth/provider/gitlab/callback\nScopes: api and read_user\n```\n\n.env file\n\n```\nGITLAB_KEY=XXXXX -> Application Id\nGITLAB_SECRET=XXXXXXXXXXXXXXXXXX\nGITLAB_INSTANCE_URI=https://gitlab.com/\n```\n\n#### Bitbucket\n\nYou must create a new Bitbucket OAuth Consumer, visit [Bitbucket new consumer guide](https://confluence.atlassian.com/bitbucket/integrate-another-application-through-oauth-372605388.html), and make sure you give write permissions when creating the consumer specially on (repositories , issues)\n\n```\nname: gitscrum\nCallback URL: http://{URL is the SAME APP_URL}/auth/provider/bitbucket/callback\nURL: http://{URL is the SAME APP_URL}\nUncheck (This is a private consumer)\n```\n\n.env file\n\n```\nBITBUCKET_CLIENT_ID=XXXXX -> Bitbucket Key\nBITBUCKET_CLIENT_SECRET=XXXXXXXXXXXXXXXXXX Bitbucket Secret\n```\n\n#### Proxy\n\n.env file\n\n```\nPROXY_PORT=\nPROXY_METHOD=\nPROXY_SERVER=\nPROXY_USER=\nPROXY_PASS=\n```\n\n\n## Screens\n\n![Screenshot 0](http://i.imgur.com/jejT8hY.png)\n![Screenshot
