Inspired by Micheal Mand\'s [Laravel 4 log viewer](https://github.com/mikemand/logviewer) (works only with laravel 4.1)\n\nWhat ?\n------\nSmall log viewer for laravel. Looks like this:\n\n![capture d ecran 2014-12-01 a 10 37 18](https://cloud.githubusercontent.com/assets/1575946/5243642/8a00b83a-7946-11e4-8bad-5c705f328bcc.png)\n\nInstall (Laravel)\n-----------------\nInstall via composer\n```\ncomposer require rap2hpoutre/laravel-log-viewer\n```\n\nAdd Service Provider to `config/app.php` in `providers` section\n```php\nRap2hpoutre\\LaravelLogViewer\\LaravelLogViewerServiceProvider::class,\n```\n\nAdd a route in your web routes file:\n```php \nRoute::get(\'logs\', \'\\Rap2hpoutre\\LaravelLogViewer\\LogViewerController@index\');\n```\n\nGo to `http://myapp/logs` or some other route\n\n**Optionally** publish `log.blade.php` into `/resources/views/vendor/laravel-log-viewer/` for view customization:\n\n```\nphp artisan vendor:publish \\\n  --provider="Rap2hpoutre\\LaravelLogViewer\\LaravelLogViewerServiceProvider" \\\n  --tag=views\n``` \n\nInstall (Lumen)\n---------------\n\nInstall via composer\n```\ncomposer require rap2hpoutre/laravel-log-viewer\n```\n\nAdd the following in `bootstrap/app.php`:\n```php\n$app->register(\\Rap2hpoutre\\LaravelLogViewer\\LaravelLogViewerServiceProvider::class);\n```\n\nExplicitly set the namespace in `app/Http/routes.php`:\n```php\n$router->group([\'namespace\' => \'\\Rap2hpoutre\\LaravelLogViewer\'], function() use ($router) {\n    $router->get(\'logs\', \'LogViewerController@index\');\n});\n```\n\nTroubleshooting\n---------------\n\nIf you got a `InvalidArgumentException in FileViewFinder.php` error, it may be a problem with config caching. Double check installation, then run `php artisan config:clear`.\n\n'
