Require the package using composer:\n\n    ```\n    composer require jeroennoten/laravel-adminlte\n    ```\n\n2. Updating\n\n1. To update this package, first update the composer package:\n\n    ```\n    composer update jeroennoten/laravel-adminlte\n    ```\n\n2. Then, we need to update the assets\n\n    > If you using AdminLTE for Laravel 5.x and are upgrading Laravel 6 version, delete the folder adminlte inside your public/vendor folder.\n    \n    And then use this command to publish new assets\n    \n    ```\n    php artisan adminlte:update\n    ```\n   \n3. If you have [published](#8-customize-views) and modified the default master, page views or login views, you will need to update them too.\n\n    Option 1:\n    - Make a copy of the views you modified.\n    - Publish the views again, using\n        ```\n       php artisan vendor:publish --provider="JeroenNoten\\LaravelAdminLte\\AdminLteServiceProvider" --tag=views\n        ```\n   - Redo the modifications you did.\n  \n   Option 2:\n   - Modify in the css, js and other assets location in the master and page views. \n\n## 4. Usage\n\nTo use the template, create a blade file and extend the layout with `@extends(\'adminlte::page\')`.\nThis template yields the following sections:\n\n- `title`: for in the `<title>` tag\n- `content_header`: title of the page, above the content\n- `content`: all of the page\'s content\n- `css`: extra stylesheets (located in `<head>`)\n- `js`: extra javascript (just before `</body>`)\n\nAll sections are in fact optional. Your blade template could look like the following.\n\n```html\n{{-- resources/views/admin/dashboard.blade.php --}}\n\n@extends(\'adminlte::page\')\n\n@section(\'title\', \'Dashboard\')\n\n@section(\'content_header\')\n    <h1>Dashboard</h1>\n@stop\n\n@section(\'content\')\n    <p>Welcome to this beautiful admin panel.</p>\n@stop\n\n@section(\'css\')\n    <link rel="stylesheet" href="/css/admin_custom.css">\n@stop\n\n@section(\'js\')\n    <script> console.log(\'Hi!\'); </script>\n@stop\n```\n\nYou now just return this view from your controller, as usual. Check out [AdminLTE](https://almsaeedstudio.com) to find out how to build beautiful content for your admin panel.\n\n## 5. Artisan Console Commands\n\n### 5.1 The `adminlte:install` Command\n\nYou can install all required & additional resources with the `adminlte:install` command.\n\nWithout any option it will install AdminLTE assets, config & translations.\nYou can also install the Authentication Views with adding `--type=enhanced` or additional to the Authentication Views also the Basic Views & Routes with adding `--type=full` to the `adminlte:install` command.\n\n#### 5.1.1 Options\n\n - `--force`: Overwrite existing views by default\n - `--type=`: Installation type, Available type: none, basic, enhanced & full.\n - `--only=`: Install only specific part, Available parts: assets, config, translations, auth_views, basic_views, basic_routes & main_views. This option can not used with the with option.\n - `--with=*`: Install basic assets with specific parts, Available parts: auth_views, basic_views & basic_routes. Can be use multiple\n - `--interactive` : The installation will guide you through the process\n\n\n### 5.2 The `adminlte:plugins` Command\n\nIf you won\'t use cdn for the plugins, you can manage the optional plugins assets with the `adminlte:plugins` command.\n\nYou can list all available plugins, install/update/remove all or specific plugins. Here are some examples for the command:\n\nInstall all plugin assets\n- `artisan adminlte:plugins install`\nInstall only Pace Progress & Select2 plugin assets\n- `artisan adminlte:plugins install --plugin=paceProgress --plugin=select2`\n\nUpdate all Plugin assets\n- `artisan adminlte:plugins update`\nUpdate only Pace Progress plugin assets\n- `artisan adminlte:plugins update`\n\nRemove all Plugin assets\n- `artisan adminlte:plugins remove`\nRemove only Select2 plugin assets\n- `artisan adminlte:plugins remove --plugin=select2`\n\n#### 5.2.1 Options\n\n - `operation`: Operation command, Available commands; list (default), install, update & remove.\n - `--plugin=`: Plugin Key. (Can used multiple times for the desired)\n - `--interactive`: The installation will guide you through the process.\n\n\n### 5.3 The `adminlte:update` Command\n\nThis command is only a shortcut for `adminlte:install --force --only=assets`.\n\n\n### 5.4 Authentication views\n\n> Note: only for Laravel 5.2 and higher\n\nThis package ships the following command to replaces the authentication views with AdminLTE style views.\n\n```\nphp artisan adminlte:install --only=auth_views\n```\n\n### 5.4.1 Using the authentication views without the `adminlte:install` command\n\nIf you want to use the included authentication related views manually, you can create the following files and only add one line to each file:\n\n- `resources/views/auth/login.blade.php`:\n```\n@extends(\'adminlte::login\')\n```\n- `resources/views/auth/register.blade.php`\n```\n@extends(\'adminlte::register\')\n```\n- `resources/views/auth/passwords/email.blade.php`\n```\n@extends(\'adminlte::passwords.email\')\n```\n- `resources/views/auth/passwords/reset.blade.php`\n```\n@extends(\'adminlte::passwords.reset\')\n```\n\nBy default, the login form contains a link to the registration form.\nIf you don\'t want a registration form, set the `register_url` setting to `null` and the link will not be displayed.\n\n## 6. Configuration\n\nFirst, publish the configuration file:\n\n```\nphp artisan adminlte:install --only=config\n```\n\nNow, edit `config/adminlte.php` to configure the title, skin, menu, URLs etc. All configuration options are explained in the comments. You can override it per page with the title section.\nYou can optionally also specify a title prefix and/or postfix.\n\nThe following config options available:\n- __`title`__\n\n    Default title\n- __`title_prefix`__\n\n    Title prefix   \n- __`title_postfix`__\n\n    Title postfix\n\n\n### 6.2 Logo\nThe logo is displayed at the upper left corner of your admin panel. You can use basic HTML here if you want for a simple text logo with a small image logo (e.g. 50 x 50 pixels), or you can use two images one big (e.g. 210 x 33 pixels) and one small (e.g. 50 x 50 pixels). You can also change the sizes of the images and the alt text for both logos.\n\n- __`logo`__\n\n    Text logo content, can be HTML.\n- __`logo_img`__\n\n    Small logo image, beside text logo.\n\n    _Recommend size: 50x50px_\n- __`logo_img_class`__\n\n    Extra classes for small logo image.\n- __`logo_img_xl`__\n\n    Large logo image, if you set a img url it will replace the text logo & small logo with one big logo and on collapsed sidebar it displays the small logo.\n\n    _Recommend size: 210x33px_\n- __`logo_img_xl_class`__\n\n    Extra classes for small logo image.\n- __`logo_img_alt`__\n\n    Logo image alt text.\n\n\n### 6.3 Layout\nIt\'s possible change the layout, you can use a top navigation (navbar) only layout, a boxed layout with sidebar and you can enable fixed mode for sidebar, navbar and footer.\n\nThe following config options available:\n- __`layout_topnav`__\n\n    Enables/Disables top navigation only layout.\n\n- __`layout_boxed`__\n\n    Enables/Disables Enables/Disables boxed layout, can\'t used with `layout_topnav`.\n\n- __`layout_fixed_sidebar`__\n\n    Enables/Disables fixed sidebar, can\'t used with `layout_topnav`. \n\n- __`layout_fixed_navbar`__\n\n    Enables/Disables fixed navbar (top navigation), here you can set `true` or pass an array for responsive usage.\n\n- __`layout_fixed_footer`__\n\n    Enables/Disables fixed footer, here you can set `true` or pass an array for responsive usage.\n\n\n__Responsive Usage for `layout_fixed_navbar` & `layout_fixed_footer`__\n\nWith responsive you can disable or enable the fixed navbar/footer for specific viewport sizes.\n\nThe array the following keys available, you can set them to `true` or `false`:\n- `xs` from 0px to 575.99px\n- `sm` from 576px to 767.99px\n- `md` from 768px to 991.99px\n- `lg` from 992px to 1199.99px\n- `xl` from 1200px\n\n__Examples__\n\n- `[\'xs\' => true, \'lg\' => false]`\n\n    Fixed from mobile to small tablet (<= 991.99px)\n\n- `[\'lg\' => true]`\n\n    Fixed starting from desktop (>= 992px)\n\n- `[\'xs\' => true, \'md\' => false, \'xl\' => true]`\n\n    Fixed from mobile (<= 767.99px) and extra large desktops (>= 1200px) but not for small tablet and desktop (>= 768px & <= 1199.99px)\n\n\n### 6.4 Classes\nYou can change the look and behavior of the admin panel, you can add extra classes to body, brand, sidebar, sidebar Each menu item should have a text and a URL. You can also specify an icon from Font Awesome. A string instead of an array represents a header in sidebar layout. The \'can\' is a filter on Laravel\'s built in Gate functionality.\n\nYou can configure your menu as follows:\n\n```php\n\'menu\' => [\n    \'MAIN NAVIGATION\',\n    [\n        \'text\' => \'Blog\',\n        \'url\' => \'admin/blog\',\n    ],\n    [\n        \'text\' => \'Pages\',\n        \'url\' => \'admin/pages\',\n        \'icon\' => \'fas fa-fw fa-file\'\n    ],\n    [\n        \'text\' => \'Show my website\',\n        \'url\' => \'/\',\n        \'target\' => \'_blank\'\n    ],\n    \'ACCOUNT SETTINGS\',\n    [\n        \'text\' => \'Profile\',\n        \'route\' => \'admin.profile\',\n        \'icon\' => \'fas fa-fw fa-user\'\n    ],\n    [\n        \'text\' => \'Change Password\',\n        \'route\' => \'admin.password\',\n        \'icon\' => \'fas fa-fw fa-lock\'\n    ],\n],\n```\n\nWith a single string, you specify a menu header item to separate the items.\nWith an array, you specify a menu item. `text` and `url` or `route` are required attributes.\nThe `icon` attribute is optional, you get an [open circle](https://fontawesome.com/icons/circle?style=regular&from=io) if you leave it out.\nThe available icons that you can use are those from [Font Awesome](https://fontawesome.io/icons/).\nJust specify the name of the icon and it will appear in front of your menu item.\n\nIt\'s also possible to add menu items to the top navigation while sidebar is enabled, you just need to set the `topnav` attribute to `true`. \nThis will ignored if the top navigation layout is enabled, all menu items will appear in top navigation.\n\nUse the `can` attribute if you want conditionally show the menu item. This integrates with Laravel\'s `Gate` functionality. If you need to conditionally show headers as well, you need to wrap it in an array like other menu items, using the `header` attribute:\n\n```php\n[\n    [\n        \'header\' => \'BLOG\',\n        \'url\' => \'admin/blog\',\n        \'can\' => \'manage-blog\'\n    ],\n    [\n        \'text\' => \'Add new post\',\n        \'url\' => \'admin/blog/new\',\n        \'can\' => \'add-blog-post\'\n    ],\n]\n```\n\n#### 6.9.1 Adding a Search Input\n\nIt\'s possible to add a search input in your menu, using a menu item with the following configuration:\n\n```php\n        [\n            \'search\' => true,\n            \'href\' => \'test\',  //form action\n            \'method\' => \'POST\', //form method\n            \'input_name\' => \'menu-search-input\', //input name\n            \'text\' => \'Search\', //input placeholder\n        ],\n```\n\n#### 6.9.2 Custom Menu Filters\n\nIf you need custom filters, you can easily add your own menu filters to this package. This can be useful when you are using a third-party package for authorization (instead of Laravel\'s `Gate` functionality).\n\nFor example with Laratrust:\n\n```php\n<?php\n\nnamespace MyApp;\n\nuse JeroenNoten\\LaravelAdminLte\\Menu\\Builder;\nuse JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\FilterInterface;\nuse Laratrust;\n\nclass MyMenuFilter implements FilterInterface\n{\n    public function transform($item, Builder $builder)\n    {\n        if (isset($item[\'permission\']) && ! Laratrust::can($item[\'permission\'])) {\n            return false;\n        }\n\n        return $item;\n    }\n}\n```\n\nAnd then add to `config/adminlte.php`:\n\n```php\n\'filters\' => [\n    JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\ActiveFilter::class,\n    JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\HrefFilter::class,\n    JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\SubmenuFilter::class,\n    JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\ClassesFilter::class,\n    //JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\GateFilter::class, Comment this line out\n    MyApp\\MyMenuFilter::class,\n]\n```\n\n#### 6.9.3 Menu configuration at runtime\n\nIt is also possible to configure the menu at runtime, e.g. in the boot of any service provider.\nUse this if your menu is not static, for example when it depends on your database or the locale.\nIt is also possible to combine both approaches. The menus will simply be concatenated and the order of service providers\ndetermines the order in the menu.\n\nTo configure the menu at runtime, register a handler or callback for the `MenuBuilding` event, for example in the `boot()` method of a service provider:\n\n```php\nuse Illuminate\\Contracts\\Events\\Dispatcher;\nuse JeroenNoten\\LaravelAdminLte\\Events\\BuildingMenu;\n\nclass AppServiceProvider extends ServiceProvider\n{\n\n    public function boot(Dispatcher $events)\n    {\n        $events->listen(BuildingMenu::class, function (BuildingMenu $event) {\n            $event->menu->add(\'MAIN NAVIGATION\');\n            $event->menu->add([\n                \'text\' => \'Blog\',\n                \'url\' => \'admin/blog\',\n            ]);\n        });\n    }\n\n}\n```\nThe configuration options are the same as in the static configuration files.\n\nA more practical example that actually uses translations and the database:\n\n```php\n    public function boot(Dispatcher $events)\n    {\n        $events->listen(BuildingMenu::class, function (BuildingMenu $event) {\n            $event->menu->add(trans(\'menu.pages\'));\n\n            $items = Page::all()->map(function (Page $page) {\n                return [\n                    \'text\' => $page[\'title\'],\n                    \'url\' => route(\'admin.pages.edit\', $page)\n                ];\n            });\n\n            $event->menu->add(...$items);\n        });\n    }\n```\n\nThis event-based approach is used to make sure that your code that builds the menu runs only when the admin panel is actually displayed and not on every request.\n\n#### 6.9.4 Active menu items\n\nBy default, a menu item is considered active if any of the following holds:\n- The current path matches the `url` parameter\n- The current path is a sub-path of the `url` parameter\n- If it has a submenu containing an active menu item\n\nTo override this behavior, you can specify an `active` parameter with an array of active URLs, asterisks and regular expressions are supported. \n\nTo utilize regex, simply prefix your pattern with `regex:` and it will get evaluated automatically. The pattern will attempt to match the path of the URL, returned by `request()->path()`, which returns the current URL without the domain name. Example:\n\n```php\n[\n    \'text\' => \'Pages\',\n    \'url\' => \'pages\',\n    \'active\' => [\'pages\', \'content\', \'content/*\', \'regex:@^content/[0-9]+$@\']\n]\n```\n\n\n### 6.10 Menu Filters\nHere we can set the filters you want to include for rendering the menu.\nYou can add your own filters to this array after you\'ve created them. You can comment out the GateFilter if you don\'t want to use Laravel\'s built in Gate functionality\n\n- __`filters`__\n\n    Array of menu filters\n\n\nDefault menu filters:\n- `JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\HrefFilter::class`\n- `JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\SearchFilter::class`\n- `JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\ActiveFilter::class`\n- `JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\SubmenuFilter::class`\n- `JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\ClassesFilter::class`\n- `JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\GateFilter::class`\n- `JeroenNoten\\LaravelAdminLte\\Menu\\Filters\\LangFilter::class`\n\n\n### 6.11 Plugins\nConfigure which JavaScript plugins should be included. At this moment, DataTables, Select2, Chartjs and SweetAlert are added out-of-the-box, including the Javascript and CSS files from a CDN via script and link tag. Plugin Name, active status and files array (even empty) are required. Files, when added, need to have type (js or css), asset (true or false) and location (string). When asset is set to true, the location will be output using asset() function.\n\nBy default the [DataTables](https://datatables.net/), [Select2](https://select2.github.io/), [ChartJS](https://www.chartjs.org/), [Pace](http://github.hubspot.com/pace/docs/welcome/) and [SweetAlert2](https://sweetalert2.github.io/) plugins are supported and but not active.\nYou can activate them with changing the config file to load it on every page or add a section in specific blade files, this will automatically injecting their CDN files. \n\nSection example: `@section(\'plugins.Datatables\', true)`\n\nYou can use these plugins by default:\n- `Datatables`\n- `Select2`\n- `Chartjs`\n- `Sweetalert2`\n- `Pace`\n\nYou can also add and configure new plugins, modifying the plugin variable using the example structure below:\n\n```php\n\'plugins\' => [\n    [\n        \'name\' => \'Plugin Name\',\n        \'active\' => true,\n        \'files\' => [\n            [\n                \'type\' => \'js\',\n                \'asset\' => false,\n                \'location\' => \'//cdn.plugin.net/plugin.min.js\',\n            ],\n            [\n                \'type\' => \'css\',\n                \'asset\' => true,\n                \'location\' => \'css/plugin.min.css\',\n            ],\n        ],\n    ],\n]\n```\n\nWith the name string you specify the plugin name, and the active value will enable/disable the plugin injection.\nEach plugin have a files array, with contain arrays with file type (`js` or `css`), and `location`. \n\nIf the asset value is `true`, the injection will use the asset() function.\n\n#### 6.11.1 Pace Plugin Configuration\n\nYou can change the Pace plugin appearence, when using the CDN injection modifying the css file location.\n```\n    \'location\' => \'//cdnjs.cloudflare.com/ajax/libs/pace/1.0.2/themes/{{color}}/pace-theme-{{theme}}.min.css\',\n``` \n\nColor values: black, blue (default), green, orange, pink, purple, red, silver, white & yellow\n\nTheme values: barber-shop, big-counter, bounce, center-atom, center-circle, center-radar (default), center-simple, corner-indicator, fill-left, flash, flat-top, loading-bar, mac-osx, minimal\n\n\n## 7. Translations\n\nAt the moment, English, German, French, Dutch, Portuguese and Spanish translations are available out of the box.\nJust specifiy the language in `config/app.php`.\nIf you need to modify the texts or add other languages, you can publish the language files:\n\n```\nphp artisan adminlte:install --only=translations\n```\n\nNow, you can edit translations or add languages in `resources/lang/vendor/adminlte`.\n\n### 7.1. Menu Translations\n\nThis resource allow you to use lang files, and is active by default.\n\n#### Configurating Menu Using Lang:\n\nFirst, configure the menu using the key `text` as translation string.\nThis is an example of configuration:\n\n```php\n    [\n        \'header\' => \'account_settings\'\n    ],\n        [\n            \'text\' => \'profile\',\n            \'url\'  => \'admin/settings\',\n            \'icon\' => \'user\',\n        ],\n```\n\n#### Lang Files\n\nAll the translation strings must be added in the `menu.php` file of each language needed.\nThe translations files are located at `resources/lang/vendor/adminlte/`\n\nThis is an example of the `menu.php` lang file:\n\n```php\nreturn [\n    \'account_settings\'  => \'ACCOUNT SETTINGS\',\n    \'profile\'           => \'Profile\',\n];\n\n```\n\nTo translate the menu headers, just use the `header` param. Example:\n\n```php\n    [\n        \'header\' => \'account_settings\'\n    ],\n        [\n            \'text\' => \'profile\',\n            \'url\'  => \'admin/settings\',\n            \'icon\' => \'user\',\n        ],\n```\n\n\n## 8. Customize views\n\nIf you need full control over the provided views, you can publish them:\n\n```\nphp artisan adminlte:install --only=main_views\n```\n\nNow, you can edit the views in `resources/views/vendor/adminlte`.\n\n## 9.
