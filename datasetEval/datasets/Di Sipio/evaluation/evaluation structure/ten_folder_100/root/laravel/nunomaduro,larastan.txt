Larastan focuses on **finding errors in your code** without actually running it. It catches whole classes of bugs even **before you write tests** for the code.\n\n- Supports [Laravel](https://laravel.com)\'s **beautiful magic**.\n- This package is still in development. Please report false positives using GitHub [issues](https://github.com/nunomaduro/larastan/issues).\n\n## Installation & Usage\n\n> **Requires:**\n- **[PHP 7.1.3+](https://php.net/releases/)**\n- **[Laravel 5.6+](https://github.com/laravel/laravel)**\n\nYou may use [Composer](https://getcomposer.org) to install Larastan as a development dependency into your Laravel project:\n```bash\ncomposer require --dev nunomaduro/larastan\n```\n\n### Usage in Laravel Applications\n\nOnce you have installed Larastan, you may start analyzing your code using the `code:analyse` Artisan command.\n```bash\nphp artisan code:analyse\n```\n\n#### Custom configuration\n\nIf you would like to provide your own configuration, you can create a `phpstan.neon` or `phpstan.neon.dist` file in the root of your application. It might look like this:\n```\nincludes:\n    - ./vendor/nunomaduro/larastan/extension.neon\nparameters:\n    level: 5\n    ignoreErrors:\n        - \'#Access to an undefined property App\\\\Demo\\\\[a-zA-Z0-9\\\\_]+::\\$[a-zA-Z0-9\\\\_]+\\.#\'\n        - \'#Call to an undefined method App\\\\Http\\\\Resources\\\\DemoResource::DemoMethod\\(\\)\\.#\'\n    excludes_analyse:\n        - /*/*/FileToBeExcluded.php\n```\n\nFor all available options, please take a look at the [PHPStan documentation](https://github.com/phpstan/phpstan).\n\n### Lumen installation\n\nAdd the following lines into your app/Providers/AppServiceProvider:\n```\npublic function register()\n{\n    $this->app->register(\\NunoMaduro\\Larastan\\LarastanServiceProvider::class);\n    $this->app->instance(\'path.storage\', app()->basePath() . DIRECTORY_SEPARATOR . \'storage\');\n    $this->app->instance(\'path.config\', app()->basePath() . DIRECTORY_SEPARATOR . \'config\');\n}\n```\n\n### Usage in Packages for Laravel\n\n> **Requires:**\n- **[orchestra/testbench 3.6+](https://github.com/orchestral/testbench)**\n\nOnce you have installed Larastan, also install `orchestra/testbench`:\n\n```sh\ncomposer require --dev orchestra/testbench\n```\n\nThen create a file named `phpstan.neon.dist` in the root of your package with the content:\n```\nincludes:\n    - ./vendor/nunomaduro/larastan/extension.neon\nparameters:\n    level: 5\n    paths:\n        - src\n```\n\nStart analyzing your code using the `analyse` PHPStan command.\n```bash\n./vendor/bin/phpstan analyse\n```\n\n\n### Rule levels\n\nYou can choose from currently 8 levels: (0 is the loosest and 7 is the strictest) by passing `--level` to `analyse` command. Default level is `5`. You can also use `--level max` as an alias for the highest level.\n\n```bash\nphp artisan code:analyse --level=max\n```\n\n### Paths\n\nYou can choose which directories you want to analyze, by default, the analyzed directory will be the `app`.\n\n```bash\nphp artisan code:analyse --paths="modules,app,domain"\n```\n\n### Error formats\n\nBy default, Larastan outputs found errors into tables grouped by files to be easily human-readable. To change the output, you can use the --error-format CLI option.\n\nThe available options are: `checkstyle`, `raw`, `table`, `json`, `prettyJson`.\n\n```bash\nphp artisan code:analyse --error-format table\n```\n\n## Contributing\n\nThank you for considering to contribute to Larastan.
