b'# React Redux Universal Boilerplate\n\n## Introduction\n\n> I started this project to learn tools like [React](https://facebook.github.io/react/), [Redux](https://github.com/reactjs/redux), [Webpack](https://webpack.js.org/), [babeljs.io](https://babeljs.io/), [ES6/ES2015](https://babeljs.io/docs/learn-es2015/)... I did it mainly for fun. But it will be maintained and then I used it as Boilerplate for my React|Redux projects. So don\'t worry it works :p. It\'s not perfect but it works :).\n\n> Enjoy it! :)\n\nAn Universal ReactJS/Redux Boilerplate.\n\n## Requirements\n\n * [nodejs](http://nodejs.org/)\n\n> Node `>=7.6.0`\n\n## Optional\n\n * [yarn](https://yarnpkg.com/)\n\n## Installation\n\n```shell\n$ git clone https://github.com/kiki-le-singe/react-redux-universal-boilerplate.git <name>\n$ cd <name>\n$ npm install or yarn\n$ on postinstall you should choose between SASS or CSSNEXT\n```\n\n## Run\n\n* npm start (dev mod)\n* npm run deploy (prod mod - Runs npm run build:client and npm run build:server scripts)\n* cd readyToDeploy, npm install or yarn then npm start (prod mod)\n\n## Some NPM Script Commands\n\n\n### Storybook\n\n```shell\n$ npm run storybook\n```\n\nStarts Storybook on `http://localhost:6006/``\n\n\n### Development\n\n```shell\n$ npm run setup\n```\n\nAllows to choose between sass or cssnext and clean up unnecessary files. This question you will be asked: `You will use SASS as CSS extension language \xf0\x9f\x98\x89 . Do you wish to use CSSNEXT \xf0\x9f\x98\x80 ? This choice is irreversible. Obviously you can install the project again or just use your version control system to discard changes in working directory.`\n\n> This script runs on postinstall script (see package.json).\n\n\n```shell\n$ npm run dev\n```\n\nServes your app at `localip:3000`. HMR will be enabled in development. A proxy is used for when you  request http://localip:3000/, it will fetch http://localip:3001/ and return.\n\n```shell\n$ npm start\n```\n\nRuns `npm run dev` script.\n\n```shell\n$ npm run start:server\n```\n\nStarts the `dev` server with nodemon to serve your app at `localip:3000`.\n\n```shell\n$ npm run start:client:server\n```\n\nStarts the `webpack dev server` to serve your `webpack bundle` at `localip:3001` and enable HMR in development.\n\n\n### Production\n\n```shell\n$ npm run build:client\n```\n\nIt does some optimizations and compiles the client app, for the production, to disk (`~/readyToDeploy/static/dist`).\n\n```shell\n$ npm run build:server\n```\n\nIt does some optimizations and compiles the server app, for the production, to disk (`~/readyToDeploy/server.js`).\n\n```shell\n$ npm run deploy\n```\n\nRuns `npm run build:client` and `npm run build:server` scripts.\n\n```shell\n$ cd readyToDeploy\n$ npm install or yarn\n$ npm start\n```\n\nStarts the `prod` server to serve your app at `localip:3000`.\n\n\n### Test\n\n```shell\n$ npm run test\n```\n\nRuns unit tests with Karma. It will generate a coverage report to `~/coverage`.\n\n```shell\n$ npm run test:dev\n```\n\nSame as `npm run test` except it watches for changes to re-run tests.\n\n\n### Linter\n\n```shell\n$ npm run lint\n```\n\nLint all files in `~/src` and `~/__tests__`.\n\n\n## Deployment\n\nRuns `npm run deploy` and everything is in the `~/readyToDeploy` folder.\n\nYou can read the [README](readyToDeploy/README.md).\n\n\n## DEBUG\n\nYou should just install an extension [Redux DevTools Extension](https://github.com/zalmoxisus/redux-devtools-extension).\nPersonally I use [Redux DevTools for Chrome](https://chrome.google.com/webstore/detail/redux-devtools/lmhkpmbekcpmknklioeibfkpmmfibljd)\n\n> Are there any other alternatives? Sure!\n> You can also use [Redux DevTools](https://github.com/gaearon/redux-devtools). You don\'t need to add prefixes like `-webkit`.\n> See:\n> * [automatic vendor prefixes](http://cssnext.io/features/#automatic-vendor-prefixes) from `postcss-cssnext features`\n> * [Autoprefixer](https://github.com/postcss/autoprefixer) (used for sass mode)\n\nSee the `~/src/common/styles/global` directory to implement global styles (site\'s theme for example) and see an example of use case css module `~/src/common/views/AboutView`. There is also a `~/src/common/styles/local` directory for common local styles (this could allow to use [css modules\' composition](https://github.com/css-modules/css-modules#composition) between components).\n\nAre there any other solutions ? Fortunately yes!\n\n> I could try one of these following options if what I implemented doesn\'t work very well...\n\n* It\'s possible to use `css-modules` for [`Theming`](https://github.com/css-modules/css-modules/blob/master/docs/theming.md)\n* Obviously you can use the traditional method (it works very well) to do your own css or sass:\n\n```css\n.foo-namespace {\n  .baz {\n    ...\n  }\n  .bar {\n    ...\n  }\n}\n```\n\n```html\n<div className="foo-namespace">\n  <div className="baz">baz</div>\n  <div className="bar">bar</div>\n</div>\n```\n\n\n## Globals\n\nThese are global variables available to you anywhere in your source code. They can be found  in `~/config/index.js`.\n\n```js\nnew webpack.DefinePlugin({\n  __CLIENT__: projectConfig.__CLIENT__,\n  __SERVER__: projectConfig.__SERVER__,\n  __DEV__: projectConfig.__DEV__,\n  __PROD__: projectConfig.__PROD__,\n})\n```\n\n> Webpack is made for client side code development only. So we also have to define them on the server side\n\n```js\n/**\n * Define isomorphic constants.\n */\n\n// src/server/index.js\n\nglobal.__CLIENT__ = false\nglobal.__SERVER__ = true\nglobal.__DEV__ = projectConfig.__DEV__\nglobal.__PROD__ = projectConfig.__PROD__\n```\n\n## Unit Tests\n\nTests are in `~/__tests__`.
