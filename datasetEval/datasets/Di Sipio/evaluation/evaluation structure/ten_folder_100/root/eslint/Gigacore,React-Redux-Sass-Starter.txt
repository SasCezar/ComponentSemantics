b"![logos](https://image.ibb.co/m8S9ew/react_redux_sass.jpg)\n\n# React, Redux & Sass Starter\nA simple and quick way to get you going with developing React applications with Redux and Sass.\n\nComes with pre-configured Webpack for bundling your application to help save time in setting up your project. Keeps CSS out of JavaScript to maintain separation of concerns. Redux takes care of State Management. If you are looking for _CSS-in-JS_ approach, this starter pack is not for you.\n\n**Demo:** https://www.gigacore.in/demos/react-redux-demo/\n\nFor a complex example with Async actions, take a look at: [react-weather-forecast](https://github.com/Gigacore/react-weather-forecast)\n\n### Components \xf0\x9f\x94\xa9 Styles \xf0\x9f\x8e\xa8 Action! \xf0\x9f\x8e\xac\n\nWith all necessary tools in place, all you need to do is start creating React Components, style them and optionally use Redux for the State management. It is recommended that you use Redux if the applications you are building require interactions between multiple components or that requires pulling data from external source.\n\nAll the required code (JS + Sass) goes inside the ```src``` folder. Webpack will compile, bundle and output in the ```public``` folder, which you can upload/deploy to the root directory of your web server to publish your app!\n\n### Get started!\n```\ngit clone\ncd React-Redux-Sass-Starter\nnpm i\n```\n\n### Start the dev server:\n```\n\nnpm run start:dev\n\n```\n\n### Build:\n```\n\nnpm run build\n\n```\n\n#### Note:\n* Changes made to your CSS / JS triggers a full page auto-reload.\n* Running a build writes the ```bundle.js``` and ```bundle.css``` to the ```/public/dist/``` directory.\n\n### Pre-configured tools and plugins\n\n* Module bundler: [Webpack](https://webpack.js.org/)\n* ES6 Transpiler: [babel-loader](https://github.com/babel/babel-loader)\n* Linting: [eslint](https://eslint.org/)\n* JS Uglification: [UglifyJsPlugin](https://webpack.js.org/plugins/uglifyjs-webpack-plugin/)\n* Dev Server: [webpack-dev-server](https://github.com/webpack/webpack-dev-server) (```npm run start:dev```)\n\n#### Codebase overview.\n\n```\n/\n\xe2\x94\x9c\xe2\x94\x80 public/\n|  \xe2\x94\x9c\xe2\x94\x80 dist/\n|  |  \xe2\x94\x9c\xe2\x94\x80 bundle.css           # CSS that's generated from SASS\n|  |  \xe2\x94\x9c\xe2\x94\x80 bundle.js            # Contains scripts of components & vendor(s).\n|  \xe2\x94\x9c\xe2\x94\x80 index.html              # Contains root element to mount your app.\n\xe2\x94\x94\xe2\x94\x80 src/\n   \xe2\x94\x9c\xe2\x94\x80 actions/                # Create actions here.\n   |  \xe2\x94\x9c\xe2\x94\x80 counterActions.js\n   \xe2\x94\x9c\xe2\x94\x80 components/             # Create components here\n   |  \xe2\x94\x9c\xe2\x94\x80 Controls.js\n   |  \xe2\x94\x9c\xe2\x94\x80 Counter.js\n   |\xe2\x94\x80 containers/             # Containers bridge Components with the State.\n   |  \xe2\x94\x9c\xe2\x94\x80 Controls.js\n   |  \xe2\x94\x9c\xe2\x94\x80 Counter.js\n   \xe2\x94\x9c\xe2\x94\x80 constants/              # Define action types here.\n   |  \xe2\x94\x9c\xe2\x94\x80 ActionTypes.js\n   \xe2\x94\x9c\xe2\x94\x80 reducers/               # Create reducers here.\n   |  \xe2\x94\x9c\xe2\x94\x80 counter.js\n   |  \xe2\x94\x9c\xe2\x94\x80 index.js             # Combine your reducers here.\n   \xe2\x94\x9c\xe2\x94\x80 styles/                 # All styling goes here.\n   |  \xe2\x94\x9c\xe2\x94\x80 main.scss\n   \xe2\x94\x9c\xe2\x94\x80 App.js                  # Include your main component(s).\n   \xe2\x94\x9c\xe2\x94\x80 index.js                # Passes state to App. Renders markup to DOM.\n   \xe2\x94\x94\xe2\x94\x80 store.js                # Creates and exports store.\n```\n\n### Learn\n\nThis starter kit assumes, you are familiar with JavaScript (ES6).
