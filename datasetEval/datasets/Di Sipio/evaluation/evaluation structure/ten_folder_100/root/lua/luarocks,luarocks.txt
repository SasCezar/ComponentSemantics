b'<p align="center"><a href="http://luarocks.org"><img border="0" src="http://luarocks.github.io/luarocks/luarocks.png" alt="LuaRocks" width="500px"></a></p>\n\nA package manager for Lua modules.\n\n[![Build Status](https://travis-ci.org/luarocks/luarocks.svg?branch=master)](https://travis-ci.org/luarocks/luarocks)\n[![Build Status](https://ci.appveyor.com/api/projects/status/4x4630tcf64da48i/branch/master?svg=true)](https://ci.appveyor.com/project/hishamhm/luarocks/branch/master)\n[![Coverage Status](https://codecov.io/gh/luarocks/luarocks/coverage.svg?branch=master)](https://codecov.io/gh/luarocks/luarocks/branch/master)\n[![Join the chat at https://gitter.im/luarocks/luarocks](https://badges.gitter.im/luarocks/luarocks.svg)](https://gitter.im/luarocks/luarocks)\n\nMain website: [luarocks.org](http://www.luarocks.org)\n\nIt allows you to install Lua modules as self-contained packages called\n[*rocks*][1], which also contain version [dependency][2] information. This\ninformation can be used both during installation, so that when one rock is\nrequested all rocks it depends on are installed as well, and also optionally\nat run time, so that when a module is required, the correct version is loaded.\nLuaRocks supports both local and [remote][3] repositories, and multiple local\nrocks trees.\n\n## Installing\n\n* [Installation instructions for Unix](http://luarocks.org/en/Installation_instructions_for_Unix) (Linux, BSDs, etc.)\n* [Installation instructions for macOS](http://luarocks.org/en/Installation_instructions_for_macOS)\n* [Installation instructions for Windows](http://luarocks.org/en/Installation_instructions_for_Windows)\n\n## License\n\nLuaRocks is free software and uses the [MIT license](http://luarocks.org/en/License), the same as Lua 5.x.\n\n[1]: http://luarocks.org/en/Types_of_rocks\n[2]: http://luarocks.org/en/Dependencies\n[3]: http://luarocks.org/en/Rocks_repositories\n'
