coding,\nand debugging support for various Lua engines\n([Lua 5.1](http://studio.zerobrane.com/doc-lua-debugging),\n[Lua 5.2](http://studio.zerobrane.com/doc-lua52-debugging),\n[Lua Quick](http://notebook.kulchenko.com/zerobrane/marmalade-quick-debugging-with-zerobrane-studio),\n[Cocos2d-x](http://notebook.kulchenko.com/zerobrane/cocos2d-x-simulator-and-on-device-debugging-with-zerobrane-studio),\n[OpenResty/Nginx](http://notebook.kulchenko.com/zerobrane/debugging-openresty-nginx-lua-scripts-with-zerobrane-studio),\n[Torch7](http://notebook.kulchenko.com/zerobrane/torch-debugging-with-zerobrane-studio),\n[Redis](http://notebook.kulchenko.com/zerobrane/redis-lua-debugging-with-zerobrane-studio),\n[GSL-shell](http://notebook.kulchenko.com/zerobrane/gsl-shell-debugging-with-zerobrane-studio),\n[Adobe others).\n\n![ZeroBrane Studio debugger screenshot](http://studio.zerobrane.com/images/debugging.png)\n\n## Features\n\n* Small, portable, and **cross-platform** (Windows, macOS, and Linux).\n* Written in Lua and is extensible with Lua packages.\n* **Syntax highlighting** and folding for 125+ languages and file formats.\n* **Project view** with auto-refresh and ability to hide files and directories from the list.\n* Bundled with several of **the most popular Lua modules**\n([luasocket](https://github.com/diegonehab/luasocket),\n[luafilesystem](https://github.com/keplerproject/luafilesystem),\n[lpeg](http://www.inf.puc-rio.br/~roberto/lpeg/),\nand [luasec](https://github.com/brunoos/luasec))\ncompiled for all supported Lua versions.\n* **Auto-complete** for functions, keywords, and custom APIs with **scope-aware completion** for variables.\n* [Scope-aware variable indicators](http://notebook.kulchenko.com/zerobrane/scope-aware-variable-indicators-zerobrane-studio) for Lua code.\n* **Integrated debugger** with local and [remote debugging](http://studio.zerobrane.com/doc-remote-debugging)\nfor [Lua 5.1](http://studio.zerobrane.com/doc-lua-debugging),\n[Lua 5.2](http://studio.zerobrane.com/doc-lua52-debugging),\n[Lua 5.3](http://studio.zerobrane.com/doc-lua53-debugging),\n[Lua 5.4](http://studio.zerobrane.com/doc-lua54-debugging),\n[LuaJIT](http://studio.zerobrane.com/doc-luajit-debugging),\nand [other Lua engines](http://studio.zerobrane.com/documentation#debugging).\n* **Interactive console** to directly test code snippets with local and remote execution.\n* [Live coding](http://studio.zerobrane.com/documentation#live_coding)\nwith [Lua](http://notebook.kulchenko.com/zerobrane/live-coding-in-lua-bret-victor-style),\n[L\xc3\x96VE](http://notebook.kulchenko.com/zerobrane/live-coding-with-love),\n[Gideros](http://notebook.kulchenko.com/zerobrane/gideros-live-coding-with-zerobrane-studio-ide),\n[Moai](http://notebook.kulchenko.com/zerobrane/live-coding-with-moai-and-zerobrane-studio),\n[Corona SDK](http://notebook.kulchenko.com/zerobrane/debugging-and-live-coding-with-corona-sdk-applications-and-zerobrane-studio),\nGSL-shell, and other engines.\n* **Static analysis** to catch errors and typos during development.\n* **Function outline**.\n* **Go to definition** navigation.\n* **Multi-cursor editing** with **scope-aware variable selection and renaming**.\n* **Fuzzy search** with `Go To File`, **project-wide** `Go To Symbol` navigation, and `Insert Library Function`.\n* Find and replace in multiple files with **preview and undo**.\n* Several **ways to extend** the current functionality:\n  - packages (`packages/`): [plugins](http://studio.zerobrane.com/doc-plugin) that provide additional functionality;\n  - translations (`cfg/i18n/`): [translations](http://studio.zerobrane.com/doc-translation) of the menus and messages to other languages;\n  - user configuration (`cfg/`): settings for various components, styles, color themes, and other preferences.\n\n## Documentation\n\n* A [short and simple overview](http://studio.zerobrane.com/doc-getting-started) for those who are new to this development environment.\n* A list of There are three options to install it:\n\n* Download [installation package for the latest release](https://studio.zerobrane.com/) for individual platforms (Windows, OSX, or Linux);\n* Download [snapshot of the repository for each of the releases](releases), which works for all platforms;\n* Clone the repository to access the current development version; this option also works for all platforms.\n\n**No compilation is needed** for any of the installation options, although the scripts to compile required libraries for all supported platforms are available in the `build/` directory.\n\nSee the [installation section](https://studio.zerobrane.com/doc-installation) in the documentation for further details and uninstallation instructions.\n\n## Usage\n\nThe IDE can be launched by using the `zbstudio` command with slight variations depending on whether a packaged installation or a repository copy is used:\n\n* **Windows**: Run `zbstudio` from the directory that the IDE is installed to or create a shortcut pointing to `zbstudio.exe`.\n* **Linux**: Run `zbstudio` when installed from the package installation or run `./zbstudio.sh` when using a snapshot/clone of the repository.\n* **macOS**: Launch the `ZeroBrane Studio` application if installed or run `./zbstudio.sh` when using a snapshot/clone of the repository.\n\nThe general command for launching is the following: `zbstudio [option] [<project directory>] [<filename>...]`.\n\n* **Open files**: `zbstudio <filename> [<filename>...]`.\n* **Set project directory** (and optionally open files): `zbstudio <project directory> [<filename>...]`.\n* **Overwrite default configuration**: `zbstudio -cfg "string with configuration settings"`, for example: `zbstudio -cfg "editor.fontsize=12; editor.usetabs=true"`.\n* **Load custom configuration file**: `zbstudio -cfg <filename>`, for example: `zbstudio -cfg cfg/xcode-keys.lua`.\n\nAll configuration changes applied from the command line are only effective for the current session.\n\nIf you are loading a file, you can also **set the cursor** on a specific line or at a specific position by using `filename:<line>` and `filename:p<pos>` syntax (0.71+).\n\nIn all cases only one instance of the IDE will be allowed to launch by default:\nif one instance is already running, the other one won\'t launch, but the directory and file parameters\npassed to the second instance will trigger opening of that directory and file(s) in the already started instance.\n\n## Contributing\n\nSee [CONTRIBUTING](CONTRIBUTING.md).\n\n## Author\n\n### ZeroBrane Studio and MobDebug\n\n  **ZeroBrane LLC:** Paul Kulchenko (paul@zerobrane.com)\n\n### Estrela Editor\n\n  **Luxinia Dev:** Christoph Kubisch (crazybutcher@luxinia.de)\n\n## Where is Estrela?\n\nThe [Estrela project](http://www.luxinia.de/index.php/Estrela/) that this IDE is based on has been merged into ZeroBrane Studio.\nIf you have used Estrela for graphics shader authoring, you can use [this GraphicsCodePack](https://github.com/pixeljetstream/zbstudio-graphicscodepack)\nto get access to all API files, specifications and tools.\n  \n## License\n\nSee [LICENSE](LICENSE).\n'
