b'<p align ="center"><img src=SWUpdate.svg width=400 height=400 /></p>\n\nSWUpdate - Software Update for Embedded Systems\n===============================================\n\n[![Build Status](https://travis-ci.org/sbabic/swupdate.svg?branch=master)](https://travis-ci.org/sbabic/swupdate)\n\nSWUpdate is a Linux Update agent with the goal to\nprovide an efficient and safe way to update\nan embedded system. SWUpdate supports local and remote\nupdates, multiple update strategies and it can\nbe well integrated in the [Yocto](https://www.yoctoproject.org) build system by adding\nthe [meta-swupdate](https://layers.openembedded.org/layerindex/branch/master/layer/meta-swupdate/) layer.\n\nExamples using this layer with evaluation boards (Beaglebone, RPI3) are provided in \n[meta-swupdate-boards](https://layers.openembedded.org/layerindex/branch/master/layer/meta-swupdate-boards/) layer.\n\nIt supports the common media on embedded devices \nsuch as NOR / NAND flashes, UBI volumes, SD / eMMC, and can\nbe easily extended to introduce project specific update\nprocedures.\n\nPre- and postinstall scripts are supported, and a Lua\ninterpreter helps to customize the update process.\n\nAn update package is described by the sw-description file,\nusing the libconfig syntax or JSON. It is even possible to\nuse Lua with a custom syntax.\n\nHere a short list of the main features:\n\n- Install on embedded media (eMMC, SD, Raw NAND, NOR and SPI-NOR flashes)\n- Allow delivery single image for multiple devices\n- Multiple interfaces for getting software\n    - local storage\n    - integrated web server\n    - integrated REST client connector to [hawkBit](https://projects.eclipse.org/projects/iot.hawkbit)\n    - remote server download\n- Software delivered as images, gzipped tarball, etc.\n- Allow custom handlers for installing FPGA firmware, microcontroller firmware via custom protocols.\n- Power-Off safe\n- Hardware / Software compatibility.\n\nDifferent components of this software are under different licenses (a mix\nof MIT, GPLv2 and GPLv2+). License information for any file is either explicitly stated\nor defaults to GPL version 2.0+.\n\nPlease check inside doc directory for documentation or\nthe online documentation (generated from doc/) at:\nhttp://sbabic.github.io/swupdate\n\n\nContributing to the project\n---------------------------\n\nContributions are welcome !  You can submit your patches (or post questions\nregarding the project) to the swupdate Mailing List:\n\n\tswupdate@googlegroups.com\n\nPlease read the [contributing](http://sbabic.github.io/swupdate/contributing.html)\nchapter in the documentation how to contribute to the project.\n'
