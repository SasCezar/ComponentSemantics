b'<p align="center">\n    <img width="150px" src="https://cdn.rawgit.com/MichaelMure/git-bug/master/misc/logo/logo-alpha-flat-bg.svg">\n</p>\n<h1 align="center">git-bug</h1>\n\n<div align="center">\n\n[![Build Status](https://travis-ci.org/MichaelMure/git-bug.svg?branch=master)](https://travis-ci.org/MichaelMure/git-bug)\n[![Backers on Open Collective](https://opencollective.com/git-bug/backers/badge.svg)](#backers) [![Sponsors on Open Collective](https://opencollective.com/git-bug/sponsors/badge.svg)](#sponsors) [![License: GPL v3](https://img.shields.io/badge/License-GPLv3+-blue.svg)](http://www.gnu.org/licenses/gpl-3.0)\n[![GoDoc](https://godoc.org/github.com/MichaelMure/git-bug?status.svg)](https://godoc.org/github.com/MichaelMure/git-bug)\n[![Go Report Card](https://goreportcard.com/badge/github.com/MichaelMure/git-bug)](https://goreportcard.com/report/github.com/MichaelMure/git-bug)\n[![Gitter chat](https://badges.gitter.im/gitterHQ/gitter.png)](https://gitter.im/the-git-bug/Lobby)\n\n</div>\n\n`git-bug` is a bug tracker that:\n\n- **fully embed in git**: you only need your git repository to have a bug tracker\n- **is distributed**: use your normal git remote to collaborate, push and pull your bugs !\n- **works offline**: in a plane or under the sea ? keep reading and writing bugs\n- **prevent vendor locking**: your usual service is down or went bad ? you already have a full backup\n- **is fast**: listing bugs or opening them is a matter of milliseconds\n- **doesn\'t pollute your project**: no files are added in your project\n- **integrate with your tooling**: use the UI you like (CLI, terminal, web) or integrate with your existing tools through the CLI or the GraphQL API\n- **bridge with other bug trackers**: [bridges](#bridges) exist to import and export to other trackers.\n\n:construction: This is now more than a proof of concept, but still not fully stable. Expect dragons and unfinished business. :construction:\n\n## Install\n\n<details><summary>Pre-compiled binaries</summary>\n\n1. Go to the [release page](https://github.com/MichaelMure/git-bug/releases/latest) and download the appropriate binary for your system.\n2. Copy the binary anywhere in your PATH\n3. Rename the binary to `git-bug` (or `git-bug.exe` on windows)\n\nThat\'s all !\n\n</details>\n\n<details><summary>Linux packages</summary>\n\n* [Archlinux (AUR)](https://aur.archlinux.org/packages/?K=git-bug)\n\n</details>\n\n<details><summary>go get (unstable)</summary>\n\n```shell\ngo get -u github.com/MichaelMure/git-bug\n```\n\nIf it\'s not done already, add golang binary directory in your PATH:\n\n```bash\nexport PATH=$PATH:$(go env GOROOT)/bin:$(go env GOPATH)/bin\n```\n\n</details>\n\n## CLI usage\n\nCreate a new identity:\n\n```\ngit bug user create\n```\n\nCreate a new bug:\n\n```\ngit bug add\n```\n\nYour favorite editor will open to write a title and a message.\n\nYou can push your new entry to a remote:\n```\ngit bug push [<remote>]\n```\n\nAnd pull for updates:\n```\ngit bug pull [<remote>]\n```\n\nList existing bugs:\n```\ngit bug ls\n```\n\nFilter and sort bugs using a [query](doc/queries.md):\n```\ngit bug ls "status:open sort:edit"\n```\n\nYou can now use commands like `show`, `comment`, `open` or `close` to display and modify bugs. For more details about each command, you can run `git bug <command> --help` or read the [command\'s documentation](doc/md/git-bug.md).\n\n## Interactive terminal UI\n\nAn interactive terminal UI is available using the command `git bug termui` to browse and edit bugs.\n\n![Termui recording](misc/termui_recording.gif)\n\n## Web UI (status: WIP)\n\nYou can launch a rich Web UI with `git bug webui`.\n\n<p align="center">\n  <img src="misc/webui1.png" alt="Web UI screenshot 1" width="738">\n</p>\n\n<p align="center">\n  <img src="misc/webui2.png" alt="Web UI screenshot 2" width="738">\n</p>\n\nThis web UI is entirely packed inside the same go binary and serve static content through a localhost http server.\n\nThe web UI interact with the backend through a GraphQL API. The schema is available [here](graphql/).\n\n## Bridges\n\n### Importer implementations\n\n| | Github | Gitlab | Launchpad |\n| --- | --- | --- | --- |\n| **incremental**<br/>(can import more than \\\n    --target=github \\\n    --url=https://github.com/michaelmure/git-bug \\\n    --token=$TOKEN\n```\n\nImport bugs:\n\n```bash\ngit bug bridge pull [<name>]\n```\n\nExport modifications:\n\n```bash\ngit bug bridge push [<name>]\n```\n\nDeleting a bridge:\n\n```bash\ngit bug bridge rm [<name>]\n```\n\n## Internals\n\nInterested by how it works ?
