While we cannot\n  (yet) claim that Magit wraps and improves upon each and every Git\n  command, it is complete enough to allow even experienced Git users\n  to perform almost all of their daily version control tasks directly\n  from within Emacs.  While many fine Git clients exist, only Magit\n  and Git itself deserve to be called porcelains.\n</p>\n<hr>\n\n<div align="center">\n  Keeping its users <a href= "https://magit.vc/quotes">this Started\n===============\n\nIf you are new to Magit, then either one of the following two\narticles should help understanding how it differs from other Git\nclients.\n\n- [Visual Magit walk-through](https://emacsair.me/2017/09/01/magit-walk-through)\n\n  If you are completely new to Magit, then this article is a good\n  visual introduction.\n\n  Almost everything that you see in Magit can be acted on by pressing\n  some key, but that\'s not obvious from just seeing how Magit looks.\n  The screenshots and accompanying text of this article explain how to\n  perform a variety of actions on Magit\'s output.\n\n- [Magit, the magical Git interface](https://emacsair.me/2017/09/01/the-magical-git-interface)\n\n  Magit differs significantly from other Git interfaces, and its\n  advantages are not immediately obvious simply from looking at a few\n  screenshots as presented in the preceding article.\n\n  This article discusses Magit\'s properties in somewhat more abstract\n  terms.\n\nSupport\n=======\n\nWhen something doesn\'t work as expected then please first see the\n[FAQ][faq].  Then also try the list of [open issues][issues] and use\nthe search box at the top of that page to find older related issues.\nYou should also consult the [manual][manual] and ask a general-purpose\nsearch engine.\n\nIf that doesn\'t answer your question, then ask for help on the\n**[Emacs Stackexchange site][forum]** or the [mailing list][list].\nWe only use the GitHub issue tracker for feature requests and bug\nreports, so please don\'t ask for help there.\n\nContributing\n============\n\nTo report bugs and suggest new feature use the\n[issue tracker][issues].  If you have some code which you would like\nto be merged, then open a [pull request][pulls].
