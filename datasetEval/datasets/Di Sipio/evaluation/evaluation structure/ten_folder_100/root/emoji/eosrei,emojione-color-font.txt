It allows font creators to embed complete SVG files\nwithin a font enabling full color and even animations. There are more details\nin the [SVGinOT proposal][6] and the [OpenType SVG table specifications][7].\n\nSVGinOT Font demos (Firefox only):\n\n* https://hacks.mozilla.org/2014/10/svg-colors-in-opentype-fonts/\n* http://xerographer.github.io/reinebow/\n* http://xerographer.github.io/multicoloure/\n\n[6]: https://www.w3.org/2013/10/SVG_in_OpenType/\n[7]: https://www.microsoft.com/typography/otspec/svg.htm\n\n## Install on Linux\nThe font can be installed for a user or system-wide. Get the latest version\nfrom releases: https://github.com/eosrei/emojione-color-font/releases\n\n*Note: This requires `Bitstream Vera` is installed and will change your\nsystems default serif, sans-serif and monospace fonts.*\n\n### Why Bitstream Vera\nThe default serif, sans-serif and monospace font for most Linux distributions is\n`DejaVu`. `DejaVu` includes a wide range of symbols which override the\n`EmojiOne Color` characters. The previous solution was to make\n`EmojiOne Color` the default system font, but that causes a number of issues.\nA better solution is a different font that doesn\'t override any emoji characters\nsuch as `Bitstream Vera`. `Bitstream Vera` is the source of the glyphs used in\n`DejaVu`, so it\'s not very different. 99%+ of people will not notice the\ndifference.\n\n### Additional default font options\nThe `Noto` and `Roboto` font families conflict far less than `DejaVu`. You may\nwant to try them. Primary issues are the 0x2639 and 0x263a characters.\n\n### Known issues\n\n* [Symbols/emoji in monospace formatted text cause incorrect character alignment][8].\n  The whitespace character widths from the most recently selected\n  fallback font are used in Pango/GTK applications.\n* [[Issue #31][9]] [Some font families are not matched correctly in Linux Firefox][10].\n  Workaround: Open `about:config` set\n  `gfx.font_rendering.fontconfig.fontlist.enabled` to `false`.\n  [Note: May cause crashes in Firefox <48.][11]\n\n[8]:https://bugzilla.gnome.org/show_bug.cgi?id=757785\n[9]:https://github.com/eosrei/emojione-color-font/issues/31\n[10]:https://bugzilla.mozilla.org/show_bug.cgi?id=1245811\n[11]:https://bugzilla.mozilla.org/show_bug.cgi?id=1266341\n\n### Manual install on any Linux\nInstall for the current user without root:\n```sh\n# 1.
