The tool is commercial (license available online), with a free download option.\n\n## JSqlParser\n\nJSqlParser is a SQL statement parser. It translates SQLs in a traversable hierarchy of Java classes. JSqlParser is not limited to one database but provides support for a lot of specials of Oracle, SqlServer, MySQL, PostgreSQL ... To name some, it has support for Oracles join syntax using (+), PostgreSQLs cast syntax using ::, relational operators like != and so on.\n\n## Support\nIf you need help using JSqlParser feel free to file an issue or contact me.\n\n## Contributions\nTo help JSqlParser's development you are encouraged to provide \n* feedback\n* bugreports\n* pull requests for new features\n* improvement requests\n* fund new features or sponsor JSqlParser ([**Sponsor**](https://www.paypal.me/wumpz))\n\n**Please write in English, since it's the language most of the dev team knows.**\n\nAlso I would like to know about needed examples or documentation stuff.\n\n## Extensions in the latest SNAPSHOT version 3.2\n\n\n## Extensions of JSqlParser releases\n\n* [Release Notes](https://github.com/JSQLParser/JSqlParser/releases)\n* Modifications before GitHub's release tagging are listed in the [Older Releases](https://github.com/JSQLParser/JSqlParser/wiki/Older-Releases) page.\n\n\n## Building from the sources\n\nAs the project is a Maven project, building is rather simple by running:\n\n\tmvn package\n    \nThe project requires the following to build:\n- Maven \n- JDK 8 or later. The jar will target JDK 8, but the version of the maven-compiler-plugin that JsqlParser uses requires JDK 8+\n\nThis will produce the jsqlparser-VERSION.jar file in the target/ directory.\n\n**To build this project without using Maven, one has to build the parser by JavaCC using the CLI options it provides.**\n\n## Debugging through problems\n\nRefer to the [Visualize Parsing](https://github.com/JSQLParser/JSqlParser/wiki/Examples-of-SQL-parsing#visualize-parsing) section to learn how to run the parser in debug mode.\n\n## Source Code conventions\n\nRecently a checkstyle process was integrated into the build process. JSqlParser follows the sun java format convention. There are no TABs allowed. Use spaces.\n\n```java\npublic void setUsingSelect(SubSelect usingSelect) {\n    this.usingSelect = usingSelect;\n    if (this.usingSelect != null) {\n        this.usingSelect.setUseBrackets(false);\n    }\n}\n```\n\nThis is a valid piece of source code:\n* blocks without braces are not allowed\n* after control statements (if, while, for) a whitespace is expected\n* the opening brace should be in the same line as the control statement\n\n## Maven Repository\n\nJSQLParser is deployed at sonatypes open source maven repository. \nStarting from now I will deploy there.
