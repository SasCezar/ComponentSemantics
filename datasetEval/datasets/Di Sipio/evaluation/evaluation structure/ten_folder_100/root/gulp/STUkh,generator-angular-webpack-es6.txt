folder.\n\n### Test\n##### Unit testing\nThe app uses [Karma](http://karma-runner.github.io/2.0/index.html) to run the unit tests, which you can find near the test target (*.spec.js files). see example test in the above directory structure.\nFor running these tests run this command in project directory:\n```\nnpm test\n```\nThis command will automatically watch for changes that happening in test files\nand rerun the test suite\nTo disable the above behaviour , please check package.json file\n\n##### End-to-end testing\nThe app uses [Protractor](https://github.com/angular/protractor), an end-to-end test framework designed for AngularJS apps, to the end-to-end tests, which you can find in the e2e folder.\n\n##### Setup development environment for running end-to-end tests\n* First make sure that you did `npm install`.\n* Download the necessary binaries for Selenium Server `npm run webdriver-update`.\n* Open onother command line and run the development server `npm start` and make sure the running port match the baseUrl port in protractor.conf.js file.\n* Finally run `npm run test:e2e`.\n\n##### Example tests\n* Check the example test for unit tests using jasmin api in\n     ```\n     /src/app/pages/main/main.controller.spec.js\n     ```\n    All your unit test files must end with `.spec.js`\n\n* Check the example test for e2e test in e2e directory.\n     ```\n     /e2e/main.component.spec.js\n     ```\n\n\n### Known bugs:\n> * **Problem**: Webpack2 unable to import function with only export default value.\n\n>   **Workaround**: Use ```import * as variable from
