b'# tic\n\n[![Travis-CI Build Status](https://travis-ci.org/ropenscilabs/tic.svg?branch=master)](https://travis-ci.org/ropenscilabs/tic)\n[![Build status](https://ci.appveyor.com/api/projects/status/r8w1psd0f5r4hs6t/branch/master?svg=true)](https://ci.appveyor.com/project/ropensci/tic/branch/master)\n[![CircleCI](https://circleci.com/gh/ropenscilabs/tic.svg?style=svg)](https://circleci.com/gh/ropenscilabs/tic)\n[![CRAN status](https://www.r-pkg.org/badges/version/tic)](https://cran.r-project.org/package=tic)\n[![lifecycle](https://img.shields.io/badge/lifecycle-maturing-blue.svg)](https://www.tidyverse.org/lifecycle/#maturing)\n[![](https://badges.ropensci.org/305_status.svg)](https://github.com/ropensci/software-review/issues/305)\n\nThe goal of tic is to enhance and simplify working with continuous integration (CI) systems.\n\nThe following ones are supported: \n\n- [Travis CI](https://travis-ci.org) (Linux)\n- [AppVeyor](https://www.appveyor.com/) (Windows)\n- [Circle CI](https://circleci.com/) (Linux)\n\nTo learn more about CI, read [the "Travis CI for test automation" blog post](http://mahugh.com/2016/09/02/travis-ci-for-test-automation/) and our [Getting Started](https://ropenscilabs.github.io/tic/articles/tic.html#prerequisites) vignette.\n\nThe most important improvements over existing solutions are:\n\n1. Deployment to a Git repository is greatly simplified. Update your repository with results from the CI build.\n\n1. Support for R packages and other kinds of project (bookdown, blogdown, etc.), with predefined templates. \n   Set up your project to deploy rendered versions of your book or blog with a single push to Git.\n\n1. Workflow specification in a single `.R` file, regardless of CI system used.  \n   Forget about `.yml` files or web browser configurations.\n\n## Installation\n\nIt can be installed from GitHub with:\n\n``` r\n# install.packages("remotes")\nremotes::install_github("ropenscilabs/tic")\n```\n\n## Setup\n\nBy calling `tic::use_tic()` a production ready CI setup is initialized, tailored to your specific R project.\nThe created templates will use the providers https://travis-ci.org and https://appveyor.com.\nFor an R package, the following steps will be set up for the CI workflow:\n\n- Installation of required dependencies for the project\n- Satisfying build-time dependencies of steps to be run in all CI stages\n- Running `rcmdcheck::rcmdcheck()`\n- Building of a `pkgdown` site with deployment GitHub\n- Running a code coverage and uploading it to [codecov.io](https://codecov.io/)\n\nSee the [Getting Started](https://ropenscilabs.github.io/tic/articles/tic.html) vignette for more information and links to [minimal example repositories](https://ropenscilabs.github.io/tic/articles/tic.html#examples-projects) for various R projects (package, blogdown, bookdown and more).\n\n## Examples\n\nAll examples listed here work with Travis, some work with AppVeyor too. Multi-project repositories are not supported, see [the comment by @jwijffels](https://github.com/ropenscilabs/tic/issues/117#issuecomment-460814990) for guidance to work around this limitation.\n\n---\n\n[![ropensci_footer](https://ropensci.org/public_images/ropensci_footer.png)](https://ropensci.org)\n'
