b"# GitLab CI Monitor\n\nA simple dashboard for monitoring [GitLab CI][gitlab-ci] builds.\n**Alpha version**.\n\nGitlab Support: 9.0 (API V4)\n\n[gitlab-ci]: https://about.gitlab.com/gitlab-ci/\n\n\n![Example][example]\n\n[example]: images/gitlab-ci-monitor-example.png\n\n\n## Usage\n\nThis project runs completely in the browser. It expects a few parameters\nin the query string:\n\n- **gitlab**: your gitlab server address (not needed if you deployed the monitor on the gitlab instance)\n- **token**: your gitlab token (if you deployed the monitor on the gitlab instance you may set this to `use_cookie`)\n- **projects**: a comma separated list of projects in the form GROUP_NAME/PROJECT_NAME/BRANCH_NAME you want to monitor.\n- **groups**: a comma separated list of groups or subgroups\n- **order**: a comma separated list of fields to order projects, optionally followed by .asc or .desc modifier\n\nAt least one of `groups` or `projects` need to be set.\n\nExample:\n\n```\nhttp://gitlab-ci-monitor.example.com/?gitlab=gitlab.example.com&token=12345&projects=namespace/project1/master,namespace/project1/branch1,namespace/project2/master&order=status.desc,project,author.asc\n```\n\nWith these parameters, it will try to fetch the list of projects that this\ntoken has access. Then, it will filter the list by the **projects** parameter\nand show only the ones that have builds (i.e., that have GitLab CI enabled).\n\nIf you set `groups` it will show the status of the default branch of those\nprojects in the group, which are active and have jobs enabled.\n\nExample:\n```\nhttps://gitlab.example.com/?token=use_cookie&groups=group1,base/group2,group3\n```\n\nThis requires installation on the gitlab server. With these parameters you see the pipeline state\nof the default branch of all projects in groups group1, base/group2 and group3.\n\n* The project name in title of a box links to the project, the branch name to the tree view of the branch.\n* The commit hash on the lower left links to the tree view of the commit.\n* The commit title links to the pipeline overview.\n* Hovering over the time on the lower right will give you the exact date.\n\n## Installation on a GitLab instance\n\n* Download and unpack to a directory of your choice:\n```\nrm -rf /usr/local/share/gitlab/monitor\nmkdir -p /usr/local/share/gitlab/monitor\ncurl -s -L https://github.com/globocom/gitlab-ci-monitor/archive/master.tar.gz | tar xz -C /usr/local/share/gitlab/monitor --strip-components=1\n```\n\nor\n\n```\nmake install\n```\n\n* Now insert the following to your `/etc/gitlab/gitlab.rb` (the entry already exists but it commented):\n```\nnginx['custom_gitlab_server_config'] = <<-NGINX_CONF\n  location /_internal/monitor/ {\n    index index.html;\n    proxy_cache gitlab;\n    alias /usr/local/share/gitlab/monitor/;\n}\nNGINX_CONF\n```\n* Run `gitlab-ctl reconfigure`\n* Now you may use the monitor without providing the parameter `gitlab` and set `token=use_cookie`\n\n\n## License\n\nGitLab CI Monitor is licensed under the [MIT license](LICENSE).\n"
