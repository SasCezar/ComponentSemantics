b'# Simplified LNCS Template [![Build Status](https://circleci.com/gh/latextemplates/LNCS/tree/master.svg?style=shield)](https://circleci.com/gh/latextemplates/LNCS/)\n\n> Quick start for modern LaTeXing with [LNCS](http://www.springer.com/computer/lncs).\n\n## TOC\n\n<!-- toc -->\n\n- [Features](#features)\n- [Background](#background)\n- [Quick start](#quick-start)\n- [Tool hints](#tool-hints)\n- [Using the template with your git repository](#using-the-template-with-your-git-repository)\n- [FAQ](#faq)\n- [Development](#development-)\n- [Links](#links)\n\n<!-- tocstop -->\n\n## Features\n\n* Support for German documents (without broken headers):\n  Contains a fix to increase compatibility with Babel.\n  See <https://tex.stackexchange.com/a/441701/9075> for details.\n* Provides a skeletal [paper.tex](paper.tex) file.\n* Generated PDF allows for copy and paste of text without getting words with ligatures such as "workflow" destroyed.\n  This is enabled by `glyphtounicode`, which encodes ligatures (such as fl) using unicode characters.\n* Automatic setting of "Fig." and "Section"/"Sect." according to the LNCS style.\n  Just use `\\Cref{sec:xy}` at the beginning of a sentence and `\\cref{sec:xy}` in the middle of a sentence.\n  Thanx to [cleveref].\n* Support of hyperlinked references without extra color thanx to [hyperref].\n* Better breaking of long URLs.\n* Sharper font (still compatible with Springer\'s requirements).\n* Support for `\\powerset` command.\n* Support todos as pdf annotations. This is enabled by the [pdfcomment] package.\n* [microtypographic extensions](https://www.ctan.org/pkg/microtype) for a better look of the paper.\n* Adds modern packages such as [microtype], [cleveref], [csquotes], [paralist], [hyperref], [hypcap], [upquote], [natbib], [booktabs], [cfr-lm].\n* Optional: Support for [minted] package. Uncomment `\\usepackage[newfloat]{minted}` to get started.\n* Optional: Compile with `lualatex` instead of `pdflatex`.\n* Ready-to-go configuration for [latexindent].\n\nExamples:\n\n- [paper.pdf](https://latextemplates.github.io/LNCS/paper.pdf) - normal paper.\n- [paper-minted.pdf](https://latextemplates.github.io/LNCS/paper-minted.pdf) - paper showing minted in action.\n- [paper-newtx.pdf](http://latextemplates.github.io/LNCS/paper-newtx.pdf) - paper typeset in Times Roman to save some space.\n- [paper-minted-newtx.pdf](http://latextemplates.github.io/LNCS/paper-minted-newtx.pdf) - paper typeset in Times Roman to save some space.\n\n## Background\n\nThe official template is available at <https://www.springer.com/gp/computer-science/lncs/conference-proceedings-guidelines> --> "Templates, samples files & useful links" --> "LaTeX2e Proceedings Templates (zip)"\nDeep link: <ftp://ftp.springernature.com/cs-proceeding/llncs/llncs2e.zip>.\n\n> **The class file authored by Springer is needed to get the template working:**\n> `llncs.cls`\n>  You get it from inside the ZIP of `llncs2e.zip` available at <ftp://ftp.springernature.com/cs-proceeding/llncs/llncs2e.zip>.\n\nReason: Licensing restrictions of Springer do not allow distribution outside of springer.\nSee [message #47 for debian bug 31897](https://bugs.debian.org/cgi-bin/bugreport.cgi?bug=31897#47) for details.\nTherefore, the required file `llncs.cls` has to be downloaded in some way.\nFollow the quick start instructions.\n\n## Quick start\n\n* Click on `Download ZIP` or [here](https://github.com/latextemplates/LNCS/archive/master.zip).\n* Extract `LNCS-master.zip` in the folder where you want to write your paper.\n* Place `llncs.cls` into the directory\n  - Download `llncs2e.zip` from <ftp://ftp.springernature.com/cs-proceeding/llncs/llncs2e.zip> and extract it in the directory.\n    On Linux, just execute `download-llncs-files-from-springer.sh`.\n  - In case ftp does not work at your side, you can try online ftp services such as http://www.net2ftp.com/ to download the files.\n    Open the connection to `ftp.springernature.com` and navigate to `cs-proceeding`, `llncs`, and download the ZIP archive.\n* Edit [paper.tex](paper.tex).\n* `latexmk paper`.\n\n As you see on CircleCI, the paper compiles out of the box.\n There is no need to adjust the packages or to remove some of them.\n This might lead to undesiered results such as hyperlinks not working any more or no good microtypographic features.\n In case you think, a package needs to be altered or added, feel free to open an issue.\n\n## Tool hints\n\nThere is currently no official biblatex support.\nA first step towards that is done at [biblatex-lncs](https://github.com/neapel/biblatex-lncs/).\n\nMiKTeX installation hints are given at <https://github.com/latextemplates/scientific-thesis-template/blob/template/README.md#installation-hints-for-windows>.\n\n- Grammar and spell checking is available at [TeXstudio].\n  Please download [LanguageTool] (Windows: `choco install languagetool`) and [configure TeXstudio to use it](http://wiki.languagetool.org/checking-la-tex-with-languagetool#toc4).\n  Note that it is enough to point to `languagetool.jar`.\n  **If TeXstudio doesn\'t fit your need, check [the list of all available LaTeX Editors](http://tex.stackexchange.com/questions/339/latex-editors-ides).**\n- Use [JabRef] to manage your bibliography (Windows: `choco install jabref`).\n\n\nIn case you want to get started using minted, do following steps:\n\n1. Install python: `choco install python` - that uses [chocolatey](https://chocolatey.org/) to install Python\n2. Install [pygments]: `pip instal pygments` - that uses the Pyhton package manager to install the pygments library\n3. When latexing, use `-shell-escape`: `pdflatex -shell-escape paper`.\n   You can also just execute `latexmk paper`.\n\n## Using the template with your git repository\n\n1. Initialize your git repository as usual\n2. Add this repository as upstream: `git remote add upstream https://github.com/latextemplates/LNCS.git`\n3. Merge the branch `upstream/master` into your `master` branch: `git merge upstream/master`.\n\nAfter that you can use and push the `master` branch as usual.\nNotes on syncing with the upstream repository [are available from GitHub](https://help.github.com/articles/syncing-a-fork/).\n\n## FAQ\n\n### Q: ShareLaTeX outputs a warning regarding the llncs class\n\nShareLaTeX might output following warning:\n\n> LaTeX Warning: You have requested, on input line 8, version\n> 2018/03/10\' of document class llncs, but only version 2004/08/17 v2.14\n> LaTeX document class for Lecture Notes in Computer Science\'\n> is available.\n\nThe reason is that you did not use `llncs.cls` from a recent llncs2e.zip (which can be downloaded from <ftp://ftp.springernature.com/cs-proceeding/llncs/llncs2e.zip>), but a copy from somewhere else.\nPlease use the latest version offered by Springer.\n\n### Q: I get the error  `! pdfTeX error (font expansion): auto expansion is only possible with scalable fonts.`\n\nInstall the `cm-super` package using the MiKTeX package manager. Then, run `initexmf --mkmaps` on the command line. (Long description: http://tex.stackexchange.com/a/324972/9075)\n\n### Q: I need more space. What can I do?\n\nThe most simple solution to get more space is to exchange the font.\nYou can disable the [cfr-lm] package by the [newtx] package\'s `newtxtext` and `newtxmath`.\nThis effectively switches the font from Computer Modern to Times Roman.\n\nThis switch is prepared in the template.\nExchange\n\n```latex\n\\iftrue % use default-font\n```\n\nby\n\n```latex\n\\iffalse\n```\n\n### Q: How can I reformat my .tex files?\n\nExecute following command:\n\n```shell\nlatexindent -l -s -sl -w paper.tex\n```\n\n### Q: I want to obey the one-sentence-per-line rule. How can I do that?\n\nExecute following command:\n\n```shell\nlatexindent -m -l -s -sl -w paper.tex\n```\n\nAttention! This is work in progress and does not always produce best results.\n\n### Q: Is it possible to have a footer indicating that the paper is intended to be submitted/submitted/published?\n\nActivate the `llncsconf` package.\nThe possible options are listed in `paper.tex`.\n\n### Q: Can I also write in German?\n\nYes.\n\n1. Pass parameter `ngerman` to the document class:\n\n    ```latex\n    \\documentclass[english,runningheads,a4paper]{llncs}[2018/03/10]\n    ```\n\n1. Please search for `babel` in `paper.tex` and use\n\n    ```latex\n    \\usepackage[english,ngerman]{babel}\n    ```\n\n    instead of\n\n    ```latex\n    \\usepackage[ngerman,english]{babel}\n    ```\n\n## Development\n\nReindent:\n\n```shell\nlatexindent -y="indentPreamble:1,defaultIndent:\'  \'" -m -w paper.tex`\n```\n\nIf you like this work, please consider donating via [Liberapay](https://liberapay.com/koppor)!\n\n## Links\n\n* tex.stackexchange.com questions regarding LNCS: <https://tex.stackexchange.com/questions/tagged/lncs>\n* German: Hinweise zu Ausarbeitungen: <http://wiki.flupp.de/studium/ausarbeitungen>\n* Other templates: <https://latextemplates.github.io/>\n* Original LNCS demonstration (without the improvements): [llncs-dem.pdf](llncs-dem.pdf)\n* Original LNCS documentation (without the improvements): [llncs-doc.pdf](llncs-doc.pdf)\n\n  [booktabs]: https://ctan.org/pkg/booktabs\n  [cfr-lm]: https://www.ctan.org/pkg/cfr-lm\n  [cleveref]: https://ctan.org/pkg/cleveref\n  [csquotes]: https://www.ctan.org/pkg/csquotes\n  [hypcap]: https://www.ctan.org/pkg/hypcap\n  [hyperref]: https://ctan.org/pkg/hyperref\n  [latexindent]: https://ctan.org/pkg/latexindent\n  [microtype]: https://ctan.org/pkg/microtype\n  [minted]: https://ctan.org/pkg/minted\n  [natbib]: https://ctan.org/pkg/natbib\n  [newtx]: https://ctan.org/pkg/newtx\n  [paralist]: https://www.ctan.org/pkg/paralist\n  [pdfcomment]: https://www.ctan.org/pkg/pdfcomment\n  [upquote]: https://www.ctan.org/pkg/upquote\n\n  [JabRef]: https://www.jabref.org\n  [LanguageTool]: https://languagetool.org/\n  [TeXstudio]: http://texstudio.sourceforge.net/\n  [pygments]: http://pygments.org/\n\n  [llncs2e.zip]: ftp://ftp.springernature.com/cs-proceeding/llncs/llncs2e.zip\n'
