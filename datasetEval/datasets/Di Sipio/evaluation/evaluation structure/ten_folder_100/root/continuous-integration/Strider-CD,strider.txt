b'# ![Strider][logo]\n\n[![NPM][npm-badge-img]][npm-badge-link] [![Code Climate][cc-badge]][cc-badge-link] [![Dependency Status][david-badge]][david-badge-link] [![Build Status][travis-badge]][travis-badge-link]  \n[![Gitter][gitter-badge]][gitter-badge-link]\n[![Backers on Open Collective][backers-badge-img]](#backers)\n[![Sponsors on Open Collective][sponsors-badge-img]](#sponsors)\n\n---\n\n### Brilliant Continuous Deployment\n\n![Strider Screenshot][screenshot]\n[more screenshots][more-screenshots]\n\n\nOverview\n========\n\n`Strider` is an Open Source Continuous Deployment / Continuous Integration\nplatform. It is written in Node.JS / JavaScript and uses MongoDB as a backing\nstore. It is published under the BSD license.\n\nStrider is extremely customizable through plugins. Plugins can \n\n- add hooks to perform arbitrary actions during build.\n- modify the database schema to add custom fields.\n- register their own HTTP routes.\n- subscribe to and emit socket events.\n- create or modify user interfaces within Strider.\n- so much more! just use your imagination!\n\n## General Requirements\n\n- [nodejs] >= 6\n- [git] >= 2.0\n- [mongodb][mongo-download] (local or remote)\n- [node-gyp]\n\n_Note: Installing on OS X might require XCode to be installed._\n\n- The package `krb5-devel`/`libkrb5-dev` might have to be installed to resolve Kerberos related build issues on some systems.\n\n\n## Running on Infrastructure\n\nMake sure you have MongoDB installed on your system. You can get the latest version at [mongodb.org][mongo-download].\n\nNext you will need Node.JS. You can get binary packages for most platforms at [nodejs.org][nodejs].\n\nOnce you have Node.JS on your system, you can fetch & install all the dependencies for your Strider clone\nby executing the following command in the project root:\n\n```no-highlight\nnpm install\n```\n\n> Note: Sometimes there are issues with permissions and installing global modules, in those cases run `npm config set prefix ~/npm` to set your global modules directory to \'~/npm\'. You will also have to add `~/npm/bin` to your `PATH` environment variable.\n\n### Configuring\n\n`Strider` configuration comes from environment variables. Most of the default\nvalues should work fine for running on localhost, however for an\nInternet-accessible deployment the following variables will need to be exported:\n\n  - `SERVER_NAME` - **Required**; Address at which server will be accessible on the Internet. E.g. `https://strider.example.com` (note: no trailing slash, and included protocol)\n  - `HOST` - Host where strider listens, optional (defaults to 0.0.0.0).\n  - `PORT` - Port that strider runs on, optional (defaults to 3000).\n  - `CONCURRENT_JOBS` - How many jobs to run concurrently (defaults to 1). Concurrency only works across different project and branch combinations. So if two jobs come in for the same project and branch, concurrency will always be 1.\n  - `STRIDER_CLONE_DEST` - Where the repositories are cloned to (defaults to ~/.strider)\n  - `DB_URI` - MongoDB DB URI (with port number if local, e.g. localhost:27017) if not localhost (you can safely use [MongoLab free plan][mongolab] - works great)\n  - `HTTP_PROXY` - Proxy support, optional (defaults to null)\n  - If you want email notifications, configure an SMTP server (we recommend [Mailgun] for SMTP if you need a server - free account gives 200 emails / day):\n    - `SMTP_HOST` - SMTP server hostname e.g. smtp.example.com\n    - `SMTP_PORT` - SMTP server port e.g. 587 (default)\n    - `SMTP_SECURE` - SMTP server TLS or SSL ("true" or "false")\n    - `SMTP_USER` - SMTP auth username e.g. "myuser"\n    - `SMTP_PASS` - SMTP auth password e.g. "supersecret"\n    - `SMTP_FROM` - Default FROM address e.g. "Strider <noreply@stridercd.com>" (default)\n\n#### Additional Configurations\n\n- `BODY_PARSER_LIMIT` - Increase the maximum payload size that our [body parser][body-parser] will attempt to parse. Useful for github web hooks.\n- `DEBUG` - Set this to `strider*` to enable all debug output. This is very helpful when troubleshooting issues or finding the cause of bugs in Strider. For more information see https://www.npmjs.com/package/debug\n- `JOBS_QUANTITY_ON_PAGE_ENABLED` - Whether users can set quantity in Account Management\n- `JOBS_QUANTITY_ON_PAGE_DEFAULT` - Number of jobs to display when not enabled\n- `JOBS_QUANTITY_ON_PAGE_MIN` - Minimal value\n- `JOBS_QUANTITY_ON_PAGE_MAX` - Maximum value\n\nYou might need to follow these instructions if you use any of these, please do so before filing issues.\n\n- [Github][github-config]  \n- [Bitbucket][bitbucket-config]  \n- [Gitlab][gitlab-config]  \n- [Heroku][heroku-config]  \n\n### Adding Initial Admin User\n\n`Strider` isn\'t much use without an account to login with. Once you create an administrative user, you can invite as many\nother people as you like to your instance. There is a simple CLI subcommand to help you create the initial user:\n\n```no-highlight\nnode bin/strider addUser\n```\n\nExample run:\n\n```no-highlight\n$ node bin/strider addUser\nEnter email []: strider@example.com\nIs admin? (y/n) [n]: y\nEnter password []: *******\n\nEmail:    strider@example.com\nPassword: ****\nisAdmin:  true\nOK? (y/n) [y]:\n22 Oct 21:21:01 - info: Connecting to MongoDB URL: mongodb://localhost:27017/strider-foss\n22 Oct 21:21:01 - info: User added successfully! Enjoy.\n```\n\nSee the [strider-cli] for more details.  \n\nIf you want to connect to your ldap server to authorization  \nyou can also add the `ldap.json` config file to project root  \nthe config like so:  \n\n```javascript\n {\n    "url": ldap://host:port,\n    "baseDN": dnString,\n    "username": username,\n    "password": password,\n    // If you want to set a admin group\n    "adminDN": dnString\n }\n```\n\n### Starting Strider\n\nOnce `Strider` has been installed and configured, it can be started with:\n\n```no-highlight\nNODE_ENV=production npm start\n```\n\n### Strider on Heroku\n\n[![Deploy](https://www.herokucdn.com/deploy/button.png)](https://heroku.com/deploy)\n\nTo get up and running quickly on Heroku, you can simply use the button above.\nIf you run into any issues, see the [wiki entry](https://github.com/Strider-CD/strider/wiki/Strider-on-Heroku).\n\n\n### Strider in Docker\n\nMany users like to run Strider within a Docker container.\nAlthough this works well, supporting it is outside the scope of the Strider project.\n\nWe recommend using [docker-strider](https://github.com/Strider-CD/docker-strider) as a base image when designing your Docker-based Strider installation.\nPlease post related issues in the [issues section](https://github.com/Strider-CD/docker-strider/issues) for that repository.\n\n\n## Resources\n\n- [Strider Tutorial Series][resource-strider-futurestudio-tutorials] - Extensive guides about Strider covering platform setup, 3rd party integrations (GitHub, GitLab, etc), continuous deployments (Heroku, SSH), notifications (email, Slack, HipChat), how to create your own Strider plugin and many more.\n- [Strider on DigitalOcean][resource-digitalocean] - Covers setting up an Ubuntu machine with Strider using upstart.\n- [Strider plugin template][resource-plugin-template] - Simple setup for getting started with your own plugin.\n- [Panamax Strider template][resource-panamax-template] - Strider template for use with Panamax.\n\n\n## Advanced Topics\n\nAdvanced topics are located in the [Wiki](https://github.com/Strider-CD/strider/wiki), here\'s a small\nsubset of what\'s covered:\n\n- [Advanced Configuration](https://github.com/Strider-CD/strider/wiki/Advanced-Configuration)\n- [Requiring Strider](https://github.com/Strider-CD/strider/wiki/Requiring-Strider)\n- [Managing Plugins](https://github.com/Strider-CD/strider/wiki/Managing-Plugins)\n\n## API Documentation\n\nAn effort has been started to document the existing REST API, and to have versioned documentation going forward.\nWe use [apiDoc] for the documentation.\n\nTo build the documentation run `npm run docs` and the documentation will be accessable from `apidocs/index.html`.\n\n**[View Strider API Docs](http://strider-api-docs.surge.sh/)**\n\n## Backers\n\nSupport us with a monthly donation and help us continue our activities.
