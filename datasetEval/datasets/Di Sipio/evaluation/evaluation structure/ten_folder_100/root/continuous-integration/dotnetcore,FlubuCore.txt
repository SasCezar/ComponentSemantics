You can define your build and deployment scripts in C# using an intuitive fluent interface. This gives you code completion, IntelliSense, debugging, FlubuCore custom analyzers, and native access to the whole .NET ecosystem inside of your scripts.\n\n![FlubuCore in action](https://raw.githubusercontent.com/flubu-core/flubu.core/master/assets/demo.gif)\n\nFlubuCore offers a .net (core) console application that uses power of roslyn to compile and execute scripts. Above example can be run from console with:\n\n* FlubuCore runner (.NET 4.62+)  ``` flubu.exe Default ```\n* FlubuCore dotnet cli tool (.NET Core 1.0+) ``` dotnet flubu Default ```\n* FlubuCore global tool (.NET Core 2.1+) ``` flubu Default ```\n## Features and Advantages\n\n* Intuitive an easy to learn. C#, fluent interface, and IntelliSense make even most complex script creation a breeze.\n\n```cs\ncontext.CreateTarget("Example")\n  .DependsOn(fetchBuildVersionTarget)\n  .AddTask(x => x.CompileSolutionTask())\n  .AddTask(x => x.PublishNuGetPackageTask("packageId", "pathToNuspec"))\n      .When(c => c.BuildSystems().Jenkins().IsRunningOnJenkins);\n```\n          \n* [Large number of often used built-in tasks](https://flubucore.dotnetcore.xyz/tasks/) like e.g. running tests, managing IIS, creating deployment packages, publishing NuGet packages, docker tasks, executing PowerShell scripts and many more.\n\n```cs\ncontext.CreateTarget("build")\n   .AddTask(x => x.GitVersionTask())\n   .AddTask(x => x.CompileSolutionTask("MySolution.sln").BuildConfiguration("Release");\n\ncontext.CreateTarget("run.tests")\n   .AddTask(x => x.XunitTaskByProjectName("MyProject").StopOnFail())\n   .AddTask(x => x.NUnitTask(NunitCmdOptions.V3, "MyProject2").ExcludeCategory("Linux"))\n   .AddCoreTask(x => x.CoverletTask("MyProject.dll"));\n```\n\n* [Execute your own custom C# code.](https://flubucore.dotnetcore.xyz/buildscript-fundamentals#Custom-code)\n\n```cs\ncontext.CreateTarget("MyCustomBuildTarget")\n     .AddTask(x => x.CompileSolutionTask())\n     .Do(MyCustomMethod)\n     .Do(NuGetPackageReferencingExample);\n```\n\n* [assembly references and nuget packages are loaded automatically](https://flubucore.dotnetcore.xyz/referencing-external-assemblies/) when script is used together with project file. When script is executed alone (for example when deploying with FlubuCore script on production environment) references can be added with attributes.\n\n```cs\n[NugetPackage("Newtonsoft.json", "11.0.2")]\n[Assembly(".\\Lib\\EntityFramework.dll")]\npublic class BuildScript : DefaultBuildScript\n{\n   public void NuGetPackageReferencingExample(ITaskContext context)\n    {\n        JsonConvert.SerializeObject("Example");\n    }\n}\n```\n\n* [Easily run any external program or console command in your script.](https://flubucore.dotnetcore.xyz/buildscript-fundamentals#Run-any-program)\n\n```cs\ncontext.CreateTarget("Run.Libz")\n    .AddTask(x => x.RunProgramTask(@"packages\\LibZ.Tool\\1.2.0\\tools\\libz.exe")\n        .WorkingFolder(@".\\src")\n        .WithArguments("add")\n        .WithArguments("--libz", "Assemblies.libz"));\n```\n\n* [Pass command line arguments, settings from json configuration file or environment variables to your script.](https://flubucore.dotnetcore.xyz/buildscript-fundamentals#Script-arguments)\n\n ```cs\n public class SimpleScript : DefaultBuildScript\n {\n    [FromArg("sn", "If true app is deployed on second node. Otherwise not.")]\n    public bool deployOnSecondNode { get; set; }\n\n \n     protected override void ConfigureTargets(ITaskContext context)\n     {\n         context.CreateTarget("compile")\n            .AddTask(x => x.CompileSolutionTask()\n                .ForMember(y => y.SolutionFileName("someSolution.sln"), "solution", "The solution to build.")); \n     }\n  }\n ```\n \n ```\n  flubu.exe compile -solution=someOtherSolution.sln -sn=true\n ```\n* [Extending FlubuCore fluent interface by writing your own tasks within FlubuCore plugins.](https://flubucore.dotnetcore.xyz/write-plugins)\n\n    ```cs\n    public class ExampleFlubuPluginTask : TaskBase<int, ExampleFlubuPluginTask>\n    {\n        protected override int DoExecute(ITaskContextInternal context)\n        {\n            // Write your task logic here.\n            return 0;\n        }\n    }\n    ```\n* [Growing list of FlubuCore plugins complements built in tasks.](https://flubucore.dotnetcore.xyz/AwesomePlugins/awesome-plugins/)\n\n* [Asynchronous or parallel execution of tasks, target dependencies and custom code.](https://flubucore.dotnetcore.xyz/buildscript-fundamentals#Async-execution)\n\n    ```cs\n    context.CreateTarget("Run.Tests")\n        .AddTaskAsync(x => x.NUnitTaskForNunitV3("TestProjectName1"))\n        .AddTaskAsync(x => x.NUnitTaskForNunitV3("TestProjectName1"))\n        .AddTaskAsync(x => x.NUnitTaskForNunitV3("TestProjectName3"));\n    ```\n\n* [Override existing options or add additional options to tasks through console](https://flubucore.dotnetcore.xyz/override-add-options/)\n\n   ```c#\n  context.CreateTarget("Example")`\n      .AddCoreTask(x => x.Build("MySolution.sln").Configuration("Release"); \n   ``` \n\n  `flubu example --configuration=Debug`\n\n   flubu would execute `dotnet build MySolution.sln --configuration Debug`    \n\n* [Full .NET Core support including the global CLI tool](https://flubucore.dotnetcore.xyz/getting-started#getting-started-net-core)\n\n    ```\n    dotnet tool install --global FlubuCore.GlobalTool\n    flubu compile\n    ```\n    \n* [FlubuCore interactive mode](https://flubucore.dotnetcore.xyz/build-script-runner-interactive/) which offers target tab completition, options tab completition, toogle targets/options, executed commands history. It is also possible to execute external commands and operable programs. For some of them FlubuCore offers tab completion with help displayed at the bottom of console out of the box(such as dotnet, git..)\n\n![FlubuCore interactive mode](https://raw.githubusercontent.com/dotnetcore/flubu.core/master/assets/FlubuCore_Interactive_mode_full.gif)\n\n* [Possibility to test and debug your build scripts.](https://flubucore.dotnetcore.xyz/Tests-debugging)\n\n    ```cs\n    context.WaitForDebugger();\n    ```\n\n* [Easily automate deployments remotely via the FlubuCore Web API.](https://flubucore.dotnetcore.xyz/WebApi/getting-started/)\n\n* [Possibility to use FlubuCore tasks in any other .NET application.](https://github.com/flubu-core/examples/blob/master/NetCore_csproj/BuildScript/BuildScriptTests.cs)\n\n* Improved developer experience with FlubuCore custom analyzers.\n\n![FlubuCore analyzers in action](https://raw.githubusercontent.com/flubu-core/flubu.core/master/assets/FlubuCoreCustomAnalyzerDemo.png)\n\n## Getting Started\nUsing FlubuCore is straightforward and very simple :-) It is also fully and throughly documented.\n\nThe [Getting Started](https://flubucore.dotnetcore.xyz/getting-started/) chapter in [FlubuCore Documentation](https://flubucore.dotnetcore.xyz) will help you set up your first FlubuCore build in no time.\n\nA comprehensive list of features that FlubuCore has to offer with descriptions can be found in the [Build Script Fundamentals](https://flubucore.dotnetcore.xyz/buildscript-fundamentals) chapter.\n\nOnce you have your build and deployment scripts defined, the following Wiki chapters will explain how to run them:\n* For .NET Framework projects use [FlubuCore.Runner](https://flubucore.dotnetcore.xyz/getting-started#Installation.net)\n* For .NET Core projects use [FlubuCore CLI global tool](https://flubucore.dotnetcore.xyz/getting-started#Installation-.net-core)\n\n## Examples\nAside from the detailed Wiki FlubuCore comes with example projects that reflect real-life situations.
