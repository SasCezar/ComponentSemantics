b'![monitorr](https://raw.githubusercontent.com/Monitorr/Monitorr/master/assets/img/monitorrbanner.png)\r\n\r\n\r\n## - *Monitorr* is a webfront to live display the status of any webapp or service\r\n\r\n[![GitHub Master-release](https://img.shields.io/github/release/monitorr/monitorr.svg?style=flat)](https://github.com/monitorr/monitorr/releases) [MASTER]\r\n\r\n[![Docker build](https://img.shields.io/docker/build/monitorr/monitorr.svg?maxAge=2592000)](https://hub.docker.com/r/monitorr/monitorr/) [DOCKER]\r\n\r\n[![GitHub (pre-)release](https://img.shields.io/github/release/monitorr/monitorr/all.svg)](https://github.com/monitorr/monitorr/releases) [DEVELOP]\r\n\r\n\r\n**NOTICE (27 June 18)**: If updating to version **1.7** from ANY previous version, **BEFORE updating backup your custom.css file** located at: _`[Monitorr install path]/assets/data/css/custom.css`_ . After the update is complete, simply restore the custom.css file to the same location overwriting the custom.css file that was updated. This is due to a bug which has been resolved in version 1.7 therefore this process will no longer be needed in future updates. See the [WiKi Update Notice](https://github.com/Monitorr/Monitorr/wiki/NOTICE:-Updating-Monitorr) for more information.\r\n\r\n\r\n## Features:\r\n\r\n**Latest major change:** Multi HD display / Service ping response time display\r\n\r\n- LIVE! (w/ option to pause live updating)\r\n- Self-hosted.\r\n- Responsive mobile display.\r\n- Monitor any app/service on any domain (Curl as primary check, fsockopen as fallback).\r\n- Enable/disable hot linking for each service.\r\n- Service ping response time display. (NEW)\r\n- Host system resources display (CPU, MEM, HD, PING, Uptime) (UPDATED)\r\n- Integrated settings page w/ authentication.\r\n- User customizable system threshold colors.\r\n- Custom CSS tool. (NEW)\r\n- Server DTG data. (UPDATED)\r\n- Update tool via web UI / branch switching w/ update notification.\r\n- Minimal UI for iFrame displays. (See: [WIKI](https://github.com/Monitorr/Monitorr/wiki/05-Integration:--Organizr))\r\n\r\n\r\n**Features in development:**\r\n\r\n- Python daemon (See: [Features 160](https://github.com/Monitorr/Monitorr/issues/160))\r\n- Alerting (E-Mail, text, etc) (See: [Features 126](https://github.com/Monitorr/Monitorr/issues/126))\r\n\r\n\r\n## Screenshots:\r\n\r\n![Monitorr Screenshot](https://i.imgur.com/2wGLfLp.png)\r\n<img src="https://i.imgur.com/UxDNcct.jpg?1" width="48%"> <img src="https://i.imgur.com/0smwRRu.jpg?1" width="50%">\r\n<img src="https://i.imgur.com/BpBplql.png?1" width="32%">  <img src="https://i.imgur.com/PWWpDkn.jpg?1" width="32%"> <img src="https://i.imgur.com/Mnu0QJg.png?1" width="32%"> \r\n\r\n\r\n### Mobile:\r\n\r\n![Monitorr Mobile SS](https://i.imgur.com/JLANkme.jpg?3)\r\n\r\n\r\n## Prerequisites:\r\n\r\n1) [PHP](https://secure.php.net/downloads.php) (7.1+ recommended)\r\n2) [PHP cURL](https://secure.php.net/manual/en/book.curl.php)\r\n3) [PHP ZipArchive](http://www.php.net/manual/en/zip.installation.php)\r\n4) [PHP PDO](http://php.net/manual/en/book.pdo.php)\r\n5) [SQLite](https://www.sqlite.org/index.html)\r\n6) [GIT](https://git-scm.com/download/win) (Recommended for Windows hosts (see wiki)\r\n\r\n\r\n## Quick Start:\r\n\r\n- See full configuration instructions in the WiKi: https://github.com/Monitorr/Monitorr/wiki\r\n\r\n1) Clone/download the Monitorr repository to your webserver.
