b'# Meemo\n\nMeemo is a personal data manager. It lets you simply input any kind of information like notes, thoughts, ideas as well as acts as a bookmarkmanager and todo list.\nThe user interface resembles a news feed organized with tags. Full text search further allows you to quickly find information in your pile of accumulated data.\n\nFor better bookmarking, there are chrome and firefox webextensions available.\n\nSupport meemo development: [![Flattr Meemo](https://button.flattr.com/flattr-badge-large.png)](https://flattr.com/submit/auto?user_id=cloudron&url=https://cloudron.io&title=Cloudron&tags=opensource&category=software)\n\n[![Build Status](https://travis-ci.org/nebulade/meemo.svg?branch=master)](https://travis-ci.org/nebulade/meemo)\n\n## Installation\n\n[![Install](https://cloudron.io/img/button32.png)](https://cloudron.io/button.html?app=de.nebulon.guacamoly)\n\nor using the [Cloudron command line tooling](https://cloudron.io/references/cli.html)\n\n```\ncloudron install --appstore-id de.nebulon.guacamoly\n```\n\nTo you run Meemo outside a Cloudron environment, those dependencies are required:\n```\nnodejs  >= 4.1.1\nmongodb >= 2.6\n```\nand the installation consist of:\n```\ncd meemo\nnpm i\n./node_modules/.bin/gulp\n./app.js\n```\n\nPossible env variables for configuration are:\n```\nPORT=3000\nBIND_ADDRESS=0.0.0.0\nAPP_ORIGIN="https://example.com"\nMONGODB_URL="mongodb://username:password@127.0.0.1:27017/meemo" # username and password are optional\nATTACHMENT_DIR="./storage"\n\n# using LDAP user management\nLDAP_URL="ldap://my.ldap.server"\nLDAP_USERS_BASE_DN="ou=users,dc=example"\nLDAP_BIND_DN="cn=admin,ou=users,dc=example"\nLDAP_BIND_PASSWORD=""\n\n# using local file user management via admin cli tool\nLOCAL_AUTH_FILE=".users.json"\t# also pass this for the admin tool to find the correct file\n\n# to enable email receiving\nMAIL_IMAP_SERVER="my.mail.server"\nMAIL_IMAP_PORT=993\nMAIL_IMAP_USERNAME=""\nMAIL_IMAP_PASSWORD=""\nMAIL_DOMAIN="example.com"\n```\n\n## Building\n\nThe app package can be built using the [Cloudron command line tooling](https://cloudron.io/references/cli.html).\n\n```\ncd meemo\n\ncloudron build\ncloudron install\n```\n\n## Development\n\nThe app can also be run locally for development. It depends on a locally running mongodb and optionally on an instance of the [ldap test server](https://github.com/nebulade/ldapjstestserver).\n\n```\ncd meemo\n\nnpm install\n\n# with LDAP\nLDAP_BIND_DN="cn=admin,ou=users,dc=example" LDAP_BIND_PASSWORD="password" LDAP_USERS_BASE_DN="ou=users,dc=example" LDAP_URL="ldap://localhost:3002" ./app.js\n\n# without LDAP\n./admin user-add --username test --password test --display-name "Test User"\n./app.js\n```\n'
