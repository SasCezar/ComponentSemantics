?\n\nFerdi is a messaging browser that allows you to combine your favorite messaging services into one application. It is based on Franz - a software already used by thousands of people - with the difference that Ferdi gives you many additonal features and doesn\'t restrict its usage! Ferdi is compatible with your existing Franz account so you can continue right where you left off. Find out more about Ferdi and its features on [getferdi.com](https://getferdi.com).\n\n### What Ferdi looks like ?\n\n<details>\n<summary>Screenshots</summary>\n<p align="center">\n<img alt="Keep all your messaging services in one place." src="./branding/screenshots/hero.png">\n<em>"Keep all your messaging services in one place."</em>\n<img alt="Order your services with Ferdi Workspaces." src="./branding/screenshots/workspaces.png">\n<em>"Order your services with Ferdi Workspaces."</em>\n<img alt="Always keep your Todo list open with Ferdi Todos." src="./branding/screenshots/todos.png">\n<em>"Always keep your Todo list open with Ferdi Todos."</em>\n<img alt="Supporting all your services." src="./branding/screenshots/service-store.png">\n<em>"Supporting all your services."</em>\n</p>\n</details>\n\n## Download Ferdi\n\nYou can find the installers in the [latest stable release](https://github.com/getferdi/ferdi/releases/latest) assets and [all the other release here](https://github.com/getferdi/ferdi/releases).\n\n### Or use homebrew (macOS only)\n\n`$ brew cask install ferdi`\n\n(Don\'t know homebrew? [brew.sh](https://brew.sh/))\n\n### Or use AUR (Arch Linux)\n\nFerdi has two seperate AUR packages you can use:\n- **ferdi-build**: Uses your debian build and extracts it to Arch\n- **ferdi-git**: Uses system electron version\n\nIf you use a AUR Helper e.g. yay, simply install it via \xc2\xb4yay -S ferdi-bin\xc2\xb4\n\n`ferdi-git` may not work on all systems so we advice you to use `ferdi-bin` instead.\n\n## Ferdi-specific Features\n\n- [x] Removes the counter-productive fullscreen app delay inviting users to upgrade\n- [x] Removes pages begging you to donate after registration\n- [x] branding proper to Ferdi\n\n## Development\n\n### Install OS dependencies\n\n#### Node.js\n\nPlease make sure you are running NodeJS v10 ([v10.16.3](https://nodejs.org/dist/v10.16.3/) suggested). Versions above will throw an errow when trying to install due to an [old fsevent dependency](https://github.com/fsevents/fsevents/issues/278).\n\n#### Git\n\nThe version [2.23.0](https://github.com/git-for-windows/git/releases/tag/v2.23.0.windows.1) for Git is working fine for development. You can then use the console from Git to do the development procedure.\n\n#### Debian/Ubuntu\n\n```bash\n$ apt install libx11-dev libxext-dev libxss-dev libxkbfile-dev\n```\n\n#### Fedora\n\n```bash\n$ dnf install libX11-devel libXext-devel libXScrnSaver-devel libxkbfile-devel\n```\n\n#### Windows\n\n```bash\n$ npm install --global windows-build-tools --vs2015\n```\n\n### Clone repository with submodule\n\n```bash\n$ git clone https://github.com/getferdi/ferdi.git\n$ cd ferdi\n$ git submodule update --init --recursive\n```\n\nIt is important you execute the last command to get the required submodules (recipes, server).\n\n### Install dependencies\n\nRun the following command to install all dependencies, and link sibling modules with Ferdi.\n\n```bash\n$ npx lerna bootstrap\n```\n\nIf you previously ran `npm install` it sometimes is necessary to delete your `node_modules` folder before running `npx lerna bootstrap`.\n\n### Fix native modules to match current electron node version\n\n```bash\n$ npm run rebuild\n```\n\n### Start development app\n\nRun these two commands **simultaneously** in different console tabs:\n\n```bash\n$ npm run dev\n$ npm run start\n```\n\nBe aware that the development database will be reset regularly.\n\n### Packaging\n\n```bash\n$ npm run build\n```\n\nDeliverables will be available in the `out` folder.\n\n### Release\n\n```bash\n$ git checkout develop && git pull && git checkout master\n$ git merge --no-ff develop\n$ git tag v5.3.4-beta.4\n$ git push --tags\n```\n\nWhen pushing a new tag, the CI builds will create a draft GitHub release and upload the deliverables in the draft release assets. Wait for all the assets to be uploaded before publishing the draft release.\n\n## Contributors \xe2\x9c\xa8\n\nThanks goes to these wonderful people ([emoji
