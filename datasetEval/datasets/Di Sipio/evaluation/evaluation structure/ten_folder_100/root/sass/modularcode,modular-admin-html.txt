[Download ZIP](https://github.com/modularcode/modular-admin-html/releases/latest) or Git Clone\n\n```\ngit clone https://github.com/modularcode/modular-admin-html.git\n```\n### 2. Build the project\n\nThe cloned/downloaded repository doesn\'t contain prebuilt version of the project and you need to build it. You need to have [NodeJs](https://nodejs.org/en/) (v8+) with npm (v3+) installed.\n\n\nInstall npm dependencies \n```\nnpm install\n```\n\nBuild the project and start local web server\n```\nnpm start\n```\n\nOpen the project [http://localhost:4000](http://localhost:4000).\n\n\n**Warning!** all changes made in ```dist/``` folder would be overwriten on application build.\n\n<br>\n\nYou can also <strong>[run the project in docker](#running-in-docker)</strong> thanks to @japrogramer\n\n<br>\n\n## Folder Structure\n\n```\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 build/               # app build tasks and tools\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 config/              # build configs and paths definitions\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 dist/                # compiled result\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 node_modules/        # node dependencies        \n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 src/                 # source files\n\xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 package.json         # npm configuration file\n```\n\n#### ```config/``` folder\n\nThis folder contains application build configurations and paths definitions.\nFor **adding/removing NPM dependencies** you need to **manually define the paths** in `config/index.js` file after the module installation.\n\n#### ```build/``` folder\n\nThis folder contains files related to our application compilation. That can be styles preprocessing (LESS,SASS,PostCSS) and template engine compilation, script file concatenation and minification and other related tasks.\n\n```\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 tasks/                           # tasks folder\n|   \xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 {different tasks}            # each file represents a single build task\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 utils/                           # some utils\n\xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 gulpfile.js                      # main build file for gulp build system\n\n```\n\n\n#### ```src/``` folder\n\nThis folder contains our application source files. \nThe folder structure reflects the app component structure.\n\n\nEach non-underscored folder represents a single component module. Modules can be nested inside each other.\n\nThere are also special folders which start with an underscore. \nFor example ```_common/``` folder contains common components that are used by other components at the same level.\n\nThis file structuring makes our app file organization very semantic and scalable. Also It\'s very easy to work on separate components even if you\'re developing large-scale applications.\n\n```\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 _assets/                           # application assets\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 _common/                           # common components\n|   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 helpers/                       # handlebars helpers\n|   \xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 styles/                        # application common styles\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 _themes/                           # different theme versions\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 app/                               # app module (dashboard view)\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 _common/                       # app common components\n\xe2\x94\x82   |   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 editor/                    # wysiwyg editor files\n\xe2\x94\x82   |   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 footer/                    # footer files\n\xe2\x94\x82   |   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 header/                    # header files\n\xe2\x94\x82   |   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 modals/                    # common modal dialogs (confirm, image library, etc)\n\xe2\x94\x82   |   \xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 sidebar/                   # sidebar files\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 {different modules}\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 app-layout.hbs                 # app view layout\n\xe2\x94\x82   \xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 app.scss                       # main app view styles\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 auth/                              # auth module (login/signup/recover)\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 {different modules}\n\xe2\x94\x82   \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 auth-layout.hbs                # auth view layout\n\xe2\x94\x82   \xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 auth.scss                      # main auth view styles\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 _context.js                        # main handlebars variables\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 _main.scss                         # main styles\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 _variables.scss                    # variables\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 config.js                          # javascript configs\n\xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 main.js                            # main script file\n\n```\n\n\n\n#### ```dist/``` folder\n\nCompiled state of our app with processed styles, templates, scripts and assets.\n\n**Warning! Never work inside this folder, because your changes would be overwritten on every build**\n\n<br>\n\n## File Types\n\nOur app consists of different file types.\n\n#### Styles (*.scss)\n\nWe use [SASS](http://sass-lang.com/) as CSS preprocessor language. \nMain variables are defined in ```src/_variables.scss``` folder. \nFor making life easier we broke down styles into components, and on build we\'re just merging all ```.scss``` files together and processing it to ```dist/css/app.css``` file. Style files are merged in the following order\n\n```\n{variables.scss}\n{bootstrap variables}\n{bootstrap mixins}\n{rest style files}\n```\nThe remaining style files are merged in the alphabetical order.\n\nThere are also different theme variations located in ```src/_themes/ folder```, where you can change the main variables to get different themes. There are a few predefined themes built in. You can add new themes by adding a new file in ```src/_themes/``` folder. The file name must end with ```-theme.scss```.\n\n#### Scripts (*.js)\n\nWe separate application\'s scripts across its components. For simplicity we use ES5 in this version, and just wrap each component\'s script in jQuery ```$(function() { })```. JS configurations are defined in ```src/config.js``` file. On build, application script files are merged together and copied to ```dist/js/app.js``` folder. The script files are merged in the following order.\n\n```\n{config.js}\n{all .js files except main.js}\n{main.js}\n```\n\n#### Templates (*.hbs)\n\nTemplates are pieces of HTML files written in template engine language. We use [Handlebars](http://handlebarsjs.com/), which allows to have conditions in HTML, reuse partials in different pages (e.g. sidebars, footers), use loops, layouts etc.\n\n#### Pages (*-page.hbs)\n\nTemplates themselves are just parts of the markup, and aren\'t compiled as separate files. What we really want in the final output is a ```.html``` page in the ```dist/``` folder. There are special handlebar templates for it, their filenames ending with ```-page.hbs```. Each ```{pagename}-page.hbs``` file would be compiled to ```dist/{pagename}.html``` page with a flatened file structure.\n\nPages can consist of different templates (partials) which can be included thanks to handlebars partial including feature. Also each page has its context, which is a data passed into the template on rendering. That data is used in template expressions and variables. page contexts can be defined in two ways:\n\n**YAML** headers ([example](https://github.com/modularcode/modular-admin-html/blob/master/src/app/dashboard/index-page.hbs))\n\n```\n---\nfoo: bar\nlist: \n  - One\n  - Two\n  - Three\n---\n```\nand **_context.js** files.\n```\nmodule.exports = {\n  foo: \'bar\',\n  foo2: function() {\n    // do some magic, return some string\n  },\n  list: [\n    \'One\', \'Two\', \'Three\'\n  ]\n}\n```\n\nThe final result of page context is a combination of both ways. Moreover, different depth level _context.js files are extending each other and then are extended with YAML headers data. For simplicity we use only **YAML** headers.\n\n#### Layouts (*-layout.hbs)\n\nIf different pages have a lot of common components like sidebars, headers, footers, then it\'s a good idea to define a layout for those common pages, and define in page files only the content which is unique.\n\nLayout is a page content wrapper. If the page has a layout in output we\'ll get page\'s content inserted into the layout. Layouts should have ```{{{body}}}``` handlebars tag, which is entry point for the page content. ([example](https://github.com/modularcode/modular-admin-html/blob/master/src/app/app-layout.hbs))\n\nTo define a page layout you need to specify page file context\'s ```layout``` variable. It can be done both with a YAML header or a _context.js file. ([example](https://github.com/modularcode/modular-admin-html/blob/master/src/app/forms/forms-page.hbs)).\n\nLayouts can also have contexts and parent layouts.\n\n```\n{_main-layout.hbs}                  # main layout with doctype, head, scripts declaration\n    {app/app-layout.hbs}            # dashboard layout with sidebar, header and footer\n        {app/forms/forms-page.hbs}  # any dashboard page\n```\n\n\nIf you need more advanced layouting with multiple content blocks at the same time you can use [handlebar-layouts](https://www.npmjs.com/package/handlebars-layouts) helper approach, which is also available out of the box.\n\n<br>\n\n## Running in Docker\n\nYou can run the project in docker. To build the container, you need to install docker and docker-compose than launch the docker daemon. After launching the daemon run the following commands from the project folder:\n\nBuild the image\n```\ndocker-compose build\n```\n\nLaunch the container\n```\ndocker-compose up\n```\n\n\n## Support me!\n\n### [Support me on Patreon](https://www.patreon.com/modularcoder)\n\nCreating a good quality project takes a lot\'s of time, so any help is really appreciated!\n\n[https://www.patreon.com/modularcoder](https://www.patreon.com/modularcoder)\n\n### Contribute\n\nBe part of our team!
