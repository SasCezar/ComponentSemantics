b'A [Select2](https://select2.github.io/) v4 [Theme](https://select2.github.io/examples.html#themes) for Bootstrap 3  \n![select2-bootstrap-theme version](https://img.shields.io/badge/select2--bootstrap--theme-v0.1.0--beta.10-brightgreen.svg)\n[![License](http://img.shields.io/badge/License-MIT-blue.svg)](http://opensource.org/licenses/MIT)\n\nDemonstrations available at\n[select2.github.io/select2-bootstrap-theme](http://select2.github.io/select2-bootstrap-theme/)\n\n#### Compatibility\n\nBuilt and tested with Bootstrap v3.3.7 and Select2 v4.0.3 in latest Chrome, Firefox and Safari (Mac) and Internet Explorer 11, 10 and 9 (should work in IE8).\n\n#### Installation\n\nYou can [download select2-bootstrap-theme from this GitHub repo](https://github.com/select2/select2-bootstrap-theme/releases), install it using Bower or npm \xe2\x80\x93 required if you want to integrate the Sass or Less sources in your build process \xe2\x80\x93 or source the compiled CSS directly from CDNJS or jsDelivr.\n\n##### Install using Bower or npm/yarn\n\nYou may install select2-bootstrap-theme with [Bower](https://bower.io/), [npm](https://www.npmjs.com/) or [Yarn](https://yarnpkg.com/):\n\n```shell\n// Bower\nbower install select2-bootstrap-theme\n\n// npm\nnpm install select2-bootstrap-theme\n\n// yarn\nyarn add select2-bootstrap-theme\n```\n\n##### Source select2-bootstrap-theme from CDNJS or jsDelivr\n\nselect2-bootstrap-theme [is also available on CDNJS](https://cdnjs.com/libraries/select2-bootstrap-theme/) and [jsDelivr](http://www.jsdelivr.com/projects/select2-bootstrap-theme).\n\n#### Usage\n\nselect2-bootstrap-theme only works with Select2 v4.x. Applying the theme requires `select2-bootstrap.css` referenced after the default `select2.css` that comes with Select2:\n\n```html\n<link rel="stylesheet" href="select2.css">\n<link rel="stylesheet" href="select2-bootstrap.css">\n```\n\nTo apply the theme, tell Select2 to do so by passing `bootstrap` to the [`theme`](https://select2.github.io/examples.html#themes) option when initializing Select2:\n\n```js\n$( "#dropdown" ).select2({\n    theme: "bootstrap"\n});\n```\n\nYou may also set it as the default theme for all Select2 widgets like so:\n\n```js\n$.fn.select2.defaults.set( "theme", "bootstrap" );\n```\n\n#### Changelog\n\n##### 0.1.0-beta.10\n\n* Compiled with grunt-sass v2.0.0 (was v1.2.1).\n* Merged pull request [#54](https://github.com/select2/select2-bootstrap-theme/pull/54) by @odahcam (and fixed it: `:first-child/:not(:first-child)/:last-child` for `.select2-container--bootstrap` won\xe2\x80\x99t play nice in our case; The demo pages are built using [Jekyll](http://jekyllrb.com/) and there are a bunch of [Grunt](http://gruntjs.com/) tasks to ease development.\n\nWith [Jekyll](http://jekyllrb.com/), [node.js](http://nodejs.org/) and [Less](http://lesscss.org/) installed, run\n\n```sh\nnpm install\n```\n\nto install all necessary development dependencies (Sass is compiled v[LibSass](https://github.com/sass/libsass/)/[node-sass](https://github.com/sass/node-sass)).\n\n * `grunt build` builds `docs`\n * `grunt serve` builds `docs` and serves them via Jekyll\'s `--watch` flag on http://localhost:3000\n\nDevelop in `src/select2-bootstrap.scss` and test your changes using `grunt serve`. Ideally, port your changes to `lib/select2-bootstrap.less` and make sure tests are passing to verify that both Less and Sass compile down to the target CSS via `npm test`.\n\n`grunt scss2less` helps in converting the Sass source to its Less counterpart (and overwrites the existing `src/select2-bootstrap.less`), but doesn\'t do the full job \xe2\x80\x93 please review the changes to the Less source file and make the necessary adjustments.\n\n#### Copyright and license\n\nThe license is available within the repository in the [LICENSE](LICENSE) file.\n'
