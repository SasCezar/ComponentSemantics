b'<img src="assets/logo.svg" height="50px"/> The Futhark Programming Language\n==========\n\n[![Join the chat at https://gitter.im/futhark-lang/Lobby](https://badges.gitter.im/futhark-lang/Lobby.svg)](https://gitter.im/futhark-lang/Lobby?utm_source=badge&utm_medium=badge&utm_campaign=pr-badge&utm_content=badge)[![Build Status](https://travis-ci.org/diku-dk/futhark.svg?branch=master)](https://travis-ci.org/diku-dk/futhark)[![Build Status](https://dev.azure.com/futhark/futhark/_apis/build/status/diku-dk.futhark?branchName=master)](https://dev.azure.com/futhark/futhark/_build/latest?definitionId=1&branchName=master)\n\nFuthark is a purely functional data-parallel programming language.\nIts optimising compiler is able to compile it to typically very\nperformant GPU code.  The language and compiler is developed at\n[DIKU](http://diku.dk) at the University of Copenhagen, originally as\npart of the [HIPERFIT centre](http://hiperfit.dk).  The language and\ncompiler is quite stable, and suitable for practical programming.\n\nFor more information, see [the website](http://futhark-lang.org).\n\nThere also exists a book, [Parallel Programming in Futhark](https://futhark-book.readthedocs.io/en/latest/), that serves as an extensive introduction and guide.\n\nAlso see the [compiler and language\ndocumentation](http://futhark.readthedocs.io) and the [basis library\ndocumentation](https://futhark-lang.org/docs).\n\n[Installation instructions here.](http://futhark.readthedocs.io/en/latest/installation.html)\n\n[![Packaging status](https://repology.org/badge/vertical-allrepos/futhark.svg)](https://repology.org/project/futhark/versions)\n\nUsage\n=====\n\nTo compile a Futhark program to sequential C:\n\n    futhark c prog.fut -o prog\n\nOr maybe OpenCL:\n\n    futhark opencl prog.fut -o prog\n\nAnd then run it:\n\n    ./prog < prog.input\n\nTo interpret a Futhark program:\n\n    futhark run prog.fut < prog.input\n\nHacking\n=======\n\nWe try to make use of Github issues for organising our work.  Issues\ntagged with\n[good first issue](https://github.com/diku-dk/futhark/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22)\ndo not require deep knowledge of the code base.\n\nTesting\n=======\n\nRun `futhark test tests` to check how well we\'re doing.  Use `futhark\ntest -t` if you\'re in a hurry and only want to check that all the\ntests type-check.\n'
