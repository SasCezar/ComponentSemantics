b'# Dark Reader for Google Chrome and Mozilla Firefox\n![Dark Reader screenshot](https://lh3.googleusercontent.com/tTJIIIAqfJWymqPM9LAmivl11kWmF-XXLABues4OwfjOED_ntsJZdLY0VTG0XFCW0W_wYSll7Q=w640-h400-e365)\n\nThis extension **inverts brightness** of web pages and aims to **reduce eyestrain** while browsing the web.\nVisit [Chrome Web Store](https://chrome.google.com/webstore/detail/dark-reader/eimadpbcbfnmbkopoojfekhnkhdbieeh)\nand [Firefox Add-ons](https://addons.mozilla.org/en-US/firefox/addon/darkreader/)\nfor more info.\n\n## How to contribute\n\n### Sponsorship\n[Donate](https://opencollective.com/darkreader) via Open Collective.\n\n### Translation\nImprove or suggest a translation [here](https://github.com/darkreader/darkreader/tree/master/src/_locales).\nSee the list of supported [language codes](https://developer.chrome.com/webstore/i18n#localeTable).\n\n### Fixing wrong inversion\n\nIf some website is **already dark** (has all pages dark by default), you can **add it to [dark-sites.config](https://github.com/alexanderby/darkreader/blob/master/src/config/dark-sites.config) file**\n*(please, preserve alphabetical order)*.\n\nIf some **parts** of web-pages are **wrongly inverted**,\nyou can specify necessary **CSS selectors** at\n**[dynamic-theme-fixes.config](https://github.com/alexanderby/darkreader/blob/master/src/config/dynamic-theme-fixes.config) file**\n(for Dynamic Theme mode)\nor **[inversion-fixes.config](https://github.com/alexanderby/darkreader/blob/master/src/config/inversion-fixes.config) file**\n(for Filter and Filter+ modes)\n*(please, preserve alphabetical order by URL, use short selectors, preserve code style)*.\n\n~~Please note, that merged changes to these files are automatically delivered to all users **within 15 minutes**.~~\nConfig syncing was disabled, cause GitHub team doesn\'t allow using GitHub as a CDN. Storing these files and making requests to other resources will be expensive and look suspicious.\n\n### Using Dev Tools\n\nDev Tools should be used to **fix minor issues** on a web page\n*(like dark icon on dark background, removing bright background, adding white background to transparent image etc.)*.\nIf page looks partially dark and bright in **Dynamic mode**, it should be considered as a bug.\nFor **Filter mode** it is a common practice to invert already dark page parts.\n\n- Open **Chrome Dev Tools** (F12).\n- Click on **element picker** (top-left corner).\n- Pick wrongly inverted element.\n- Choose a **[selector](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_Selectors)** for that element or all similar elements (e.g. if element has `class="icon small"` selector may look like `.icon`).\n- Click **Dark Reader icon**.\n- Click **Open developer tools** (at bottom).\n- Find or add a block containing URL and selectors to invert.\n```\ndynamic-theme-fixes.config\n================================\n\nexample.com\n\nINVERT\n.icon\n\nCSS\n.wrong-element-colors {\n    background-color: ${white} !important;\n    color: ${black} !important;\n}\n\n```\n- `INVERT` rule inverts specified elements.\nFor **Dynamic mode** use `INVERT` only for dark images, that are invisible on dark backgrounds (icons, diagrams, charts, `<img>` and `<svg>` elements).\n- `CSS` rule adds custom CSS to a web page.\n`!important` keyword should be specified for each CSS property to prevent overrides by other stylesheets.\n**Dynamic mode** supports `${COLOR}` template, where `COLOR` is a color value before the inversion (`white` will become `black` in dark mode).\n- **Special notice for Filter and Filter+ config**.\nIt works by inverting the whole web page and reverting necessary parts (images, videos etc.) back, listed in `INVERT` section.\nIf inverted element contains images or other content that becomes wrongly displayed, `NO INVERT` rule can be used.\n`REMOVE BG` removes background image from element and forces black background.\n```\ninversion-fixes.config\n================================\n\nexample.com\n\nINVERT\n.icon\n.button\n#player\n\nNO INVERT\n#player *\n\nREMOVE BG\n.bg-photo\n\nCSS\n.overlay {\n    background: rgba(255, 255, 255, 0.5);\n}\n```\n- Click **Apply**.\n- Check how it looks both in **Light** and **Dark** mode.\n- If the **fix worked** open\n**[dynamic-theme-fixes.config](https://github.com/alexanderby/darkreader/blob/master/src/config/dynamic-theme-fixes.config) file**\nor **[inversion-fixes.config](https://github.com/alexanderby/darkreader/blob/master/src/config/inversion-fixes.config) file**.\n- Click **edit** (sign-in to GitHub first).\n- **Insert your fix** there. Preserve **alphabetical order** by URL.\n- Provide a **short description** of what you have done.\n- Click **Propose file change**.\n- Review your changes. Click **Create pull request**.\n- The Travis CI will run tests reviewing your changes.\n- If you see a **red cross** click **Details** and see what is wrong and edit existing Pull Request.\n- When you see a **green checkmark** then everything is fine.\n- Dark Reader developer will **review** and merge your changes making them available for all users.\n\n### Adding new features or fixing bugs\n\nIf you would like to **add new feature** to Dark Reader or **fix a bug**, **submit an issue** in GitHub (if there is no existing one), **discuss** it with active contributors, wait for **approvement**.\n\nIn order to build and debug the extension **install the [Node.js](https://nodejs.org/)** LTS.\nInstall development dependencies by running `npm install` in project root folder.\nThen execute `npm run debug`.\n\n#### Chrome\nOpen the `chrome://extensions` page.\nDisable the official Dark Reader version.\nEnable the **Developer mode**.\nClick **Load unpacked extension** button, navigate to project\'s `debug/` folder.\n\n#### Firefox\nOpen `about:addons` page.\nDisable the official Dark Reader version.\nOpen `about:debugging#addons` page.\nClick **Load Temporary Add-on** button, open `debug-firefox/manifest.json` file.\n\nAfter making any code changes the project will be automatically recompiled.\nIf the extension **didn\'t reload** automatically it can be reloaded manually on the extensions page.\n\nFor editing the code you can use any text editor or web IDE (like [Visual Studio Code](https://code.visualstudio.com), [Atom](https://atom.io/), [WebStorm](https://www.jetbrains.com/webstorm/)).\n**Preserve code style** (whitespaces etc).\n\nRun tests by executing `npm test`.\n\nSubmit a **pull request**, wait for **review**.\n\n## Building for use\nYou can install the extension from a file.\nInstall [Node.js LTS](https://nodejs.org/en/). Download the source code (or checkout from git).\nOpen terminal in root folder and run:\n- `npm install`\n- `npm run release`\n\nThis will generate `build.zip` for use in Chromium browsers and `build-firefox.xpi` for use in Firefox.\n\n## Using for a website\n\nYou can use Dark Reader to enable dark mode on your website.\nInstall the package from NPM (`npm install darkreader`)\nor download from CDN like `https://unpkg.com/darkreader`.\nThen use the following API\n```javascript\nDarkReader.enable({\n    brightness: 100,\n    contrast: 90,\n    sepia: 10\n});\n\nDarkReader.disable();\n```\n... or if you are using ES modules\n```javascript\nimport {\n    enable as enableDarkMode,\n    disable as disableDarkMode,\n} from \'darkreader\';\n\nenableDarkMode({\n    brightness: 100,\n    contrast: 90,\n    sepia: 10,\n});\n\ndisableDarkMode();\n```\n\n## Contributors\n\nThis project exists thanks to all the people who
