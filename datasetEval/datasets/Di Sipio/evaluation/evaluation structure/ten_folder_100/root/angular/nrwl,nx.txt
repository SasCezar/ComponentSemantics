Nx sets up these tools and allows you to use them seamlessly. Nx fully integrates with the other modern tools you already use and love.\n\n### Build Full-Stack Applications\n\nWith Nx, you can build full-stack applications using modern frameworks. You can share code between the frontend and the backend. And you can use the same `build/test/serve` commands throughout the whole dev experience.\n\n### Develop like Google, Facebook, and Microsoft\n\nWith Nx, you can develop multiple full-stack applications holistically and share code between them all in the same workspace. Nx provides advanced tools which help you scale your enterprise development. Nx also helps enforce your organization\xe2\x80\x99s standards and community best practices.\n\n# Getting Started\n\n## Creating an Nx Workspace\n\n**Using `npx`**\n\n```bash\nnpx create-nx-workspace myworkspace\n```\n\n**Using `npm init`**\n\n```bash\nnpm init nx-workspace myworkspace\n```\n\n**Using `yarn create`**\n\n```bash\nyarn create nx-workspace myworkspace\n```\n\nIf it\'s your first Nx project, the command will recommend you to install `@nrwl/cli` globally, so you can invoke `nx` directly without going through yarn or npm.\n\n## Creating First Application\n\nBy default, an Nx workspace starts blank. There are no applications to build, serve, and test. To create one, you need to add capabilities to the workspace.\n\n**To add a web components app, run:**\n\n```bash\nyarn add --dev @nrwl/web\nnx g @nrwl/web:app myapp # or just "nx g myapp"\n```\n\n```bash\nnpm install --save-dev @nrwl/web\nnx g @nrwl/web:app myapp # or just "nx g myapp"\n```\n\n**To add an Angular app, run:**\n\n```bash\nyarn add --dev @nrwl/angular\nnx g @nrwl/angular:app myapp # or just "nx g myapp"\n```\n\n```bash\nnpm install --save-dev @nrwl/angular\nnx g @nrwl/angular:app myapp # or just "nx g myapp"\n```\n\n**To add a React app, run:**\n\n```bash\nyarn add --dev @nrwl/react\nnx g @nrwl/react:app myapp # or just "nx g myapp"\n```\n\n```bash\nnpm install --save-dev @nrwl/react\nnx g @nrwl/react:app myapp # or just "nx g myapp"\n```\n\nIf `nx g` fails, use: `yarn nx g @nrwl/web:app myapp` or `npm run nx -- g @nrwl/web:app myapp`.\n\nRegardless of what framework you chose, the resulting file tree will look like this:\n\n```treeview\n<workspace name>/\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 apps/\n\xe2\x94\x82\xc2\xa0\xc2\xa0 \xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 myapp/\n\xe2\x94\x82\xc2\xa0\xc2\xa0 \xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 myapp-e2e/\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 libs/\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 tools/\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 nx.json\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 package.json\n\xe2\x94\x9c\xe2\x94\x80\xe2\x94\x80 tsconfig.json\n\xe2\x94\x94\xe2\x94\x80\xe2\x94\x80 tslint.json\n```\n\n## Serving Application\n\n- Run `nx serve myapp` to serve the newly generated application!\n- Run `nx test myapp` to test it.\n- Run `nx e2e myapp-e2e` to run e2e tests for it.\n\nAngular users can also run `ng g/serve/test/e2e`.\n\nYou are good to go!\n\n### Documentation\n\n- [Nx Documentation and Guides](https://nx.dev)\n- [Nx blog posts](https://blog.nrwl.io/nx/home)\n\n### Quick Start
