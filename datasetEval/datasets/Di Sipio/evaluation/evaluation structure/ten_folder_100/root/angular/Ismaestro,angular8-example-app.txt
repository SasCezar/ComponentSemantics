You can navigate through every language and reload (and share) every page in the application without losing context. This is very useful for SEO purposes and you almost have a ready for production app.\nIf you want to translate the messages you can use this awesome tool, [Tiny Translator](https://martinroob.github.io/tiny-translator/en/#/translate) or follow [this tutorial](https://github.com/martinroob/ngx-i18nsupport/wiki/Tutorial-for-using-xliffmerge-with-angular-cli).\n\nI\'ve created a medium post where you can find a tutorial to apply this concepts to your own project. Also Hosting and Functions to deploy the app with Universal.\n\n## Travis CI\n\nWe use Travis CI to run this tasks in order:\n* Linter\n* Unit tests\n* End to end tests\n* Build for production of browser and server\n* Validate that server generated using curl\n* Sonar\n* Deploy to Firebase\n\n## Quick start\n\n**WARNING**\n\n> Verify that you are running node 10.16.0 by running node -v in a terminal/console window. Older versions produce errors, but newer versions are fine.\n\n ```bash\n npm i\n npm start\n ```\n\nTasks                       | Description\n----------------------------|---------------------------------------------------------------------------------------\nnpm start                   | Start the app in development mode with the english language only\nnpm start:es                | Start the app in development mode with the spanish language only\nstart:ssr                   | Start the server like SSR\nextract-i18n                | Extract all messages from templates and ts files and update the language files with new translations\nnpm run lint                | Run the linter (tslint)\nnpm run test                | Run all unit tests with karma and jasmine\nnpm run test:app:watch      | Run app unit tests and wait for changes\nnpm run test:library:watch  | Run library unit tests and wait for changes\nnpm run e2e                 | Run end to end tests with protractor\nnpm run build:prod:en       | Build the app for production with english translations\nnpm run build:prod:es       | Build the app for production with spanish translations\nnpm run build:server:prod   | Build the server version for production\nnpm run compile:server      | Compiles the server with webpack\nnpm run build:ssr           | Complete task with all the build subtasks for SSR\nnpm run build:library       | Build the library\nnpm run serve:ssr           | Start the node server for angular universal\nnpm run validate:server     | Script that validate if server generated previously starts without error\nnpm run bundle-report       | Build and run webpack-bundle-analyzer over stats json\nnpm run release:minor       | Create a new minor release using standard-version\nnpm run release:major       | Create a new major release using standard-version\nnpm run ci                  | Execute linter, tests and production builds\nnpm run deploy              | Build the app and deploy it to firebase hosting\n\n## Bugs and feature requests\n\nHave a bug or a feature request?
