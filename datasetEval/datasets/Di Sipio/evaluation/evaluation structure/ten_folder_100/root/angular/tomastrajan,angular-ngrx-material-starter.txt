https://github.com/tomastrajan/angular-ngrx-material-starter.git new-project\ncd new-project\nnpm install\nnpm start\n```\n\n## Useful Commands\n\n- `npm start` - starts a dev server and opens browser with running app\n- `npm run start:prod` - runs full prod build and serves prod bundle\n- `npm run test` - runs lint and tests\n- `npm run watch` - runs tests in watch mode\n- `npm run prettier` - runs prettier to format whole code base (`.ts` and `.scss`)\n- `npm run analyze` - runs full prod build and `webpack-bundle-analyzer` to visualize how much code is shipped (dependencies & application)\n\n![analzye](https://raw.githubusercontent.com/tomastrajan/angular-ngrx-material-starter/master/meta-assets/analyze.png)\n\n## Make It Your Own\n\nWhen using this starter project to build your own app you might consider some of the following steps:\n\n- use `search and replace` functionality of your favourite IDE to replace `anms` with `<your-app-prefix>`\n- rename project in `package.json` `name` property and set appropriate version (eg `0.0.0` or `1.0.0`)\n- remove / rename context path config `-- --deploy-url /angular-ngrx-material-starter/ --base-href /angular-ngrx-material-starter` in `package.json`, this is used to configure url (context path) on which the application will be available (eg. `https://www.something.com/<context-path>/`)\n- rename app in `/environments/` files (will be shown in browser tab)\n- delete pre-existing `CHANGELOG.md` (you will generate your own with future releases of your features)\n- delete `CODE_OF_CONDUCT.md`, `CONTRIBUTING.md` and `BUILT_WITH.md` files as they are relevant only if project is open sourced on Github\n- edit the title and Open Graph metadata properties in `index.html`\n- remove or adjust links in the [footer](https://github.com/tomastrajan/angular-ngrx-material-starter/blob/master/src/app/app.component.html#L79)\n- replace logo in `/assets` folder ( currently 128 x 128 pixel `png` file )\n- adjust colors in `/themes/default-theme.scss`\n- create a pull request in the [original repository](https://github.com/tomastrajan/angular-ngrx-material-starter/) to update `BUILT_WITH.md` [file](https://github.com/tomastrajan/angular-ngrx-material-starter/blob/master/BUILT_WITH.md) with a link and short description of your project\n\n#### Continuous Integration\n\nStarter project is using [Travis CI](https://travis-ci.org/) for running linters and tests on every commit.\nBased on your preferences and needs you can either:\n\n- not use / use other CI server and delete both `.travis.yml` and `.travis-deploy.sh`\n- create Travis CI account and link it to your projects Github repo and [configure build](https://medium.com/@tomastrajan/continuous-deployment-of-client-side-apps-with-github-pages-travis-ci-10e9d641a889)\n  with `GH_REF` and `GH_TOKEN` environment variables for automatic deployment of releases to Github Pages\n\n## Goals\n\nThe main goal of this repository is to provide an up to date example of Angular application following all recent best practices in various areas like:\n\n- `@ngrx/store` - including reducers, actions, selectors\n- `@ngrx/effects` - for implementation of side effects like `http` requests, logging, notifications,...\n- `@ngrx/entity` - for CRUD operations\n- `@ngrx/router-store` - to connect the Angular Router to @ngrx/store\n- `@ngrx/store-devtools` - to enable a powerful time-travelling debugger.\n- `@angular/material` - material design component library, theming, ...\n- routing\n- testing of all the above mentioned concepts\n- Angular CLI configuration (prod build, budgets, ...)\n\nThis repository will also strive to always stay in sync with releases of Angular and the related libraries.\nThe nature of the repository is also a great match for first time open source contributors who can add\nsimple features and enhance test coverage, all contributors are more than welcome!\n\n## Learning Materials\n\nArticles with content that explains various approaches used to build this starter project.\n\n- [Blog post about Best (4 themes included)\n- lazy-loading of feature modules\n- lazy reducers\n- localStorage ui state persistence\n- `@ngrx/effects` for API requests\n- fully responsive design\n- angular-material and custom components in `SharedModule`\n\n## Stack\n\n- Angular\n- ngrx (or try [ngx-model](https://github.com/tomastrajan/ngx-model) if you prefer less boilerplate)\n- Angular Material\n- Bootstrap 4 (only reset, utils and grids)\n\n## Troubleshooting\n\n- **Blocking at emitting LicenseWebpackPlugin when npm start** - try using [cnpm](https://github.com/cnpm/cnpm) instead of npm\n\n##
