There can be a lot of boilerplate code in state management, thus a main goal of NGXS is to reduce boilerplate allowing you to do more things with less. It is also not necessary to be super familiar with RxJs.\n\n### The Goal of NGXS Labs\n\nThe idea with this github organisation is to provide a place for the community to create libraries that augment the main framework with functionality that does not need to be integrated directly into the framework and therefore can evolve through their initial iterations of experimentation without affecting the main @ngxs/store library.\n\n### Getting Started - Local Development\n\n#### Installation\n\nTo get started locally, follow these instructions:\n\n1. If you haven\'t done it already, make a fork of this repo.\n2. Clone to your local computer using git.\n3. Make sure that you have installed NodeJS.\n4. Make sure that you have yarn installed.\n5. Run `yarn install`.\n6. Run `yarn build:packages`.\n\n#### Creating new packages or add feature/fix\n\n##### if you make changes @ngxs/store\n\n1. Run development mode `yarn build:packages --package store --watch`\n2. Run serve integration examples `yarn start`\n3. **...development...**\n4. Run tests `yarn test:ci`\n5. Create pull request\n\n##### if you add a new package @ngxs/my-super-plugin\n\n1. Create a new project folder `packages/my-super-plugin`\n2. Create template library with ngPackagr\n3. Add your project to package.json\n4. Run development mode `yarn build:packages --package my-super-plugin --watch`\n5. **...development...**\n6. Run build `yarn build:packages --package my-super-plugin`\n7. Run tests `yarn test:ci`\n8. Create pull request\n\n#### NGXS Labs\n\nIf you have ideas for creating unique libraries, you can join us. Email us at `ngxs.lead@gmail.com`.
