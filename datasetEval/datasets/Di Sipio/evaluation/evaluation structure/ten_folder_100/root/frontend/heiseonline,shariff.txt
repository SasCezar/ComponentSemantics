b'# Shariff \xe2\x80\x93 Give Social Media Buttons Some Privacy [![Build Status](https://travis-ci.org/heiseonline/shariff.svg?branch=master)](https://travis-ci.org/heiseonline/shariff)\n\n\nShariff enables website users to share their favorite content without compromising their privacy. [Demo](http://heiseonline.github.io/shariff/)\n\n![Shariff Logo \xc2\xa9 2015 Heise Medien](http://www.heise.de/icons/ho/shariff-logo.png)\n\nSome social service providers like e.g. Facebook supply official sharing code snippets which quietly siphon personal data from all page visitors. Shariff enables visitors to see how popular your page is on Facebook and share your content with others without needless data leaks.\n\nShariff `(/\xcb\x88\xca\x83\xc9\x9b\xc9\xb9\xc9\xaaf/)` is an open-source, low-maintenance, high-privacy solution maintained by German computer magazine c\'t and heise online.\n\nShariff consists of two parts: a simple JavaScript client library and an optional server-side component. The latter fetches the number of likes or shares. Share buttons and share counts work without a connection between your visitors\' browsers and *social networks* (unless they decide to share, of course).\n\n## Getting Started\n\n1. Download the [latest release](https://github.com/heiseonline/shariff/releases/latest)\n2. Upload all files included in the release\n3. Include CSS in `<head>`:\n    * `shariff.complete.css` uses the dependencies included in the release files\n    * if [Font Awesome](https://github.com/FortAwesome/Font-Awesome) is already included in your site, use `build/shariff.min.css`\n4. Include JavaScript right before `</body>`:\n    * `shariff.complete.js` uses the dependencies included in the release files\n    * if [jQuery](https://github.com/jquery/jquery) is already included in your site, use `build/shariff.min.js`\n5. Insert one or more `<div class="shariff">` elements.\n6. Customize the look using data-* attributes.\n\nTo enable the counters in the buttons, see section [Backends](#backends).\n\nUsage example:\n\n```html\n<!DOCTYPE html>\n<html>\n<head>\n    <link href="/path/to/shariff.min.css" rel="stylesheet">\n</head>\n<body>\n    <h1>My article</h1>\n    <p>Lorem ipsum dolor sit amet, consectetur adipisicing elit.</p>\n\n    <h2>Minimum buttons:</h2>\n    <div class="shariff"></div>\n\n    <h2>More advanced buttons:</h2>\n    <div class="shariff" data-backend-url="/path/to/backend" data-url="http://www.example.com/my-article.html" data-theme="grey" data-orientation="vertical"></div>\n\n    <!-- immediately before </body> -->\n    <script src="/path/to/shariff.min.js"></script>\n</body>\n</html>\n```\n\n## Getting Started using `npm`\n\nYou can also use Shariff\'s node package by installing it via `npm`:\n\n```sh\n$ cd my-project\n$ npm install shariff --save\n```\n\nEdit your JS main script, include Shariff and initialize it in one or more containers:\n\n```js\n// my-app.js\nvar Shariff = require(\'shariff\');\nvar $ = require(\'jquery\');\nvar buttonsContainer = $(\'.some-selector\');\nnew Shariff(buttonsContainer, {\n    orientation: \'vertical\'\n});\n```\n\n## Running demo site\n\nAfter downloading Shariff, install its dependencies by running `npm install`.\n\n```sh\n$ git clone https://github.com/heiseonline/shariff.git\n$ cd shariff\n$ npm install\n$ npm run dev\n```\n\n## Options (data attributes)\n\n| Attribute        | Description | Default |\n|------------------|-------------|---------|\n| `data-backend-url` | The path to your Shariff backend, see below. Setting the value to `null` disables the backend feature. No counts will occur.  | `null` |\n| `data-button-style` | How to display the buttons. Values: `standard`, `icon`, `icon-count`. With `icon` only the icon is shown, with `icon-count` icon and counter and with `standard` icon, text and counter are shown, depending on the display size.  | `standard` |\n| `data-flattr-category` | Category to be used for Flattr. | `null` |\n| `data-flattr-user` | User that receives Flattr donation. | `null` |\n| `data-info-url` | URL of the info page. | `http://ct.de/-2467514` |\n| `data-info-display` | How to display the info page. Values: `blank`, `popup`, `self`. | `blank` |\n| `data-lang`      | The localisation to use. Available: `bg`, `cs`, `da`, `de`, `en`, `es`, `fi`, `fr`, `hr`, `hu`, `it`, `ja`, `ko`, `nl`, `no`, `pl`, `pt`, `ro`, `ru`, `sk`, `sl`, `sr`, `sv`, `tr`, `zh` | `de` |\n| `data-mail-body` | If a `mailto:` link is used in `data-mail-url`, then this value is used as the mail body. The body text should contain the placeholder `{url}` which will be replaced with the share URL. | see `data-url`  |\n| `data-mail-subject` | If a `mailto:` link is used in `data-mail-url`, then this value is used as the mail subject. | see `data-title` |\n| `data-mail-url`  | The url target used for the `mail` service button | `?view=mail` |\n| `data-media-url` | Media url to be shared (pinterest) | `null` |\n| `data-orientation` | `vertical` will stack the buttons vertically. | `horizontal`  |\n| `data-referrer-track` | A string that will be appended to the share url. Can be disabled using `null`. | `null` |\n| `data-services`   | An entity-encoded JSON string containing an array of service names to be enabled. Example: `data-services="[&quot;facebook&quot;,&quot;twitter&quot;]"` Available service names: `twitter`, `facebook`, `linkedin`, `pinterest`, `xing`, `whatsapp`, `mail`, `info`, `addthis`, `tumblr`, `flattr`, `diaspora`, `reddit`, `stumbleupon`, `threema`, `weibo`, `tencent-weibo`, `qzone`, `print`, `telegram`, `vk`, `flipboard`, `pocket`, `buffer` | `twitter`, `facebook`, `info` |\n| `data-theme`       | We include 3 color schemes, `standard`, `grey` and `white`. | `standard` |\n| `data-title`       | Title to be used as share text in Twitter/Whatsapp | page\'s `DC.title`/`DC.creator` or `<title>` |\n| `data-twitter-via` | Screen name of the user to attribute the Tweet to | `null` |\n| `data-url`         | The canonical URL of the page to check. | page\'s canonical URL or `og:url` or current URL |\n\n## Constructor arguments\n\nAll data attributes above are also available as constructor arguments in JavaScript. However, the\nleading `data-` is omitted and the name is in camelCase instead of kebab-case:\n\n```js\nvar buttonsContainer = $(\'.some-selector\');\nnew Shariff(buttonsContainer, {\n    backendUrl: \'/my/backend/url\',\n    orientation: \'vertical\',\n    mailUrl: \'mailto:me@example.com\',\n});\n```\n\n## Supported browsers\n\nShariff supports the following Browsers:\n\n- Firefox\n- Google Chrome\n- Internet Explorer/Edge\n- Safari\n\nThe current and previous major releases of Firefox, Google Chrome, Internet Explorer/Edge and Safari are supported on a rolling basis.\n\n## Supported services\n\nShariff supports the following social sharing services:\n\n- AddThis\n- buffer\n- diaspora*\n- facebook\n- Flattr\n- Flipboard\n- LinkedIn\n- Mail\n- Pinterest\n- Pocket\n- Print\n- Qzone\n- reddit\n- StumbleUpon\n- Telegram\n- Tencent Weibo\n- Threema\n- Tumblr\n- Twitter\n- VK\n- Weibo\n- WhatsApp\n- XING\n\nIn addition, the service `Info` provides a button to show an info page about the social sharing buttons.\nThe URL of this page can be set with an option. Default value: `http://ct.de/-2467514`, i.e. the c\'t article introducing Shariff.\n\n## Backends\n\nIn order to display share counts with Shariff, you need the following backend:\n\n* [shariff-backend-php](http://github.com/heiseonline/shariff-backend-php)\n\nThird-party backends:\n\n* [shariff-backend-java](https://github.com/shred/shariff-backend-java)\n\nOnce you have one of these backends up and running, insert its URL into the `data-backend-url` attribute. For example, if the backend runs under `http://example.com/my-shariff-backend/`, the `data-backend-url` should be `/my-shariff-backend/`. The script will handle the rest.\n\n## Third-party integrations\n\nThis is a list of integrations for third-party systems:\n\n* [Contao integration](https://github.com/hofff/contao-shariff)\n* [Drupal module](https://www.drupal.org/project/shariff)\n* [Joomla! 3 Shariff Plugin](https://github.com/joomla-agency/plg_jooag_shariff)\n* [Kirby-CMS Shariff-Plugin](https://github.com/SpicyWeb-de/kirby-plugin-shariff)\n* [MediaWiki Extension](https://github.com/vonloxley/Shariff-Mediawiki/)\n* [Open Monograph Press Plugin](https://github.com/langsci/shariff)\n* [Pagekit Extension](https://pagekit.com/marketplace/package/spqr/shariff)\n* [Serendipity](https://github.com/s9y/additional_plugins/tree/master/serendipity_event_social)\n* [SilverStripe Module](https://github.com/andrelohmann/silverstripe-shariff)\n* [Symfony ShariffBundle](https://github.com/core23/ShariffBundle)\n* [TYPO3 Plugin rx_shariff](http://typo3.org/extensions/repository/view/rx_shariff)\n* [Wordpress Plugin shariff-sharing](https://wordpress.org/plugins/shariff-sharing/)\n* [WordPress Plugin Shariff Wrapper](https://wordpress.org/plugins/shariff/)\n* [Xenforo [ITM] ctSSB for Xenforo 1.5](https://github.com/McAtze/-ITM-ctShariffSocialButtons)\n* [Xenforo [WMTech] Social Share Privacy Plugin](https://wmtech.net/products/wmtech-social-share-privacy.41/)\n* [Magento 2 Shariff Social Share](https://www.jajuma.de/en/jajuma-develop/extensions/shariff-social-share-buttons-extension-for-magento-2)\n'
