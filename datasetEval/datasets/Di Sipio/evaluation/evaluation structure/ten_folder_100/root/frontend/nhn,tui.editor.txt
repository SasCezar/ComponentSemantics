It also serves as important index to determine the future course of projects. location.hostname (e.g. > \xe2\x80\x9cui.toast.com") is to be collected and the sole purpose is nothing but to measure statistics on the usage. To disable GA, use the following `usageStatistics` options when creating editor.\n\n``` javascript\nconst options = {\n  // ...\n  usageStatistics: false\n}\n\nconst instance = new Editor(options);\n```\n\nOr, include [`tui-code-snippet`](https://github.com/nhn/tui.code-snippet)(**v1.5.0** or **later**) and then immediately write the options as follows:\n\n``` javascript\ntui.usageStatistics = false;\n```\n\n\n## \xf0\x9f\x93\x99 Documents\n\n* [Getting Started](https://github.com/nhn/tui.editor/blob/master/docs/getting-started.md)\n* [Getting Started with Bower](https://github.com/nhn/tui.editor/blob/master/docs/getting-started-with-bower.md)\n* [Writing Your Own Extension](https://github.com/nhn/tui.editor/blob/master/docs/writing-your-own-extension.md)\n* [APIs](https://nhn.github.io/tui.editor/api/latest/)\n\n\n## <img src="https://user-images.githubusercontent.com/1215767/34336735-e7c9c4b0-e99c-11e7-853b-2449b51f0bab.png" height="18px" /> Standard and Extensible\n![standard and extensible image](https://user-images.githubusercontent.com/1215767/34353629-95b58da0-ea6c-11e7-859b-df5e990dd157.png)\n\n### CommonMark + GFM Specifications\n\nToday *CommonMark* is the de-facto *Markdown* standard. *GFM (GitHub Flavored Markdown)* is another popular specification based on *CommonMark* - maintained by *GitHub*, which is the *Markdown* mostly used. TOAST UI Editor follows both [*CommonMark*](http://commonmark.org/) and [*GFM*](https://github.github.com/gfm/) specifications. Write documents with ease using productive tools provided by TOAST UI Editor and you can easily open the produced document wherever the specifications are supported.\n\n### Powerful Extensions\n\n*CommonMark* and *GFM* are great, but we often need more abstraction. The TOAST UI Editor comes with powerful **Extensions** in compliance with the *Markdown* syntax. You also get the flexibility to develop your own extensions using simple APIs.\n\nHere are some of the extensions you can start with:\n\n* **Color Picker**: [ColorPicker](https://github.com/nhn/tui.color-picker) provides an easy way to color text with a GUI tool box.\n* **Chart Code Block**: A code block marked as a \'chart\' will render [charts](https://github.com/nhn/tui.chart).\n* **UML Code Block**: A code block marked as an \'uml\' will render [UML diagrams](http://plantuml.com/screenshot).\n* **Table Merge**: You can merge columns and rows in tables.\n\n To learn more about **Extensions** check the [Using Extension](https://github.com/nhn/tui.editor/blob/master/docs/using-extensions.md).\n\n\n## \xf0\x9f\x8e\xa8 Features\n\nTOAST UI Editor provides **Markdown mode** and **WYSIWYG mode**.\n\nDepending on the type of use you want like production of *Markdown* or maybe to just edit the *Markdown*. If you are using *bower* please see [Getting Started with Bower](https://github.com/nhn/tui.editor/blob/master/docs/getting-started-with-bower.md).\n\n### Create Editor\n\n#### HTML\n\nAdd the container element where TOAST UI Editor will be created.\n\n```html\n<div id="editorSection"></div>\n```\n\n#### JavaScript\n\nTOAST UI Editor can be used by creating an instance with the constructor function. To get the constructor function, you should import the module using one of the following ways depending on your environment.\n\n``` javascript\nconst Editor = require(\'tui-editor\'); /* CommonJS */\n```\n\n``` javascript\nimport Editor from \'tui-editor\'; /* ES6 */\n```\n\nThen import styles of TOAST UI Editor and dependencies.\n\n``` javascript\nimport \'tui-editor/dist/tui-editor.css\'; // editor\'s ui\nimport \'tui-editor/dist/tui-editor-contents.css\'; // editor\'s content\nimport \'codemirror/lib/codemirror.css\'; // codemirror\nimport \'highlight.js/styles/github.css\'; // code block highlight\n```\n\nFinally you can create an instance with options and call various API after creating an instance.\n\n``` javascript\nimport \'codemirror/lib/codemirror.css\';\nimport \'tui-editor/dist/tui-editor.css\';\nimport \'tui-editor/dist/tui-editor-contents.css\';\nimport \'highlight.js/styles/github.css\';\n\nimport Editor from \'tui-editor\';\n\nconst instance = new Editor({\n  el: document.querySelector(\'#editorSection\'),\n  initialEditType: \'markdown\',\n  previewStyle: \'vertical\',\n  height: \'300px\'\n});\n\ninstance.getHtml();\n```\n\nIf you use jQuery plugin, you can use it as follows.\n\n``` javascript\n$(\'#editorSection\').tuiEditor({\n  initialEditType: \'markdown\',\n  previewStyle: \'vertical\',\n  height: \'300px\'\n});\n```\n\n#### Default Options\n\n* `height`: Height in string or auto ex) `300px` | `auto`\n* `initialValue`: Initial value. Set Markdown string\n* `initialEditType`: Initial type to show `markdown` | `wysiwyg`\n* `previewType`: Preview style of Markdown mode `tab` | `vertical`\n* `usageStatistics`: Let us know the *hostname*. We want to learn from you how you are using the Editor. You are free to disable it. `true` | `false`\n\nFind out more options [here](https://nhn.github.io/tui.editor/latest/ToastUIEditor)\n\n### Create Viewer\n\nTOAST UI Editor provides the **Viewer** in case you want to show *Markdown* content without loading the Editor. The Viewer is much **lighter** than the Editor.\n\n``` javascript\nimport \'tui-editor/dist/tui-editor-contents.css\';\nimport \'highlight.js/styles/github.css\';\n\nimport Viewer from \'tui-editor/dist/tui-editor-Viewer\';\n\nconst instance = new Viewer({\n  el: document.querySelector(\'#viewerSection\'),\n  height: \'500px\',\n  initialValue: \'# content to be rendered\'\n});\n\ninstance.getHtml();\n```\n\nBe careful not to load both the Editor and the Viewer at the same time because the Editor already contains the Viewer function, you can initialize editor [`Editor.factory()`](https://nhn.github.io/tui.editor/latest/ToastUIEditor#factory) and set the `viewer` option to value `true` in order to make the editor a viewer. You can also call [`getHtml()`](https://nhn.github.io/tui.editor/latest/ToastUIEditor#getHtml) to render the HTML.\n\n\n``` javascript\nimport Editor from \'tui-editor\';\n\nconst instance = Editor.factory({\n  el: document.querySelector(\'#viewerSection\'),\n  viewer: true,\n  height: \'500px\',\n  initialValue: \'# content to be rendered\'\n});\n```\n\nTOAST UI Editor follows *CommonMark* and *GFM*. So any *Markdown* renderer including [markdown-it](https://github.com/markdown-it/markdown-it) can handle the content made using TOAST UI Editor. You can also use any of these renderer in place of TOAST UI Editor **Viewer**.\n\n\n## \xf0\x9f\x8c\x8f Browser Support\n\n| <img src="https://user-images.githubusercontent.com/1215767/34348387-a2e64588-ea4d-11e7-8267-a43365103afe.png" alt="Chrome" width="16px" height="16px" /> Chrome | <img src="https://user-images.githubusercontent.com/1215767/34348590-250b3ca2-ea4f-11e7-9efb-da953359321f.png" alt="IE" width="16px" height="16px" /> Internet Explorer | <img src="https://user-images.githubusercontent.com/1215767/34348380-93e77ae8-ea4d-11e7-8696-9a989ddbbbf5.png" alt="Edge" width="16px" height="16px" /> Edge | <img src="https://user-images.githubusercontent.com/1215767/34348394-a981f892-ea4d-11e7-9156-d128d58386b9.png" alt="Safari" width="16px" height="16px" /> Safari | <img src="https://user-images.githubusercontent.com/1215767/34348383-9e7ed492-ea4d-11e7-910c-03b39d52f496.png" alt="Firefox" width="16px" height="16px" /> Firefox |\n| :---------: | :---------: | :---------: | :---------: | :---------: |\n| Yes | 10+ | Yes | Yes | Yes |\n\n\n## \xf0\x9f\x94\xa7 Pull Request Steps\n\nAll TOAST UI products are open source.
