b'<h1 align="center">\n  <img src="https://raw.githubusercontent.com/jeisele2/OverVue/master/src/assets/overvue-icons/apple-icon-72x72.png">\n  <br/>\n  OverVue (Beta)\n</h1>\n\n<p align="center"><b>Prototyping Tool for Vue Developers</b></p>\n\n<p>OverVue is a prototyping tool that allows developers to <em>dynamically</em> create and visualize a Vue application, implementing a real-time intuitive <em>tree display</em> of component hierarchy and a live-generated <em>code preview</em>. The resulting boilerplate can be <em>exported</em> as a template for further development.</p>\n\n<p>\xe8\xbf\x99\xe4\xb8\xaa\xe7\xa8\x8b\xe5\xba\x8f\xe8\x83\xbd\xe5\xb8\xae\xe4\xbd\xa0\xe7\x94\x9f\xe6\x88\x90Vue \xe7\xbb\x84\xe4\xbb\xb6, \xe8\xae\xbe\xe7\xbd\xae routes , \xe4\xb9\x9f\xe5\x8f\xaf\xe4\xbb\xa5\xe5\xb8\xae\xe4\xbd\xa0\xe6\x98\xbe\xe5\x83\x8fComponent Parent-Child\xe7\xbb\x84\xe4\xbb\xb6\xe6\xa0\x91\xe3\x80\x82\xe4\xbd\xa0\xe5\x8f\xaa\xe8\xa6\x81\xe5\x81\x9a\xe4\xb8\x80\xe4\xba\x9b\xe5\xb0\x8f\xe9\x85\x8d\xe7\xbd\xae\xe7\x84\xb6\xe5\x90\x8e\xe5\x8f\xaf\xe4\xbb\xa5\xe4\xb8\x8b\xe8\xbd\xbdcode boilerplate. \xe8\xbf\x99\xe6\xa0\xb7\xe4\xbd\xa0\xe5\xb0\xb1\xe5\x8f\xaf\xe4\xbb\xa5\xe5\xbe\x88\xe6\x96\xb9\xe4\xbe\xbf\xe7\xae\x80\xe6\xb4\x81\xe5\x9c\xb0\xe7\x94\x9f\xe6\x88\x90Vue\xe5\x89\x8d\xe5\x8f\xb0APP\xe4\xba\x86\xef\xbc\x81</p>\n\n![](https://raw.githubusercontent.com/jeisele2/OverVue/master/src/assets/gifs/screenshot.png)\n\n#### Features\n+ Upload a frontend mockup image\n+ Visualize draggable and resizable components\n+ Create parent-child hierarchy of components\n+ Add html elements to components\n+ Create routes to be used by Vue Router\n+ Live-generated previewable code snippets for each component\n+ Live-generated tree view to aid in visualizing parent-child hierarchy\n+ Save projects and open previous projects\n+ Export full boilerplate code for a working frontend\n\n\n#### How to use\n+ Opening the application will create by default a root App component and a root route called "HomeView"\n+ Upload a mockup from your filesystem if you\'d like. Remove the mockup and choose a new one if needed.\n![](https://raw.githubusercontent.com/jeisele2/OverVue/master/src/assets/gifs/upload-image-drawers.gif)\n\n+ To add a new component, type its name in the component name box and select any HTML elements that should be rendered by that component.\n+ HTML elements can also be added after creation by selecting the component in the display, then selecting HTML elements.\n+ Select a parent component for the new component if needed.\n+ After adding, you can move and resize the component in the display.\n![alt text](https://raw.githubusercontent.com/jeisele2/OverVue/master/src/assets/gifs/component_creation.gif)\n\n+ You can also add children to components by right-clicking the component to add children to, and you can see the tree re-render as you create new components or change the hierarchy.\n![](https://raw.githubusercontent.com/jeisele2/OverVue/master/src/assets/gifs/HTML-elements-tree-rerender.gif)\n\n+ The dashboard shows info about each component (code snippets and HTML elements). Click a component in the display to see its properties.                                               \n![](https://raw.githubusercontent.com/jeisele2/OverVue/master/src/assets/gifs/snippets-active-component.gif)\n\n+ You can add new routes and view all components and routes in the sidebar.\n![](https://raw.githubusercontent.com/jeisele2/OverVue/master/src/assets/gifs/sidebar-components-routes.gif)\n\n+ When finished creating, you can export to a file location of your choice. Below is the exported file structure:\n\n```\npublic/\n  index.html\nsrc/\n  assets/\n  components/\n    UserCreatedComponent1.vue\n    UserCreatedComponent2.vue\n    ...\n  views/\n    HomeView.vue\n    UserCreatedRouteComponent1.vue\n    UserCreatedRouteComponent2.vue\n    ...\n  App.vue\n  main.js\n  router.js\nbabel.config.js\npackage.json\n```\n\n#### Running a local version\nThis app was developed using the Quasar framework, so first you will need to install the Quasar cli\n```\nnpm i -g @quasar/cli\n```\nInstall dependencies\n```\nnpm i\n```\nTo run electron app in dev mode\n```\nquasar dev -m electron\n```\nTo build a new .dmg\n```\nquasar build -m electron\n```\n\n#### Contributing\nWe\'d love for you to test this application out and submit any issues you encounter.
