Karabiner-Elements\n\nKarabiner-Elements is a powerful utility for keyboard customization on macOS Sierra or later.\n\n## Download\n\nYou can download Karabiner-Elements from [official site](https://pqrs.org/osx/karabiner/).\n\n### Old releases\n\nYou can download previous versions of Karabiner-Elements from [here](https://pqrs.org/osx/karabiner/history.html).\n\n## Supported systems\n\n-   macOS Sierra (10.12)\n-   macOS High Sierra (10.13)\n-   macOS Mojave (10.14)\n-   macOS Catalina (10.15)\n\n## Usage\n\n<https://pqrs.org/osx/karabiner/document.html>\n\n## Donations\n\nIf you would like to contribute financially to the development of Karabiner Elements, donations can be made via <https://pqrs.org/osx/karabiner/pricing.html>\n\n---\n\n## For developers\n\n### How to build\n\nSystem requirements to build Karabiner-Elements:\n\n-   macOS 10.14+\n-   Xcode 10+\n-   Command Line Tools for Xcode\n-   CMake (`brew install cmake`)\n\n#### Step 1: Getting source code\n\nClone the source from github.\n\n```shell\ngit clone --depth 1 https://github.com/pqrs-org/Karabiner-Elements.git\n```\n\n#### Step 2: Building a package\n\n```shell\ncd Karabiner-Elements\nmake package\n```\n\nThe `make` script will create a redistributable **Karabiner-Elements-VERSION.dmg** in the current directory.\n\n#### Note: About pre-built binaries in the source tree\n\nKarabiner-Elements uses some pre-built binaries in the source tree.\n\n-   `src/vendor/Karabiner-VirtualHIDDevice/dist/*.kext`\n-   `src/vendor/Sparkle/Sparkle.framework`\n\nAbove `make package` command does not rebuild these binaries.<br/>\n(These binaries will be copied in the distributed package.)\n\nIf you want to rebuild these binaries, you have to build them manually.<br/>\nPlease follow the instruction of these projects.\n\n##### About rebuilding kext in Karabiner-VirtualHIDDevice\n\nIf you want to build kext in Karabiner-VirtualHIDDevice, macOS requires a valid certificate which be able to sign the built kext.<br/>\nUnless such certificate, macOS refuses to load the built kext.<br/>\nPlease read a documentation about [System Integrity Protection Guide](https://developer.apple.com/library/archive/documentation/Security/Conceptual/System_Integrity_Protection_Guide/KernelExtensions/KernelExtensions.html) for more details.\n\n(We are including the pre-built kext binary to avoid the restriction that macOS requires a uncommon certificate.)\n'
