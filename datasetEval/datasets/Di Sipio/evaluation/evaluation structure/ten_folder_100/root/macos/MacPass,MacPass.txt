b"[![Build Status](https://travis-ci.org/MacPass/MacPass.svg?branch=continuous)](https://travis-ci.org/MacPass/MacPass)\n\n# MacPass\n\nThere are a lot of iOS KeePass tools around but a distinct lack of a good native macOS version.\nKeePass can be used via Mono on macOS but lacks vital functionality and feels sluggish and simply out of place.\n\nMacPass is an attempt to create a native macOS port of KeePass on a solid open source foundation with a vibrant community pushing it further to become the best KeePass client for macOS.\n\n## Download\n\nAll pre-built releases can be found at [Github](https://github.com/mstarke/MacPass/releases).\n\nAn unsigned build of the current contiuous tag can be found here: [Continuous Build](https://github.com/mstarke/MacPass/releases/tag/continuous)\n\nDue to the nature of the build it might be unstable, however this version contains all the latest changes and bug fixes!\n\n## How to Contribute\n\nIf you want to contribute by fixing a bug, adding a feature or improving localization you're awesome!\n\n## How to Build\n\n* Fetch the source of MacPass\n```bash\ngit clone https://github.com/mstarke/MacPass --recursive\n```\n* Install [Carthage](https://github.com/Carthage/Carthage#installing-carthage)\n* Install all Dependencies\n```bash\ncd MacPass\ncarthage bootstrap --platform macOS\n```\nAfter that you can build and run in Xcode. The following command will build and make the application available through Spotlight. If you run into signing issues take a look at [Issue #92](https://github.com/mstarke/MacPass/issues/92). Since Sparkle is disabled only on the CI build and in Debug mode, you have to explicitly disable it in Release. Otherwise warnings on unsecure updates will appear.\n\n    xcodebuild -scheme MacPass -target MacPass -configuration Release CODE_SIGNING_REQUIRED=NO NO_SPARKLE=NO_SPARKLE\n\n## Help\n\nSome questions might be answered in the [FAQ](https://github.com/mstarke/MacPass/wiki/FAQ)\n\nAnother place to look is the IRC channel [#macpass](irc://irc.freenode.org/macpass) on [irc.freenode.org](irc://irc.freenode.org)\n\nOr follow the Twitter account [@MacPassApp](https://twitter.com/MacPassApp)\n\n## System Requirement\n\nMacPass 0.7 requires macOS 10.10 Yosemite or later.\nEarlier versions of MacPass require macOS 10.8 Mountain Lion or later.\n\n## Status\n\nThe Status can be found on the dedicated [Wiki page](https://github.com/mstarke/MacPass/wiki/Status).\n\n## What does it look like?\n\n![image](https://raw.github.com/mstarke/MacPass/master/Assets/Screenshots/MacPass.png)\n\nMore Screenshots in the [Wiki](https://github.com/mstarke/MacPass/wiki/Screenshots)\n\n## Alternatives\n \n[KeePassX](https://www.keepassx.org) and its fork [KeePassXC](https://github.com/keepassxreboot/keepassxc). Qt based cross plattform port.\n\n[KyPass Companion](http://www.kyuran.be/logiciels/kypass4mac/). Native macOS client.\n\n[KeeWeb](https://keeweb.info). Electron based cross plattform port. Since it's browser based you can pretty much run it anywhere.\n\n## License\n\nMacPass, a KeePass compatible Password Manager for OS X\nCopyright (c) 2012-2017  Michael Starke (HicknHack Software GmbH) and all [MacPass contributors](https://github.com/mstarke/MacPass/graphs/contributors)\n\nThis program is free software: you can redistribute it and/or modify\nit under the terms of the GNU General Public License as published by\nthe Free Software Foundation, either version 3 of the License, or\n(at your option) any later version.\n\nThis program is distributed in the hope that it will be useful,\nbut WITHOUT ANY WARRANTY; without even the implied warranty of\n\nMERCHANTABILITY or FITNESS FOR A PARTICULAR PURPOSE.
