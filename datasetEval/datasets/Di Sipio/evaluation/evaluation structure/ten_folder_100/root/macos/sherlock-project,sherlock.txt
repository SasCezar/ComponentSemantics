Python 3.6 or higher is required.\n\n```bash\n# clone the repo\n$ git clone https://github.com/sherlock-project/sherlock.git\n\n# change the working directory to sherlock\n$ cd sherlock\n\n# install python3 and python3-pip if they are not installed\n\n# install the requirements\n$ python3 -m pip install -r requirements.txt\n```\n[![Open in Cloud Usage\n\n```bash\n$ python3 sherlock.py --help\nusage: sherlock.py [-h] [--version] [--verbose] [--rank]\n                   [--folderoutput FOLDEROUTPUT] [--output OUTPUT] [--tor]\n                   [--unique-tor] [--csv] [--site SITE_NAME]\n                   [--proxy PROXY_URL] [--json JSON_FILE]\n                   [--proxy_list PROXY_LIST] [--check_proxies CHECK_PROXY]\n                   [--print-found]\n                   USERNAMES [USERNAMES ...]\n\nSherlock: Find Usernames Across Social Networks (Version 0.9.5)\n\npositional arguments:\n  USERNAMES             One or more usernames to check with social networks.\n\noptional arguments:\n  -h, --help            show this help message and exit\n  --version             Display version information and dependencies.\n  --verbose, -v, -d, --debug\n                        Display extra debugging information and metrics.\n  --rank, -r            Present websites ordered by their Alexa.com global\n                        rank in popularity.\n  --folderoutput FOLDEROUTPUT, -fo FOLDEROUTPUT\n                        If using multiple usernames, the output of the results\n                        will be saved to this folder.\n  --output OUTPUT, -o OUTPUT\n                        If using single username, the output of the result\n                        will be saved to this file.\n  --tor, -t             Make requests over Tor; increases runtime; requires\n                        Tor to be installed and in system path.\n  --unique-tor, -u      Make requests over Tor with new Tor circuit after each\n                        request; increases runtime; requires Tor to be\n                        installed and in system path.\n  --csv                 Create Comma-Separated Values (CSV) File.\n  --site SITE_NAME      Limit analysis to just the listed sites. Add multiple\n                        options to specify more than one site.\n  --proxy PROXY_URL, -p PROXY_URL\n                        Make requests over a proxy. e.g.\n                        socks5://127.0.0.1:1080\n  --json JSON_FILE, -j JSON_FILE\n                        Load data from a JSON file or an online, valid, JSON\n                        file.\n  --proxy_list PROXY_LIST, -pl PROXY_LIST\n                        Make requests over a proxy randomly chosen from a list\n                        generated from a .csv file.\n  --check_proxies CHECK_PROXY, -cp CHECK_PROXY\n                        To be used with the \'--proxy_list\' parameter. The\n                        script will check if the proxies supplied in the .csv\n                        file are working and anonymous.Put 0 for no limit on\n                        successfully checked proxies, or another number to\n                        institute a limit.\n  --print-found         Do not output sites where the username was not found.\n\n```\n\nTo search for only one user:\n```\npython3 sherlock.py user123\n```\n\nTo search for more than one user:\n```\npython3 sherlock.py user1 user2 user3\n```\n\nAccounts found will be stored in an individual text file with the corresponding username (e.g ```user123.txt```).\n\n## Docker Notes\nIf docker is installed you can build an image and run this as a container.\n\n```\ndocker build -t mysherlock-image .\n```\n\nOnce the image is built, sherlock can be invoked by running the following:\n\n```\ndocker run --rm -t mysherlock-image user123\n```\n\nThe optional ```--rm``` flag removes the container filesystem on completion to prevent cruft build-up. See: https://docs.docker.com/engine/reference/run/#clean-up---rm\n\nThe optional ```-t``` flag allocates a pseudo-TTY which allows colored output. See: https://docs.docker.com/engine/reference/run/#foreground\n\nUse the following command to access the saved results:\n\n```\ndocker run --rm -t -v "$PWD/results:/opt/sherlock/results" mysherlock-image -o /opt/sherlock/results/text.txt user123\n```\n\nThe ```-v "$PWD/results:/opt/sherlock/results"``` option tells docker to create (or use) the folder `results` in the\npresent working directory and to mount it at `/opt/sherlock/results` on the docker container.\nThe `-o /opt/sherlock/results/text.txt` option tells `sherlock` to output the result.\n\nOr you can use "Docker Hub" to run `sherlock`:\n```\ndocker run theyahya/sherlock user123\n```\n\n### Using `docker-compose`\n\nYou can use the `docker-compose.yml` file from the repository and use this command:\n\n```\ndocker-compose run sherlok -o /opt/sherlock/results/text.txt user123\n```\n\n## Adding New Sites\n\nPlease look at the Wiki entry on\n[adding new sites](https://github.com/TheYahya/sherlock/wiki/Adding-Sites-To-Sherlock)\nto understand the issues.\n\n**NOTE**: Sherlock is not accepting adult sites in the standard list.\n\n## Tests\nThank you for contributing to Sherlock!\n\nBefore creating a pull request with new development, please run the tests\nto ensure that everything is working great.  It would also be a good idea to run the tests\nbefore starting development to distinguish problems between your\nenvironment and the Sherlock software.\n\nThe following is an example of the command line to run all the tests for\nSherlock.  This invocation hides the progress text that Sherlock normally\noutputs, and instead shows the verbose output of the tests.\n\n```\n$ python3 -m unittest tests.all --buffer --verbose\n```\n\nNote that we do currently have 100% test coverage.  Unfortunately, some of\nthe sites that Sherlock checks are not always reliable, so it is common\nto get response errors.\n\n## Stargazers over time\n\n[![Stargazers over time](https://starcharts.herokuapp.com/TheYahya/sherlock.svg)](https://starcharts.herokuapp.com/TheYahya/sherlock)\n\n## License\n\nMIT \xc2\xa9 [Yahya SayadArbabi](https://theyahya.com)<br/>\nOriginal Creator - [Siddharth Dushantha](https://github.com/sdushantha)\n'
