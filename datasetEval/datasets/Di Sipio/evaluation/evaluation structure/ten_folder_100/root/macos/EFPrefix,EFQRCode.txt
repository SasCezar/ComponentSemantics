recognizing QRCode from image, it is based on `CoreGraphics`, `CoreImage` and `ImageIO`. EFQRCode provides you a better way to operate QRCode in your app, it works on `iOS`, `macOS`, `watchOS` and `tvOS`, and it is available through `CocoaPods`, `Carthage` and `Swift Package Manager`. This project is inspired by [qrcode](https://github.com/sylnsfar/qrcode). \n\n> [\xe4\xb8\xad\xe6\x96\x87\xe4\xbb\x8b\xe7\xbb\x8d](https://github.com/EFPrefix/EFQRCode/blob/master/README_CN.md)\n\n## Overview\n\n![](https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/QRCode5.jpg)|![](https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/QRCode6.jpg)|![](https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/QRCode7.jpg)|![](https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/QRCode8.jpg)  \n:---------------------:|:---------------------:|:---------------------:|:---------------------:\n![](https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/QRCodeGIF1.gif)|![](https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/QRCodeGIF2.gif)|![](https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/QRCodeGIF7.gif)|![](https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/QRCodeGIF8.gif)  \n\n## Demo\n\n### App Store\n\nYou can click the `App Store` button below to download demo, support `iOS`, `tvOS` and `watchOS`:\n\n<a target=\'_blank\' href=\'https://itunes.apple.com/cn/app/EFQRCode/id1242337058?mt=8\'>\n    <img src=\'https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/icon/AppStore.jpeg\' width=\'144\' height=\'49\'/>\n</a>\n\nYou can also click the `Mac App Store` button below to download demo for `macOS`:\n\n<a target=\'_blank\' href=\'https://itunes.apple.com/cn/app/EFQRCode/id1306793539?mt=8\'>\n    <img src=\'https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/icon/AppStoreMac.png\' width=\'168.5\' height=\'49\'/>\n</a>\n\n### Manual\n\nTo run the example project manually, clone the repo, demos are in the \'Examples\' folder, remember run command `sh Startup.sh` in terminal to get all dependencies first, then open `EFQRCode.xcworkspace` with Xcode and select the target you want, run.\n\nOr you can run the following command in terminal:\n\n```bash\ngit clone git@github.com:EFPrefix/EFQRCode.git; To install\nit, simply add the following line to your Podfile:\n\n```ruby\npod \'EFQRCode\', \'~> 5.1.6\'\n```\n\nTry `pod \'EFQRCode/watchOS\'` instead if you need to use it in watchOS.\n\nThen, run the following command:\n\n```bash\n$ pod install\n```\n\n### Carthage\n\n[Carthage](https://github.com/Carthage/Carthage) is a decentralized dependency manager that builds your dependencies and provides you with binary frameworks.\n\nYou can install Carthage with [Homebrew](http://brew.sh/) using the following command:\n\n```bash\n$ brew update\n$ brew install carthage\n```\n\nTo integrate EFQRCode into your Xcode project using Carthage, specify it in your `Cartfile`:\n\n```ogdl\ngithub "EFPrefix/EFQRCode" ~> 5.1.6\n```\n\nRun `carthage update` to build the framework and drag the built `EFQRCode.framework` into your Xcode project.\n\n### Swift Package Manager\n\nThe [Swift Package Manager](https://swift.org/package-manager/) is a tool for automating the distribution of Swift code and is integrated into the `swift` compiler.\n\nOnce you have your Swift package set up, adding EFQRCode as a dependency is as easy as adding it to the `dependencies` value of your `Package.swift`.\n\n```swift\ndependencies: [\n    .Package(url: "https://github.com/EFPrefix/EFQRCode.git", Version(5, 1, 6))\n]\n```\n\n## Quick Start\n\n#### 1. Import EFQRCode\n\nImport EFQRCode module where you want to use it:\n\n```swift\nimport EFQRCode\n```\n\n#### 2. Recognition\n\nGet QR Codes from CGImage, maybe there are several codes in a image, so it will return an array:\n\n```swift\nif let testImage = UIImage(named: "test.png")?.cgImage() {\n    if let tryCodes = EFQRCode.recognize(image: testImage) {\n        if tryCodes.count > 0 {\n            print("There are \\(tryCodes.count) codes in testImage.")\n            for (index, code) in tryCodes.enumerated() {\n                print("The content of \\(index) QR Code is: \\(code).")\n            }\n        } else {\n            print("There is no QR Codes in testImage.")\n        }\n    } else {\n        print("Recognize failed, check your input image!")\n    }\n}\n```\n\n#### 3. Generation\n\nCreate QR Code image, quick usage:\n\n```swift\n//                    content: Content of QR Code\n//            size (Optional): Width and height of image\n// backgroundColor (Optional): Background color of QRCode\n// foregroundColor (Optional): Foreground color of QRCode\n//       watermark (Optional): Background image of QRCode\n```\n\n```swift\nif let tryImage = EFQRCode.generate(\n    content: "https://github.com/EFPrefix/EFQRCode",\n    watermark: UIImage(named: "WWF")?.cgImage()\n) {\n    print("Create QRCode image success: \\(tryImage)")\n} else {\n    print("Create QRCode image failed!")\n}\n```\n\nResult: \n\n<img src="https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/sample1.jpg" width = "36%"/>\n\n#### 4. Generation from GIF\n\nYou can create GIF QRCode with function `generateWithGIF` of class `EFQRCode`, for example:\n\n```swift\n//                  data: Data of input GIF\n//             generator: An object of EFQRCodeGenerator, use for setting\n// pathToSave (Optional): Path to save the output GIF, default is temp path\n//      delay (Optional): Output QRCode GIF delay, default is same as input GIF\n//  loopCount (Optional): Output QRCode GIF loopCount, default is same as input GIF\n```\n\n```swift\nif let qrcodeData = EFQRCode.generateWithGIF(data: data, generator: generator) {\n    print("Create QRCode image success.")\n} else {\n    print("Create QRCode image failed!")\n}\n```\n\nYou can get more information from the demo, result will like this:\n\n<img src="https://raw.githubusercontent.com/EFPrefix/EFQRCode/assets/QRCodeGIF6.gif" width = "36%"/>\n\n#### 5. Please select a high contrast foreground and background color combinations;\n2. Magnification too high\xef\xbc\x8fSize too long\xef\xbc\x8fContent too much may cause failure;\n4. It is recommended to test the QRCode image before put it into use;\n5.
