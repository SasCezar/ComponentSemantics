b'<h1 align="center">Crunchy Data PostgreSQL Operator</h1>\n<p align="center">\n  <img width="150" src="./crunchy_logo.png?raw=true"/>\n</p>\n\n\nThe *postgres-operator* is a controller that runs within a Kubernetes cluster that provides a means to deploy and manage PostgreSQL clusters.\n\nUse the postgres-operator to:\n\n * deploy PostgreSQL containers including streaming replication clusters\n * scale up PostgreSQL clusters with extra replicas\n * add pgpool, pgbouncer, and metrics sidecars to PostgreSQL clusters\n * apply SQL policies to PostgreSQL clusters\n * assign metadata tags to PostgreSQL clusters\n * maintain PostgreSQL users and passwords\n * perform minor upgrades to PostgreSQL clusters\n * load simple CSV and JSON files into PostgreSQL clusters\n * perform database backups\n\n\n## Deployment Requirements\n\nThe PostgreSQL Operator is validated for deployment on Kubernetes, OpenShift, and VMware Enterprise PKS clusters.  Some form of storage is required, NFS, hostPath, and Storage Classes are currently supported.\n\nThe PostgreSQL Operator includes various components that get deployed to your\nKubernetes cluster as shown in the following diagram and detailed\nin the Design section of the documentation for the version you are running.\n\n![Reference](https://access.crunchydata.com/documentation/postgres-operator/latest/Operator-Architecture.png)\n\nThe PostgreSQL Operator is developed and tested on CentOS and RHEL linux platforms but is known to run on other Linux variants.\n\n## Installation\n\n### PostgreSQL Operator Installation\n\nThe PostgreSQL Operator provides a few different methods for installation.  \n\nFor an automated deployment using Ansible playbooks, please start here:\n\n -
