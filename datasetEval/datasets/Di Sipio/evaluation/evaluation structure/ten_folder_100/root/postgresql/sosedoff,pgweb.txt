b"# pgweb\n\nWeb-based PostgreSQL database browser written in Go.\n\n[![Release](https://img.shields.io/github/release/sosedoff/pgweb.svg?label=Release)](https://github.com/sosedoff/pgweb/releases)\n[![Linux Build](https://img.shields.io/travis/sosedoff/pgweb/master.svg?label=Linux)](https://travis-ci.org/sosedoff/pgweb)\n[![Windows Build](https://img.shields.io/appveyor/ci/sosedoff/pgweb/master.svg?label=Windows)](https://ci.appveyor.com/project/sosedoff/pgweb)\n[![Go Report Card](https://goreportcard.com/badge/github.com/sosedoff/pgweb)](https://goreportcard.com/report/github.com/sosedoff/pgweb)\n[![GoDoc](https://godoc.org/github.com/sosedoff/pgweb?status.svg)](https://godoc.org/github.com/sosedoff/pgweb)\n[![Docker Pulls](https://img.shields.io/docker/pulls/sosedoff/pgweb.svg)](https://hub.docker.com/r/sosedoff/pgweb/)\n\n## Overview\n\nPgweb is a web-based database browser for PostgreSQL, written in Go and works\non OSX, Linux and Windows machines. Main idea behind using Go for backend development\nis to utilize ability of the compiler to produce zero-dependency binaries for \nmultiple platforms. Pgweb was created as an attempt to build very simple and portable\napplication to work with local or remote PostgreSQL databases.\n\n[See application screenshots](SCREENS.md)\n\n## Features\n\n- Cross-platform support OSX/Linux/Windows 32/64-bit\n- Simple installation (distributed as a single binary)\n- Zero dependencies\n- Works with PostgreSQL 9.1+\n- SSH Connections\n- Multiple database sessions\n- Simple database browser\n- Execute and analyze custom SQL queries\n- Table and query data export to CSV/JSON/XML\n- Query history\n- Server bookmarks\n\nVisit [WIKI](https://github.com/sosedoff/pgweb/wiki) for more details\n\n## Pgweb Pro\n\nPgweb Pro is the next major version of Pgweb and includes features like:\n\n- Table structure editing\n- Data editing (update row content)\n- Charting\n- History persistence\n- Multiple tabs\n\nPlease get in touch via: https://goo.gl/forms/euQOGWg5uPdPH70b2\n\n## Demo\n\nVisit https://pgweb-demo.herokuapp.com to see pgweb in action.\n\n## Installation\n\n[Precompiled binaries](https://github.com/sosedoff/pgweb/releases) for supported \noperating systems are available.\n\n[More installation options](https://github.com/sosedoff/pgweb/wiki/Installation)\n\n## Usage\n\nStart server:\n\n```\npgweb\n```\n\nYou can also provide connection flags:\n\n```\npgweb --host localhost --user myuser --db mydb\n```\n\nConnection URL scheme is also supported:\n\n```\npgweb --url postgres://user:password@host:port/database?sslmode=[mode]\n```\n\n### Multiple database sessions\n\nTo enable multiple database sessions in pgweb, start the server with:\n\n```\npgweb --sessions\n```\n\nOr set environment variable:\n\n```\nSESSIONS=1 pgweb\n```\n\n## Deploy on Heroku\n\n[![Heroku Deploy](https://www.herokucdn.com/deploy/button.svg)](https://heroku.com/deploy?template=https://github.com/sosedoff/pgweb)\n\n## Testing\n\nBefore running tests, make sure you have PostgreSQL server running on `localhost:5432`\ninterface. Also, you must have `postgres` user that could create new databases\nin your local environment. Pgweb server should not be running at the same time.\n\nExecute test suite:\n\n```\nmake test\n```\n\nIf you're using Docker locally, you might also run pgweb test suite against\nall supported PostgreSQL version with a single command:\n\n```\nmake test-all\n```\n\n## Contribute\n\n- Fork this repository\n- Create a new feature branch for a new functionality or bugfix\n- Commit your changes\n- Execute test suite\n- Push your code and open a new pull request\n- Use [issues](https://github.com/sosedoff/pgweb/issues) for any questions\n- Check [wiki](https://github.com/sosedoff/pgweb/wiki) for extra documentation\n\n## Contact\n\n- Dan Sosedoff\n- [dan.sosedoff@gmail.com](mailto:dan.sosedoff@gmail.com)\n- [http://twitter.com/sosedoff](http://twitter.com/sosedoff)\n\n## License\n\nThe MIT License (MIT).
