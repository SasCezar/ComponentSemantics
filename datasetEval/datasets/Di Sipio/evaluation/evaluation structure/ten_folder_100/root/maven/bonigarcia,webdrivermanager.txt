[About](#about)\n\n## Motivation\n\nIf you use [Selenium WebDriver], you will know that in order to use some browsers such as **Chrome**, **Firefox**, **Opera**, **PhantomJS**, **Microsoft Edge**, or **Internet Explorer**, first you need to download a binary file which allows WebDriver to handle browsers. In Java, the path to this binary must be set as JVM properties, as follows:\n\n```java\nSystem.setProperty("webdriver.chrome.driver", "/path/to/binary/chromedriver");\nSystem.setProperty("webdriver.gecko.driver", "/path/to/binary/geckodriver");\nSystem.setProperty("webdriver.opera.driver", "/path/to/binary/operadriver");\nSystem.setProperty("phantomjs.binary.path", "/path/to/binary/phantomjs");\nSystem.setProperty("webdriver.edge.driver", "C:/path/to/binary/msedgedriver.exe");\nSystem.setProperty("webdriver.ie.driver", "C:/path/to/binary/IEDriverServer.exe");\n```\n\nThis is quite annoying since it forces you to link directly this binary file into your source code. In addition, you have to check manually when new versions of the binaries are released. WebDriverManager comes to the rescue, performing in an automated way all this dirty job for you. WebDriverManager can be used in 3 different ways:\n\n1. [WebDriverManager as Java dependency](#webdrivermanager-as-java-dependency) (typically from test cases).\n2. [WebDriverManager as a Command Line Interface (CLI) tool](#webdrivermanager-cli) (from the shell).\n3. [WebDriverManager as a Server](#webdrivermanager-server) (using a REST-like API).\n\nWebDriverManager is open source, released under the terms of [Apache 2.0 License].\n\n## WebDriverManager as Java dependency\n\n### Basic usage\n\nIn order to use WebDriverManager from tests in a Maven project, you need to add the following dependency in your `pom.xml` (Java 8 or upper required):\n\n```xml\n<dependency>\n    <groupId>io.github.bonigarcia</groupId>\n    <artifactId>webdrivermanager</artifactId>\n    <version>3.7.1</version>\n    <scope>test</scope>\n</dependency>\n```\n\n... or in Gradle project:\n\n```\ndependencies {\n    testCompile("io.github.bonigarcia:webdrivermanager:3.7.1")\n}\n```\n\nOnce we have included this dependency, you can let WebDriverManager to manage the WebDriver binaries for you. Take a look at this JUnit 4 example which uses Chrome with Selenium WebDriver (in order to use WebDriverManager in conjunction with **JUnit 5**, the extension [Selenium-Jupiter] is highly recommended):\n\n```java\npublic class ChromeTest {\n\n    private WebDriver driver;\n\n    @BeforeClass\n    public static void setupClass() {\n        WebDriverManager.chromedriver().setup();\n    }\n\n    @Before\n    public void setupTest() {\n        driver = new ChromeDriver();\n    }\n\n    @After\n    public void teardown() {\n        if (driver != null) {\n            driver.quit();\n        }\n    }\n\n    @Test\n    public void test() {\n        // Your test code here\n    }\n\n}\n```\n\nNotice that simply adding ``WebDriverManager.chromedriver().setup();`` WebDriverManager does magic for you:\n\n1. It checks the version of the browser installed in your machine (e.g. Chrome, Firefox).\n2. It checks the version of the driver (e.g. *chromedriver*, *geckodriver*). If unknown, it uses the latest version of the driver.\n3. It downloads the WebDriver binary if it is not present on the WebDriverManager cache (``~/.m2/repository/webdriver`` by default).\n4. It exports the proper WebDriver Java environment variables required by Selenium (not done when using WebDriverManager from the CLI or as a Server).\n\nWebDriverManager resolves the driver binaries for the browsers **Chrome**, **Firefox**, **Opera**, **PhantomJS**, **Microsoft Edge**, and **Internet Explorer**. For that, it provides several *drivers managers* for these browsers. These *drivers managers* can be used as follows:\n\n```java\nWebDriverManager.chromedriver().setup();\nWebDriverManager.firefoxdriver().setup();\nWebDriverManager.operadriver().setup();\nWebDriverManager.phantomjs().setup();\nWebDriverManager.edgedriver().setup();\nWebDriverManager.iedriver().setup();\n```\n\n**NOTE**: The old WebDriverManager API (version 1.x) has been deprecated as of version 3.x (`ChromeDriverManager.getInstance().setup();`, `FirefoxDriverManager.getInstance().setup();`, and so on).\n\nMoreover, WebDriverManager provides a generic *driver manager*. This manager which can be parameterized using Selenium driver classes (e.g. `org.openqa.selenium.chrome.ChromeDriver`, `org.openqa.selenium.firefox.FirefoxDriver`, etc), as follows: \n\n```java\nimport org.openqa.selenium.WebDriver;\nimport org.openqa.selenium.chrome.ChromeDriver;\nimport io.github.bonigarcia.wdm.WebDriverManager;\n\n// ...\n\nClass<? extends WebDriver> driverClass = ChromeDriver.class;\nWebDriverManager.getInstance(driverClass).setup();\nWebDriver driver = driverClass.newInstance();\n```\n\nThis generic *driver manager* can be also parameterized using the enumeration `DriverManagerType`. For instance as follows:\n\n```java\nimport static io.github.bonigarcia.wdm.DriverManagerType.CHROME;\n\nimport org.openqa.selenium.WebDriver;\nimport org.openqa.selenium.chrome.ChromeDriver;\nimport io.github.bonigarcia.wdm.WebDriverManager;\n\n// ...\n\nWebDriverManager.getInstance(CHROME).setup();\nWebDriver driver = new ChromeDriver();\n```\n\n### Examples\n\nCheck out the repository [WebDriverManager Examples] which contains different JUnit 4 test examples using WebDriverManager.\n\n\n### Driver versions\n\nThe relationship between browser version and driver version is managed in a internal database by WebDriverManager stored as Java properties in the file [versions.properties]. This file is maintained updated when a new driver or browser version is released. In order to use always the latest relationships between browser and driver, and due to its rapid change, the online version of [versions.properties] (master branch) is used. As alternative, the local version of this file (distributed within WebDriverManager) can be used.\n\nIn order to resolve the driver version for a given browser, first WebDriverManager try to find out the version of that browser. This mechanism depends on the browser (Chrome, Firefox, etc) and the platform (Linux, Windows, Mac). For instance, for Chrome in Linux, the command ``google-chrome --version`` is executed in the shell.  \n\nMoreover in order to improve the performance of WebDriverManager, as of version 3.1.0, resolved driver versions for browsers are stored persistently as Java preferences. The validity of this relationship (browser version and driver version) stored as preferences is linked to a *time-to-live* (ttl). By default, this value is 86400 seconds (i.e. 1 day).\n\n\n### WebDriverManager API\n\nWebDriverManager exposes its API by means of the **builder pattern**. parameters for WebDriverManager are set in the ``webdrivermanager.properties`` file:\n\n```properties\nwdm.targetPath=~/.m2/repository/webdriver\nwdm.forceCache=false\nwdm.override=false\nwdm.useMirror=false\nwdm.useBetaVersions=false\nwdm.avoidExport=false\nwdm.avoidOutputTree=false\nwdm.avoidAutoVersion=false\nwdm.avoidAutoReset=false\nwdm.avoidPreferences=false\nwdm.timeout=30\nwdm.serverPort=4041\nwdm.ttl=86400\n\nwdm.chromeDriverUrl=https://chromedriver.storage.googleapis.com/\nwdm.chromeDriverMirrorUrl=http://npm.taobao.org/mirrors/chromedriver\nwdm.chromeDriverExport=webdriver.chrome.driver\n\nwdm.geckoDriverUrl=https://api.github.com/repos/mozilla/geckodriver/releases\nwdm.geckoDriverMirrorUrl=http://npm.taobao.org/mirrors/geckodriver\nwdm.geckoDriverExport=webdriver.gecko.driver\n\nwdm.operaDriverUrl=https://api.github.com/repos/operasoftware/operachromiumdriver/releases\nwdm.operaDriverMirrorUrl=http://npm.taobao.org/mirrors/operadriver\nwdm.operaDriverExport=webdriver.opera.driver\n\nwdm.phantomjsDriverUrl=https://bitbucket.org/ariya/phantomjs/downloads/\nwdm.phantomjsDriverMirrorUrl=http://npm.taobao.org/mirrors/phantomjs\nwdm.phantomjsDriverExport=phantomjs.binary.path\n\nwdm.edgeDriverUrl=https://developer.microsoft.com/en-us/microsoft-edge/tools/webdriver/\nwdm.edgeDriverExport=webdriver.edge.driver\n\nwdm.internetExplorerDriverUrl=https://selenium-release.storage.googleapis.com/\nwdm.internetExplorerDriverExport=webdriver.ie.driver\n\nwdm.seleniumServerStandaloneUrl=https://selenium-release.storage.googleapis.com/\n\nwdm.versionsPropertiesOnlineFirst=true\nwdm.versionsPropertiesUrl=https://raw.githubusercontent.com/bonigarcia/webdrivermanager/master/src/main/resources/versions.properties\n```\n\nFor instance, the variable ``wdm.targetPath`` is the default folder in which WebDriver binaries are going to be stored. By default the path of the Maven local repository is used. This property can be overwritten by Java system properties, for example:\n\n```java\nSystem.setProperty("wdm.targetPath", "/my/custom/path/to/driver/binaries");\n```\n\n... or by command line, for example:\n\n```properties\n-Dwdm.override=true\n```\n\nBy default, WebDriverManager downloads the latest version of the WebDriver binary. But concrete versions of WebDriver binaries can be forced by changing the value of the variables ``wdm.chromeDriverVersion``, ``wdm.operaDriverVersion``,  ``wdm.internetExplorerDriverVersion``, or  ``wdm.edgeDriverVersion`` to a concrete version. For instance:\n\n```properties\n-Dwdm.chromeDriverVersion=2.25\n-Dwdm.internetExplorerDriverVersion=2.46\n-Dwdm.operaDriverVersion=0.2.0\n-Dwdm.edgeDriverVersion=3.14366\n-Dwdm.phantomjsDriverVersion=2.1.1\n-Dwdm.geckoDriverVersion=0.11.1\n```\n\nIf no version is specified, WebDriverManager sends a request to the server hosting the binary. In order to avoid this request and check if any binary has been previously downloaded, the key `wdm.forceCache` can be used.\n\nThe value of these properties can be overridden by means of *environmental variables*. The name of these variables result from putting the name in uppercase and replacing the symbol `.` by `_`. For example, the property ``wdm.targetPath`` can be overridden by the environment variable ``WDM_TARGETPATH``.\n\nMoreover, configuration values can be customized from Java code. The configuration object is unique per driver manager (i.e. ``WebDriverManager.chromedriver()``,  ``WebDriverManager.firefoxdriver()``, etc.) and is accessed using the method ``config()``. The configuration parameters can be reset to default values using the method ``reset()``. For example:\n\n```java\nWebDriverManager.chromedriver().config().setProperties("/path/to/my-wdm.properties");\nWebDriverManager.firefoxdriver().config().setForceCache(true);\nWebDriverManager.operadriver().config().setOverride(true);\nWebDriverManager.chromedriver().config().reset();\n```\n\nFinally, if a single configuration object needs to be specified for all the driver managers, it can be done using the static method ``globalConfig()``. This global configuration is reset as usual using the ``reset()`` method. For example:\n\n```java\nWebDriverManager.globalConfig().setTargetPath("/path/to/my-wdm-cache");\nWebDriverManager.globalConfig().reset();\n```\n\n## WebDriverManager CLI\n\nAs of version 2.2.0, WebDriverManager can used interactively from the Command Line Interface (CLI), i.e. the shell, to resolve and download binaries for the supported browsers. There are two ways of using this feature:\n\n* Directly from the source code, using Maven. The command to be used is ``mvn exec:java -Dexec.args="browserName"``. For instance:\n\n```\n> mvn exec:java -Dexec.args="chrome"\n[INFO] Scanning for projects...\n[INFO]\n[INFO] ------------------------------------------------------------------------\n[INFO] Building WebDriverManager 3.7.1\n[INFO] ------------------------------------------------------------------------\n[INFO]\n[INFO] --- exec-maven-plugin:1.6.0:java (default-cli) @ webdrivermanager ---\n[INFO] Using WebDriverManager to resolve chrome\n[INFO] Using chromedriver 73.0.3683.68 (since Google Chrome 73 is installed in your machine)\n[INFO] Reading https://chromedriver.storage.googleapis.com/ to seek chromedriver\n[INFO] Downloading https://chromedriver.storage.googleapis.com/73.0.3683.68/chromedriver_win32.zip\n[INFO] Extracting binary from compressed file chromedriver_win32.zip\n[INFO] Resulting binary D:\\projects\\webdrivermanager\\chromedriver.exe\n[INFO] ------------------------------------------------------------------------\n[INFO] BUILD SUCCESS\n[INFO] ------------------------------------------------------------------------\n[INFO] Total time: 14.919 s\n[INFO] Finished at: 2019-04-11T00:36:32+02:00\n[INFO] ------------------------------------------------------------------------\n```\n\n* Using WebDriverManager as a *fat-jar* (i.e. WebDriverManager with all its dependencies in a single executable JAR file). This JAR file can downloaded from [here](https://github.com/bonigarcia/webdrivermanager/releases/download/webdrivermanager-3.7.1/webdrivermanager-3.7.1-fat.jar) and also it can be created using the command ``mvn compile assembly:single`` from the source code. Once you get the *fat-jar*, you simply need to use the command ``java -jar webdrivermanager-3.7.1-fat.jar browserName``, for instance:\n\n```\n> java -jar webdrivermanager-3.7.1-fat.jar chrome\n[INFO] Using WebDriverManager to resolve chrome\n[INFO] Using chromedriver 73.0.3683.68 (since Google Chrome 73 is installed in your machine)\n[INFO] Reading https://chromedriver.storage.googleapis.com/ to seek chromedriver\n[INFO] Downloading https://chromedriver.storage.googleapis.com/73.0.3683.68/chromedriver_win32.zip\n[INFO] Extracting binary from compressed file chromedriver_win32.zip\n[INFO] Resulting binary D:\\projects\\webdrivermanager\\chromedriver.exe\n```\n\n## WebDriverManager server\n\nAs of version 3.0.0, WebDriverManager can used as a server. To start this mode, the shell is used. Once again, two options are allowed:\n\n* Directly from the source code and Maven. The command to be used is ``mvn exec:java -Dexec.args="server <port>"``. If the second argument is not specified, the default port will be used (4041):\n\n```\n$ mvn exec:java -Dexec.args="server"\n[INFO] Scanning for projects...\n[INFO]\n[INFO] ------------------------------------------------------------------------\n[INFO] Building WebDriverManager 3.7.1\n[INFO] ------------------------------------------------------------------------\n[INFO]\n[INFO] --- exec-maven-plugin:1.6.0:java (default-cli) @ webdrivermanager ---\n[INFO] WebDriverManager server listening on port 4041\n```\n\n* Using WebDriverManager as a [fat-jar](https://github.com/bonigarcia/webdrivermanager/releases/download/webdrivermanager-3.7.1/webdrivermanager-3.7.1-fat.jar). For instance:\n\n```\n> java -jar webdrivermanager-3.7.1-fat.jar server\n[INFO] WebDriverManager server listening on port 4041\n```\n\nWhen the WebDriverManager is up and running, a REST-like API using HTTP request can be done to resolve driver binaries (*chromedriver*, *geckodriver*, etc.). For instance, supposing that WebDriverManager is running the local host and in the default port:\n\n* http://localhost:4041/chromedriver : To download the latest version of *chromedriver*\n* http://localhost:4041/firefoxdriver : To download the latest version of *geckodriver*\n* http://localhost:4041/operadriver : To download the latest version of *operadriver*\n* http://localhost:4041/phantomjs : To download the latest version of *phantomjs* driver\n* http://localhost:4041/edgedriver : To download the latest version of *msedgedriver*\n* http://localhost:4041/iedriver : To download the latest version of *IEDriverServer*\n\nThese requests use HTTP GET and can be done using a normal browser. The driver binary is automatically downloaded by the browser since it is sent as an attachment in the HTTP response.\n\nIn addition, configuration parameters can be specified in the URL using query arguments. The name of these arguments are identical to the parameters in the ``webdrivermanager.properties`` file but skipping the prefix ``wdm.`` (see [configuration](#webdrivermanager-api) section). For instance: \n\n| Example                                                           | Description                                                       |\n|-------------------------------------------------------------------|-------------------------------------------------------------------|\n| http://localhost:4041/chromedriver?chromeDriverVersion=2.40       | Downloads the version 2.40 of *chromedriver*                      |\n| http://localhost:4041/firefoxdriver?geckoDriverVersion=0.21.0     | Downloads the version 0.21.0 of *geckodriver*                     |\n| http://localhost:4041/operadriver?os=WIN&forceCache=true          | Force to use the cache version of *operadriver* for Windows       |\n\nFinally, requests to WebDriverManager server can be done interactively using tools such as [curl], as follows:\n\n```\ncurl -O -J http://localhost:4041/chromedriver\ncurl -O -J "http://localhost:4041/chromedriver?chromeDriverVersion=2.40&forceCache=true"\n\n```\n\n## Known issues\n\n### HTTP response code 403\n\nSome of the binaries (for Opera and Firefox) are hosted on GitHub. When several consecutive requests are made by WebDriverManager, GitHub servers return an **HTTP 403 error** response as follows:\n\n```\nCaused by: java.io.IOException: Server returned HTTP response code: 403 for URL: https://api.github.com/repos/operasoftware/operachromiumdriver/releases\n    at sun.net.www.protocol.http.HttpURLConnection.getInputStream0(HttpURLConnection.java:1840)\n    at sun.net.www.protocol.http.HttpURLConnection.getInputStream(HttpURLConnection.java:1441)\n    at sun.net.www.protocol.https.HttpsURLConnectionImpl.getInputStream(HttpsURLConnectionImpl.java:254)\n    at io.github.bonigarcia.wdm.BrowserManager.openGitHubConnection(BrowserManager.java:463)\n    at io.github.bonigarcia.wdm.OperaDriverManager.getDrivers(OperaDriverManager.java:55)\n    at io.github.bonigarcia.wdm.BrowserManager.manage(BrowserManager.java:168)\n```\n\n```\nCaused by: java.io.IOException: Server returned HTTP response code: 403 for URL: https://api.github.com/repos/mozilla/geckodriver/releases\n    at sun.net.www.protocol.http.HttpURLConnection.getInputStream0(HttpURLConnection.java:1840)\n    at sun.net.www.protocol.http.HttpURLConnection.getInputStream(HttpURLConnection.java:1441)\n    at sun.net.www.protocol.https.HttpsURLConnectionImpl.getInputStream(HttpsURLConnectionImpl.java:254)\n    at io.github.bonigarcia.wdm.FirefoxDriverManager.getDrivers(FirefoxDriverManager.java:61)\n    at io.github.bonigarcia.wdm.BrowserManager.manage(BrowserManager.java:163)\n```\n\nIn order to avoid this problem, [authenticated requests] should be done. The procedure is the following:\n\n1. Create a token/secret pair in your [GitHub account].\n2. Tell WebDriverManager the value of this pair token/secret. To do that you should use the configuration keys ``wdm.gitHubTokenName`` and ``wdm.gitHubTokenSecret``. You can pass them as command line Java parameters as follows:\n\n```properties\n-Dwdm.gitHubTokenName=<your-token-name>\n-Dwdm.gitHubTokenSecret=<your-token-secret>\n```\n\n... or as environment variables (e.g. in Travis CI) as follows:\n\n```properties\nWDM_GITHUBTOKENNAME=<your-token-name>\nWDM_GITHUBTOKENSECRET=<your-token-secret>\n```\n\n### Tons of org.apache.http DEBUG log\n\nWebDriverManager uses [Apache HTTP Client] to download WebDriver binaries from online repositories. Internally, Apache HTTP client writes a lot of logging information using the `DEBUG` level of `org.apache.http` classes. To reduce this amount of logs, the level of this logger might be reduced. For instance, in the case of [Logback], the log configuration file should include the following:\n\n```xml\n<configuration>\n    <logger name="org.apache.http" level="WARN" />\n</configuration>\n```\n\nYou can find further information about others logging implementations in the [Apache HTTP Client logging practices] page.\n\n\n## Help\n\nIf you have questions on how to use WebDriverManager properly with a special configuration or suchlike, please consider asking a question on [Stack Overflow] and tag it with  *webdrivermanager-java*.\n\n\n## Backers\n\nThank you to all our backers!
