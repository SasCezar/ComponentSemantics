demonstration of a completely stateless and RESTful token-based authorization system using JSON Web Tokens (JWT) and Spring Security.\n\n## Why?\nFor an API to be truly RESTful, no application state can be stored on the server itself. One particular challenge in implementing this is ensuring that your API is secure. Cerberus is the answer to this problem; access to the endpoints in the API requires a JSON Web Token to be present in the request header. This token is obtained by successfully performing an authentication request with the API, and afterwards this token will grant access to the API based on the authorities granted to the specified user.\n\n## Requirements\nCerberus requires Maven and Java 1.7 or greater.\n\n## Usage\nTo use start Cerberus, run in the terminal `mvn spring-boot:run`. Cerberus will now be running at `http://localhost:8080/api/`\n\nThere are two built-in user accounts to demonstrate the differing levels of access to the endpoints in the API:\n```\nUser - user:password\nAdmin - admin:admin\n```\n\nCerberus also has two endpoints. The first is the authentication endpoint, which is unrestricted. The second is a protected endpoint which only admin users may access (provided the correct JWT token is present in the request header):\n```\n/api/auth\n/api/protected\n```\n\nTo authenticate with Cerberus, you can curl a POST request with the following credentials to receive a JWT token:\n```\ncurl -i -H "Content-Type: application/json" -X POST -d \'{"username":"admin","password":"admin"}\' http://localhost:8080/api/auth\n```\n\nThe response should look like this:\n```\n{\n  "token" : "eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiJhZG1pbiJ9.bKweskM-3QqOY8ScxhC9AcREOCG2UDY0Ylezdv1h81ALFg_v0QYBgxwfUjtf_Ns7RqAQIh_kFg1ZkeFV-szRUg"\n}\n```\n\nYou can now insert this token into your request header for GET access to `/api/protected`:\n```\ncurl -i -H "Content-Type: application/json" -H "X-Auth-Token: eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiJhZG1pbiJ9.bKweskM-3QqOY8ScxhC9cREOCG2UDY0Ylezdv1h81ALFg_v0QYBgxwfUjtf_Ns7RqAQIh_kFg1ZkeFV-szRUg" -X GET http://localhost:8080/api/protected\n```\n\nYou should get an HTTP 200 and the response `:O`\n\nTokens are configured to expire after a week. To ensure that a token remains fresh and does not expire, you can refresh an existing token by sending a GET to `/api/auth/refresh` with the token set in the request header. The response will be a new token with an updated expiration date. This refresh mechanism only works for tokens that have not expired yet, unless the token was provided to a mobile device. Tokens for mobile devices can always be refreshed.\n\n## Testing\nTo run Cerberus\'s unit tests, run in the terminal `mvn clean package`.\n'
