b'# [<img src="ecs-logo.png" width="30%">](https://aws.amazon.com/ecs/) [<img src="fargate-logo.png" width="33%">](https://aws.amazon.com/fargate/)\n\nA curated list of guides, development tools, and resources for [Amazon Elastic Container Service](https://aws.amazon.com/ecs/) (ECS). This list includes both community created content as well as content created by AWS.\n\n_Want to add something? Open a PR!_ \xf0\x9f\x99\x82\n\n### Pick your container hosting strategy:\n\n- <a href="#aws-fargate">AWS Fargate</a> - AWS Fargate is a technology for Amazon ECS that allows you to run containers without having to manage servers or clusters.\n- <a href="#self-hosted-in-ec2">Self hosted in EC2</a> - Running your own cluster of EC2 instances to host your containers gives you the most control over price (ability to run on spot instances or reserved instances) as well as configuration.\n\n<br />\n\n## AWS Fargate\n\n### Setting up Fargate\n\n- [fargate](http://somanymachines.com/fargate/) - Command line tool for interacting with AWS Fargate. With just a single command you can build, push, and launch your container in Fargate, orchestrated by ECS.\n- [CloudFormation Templates](https://github.com/awslabs/aws-cloudformation-templates/tree/master/aws/services/ECS) - Sample CloudFormation templates to help you spin up a Fargate cluster, and service in that cluster automatically\n- [Terraform](https://thecode.pub/easy-deploy-your-docker-applications-to-aws-using-ecs-and-fargate-a988a1cc842f) - Use Terraform to deploy your docker containers in Fargate\n- [Terraform Airship Modules](https://airship.tf/getting_started/) Full getting started guide to setup your Fargate ECS Services.\n- [fargate-create](https://github.com/turnerlabs/fargate-create) - A CLI tool for creating new projects based on Terraform templates and [Fargate CLI](https://github.com/turnerlabs/fargate). Supported stacks:\n  - [Web Application (ALB - HTTP/HTTPS)](https://github.com/turnerlabs/terraform-ecs-fargate)\n  - [Network Application (NLB)](https://github.com/turnerlabs/terraform-ecs-fargate-nlb)\n  - [Background Worker (Service)](https://github.com/turnerlabs/terraform-ecs-fargate-background-worker)\n  - [Scheduled Task (Job)](https://github.com/turnerlabs/terraform-ecs-fargate-scheduled-task)\n\n### Application Deployments in Fargate\n\n- [Deploying a Rails app to Fargate](http://blog.scoutapp.com/articles/2018/01/08/deploying-to-aws-part-i-running-a-rails-app-on-fargate) - Step by step walkthrough of deploying a Ruby + RDS app, with helpful debugging tips\n- [How to use AWS Fargate and Lambda for long-running processes in a Serverless app](https://serverless.com/blog/serverless-application-for-long-running-process-fargate-lambda/) - Great tutorial showing how to leverage the power of long running docker containers in Fargate alongside Lambda. Example application processes video files to extract thumbnails using just S3, Lambda, and Fargate... no EC2.\n- [Microservice CI/CD to AWS Fargate](https://ecsworkshop.com/) - Learn how to use Stelligent Mu to setup a CI/CD pipeline for deploying a 3 tier, polyglot, microservice based application to AWS Fargate\n- [Wonqa](https://www.npmjs.com/package/wonqa) is a tool for spinning up disposable QA environments in AWS Fargate, with SSL enabled by Let\'s Encrypt. More details about Wonqa on the [Wonder Engineering blog](https://medium.com/wonder-engineering/on-demand-qa-environments-with-aws-fargate-c23b41f15a0c).\n- [Setting up service discovery for AWS Fargate using CloudFormation](https://paul.annesley.cc/ecs-service-discovery-cloudformation/)\n- [Using the Clair image scanner with Fargate, ECR and CodeBuild/CodePipeline](https://github.com/jasonumiker/clair-ecs-fargate)\n- [A lighter way to deploy to ECS/Fargate](https://ramblingsofasoftwaredevelopermanager.wordpress.com/2019/05/18/a-lighter-way-to-deploy-to-aws-ecs/) - combining update-service/task data from several sources including metadata of the docker image itself to deploy new versions.\n\n<br />\n\n## Self hosted in EC2\n\n- <a href="#getting-started">Getting Started</a>\n- <a href="#infrastructure-as-code">Infrastructure as Code</a>\n- <a href="#build-and-deploy-tools">Build and Deploy Tools</a>\n- <a href="#implementation-guides">Implementation Guides</a>\n- <a href="#open-source">Open Source</a>\n- <a href="#reference-architectures">Reference Architectures</a>\n- <a href="#continuous-integration--continuous-deployment">Continuous Integration / Continuous Deployment</a>\n- <a href="#presentations">Presentations</a>\n- <a href="#tech-blogs">Tech Blogs</a>\n\n### Getting Started\n  - [The Hitchhikers Guide to AWS ECS and Docker](http://start.jcolemorrison.com/the-hitchhikers-guide-to-aws-ecs-and-docker/) by [J.
