b'![Foundatio](https://raw.githubusercontent.com/FoundatioFx/Foundatio/master/media/foundatio.png "Foundatio")\n\n[![Build status](https://ci.appveyor.com/api/projects/status/8ccl17fc2tahbwuu?svg=true)](https://ci.appveyor.com/project/Exceptionless/foundatio)\n[![NuGet Version](http://img.shields.io/nuget/v/Foundatio.svg?style=flat)](https://www.nuget.org/packages/Foundatio/)\n[![Slack Status](https://slack.exceptionless.com/badge.svg)](https://slack.exceptionless.com)\n\nPluggable foundation blocks for building loosely coupled distributed apps.\n- [Caching](#caching)\n- [Queues](#queues)\n- [Locks](#locks)\n- [Messaging](#messaging)\n- [Jobs](#jobs)\n- [File Storage](#file-storage)\n- [Metrics](#metrics)\n\nIncludes implementations in Redis, Azure, AWS and in memory (for development). \n\n## Why Foundatio?\nWhen building several big cloud applications we found a lack of great solutions (that\'s not to say there isn\'t solutions out there) for many key pieces to building scalable distributed applications while keeping the development experience simple. Here are a few examples of why we built and use Foundatio:\n * Wanted to build against abstract interfaces so that we could easily change implementations.\n * Wanted the blocks to be dependency injection friendly.\n * Caching: We were initially using an open source Redis cache client but then it turned into a commercial product with high licensing costs. Not only that, but there weren\'t any in memory implementations so every developer was required to set up and configure Redis.\n * Message Bus: We initially looked at [NServiceBus](http://particular.net/nservicebus) (great product) but it had high licensing costs (they have to eat too) but was not OSS friendly. We also looked into [MassTransit](http://masstransit-project.com/) but found Azure support lacking and local set up a pain. We wanted a simple message bus that just worked locally or in the cloud.\n * Storage: We couldn\'t find any existing project that was decoupled and supported in memory, file storage or Azure Blob Storage.\n\nTo summarize, if you want pain free development and testing while allowing your app to scale, use Foundatio!\n\n## Implementations\n- [Redis](https://github.com/FoundatioFx/Foundatio.Redis) - Caching, Storage, Queues, Messaging, Locks, Metrics\n- [Azure Storage](https://github.com/FoundatioFx/Foundatio.AzureStorage) - Storage, Queues\n- [Azure ServiceBus](https://github.com/FoundatioFx/Foundatio.AzureServiceBus) - Queues, Messaging\n- [AWS](https://github.com/FoundatioFx/Foundatio.AWS) - Storage, Queues, Metrics\n- [RabbitMQ](https://github.com/FoundatioFx/Foundatio.RabbitMQ) - Queues\n- [Minio](https://github.com/FoundatioFx/Foundatio.Minio) - Storage\n- [Aliyun](https://github.com/FoundatioFx/Foundatio.Aliyun) - Storage\n- [SshNet](https://github.com/FoundatioFx/Foundatio.Storage.SshNet) - Storage\n\n## Getting Started (Development)\n\n[Foundatio can be installed](https://www.nuget.org/packages?q=Foundatio) via the [NuGet package manager](https://docs.nuget.org/consume/Package-Manager-Dialog). Please let us know if you have any questions or need assistance!\n\n### [Caching](https://github.com/FoundatioFx/Foundatio/tree/master/src/Foundatio/Caching)\n\nCaching allows you to store and access data lightning fast, saving you exspensive operations to create or get data. We provide four different cache implementations that derive from the [`ICacheClient` interface](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Caching/ICacheClient.cs):\n\n1. [InMemoryCacheClient](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Caching/InMemoryCacheClient.cs): An in memory cache client implementation. This cache implementation is only valid for the lifetime of the process. It\'s worth noting that the in memory cache client has the ability to cache the last X items via the `MaxItems` property. We use this in [Exceptionless](https://github.com/exceptionless/Exceptionless) to only [keep the last 250 resolved geoip results](https://github.com/exceptionless/Exceptionless/blob/master/src/Exceptionless.Core/Geo/MaxMindGeoIpService.cs).\n2. [HybridCacheClient](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Caching/HybridCacheClient.cs): This cache implementation uses the `InMemoryCacheClient` and uses the `IMessageBus` to keep the cache in sync across processes.\n3. [RedisCacheClient](https://github.com/FoundatioFx/Foundatio.Redis/blob/master/src/Foundatio.Redis/Cache/RedisCacheClient.cs): A Redis cache client implementation.\n4. [RedisHybridCacheClient](https://github.com/FoundatioFx/Foundatio.Redis/blob/master/src/Foundatio.Redis/Cache/RedisHybridCacheClient.cs): This cache implementation uses both the `RedisCacheClient` and `InMemoryCacheClient` implementations and uses the `RedisMessageBus` to keep the in memory cache in sync across processes. This can lead to **huge wins in performance** as you are saving a serialization operation and call to Redis if the item exists in the local cache.\n5. [ScopedCacheClient](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Caching/ScopedCacheClient.cs): This cache implementation takes an instance of `ICacheClient` and a string `scope`. The scope is prefixed onto every cache key. This makes it really easy to scope all cache keys and remove them with ease.\n\n#### Sample\n\n```csharp\nusing Foundatio.Caching;\n\nICacheClient cache = new InMemoryCacheClient();\nawait cache.SetAsync("test", 1);\nvar value = await cache.GetAsync<int>("test");\n```\n\n### [Queues](https://github.com/FoundatioFx/Foundatio/tree/master/src/Foundatio/Queues)\n\nQueues offer First In, First Out (FIFO) message delivery. We provide four different queue implementations that derive from the [`IQueue` interface](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Queues/IQueue.cs):\n\n1. [InMemoryQueue](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Queues/InMemoryQueue.cs): An in memory queue implementation. This queue implementation is only valid for the lifetime of the process.\n2. [RedisQueue](https://github.com/FoundatioFx/Foundatio.Redis/blob/master/src/Foundatio.Redis/Queues/RedisQueue.cs): An Redis queue implementation.\n3. [AzureServiceBusQueue](https://github.com/FoundatioFx/Foundatio.AzureServiceBus/blob/master/src/Foundatio.AzureServiceBus/Queues/AzureServiceBusQueue.cs): An Azure Service Bus Queue implementation.\n4. [AzureStorageQueue](https://github.com/FoundatioFx/Foundatio.AzureStorage/blob/master/src/Foundatio.AzureStorage/Queues/AzureStorageQueue.cs): An Azure Storage Queue implementation.\n\n#### Sample\n\n```csharp\nusing Foundatio.Queues;\n\nIQueue<SimpleWorkItem> queue = new InMemoryQueue<SimpleWorkItem>();\n\nawait queue.EnqueueAsync(new SimpleWorkItem {\n    Data = "Hello"\n});\n\nvar workItem = await queue.DequeueAsync();\n```\n\n### [Locks](https://github.com/FoundatioFx/Foundatio/tree/master/src/Foundatio/Lock)\n\nLocks ensure a resource is only accessed by one consumer at any given time. We provide two different locking implementations that derive from the [`ILockProvider` interface](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Lock/ILockProvider.cs):\n\n1. [CacheLockProvider](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Lock/CacheLockProvider.cs): A lock implementation that uses cache to communicate between processes.\n2. [ThrottlingLockProvider](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Lock/ThrottlingLockProvider.cs): A lock implementation that only allows a certain amount of locks through. You could use this to throttle api calls to some external service and it will throttle them across all processes asking for that lock.\n3. [ScopedLockProvider](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Lock/ScopedLockProvider.cs): This lock implementation takes an instance of `ILockProvider` and a string `scope`. The scope is prefixed onto every lock key. This makes it really easy to scope all locks and release them with ease.\n\nIt\'s worth noting that all lock providers take a `ICacheClient`. This allows you to ensure your code locks properly across machines.\n\n#### Sample\n\n```csharp\nusing Foundatio.Lock;\n\nILockProvider locker = new CacheLockProvider(new InMemoryCacheClient(), new InMemoryMessageBus());\nvar lock = await locker.AcquireAsync("test");\n// ...\nawait lock.ReleaseAsync();\n\nILockProvider throttledLocker = new ThrottlingLockProvider(new InMemoryCacheClient(), 1, TimeSpan.FromMinutes(1));\nvar throttledLock = await throttledLocker.AcquireAsync("test");\n// ...\nawait throttledLock.ReleaseAsync();\n```\n\n### [Messaging](https://github.com/FoundatioFx/Foundatio/tree/master/src/Foundatio/Messaging)\n\nAllows you to publish and subscribe to messages flowing through your application. We provide four different message bus implementations that derive from the [`IMessageBus` interface](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Messaging/IMessageBus.cs):\n\n1. [InMemoryMessageBus](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Messaging/InMemoryMessageBus.cs): An in memory message bus implementation. This message bus implementation is only valid for the lifetime of the process.\n2. [RedisMessageBus](https://github.com/FoundatioFx/Foundatio.Redis/blob/master/src/Foundatio.Redis/Messaging/RedisMessageBus.cs): A Redis message bus implementation.\n3. [RabbitMQMessageBus](https://github.com/FoundatioFx/Foundatio.RabbitMQ/blob/master/src/Foundatio.RabbitMQ/Messaging/RabbitMQMessageBus.cs): A RabbitMQ implementation.\n4. [AzureServiceBusMessageBus](https://github.com/FoundatioFx/Foundatio.AzureServiceBus/blob/master/src/Foundatio.AzureServiceBus/Messaging/AzureServiceBusMessageBus.cs): An Azure Service Bus implementation.\n\n#### Sample\n\n```csharp\nusing Foundatio.Messaging;\n\nIMessageBus messageBus = new InMemoryMessageBus();\nawait messageBus.SubscribeAsync<SimpleMessageA>(msg => {\n  // Got message\n});\n\nawait messageBus.PublishAsync(new SimpleMessageA { Data = "Hello" });\n```\n\n### [Jobs](https://github.com/FoundatioFx/Foundatio/tree/master/src/Foundatio/Jobs)\n\nAllows you to run a long running process (in process or out of process) without worrying about it being terminated prematurely. We provide three different ways of defining a job, based on your use case:\n\n1. **Jobs**: All jobs must derive from the [`IJob` interface](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Jobs/IJob.cs). We also have a [`JobBase` base class](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Jobs/JobBase.cs) you can derive from which provides a JobContext and logging. You can then run jobs by calling `RunAsync()` on the job or by creating a instance of the [`JobRunner` class](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Jobs/JobRunner.cs) and calling one of the Run methods. The JobRunner can be used to easily run your jobs as Azure Web Jobs.\n\n  #### Sample\n\n  ```csharp\n  using Foundatio.Jobs;\n\n  public class HelloWorldJob : JobBase {\n    public int RunCount { get; set; }\n\n    protected override Task<JobResult> RunInternalAsync(JobContext context) {\n       RunCount++;\n       return Task.FromResult(JobResult.Success);\n    }\n  }\n  ```\n\n  ```csharp\n  var job = new HelloWorldJob();\n  await job.RunAsync(); // job.RunCount = 1;\n  await job.RunContinuousAsync(iterationLimit: 2); // job.RunCount = 3;\n  await job.RunContinuousAsync(cancellationToken: new CancellationTokenSource(10).Token); // job.RunCount > 10;\n  ```\n\n2. **Queue Processor Jobs**: A queue processor job works great for working with jobs that will be driven from queued data. Queue Processor jobs must derive from [`QueueJobBase<T>` class](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Jobs/QueueJobBase.cs). You can then run jobs by calling `RunAsync()` on the job or passing it to the [`JobRunner` class](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Jobs/JobRunner.cs). The JobRunner can be used to easily run your jobs as Azure Web Jobs.\n\n  #### Sample\n\n  ```csharp\n  using Foundatio.Jobs;\n\n  public class HelloWorldQueueJob : QueueJobBase<HelloWorldQueueItem> {\n    public int RunCount { get; set; }\n\n    public HelloWorldQueueJob(IQueue<HelloWorldQueueItem> queue) : base(queue) {}\n\n    protected override Task<JobResult> ProcessQueueEntryAsync(QueueEntryContext<HelloWorldQueueItem> context) {\n       RunCount++;\n\n       return Task.FromResult(JobResult.Success);\n    }\n  }\n\n  public class HelloWorldQueueItem {\n    public string Message { get; set; }\n  }\n  ```\n\n  ```csharp\n   // Register the queue for HelloWorldQueueItem.\n  container.AddSingleton<IQueue<HelloWorldQueueItem>>(s => new InMemoryQueue<HelloWorldQueueItem>());\n\n  // To trigger the job we need to queue the HelloWorldWorkItem message.\n  // This assumes that we injected an instance of IQueue<HelloWorldWorkItem> queue\n\n  IJob job = new HelloWorldQueueJob();\n  await job.RunAsync(); // job.RunCount = 0; The RunCount wasn\'t incremented because we didn\'t enqueue any data.\n\n  await queue.EnqueueAsync(new HelloWorldWorkItem { Message = "Hello World" });\n  await job.RunAsync(); // job.RunCount = 1;\n\n  await queue.EnqueueAsync(new HelloWorldWorkItem { Message = "Hello World" });\n  await queue.EnqueueAsync(new HelloWorldWorkItem { Message = "Hello World" });\n  await job.RunUntilEmptyAsync(); // job.RunCount = 3;\n  ```\n\n3. **Work Item Jobs**: A work item job will run in a job pool among other work item jobs. This type of job works great for things that don\'t happen often but should be in a job (Example: Deleting an entity that has many children.). It will be triggered when you publish a message on the `message bus`. The job must derive from the [`WorkItemHandlerBase` class](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Jobs/WorkItemJob/WorkItemHandlerBase.cs). You can then run all shared jobs via [`JobRunner` class](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Jobs/JobRunner.cs). The JobRunner can be used to easily run your jobs as Azure Web Jobs.\n\n  #### Sample\n\n  ```csharp\n  using System.Threading.Tasks;\n  using Foundatio.Jobs;\n\n  public class HelloWorldWorkItemHandler : WorkItemHandlerBase {\n    public override async Task HandleItemAsync(WorkItemContext ctx) {\n      var workItem = ctx.GetData<HelloWorldWorkItem>();\n\n      // We can report the progress over the message bus easily.\n      // To receive these messages just inject IMessageSubscriber\n      // and Subscribe to messages of type WorkItemStatus\n      await ctx.ReportProgressAsync(0, "Starting Hello World Job");\n      await Task.Delay(TimeSpan.FromSeconds(2.5));\n      await ctx.ReportProgressAsync(50, "Reading value");\n      await Task.Delay(TimeSpan.FromSeconds(.5));\n      await ctx.ReportProgressAsync(70, "Reading value.");\n      await Task.Delay(TimeSpan.FromSeconds(.5));\n      await ctx.ReportProgressAsync(90, "Reading value..");\n      await Task.Delay(TimeSpan.FromSeconds(.5));\n\n      await ctx.ReportProgressAsync(100, workItem.Message);\n    }\n  }\n\n  public class HelloWorldWorkItem {\n    public string Message { get; set; }\n  }\n  ```\n\n  ```csharp\n  // Register the shared job.\n  var handlers = new WorkItemHandlers();\n  handlers.Register<HelloWorldWorkItem, HelloWorldWorkItemHandler>();\n\n  // Register the handlers with dependency injection.\n  container.AddSingleton(handlers);\n\n  // Register the queue for WorkItemData.\n  container.AddSingleton<IQueue<WorkItemData>>(s => new InMemoryQueue<WorkItemData>());\n\n  // The job runner will automatically look for and run all registered WorkItemHandlers.\n  new JobRunner(container.GetRequiredService<WorkItemJob>(), instanceCount: 2).RunInBackground();\n  ```\n\n  ```csharp\n   // To trigger the job we need to queue the HelloWorldWorkItem message.\n   // This assumes that we injected an instance of IQueue<WorkItemData> queue\n\n   // NOTE: You may have noticed that HelloWorldWorkItem doesn\'t derive from WorkItemData.\n   // Foundatio has an extension method that takes the model you post and serializes it to the\n   // WorkItemData.Data property.\n   await queue.EnqueueAsync(new HelloWorldWorkItem { Message = "Hello World" });\n  ```\n\n### [File Storage](https://github.com/FoundatioFx/Foundatio/tree/master/src/Foundatio/Storage)\n\nWe provide different file storage implementations that derive from the [`IFileStorage` interface](https://github.com/FoundatioFx/Foundatio/blob/master/src/Foundatio/Storage/IFileStorage.cs):\n\n1. [AzureFileStorage](https://github.com/FoundatioFx/Foundatio.AzureStorage/blob/master/src/Foundatio.AzureStorage/Storage/AzureFileStorage.cs): An Azure Blob storage implementation.\n4. [S3FileStorage](https://github.com/FoundatioFx/Foundatio.AWS/blob/master/src/Foundatio.AWS/Storage/S3FileStorage.cs): An AWS S3 file storage implementation.\n5. [RedisFileStorage](https://github.com/FoundatioFx/Foundatio.Redis/blob/master/src/Foundatio.Redis/Storage/RedisFileStorage.cs): An Redis file storage implementation.\n6. [MinioFileStorage](https://github.com/FoundatioFx/Foundatio.Minio/blob/master/src/Foundatio.Minio/Storage/MinioFileStorage.cs) An Minio file storage implementation.\n7. [AliyunFileStorage](https://github.com/FoundatioFx/Foundatio.Aliyun/blob/master/src/Foundatio.Aliyun/Storage/AliyunFileStorage.cs): An Aliyun file storage implementation.\n8.
