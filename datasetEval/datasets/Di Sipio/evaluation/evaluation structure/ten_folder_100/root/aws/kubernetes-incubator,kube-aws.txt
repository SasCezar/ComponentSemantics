b'# Kubernetes on AWS (kube-aws)\n\n[![Go Report Card](https://goreportcard.com/badge/github.com/kubernetes-incubator/kube-aws)](https://goreportcard.com/report/github.com/kubernetes-incubator/kube-aws)\n[![Build Status](https://travis-ci.org/kubernetes-incubator/kube-aws.svg?branch=master)](https://travis-ci.org/kubernetes-incubator/kube-aws)\n[![License](https://img.shields.io/badge/license-Apache%20License%202.0-blue.svg)](LICENSE)\n\n**Note**: The `master` branch may be in an *unstable or even broken state* during development. Please use [releases](https://github.com/kubernetes-incubator/kube-aws/releases) instead of the `master` branch in order to get stable binaries.\n\n`kube-aws` is a command-line tool to create/update/destroy Kubernetes clusters on AWS.\n\n## Features\n\n* Create, update and destroy Kubernetes clusters on AWS\n* Review changes before applying\n* Highly available and scalable Kubernetes clusters backed by multi-AZ deployment and Node Pools\n* Deployment to an existing VPC\n* Powered by various AWS services including CloudFormation, KMS, Auto Scaling, Spot Fleet, EC2, ELB, S3, etc.\n\n## Getting Started / Manual\n\n[View the latest manual for `kube-aws`](https://kubernetes-incubator.github.io/kube-aws/)\n\nCheck out our [getting started tutorial](https://kubernetes-incubator.github.io/kube-aws/getting-started/) \nto launch your first Kubernetes cluster on AWS.\n\n## Global options\n\nEach command supports following options:\n\n - `-s` `--silent` do not show messages\n - `-v` `--verbose` show debug messages\n - `--color` use color for messages\n\n## Examples\n\nGenerate `cluster.yaml`:\n\n```\n$ mkdir my-cluster\n$ cd my-cluster\n$ kube-aws init \\\n--cluster-name=my-cluster \\\n--region=us-west-1 \\\n--availability-zone=us-west-1c \\\n--hosted-zone-id=<my-hosted-zone> \\\n--external-dns-name=<my-cluster-endpoint> \\\n--key-name=<key-pair-name> \\\n--kms-key-arn="arn:aws:kms:us-west-1:xxxxxxxxxx:key/xxxxxxxxxxxxxxxxxxx" \\\n--s3-uri=s3://examplebucket/mydir\n```\n\nHere `us-west-1c` is used for parameter `--availability-zone`, but supported availability zone varies among AWS accounts.\nPlease check if `us-west-1c` is supported by `aws ec2 --region us-west-1 describe-availability-zones`, if not switch to other supported availability zone. (e.g., `us-west-1a`, or `us-west-1b`)\n\nGenerate assets:\n\n```\n$ kube-aws render credentials --generate-ca\n$ kube-aws render stack\n```\n\nView generated certificates:\n\n```\n$ kube-aws show certificates\n```\n\nValidate configuration:\n\n```\n$ kube-aws validate\n```\n\nLaunch:\n\n```\n$ kube-aws apply\n\n# Or export your cloudformation stack and dependent assets into the `exported/` directory\n$ kube-aws apply --export\n\n# Access the cluster\n$ KUBECONFIG=kubeconfig kubectl get nodes --show-labels\n```\n\nUpdate:\n\n```\n# Modify your cluster.yaml\n$ $EDITOR cluster.yaml\n\n# Reviews changes to cfn stacks and EC2 userdata\n$ kube-aws diff --context 3 --color\n\n# Update all the cfn stacks including the one for control-plane and the ones for worker node pools\n$ kube-aws apply\n```\n\nDestroy:\n\n```\n# Destroy all the cfn stacks including the one for control-plane and the ones for worker node pools. Use `--force` for skip confirmation. \n$ kube-aws destroy\n```\n\n## Other Resources\n\nExtra or advanced topics in for kube-aws:\n\n* [Known Limitations](/docs/troubleshooting/known-limitations.md)\n* [Roadmap](/ROADMAP.md)\n\nThe following links can be useful for development:\n\n- [AWS CloudFormation resource types](http://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/aws-template-resource-type-ref.html)\n\nPlease feel free to reach out to the kube-aws community on: [#kube-aws in the kubernetes slack](https://kubernetes.slack.com/messages/C5GP8LPEC/)\n\n## Kubernetes Incubator\n\nThis is a [Kubernetes Incubator project](https://github.com/kubernetes/community/blob/master/incubator.md).
