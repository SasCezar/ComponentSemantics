b'<p align="center"><a href="https://lovr.org"><img src="https://lovr.org/static/img/README.png" width="256"/></a></p>\n\n# L\xc3\x96VR\n\n> **A simple Lua framework for rapidly building VR experiences.**\n\nYou can use L\xc3\x96VR to easily create VR experiences without much setup or programming experience.  The framework is tiny, fast, open source, and supports lots of different platforms and devices.\n\n[![Build status](https://ci.appveyor.com/api/projects/status/alx3kdi35bmxka8c/branch/master?svg=true)](https://ci.appveyor.com/project/bjornbytes/lovr/branch/master)\n[![Version](https://img.shields.io/github/release/bjornbytes/lovr.svg?label=version)](https://github.com/bjornbytes/lovr/releases)\n[![Slack](https://img.shields.io/badge/chat-slack-7e4e76.svg)](https://lovr.org/slack)\n\n[**Homepage**](https://lovr.org) | [**Documentation**](https://lovr.org/docs) | [**FAQ**](https://lovr.org/docs/FAQ)\n\n<p align="left">\n  <span><img src="http://lovr.org/static/img/wattle.jpg" width="32%"/></span>\n  <span><img src="http://lovr.org/static/img/levrage.jpg" width="32%"/></span>\n  <span><img src="http://lovr.org/static/img/planets.jpg" width="32%"/></span>\n</p>\n\nFeatures\n---\n\n- **Cross-Platform** - Runs on Windows, Mac, Linux, Android, and even on the web using WebAssembly and WebVR.\n- **Cross-Device** - Supports HTC Vive, Oculus Touch/Go/Quest, Windows MR, and includes a keyboard/mouse VR simulator.\n- **Beginner-friendly** - Simple VR scenes can be created in just a few lines of Lua.\n- **Fast** - Writen in C99 and scripted with LuaJIT, includes optimized single-pass stereo rendering.\n- **Asset Import** - Supports 3D models (glTF, OBJ), skeletal animation, HDR textures, cubemaps, fonts, etc.\n- **Spatialized Audio** - Audio is automatically spatialized using HRTFs.\n- **Vector Library** - Efficient first-class support for 3D vectors, quaternions, and matrices.\n- **3D Rigid Body Physics** - Including 4 collider shapes and 4 joint types.\n- **Compute Shaders** - For high performance GPU tasks, like particles.\n- **Multiplayer** - Includes enet for multi-user VR experiences.\n\nGetting Started\n---\n\nIt\'s really easy to get started making things with L\xc3\x96VR.  Grab a copy of the executable from <https://lovr.org/download>,\nthen write a `main.lua` script and drag its parent folder onto the executable.  Here are some example projects to try:\n\n#### Hello World\n\n```lua\nfunction lovr.draw()\n  lovr.graphics.print(\'Hello World!\', 0, 1.7, -3, .5)\nend\n```\n\n#### Spinning Cube\n\n```lua\nfunction lovr.draw()\n  lovr.graphics.cube(\'line\', 0, 1.7, -1, .5, lovr.timer.getTime())\nend\n```\n\n#### Hand Tracking\n\n```lua\nfunction lovr.draw()\n  for _, hand in ipairs(lovr.headset.getHands()) do\n    lovr.graphics.sphere(vec3(lovr.headset.getPosition(hand)), .1)\n  end\nend\n```\n\n#### 3D Models\n\n```lua\nfunction lovr.load()\n  model = lovr.graphics.newModel(\'model.gltf\')\nend\n\nfunction lovr.draw()\n  model:draw(x, y, z)\nend\n```\n\nYou can try more examples in your browser on the [docs page](https://lovr.org/docs/Hello_World).\n\nBuilding\n---\n\nYou can build L\xc3\x96VR from source using CMake.  Here are the steps using the command line:\n\n```console\nmkdir build\ncd build\ncmake ..\ncmake --build .\n```\n\nSee the [Compiling Guide](https://lovr.org/docs/Compiling) for more info.\n\nResources\n---\n\n- [**Documentation**](https://lovr.org/docs): Guides, tutorials, examples, and API documentation.\n- [**FAQ**](https://lovr.org/docs/FAQ): Frequently Asked Questions.\n- [**Slack Group**](https://lovr.org/slack): For general L\xc3\x96VR discussion and support.\n- [**matrix.org**](https://matrix.to/#/!XVAslexgYDYQnYnZBP:matrix.org): Decentralized alternative to Slack.\n- [**Nightly Builds**](https://lovr.org/download/nightly): Nightly builds for Windows.\n- [**Compiling Guide**](https://lovr.org/docs/Compiling): Information on compiling L\xc3\x96VR from source.\n- [**Contributing**](CONTRIBUTING.md): Guide for helping out with development \xf0\x9f\x92\x9c\n- [**L\xc3\x96VE**](https://love2d.org): L\xc3\x96VR is heavily inspired by L\xc3\x96VE, a 2D game framework.\n\nContributors\n---\n\n- [@bjornbytes](https://github.com/bjornbytes)\n- [@shakesoda](https://github.com/shakesoda)\n- [@bcampbell](https://github.com/bcampbell)\n- [@mcclure](https://github.com/mcclure)\n\nLicense\n---\n\nMIT, see [`LICENSE`](LICENSE) for details.\n'
