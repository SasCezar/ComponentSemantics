[Finished Games](#finished-games)\n\n***\n\n## Installation <a name="installation"/>\n`go get -u github.com/oakmound/oak/...`\n\n\n## Motivation <a name="motivation"/>\nThe initial version of oak was made to support Oakmound Studio\'s game,\n[Agent Blue](https://github.com/OakmoundStudio/AgentRelease), and was developed in parallel.\nOak supports Windows with no dependencies and Linux with limited audio dependencies. We don\'t own a machine to check with, but hypothetically it supports OSX as well.\n We hope that users will be able to make great pure Go games with oak and welcome improvements.\n \n Because Oak wants to have as few non-Go dependencies as possible, Oak does not use OpenGL or [GLFW](https://github.com/go-gl/glfw).\n We\'re open to adding support for these in the future for performance gains, but we always want\n an alternative that requires zero or near-zero dependencies.\n\n \n## Features <a name="features"></a>\n1. Window Rendering\n    - Windows and key events forked from [shiny](https://github.com/oakmound/shiny)\n    - Logical frame rate distinct from Draw rate\n    - Fullscreen, Window Positioning support\n    - Auto-scaling for screen size changes\n1. [Image Management](https://godoc.org/github.com/oakmound/oak/render)\n    - `render.Renderable` interface\n    - Sprite Sheet Batch Loading at startup\n    - Manipulation\n        - `render.Modifiable` interface\n        - Built in Transformations and Filters\n        - Some built-ins via [gift](https://github.com/disintegration/gift)\n        - Extensible Modification syntax `func(image.Image) *image.RGBA`\n    - Built in `Renderable` types covering common use cases\n        - `Sprite`, `Sequence`, `Switch`, `Composite`\n        - Primitive builders, `ColorBox`, `Line`, `Bezier`\n        - History-tracking `Reverting`\n    - Primarily 2D\n1. [Particle System](https://godoc.org/github.com/oakmound/oak/render/particle) \n    - <details>\n      <summary>Click to see gif captured in examples/particle-demo</summary>\n      \n        ![particles!](examples\\particle-demo\\overviewExample.gif)\n    </details>\n1. [Mouse Handling](https://godoc.org/github.com/oakmound/oak/mouse)\n    - Click Collision\n    - MouseEnter / MouseExit reaction events\n    - Drag Handling\n1. [Joystick Support](https://godoc.org/github.com/oakmound/oak/joystick)\n    - <details>\n      <summary>Click to see gif captured in examples/joystick-viz</summary>\n      \n        ![particles!](examples\\joystick-viz\\example.gif)\n    </details>\n1. [Audio Support](https://godoc.org/github.com/oakmound/oak/audio)\n    - Positional filters to pan and scale audio based on a listening position\n1. [Collision](https://godoc.org/github.com/oakmound/oak/collision)\n    - Collision R-Tree forked from [rtreego](https://github.com/dhconnelly/rtreego)\n    - [2D Raycasting](https://godoc.org/github.com/oakmound/oak/collision/ray)\n    - Collision Spaces\n        - Attachable to Objects\n        - Auto React to collisions through events\n        - OnHit bindings `func(s1,s2 *collision.Space)`\n        - Start/Stop collision with targeted objects\n1. [2D Physics System](https://godoc.org/github.com/oakmound/oak/physics)\n    - Vectors\n        - Attachable to Objects / Renderables\n        - Friction\n1. [Event Handler, Bus](https://godoc.org/github.com/oakmound/oak/event)\n    - PubSub system: `event.CID` can `Bind(fn,eventName)` and `Trigger(eventName)` events\n1. [Shaping](https://godoc.org/github.com/oakmound/oak/shape)\n    - Convert shapes into: \n        - Containment checks\n        - Outlines\n        - 2D arrays\n1. [Custom Console Commands](debugConsole.go)\n1. [Logging](https://godoc.org/github.com/oakmound/oak/dlog)\n    - Swappable with custom implementations\n    - Default Implementation: 4 log levels, writes to file and stdout\n\n## Support <a name="support"></a>\n\nFor discussions not significant enough to be an Issue or PR, see the #oak channel on the [gophers slack](https://invite.slack.golangbridge.org/). \n\n## Quick Start <a name="quick-start"></a>\nThis is an example of the most basic oak program:\n\n```go\n\noak.Add("firstScene",\n    // Initialization function\n    func(prevScene string, inData interface{}) {}, \n    // Loop to continue or stop the current scene\n    func() bool {return true}, \n    // Exit to transition to the next scene\n    func() (nextScene string, result *scene.Result) {return "firstScene", nil}) \noak.Init("firstScene")\n```\n\nSee the [examples](examples) folder for longer demos, [godoc](https://godoc.org/github.com/oakmound/oak) for reference documentation, and the [wiki](https://github.com/oakmound/oak/wiki) for more guided feature sets, tutorials and walkthroughs.\n\n## Implementation and Examples <a name="examples"></a>\n\n### Platformer\n\n![Platformer](examples/platformer-tutorial/6-complete/example.gif)\n\n\nBuild up to having a simple platforming game by doing the following: Setup a character, get it to move, set basic gravity, get it to jump, make it only jump on solid ground, put it all together.\n\n\n```go\n\nchar := entities.NewMoving(100, 100, 16, 32,\n\trender.NewColorBox(16, 32, color.RGBA{255, 0, 0, 255}),\nnil, 0, 0)\n```\n\n\n```go \n\nchar.Bind(func(id int, nothing interface{}) int {\n\tchar := event.GetEntity(id).(*entities.Moving)\n\n\t// Move left and right with A and D\n\tif oak.IsDown(key.A) {\n\t\tchar.Delta.SetX(-char.Speed.X())\n\t} else if oak.IsDown(key.D) {\n\t\tchar.Delta.SetX(char.Speed.X())\n\t} else {\n\t\tchar.Delta.SetX(0)\n\t}\n\toldX, oldY := char.GetPos()\n    char.ShiftPos(char.Delta.X(), char.Delta.Y())\n\treturn 0\n}, event.Enter)\n```\n\n\n### Top Down Shooter\n\nLearn to use the collision library and make short term shots that
