More info in the [wiki](https://github.com/rundeck/rundeck/wiki/Building-and-Testing).\n\nGradle Build\n=====\n\n    ./gradlew build\n\nArtifacts: \n\n* `rundeckapp/build/libs/rundeck-X.Y.war`\n\n\nOther builds\n======\n\nThe documentation can be built with [pandoc](http://johnmacfarlane.net/pandoc/).\n    \nBuild the documentation. Artifacts in `docs/en/dist`:\n\n    cd docs\n    make\n\nYou can build .rpm or .deb files (requires pandoc to build the docs):\n\nBuild the RPM. Artifacts in `packaging/rpmdist/RPMS/noarch/*.rpm`\n\n    make rpm\n    \nBuild the .deb. Artifacts in `packaging/*.deb`:\n\n    make deb\n\nTo build clean:\n\n    make clean\n\nInstallation\n======\n\nThere are several install options: a self-contained war file, or RPM, or Debian.\n\nTo start from the rundeck-X.Y.war, put it in a directory named ~/rundeck, then execute:\n\n    java -Xmx1024m -Xms256m -XX:MaxMetaspaceSize=256m -server -jar rundeck-X.Y.war\n\nIf you\'d like to install via RPM, you can use Yum:\n\n    rpm -Uvh http://repo.rundeck.org/latest.rpm \n    yum install rundeck\n\nOR install directly from RPM:\n\n    rpm -ivh rundeck-2.0.0-xxx.rpm rundeck-config-2.0.0-xxx.rpm\n\nOnce the RPM is installed, execute:\n\n    sudo /etc/init.d/rundeckd start\n\nThe server should launch on port 4440, with default username/password of `admin/admin`.\n\nFor Debian, download the .deb from the [downloads page](http://rundeck.org/downloads.html), then run:\n\n    dpkg -i rundeck-2.0.0-x.deb\n\n* For more info and configuration information, see the [Rundeck docs](http://docs.rundeck.org).\n\nRequirements\n=======\n\nJava 8 (openjdk, oracle)\n\n[Pandoc](http://johnmacfarlane.net/pandoc/) (documentation build only)\n\nDocumentation\n======\n\nAvailable online at <http://rundeck.org/docs>\n\nFAQ: <https://github.com/rundeck/rundeck/wiki/FAQ>\n\nDevelopment\n======\n\nRefer to the [IDE Development Environment](https://github.com/rundeck/rundeck/wiki/IDE-Development-Environment) to get set up using IntelliJ IDEA or Eclipse/STS.\n\n* [Issue tracker](https://github.com/rundeck/rundeck/issues) at github.com\n* [Fresh builds](http://build.rundeck.org) served by Jenkins\n\nDo you have changes to contribute?
