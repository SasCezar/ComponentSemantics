b'# [Worldview](https://worldview.earthdata.nasa.gov)\n\n[![Worldview Screenshot](/web/images/preview.gif)](https://worldview.earthdata.nasa.gov)\n\n[![Build Status](https://api.travis-ci.org/nasa-gibs/worldview.svg?branch=master)](https://travis-ci.org/nasa-gibs/worldview)\n\nInteractive interface for browsing full-resolution, global satellite imagery\n\n## Background\n\nThis app from NASA\'s [EOSDIS](https://earthdata.nasa.gov/) provides the\ncapability to interactively browse over 900 global, full-resolution satellite\nimagery layers on desktop and mobile devices. Many of the imagery layers are\nupdated daily and are within three hours of observation - showing the entire Earth as it is\n"right now". This supports time-critical applications such as wildfire\nmanagement, air quality measurements, and flood monitoring. Some satellite\nimagery layers span almost 30 years, providing a long term view of our dynamic\nplanet. The underlying data is available for download, and Arctic and Antarctic\nviews of several imagery layers are available for a \xe2\x80\x9cfull globe\xe2\x80\x9d perspective.\n\nWorldview uses [OpenLayers](http://openlayers.org/) to display imagery from the\n[Global Imagery Browse Services (GIBS)](https://earthdata.nasa.gov/gibs). This\nimagery can also be used [with libraries such as Leaflet, Cesium, Google Maps](https://wiki.earthdata.nasa.gov/display/GIBS/Map+Library+Usage)\nor [custom GDAL scripts](https://wiki.earthdata.nasa.gov/display/GIBS/Map+Library+Usage#expand-GDALBasics).\nWe encourage interested developers to fork Worldview or build their own clients\nusing GIBS services.\n\nCheck out our [roadmap](https://github.com/nasa-gibs/worldview/projects/7)\nto see what we\'re working on and follow our [blog](https://wiki.earthdata.nasa.gov/pages/viewrecentblogposts.action?key=GIBS)\nto find out the latest features and imagery available.\n\n## Install\n\nThis project uses Node and Python. See the [dependencies](#dependencies) section for more information.\n\n```bash\ngit clone https://github.com/nasa-gibs/worldview.git\ncd worldview\nnpm install\n```\n\nView the [Configuration](doc/configuration.md) section for information on how to install the official EOSDIS Worldview configuration, or to add your own custom configuration.\n\n### Dependencies\n\nThe following are required to install and run Worldview:\n\n- [Node v8.15.1 or Later](https://nodejs.org/en/download/)\n  - **Note:** Ubuntu users may run into issues with the `node` command not being available. See [this question on StackOverflow](https://stackoverflow.com/q/18130164/417629) for possible solutions.\n- [Python v3.7.1 or Later](https://www.python.org/)\n\n  - **Note:** It is recommended that Windows users install Python with [Windows Build Tools](https://www.npmjs.com/package/windows-build-tools).\n  - macOS users should use homebrew to install **Python v3**\n    - `brew install python`\n\nWindows users will also need the following:\n\n- [Git Bash](https://git-scm.com/downloads), or [MinGW-w64](https://sourceforge.net/projects/mingw-w64/files/External%20binary%20packages%20%28Win64%20hosted%29/MSYS%20%2832-bit%29/)\n- [.NET Framework](https://www.microsoft.com/net/download/dotnet-framework-runtime) or [Visual Studio Team Services](https://www.visualstudio.com/)\n- [Add `python` to the path](https://docs.python.org/3/using/windows.html#installation-steps)\n\n## Usage\n\n```bash\nnpm run build\nnpm start\n```\n\nNavigate to [`http://localhost:3000`](http://localhost:3000) in a browser. To stop Worldview, press Control+C in the terminal.\n\nSee [Developing](doc/developing.md) for more usage details.\n\n## Updates\n\nTo update Worldview, pull down any branch or tag from GitHub. From the `master` branch (default), to update to the latest stable version of Worldview, run `git pull`.\n\n**Note:** This project uses [Semantic Versioning](https://semver.org/). Updates to the major version number in [package.json](package.json) indicate a breaking change; _update with caution_.\n\n## Other Information\n\n- [Configuration](doc/configuration.md)\n- [Custom Branding](doc/branding.md)\n- [Optional Features](doc/features.md)\n- [Data Download](doc/data_download.md)\n- [URL Parameters](doc/url_parameters.md)\n- [Developing](doc/developing.md)\n- [Deployment](doc/deployment.md)\n- [Testing](doc/testing.md)\n- [Uploading](doc/upload.md)\n- [Docker](doc/docker.md)\n\n## Contact\n\nContact us via GitHub or by sending an email to\n[support@earthdata.nasa.gov](mailto:support@earthdata.nasa.gov).\n\n## Contribute\n\nWe welcome your contributions!
