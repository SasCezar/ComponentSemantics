Dependency\n\n|[D3](https://d3js.org/) (required)|\n| --- |\n| 4+ |\n\nLoad billboard.js after D3.js.\n\n```html\n<!-- 1) Load D3.js and billboard.js separately -->\n    <!-- Load D3 -->\n    <script src="https://d3js.org/d3.v5.min.js"></script>\n\n    <!-- Load billboard.js with base(or theme) style -->\n    <link rel="stylesheet" href="$YOUR_PATH/billboard.css">\n    <script src="$YOUR_PATH/billboard.js"></script>\n\n<!-- 2) or Load billboard.js packaged with D3.js -->\n    <link rel="stylesheet" href="$YOUR_PATH/billboard.css">\n    <script src="$YOUR_PATH/billboard.pkgd.js"></script>\n```\n\nor use importing ESM.\n\n```js\n// 1) import billboard.js\n// as named import\nimport {bb} from "billboard.js";\n\n// or as importing default\nimport bb from "billboard.js";\n\n// 2) import css if your dev-env supports. If don\'t, include them via <link>\nimport "billboard.js/dist/billboard.css";\n\n// or theme style. Find more themes from \'theme\' folder\nimport "billboard.js/dist/theme/insight.css"\n```\n\n> **Note**\n> For migration from C3.js, checkout the [migration guide](https://github.com/naver/billboard.js/wiki/How-to-migrate-from-C3.js%3F)\n\n## Basic usage example\n\n#### 1) Create chart holder element\n```html\n<div id="chart"></div>\n```\n\n#### 2) Generate a chart with options\n```js\n// generate the chart\nvar chart = bb.generate({\n    bindto: "#chart",\n    data: {\n      type: "line",\n        columns: [\n            ["data1", 30, 200, 100, 400, 150, 250]\n        ]\n    }\n});\n\n// call some API\nchart.load( ... );\n```\n\n## How to start developing billboard.js?\n\nFor anyone interested in developing billboard.js, follow the instructions below.\n> Required Node.js version: `10.10.0+`\n\n### Development Environment\n\n#### 1. Clone the repository\n\nClone the billboard.js repository and install the dependency modules.\n\n```bash\n# Create a folder and move.\n$ mkdir billboard.js && cd billboard.js\n\n# Clone the repository.\n$ git clone https://github.com/naver/billboard.js.git\n```\n\n#### 2. Install dependencies\n`npm` and `Yarn` are supported.\n\n```\n# Install the dependency modules.\n$ npm install\n\n# or\n$ yarn\n```\n\n#### 3. Build\n\nUse npm script to build billboard.js\n\n```bash\n# Run webpack-dev-server for development\n$ npm start\n\n# Build\n$ npm run build\n\n# Generate jsdoc\n$ npm run jsdoc\n```\n\nTwo folders will be created after the build is completed.\n\n- **dist** folder: Includes the **billboard.js** and **billboard.min.js** files.\n- **doc** folder: Includes API documentation. The home page for the documentation is **doc/index.html**.\n\n### Linting\n\nTo maintain the same code style and quality, we adopted [ESLint](https://eslint.org/). The [rules](https://github.com/naver/eslint-config-naver/tree/master/rules) are based on the [Airbnb JavaScript Style Guide](https://github.com/airbnb/javascript) with some modifications.\nSetup your editor for check or run the below command for linting.\n\n```bash\n$ npm run lint\n```\n\n### Test\n\nOnce you created a branch and finished the development, you must perform a test with `npm test` command before the push to a remote repository.\n\n```bash\n$ npm test\n```\nRunning the `npm test` command will start [Mocha](https://mochajs.org/) tests via [Karma-runner](https://karma-runner.github.io/).\n\n## Bug Report\n\nIf you find a bug, please report to us by posting [issues](https://github.com/naver/billboard.js/issues) on GitHub.\n\n## License\nbillboard.js is released under the [MIT license](https://github.com/naver/billboard.js/blob/master/LICENSE).\n\n```\nCopyright (c) 2017 ~ present NAVER Corp.\n\nPermission is hereby granted, free of charge, to any person obtaining a copy\nof this software and associated documentation files (the "Software"), to deal\nin the Software without restriction, including without limitation the rights\nto use, copy, modify, merge, publish, distribute, sublicense, and/or sell\ncopies of the Software, and to permit persons to whom the Software is\nfurnished to do so, subject to the following conditions:\n\nThe above copyright notice and this permission notice shall be included in\nall copies or substantial portions of the Software.\n\nTHE SOFTWARE IS PROVIDED "AS IS", WITHOUT WARRANTY OF ANY KIND, EXPRESS OR\nIMPLIED, INCLUDING BUT NOT LIMITED TO THE WARRANTIES OF MERCHANTABILITY,\nFITNESS FOR A PARTICULAR PURPOSE AND NONINFRINGEMENT.
