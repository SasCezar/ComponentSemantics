Unity native iOS/Android console for ease of testing and debugging.  \n\n- Check [Official Wiki](https://github.com/SpaceMadness/lunar-unity-console/wiki) for additional information, troubleshooting and guides.\n- Visit [Unity Forums Thread](http://forum.unity3d.com/threads/lunar-mobile-console-high-performance-unity-ios-android-logger-built-with-native-platform-ui.347650/) for discussions and general questions.\n\n### Platform Support\n- **iOS**: requires iOS 8 or later\n- **Android**: requires API Level 14 or later\n  \n### Key Benefits\n- Native C/Objective-C/Java code with a low memory footprint.\n- Works well with huge logs amount (up to 65536 entries).\n- Completely built with native iOS/Android UI (does NOT rely on Unity GUI).\n- Resolution independent (looks great on highres/retina displays).\n- Does NOT modify scenes and does NOT add new assets.\n- Removes completely from the release build with a single mouse click or from command line (absolutely NO traces left).\n\n### Features\n- Instant error notification (never miss an unhandled exception again):  \n<img Installation\n- **Automatic**:  \n  Unity Editor Menu:  Window \xe2\x96\xb6 Lunar Mobile Console \xe2\x96\xb6 Install...\n  \n- **Manual**:  \n  Drag\'n\'Drop `LunarConsol.prefab` (Assets/LunarConsole/Scripts/LunarConsole.prefab) into your current scene hierarchy and save your changes. You only need to do it once for your startup scene.\n\n## Usage\nYou can open the console with a multi touch gesture or using the API call from a script (see [API Guide](https://github.com/SpaceMadness/lunar-unity-console/wiki/API-Guide) for details).\n\n### Configuration\n- Select `LunarConsole` game object in the `Hierarchy` window.  \n  <img src="https://cloud.githubusercontent.com/assets/786644/18031101/ada34058-6c85-11e6-947b-2f85d657a8ea.png" width=244/>\n- Find `Lunar Console` script settings in the `Inspector` window.  \n  <img src="https://cloud.githubusercontent.com/assets/786644/18031112/2eb5120c-6c86-11e6-9e32-7b947897797d.png" width=276/>  \n  - Set the capacity (the maximum number of lines the console can hold). It\'s advised to keep this amount low (the more you have - the more memory it will take).\n  - Set the trim amount (how many lines will be removed from the beginning of the log when console overflows).\n  - Choose a gesture from the drop down list or select `None` to completely disable multi touch gestures (you would still be able to open the console from your scripts).\n  - Check "Remove Rich Text Tags" to remove rich text tags from the output (may cause performance overhead).\n\n### User-Defined Actions and Variables (only available in PRO version)\nFor more information on Actions and Variables check the user [guide](https://github.com/SpaceMadness/lunar-unity-console/wiki/Actions-and-Variables).\n\n### Stack Trace Frames\nTouch the log entry to view its stack trace.\n\n*Important*: make sure to properly configure your build settings (File \xe2\x96\xb6 Build Settings...) otherwise you will only be able to see exceptions traces.\n\n- iOS: check the "Development Build" checkbox  \n<img src="https://cloud.githubusercontent.com/assets/786644/18031076/81a757e2-6c84-11e6-870b-49fbf67473dd.png" width=630/>  \n\n- Android: check the "Development Build" checkbox  \n<img src="https://cloud.githubusercontent.com/assets/786644/18031075/81a3e36e-6c84-11e6-8484-627441e7fb91.png" width=630/>  \n\nFor more info see:\nhttp://docs.unity3d.com/Manual/PublishingBuilds.html\n\n### Enabling/Disabling Plugin\n\nIf Lunar Mobile Console is\n- _enabled_ - the plugin files would appear in your iOS/Android build: you can access the console with a multi touch gesture or from a script.\n- _disabled_ - the plugin files would **NOT** appear in your iOS/Android build: you can\'t access the console (multi touch gestures or API calls would be ignored).\n\nTo\n- disable:  \n  Window \xe2\x96\xb6 Lunar Mobile Console \xe2\x96\xb6 Disable\n- enable:  \n  Window \xe2\x96\xb6 Lunar Mobile Console \xe2\x96\xb6 Enable\n\n**Important:** make sure to "replace" your generated Xcode project whenever you switch the plugin from enabled to disabled (or back). Failing to do so may lead to an unexpected result!  \n\n<img width="404" src="https://cloud.githubusercontent.com/assets/786644/18031143/985708a4-6c87-11e6-995a-69073b6eff6b.png">\n\nFor more detailed information check Wiki page: [Enabling and Disabling Lunar Mobile Console](https://github.com/SpaceMadness/lunar-unity-console/wiki/Enabling-and-Disabling-Lunar-Mobile-Console).\n\n### Unity Cloud Build Support\nLunar Mobile Console is fully compatible with Unity Cloud Build.  \n- For more information check the user [guide](https://github.com/SpaceMadness/lunar-unity-console/wiki/Unity-Cloud-Build-Support)\n- For Cloud Build related issues check the troubleshooting [guide](https://github.com/SpaceMadness/lunar-unity-console/wiki/Troubleshooting#unity-cloud-build)\n\n### Build System Support\nYou can enable/disable the plugin from the command line (and make it a part of your build process)\n\n- To disable:  \n  ```<UNITY_BIN_PATH>  -quit -batchmode  -executeMethod LunarConsoleEditorInternal.Installer.DisablePlugin```\n- To enable:  \n  ```<UNITY_BIN_PATH>  -quit -batchmode  -executeMethod LunarConsoleEditorInternal.Installer.EnablePlugin```\n\n```<UNITY_BIN_PATH>``` locations:\n- Mac OS X: ```/Applications/Unity/Unity.app/Contents/MacOS/Unity```\n- Windows: ```c:\\Program Files\\Unity\\Editor\\Unity.exe```\n- Linux: _TBD_  \n**Note:** if you\'re using Lunar Mobile Console older than v0.8.0 (v0.0.1b-v0.7.1) you would need to use ```LunarConsolePluginInternal.Installer.DisablePlugin``` and ```LunarConsolePluginInternal.Installer.EnablePlugin``` instead.\n \n## Troubleshooting\nPlease, visit the
