Architecture\n\n![demo](https://raw.githubusercontent.com/youngsoft/MyLinearLayout/master/MyLayout/MyLayoutClass.png)\n\n### MyLayoutPos\n`MyLayoutPos` is represent to the position of a view. UIView provides six extension variables:leftPos, topPos, bottomPos, rightPos, centerXPos, centerYPos to set view\'s margin or space distance between self and others. You can use `equalTo` method to set NSNumber or MyLayoutPos or NSArray<MyLayoutPos*> value. there are six simple variables:myLeft,myTop,myBottom,myRight,myCenterX,myCenterY use to set NSNumber value. eg. `A.leftPos.equalTo(@10); <==> A.myLeft = 10;`\n\n\n### MyLayoutSize\n`MyLayoutSize` is represent to the size of a view. UIView provides two extension variables:widthSize,heightSize to set view\'s width and height dimension. You can use `equalTo` method to set NSNumber or MyLayoutSize or NSArray<MyLayoutSize*> value. there are two simple variables: myWidth, myHeight use to set NSNumber value. eg. `A.widthSize.equalTo(@10); <==> A.myWidth = 10;`\n\n\n### MyLinearLayout\n> Is equivalent to: UIStackView of iOS and LinearLayout of Android.\n\nLinear layout is a single line layout view that the subviews are arranged in sequence according to the added order\xef\xbc\x88from top to bottom or from left to right). So the subviews\' origin&size constraints are established by the added order. Subviews arranged in top-to-bottom order is called vertical linear layout view, and \nthe subviews arranged in left-to-right order is called horizontal linear layout.\n\n![\xe6\xbc\x94\xe7\xa4\xba\xe6\x95\x88\xe6\x9e\x9c\xe5\x9b\xbe](https://raw.githubusercontent.com/youngsoft/MyLinearLayout/master/MyLayout/ll.png)\n\nSample code:\n\n```objective-c\n\n-(void)loadView\n{\n    [super loadView];\n    \n    MyLinearLayout *S = [MyLinearLayout linearLayoutWithOrientation:MyOrientation_Vert];\n    S.myWidth = 120;\n    S.subviewSpace = 10;\n    \n    UIView *A = [UIView new];\n    A.myLeft = A.myRight = 5;\n    A.myHeight = 40;\n    [S addSubview:A];\n    \n    UIView *B = [UIView new];\n    B.myLeft = 20;\n    B.myWidth = B.myHeight = 40;\n    [S addSubview:B];\n    \n    UIView *C = [UIView new];\n    C.myRight = 40;\n    C.myWidth = 50;\n    C.myHeight = 40;\n    [S addSubview:C];\n    \n    UIView *D = [UIView new];\n    D.myLeft = D.myRight = 10;\n    D.myHeight = 40;\n    [S addSubview:D];\n    \n    [self.view addSubview:S];\n    S.backgroundColor = [UIColor redColor];\n    A.backgroundColor = [UIColor greenColor];\n    B.backgroundColor = [UIColor blueColor];\n    C.backgroundColor = [UIColor orangeColor];\n    D.backgroundColor = [UIColor cyanColor];\n }\n\n```\n\n\n### MyRelativeLayout\n> Is equivalent to: AutoLayout of iOS and RelativeLayout of Android.\n\nRelative layout is a layout view that the subviews layout and position through mutual constraints.The subviews in the relative layout are not depended to the adding order but layout and position by setting the subviews\' constraints.\n\n![\xe6\xbc\x94\xe7\xa4\xba\xe6\x95\x88\xe6\x9e\x9c\xe5\x9b\xbe](https://raw.githubusercontent.com/youngsoft/MyLinearLayout/master/MyLayout/rl.png)\n\nSample code:\n\n```objective-c\n\n-(void)loadView\n{\n    [super loadView];\n    \n    MyRelativeLayout *S = [MyRelativeLayout new];\n    S.widthSize.equalTo(@170);\n    S.heightSize.equalTo(@280);\n    \n    UIView *A = [UIView new];\n    A.leftPos.equalTo(@20);\n    A.topPos.equalTo(@20);\n    A.widthSize.equalTo(@40);\n    A.heightSize.equalTo(A.widthSize);\n    [S addSubview:A];\n    \n    UIView *B = [UIView new];\n    B.leftPos.equalTo(A.centerXPos);\n    B.topPos.equalTo(A.bottomPos).offset(10);\n    B.widthSize.equalTo(@60);\n    B.heightSize.equalTo(A.heightSize);\n    [S addSubview:B];\n    \n    UIView *C = [UIView new];\n    C.leftPos.equalTo(B.rightPos).offset(10);\n    C.bottomPos.equalTo(B.bottomPos);\n    C.widthSize.equalTo(@40);\n    C.heightSize.equalTo(B.heightSize).multiply(0.5);\n    [S addSubview:C];\n    \n    UIView *D = [UIView new];\n    D.bottomPos.equalTo(C.topPos).offset(10);\n    D.rightPos.equalTo(@15);\n    D.heightSize.equalTo(A.heightSize);\n    D.widthSize.equalTo(D.heightSize);\n    [S addSubview:D];\n    \n    UIView *E = [UIView new];\n    E.centerYPos.equalTo(@0);\n    E.centerXPos.equalTo(@0);\n    E.heightSize.equalTo(@40);\n    E.widthSize.equalTo(S.widthSize).add(-20);\n    [S addSubview:E];\n    //.. F, G\n    \n    [self.view addSubview:S];\n    S.backgroundColor = [UIColor redColor];\n    A.backgroundColor = [UIColor greenColor];\n    B.backgroundColor = [UIColor blueColor];\n    C.backgroundColor = [UIColor orangeColor];\n    D.backgroundColor = [UIColor cyanColor];\n    E.backgroundColor = [UIColor magentaColor];\n}\n\n```\n\n\n### MyFrameLayout\n> Is equivalent to: FrameLayout of Android.\n\nFrame layout is a layout view that the subviews can be overlapped and gravity in a special location of the superview.The subviews\' layout position&size is not depended to the adding order and establish dependency constraint with the superview. Frame layout devided the vertical orientation to top,vertical center and bottom, while horizontal orientation is devided to left,horizontal center and right. Any of the subviews is just gravity in either vertical orientation or horizontal orientation.\n\n![\xe6\xbc\x94\xe7\xa4\xba\xe6\x95\x88\xe6\x9e\x9c\xe5\x9b\xbe](https://raw.githubusercontent.com/youngsoft/MyLinearLayout/master/MyLayout/fl.png)\n\nSample code:\n\n```objective-c\n\n -(void)loadView\n{\n    [super loadView];\n    \n    MyFrameLayout *S = [MyFrameLayout new];\n    S.mySize = CGSizeMake(320,500);\n    \n    UIView *A = [UIView new];\n    A.mySize = CGSizeMake(40,40);\n    [S addSubview:A];\n    \n    UIView *B = [UIView new];\n    B.mySize = CGSizeMake(40,40);\n    B.myRight = 0;\n    [S addSubview:B];\n    \n    UIView *C = [UIView new];\n    C.mySize = CGSizeMake(40,40);\n    C.myCenterY = 0;\n    [S addSubview:C];\n    \n    UIView *D = [UIView new];\n    D.mySize = CGSizeMake(40,40);\n    D.myCenter = CGPointZero;\n    [S addSubview:D];\n    \n    //..E\xef\xbc\x8cF,G\n    \n    [self.view addSubview:S];\n    S.backgroundColor = [UIColor redColor];\n    A.backgroundColor = [UIColor greenColor];\n    B.backgroundColor = [UIColor blueColor];\n    C.backgroundColor = [UIColor orangeColor];\n    D.backgroundColor = [UIColor cyanColor];  \n  }\n  \n```\n\n\n### MyTableLayout\n> Is equivalent to: TableLayout of Android and table of HTML.\n\nTable layout is a layout view that the subviews are multi-row&col arranged like a table. First you must create a rowview and add it to the table layout, then add the subview to the rowview. If the rowviews arranged in top-to-bottom order,the tableview is caled vertical table layout,in which the subviews are arranged from left to right; If the rowviews arranged in in left-to-right order,the tableview is caled horizontal table layout,in which the subviews are arranged from top to bottom.\n\n![\xe6\xbc\x94\xe7\xa4\xba\xe6\x95\x88\xe6\x9e\x9c\xe5\x9b\xbe](https://raw.githubusercontent.com/youngsoft/MyLinearLayout/master/MyLayout/tl.png)\n\nSample code:\n\n```objective-c\n\n -(void)loadView\n{\n    [super loadView];\n    \n    MyTableLayout *S = [MyTableLayout tableLayoutWithOrientation:MyOrientation_Vert];\n    S.wrapContentWidth = YES;\n    S.subviewHSpace = 10;\n    S.subviewVSpace = 10;\n    \n \xc2\xa0 \xc2\xa0[S addRow:MyLayoutSize.wrap colSize:MyLayoutSize.wrap];\n    \n    UIView *A = [UIView new];\n    A.mySize = CGSizeMake(50,40);\n    [S addSubview:A];\n    \n    UIView *B = [UIView new];\n    B.mySize = CGSizeMake(100,40);\n    [S addSubview:B];\n    \n    UIView *C = [UIView new];\n    C.mySize = CGSizeMake(30,40);\n    [S addSubview:C];\n    \n    [S addRow:MyLayoutSize.wrap colSize:MyLayoutSize.wrap];\n    \n    UIView *D = [UIView new];\n    D.mySize = CGSizeMake(200,40);\n    [S addSubview:D];\n    \n    //...E,F  \n    \n    \n    [self.view addSubview:S];\n    S.backgroundColor = [UIColor redColor];\n    A.backgroundColor = [UIColor greenColor];\n    B.backgroundColor = [UIColor blueColor];\n    C.backgroundColor = [UIColor orangeColor];\n    D.backgroundColor = [UIColor cyanColor];\n}  \n  \n```\n\n\n### MyFlowLayout\n> Is equivalent to: flexbox of CSS3.\n\nFlow layout is a layout view presents in multi-line that the subviews are arranged in sequence according to the added order, and when meeting with a arranging constraint it will start a new line and rearrange. The constrains mentioned here includes count constraints and size constraints. The orientation of the new line would be vertical and horizontal, so the flow layout is divided into: count constraints vertical flow layout, size constraints vertical flow layout, count constraints horizontal flow layout,  size constraints horizontal flow layout. Flow layout often used in the scenes that the subviews is  arranged regularly, it can be substitutive of UICollectionView to some extent. the MyFlowLayout is almost implement the flex-box function of the HTML/CSS.\n\n![\xe6\xbc\x94\xe7\xa4\xba\xe6\x95\x88\xe6\x9e\x9c\xe5\x9b\xbe](https://raw.githubusercontent.com/youngsoft/MyLinearLayout/master/MyLayout/fll.png)\n\nSample code:\n\n```objective-c\n\n\n  -(void)loadView\n{\n    [super loadView];\n    \n    MyFlowLayout *S = [MyFlowLayout flowLayoutWithOrientation:MyOrientation_Vert arrangedCount:4];\n    S.wrapContentHeight = YES;\n    S.myWidth = 300;\n    S.padding = UIEdgeInsetsMake(10, 10, 10, 10);\n    S.gravity = MyGravity_Horz_Fill;\n    S.subviewSpace = 10;\n    \n    for (int i = 0; i < 10; i++)\n    {\n        UIView *A = [UIView new];\n        A.heightSize.equalTo(A.widthSize);\n        [S addSubview:A];\n        \n        A.backgroundColor = [UIColor greenColor];\n\n    }\n    \n    \n    [self.view addSubview:S];\n    S.backgroundColor = [UIColor redColor];\n}\n\n   \n   \n\n```\n\n\n\n### MyFloatLayout\n> Is equivalent to: float of CSS.\n\nFloat layout is a layout view that the subviews are floating gravity in the given orientations, when the size is not enough to be hold, it will automatically find the best location to gravity. float layout\'s conception is reference from the HTML/CSS\'s floating positioning technology, so the float layout can be designed in implementing irregular layout. According to the different orientation of the floating, float layout can be divided into left-right float layout and up-down float layout.\n\n![\xe6\xbc\x94\xe7\xa4\xba\xe6\x95\x88\xe6\x9e\x9c\xe5\x9b\xbe](https://raw.githubusercontent.com/youngsoft/MyLinearLayout/master/MyLayout/flo.png)\n\nSample code:\n\n```objective-c\n\n     -(void)loadView\n{\n    [super loadView];\n    \n    MyFloatLayout *S  = [MyFloatLayout floatLayoutWithOrientation:MyOrientation_Vert];\n    S.wrapContentHeight = YES;\n    S.padding = UIEdgeInsetsMake(10, 10, 10, 10);\n    S.subviewSpace = 10;\n    S.myWidth = 300;\n    \n    UIView *A = [UIView new];\n    A.mySize = CGSizeMake(80,70);\n    [S addSubview:A];\n    \n    UIView *B = [UIView new];\n    B.mySize = CGSizeMake(150,40);\n    [S addSubview:B];\n    \n    UIView *C = [UIView new];\n    C.mySize = CGSizeMake(70,40);\n    [S addSubview:C];\n    \n    UIView *D = [UIView new];\n    D.mySize = CGSizeMake(100,140);\n    [S addSubview:D];\n    \n    UIView *E = [UIView new];\n    E.mySize = CGSizeMake(150,40);\n    E.reverseFloat = YES;\n    [S addSubview:E];\n    \n    UIView *F = [UIView new];\n    F.mySize = CGSizeMake(120,60);\n    [S addSubview:F];\n    \n    \n    [self.view addSubview:S];\n    S.backgroundColor = [UIColor redColor];\n    A.backgroundColor = [UIColor greenColor];\n    B.backgroundColor = [UIColor blueColor];\n    C.backgroundColor = [UIColor orangeColor];\n    D.backgroundColor = [UIColor cyanColor];\n    E.backgroundColor = [UIColor blackColor];\n    F.backgroundColor = [UIColor whiteColor];\n}     \n\n```\n\n\n### MyPathLayout\n> Is unique characteristic layout view of iOS.\n\n Path layout is a layout view that the subviews are according to a specified path curve to layout. You must provide a type of Functional equation\xef\xbc\x8ca coordinate and a type of distance setting to create a Path Curve than all subview are equidistance layout in the Path layout. path layout usually used to create some irregular and gorgeous UI layout.\n\n![\xe6\xbc\x94\xe7\xa4\xba\xe6\x95\x88\xe6\x9e\x9c\xe5\x9b\xbe](https://raw.githubusercontent.com/youngsoft/MyLinearLayout/master/MyLayout/pl.png)\n\nSample code:\n \n ```objective-c\n \n    -(void)loadView\n{\n    [super loadView];\n    \n    MyPathLayout *S = [MyPathLayout new];\n    S.mySize = CGSizeMake(320,320);\n    S.coordinateSetting.isReverse = YES;\n    S.coordinateSetting.origin = CGPointMake(0.5, 0.2);\n    \n    S.polarEquation = ^(CGFloat angle)\n    {\n        return 80 * (1 + cos(angle));\n    };\n    \n    for (int i = 0; i < 4; i++)\n    {\n        UIView *A = [UIView new];\n        A.mySize = CGSizeMake(40,40);\n        [S addSubview:A];\n        \n        A.backgroundColor = [UIColor greenColor];\n    }\n\n    [self.view  addSubview:S];\n    S.backgroundColor = [UIColor redColor];\n }\n\n \n ```\n  \n  \n### MyGridLayout\n> Is unique characteristic layout view of iOS.\n\nGrid layout is a view to a rectangular area according to the row or column divided into multiple sub areas, sub area according to the requirements of layout can continue recursive partitioning, grid layout inside the child view will be in accordance with the sequence to add to the corresponding leaf area filling in the layout of the mechanism. Grid layout through a system of the layout of the custom to the position and size, added to the inside of the grid layout child view will no longer need to specify the location and size but by grid layout of the grid to complete, so can be very convenient to adjust the layout structure, so as to realize the ability of dynamic layout.\n\n![\xe6\xbc\x94\xe7\xa4\xba\xe6\x95\x88\xe6\x9e\x9c\xe5\x9b\xbe](https://raw.githubusercontent.com/youngsoft/MyLinearLayout/master/MyLayout/gl.png)\n   \n   \n   Sample code:\n \n ```objective-c\n   -(void)loadView\n{\n    [super loadView];\n    \n    MyGridLayout *S = [MyGridLayout new];\n    S.mySize = CGSizeMake(320,320);\n    S.backgroundColor = [UIColor redColor];\n    [self.view addSubview:S];\n\n    //add grids\n    [S addRow:50];\n    id<MyGrid> g2 = [S addRow:MyLayoutSize.fill];\n    [g2 addCol:0.2];\n    [g2 addCol:0.2];\n    id<MyGrid> g23 = [g2 addCol:0.6];\n    [g23 addRow:0.5];\n    [g23 addRow:0.5];\n    \n    \n    //add subviews\n    UIView *A = [UIView new];\n    A.backgroundColor = [UIColor greenColor];\n    [S addSubview:A];\n    \n    UIView *B = [UIView new];\n    B.backgroundColor = [UIColor blueColor];\n    [S addSubview:B];\n\n    UIView *C = [UIView new];\n    C.backgroundColor = [UIColor orangeColor];\n    [S addSubview:C];\n\n    UIView *D = [UIView new];\n    D.backgroundColor = [UIColor cyanColor];\n    [S addSubview:D];\n\n    UIView *E = [UIView new];\n    E.backgroundColor = [UIColor blackColor];\n    [S addSubview:E];    \n }\n \n ```\n\n \n \n###  MySizeClass\n> Is equivalent to: Size Classes of iOS.\n\nMyLayout provided support to SizeClass in order to fit the different screen sizes of devices. You can combinate the SizeClass with any of the 6 kinds of layout views mentioned above to perfect fit the UI of all equipments. there are two UIView extension method: rootLayout.padding = UIEdgeInsetsMake(10, 10, 10, 10);\n    rootLayout.wrapContentHeight = NO;\n    rootLayout.gravity = MyGravity_Horz_Fill;\n\n//MySizeClass_wAny | MySizeClass_hCompact is iPhone
