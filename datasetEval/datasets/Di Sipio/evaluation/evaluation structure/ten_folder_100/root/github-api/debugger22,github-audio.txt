b'GitHub Audio\n============\n\nListen to music generated by events happening across GitHub\n\n_Imitation is the sincerest form of flattery._ - __Oscar Wilde__\n\nBut seriously, [listen-to-wikipedia](https://github.com/hatnote/listen-to-wikipedia) by [@slaporte](https://github.com/slaporte) and [@mahmoud](https://github.com/mahmoud) is the hotness.\n\n<sub><sup><sub><sup>Use [`ULTIMATE_DREAM_KILLER`](https://github.com/debugger22/github-audio/commit/ed47067f5e56ab70d65fa31f72bf2dbc513f8f56) to filter all events except closed PRs.\n\n\nMedia Attention\n-----\n* [The Next Web](http://thenextweb.com/apps/2016/10/03/this-site-tracks-events-across-github-to-generate-calming-work-music/)\n* [Product Hunt](https://www.producthunt.com/tech/github-audio)\n\n\n# Installing dependencies\n\nThis application requires:\n* node\n* npm\n* redis\n\nDependency Installation on OSX\n-------------------\n\n```bash\n$ brew install nodejs\n$ brew install npm\n$ brew install redis\n```\n\nDependency Installation on Linux\n---------------------\n\n```bash\n$ sudo apt-get update\n$ sudo apt-get install nodejs\n$ sudo apt-get install npm\n$ sudo apt-get install redis-server\n```\n\n# Running the Application\nInstall node packages\n---------------------\n\nNavigate to the project directory and run\n\n```bash\n$ npm install\n```\n\nEnvironment variables\n---------------------\n\n```\n$ export GITHUB_OAUTH_KEY=<your_github_oauth_key>\n```\n\nNote: Without the GitHub oauth key the number of requests is throttled at 60 per hour. It can be increased to 5000 per hour by using an oauth key.\n\nRun Redis and Server\n----------\n\nIf you are running locally then run local redis server\n\n```bash\n$ redis-server\n```\n\nOn heroku, you can set up `Heroku Redis` add-on and it sets `REDIS_URL` enviornment variable.\n\nIn a separate window:\n```bash\n$ node server\n```\n\nNote: For production run `export NODE_ENV="production"` before starting the server.\n'
