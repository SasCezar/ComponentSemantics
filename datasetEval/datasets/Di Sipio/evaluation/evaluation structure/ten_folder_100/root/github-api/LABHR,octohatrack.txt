b'# \xf0\x9f\x90\x99\xf0\x9f\x91\x92 - octohatrack\n\n[![Travis](https://img.shields.io/travis/LABHR/octohatrack.svg)](https://travis-ci.org/LABHR/octohatrack)\n![PyPI](https://img.shields.io/pypi/v/octohatrack.svg)\n![PyPI](https://img.shields.io/pypi/pyversions/octohatrack.svg)\n![PyPI](https://img.shields.io/pypi/l/octohatrack.svg)\n![PyPI](https://img.shields.io/pypi/implementation/octohatrack.svg)\n[![Say Thanks](https://img.shields.io/badge/Say%20Thanks-!-1EAEDB.svg)](https://saythanks.io/to/glasnt)\n\nIt\'s easy to see some [code contributions](https://help.github.com/articles/why-are-my-contributions-not-showing-up-on-my-profile/)\non a GitHub repo, but what about everything else?\n\n**Octohatrack** takes a GitHub repo name, and returns two lists: \n\n  - A list of all users as defined by GitHub as a contributor\n  - A list of all contributors to a project \n\n**What is a \'direct contributor\'?**\n\nOn any GitHub repo page, the header at the top of the file listings shows a number of commits, branches, releases and contributors. If you [click the \'contributors\' link](https://github.com/LABHR/octohatrack/graphs/contributors), you get a list of users that contributed code to the master branch of the repo, ordered by the commits and lines of code contributed. This list is limited to the top 100 users.\n\nGitHub also acknowledges \'Community Contributors\', those that have contributed code to the master branch of repos that are dependencies of the current repo. The total count of these contributors is visible by hovering over the \'contributors\' link on the main repo. \n\n*Update 2019-06-13 - GitHub now uses the term Direct and Communtiy contributor*\n\n**So, what are \'all contributors\', then?**\n\nThat\'s everyone who\'s worked on a GitHub project. It compiles a complete list of the GitHub-defined contributors (not just the top 100), plus everyone who\'s created an issue, opened a pull requests, commented on an issue, replied to a pull request, made any in-line comments on code, edited the repo wiki, or in any other way interacted with the repo. \n\nIt also adds anyone manually added to the `CONTRIBUTORS` file on a repo (if it exists). See the bottom of [CONTRIBUTORS](https://github.com/LABHR/octohatrack/blob/master/CONTRIBUTORS) for details on the formatting of this file. \n\n**Limitations**\n\n* [GitHub Reactions](https://github.com/blog/2119-add-reactions-to-pull-requests-issues-and-comments) are not counted. [Issue #87](https://github.com/LABHR/octohatrack/issues/87)\n* Does not iterate over dependencies (although `octohatrack` could be run over these independently.)\n \n\n**#LABHR**\n\n"Let\'s All Build a Hat Rack" ([\\#LABHR](https://twitter.com/search?q=%23LABHR&src=typd)) is an\noriginal concept by [Leslie Hawthorn](http://hawthornlandings.org/2015/02/13/a-place-to-hang-your-hat/)\n\nRead more about octohatrack:\n\n-   [A tool for tracking GitHub contributions](https://opensource.com/life/15/10/octohatrack-github-non-code-contribution-tracker) on OpenSource.com\n-   [Acknowledging Non-Coding Contributions](https://modelviewculture.com/pieces/acknowledging-non-coding-contributions) on ModelViewCulture.com\n-   [Build a Better Hat Rack: All Contributions Welcome](https://www.youtube.com/watch?v=wQxFKxbWcFM) from KiwiPyCon (YouTube video)\n-   [Read about the project name change](http://glasnt.com/blog/2015/11/21/goodbye-octohat.html)\n\n## Installation\n\n```\npip install octohatrack\n```\n\n`octohatrack` requires Python 3. More specifically, there are some system utils that are Python 3 only, and Unicode support in Python 3 is **so** much easier than in Python 2.\n\nIf you are having issues installing and are getting a `octohatrack requires a Python 3 environment` error, check: \n - `python --version`\n - `pip --version`\n\nIf you are running in an environment with both Python 2 and Python 3, you may need to use `pip3` to install. \n\nThere are two checks in `setup.py` and `__main__.py` that will end the installation or execution, respectively, running if it doesn\'t detect a Python 3 environment. \n\n\nIf you *are* running in a Python 3 environment and it kicks you out, please [log an issue](https://github.com/LABHR/octohatrack/issues/new), including your `python --version`, and if you\'re running in a virtualenv. \n\n### Cache\n\nIn order to not make duplicate API calls, octohatrack uses caching via [`requests-cache`](https://github.com/reclosedev/requests-cache).  \n\nPrevious versions used a local `cache_file.json`. \n\nAny time an external API call is made, it gets saved to a local\ncache file so that any subsequent calls don\'t have to burn an API call.\n\nYou can disable the cache by using the `--no-cache` flag. \n\nTo reset the cache, remove the `cache_file.json` or `cache.sqlite` file.\n\nIf you experience ongoing issues with the caching,\nplease [log a detailed issue describing what you\'re seeing](https://github.com/LABHR/octohatrack/issues/new)\n\n### Rate limiting\n\nEven if you define a `GITHUB_TOKEN`, you may be rate limited for a popular repository. Using `--wait-for-reset` will have Octohatrack sleep until GitHub says your token is usable again.\n\n### Wiki\n\nBecause GitHub doesn\'t have an API endpoint for being able to parse gollum-based repo-wikis, I\'ve had to default to cloning repos locally and parsing via gitpython. \n\nIf there are issues cloning the wiki, or other issues, it shouldn\'t break an octohatrack run, but if you do encounter issues, please [log an issue](https://github.com/LABHR/octohatrack/issues/new), and be sure to include platform information (this functionality has been tested on Mac OSX Yosemite and Ubuntu Xenial)\n\n\n## To do\n\n-   include merge-only contributors\n\n## Code of Conduct\n\nOctohatrack operates under a [Code of\nConduct](https://github.com/labhr/octohatrack/blob/master/code-of-conduct.md).\n\n## License\n\nOctohatrack is distributed under the [MIT license](https://github.com/labhr/octohatrack/blob/master/LICENSE).\n\nThis project is not affiliated with GitHub.\n'
