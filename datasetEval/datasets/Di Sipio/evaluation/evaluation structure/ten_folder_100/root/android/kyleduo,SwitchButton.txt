Diagram: How SwitchButton Measure\n\nThis diagram shows how SwitchButton measure itself and what does those nouns mean. To measure width is much complex than the height, so if you know how to measure width, you know how to measure height. And text measurement and location increase the complexity.\n\n![demo_preview](./images/how_switchbutton_measure.png)\n\n***\n\n\nDemo\n---\nI create a new demo apk to show you how to style the cute widget and use it. There\'s some screenshots of the new demo.\n\n![demo_preview](./images/demo_preview.png)\n\n***\n\n\nUsage\n---\n\nThe usage of SwitchButton is just like CheckBox. The basic control APIs of SwitchButton.\n\n* __setChecked(boolean)__\n* __toggle()__\n\nSince SwitchButton has addition animation when checked status changed, there are two addition methods for disable animation for single operation.\n\n* __setCheckedImmediately(boolean)__: like setChecked but NO animation.\n* __toggleImmediately()__: like toggle but NO animation.\n\nFrom **version 1.4.1** on, SwitchButton support operation without onCheckedChanged callback. It makes changing state in code more convenient. Using these methods to achieve that feature.\n\n\n* __setCheckedNoEvent(boolean)__\n* __setCheckedImmediatelyNoEvent(boolean)__\n* __toggleNoEvent()__\n* __toggleImmediatelyNoEvent()__\n\n***\n\n\nStyle\n---\n\nIn 1.3.0, I updated the usage of __SwitchButton__ library. To make it more Android way to use, I\'ve combined the thumb and back style each to StateListColor/StateListDrawable. So you are free to create styles in different states.\n\nIn __xml__ layout file, you can configure the face of switch button using these attrs.\n\n*   __kswThumbDrawable__: drawable for thumb\n*   __kswThumbColor__: color for thumb\n*   __kswThumbMargin__: margin from thumb to back, can be negative. maybe cover by single direction margins\n*   __kswThumbMarginTop__: same to __kswThumbMargin__, just top\n*   __kswThumbMarginBottom__: same to __kswThumbMargin__, just bottom\n*   __kswThumbMarginLeft__: same to __kswThumbMargin__, just left\n*   __kswThumbMarginRight__: same to __kswThumbMargin__, just right\n*   __kswThumbWidth__: width of thumb\n*   __kswThumbHeight__: height of thumb\n*   __kswThumbRadius__: radius of thumb rect, only work with __kswThumbColor__\n*   __kswBackRadius__: radius of background rect, only work with __kswBackColor__\n*   __kswBackDrawable__: drawable for background\n*   __kswBackColor__: color for background\n*   __kswFadeBack__: fade background color/drawable when drag or animate between on/off status or not\n*   __kswAnimationDuration__: duration of animation between 2 status\n*   __kswTintColor__: change SwitchButton\'s style just by __one__ property, all relevant color will be generate automatically. Do not support `SwitchButtonMD` or other style created by xml resources.\n*   __kswTextOn__: text for checked status.\n*   __kswTextOff__: text for unchecked status.\n*   __kswTextThumbInset (since 2.0.0)__: length of the part of text under the thumb.\n*   __kswTextExtra (since 2.0.0)__: extra space needed by background besides the actual text width.\n*   __kswTextAdjust (since 2.0.0)__: move the text after position text on the center of text area.\n*   **kswThumbRangeRatio (since 2.0.0)**: (thumb move range width / thumb\'s width). float value. *see measure diagram*\n*   ~~__kswBackMeasureRatio__: (background\'s width / thumb\'s width). float value.~~ *Removed since 2.0.0*\n*   ~~**kswTextMarginH**: horizontal margin of text.~~ *Removed since 2.0.0*\n*   ~~__kswAutoAdjustTextPosition__: **(since 1.4.4)** whether auto adjust text position to make them looks centered (NOT really centered) when there are round corners. You should set this to false when you don\'t need this feature.~~ *Removed since 2.0.0*\n\nYou can alse change the configuration of SwitchButton ___in code___. You can find the api from Demo apk.
